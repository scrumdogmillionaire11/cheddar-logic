<objective>
Fix bug where PASS projections ("No edge at current price") are surfaced to the UI with edge data visible, breaking the play/pass contract and creating user confusion.

**Current state:** WI-0967 filtered PASS rows from directional tallying (over/under records) but did NOT address rendering/display layer. PASS projections still appear in UI with finalEdge, drivers, confidence visible.

**Goal:** Exclude PASS projections from all UI surfaces except where explicitly aggregating/auditing them. PLAY/LEAN/FIRE projections continue displaying normally.

**Output:** Projections marked PASS are either:
- Not rendered in projection cards, OR
- Rendered with minimal info (no edge breakdowns)
</objective>
