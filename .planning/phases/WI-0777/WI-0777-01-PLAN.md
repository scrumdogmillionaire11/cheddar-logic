<objective>
Wire npm workspaces at the repo root so that `packages/data`, `packages/adapters`,
`packages/models`, `packages/odds`, `apps/worker`, and `web` are first-class workspace
members under a single `npm install`.

Purpose: Collapses duplicate node_modules hoisting, makes @cheddar-logic/* inter-package
resolution declarative, and removes the orphaned apps/worker/packages/data/ stub.

Output:
- Root package.json gains `workspaces` array
- Orphaned apps/worker/packages/data/ deleted from repo
- Root lockfile regenerated to cover all workspace members
- Worker and web tests remain green
</objective>

<task type="auto">
  <name>Task 1: Declare workspaces in root package.json and delete orphaned stub</name>
  <files>package.json, apps/worker/packages/data/ (delete)</files>
  <action>
1. Edit `package.json` — add `"workspaces"` array immediately after the `"private": true` line
   (or after `"name"` if private is absent). The value must be:

   ```json
   "workspaces": ["packages/*", "apps/*", "web"]
   ```

   Do NOT touch `"scripts"` or `"devDependencies"`.

2. Delete the orphaned stub directory `apps/worker/packages/data/`:
   ```bash
   rm -rf apps/worker/packages/data
   git rm -rf apps/worker/packages/data 2>/dev/null || true
   ```
   This directory contains only an empty `cheddar.db` (0 bytes) and an empty
   `package-lock.json`. It has no `src/` and is not the production DB
   (production DB lives at /opt/data/cheddar-prod.db).

3. Do NOT change `file:` dependency references in `apps/worker/package.json` or
   `web/package.json`. npm workspaces resolves workspace members referenced via
   `file:` paths correctly — it creates symlinks just as with workspace: protocol.
   Changing them to `*` risks falling back to the npm registry if resolution fails.
  </action>
  <verify>
node -e "const p=require('./package.json'); console.log(p.workspaces)" 
# Expected: [ 'packages/*', 'apps/*', 'web' ]

ls apps/worker/packages/ 2>/dev/null || echo "apps/worker/packages gone (OK)"
  </verify>
  <done>
- package.json contains workspaces array with three entries
- apps/worker/packages/data/ no longer exists on disk or in git index
  </done>
</task>

<task type="auto">
  <name>Task 2: Run npm install at root and verify workspace resolution</name>
  <files>package-lock.json</files>
  <action>
1. Run workspace install from repo root:
   ```bash
   npm install
   ```
   This regenerates `package-lock.json` to cover all workspace members.
   The per-app lockfiles (`apps/worker/package-lock.json`, `web/package-lock.json`)
   become redundant — leave them in place for now (removing them is out of scope).

2. Verify all six members are listed:
   ```bash
   npm ls --workspaces --depth=0 2>&1 | head -40
   ```

3. Verify @cheddar-logic/data resolves as a symlink from worker:
   ```bash
   node -e "require('@cheddar-logic/data'); console.log('OK')" --prefix apps/worker
   # OR
   cd apps/worker && node -e "require('@cheddar-logic/data'); console.log('OK')"
   ```

4. Confirm the symlink target:
   ```bash
   ls -la node_modules/@cheddar-logic/data
   # Should be -> ../../packages/data
   ```

5. Run the worker test suite:
   ```bash
   npm --prefix apps/worker run test --no-coverage 2>&1 | tail -15
   ```
   All tests must pass (pre-existing failures are acceptable if they were failing
   before this change — check git diff for whether new failures appeared).
  </action>
  <verify>
npm ls --workspaces --depth=0 2>&1 | grep -E "cheddar-logic-root|worker|web|@cheddar-logic"
# Must show 6 workspace members without "UNMET" or "extraneous" errors

ls -la node_modules/@cheddar-logic/data | grep "->"
# Must be a symlink

npm --prefix apps/worker run test --no-coverage 2>&1 | tail -5
  </verify>
  <done>
- `npm install` exits 0
- `npm ls --workspaces` shows packages/adapters, packages/data, packages/models,
  packages/odds, apps/worker, web with no UNMET warnings
- node_modules/@cheddar-logic/data is a symlink to ../../packages/data
- Worker test suite passes (same count as pre-change baseline)
  </done>
</task>
