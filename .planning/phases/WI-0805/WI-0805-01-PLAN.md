In `edge-calculator.js`, add and export a new pure helper function immediately
after the existing `removeVig` helper:

```js
/**
 * Return the no-vig fair implied probability for side A in a two-sided market.
 *
 * Formula: rawA / (rawA + rawB)
 *   where rawA = impliedProbFromAmerican(priceA)
 *         rawB = impliedProbFromAmerican(priceB)
 *
 * Returns null if either price is invalid.
 *
 * Example: twoSidedFairProb(-110, -110) → 0.5000 (not 0.5238)
 * Example: twoSidedFairProb(-115, -105) → 0.5227 (not 0.5349)
 */
function twoSidedFairProb(priceA, priceB) {
  const rawA = impliedProbFromAmerican(priceA);
  const rawB = impliedProbFromAmerican(priceB);
  if (rawA == null || rawB == null) return null;
  const total = rawA + rawB;
  if (total <= 0) return null;
  return rawA / total;
}
```

Add it to `module.exports`.

## run_nhl_player_shots_model.js (inline BLK multi-line loop)

The multi-line BLK loop (~L3335+) computes edge inline:
```js
const implOvr = americanToImplied(extraCandidate.over_price);
extraEdgeOverPp = extraFairOverProb - implOvr;
```
Replace with:
```js
const deviggiedOvr = (extraCandidate.over_price != null && extraCandidate.under_price != null)
  ? (americanToImplied(extraCandidate.over_price) / (americanToImplied(extraCandidate.over_price) + americanToImplied(extraCandidate.under_price)))
  : americanToImplied(extraCandidate.over_price);
extraEdgeOverPp = extraFairOverProb - deviggiedOvr;
```

Same pattern for the under side.

payout calculations (which should remain raw) and for the `impliedOverProb`
field stored in the card payload for display purposes. Only the edge delta
values (`edge_over_pp`, `edge_under_pp`) need devigging.
