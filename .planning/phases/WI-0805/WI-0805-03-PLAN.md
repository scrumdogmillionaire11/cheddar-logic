  <action>
Find the F5 ML edge computation block (~line 664). Currently:

```js
// American odds → implied probability (includes vig)
function mlToImplied(ml) { ... }
const impliedHome = mlToImplied(mlF5Home);
const impliedAway = mlToImplied(mlF5Away);
const homeEdge = winProbHome - impliedHome;
const awayEdge = (1 - winProbHome) - impliedAway;
```

Replace with:

```js
// Remove bookmaker vig via normalization: fair = raw / (rawHome + rawAway)
function mlToImplied(ml) { ... }  // keep the helper, used for null checks
const rawHome = mlToImplied(mlF5Home);
const rawAway = mlToImplied(mlF5Away);
if (rawHome === null || rawAway === null) return null;
const overround = rawHome + rawAway;
const fairImpliedHome = rawHome / overround;
const fairImpliedAway = rawAway / overround;
const homeEdge = winProbHome - fairImpliedHome;
const awayEdge = (1 - winProbHome) - fairImpliedAway;
```

Update the reasoning string to use `fairImpliedHome` / `fairImpliedAway` instead
of `impliedHome` / `impliedAway` (they're the same variables, just renamed).

Update the comment from "includes vig" to "no-vig via normalization".

Do NOT change MLB_F5_EDGE_THRESHOLD (0.5 raw points). That threshold is for
the total line edge (runs above/below line), not the probability edge.
