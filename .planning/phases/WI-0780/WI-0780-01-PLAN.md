--- CREATE apps/worker/src/schedulers/nfl.js ---

The NFL scheduler is minimal: only fixed-time (09:00, 12:00 ET) and T-minus model runs.
There are no sport-specific ingestion or availability sync jobs for NFL.

Interface follows the same pattern as player-props.js — receives a context object:
  computeNflDueJobs(nowEt, { nowUtc, games, dryRun, quotaTier, maybeQueueTeamMetricsRefresh })

Implementation:
  - Import { isFixedDue, keyFixed, keyTminus, dueTminusMinutes } from './windows'
  - Import { runNFLModel } from '../jobs/run_nfl_model'
  - Check process.env.ENABLE_NFL_MODEL !== 'false' at top of function; return [] if disabled
  - Fixed-time loop over ['09:00', '12:00']: call maybeQueueTeamMetricsRefresh, emit keyFixed job
  - Per-game T-minus loop: use games filtered to sport==='nfl', call dueTminusMinutes,
    emit T-minus pre-model odds pull if quotaTier === 'FULL' (pull_odds_hourly, same
    pattern as the existing T-minus block in main.js section 3), then emit run_nfl_model job
  - DO NOT import pullOddsHourly inside nfl.js — receive it as a context param:
      computeNflDueJobs(nowEt, { nowUtc, games, dryRun, quotaTier, maybeQueueTeamMetricsRefresh,
                                  claimTminusPullSlot, pullOddsHourly, ENABLE_WITHOUT_ODDS_MODE })
    This avoids circular deps and makes the dependency explicit.
  - module.exports = { computeNflDueJobs }
