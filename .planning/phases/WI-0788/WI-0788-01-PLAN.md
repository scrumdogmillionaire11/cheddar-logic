  <action>
Add a new exported function `recomputeTrackingStats(opts = {})` that converts
projection_audit rows into tracking_stats rows via GROUP BY aggregation.

Signature (from WI-0788):
```js
/**
 * Recompute tracking_stats aggregates from projection_audit rows.
 * Replaces the live-increment pattern with a recompute-from-source pattern.
 *
 * @param {object} [opts]
 * @param {string} [opts.sport]          - Limit to one sport (optional)
 * @param {string} [opts.sinceSettledAt] - ISO8601 lower bound (optional, default: all time)
 * @param {boolean} [opts.fullReplace]   - If true, DELETE existing rows for scope before upsert
 */
function recomputeTrackingStats(opts = {}) { ... }
```

Implementation:

1. Build WHERE clause from opts (sport filter, sinceSettledAt filter). Always
   include `AND (period IS NULL OR period != '1P')` to exclude 1st-period totals.

2. Run TWO GROUP BY queries on projection_audit:

   **Query A — market aggregate** (stat_key pattern: `${sport}|${marketType}|all|all|all|alltime`):
   ```sql
   SELECT
     sport, market_type,
     COUNT(CASE WHEN result = 'win'  THEN 1 END) AS wins,
     COUNT(CASE WHEN result = 'loss' THEN 1 END) AS losses,
     COUNT(CASE WHEN result = 'push' THEN 1 END) AS pushes,
     SUM(pnl_units) AS total_pnl
   FROM projection_audit
   WHERE (period IS NULL OR period != '1P')
     [AND sport = ?]
     [AND settled_at >= ?]
   GROUP BY sport, market_type
   ```

   **Query B — verification-segmented** (stat_key pattern: `${sport}|${marketType}|all|all|edge_verification:${sharpStatus}|alltime`):
   ```sql
   SELECT
     sport, market_type,
     COALESCE(sharp_price_status, 'UNTAGGED') AS sharp_status,
     COUNT(CASE WHEN result = 'win'  THEN 1 END) AS wins,
     COUNT(CASE WHEN result = 'loss' THEN 1 END) AS losses,
     COUNT(CASE WHEN result = 'push' THEN 1 END) AS pushes,
     SUM(pnl_units) AS total_pnl
   FROM projection_audit
   WHERE (period IS NULL OR period != '1P')
     [AND sport = ?]
     [AND settled_at >= ?]
   GROUP BY sport, market_type, COALESCE(sharp_price_status, 'UNTAGGED')
   ```

3. If `opts.fullReplace` is true, DELETE matching rows from tracking_stats
   before upserting (scoped to sport if provided, else all rows whose stat_key
   matches the 'all|alltime' or 'edge_verification' patterns).

4. For each row from Query A, call the existing `upsertTrackingStat({...})` with:
   - `id`: `stat-${sport}-${marketType}-alltime`
   - `statKey`: `${sport}|${marketType}|all|all|all|alltime`
   - `sport`, `marketType`, `direction: 'all'`, `confidenceTier: 'all'`
   - `driverKey: 'all'`, `timePeriod: 'alltime'`
   - `totalCards`, `settledCards`, `wins`, `losses`, `pushes`, `totalPnlUnits`
   - `winRate`, `avgPnlPerCard` computed from counts
   - `metadata: { recomputedAt: new Date().toISOString() }`

5. For each row from Query B, call `upsertTrackingStat({...})` with:
   - `id`: `stat-${sport}-${marketType}-${sharpStatus}-alltime`
   - `statKey`: `${sport}|${marketType}|all|all|edge_verification:${sharpStatus}|alltime`
   - same numeric fields
   - `driverKey`: `edge_verification:${sharpStatus}`

6. Wrap both queries + upserts in a single `db.transaction(...)` call so the
   update is atomic.

7. Return `{ rows: N }` where N is total upserts performed.

Add `recomputeTrackingStats` to the module.exports object at the bottom of
tracking.js (alongside `upsertTrackingStat`, `incrementTrackingStat`).

Do NOT remove or modify `incrementTrackingStat` — it will continue to exist
until it is removed in Plan 02's job wiring step.
