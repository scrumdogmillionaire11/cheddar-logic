Add `impactPlayerOutList` computation before the return:

```js
// Derive impact player out list from ESPN injury designations
const impactPlayerOutList = [];
const injuries = teamInfo?.injuries ?? [];
const recentGameScorers = (games ?? [])
  .flatMap(g => g.players ?? [])
  .reduce((acc, p) => {
    if (!acc[p.name]) acc[p.name] = [];
    acc[p.name].push(p.points ?? 0);
    return acc;
  }, {});

// Top-3 scorers by avg points in recent games
const topScorers = Object.entries(recentGameScorers)
  .map(([name, pts]) => ({ name, avg: pts.reduce((a, b) => a + b, 0) / pts.length }))
  .sort((a, b) => b.avg - a.avg)
  .slice(0, 3)
  .map(p => p.name);

for (const player of injuries) {
  const designation = (player.status ?? '').toUpperCase();
  if (!['OUT', 'DOUBTFUL', 'DTD'].includes(designation)) continue;
  const isStarter = (player.position ?? '').toLowerCase() === 'starter'
    || player.starter === true;
  const isTopScorer = topScorers.includes(player.displayName ?? player.name);
  if (isStarter || isTopScorer) {
    impactPlayerOutList.push(player.displayName ?? player.name);
  }
}
```

Adapt field names (e.g. `player.displayName`, `player.status`, `player.starter`)
to whatever ESPN actually returns — read the existing ESPN parsing code carefully
before writing. If injury data structure differs, use the closest equivalent.

If ESPN injury data is unavailable (null/empty), `impactPlayerOutList` defaults
to `[]` — never throw.

Add `impactPlayerOutList` to the returned object. Maintain all existing returned fields.
