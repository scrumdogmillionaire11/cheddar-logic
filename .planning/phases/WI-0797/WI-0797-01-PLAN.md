```

Add a helper `shouldSendAlert(phase, checkName, consecutiveRequired, cooldownMinutes)`:
- Queries `pipeline_health` for the last `consecutiveRequired` rows for the given `phase + check_name`, ordered by `created_at DESC`.
- Returns `false` if fewer than `consecutiveRequired` rows exist OR if any of those rows is NOT `status='failed'` (streak not yet reached).
- If all N are `failed`, check whether the row at position N (i.e., the earliest in the streak) was inserted within `cooldownMinutes` ago. If it was (streak is fresh / just crossed the threshold), return `true`. If not (streak is older than cooldown window), return `false` — already alerted on a prior tick.
  - In other words: the streak of N `failed` rows must have its OLDEST row within `cooldownMinutes` to be treated as "just crossed threshold". Once the oldest row is > cooldownMinutes old, suppress.

Add a helper `buildHealthAlertMessage(failedChecks)`:
- Accepts array of `{phase, checkName, reason}` objects.
- Returns a Discord-formatted string, e.g.:

The following checks are failing:
• `schedule / freshness` — No upcoming games in next 48h (schedule may be stale)
• `cards / freshness` — 3/5 games within T-2h missing/stale cards (>30m old)
```
