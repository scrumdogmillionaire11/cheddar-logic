<objective>
Add a human-readable explanation to Play of the Day that can be displayed with the pick without inventing a separate narrative system.

Purpose: the current POTD pipeline already computes `lineValue`, `marketConsensus`, `edgePct`, `modelWinProb`, and, for MLB moneyline, `projection_source`, but none of that is surfaced as a short rationale on the POTD page. The implementation should derive one concise string from those existing facts, persist it at publish time, and expose it through the existing read-only POTD API/page contract.

Output:
- one nullable `reasoning` column on `potd_plays`
- one deterministic POTD reasoning string on the selected play and card payload
- `/api/potd` and `/play-of-the-day` rendering that reasoning
</objective>
