<objective>
Emit `[ESPN_NULL]` structured logs and a Discord alert when ESPN team metrics
return all-null for 2+ teams in a single model run. Operators currently have no
real-time signal when the ESPN data layer fails wholesale.

Purpose: Reduces time-to-detect ESPN failures from the next Dr. Claire health
report (hours) to the current live trading session (minutes).

Output: Alerting logic in run_nba_model.js + run_nhl_model.js; neutral() reason
field in team-metrics.js; one new test.
</objective>

**team-metrics.js:**
Locate the `neutral()` helper (used when ESPN metrics are all-null). Add a `reason`
field if not already present. When returning neutral(), include a reason:
`neutral('RATE_LIMITED')`, `neutral('TEAM_MAP_MISS')`, `neutral('FETCH_ERROR')`, etc.,
based on the catch or miss path that led there. Default reason: `'UNKNOWN'`.
If `neutral()` is a factory, add `reason` as an optional parameter.

**run_nba_model.js and run_nhl_model.js (same pattern for both):**

1. Before the ESPN fetch loop, declare accumulator:
   `const nullMetricTeams = [];`

2. After each team's metrics are fetched, detect neutral result:
   ```js
   if (metrics.avgPoints === null) {
     const reason = metrics.espnNullReason ?? 'UNKNOWN';
     console.warn(`[ESPN_NULL] sport=NBA team=${teamName} reason=${reason}`);
     nullMetricTeams.push({ team: teamName, reason });
   }
   ```
   Use 'NHL' for the NHL runner. `teamName` is whatever identifier is available
   (abbr or display name).

3. After the game loop (after all teams processed), add threshold check and Discord alert:
   ```js
   const ESPN_NULL_THRESHOLD = parseInt(process.env.ESPN_NULL_ALERT_THRESHOLD ?? '2', 10);
   if (nullMetricTeams.length >= ESPN_NULL_THRESHOLD) {
     // Reuse cooldown pattern from check_pipeline_health.js
     const now = Date.now();
     if (!global._espnNullLastAlerted || now - global._espnNullLastAlerted > 60 * 60 * 1000) {
       global._espnNullLastAlerted = now;
       const teamsStr = nullMetricTeams.map(t => `${t.team}(${t.reason})`).join(', ');
       // Use existing Discord webhook helper — import or require from wherever check_pipeline_health uses it
       await sendDiscordAlert(`⚠️ ESPN_NULL: ${nullMetricTeams.length} teams with null metrics: ${teamsStr}`);
     }
   }
   ```

Read `apps/worker/src/jobs/check_pipeline_health.js` to find the exact Discord
helper function name and import path — reuse it exactly. Do not create a new webhook
call pattern. If `sendDiscordAlert` is named differently, use the correct name.

Do NOT add cooldown logic to run_nba_model or run_nhl_model if they run without
async context — make the cooldown best-effort via in-memory global. The in-memory
approach matches the WI spec.
