  <action>
Create `apps/worker/src/models/input-gate.js` with exactly the API specified in WI-0820.md, plus:

1. `classifyModelStatus(featureMap, required, optional = [])`:
   - `missingCritical`: required keys where value is null, undefined, or NaN
   - `missingOptional`: optional keys where value is null, undefined, or NaN
   - status: NO_BET if missingCritical.length > 0, else DEGRADED if missingOptional.length > 0, else MODEL_OK
   - Asymmetric failure: because ALL home+away features are listed in `required`, one side missing → NO_BET automatically. No special-case needed — document this clearly in JSDoc.

2. `buildNoBetResult(missingCritical, context = {})`:
   - Returns: `{ status: 'NO_BET', reason: 'MISSING_CORE_INPUTS', missingCritical, prediction: null, confidence: 0, projection: null, ...context }`

3. Export a frozen `DEGRADED_CONSTRAINTS` object documenting the behavioral contract:
   ```js
   const DEGRADED_CONSTRAINTS = Object.freeze({
     MAX_CONFIDENCE: 0.55,
     FORBIDDEN_TIERS: ['PLAY'],  // LEAN and PASS may still emit
   });
   ```
   Export this alongside the functions. Callers use it to cap confidence and block tier escalation.

4. `module.exports = { classifyModelStatus, buildNoBetResult, DEGRADED_CONSTRAINTS }`

Do NOT add any sport-specific logic here. Pure utility only.
