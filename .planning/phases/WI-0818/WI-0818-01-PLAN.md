<objective>
Attach a `price_staleness_warning` object to hard-locked card payloads when the
locked price diverges from the current market price inside T-60 minutes. Advisory
only — no lock logic changes. Prevents users from attempting bets at unavailable prices.

Purpose: Gives Discord and UI consumers an explicit signal when a T-120 hard lock
has drifted, so the operator can warn before a user shops a stale line.

Output: Modified `decision-publisher.js` with warning attachment logic + Discord
embed update + two new tests.
</objective>

  <files>apps/worker/src/utils/decision-publisher.js</files>
  <action>
1. Update `applyPublishedDecisionToPayload` signature to accept an optional
   options object as the last parameter:
   `function applyPublishedDecisionToPayload(card, decision, market, decisionKey, gateReason, { minutesToStart = 9999, candidatePrice = null } = {})`

2. Inside the function, after the locked price is applied (`payload.price = decision.recommended_price`),
   compute priceDelta and conditionally attach warning:

   ```js
   const lockedPrice = decision.recommended_price ?? null;
   const priceDelta =
     Number.isFinite(lockedPrice) && Number.isFinite(candidatePrice)
       ? Math.abs(candidatePrice - lockedPrice)
       : null;

   if (priceDelta != null && priceDelta > 0 && minutesToStart < 60) {
     payload.price_staleness_warning = {
       locked_price: lockedPrice,
       current_candidate_price: candidatePrice,
       delta_american: Number(priceDelta.toFixed(0)),
       minutes_to_start: Math.round(minutesToStart),
       reason: 'HARD_LOCK_PRICE_DRIFT',
     };
     payload.tags = Array.from(
       new Set([...(payload.tags || []), 'PRICE_STALENESS_WARNING']),
     );
   }
   ```

3. At the call site inside `publishDecisionForCard`, pass the new options:
   `applyPublishedDecisionToPayload(card, current, market, decisionKey, gateResult.reason_code, { minutesToStart, candidatePrice: price })`

   `price` and `minutesToStart` are already computed by this point in
   `publishDecisionForCard` — do not add new lookups.

Do NOT modify the hard-lock threshold (120 min), flip logic, or any other gating.
