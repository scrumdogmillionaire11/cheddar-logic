Find the CF secrets missing branch at line ~516 (inside the `if [ -z "${CF_API_TOKEN:-}" ]...` block):

```
              echo "WARNING: Cloudflare secrets (CF_API_TOKEN / CF_ZONE_ID) not set — skipping CF cache purge. Set them as GitHub Actions secrets to enable automatic purge on every deploy."
```

REPLACE WITH (three lines, same indentation):
```
              echo "::warning::CF secrets missing — Cloudflare purge skipped. Users may see stale assets until CF TTL expires. Set CF_API_TOKEN and CF_ZONE_ID as GitHub Actions secrets to restore automatic purge."
              echo "⚠️ **CF purge skipped** — \`CF_API_TOKEN\` or \`CF_ZONE_ID\` not set. Users may see stale Cloudflare-cached HTML/assets until TTL expires." >> "${GITHUB_STEP_SUMMARY:-/dev/null}"
```

Notes:
- `::warning::` is the GitHub Actions workflow command syntax; it creates an orange annotation visible in the run summary and file annotations. No title needed (default title = "Warning").
- `>> "${GITHUB_STEP_SUMMARY:-/dev/null}"` guards against local runs where `$GITHUB_STEP_SUMMARY` is unset (e.g., manual test or act dry-runs). This avoids breaking the step if run outside GHA.
- Keep `CF_PURGE_SKIPPED=true` on the line that immediately follows — do not modify it.
- No other lines in the file change. The `else` branch (secrets present → curl purge) is untouched.
