# WI-0640: Enrich F5 projection with rest, stamina, K-trend, and weather signals

**ID**: WI-0640
**Goal**: Upgrade `projectF5Total` and `projectF5ML` to use all available pitcher and game-context signals instead of ERA/WHIP/K9 alone.

**Background**:
`projectF5Total` currently uses only 3 pitcher fields (ERA required; WHIP and K/9 with hard-coded defaults). The pitcher object from `buildPitcherKObject` already carries `recent_k_per_9`, `days_since_last_start`, `last_three_ip`, `k_pct_last_4_starts`, and `k_pct_prior_4_starts`. Game weather (`temp_f`, `wind_mph`, `wind_dir`) is already attached to the `mlb` context in `enrichPitcherStats` but never passed to the projection functions. No new data collection is required — this is purely a model-enrichment change.

**Scope**:
- `apps/worker/src/models/mlb-model.js`

**Out of scope**:
- `apps/worker/src/jobs/run_mlb_model.js` (no change needed — `computeMLBDriverCards` already gets full `oddsSnapshot` with weather)
- `apps/worker/src/jobs/pull_mlb_pitcher_stats.js`
- DB migrations (all signals already stored)
- K engine strikeout model (separate concern)
- Park factor data collection (would need new data source — future WI)

**Changes required**:

1. **`projectF5Total(homePitcher, awayPitcher, gameCtx?)`** — add optional `gameCtx = {}`:
   - **Recent K/9**: use `pitcher.recent_k_per_9 ?? pitcher.k_per_9 ?? 8.5` so recent form takes priority over season average
   - **Rest overlay**: if `days_since_last_start >= 5` → `base *= 0.96` (rested pitcher limits early runs); if `days_since_last_start <= 3` → `base *= 1.05` (short rest); only apply if both pitchers have the field
   - **Stamina signal**: compute each pitcher's `avg_last3_ip = mean(last_three_ip)`; if avg < 5.0 → `base *= 1.07` (pitcher likely on short leash, F5 run environment opens up)
   - **K% trend**: if `k_pct_last_4_starts > k_pct_prior_4_starts + 0.03` → confidence += 1 (improving); if `k_pct_last_4_starts < k_pct_prior_4_starts - 0.03` → confidence -= 1 (declining)
   - **Weather — temperature**: `gameCtx.temp_f < 45` → `base *= 0.93`; `gameCtx.temp_f > 85` → `base *= 1.04`
   - **Weather — wind**: `gameCtx.wind_mph >= 12` and wind_dir contains "Out" (or compass directions SSW/S/SW for most parks) → `base *= 1.07`; wind_mph >= 12 and "In" direction → `base *= 0.93`
   - Update confidence: add rest/stamina signals to existing era/whip gates

2. **`projectF5TotalCard(homePitcher, awayPitcher, f5Line, gameCtx?)`** — thread `gameCtx` through to `projectF5Total`

3. **`projectF5ML(homePitcher, awayPitcher, mlF5Home, mlF5Away, gameCtx?)`** — use `recent_k_per_9 ?? k_per_9` for run expectation (lines 186–188); thread `gameCtx` for any base adjustment

4. **`computeMLBDriverCards(gameId, oddsSnapshot)`** — build `gameCtx = { temp_f: mlb.temp_f ?? null, wind_mph: mlb.wind_mph ?? null, wind_dir: mlb.wind_dir ?? null }` and pass to `projectF5TotalCard` and `projectF5ML`

5. **Update reasoning string** in `projectF5TotalCard` to include active overlays (rest, stamina, weather) so card output is explainable

**Acceptance**:
- `projectF5Total` uses `recent_k_per_9` when available (falls back to `k_per_9 ?? 8.5`)
- Rest overlay fires correctly: well-rested pitcher (days >= 5) lowers projected F5 runs, short rest (days <= 3) raises them
- Stamina signal fires: avg last-3 IP < 5.0 → projected runs increase
- K% trend adjusts confidence ±1 when trend delta exceeds 3pp
- Weather adjustments fire for outdoor games when temp < 45°F or wind >= 12 mph
- Indoor/dome games (`gameCtx.temp_f == null`) are unaffected (null-safe)
- `computeMLBDriverCards` passes weather from `mlb.temp_f/wind_mph/wind_dir`
- Reasoning string in card output lists active overlays
- All existing MLB model unit tests continue to pass
