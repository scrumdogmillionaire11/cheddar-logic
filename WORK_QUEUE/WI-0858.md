# WI-0858: Fix POTD trigger window collapse and silent null-metadata skip

| Field | Value |
|---|---|
| **ID** | WI-0858 |
| **Goal** | Prevent POTD from silently never firing on days where all games tip after ~5:30 PM ET (collapsed window) and log explicitly when `computePotdScheduleMetadata` returns null. |
| **Priority** | High — production silent failure |
| **Depends on** | None |
| **Coordination flag** | solo |

---

## Context

### Problem 1 — Collapsed trigger window

`computePotdScheduleMetadata` clamps `targetPostTimeEt` to `[12:00 PM, 4:00 PM]`. When all eligible games tip at or after ~5:30 PM ET (the common case for NBA/NHL days), `unclampedTarget = gameTime - 90min ≥ 5:30 PM > windowEnd`, so `targetPostTimeEt` is set to `windowEnd = 4:00 PM`.

The trigger condition in `computeDueJobs` is:
```js
nowEt >= targetPostTimeEt && nowEt <= windowEnd
```

Both sides equal 4:00 PM, so the effective window is exactly **one scheduler tick**. If that tick is skipped (previous tick still running, PM2 restart, clock drift), POTD silently never fires for that day.

### Problem 2 — Silent null return from `computePotdScheduleMetadata`

When there are no eligible games today, `computePotdScheduleMetadata` returns `null` and the entire POTD block is silently skipped with no log output. Operators cannot distinguish "POTD ran and produced a pick" from "POTD was never eligible" without querying the database.

---

## Fix

### Fix 1 — Introduce `postDeadlineEt` and `windowCollapsed` in `computePotdScheduleMetadata`

In `computePotdScheduleMetadata` (`apps/worker/src/schedulers/main.js`), add:
- `postDeadline = todayStart.set({ hour: 16, minute: 15 })` — the latest the scheduler may still fire the primary job
- Expose it as `postDeadlineEt` in the return object
- Add `windowCollapsed: targetPostTimeEt.toMillis() === windowEnd.toMillis()` — true when all games tip after ~5:30 PM

`windowEnd` (4:00 PM) is unchanged and still used for clamping `targetPostTimeEt`. `postDeadlineEt` is a separate concept: the last tick at which `run_potd_engine` may be enqueued.

### Fix 2 — Update trigger condition to use `postDeadlineEt`

In the POTD block of `computeDueJobs` (`apps/worker/src/schedulers/main.js`):

```js
const meta = computePotdScheduleMetadata(nowEt, games);

if (!meta) {
  console.warn('[POTD] No eligible games today — skipping');
  // no job pushed
} else {
  const targetPostTimeEt = DateTime.fromISO(meta.targetPostTimeEt, { zone: TZ });
  const postDeadlineEt   = DateTime.fromISO(meta.postDeadlineEt,   { zone: TZ });

  if (meta.windowCollapsed) {
    console.warn(
      `[POTD] Window collapsed — all games tip after 5:30 PM ET. ` +
      `Effective window: [${targetPostTimeEt.toFormat('h:mm a')} – ${postDeadlineEt.toFormat('h:mm a')} ET]`
    );
  }

  const publishJobKey = `potd|${nowEt.toISODate()}`;
  if (
    nowEt >= targetPostTimeEt &&
    nowEt <  postDeadlineEt &&   // strict < gives 15 ticks instead of 1
    shouldRunJobKey(publishJobKey)
  ) {
    jobs.push({
      jobName: 'run_potd_engine',
      jobKey:  publishJobKey,
      execute: runPotdEngine,
      args:    { jobKey: publishJobKey, dryRun, schedule: meta },
      reason:  `potd engine (${meta.targetPostTimeEt})`,
    });
  }
}
```

Note: the local `schedule` variable in the existing code maps to `meta` here — rename accordingly.

1. `computePotdScheduleMetadata` return object includes `postDeadlineEt` (ISO string, 4:15 PM ET) and `windowCollapsed` (boolean)

5. When `games = []`, no POTD job is pushed and `[POTD] No eligible games today` appears in output

7. New tests added (see below) for collapsed window and null-game cases
