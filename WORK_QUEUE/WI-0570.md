# WI-0570: Market evaluator — misprice detector

**ID**: WI-0570
**Goal**: Add a misprice detection layer to `market_evaluator.js` that identifies when a market contains a likely stale, soft, or genuinely mispriced number relative to consensus. Surface flags per-snapshot so the model and UI can react to them.

**Background**:
Four types of misprice matter (in order of signal strength):

- **Type 1 — Consensus vs projection gap**: model fair value materially disagrees with consensus line. This is model-driven edge.
- **Type 2 — Best available beats consensus by threshold**: one book is ≥1.5 pts better than consensus for a given side. This is likely stale or soft.
- **Type 3 — Price misprice on same line**: most books are -110; one book has +102 on the same line. Pure execution edge, no line movement.
- **Type 4 — Wide book disagreement**: stddev across books is abnormally high. Not auto-playable — surface as a review flag.

Types 1 and 4 require projection data (not available at ingest time). Types 2 and 3 are purely market-side and can be computed during ingest.

This WI implements Types 2, 3, and 4. Type 1 is handled in WI-0571 (projection comparator).

**Scope**:
- `packages/odds/src/market_evaluator.js`
  - `detectMisprice(consensus, executionBlock)` — returns:
    ```
    {
      is_mispriced: boolean,
      misprice_type: "SOFT_LINE" | "PRICE_ONLY" | "HIGH_DISPERSION" | null,
      misprice_strength: number | null,      // pts for line, bps for price
      outlier_book: string | null,
      outlier_delta_vs_consensus: number | null,
      stale_or_soft_flag: boolean,
      review_flag: boolean                   // Type 4 — wide disagreement
    }
    ```
  - Thresholds (configurable constants at top of file):
    - Type 2 (soft line): `best_line - consensus_line > 1.5` pts for either side
    - Type 3 (price only): `best_price - median_price > 800` bps (e.g., +102 vs -110 = ~1100 bps)
    - Type 4 (high dispersion): `dispersion_stddev > 1.5`
- `packages/odds/src/normalize.js`
  - Call `detectMisprice` after computing consensus and execution blocks
  - Add misprice fields to the `odds` object
- `packages/data/db/migrations/040_add_misprice_to_odds_snapshots.sql`
  - `spread_is_mispriced INTEGER` (0/1)
  - `spread_misprice_type TEXT`
  - `spread_misprice_strength REAL`
  - `spread_outlier_book TEXT`
  - `spread_outlier_delta REAL`
  - `spread_review_flag INTEGER` (0/1)
  - Same columns for `total_*` prefix
- `packages/data/src/db.js` — `insertOddsSnapshot`
- `apps/worker/src/jobs/pull_odds_hourly.js` + `refresh_stale_odds.js`
- `web/src/app/api/games/route.ts` — expose misprice fields
- `web/src/components/cards-page-client.tsx`
  - When `spread_is_mispriced` and `spread_misprice_type === 'SOFT_LINE'`: show a small flag in the card's math detail section: "⚠ Soft line at [Book] (+X.X vs consensus)"
  - When `review_flag`: show a dimmer "Market disagreement" note

**Out of scope**:
- Type 1 misprice (projection vs consensus) — that's WI-0571
- Alerting / async scanning — that's a future agent layer
- Props and team totals

**Acceptance**:
- For a game where one book's spread is 2+ pts off the others: `spread_is_mispriced = true`, `spread_misprice_type = 'SOFT_LINE'`, `outlier_book` identifies the outlier, `outlier_delta_vs_consensus` is correct.
- For a game where all lines are equal but one book offers +105 vs others at -110: `misprice_type = 'PRICE_ONLY'`, `misprice_strength` is in bps.
- `review_flag = true` when spread stddev > 1.5 pts, even if no single book is an outlier.
- Flag renders in card UI for games that are soft or high-dispersion.
- TypeScript compiles cleanly.
