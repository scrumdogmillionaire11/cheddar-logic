# WI-0335: Eliminate Card Snapshot Race Conditions Across APIs and Model Jobs

**ID**: WI-0335  
**Goal**: Remove remaining run publication races and run-state inconsistencies that cause cards/plays to disappear, flicker, or diverge between endpoints.

**Scope**:

- `packages/data/src/db.js`
- `apps/worker/src/jobs/run_nba_model.js`
- `apps/worker/src/jobs/run_nhl_model.js`
- `apps/worker/src/jobs/run_ncaam_model.js`

**Out of scope**:

- DB migrations
- CI/workflow files
- UI component redesign
- Settlement job logic

**Acceptance**:

- Worker write paths no longer clear currently published cards before new run publish.
- All major card-producing model jobs stamp cards with `run_id` and publish per-sport `run_state` on success.
- `/api/cards`, `/api/games`, and `/api/cards/[gameId]` apply consistent active-run filtering semantics and safe fallback behavior when run state is missing.
- Card dedupe/select ordering is deterministic under timestamp ties.
- No syntax errors in touched files.
