## WI: Canonical Decision Authority

**Goal:** One module owns Play / Slight Edge / Pass truth across worker, web, results, Discord, and POTD.

**Why:** The mapper found worker truth is recomputed by web/read layers, with `decision_v2`, `final_play_state`, `status`, `action`, and `classification` competing. First fix is one decision-authority module and fail-closed reads when canonical fields are missing.

**Core contract**

```ts
CanonicalDecision {
  official_status: "PLAY" | "SLIGHT_EDGE" | "PASS";
  is_actionable: boolean;
  tier: "PLAY" | "SLIGHT_EDGE" | "PASS";
  reason_code: string;
  source: "decision_authority";
  lifecycle: DecisionStage[];
}

DecisionStage {
  stage: "parser" | "model" | "publisher" | "watchdog" | "read_api";
  status: "CLEARED" | "DOWNGRADED" | "BLOCKED" | "PASS";
  reason_code: string;
}
```

**Acceptance**

1. Cards, games, results, Discord card publishing, and POTD candidate selection all read canonical decision state.
2. Active APIs fail closed if canonical decision is missing.
3. Legacy repair is historical-only; no active global fallback.
4. Watchdog only vetoes trust/safety failures, not betting-edge logic.
5. `NO_QUALIFIED_PROPS` and `SKIP_MARKET_NO_EDGE` always produce empty official plays. No "best trash wins." Existing watchdog/orchestrator docs already require this behavior.
