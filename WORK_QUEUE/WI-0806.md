## Root Cause Investigation

The quant audit found 1,384 settled cards — all game-level markets (MONEYLINE / SPREAD / TOTAL / PUCKLINE). Zero prop cards have settled. 66,802 cards are in `status = 'error'` (~80% of all generated), but it is unknown whether these are predominantly props or game cards.

**Step 1 — Diagnose before touching any code.** SSH to Pi and run:

```sql
-- Break down error cards by market_type
SELECT
  json_extract(payload, '$.market_type') AS market_type,
  COUNT(*)                               AS n
FROM card_results
WHERE status = 'error'
GROUP BY 1
ORDER BY 2 DESC;

-- Sample the actual error messages for props
SELECT
  json_extract(payload, '$.market_type')                       AS market_type,
  json_extract(payload, '$.settlement_error')                  AS err,
  json_extract(payload, '$.card_type')                         AS card_type,
  COUNT(*)                                                     AS n
FROM card_results
WHERE status = 'error'
  AND json_extract(payload, '$.market_type') = 'PLAYER_PROP'
GROUP BY 1, 2, 3
ORDER BY 4 DESC
LIMIT 30;

-- Confirm any settled props exist
SELECT COUNT(*) FROM card_results
WHERE status = 'settled'
  AND json_extract(payload, '$.market_type') = 'PLAYER_PROP';
```

Document diagnosis findings before writing any code.

## Hypotheses (ranked by likelihood)

1. **Settlement path never branches on `PLAYER_PROP`** — `settle_pending_cards.js` queries `card_results WHERE status = 'pending'` but the settlement logic only accesses game-level result fields (`home_score`, `away_score`, `goals`, etc.) and never reads prop stat fields (`pts`, `sog`, `blk`, `k`, `assists`). Prop cards hit a code path that sets `status = 'error'` with a missing-result sentinel.

2. **`game_id` join mismatch** — Prop cards may store `game_id` from the odds API in a format that doesn't match the `game_results` table populated by `settle_game_results.js`. A null join sends the card to the error branch.

3. **`market_type` normalization strips `PLAYER_PROP` to something unrecognized** — `settle_pending_cards.js` line 1237 does `market_type` normalization. If the normalization map lacks `PLAYER_PROP`, cards get an unexpected value and fall through to error.

4. **`odds_snapshots` closing-line fetch fails for props** — Not applicable to settlement result (WIN/LOSS) but may cause partial failures.
