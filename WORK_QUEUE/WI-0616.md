# WI-0616: Rename welcome-home-v2 card_type to welcome-home + DB alias

**ID**: WI-0616
**Goal**: Change the production `cardType` value from `'welcome-home-v2'` to `'welcome-home'` so the version suffix no longer leaks into the DB, API responses, and card payloads. Wire a read-time alias for backward compat with existing rows.

**Background**:
`cardType = 'welcome-home-v2'` is the live identifier emitted into `card_payloads` by `run_nhl_model.js`, `run_nba_model.js`, and `welcome-home-v2.js`. v1 no longer exists, making "v2" permanently the canonical name — any future v3 would require a migration. The name leaks into the DB and API responses, confusing consumers who don't know the versioning history.

**Scope**:
- `apps/worker/src/models/welcome-home-v2.js` — change `cardType` export from `'welcome-home-v2'` to `'welcome-home'`
- `apps/worker/src/jobs/run_nhl_model.js`, `run_nba_model.js` — update any hardcoded `'welcome-home-v2'` string references
- `apps/worker/src/models/index.js` — update registration key if keyed by card type string
- `web/src/app/api/games/route.ts` — add `'welcome-home-v2'` as alias in card_type query/filter so old DB rows still match
- `web/src/lib/game-card/transform.ts` — handle both `'welcome-home'` and `'welcome-home-v2'` during transition

**Out of scope**:
- Renaming the file `welcome-home-v2.js` (deferred; file rename is a separate low-risk commit)
- Backfilling old DB rows (alias handles backward compat)

**Acceptance**:
- New cards emit `card_type = 'welcome-home'`.
- Existing `card_type = 'welcome-home-v2'` rows still appear in queries via alias.
- `npm --prefix apps/worker test` passes.
- `npx tsc --noEmit --project web/tsconfig.json` passes.
