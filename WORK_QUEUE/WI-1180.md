# WI-1180: Prevent Silent Consensus Fallback for Incomplete Model Payloads in POTD

**ID**: WI-1180
**Goal**: Ensure POTD does not silently downgrade incomplete model payloads to consensus candidates and instead records explicit rejection reasons.

**Context**:

- Current POTD scoring can produce high-scoring consensus fallback candidates when model payload exists but fails extraction/actionability.
- This obscures model data quality failures and causes `NON_MODEL_SOURCE` noise without actionable diagnostics.

**Scope**:

- `apps/worker/src/jobs/potd/signal-engine.js`
- `apps/worker/src/jobs/potd/run_potd_engine.js`
- `apps/worker/src/jobs/potd/__tests__/signal-engine.test.js`
- `apps/worker/src/jobs/potd/__tests__/run-potd-engine.test.js`
- `WORK_QUEUE/WI-1180.md`
- `.planning/ROADMAP.md`
- `.planning/STATE.md`
- `.planning/phases/WI-1180/**`

**Out of scope**:

- Any NHL/MLB producer payload write-path changes.
- Adjusting POTD scoring weights, noise floors, or NBA sigma logic from WI-1178.
- Modifying `EDGE_SOURCE_CONTRACT`, `edgeSourceTag`, or `isModelBackedCandidate` behavior contract.

**Acceptance**:

- When a model payload is present but incomplete/non-actionable for a contract-`MODEL` market, POTD records explicit failure diagnostics (for example `MODEL_SIGNAL_INCOMPLETE`) instead of silently falling through to a normal consensus candidate path.
- Consensus fallback remains allowed only for markets whose contract expects `CONSENSUS_FALLBACK`.
- MLB modern actionable model rows can be scored as `MODEL` once extractor data is available.
- NHL PASS/evidence rows remain non-actionable and are surfaced with explicit rejection diagnostics.
- Tests cover: incomplete model payload path emits explicit rejection diagnostics, no silent consensus fallback for contract-`MODEL` markets, existing eligibility gates still pass.
