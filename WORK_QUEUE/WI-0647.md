# WI-0647: Cross-market settlement win-rate audit (ROI report)

**ID**: WI-0647
**Goal**: Write a SQL-backed analysis script that queries the `card_results_ledger` for all settled cards since January 2026 and produces a per-market win-rate, ROI (units), and coverage report that can be run on-demand to inform model promotion/quarantine decisions.

**Background**:
`market_period_token` is now persisted at settlement (WI-0607 ✓), `settle_mlb_f5` doubleheader bug is fixed (WI-0626 ✓), and the CLV ledger is wired (WI-0557 ✓). This is the first time a reliable cross-market results read is possible. Settlement infrastructure is clean but no one has ever run an actual ROI cut. This report is the prerequisite for any evidence-based model promotion or quarantine decision.

**Scope**:
- `scripts/settlement-roi-report.js` (new script — read-only DB access via read-only client)
- `docs/SETTLEMENT_ROI_GUIDE.md` (new — documents how to run, interpret, and act on the report)

**Out of scope**:
- Changes to settlement logic
- Changes to any model runner
- Automated scheduling of this report (separate WI if needed)

**Acceptance**:
- `node scripts/settlement-roi-report.js` runs against prod DB (read-only) and outputs a table: `market_key | period | settled_count | win_count | win_rate | avg_clv | units_won`
- Covers all market keys present in `card_results_ledger` from 2026-01-01 onward
- Output includes a per-sport rollup and a recommendation flag (`PROMOTE` / `WATCH` / `QUARANTINE`) based on win-rate thresholds: >54% = PROMOTE, 50–54% = WATCH, <50% = QUARANTINE (min 20 settled cards)
- Uses read-only DB access only (`openDatabaseReadOnly`)
- `docs/SETTLEMENT_ROI_GUIDE.md` contains: how to run, column definitions, threshold rationale, how to update quarantine list from output
