# WI-1210: Enforce /api/results Performance Budget + High-Cardinality Regression Harness

**ID**: WI-1210
**Goal**: Add enforceable latency and heap regression coverage for `/api/results` under high-cardinality datasets so performance regressions fail CI before production impact.

**Scope**:

- `web/src/app/api/results/route.ts`
- `web/src/lib/results/query-layer.ts`
- `web/src/lib/results/transform-layer.ts`
- `web/src/__tests__/integration/results-behavioral-contract.test.ts` (extend if appropriate)
- `web/src/__tests__/` (new performance-focused results test file(s))
- `web/package.json` (test script wiring only, if needed)

**Out of scope**:

- Full pagination/cursor redesign of `/api/results`
- Worker settlement logic changes
- UI rendering/virtualization changes on `/results`

**Acceptance**:

- A deterministic high-cardinality fixture path exists for `/api/results` performance testing (seeded and repeatable)
- Automated test(s) assert max response latency budget for key query variants:
  - default `/api/results`
  - `/api/results?limit=200`
  - `/api/results?limit=200&dedupe=0`
  - at least one filtered variant (`sport`, `market`, or `min_confidence`)
- Automated test(s) assert heap budget deltas (or equivalent memory pressure guard) during request execution
- Tests fail clearly when latency/heap budgets are exceeded
- Test runtime is stable enough for CI (no flaky threshold behavior under normal local/CI variance)
- Existing results contract/behavior tests still pass
