# WI-0581 — Rename `decision_v2.edge_pct` → `edge_delta_pct` for Semantic Clarity

**ID:** WI-0581
**Goal:** Rename the `edge_pct` field in `decision_v2` payloads to `edge_delta_pct` to distinguish it from probability-based edge (`edge_over_pp`). Both are currently labelled as "edge" despite measuring fundamentally different quantities.
**Severity:** HIGH
**Owner agent:** unassigned
**Status:** OPEN
**Time window:** —
**Coordination flag:** needs-sync (public API surface change — `decision_v2` payload schema)

## Problem

`decision_v2.edge_pct` is `(mu − line) / line × 100` — a projection-delta percentage.  
`decision_v2.edge_over_pp` is the probability edge: `p_fair − p_implied`.  

Both surface as "edge" in downstream code, Discord posts, and the UI. Consumers (and agents) conflate them, leading to incorrect EV reasoning. For example: a positive `edge_pct` of +4% does NOT mean +4% EV — it means the model projects 4% more SOG than the line, which may not be a positive-EV bet if the price is bad.

## Scope

- `apps/worker/src/jobs/run_nhl_player_shots_model.js` — rename `edge_pct` → `edge_delta_pct` at every write site in the `decision_v2` block.
- Search all consumers: `web/src/`, `apps/worker/src/utils/`, `packages/` for reads of `decision_v2.edge_pct` and update them.
- Add a comment in the schema definition explaining both fields.

## Out of Scope

- Renaming `edge_over_pp` (the probability edge field — name is already accurate).
- Changes to the V1 edge fields outside `decision_v2`.
- Database migration (field lives in JSON blob inside `card_payloads.payload`, no schema change needed).
