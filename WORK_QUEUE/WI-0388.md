# WI-0388: Omit Started Games from `/api/games` (Default) for Cleaner `/cards`

**ID**: WI-0388
**Goal**: Reduce `/cards` noise by omitting started games from `/api/games` by default, with an explicit diagnostics override.

**Scope**:
- `WORK_QUEUE/WI-0388.md`
- `web/src/app/api/games/route.ts`
- `web/src/__tests__/api-games-filter.test.js`
- `web/src/__tests__/api-games-repair-budget.test.js`

**Out of scope**:
- `/api/cards` and `/api/cards/[gameId]` behavior changes
- Worker model/job logic changes
- DB schema or migration changes
- UI redesign/toggle work in `cards-page-client.tsx`

**Acceptance**:
- `/api/games` defaults to upcoming-only results (`game_time_utc > now`) so started games are omitted.
- `/api/games?include_started=true` includes started games (restores same-day visibility for diagnostics).
- Non-prod `meta.diagnostics.query_window` includes explicit started-game mode indicators.
- Existing `/cards` client works without query changes and reflects cleaner board output.
- Regression tests cover default exclusion and override inclusion behavior.
