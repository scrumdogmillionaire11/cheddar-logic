# WI-1020: NBA Phase 1C - Rolling bias calibration loop

**ID**: WI-1020
**Goal**: Apply a rolling mean-error bias correction to the NBA pace-anchor total projection at job start, eliminating the systemic +9 pt over-projection - with a strict anti-leak contract so calibration does not become a drunk mechanic with a wrench.

**Depends on**: WI-1019, WI-1027 (accuracy tracking and schema hardening must be active; calibration rows require `raw_total`, `actual_total`, and context fields)

**Scope**:
- `apps/worker/src/jobs/run_nba_model.js`
- `apps/worker/src/jobs/__tests__/run_nba_model.test.js`
- `apps/worker/src/utils/nba-bias.js` (only if extracting helper)

**Out of scope**:
- Sigma (variance) computation - `computeSigmaFromHistory` is unchanged
- Other sport models
- UI changes
- `projection-accuracy.js` data layer (WI-1019/WI-1027)

**Acceptance**:
- New function `computeNbaRollingBias({ db, windowGames })` (inline or extracted to `apps/worker/src/utils/nba-bias.js`) returns `{ bias, games_sampled, source }` where:
  - `source: 'computed'` when >= 50 settled games exist
  - `source: 'fallback'` with `bias: 0` when < 50 settled games exist
- When `source: 'fallback'`, logs: `[NBAModel] [CALIBRATION_INACTIVE] N settled games - minimum 50 required before correction activates`
- When `source: 'computed'`, logs: `[NBAModel] [BIAS_CORRECTION] rolling_bias=+N.N games_sampled=M`
- Bias correction is applied before blend in `applyNbaTeamContext`: `correctedAnchor = paceAnchorTotal - rollingBias`
- Maximum absolute correction is capped at **+/-4.0 points**. If `abs(rollingBias) > 4`, log a warning: `[NBAModel] [BIAS_CORRECTION] bias_raw=N.N clamped to +/-4.0 - investigate outlier`
- Rolling window is **50 most recent settled games** (default). This is a deliberate conservative choice: enough signal to be meaningful, small enough to react to mid-season shifts.
- Correction is **global** only in this WI (across all NBA total games). Segmented correction by pace_tier / total_band is reserved for WI-1024 (residual learning). This WI must not attempt per-segment correction.
- `raw_data.calibration_state` stamped on each card: `{ bias, games_sampled, source, correction_applied }`.
- Unit test: `computeNbaRollingBias` returns correct mean error for mock db with known rows; returns fallback when rows < 50; returns fallback when rows = 0.
- `run_nba_model.test.js` verifies `rollingBias` is threaded into the pace anchor blend.

**Anti-leak contract (non-negotiable)**:

The following invariants must hold and must be tested:

1. **No future leakage**: the query orders by `settled_at DESC LIMIT 50`. `settled_at` must be a verified past timestamp, not `created_at` or `updated_at`. The query must include `AND settled_at < datetime('now')`.

2. **No calibration on the same game**: the query must exclude the game currently being projected. The game_id for the current job run is not available at calibration compute time, but the `settled_at < datetime('now')` constraint is sufficient since in-progress games are not yet settled.

3. **Insufficient data means no correction**: when `games_sampled < 50`, the raw projection is used unchanged. This is not a fallback to some smaller sample - it is a hard gate. Using 10 noisy samples as calibration is worse than no calibration.

4. **Correction does not compound**: the corrected anchor is used in the blend, not re-corrected. Downstream WI-1024 residual correction is additive on top, not multiplicative. Document this explicitly in the code comment.

**Rolling bias query**:

```sql
SELECT AVG(sample.raw_total - sample.actual_total) AS mean_error, COUNT(*) AS n
FROM (
  SELECT raw_total, actual_total
  FROM projection_accuracy_line_evals
  WHERE sport = 'nba'
    AND market_family = 'NBA_TOTAL'
    AND actual_total IS NOT NULL
    AND raw_total IS NOT NULL
    AND settled_at < datetime('now')
  ORDER BY settled_at DESC
  LIMIT 50
) AS sample
```

Note: query uses `raw_total` and `actual_total` from the hardened schema (WI-1027), not `projection_raw` and `actual_value` aliases.

**Phase 1 exit gate**:

Phase 1 (WI-1018, WI-1019, WI-1020, WI-1027) is not complete until the following are true over a minimum of 50 settled NBA total games post-deployment:

- Absolute mean bias < 3.0 points (down from +9.26)
- RMSE < 16.0 (down from ~20)
- `240+` total band absolute mean bias < 4.0 points
- No single driver contributing uncapped pace inflation (verifiable from driver_contributions_json)
- Post-gate play rate is not collapsed (at least some games still producing FIRE/WATCH cards)
- `CALIBRATION_INACTIVE` no longer appearing in logs (sample has crossed 50 games)

These are the criteria, not vibes. If they are not met, do not move to Phase 2. Investigate first.
