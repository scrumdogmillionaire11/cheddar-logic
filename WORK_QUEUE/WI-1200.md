# WI-1200: Wire Web to DecisionOutcome

**ID**: WI-1200
**Goal**: Refactor web surfaces (cards, games, results) to consume `DecisionOutcome` as the single source of truth, eliminating duplicate status/classification logic and proving parity with current behavior.

**Scope**:
- `web/src/lib/game-card/decision.ts` (import canonical builder from @cheddar-logic/data, apply to decision_v2, consume outcome only)
- `web/src/lib/game-card/transform/index.ts` (consume DecisionOutcome only from decision.ts, no local status recompute)
- `web/src/app/api/cards/route.ts` (pull decision_v2 from worker, apply canonical builder, serialize DecisionOutcome)
- `web/src/app/api/games/route.ts` (same: builder then outcome)
- `web/src/app/api/results/projection-metrics.ts` (same: builder then outcome)
- `web/src/__tests__/cards-decision-outcome-parity.test.ts` (new file: verify web builder application produces expected status for 50+ fixture decisions)
- `web/src/__tests__/games-decision-outcome-parity.test.ts` (new file: same for games)
- `web/src/__tests__/results-decision-outcome-parity.test.ts` (new file: same for results)

**Out of scope**:
- Changing decision_v2 input sources (worker still produces decision_v2; web applies builder at boundary)
- Implementing web-specific decision mappers (use canonical builder from WI-1199 only)
- Deleting old mappers (WI-1203)

**Acceptance**:
- All API routes call `buildDecisionOutcomeFromDecisionV2()` from canonical builder (imported from @cheddar-logic/data) and return its result; no local mapping logic
- Parity tests compare outcome.status on 50+ fixture decisions: status, count, and presentation must match prior web behavior exactly
- Web does NOT implement its own decision_v2 → outcome mapper; any mapping must import from shared builder only

- If DecisionOutcome has `reasons.blockers`, web shows them
- All existing card/game/results tests pass
- No unused decision mappers in web code (dead code removal only; do not delete files yet)
- TypeScript strict mode passes; `buildDecisionOutcomeFromDecisionV2` import resolves cleanly
