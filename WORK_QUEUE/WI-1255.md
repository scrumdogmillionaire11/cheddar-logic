# WI-1255: Enforce Confidence Caps When Trap Inputs Are Missing

**ID**: WI-1255
**Goal**: Prevent fake confidence by hard-capping posture/confidence when critical trap prerequisites are missing or stale.

**Context**:

Projection-only outputs can still overstate confidence if missing-input conditions are not hard-gated.

**Scope**:
- `apps/worker/src/models/mlb-model.js`
- `apps/worker/src/jobs/run_mlb_model.js`
- `apps/worker/src/jobs/mlb-k-input-classifier.js`
- `apps/worker/src/jobs/__tests__/run_mlb_model.test.js`
- `apps/worker/src/jobs/__tests__/mlb-k-input-classifier.test.js`
- `docs/pitcher_ks/scoring.md`
- `docs/pitcher_ks/08datareqs.md`
- `WORK_QUEUE/WI-1255.md`

**Out of scope**:
- Any live odds-backed edge calculations
- Threshold tuning unrelated to missing-input confidence caps
- Portfolio-level constraints

**Acceptance**:
- Confidence capping rules are enforced:
  - missing opponent live-split freshness -> cap at `WATCH`
  - missing leash confidence -> cap at `WATCH`
  - static fallback opponent profile -> cap posture to `DATA_UNTRUSTED`
  - short leash plus over candidate -> force `TRAP_FLAGGED`
  - low-K opponent plus high pitcher projection -> cap posture to `UNDER_LEAN_ONLY`
- If opponent data is fallback/static, card cannot be labeled as full-trust model quality.
- If both opponent live-split freshness and leash confidence are missing:
  - suppress posture assignment entirely
  - emit `NO_OUTPUT_INSUFFICIENT_DATA` instead of any posture label
  - card is excluded from candidate output

- Tests verify each cap rule path and reason-code emission.
