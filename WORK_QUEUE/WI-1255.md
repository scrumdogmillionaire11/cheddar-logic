# WI-1255: Enforce Confidence Caps When Trap Inputs Are Missing

**ID**: WI-1255
**Goal**: Prevent fake confidence by hard-capping posture/confidence when critical trap prerequisites are missing or stale.

**Context**:

Projection-only outputs can still overstate confidence if missing-input conditions are not hard-gated.

**Scope**:
- `apps/worker/src/models/mlb-model.js`
- `apps/worker/src/jobs/run_mlb_model.js`
- `apps/worker/src/jobs/mlb-k-input-classifier.js`
- `apps/worker/src/jobs/__tests__/run_mlb_model.test.js`
- `apps/worker/src/jobs/__tests__/mlb-k-input-classifier.test.js`
- `docs/pitcher_ks/scoring.md`
- `docs/pitcher_ks/08datareqs.md`
- `WORK_QUEUE/WI-1255.md`

**Out of scope**:
- Any live odds-backed edge calculations
- Threshold tuning unrelated to missing-input confidence caps
- Portfolio-level constraints

**Acceptance**:
- Confidence capping rules are enforced using the exact field values produced by WI-1254 trap diagnostics:
  - `opp_profile_staleness === 'STALE'` -> cap posture at `WATCH`; emit reason `CAP_OPP_STALE`
  - `leash_bucket === 'UNKNOWN'` -> cap posture at `WATCH`; emit reason `CAP_LEASH_UNKNOWN`
  - `opp_profile_staleness === 'STATIC_FALLBACK'` -> cap posture to `DATA_UNTRUSTED`; emit reason `CAP_OPP_STATIC_FALLBACK`
  - `leash_bucket === 'SHORT'` AND posture candidate is over -> force `TRAP_FLAGGED`; emit reason `CAP_SHORT_LEASH_OVER`
  - `opp_k_bucket === 'LOW_K'` AND projected Ks > MID tier upper bound (6.5) -> cap posture to `UNDER_LEAN_ONLY`; emit reason `CAP_LOW_OPP_HIGH_PROJ`
- If opponent data is `STATIC_FALLBACK`, card MUST NOT be labeled as full-trust model quality.
- Full-suppression rule: if BOTH `opp_profile_staleness` is `STALE` or `STATIC_FALLBACK` AND `leash_bucket` is `UNKNOWN`:
  - suppress posture assignment entirely
  - emit `NO_OUTPUT_INSUFFICIENT_DATA` as the posture label
  - card MUST be excluded from candidate output
- `confidence_cap_reason` field (introduced as `null` in WI-1254) is populated with the applied cap reason code string, or remains `null` when no cap applies.
- Tests MUST cover all 6 scenarios as distinct cases:
  1. `opp_profile_staleness === 'STALE'` alone -> posture capped at `WATCH`, reason `CAP_OPP_STALE`
  2. `leash_bucket === 'UNKNOWN'` alone -> posture capped at `WATCH`, reason `CAP_LEASH_UNKNOWN`
  3. `opp_profile_staleness === 'STATIC_FALLBACK'` -> posture `DATA_UNTRUSTED`, reason `CAP_OPP_STATIC_FALLBACK`
  4. `leash_bucket === 'SHORT'` + over candidate -> posture `TRAP_FLAGGED`, reason `CAP_SHORT_LEASH_OVER`
  5. `opp_k_bucket === 'LOW_K'` + high projection -> posture `UNDER_LEAN_ONLY`, reason `CAP_LOW_OPP_HIGH_PROJ`
  6. Both freshness-missing AND leash-unknown -> posture `NO_OUTPUT_INSUFFICIENT_DATA`, card excluded from output
