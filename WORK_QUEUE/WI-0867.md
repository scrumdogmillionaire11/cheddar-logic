## Background

After WI-0864–0866 ship, proxy eval rows accumulate in the DB with every settlement cycle. This WI makes that data accessible to the frontend without requiring anyone to query the DB directly. The route is read-only, public (same as `/api/cards`), and requires no auth.

The data surface is accuracy signal only — no model recommendations, no picks, no prices. The route name uses "projection-accuracy" deliberately (not "results") to avoid any language suggesting a betting outcome.

## Response Shape

```typescript
// GET /api/results/projection-accuracy
// GET /api/results/projection-accuracy?family=MLB_F5_TOTAL
// GET /api/results/projection-accuracy?family=NHL_1P_TOTAL
// GET /api/results/projection-accuracy?days=30

interface ProjectionAccuracyResponse {
  generatedAt: string;              // ISO timestamp
  lookbackDays: number;
  families: ProjectionFamilySummary[];
}

interface ProjectionFamilySummary {
  card_family: string;              // 'MLB_F5_TOTAL' | 'NHL_1P_TOTAL'
  game_date_range: { gte: string; lte: string | null };
  total_games: number;
  total_proxy_decisions: number;
  wins: number;
  losses: number;
  no_bets: number;
  proxy_hit_rate: number | null;    // wins / (wins + losses)
  consensus_games: number;
  consensus_wins: number;
  consensus_hit_rate: number | null;
  split_zone_games: number;
  avg_tier_score: number | null;
  total_score: number;
  by_tier: {
    LEAN:   TierRecord;
    PLAY:   TierRecord;
    STRONG: TierRecord;
  };
  by_proxy_line: Record<string, TierRecord>;  // keyed by '1.5', '3.5', '4.5', etc.
}

interface TierRecord {
  decisions: number;
  wins: number;
  losses: number;
  hit_rate: number | null;
}
```

Define these interfaces in `web/src/lib/types/projection-accuracy.ts` and import from there in the route.

## Route Logic

```typescript
import { NextResponse } from 'next/server';
import type { ProjectionAccuracyResponse } from '@/lib/types/projection-accuracy';

const VALID_FAMILIES = ['MLB_F5_TOTAL', 'NHL_1P_TOTAL'] as const;
const DEFAULT_LOOKBACK_DAYS = 90;
const MAX_LOOKBACK_DAYS = 365;

export async function GET(request: Request) {
  const { searchParams } = new URL(request.url);

  const familyParam = searchParams.get('family');
  const daysParam = searchParams.get('days');

  // Validate family
  if (familyParam && !VALID_FAMILIES.includes(familyParam as typeof VALID_FAMILIES[number])) {
    return NextResponse.json({ error: 'Invalid family' }, { status: 400 });
  }

  // Validate days
  const lookbackDays = daysParam ? parseInt(daysParam, 10) : DEFAULT_LOOKBACK_DAYS;
  if (!Number.isFinite(lookbackDays) || lookbackDays < 1 || lookbackDays > MAX_LOOKBACK_DAYS) {
    return NextResponse.json({ error: 'Invalid days parameter (1–365)' }, { status: 400 });
  }

  const gameDateGte = new Date(Date.now() - lookbackDays * 24 * 60 * 60 * 1000)
    .toISOString()
    .split('T')[0];

  const familiesToQuery = familyParam
    ? [familyParam]
    : [...VALID_FAMILIES];

  // Import DB functions — use dynamic require consistent with other web API routes
  // that access the data package (match the import pattern used in
  // web/src/app/api/results/projection-settled/route.ts)
  const { getProjectionAccuracySummary } = await import(
    '@cheddar-logic/data/db/projection-accuracy'
  );
  const { openDatabaseReadOnly } = await import('@cheddar-logic/data');
  const db = openDatabaseReadOnly(process.env.CHEDDAR_DB_PATH!);

  const families = familiesToQuery.map((cardFamily) =>
    getProjectionAccuracySummary(db, { cardFamily, gameDateGte })
  );

  const payload: ProjectionAccuracyResponse = {
    generatedAt: new Date().toISOString(),
    lookbackDays,
    families,
  };

  return NextResponse.json(payload);
}
```

**Import pattern note:** Match the exact `require` / `import` pattern used in `web/src/app/api/results/projection-settled/route.ts` for DB access. Do not call `openDatabase()` (writer) — always `openDatabaseReadOnly()` (ADR-0002 single-writer contract).

## Acceptance Criteria

1. `GET /api/results/projection-accuracy` returns 200 with `generatedAt`, `lookbackDays`, `families` array containing both `MLB_F5_TOTAL` and `NHL_1P_TOTAL`
2. `GET /api/results/projection-accuracy?family=MLB_F5_TOTAL` returns 200 with `families` array of length 1
3. `GET /api/results/projection-accuracy?family=INVALID` returns 400
4. `GET /api/results/projection-accuracy?days=0` returns 400
5. `GET /api/results/projection-accuracy?days=365` returns 200
6. Route never calls `openDatabase()` — only `openDatabaseReadOnly()` (grep check)
7. `npx tsc --noEmit --project web/tsconfig.json` passes with 0 errors
8. Grep for `bet|pick|recommend|wager` in route and types files → 0 matches
