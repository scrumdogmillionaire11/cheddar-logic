# WI-0830: Additive Z-Score Model Core — Shared Bounded Scoring Framework

| Field | Value |
|---|---|
| **ID** | WI-0830 |
| **Goal** | Replace the per-sport multiplicative adjustment stacks with a shared additive z-score aggregation layer. Each feature contribution is normalised to a z-score, weighted, summed, and passed through a single globally-clamped sigmoid transform to produce a bounded `modelScore ∈ (0, 1)`. The current approach applies sequential multipliers (e.g. MLB offense: `xwoba × iso × hard_hit × wrc_plus`) which causes compounding distortion when any factor is near-zero or extreme. |
| **Priority** | High |
| **Depends on** | WI-0820 (NO_BET gate must be clean before changing projection math); WI-0821 (MLB offense composite); WI-0822 (NBA ORtg normalization); WI-0823 (NHL goalie composite) |

## Background

Every sport model currently builds projections by multiplying or chain-applying adjustment factors:

```
adjustedRa9 = baseRa9 * offenseAdj * parkFactor * tempFactor * ...
```

This creates three failure modes:
1. **Compounding amplification** — two 1.05x factors compound to 1.10x, not 1.05+1.05
2. **Unbounded output** — no clamp on final projection means downstream edge is uncapped
3. **Non-comparable scores** — NBA `projectedTotal` and NHL `expectedGoals` live on different scales with no shared interpretation

The fix is a normalisation layer: convert each feature to its z-score relative to a rolling population baseline, weight by importance, sum, and transform once.

## Core Algorithm

### score-engine.js API

```js
/**
 * Aggregate a set of feature contributions into a bounded model score.
 *
 * @param {Array<{value: number, mean: number, std: number, weight: number, name: string}>} features
 * @param {object} [opts]
 * @param {number} [opts.outputClampLow=0.2]   — minimum allowed output
 * @param {number} [opts.outputClampHigh=0.8]  — maximum allowed output
 * @returns {{ score: number, contributions: object, zScores: object }}
 */
function aggregate(features, opts = {})
```

Internally:
1. For each feature: `z_i = (value - mean) / std`; clamp `z_i ∈ [-3, 3]`
2. Weighted sum: `S = Σ(weight_i × z_i) / Σ(weight_i)`
3. Sigmoid: `score = 1 / (1 + exp(-k × S))` where `k = 2.0` (tunable)
4. Output clamp: `score = clamp(score, outputClampLow, outputClampHigh)`

Return `{ score, contributions, zScores }` — contributions and zScores written to card `drivers[]` for observability.

### Population baselines

Use configurable per-sport per-feature baseline objects (mean/std). Initial values derived from season-average static constants already in each model file (e.g. `MLB_F5_DEFAULT_TEAM_XWOBA`, `LEAGUE_AVG_K_PCT`). These are updated seasonally, same cadence as current defaults.
