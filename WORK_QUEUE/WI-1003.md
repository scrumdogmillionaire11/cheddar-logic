# WI-1003: Replace source-string test assertions with behavior-based decision tests

**ID**: WI-1003

**Goal**: Rewrite the brittle source-text assertions in `play-decision-edge-verification.test.js` as fixture-driven behavior tests that exercise the actual decision pipeline with known inputs and assert on structured outputs.

**Priority**: Low — tests currently pass but will false-negative any refactor that preserves behavior while changing implementation text; they provide false confidence.

**Owner agent**: Unassigned

**Time window**: TBD — can be done any time; no runtime impact.

**Coordination flag**: None — test-only change, no shared touchpoints.

---

## Background

`web/src/__tests__/play-decision-edge-verification.test.js` asserts against source code text using `source.includes(...)`:

```js
// Lines 17–37 (current)
assert(source.includes("export const EDGE_VERIFICATION_TAG = 'LINE_NOT_CONFIRMED';"), ...)
assert(source.includes('export function hasEdgeVerificationSignals('), ...)
assert(source.includes("'BLOCKED_BET_VERIFICATION_REQUIRED'"), ...)
assert(source.includes('EDGE_SANITY_GATE_CODE,'), ...)
assert(source.includes('gate.code === EDGE_SANITY_GATE_CODE'), ...)
```

These tests:
- Break on any internal rename even when behavior is unchanged
- Pass even if the functions are exported but non-functional
- Do not verify that `hasEdgeVerificationSignals` returns the correct result for known inputs
- Duplicate the coverage intent of `game-card-pass-surface-contract.test.js` without the behavioral guarantee

## Required replacements

Each source-string assertion should be replaced with a behavioral equivalent:

| Removed assertion | Replacement |
| ----------------- | ----------- |
| `EDGE_VERIFICATION_TAG = 'LINE_NOT_CONFIRMED'` exported | Import `EDGE_VERIFICATION_TAG` and assert its value equals `'LINE_NOT_CONFIRMED'` |
| `hasEdgeVerificationSignals` exported | Call it with fixtures that should return `true` and `false`; assert both |
| `BLOCKED_BET_VERIFICATION_REQUIRED` present in source | Create a play fixture with that reason code; assert it is treated as a block |
| `EDGE_SANITY_GATE_CODE` present in source | Create a gate fixture with that code; assert gate evaluation returns expected result |
| `gate.code === EDGE_SANITY_GATE_CODE` present | Covered by gate fixture test above |

## Fixture approach

Use the existing fixture pattern from `game-card-pass-surface-contract.test.js` for consistency. Minimal play/gate fixture objects with only the fields needed to drive the assertion.
