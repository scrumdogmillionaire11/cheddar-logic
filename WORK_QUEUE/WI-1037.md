# WI-1037: Enable all projection-family settlement tracking on /results + near-miss POTD tracking

**ID**: WI-1037
**Goal**: Remove NHL 1P-only rendering/query constraints so every settled projection family appears in the /results projection settlement surface, and track near-miss POTD candidates (top non-winning plays) with settled performance metrics.

**Depends on**: WI-0967 (projection settlement slice). POTD near-miss slice is otherwise independent.

**Scope**:

- `web/src/app/api/results/projection-settled/route.ts`
- `web/src/components/results/ProjectionResultsTable.tsx`
- `web/src/app/results/page.tsx`
- `apps/worker/src/jobs/potd/run_potd_engine.js`
- `apps/worker/src/jobs/potd/settle-shadow-candidates.js`
- `apps/worker/src/schedulers/main.js`
- `web/src/lib/potd-server.ts`
- `web/src/app/api/potd/route.ts`

- `apps/worker/src/jobs/potd/__tests__/run-potd-engine.test.js`
- `apps/worker/src/jobs/potd/__tests__/shadow-settlement.test.js`

**Out of scope**:

- Changes to model math/scoring in sport model files (`apps/worker/src/models/**`)
- DB schema migrations outside existing POTD shadow-candidate tables
- Any changes to odds-backed results ledger logic in `web/src/app/api/results/route.ts`
- Discord formatting and posting behavior changes (`apps/worker/src/jobs/potd/format-discord.js`)

**Acceptance**:

- `/api/results/projection-settled` no longer hard-filters to `NHL_1P_TOTAL` and returns rows for all card families present in `projection_proxy_evals` (e.g., `NHL_1P_TOTAL`, `MLB_F5_TOTAL`, `MLB_PITCHER_K`, `NHL_PLAYER_SHOTS`, `NHL_PLAYER_SHOTS_1P`, `NHL_PLAYER_BLOCKS`).
- Projection settlement UI groups and renders results by `cardFamily` instead of assuming a single NHL 1P table.
- Projection settlement section copy in `/results` no longer implies only NHL 1P/MLB F5 coverage.
- Existing empty-state behavior remains intact when no settled projection rows exist.
- Existing badge semantics remain intact (`tier`, `direction`, `gradedResult`).
- Near-miss POTD candidates are deterministically selected from ranked fireable nominees as follows:
  - exclude the official winner when `winner_status='FIRED'`;
  - take up to the top 3 remaining candidates in rank order;
  - if fewer than 3 remain, take all available;
  - if none remain, record none.
- Near-miss POTD candidates are settled via the existing shadow settlement flow and expose aggregate success metrics (wins/losses/pushes, win rate, sample size) in POTD API/UI.
- Scheduler wiring runs POTD shadow settlement automatically in the standard settlement cadence (no manual-only dependency for ongoing tracking).
