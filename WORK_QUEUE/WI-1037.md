# WI-1037: Harden deploy worker smoke check and DB lock diagnostics

**ID**: WI-1037
**Goal**: Make production deploys detect post-start worker lock crashes and surface the competing lock-holder process in DB contention errors.

**Scope**:

- `.github/workflows/deploy-production.yml`
- `packages/data/src/db/connection.js`

- Production DB lock errors include lock-owner process metadata when available.
