# WI-1188: Canonicalize MLB Model and Execution-Gate Reason Codes

**ID**: WI-1188
**Goal**: Add the active MLB model and execution-gate reason codes to the canonical reason-code registry and guard them with a registry test so the worker and web surfaces stop drifting.

**Scope**:

- `WORK_QUEUE/WI-1188.md`
- `packages/data/src/reason-codes.js`
- `packages/data/src/__tests__/reason-codes.test.js`
- `web/src/lib/game-card/reason-labels.ts`

**Out of scope**:

- Reworking MLB model policy thresholds or execution-gate logic
- Registry alignment for NHL or NBA-specific tokens
- Web UI copy changes beyond labels generated from the canonical registry

**Implementation notes**:

- `apps/worker/src/models/mlb-model.js` emits inline reason codes (`PASS_SYNTHETIC_FALLBACK`, `PASS_DEGRADED_TOTAL_MODEL`, `PASS_CONFIDENCE_GATE`, `PASS_MODEL_DEGRADED`, `PASS_INPUTS_INCOMPLETE`, `PASS_NO_DISTRIBUTION`, `MODEL_DEGRADED_INPUTS`, `MARKET_SANITY_FAIL`, `SOFT_DEGRADED_TOTAL_MODEL`, `SOFT_MARKET_SANITY_FAIL`, `SOFT_WEAK_DRIVER_SUPPORT`, `PASS_UNKNOWN`, `PASS_PROJECTION_ONLY_NO_MARKET`) that are not in the canonical registry.
- `apps/worker/src/jobs/execution-gate.js` emits `PROJECTION_ONLY_EXCLUSION` as a drop-reason code that is not in the canonical registry.
- This WI only canonicalizes the literals actually emitted from these two files.

**Acceptance**:

- All active MLB model and execution-gate reason codes are present in `ALL_REASON_CODES` with labels in `REASON_CODE_LABELS`.
- A canonical registry test fails if those MLB/execution-gate reason codes drift out of the shared registry again.
- Existing canonical reason-code tests continue to pass.
- `REASON_CODE_SCHEMA_VERSION` is bumped to reflect the additions.
