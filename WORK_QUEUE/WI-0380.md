## Design

### Arbitration function signature

```js
/**
 * Resolve scraper and user inputs into a single CanonicalGoalieState.
 *
 * @param {object} scraperInput
 * @param {string|null} scraperInput.goalie_name        - Name from MoneyPuck / name-match
 * @param {number|null} scraperInput.gsax               - Season GSaX value
 * @param {number|null} scraperInput.save_pct           - Season save %
 * @param {'SCRAPER_NAME_MATCH'|'SEASON_TABLE_INFERENCE'} scraperInput.source_type
 *
 * @param {object|null} userInput
 * @param {string|null} userInput.goalie_name           - User-supplied starter name
 * @param {'CONFIRMED'|'EXPECTED'|'UNKNOWN'|null} userInput.status
 * @param {string|null} userInput.supplied_at           - ISO8601 timestamp; used for staleness check
 *
 * @param {string} gameId
 * @param {'home'|'away'} teamSide
 *
 * @returns {CanonicalGoalieState}
 */
function resolveGoalieState(scraperInput, userInput, gameId, teamSide)
```

### Precedence rules

1. **User-supplied name + status present and not stale**
   - User name becomes `goalie_name`
   - User status maps to `starter_state`
   - `starter_source = USER_INPUT`
   - If scraper also has a name that differs → add `CONFLICTING_SOURCE_EVIDENCE` to `evidence_flags`, set `starter_state = CONFLICTING`
   - If scraper name matches → clean state, no conflict flag

2. **Scraper name-match present, no usable user input**
   - `goalie_name = scraperInput.goalie_name`
   - `starter_state = EXPECTED` (name-match is not game-day confirmation)
   - `starter_source = SCRAPER_NAME_MATCH`

3. **Season-table inference only (GSaX present, no name match, no user input)**
   - `goalie_name = null`
   - `starter_state = UNKNOWN`
   - `starter_source = SEASON_TABLE_INFERENCE`
   - `tier_confidence = LOW`
   - Add `SEASON_TABLE_INFERENCE_ONLY` to `evidence_flags`

4. **No data from any source**
   - `starter_state = UNKNOWN`
   - `starter_source = SEASON_TABLE_INFERENCE`
   - `tier_confidence = NONE`
   - `goalie_tier = UNKNOWN`

### Tier derivation from GSaX

| GSaX range    | `goalie_tier` |
|---------------|--------------|
| > 10          | ELITE        |
| 5 – 10        | STRONG       |
| -5 – 5        | AVERAGE      |
| -10 – -5      | WEAK         |
| null / absent | UNKNOWN      |

```
resolveGoalieState — season-table only (gsax present, no name) → UNKNOWN, SEASON_TABLE_INFERENCE_ONLY flag

resolveGoalieState — user input is stale → STALE_USER_INPUT in evidence_flags, falls back to scraper
resolveGoalieState — user input is malformed → MALFORMED_USER_INPUT in evidence_flags, falls back to scraper
resolveGoalieState — CONFIRMED user status → starter_state CONFIRMED
resolveGoalieState — EXPECTED user status → starter_state EXPECTED
```

## Failure Modes & Guards

### Failure mode: conflict gets swallowed by precedence
**Guard:** `evidence_flags` is populated before the winning source is applied. The losing source's name is preserved in `evidence_flags` as `OVERRIDDEN_BY_USER: <scraper_name>` or similar.

### Failure mode: user input staleness not checked
**Guard:** `resolveGoalieState()` checks `supplied_at` against a configurable staleness window (default: 6 hours before game time). Stale inputs add `STALE_USER_INPUT` and drop to scraper precedence.

### Failure mode: arbitration called after projection already ran
**Guard:** `run_nhl_model.js` must call `resolveGoalieState()` before any call to `projectNHL()` or `predictNHLGame()`. Code review acceptance criterion.
