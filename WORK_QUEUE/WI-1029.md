# WI-1029: POTD — Per-candidate gate audit logging

**ID**: WI-1029
**Goal**: Emit a structured audit log entry for every scored POTD candidate showing exactly which gate(s) rejected it and why. Currently there is zero per-candidate visibility; debugging gate calibration requires guesswork.

**Background**:
`gatherBestCandidate` in `run_potd_engine.js` silently filters candidates via `viableCandidates`. When no play fires, the only signal is aggregate counts (`candidatesCount`, `viableCount`). There is no way to know whether the Rockies ML was rejected because of edge (0.4% < 2% floor), score, or something else. This WI adds structured logging so every rejection has a traceable reason.

**Depends on**: WI-1028 (sport-specific noise floors must exist before audit log can reference per-candidate floor)

**Scope**:
- `apps/worker/src/jobs/potd/run_potd_engine.js`

**Out of scope**:
- New DB columns or migrations (log to stdout only — persisted diagnostics are already handled by `potd_shadow_candidates`)
- Changes to `signal-engine.js`
- Any UI changes

**Acceptance**:
- After `gatherBestCandidate` scores each candidate, a structured log line is emitted for every candidate in `scoredCandidates`.
- Log format (JSON-serializable object logged via `console.log` or existing logger):

```json
{
  "potd_audit": true,
  "sport": "MLB",
  "selectionLabel": "Colorado Rockies",
  "marketType": "MONEYLINE",
  "edgePct": 0.004,
  "noiseFloor": 0.03,
  "passesNoise": false,
  "totalScore": 0.782,
  "minScore": 0.3,
  "passesScore": true,
  "passesConfidence": true,
  "rejectedReason": "BELOW_NOISE_FLOOR",
  "gameId": "..."
}
```

- `rejectedReason` values (first failing gate wins):
  - `"BELOW_NOISE_FLOOR"` — `edgePct < noiseFloor`
  - `"NEGATIVE_EDGE"` — `edgePct <= 0`
  - `"BELOW_MIN_SCORE"` — `totalScore < POTD_MIN_TOTAL_SCORE`
  - `"BELOW_CONFIDENCE_LABEL"` — `confidenceLabel === 'LOW'`
  - `"VIABLE"` — passed all gates (nominated)
- The audit log fires for all candidates regardless of outcome (pass or reject).
- When `POTD_AUDIT_LOG_ENABLED=false` is set in env, the audit log is suppressed entirely (default: enabled, i.e. omitting the env var means logs fire).
- Existing tests pass without modification.
- New unit test verifies that the audit entries emitted for a known candidate set contain the correct `rejectedReason` values.

**Implementation notes**:

Add `buildCandidateAuditEntry(candidate, noiseFloor, minScore)` as an internal function in `run_potd_engine.js`:

```js
function buildCandidateAuditEntry(candidate, noiseFloor, minScore) {
  const edgePct = candidate.edgePct ?? null;
  const totalScore = candidate.totalScore ?? null;
  const confidenceLabel = String(candidate.confidenceLabel || '').toUpperCase();
  const passesNoise = typeof edgePct === 'number' && edgePct >= noiseFloor;
  const passesPositive = typeof edgePct === 'number' && edgePct > 0;
  const passesScore = typeof totalScore === 'number' && totalScore >= minScore;
  const passesConfidence = confidenceLabel !== 'LOW';

  let rejectedReason = 'VIABLE';
  if (!passesPositive) rejectedReason = 'NEGATIVE_EDGE';
  else if (!passesNoise) rejectedReason = 'BELOW_NOISE_FLOOR';
  else if (!passesScore) rejectedReason = 'BELOW_MIN_SCORE';
  else if (!passesConfidence) rejectedReason = 'BELOW_CONFIDENCE_LABEL';

  return {
    potd_audit: true,
    sport: candidate.sport ?? null,
    selectionLabel: candidate.selectionLabel ?? null,
    marketType: candidate.marketType ?? null,
    gameId: candidate.gameId ?? null,
    edgePct,
    noiseFloor,
    passesNoise,
    totalScore,
    minScore,
    passesScore,
    passesConfidence,
    rejectedReason,
  };
}
```

Emit audit entries in `gatherBestCandidate` immediately after `scoredCandidates` is fully populated:

```js
if (process.env.POTD_AUDIT_LOG_ENABLED !== 'false') {
  for (const c of scoredCandidates) {
    const floor = resolveNoiseFloor(c.sport, c.marketType, POTD_MIN_EDGE);
    console.log(JSON.stringify(buildCandidateAuditEntry(c, floor, POTD_MIN_TOTAL_SCORE)));
  }
}
```
