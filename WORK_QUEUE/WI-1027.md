# WI-1027: NBA Phase 1B — Evaluation schema hardening

**ID**: WI-1027
**Goal**: Harden the projection accuracy evaluation table so calibration, residual learning, and volatility empirics have the exact fields they need — not a partial skeleton that forces downstream WIs to guess at missing data.

**Depends on**: WI-1019 (NBA card types must be tracked before schema can be hardened)

**Scope**:
- `packages/data/db/migrations/` (new migration adding columns to `projection_accuracy_line_evals`)
- `packages/data/src/db/projection-accuracy.js` (write path: populate new columns at settlement)
- `apps/worker/src/jobs/projection_accuracy_health.js` (health job: report new fields in output)

**Out of scope**:
- Rolling bias computation (WI-1020)
- Any new card type registration (WI-1019)
- UI changes
- Other sport models

**Acceptance**:
- `projection_accuracy_line_evals` table contains the following columns (new or confirmed existing):
  - `game_id TEXT` — FK to the game
  - `settled_at TEXT` — ISO timestamp of settlement
  - `snapshot_time TEXT` — timestamp of the odds snapshot the projection was built from
  - `market_total REAL` — market total line at snap time (from `raw_data.market_total`)
  - `raw_total REAL` — model projection before calibration (= `projection_raw`)
  - `calibrated_total REAL` — model projection after calibration correction is applied (null until WI-1020)
  - `actual_total REAL` — verified final score total (= `actual_value`)
  - `total_error_raw REAL` — `raw_total - actual_total` (computed at write, positive = over-projected)
  - `total_error_calibrated REAL` — `calibrated_total - actual_total` (null until WI-1020)
  - `pace_tier TEXT` — from `raw_data.pace_tier` (FAST×FAST, SLOW×SLOW, etc.)
  - `vol_env TEXT` — from `raw_data.vol_env` (LOW / MED / HIGH)
  - `total_band TEXT` — from `raw_data.total_band` (`<220`, `220-230`, `230-240`, `240+`)
  - `injury_cloud TEXT` — from `raw_data.injury_cloud` (NONE / MODERATE / SEVERE)
  - `driver_contributions_json TEXT` — JSON array from `raw_data.driver_contributions`
  - `regime_tags_json TEXT` — JSON array of regime tags (null until WI-1025)
  - `confidence_tier TEXT` — from `payloadData.tier` at settlement time
- All new columns are nullable with no DEFAULT constraint so existing rows are unaffected.
- Migration is additive only (no DROP, no NOT NULL without DEFAULT, no data transforms).
- Settlement write path in `projection-accuracy.js` populates the new columns when the source fields are present on the card payload.
- `job:projection-accuracy-health` reports per-band and per-vol_env breakdowns using new columns, logging a summary like `[HEALTH] NBA_TOTAL pace_tier=FAST×FAST n=12 bias=-1.2 rmse=14.3`.

**Implementation notes**:

Migration template:

```sql
ALTER TABLE projection_accuracy_line_evals ADD COLUMN snapshot_time TEXT;
ALTER TABLE projection_accuracy_line_evals ADD COLUMN market_total REAL;
ALTER TABLE projection_accuracy_line_evals ADD COLUMN raw_total REAL;
ALTER TABLE projection_accuracy_line_evals ADD COLUMN calibrated_total REAL;
ALTER TABLE projection_accuracy_line_evals ADD COLUMN actual_total REAL;
ALTER TABLE projection_accuracy_line_evals ADD COLUMN total_error_raw REAL;
ALTER TABLE projection_accuracy_line_evals ADD COLUMN total_error_calibrated REAL;
ALTER TABLE projection_accuracy_line_evals ADD COLUMN pace_tier TEXT;
ALTER TABLE projection_accuracy_line_evals ADD COLUMN vol_env TEXT;
ALTER TABLE projection_accuracy_line_evals ADD COLUMN total_band TEXT;
ALTER TABLE projection_accuracy_line_evals ADD COLUMN injury_cloud TEXT;
ALTER TABLE projection_accuracy_line_evals ADD COLUMN driver_contributions_json TEXT;
ALTER TABLE projection_accuracy_line_evals ADD COLUMN regime_tags_json TEXT;
ALTER TABLE projection_accuracy_line_evals ADD COLUMN confidence_tier TEXT;
```

At settlement write time, populate from card payload:

```js
snapshot_time: payload.raw_data?.captured_at ?? null,
market_total: payload.raw_data?.market_total ?? null,
raw_total: projection_raw,
actual_total: actual_value,
total_error_raw: (projection_raw != null && actual_value != null) ? projection_raw - actual_value : null,
pace_tier: payload.raw_data?.pace_tier ?? null,
vol_env: payload.raw_data?.vol_env ?? null,
total_band: payload.raw_data?.total_band ?? null,
injury_cloud: payload.raw_data?.injury_cloud ?? null,
driver_contributions_json: payload.raw_data?.driver_contributions
  ? JSON.stringify(payload.raw_data.driver_contributions)
  : null,
confidence_tier: payload.tier ?? null,
```
