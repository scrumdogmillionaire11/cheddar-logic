# WI-1247: Add POTD Suppression Diagnostics

**ID**: WI-1247
**Goal**: Make all “missing POTD” outcomes explainable by exposing suppression reasons across the POTD engine, read API, and admin health surfaces.

**Context**:

POTD can currently hide rows because of invalid-decision enforcement, nominee pruning, shadow-only outcomes, or no-pick paths, and some of those paths do not surface clearly to operators. This WI adds diagnostics only; it does not lower POTD standards.

**Scope**:
- `apps/worker/src/jobs/potd/run_potd_engine.js`
- `web/src/app/api/potd/route.ts`
- `web/src/lib/potd-server.ts`
- `web/src/app/api/admin/model-health/route.ts`
- `web/src/app/api/admin/pipeline-health/route.ts`
- `apps/worker/src/__tests__/potd-suppression-diagnostics.test.js` (new)
- `node web/src/__tests__/api-potd-suppression-diagnostics.test.js` (new command target)
- `WORK_QUEUE/WI-1247.md`

**Out of scope**:
- Any change to POTD selection thresholds or bankroll rules
- Any widening of POTD eligibility
- Any change to wedge/results visibility rules

**Acceptance**:
- POTD diagnostics expose counts for invalid decision suppression, nominee pruning, shadow-only, and no-pick outcomes.
- Missing POTD rows become explainable through API/admin diagnostics rather than silent omission.
- Regression coverage proves each suppression path is reported.
- Official POTD selection behavior remains unchanged.
