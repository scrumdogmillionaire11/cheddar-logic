# WI-0837: ESPN Neutral Metrics Alerting — Discord Warning When All Team Inputs Are Null

| Field | Value |
|---|---|
| **ID** | WI-0837 |
| **Goal** | When `getTeamMetricsWithGames()` returns `neutral()` (all-null metrics) for a team, emit a Discord alert and a structured `[ESPN_NULL]` log line. Currently the system silently caps the card at `PROJECTION_ONLY` but no operator is notified that ESPN data is wholly absent. Cascading ESPN failures (rate-limit, DNS, team-map miss) can affect all teams on a run with no observable signal. |
| **Scope** | `apps/worker/src/jobs/run_nba_model.js`, `apps/worker/src/jobs/run_nhl_model.js`, `packages/data/src/team-metrics.js` |
| **Out of scope** | Fixing the ESPN fetch itself, changing rate-limit behavior, adding circuit breaker (separate concerns), frontend changes |
| **Acceptance** | (1) When `neutral()` is returned for any team on a model run, `[ESPN_NULL] sport=NBA team=<name> reason=<reason>` is logged. (2) If `ENABLE_DISCORD_CARD_WEBHOOKS=true` and null metric count exceeds `ESPN_NULL_ALERT_THRESHOLD` (default: 2 teams per run), a Discord alert fires. (3) The alert is deduped within a 1-hour cooldown window (reuse pipeline health cooldown pattern from `check_pipeline_health.js`). (4) No behavioral change to card generation. |
| **Owner agent** | unassigned |
| **Time window** | unassigned |
| **Coordination flag** | none |

## Context

`neutral()` returns `{ avgPoints: null, avgPointsAllowed: null, ... }`. WI-0768 correctly caps execution_status to `PROJECTION_ONLY` when this happens. However, there is no alerting when the ESPN data layer silently fails wholesale (rate-limit, unmapped team, network partition). The first indication of failure is currently the Dr. Claire health report picking up a spike in `PROJECTION_ONLY` cards — which is too late for an operator to react during a live trading session.

## Implementation Notes

1. Add `nullMetricTeams: string[]` accumulator to each model run.
2. After ESPN fetch loop, if `nullMetricTeams.length >= ESPN_NULL_ALERT_THRESHOLD`, call the existing Discord webhook helper with a summary message.
3. Log `[ESPN_NULL]` once per affected team regardless of threshold.
4. Cooldown: track last-alerted timestamp in-memory (or a simple `job_runs`-based check) to suppress duplicate alerts within 60 minutes.
5. `ESPN_NULL_ALERT_THRESHOLD` env var, default `2`.
