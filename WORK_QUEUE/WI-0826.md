## Background

The system has a `ENABLE_CLV_LEDGER` flag (deployed per WI-0807) but CLV is not aggregated into actionable market-level dashboards. There is no automated daily report of:
- How many games were `MODEL_OK` vs `DEGRADED` vs `NO_BET`
- How many bets were placed vs blocked by execution gate
- CLV per market (edge at placement vs edge at close)
- Rolling ROI, hit rate, drawdown

This WI adds the DB layer and worker job to generate these reports, and exposes the data via a new API route in the web layer.

---

## Scope

**CLV Data Architecture — canonical write path:**
`clv_ledger` (from WI-0807, corrected by WI-0838) is the canonical **pick-side** source. It holds `odds_at_pick` (the opening price at card creation) written by `settle_pending_cards.js`. This WI does **not** replace that flow. `run_clv_snapshot.js` reads pick-side data from `clv_ledger` by `card_id`, fetches the closing odds snapshot, computes `clv = closing_implied_prob - implied_prob_at_pick`, and writes the closing-side + delta to `clv_entries`. The dashboard and `/api/performance` route read from `clv_entries`. This avoids dual-write: each table has a single owner.

**Files to create:**
- `apps/worker/src/jobs/run_daily_performance_report.js` — computes and writes daily performance report
- `apps/worker/src/jobs/run_clv_snapshot.js` — joins `clv_ledger` (pick-side, written by settle) with closing odds snapshot; writes CLV delta rows to `clv_entries` (runs after game end)

**Files to change:**
- `packages/data/db/migrations/065_performance_reports.sql` — new tables
- `apps/worker/src/schedulers/main.js` — register new jobs
- `web/src/app/api/performance/route.ts` — new API route returning performance report data

**Out of scope:**
- Visual chart rendering (raw data only from API)
- Per-book CLV breakdown
- Historical backfill before this WI ships

## DB Schema

```sql
-- Migration 065
CREATE TABLE IF NOT EXISTS daily_performance_reports (
  id INTEGER PRIMARY KEY AUTOINCREMENT,
  report_date TEXT NOT NULL,    -- YYYY-MM-DD
  market TEXT NOT NULL,
  sport TEXT NOT NULL,
  eligible_games INTEGER DEFAULT 0,
  model_ok_count INTEGER DEFAULT 0,
  degraded_count INTEGER DEFAULT 0,
  no_bet_count INTEGER DEFAULT 0,
  bets_placed INTEGER DEFAULT 0,
  bets_blocked_gate INTEGER DEFAULT 0,
  hit_rate REAL,
  roi REAL,
  avg_edge_at_placement REAL,
  avg_clv REAL,                 -- avg(edge_at_close - edge_at_placement)
  brier REAL,
  ece REAL,
  max_drawdown REAL,
  computed_at TEXT NOT NULL,
  UNIQUE(report_date, market, sport)
);

CREATE TABLE IF NOT EXISTS clv_entries (
  id INTEGER PRIMARY KEY AUTOINCREMENT,
  game_id TEXT NOT NULL,
  market TEXT NOT NULL,
  side TEXT NOT NULL,
  edge_at_placement REAL,
  fair_prob_at_placement REAL,
  implied_prob_at_placement REAL,
  closing_price REAL,
  closing_implied_prob REAL,
  clv REAL,                     -- closing_implied_prob - implied_prob_at_placement
  clv_positive INTEGER,         -- 1 if CLV > 0
  outcome INTEGER,              -- 1 win, 0 loss, NULL pending
  created_at TEXT NOT NULL
);
CREATE INDEX IF NOT EXISTS idx_clv_market ON clv_entries(market, created_at);
```

## Firing vs Winning Metrics

The daily report tracks these separately to distinguish model health from execution health:

**Firing metrics** (did the model generate signals?):
- `eligible_games` — games with odds snapshots
- `model_ok_count` — games where model returned MODEL_OK
- `no_bet_count` — games blocked at input gate
- `bets_placed` vs `bets_blocked_gate`

**Winning metrics** (did signals lead to value?):
- `avg_clv` — positive = beating close
- `roi` — realized P&L per unit staked
- `hit_rate` — win rate on placed bets
- `brier` / `ece` — calibration quality
- `max_drawdown` — worst run-length loss

## CLV Computation

```js
/**
 * CLV = (closing_implied_prob) - (implied_prob_at_placement)
 * Positive CLV = you got better numbers than the closing market.
 *
 * Both probabilities should be vig-free (no-vig implied).
 */
function computeCLV(impliedProbAtPlacement, closingImpliedProb) {
  if (impliedProbAtPlacement === null || closingImpliedProb === null) return null;
  return Number((closingImpliedProb - impliedProbAtPlacement).toFixed(4));
}
```

CLV target: consistently positive CLV (≥ 0.005 per bet avg) is the primary signal of a real edge. ROI alone is too noisy on small samples.
