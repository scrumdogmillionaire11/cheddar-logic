## Problem

`player_availability` is populated by `sync_nhl_player_availability.js` for NHL and is actively used by `run_nhl_player_shots_model.js` (lines 1642, 1686) as a participation gate. No equivalent exists for NBA.

The NBA model never queries `player_availability`. A game where LeBron James, Giannis, or any other impact player is OUT has the exact same card generation path as a full-roster game. There is no downgrade, no `missing_inputs`, and no pass reason.

**Contract violation:** The NHL pattern proves the infrastructure is correct. The NBA simply never copied it. The cost is real: cards can fire with materially incorrect implied totals because key-player absence isn't factored.

---

## Input proof

`player_availability` table schema (from NHL usage):
```
sport TEXT, player_id TEXT, player_name TEXT, team_id TEXT,
status TEXT (OUT | GTD | DTD | ACTIVE), updated_at TEXT
```

ESPN API provides NBA injury reports at `http://site.api.espn.com/apis/site/v2/sports/basketball/nba/injuries` â€” same pattern used by `sync_nhl_player_availability.js` for NHL.

## Fix spec

**Step 1 â€” Build `sync_nba_player_availability.js`:**
- Mirror `sync_nhl_player_availability.js` logic
- Source: ESPN NBA injuries endpoint
- Upsert into `player_availability` with `sport='nba'`
- Schedule: hourly (same as NHL sync)

**Step 2 â€” Wire into NBA model:**
- In `run_nba_model.js` game loop, query `player_availability` for home and away rosters
- Identify "impact players" by market value proxy (minutes average or salary tier â€” use a configurable threshold, initially: any player averaging > 25 min/game)
- If an impact player is `OUT`: emit `missing_inputs: ['key_player_out']`, cap classification at `LEAN`
- If an impact player is `GTD`/`DTD`: emit `missing_inputs: ['key_player_uncertain']`, flag in `pricing_context`

**Output proof:** Card payload includes `availability_flags: [{player, status}]` in `raw_data`; classification is correctly capped.

**Failure proof:** If `player_availability` sync has not run (no rows for today's game teams), emit `missing_inputs: ['nba_availability_unresolved']` â€” same conservative pattern as NHL goalie unknown.
