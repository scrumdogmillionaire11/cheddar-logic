# WI-0831: Fit and Apply Calibration Function — Isotonic Regression per Market

| Field | Value |
|---|---|
| **ID** | WI-0831 |
| **Goal** | WI-0825 tracks Brier/ECE and can suppress cards. This WI goes one step further: fit a per-market isotonic regression calibrator on the model's historical `fair_prob` outputs vs resolved outcomes, and apply the calibration transform at inference time so that the `fair_prob` written to card payloads is the *corrected* probability, not the raw model output. |
| **Priority** | High |
| **Depends on** | WI-0825 (calibration metrics infrastructure and `model_calibration` DB table with decile buckets and outcome counts); sufficient resolved history (≥100 bets per market recommended) |

## Background

WI-0825 detects miscalibration and can kill a market via ECE threshold. That is a safety valve, not a correction. Every card written before the kill switch fires uses an inflated or deflated `fair_prob` directly, which then flows into:

- `net_edge = fair_prob - implied_prob` (execution gate in WI-0824)
- `kelly_fraction = (b × fair_prob - (1 - fair_prob)) / b` (WI-0819)
- `risk_analysis.ipynb` Monte Carlo inputs

If the model systematically outputs 0.58 when the true probability is 0.54, every downstream calculation is wrong. The Kelly fraction is ~40% overstated at those numbers.

Isotonic regression is appropriate here because:
- It is monotone (preserves ordering, never inverts predictions)
- It is non-parametric (no shape assumption)
- It can be fit with 100–500 samples per market
- It is interpretable (piecewise constant mapping, auditable)

## Scope

**Files to create:**
- `apps/worker/src/jobs/fit_calibration_models.js` — reads `model_calibration` table from WI-0825, fits isotonic regression per `(sport, market_type)` using numeric approximation (no scipy dependency), serialises as JSON breakpoints, writes to `calibration_models` table

**Files to change:**
- `apps/worker/src/utils/calibration.js` (create if not exists) — `applyCalibration(rawProb, calibrationBreakpoints)` — linear interpolation on saved breakpoints
- Card write path (sport-specific job files) — wrap `fair_prob` through `applyCalibration()` before writing card payload; fall back to raw `fair_prob` if no calibration exists for that market
- Scheduler (`main.js`) — register `fit_calibration_models` to run once daily (e.g. 06:00 ET, before model runs)

**DB migrations:**
- `packages/data/db/migrations/063_calibration_models.sql` — `calibration_models(sport TEXT, market_type TEXT, fitted_at TEXT, breakpoints_json TEXT, n_samples INTEGER, isotonic_brier REAL)`; one row per `(sport, market_type)`, upsert on conflict

**Out of scope:**
- Platt scaling (logistic) — isotonic is sufficient and safer for small samples
- Cross-sport calibration — each market type is calibrated independently
- UI display of calibrated vs raw probability
- Re-running `risk_analysis.ipynb` — that is WI-0834

### Fallback behaviour

If no calibration exists for a market (first season, insufficient samples), `applyCalibration()` returns the raw `fair_prob` unchanged. A `calibration_source: 'raw'` field is written to the card payload for monitoring.
