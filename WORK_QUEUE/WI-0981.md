# WI-0981: Add release evidence manifest and deployment audit trail

**ID**: WI-0981
**Goal**: Produce a machine-readable release manifest for every production deploy containing SHA, artifact, checks, and verification evidence.

**Depends on**: WI-0974

**Scope**:
- `.github/workflows/deploy-production.yml`
- `scripts/deployment-verify.js`
- `docs/processes/release-audit-trail.md`
- `docs/runbooks/production-release-verification.md`

**Out of scope**:
- Notification provider migration
- Historical backfill of old deploy records

**Acceptance**:
- Every production deploy emits a release manifest artifact with intended SHA, runtime SHA, artifact digest, check results, and approval metadata.
- Manifest generation is mandatory for deploy success.
- Runbook documents where release evidence is stored and how to retrieve it.

**Owner agent**: unassigned
**Time window**: 2026-04-16 -> TBD
**Coordination flag**: needs-sync

**Tests to run**:
- `npm --prefix web run lint`
- `npm --prefix web run test:ci:smoke`

**Manual validation**:
- Execute a test deploy and confirm manifest contains complete evidence fields.
- Verify release evidence can be traced from workflow run to deployed SHA.
