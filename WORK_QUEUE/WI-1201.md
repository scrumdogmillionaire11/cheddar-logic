# WI-1201: Wire Discord to DecisionOutcome

**ID**: WI-1201
**Goal**: Convert Discord formatter to a pure consumer of `DecisionOutcome`, ensuring Discord output counts and status match web exactly, eliminating drift.

**Scope**:
- `apps/worker/src/utils/discord-formatter.js` (refactor to accept DecisionOutcome only; no mapping logic)
- `apps/worker/src/jobs/publish_discord_decisions.js` (import canonical builder from @cheddar-logic/data, apply to decision_v2 list, pass outcomes to formatter)
- `apps/worker/src/jobs/__tests__/publish_discord_decisions.test.js` (add snapshot tests: same count/status as web)
- `apps/worker/src/utils/__tests__/discord-formatter.test.js` (new/refactored: test DecisionOutcome input only)

**Out of scope**:
- Implementing Discord-specific decision mappers (use canonical builder from WI-1199 only)
- Changing worker decision_v2 production
- Deleting old formatters (WI-1203)
- Webhook payload contract changes

**Acceptance**:
- `discord-formatter` accepts `DecisionOutcome` (or list thereof) and produces Discord message with zero reinterpretation
- `publish_discord_decisions.js` calls `buildDecisionOutcomeFromDecisionV2()` from canonical builder (imported from @cheddar-logic/data); does NOT implement its own mapper
- Formatter produces zero-reinterpretation output: if DecisionOutcome.status='PLAY', message says "🎯 PLAY", if 'PASS', message says "PASS"
- Snapshot test: Discord message for same 20 fixture decisions matches web's card counts (e.g., "6 plays, 12 slight edges" in Discord = same as /api/cards)
- If DecisionOutcome has `status='PLAY'` and `reasons.blockers`, Discord shows the blockers
- All existing Discord tests pass
- No Discord-specific mapper code exists (dead code removal only; do not delete files yet)
