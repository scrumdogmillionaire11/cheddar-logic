# WI-1034-c: Implement MLB verification contract and watchdog enforcement

**ID**: WI-1034-c

**Implementation requirements**:

- Ensure MLB candidates include structured `CandidateVerification` payload:
  - `verification_state = NOT_REQUIRED | PENDING | CLEARED | FAILED | EXPIRED`
  - `requirements[]` entries include `blocker_code`, `severity`, `status`, `unblock_condition`, `action_type`, `retry_policy`
  - `next_action_summary` required when `verification_state=PENDING`
- Preserve first-fail user-facing gate output (`PASS - [REASON_CODE]: ...`) while collecting all cheap-to-detect blockers internally in the same pass.
- Enforce verdict semantics:
  - `LEAN + verification_state=PENDING` = verification-blocked candidate
  - `LEAN + verification_state=CLEARED|NOT_REQUIRED` = true Slight Edge lean
  - `LEAN` without `verification_state` is invalid payload
- Enforce resolver terminal-state behavior:
  - `CLEARED` marks eligibility for re-evaluation only (must not auto-emit or force `PLAY`)
  - `FAILED` emits `PASS - [blocker_code]: [failure_reason].`
  - `EXPIRED` emits `PASS - EXPIRED: Verification window closed without resolution.` and stops further processing
- Implement watchdog contract enforcement for invalid pending-verification payloads:
  - `verification_state=PENDING` with empty `requirements` -> contract violation
  - `verification_state=PENDING` with missing/empty `next_action_summary` -> contract violation
  - stale `PENDING` past expiry still surfaced as actionable -> contract violation

**Acceptance**:

- MLB candidate payloads include `verification_state` and structured `requirements` whenever verification applies.
- Any `verification_state=PENDING` candidate with empty `requirements` is rejected/flagged by watchdog enforcement.
- Any `verification_state=PENDING` candidate with empty `next_action_summary` is rejected/flagged by watchdog enforcement.
- `CLEARED` path requires re-evaluation and does not auto-promote to `PLAY`.
- `EXPIRED` path emits `PASS - EXPIRED: Verification window closed without resolution.` and exits further processing.
- Existing MLB tests pass; add/adjust tests to cover:
  - `LEAN + verification_state=PENDING`
  - `LEAN + verification_state=CLEARED|NOT_REQUIRED`
  - missing `verification_state` with `LEAN` is invalid
  - `PENDING` with empty `requirements` contract violation
  - `PENDING` with empty `next_action_summary` contract violation
  - `CLEARED != PLAY`
  - `EXPIRED` emit and stop behavior
