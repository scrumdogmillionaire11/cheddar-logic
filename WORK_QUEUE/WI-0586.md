# WI-0586: NHL Blocked Shots prop — full pipeline (data ingest → model runner → display)

**ID**: WI-0586
**Goal**: Wire the `projectBlkV1` model into an end-to-end pipeline so NHL blocked-shot prop cards appear on the /cards props tab, backed by real per-game logs and EV/PK rate splits.

**Background**:
The ingest leg already exists: `pull_nhl_player_shots_props.js` fetches `player_blocked_shots` O/U lines from the Odds API and stores them in `player_prop_lines` with `propType = 'blocked_shots'`. The model function `projectBlkV1` also exists in `apps/worker/src/models/nhl-player-shots.js` with a fully-specified input contract. What's missing is the data that feeds the model (per-game BLK logs + EV/PK rate splits) and the runner wiring that turns those inputs into cards.

**What `projectBlkV1` needs (and where it will come from)**:

| Input field | Source |
|---|---|
| `ev_blocks_season_per60` | NST ingest (new) |
| `ev_blocks_l10_per60` | NST ingest (new) |
| `ev_blocks_l5_per60` | NST ingest (new) |
| `pk_blocks_season_per60` | NST ingest (new) |
| `pk_blocks_l10_per60` | NST ingest (new) |
| `pk_blocks_l5_per60` | NST ingest (new) |
| `toi_proj_ev` | Existing `player_shot_logs.raw_data` (season avg EV TOI) or NHL API |
| `toi_proj_pk` | NST ingest (new) |
| `opponent_attempt_factor` | `team_metrics_cache` (existing, column `attempts_for_rate`) |
| `defensive_zone_factor` | `team_metrics_cache` (existing) |
| `underdog_script_factor` | Existing game-context logic in model runner |
| `playoff_tightening_factor` | Derived from game date (existing pattern) |
| `role_stability` | `player_availability` (existing) |
| `market_line`, `market_price_over/under` | `player_prop_lines` (existing — `propType = 'blocked_shots'`) |
