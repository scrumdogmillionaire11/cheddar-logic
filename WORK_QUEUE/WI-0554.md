# WI-0554: Replace hardcoded confidence values with computed function

**ID**: WI-0554
**Goal**: Replace the hardcoded `confidence: 0.95/0.88/0.85` literals in edge-calculator and the `CONFIDENCE_MAP` in run_nba_model with a computed function that adjusts confidence based on watchdog status, missing fields, proxy usage, and conflict score.

**Scope**:
- `packages/models/src/edge-calculator.js`
- `apps/worker/src/jobs/run_nba_model.js`
- `packages/models/src/decision-pipeline-v2.js` (if confidence is also set there)

**Out of scope**:
- Changing decision threshold values (play_edge_min, lean_edge_min) — confidence is a separate signal
- Consumer UI changes to display confidence
- Confidence calibration via historical outcomes (future; this WI establishes the mechanism)

**Background**:
Three confidence values are hardcoded in edge-calculator.js: moneyline=0.95, total=0.88, spread=0.85. These never change regardless of watchdog status, missing drivers, proxy usage, or conflicting signals. In run_nba_model.js, `CONFIDENCE_MAP` maps `FIRE → 0.74` and `WATCH → 0.61` regardless of edge magnitude — a FIRE card with 2.1% edge and one with 9% edge both emit `confidence: 0.74`. Consumers using confidence for sizing get a number that carries no real information.

**Acceptance**:
- A `computeConfidence({ baseConfidence, watchdogStatus, missingFieldCount, proxyUsed, conflictScore })` function is exported from `edge-calculator.js` or a shared utility.
- Adjustment rules (minimum viable set):
  - `watchdogStatus === 'CAUTION'` → `-0.10`
  - `proxyUsed === true` → `-0.15`
  - Each missing required field → `-0.05` (capped at `-0.20` total for missing fields)
  - `conflictScore > 0.3` → `-0.10`
  - Result is floored at `0.30` and capped at the base.
- `computeMoneylineEdge`, `computeSpreadEdge`, `computeTotalEdge` all call `computeConfidence` with available context rather than returning a hardcoded literal.
- `CONFIDENCE_MAP` in `run_nba_model.js` is removed. Call card confidence is derived from edge_pct: e.g., `Math.min(0.50 + edge_pct * 3, 0.90)` or delegates to the same `computeConfidence` function.
- Unit tests confirm confidence decreases appropriately for degraded inputs (proxy, missing fields, watchdog CAUTION).
