# WI-1250: Add Scheduled-Game Coverage Watchdog To Pipeline Health

**ID**: WI-1250
**Goal**: Detect scheduled games that never produce any `card_payloads` row so upstream coverage failures fail in `pipeline_health` instead of slipping past display-log integrity checks.

**Context**:

`checkVisibilityIntegrity` only sees cards that were written to `card_payloads` but never enrolled in `card_display_log`. It cannot detect the two upstream gaps now visible in production evidence: MLB games that only produced F5 analysis and never emitted full-game rows, and NBA games that ran but never produced a publishable card row for the scheduled game at all.

**Scope**:
- `apps/worker/src/jobs/check_pipeline_health.js`
- `apps/worker/src/__tests__/check-pipeline-health.mlb.test.js`
- `apps/worker/src/__tests__/check-pipeline-health.nba.test.js`
- `WORK_QUEUE/WI-1250.md`

**Out of scope**:
- Fixing the underlying MLB full-game generation gap
- Fixing NBA zero-signal model behavior
- Changing admin UI rendering beyond what existing `pipeline_health` rows already expose

**Acceptance**:
- A new game-coverage health check compares scheduled game IDs in the near-term watchdog window against game IDs that produced at least one `card_payloads` row.
- Coverage is evaluated per sport with explicit thresholds, with initial defaults covering the observed failure modes: MLB below `50%` and NBA below `66%` must fail.
- Failed coverage writes a `pipeline_health` row with sport, coverage counts, and sample game IDs using the same degraded-state and alert flow as other health checks.
- Regression coverage exercises both an MLB missing-full-game-row scenario and an NBA zero-card-row scenario.
