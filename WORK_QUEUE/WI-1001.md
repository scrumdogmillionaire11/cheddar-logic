# WI-1001: Centralize reason-code registry into shared package export

**ID**: WI-1001

**Goal**: Consolidate the scattered reason-code sets, maps, and label definitions into a single authoritative export in `packages/data/src/` and replace all inline duplicates with imports from that canonical source.

**Priority**: Medium — silent drift between parallel definitions is the main risk; currently not causing bugs but will diverge under future blocker additions.

**Owner agent**: Unassigned

**Time window**: TBD

**Coordination flag**: `needs-sync` — touches public API surfaces in `packages/data/src/`; all consumers must update imports in the same PR.

---

## Background

The same blocker semantics are defined independently in at least four places:

| File | What it defines |
| ---- | --------------- |
| `packages/models/src/decision-pipeline-v2.js` | Inline blocker-code sets used in pipeline gating |
| `packages/models/src/decision-policy.js` | Separate inline sets for policy evaluation |
| `web/src/lib/game-card/reason-labels.ts` | `REASON_CODE_LABELS` map of code → display text |
| `web/src/lib/game-card/transform/decision-surface.ts` | Local sets cross-checked at transform time |

Any new blocker code added to one location must be manually echoed to all others. There is no lint or type check enforcing parity.

## Proposed registry shape

```ts
// packages/data/src/reason-codes.ts (new file)
export const BLOCKER_REASON_CODES = [
  'SUPPORT_BELOW_LEAN_THRESHOLD',
  'LINE_NOT_CONFIRMED',
  'EDGE_RECHECK_PENDING',
  'EDGE_NO_LONGER_CONFIRMED',
  'MARKET_DATA_STALE',
  'PRICE_SYNC_PENDING',
  // ...full list
] as const;

export type BlockerReasonCode = typeof BLOCKER_REASON_CODES[number];

export const REASON_CODE_LABELS: Record<BlockerReasonCode, string> = {
  LINE_NOT_CONFIRMED: 'Line not confirmed',
  // ...
};
```

JS-only consumers in `packages/models/` and `apps/worker/` import from the same source without the `as const` type annotation.

## Scope

| File | Change |
| ---- | ------ |
| `packages/data/src/reason-codes.ts` | **New** — canonical blocker code list, `BlockerReasonCode` type, `REASON_CODE_LABELS` map |
| `packages/data/src/index.ts` | Re-export new module |
| `packages/models/src/decision-pipeline-v2.js` | Replace inline blocker sets with imports from `@cheddar/data` |
| `packages/models/src/decision-policy.js` | Replace inline blocker sets with imports |
| `web/src/lib/game-card/reason-labels.ts` | Replace local map with import from `@cheddar/data`; keep file as thin re-export if needed by existing UI imports |
