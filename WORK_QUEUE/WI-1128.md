# WI-1128: Replace Source-String Contract Tests With Behavioral Tests

**ID**: WI-1128
**Goal**: Replace brittle source-token assertions with runtime behavioral tests that execute handlers/contracts using deterministic fixtures.

**Scope**:
- web/src/__tests__/api-admin-model-health.test.js
- web/src/__tests__/api-potd.test.js
- web/src/__tests__/ui-results-smoke.test.js
- web/src/__tests__/db-test-runtime.js
- web/src/__tests__/integration/

**Out of scope**:
- End-to-end browser test stack adoption
- Worker-side Jest suite changes
- Route implementation changes unrelated to making tests behavioral and deterministic

**Acceptance**:
- Primary assertions in scoped tests are behavioral/runtime assertions against response status, payload shape, headers, and deterministic fixture outcomes.
- Source-token assertions using string scanning (for example routeSource/pageSource/serverSource includes checks) are removed from primary pass criteria.
- Any remaining static-policy assertions are narrowly scoped to explicit non-behavioral invariants and are supplemental (cannot be the reason the test suite passes when runtime assertions fail).
- CI path for scoped tests is deterministic and does not rely on live localhost availability.
- Fallback-to-source-contract pass paths are removed for scoped tests; if runtime preconditions are unavailable, tests fail with actionable setup messaging instead of silently passing via source checks.
