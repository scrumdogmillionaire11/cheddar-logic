# WI-1122: POTD Edge Integrity and Model-Backed Eligibility Enforcement

**ID**: WI-1122
**Goal**: Ensure POTD edge uses `edgePct = modelProb - impliedProb`, enforce model-backed candidate eligibility, and prevent null/non-computable edges from reaching POTD UI surfaces.

**Scope**:

- `apps/worker/src/jobs/potd/signal-engine.js`
- `apps/worker/src/jobs/potd/run_potd_engine.js`
- `apps/worker/src/jobs/potd/__tests__/signal-engine.test.js`
- `apps/worker/src/jobs/potd/__tests__/run-potd-engine.test.js`
- `web/src/lib/potd-server.ts`

**Out of scope**:

- POTD page visual redesign or layout changes
- Non-POTD game-card transform logic
- DB schema migrations

**Acceptance**:

- Edge computation is explicitly derived as `modelProb - impliedProb` for scored POTD candidates.
- Candidate-level audit logs include `modelProb`, `impliedProb`, `edgePct`, and source class (`MODEL` or `CONSENSUS`).
- Candidates missing required pricing/model probability inputs are rejected before POTD nomination/selection.
- Only model-backed (`MODEL`) candidates are eligible for POTD evaluation and ranking.
- POTD nominees returned to UI exclude entries with missing/invalid edge values.
