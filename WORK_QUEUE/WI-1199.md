# WI-1199: Define DecisionOutcome Contract

**ID**: WI-1199
**Goal**: Establish the canonical `DecisionOutcome` type and validator so all downstream systems (web, Discord, POTD) can consume a single, deterministic decision object with no recomputation or reinterpretation.

**Scope**:
- `packages/data/src/decision-outcome.ts` (update shared type contract and declarations to match canonical runtime behavior)
- `packages/data/src/decision-outcome.js` (runtime export surface + validator wiring)
- `packages/data/src/decision-outcome.builders.js` (canonical builders: buildDecisionOutcomeFromDecisionV2, normalizeStatus, mapReasonsToOutcome)
- `packages/data/src/validators/decision-outcome.js` (strict validator for canonical contract)
- `packages/data/__tests__/decision-outcome.test.js` (validator + builder + fixture + determinism tests)
- `packages/data/src/index.ts` (export DecisionOutcome type + builder/validator declarations)
- `packages/data/src/index.js` (export runtime DecisionOutcome builders/validator)

**Out of scope**:
- Consuming existing decision_v2 code in web/Discord/POTD surfaces (that's WI-1200+)
- Changing web/Discord/POTD presentation behavior (done in later WIs)
- Deleting legacy decision mappers (WI-1203)

**Acceptance**:
- `DecisionOutcome` type is defined in shared package with clear schema:
  - `status`: 'PLAY' | 'SLIGHT_EDGE' | 'PASS'
  - `selection`: { market, side, line?, price? }
  - `edge: number | null`
  - `confidence: number | null`
  - `reasons: { pass?: string[], blockers?: string[], warnings?: string[] }`
  - `verification: { line_verified: boolean, data_fresh: boolean, inputs_complete: boolean }`
  - `source: { model: string, timestamp: string }`
- **Canonical builder functions exist and are exported:**
  - `buildDecisionOutcomeFromDecisionV2(decisionV2, metadata): DecisionOutcome` — single source of truth for mapping
  - `normalizeDecisionOutcomeStatus(status): 'PLAY' | 'SLIGHT_EDGE' | 'PASS'` — standardizes status naming
  - `mapReasonsToOutcome(decisionV2): { pass?, blockers?, warnings? }` — canonical reason extraction
- Validator function `validateDecisionOutcome()` returns `{ valid: boolean, errors?: string[] }` and rejects malformed objects
- At least 15 fixture tests covering:
  - builder behavior: decision_v2 with PLAY thresholds -> DecisionOutcome.status='PLAY'
  - builder behavior: decision_v2 with SLIGHT_EDGE thresholds -> DecisionOutcome.status='SLIGHT_EDGE'
  - builder behavior: decision_v2 with PASS blockers -> DecisionOutcome.reasons.blockers populated
  - validator: rejects status not in enum, missing required fields
  - reason mapping: all decision_v2 blockers correctly extracted to reasons
- All builders and types exported from `@cheddar-logic/data` so web/worker can import them
- No breaking changes to existing decision_v2 imports (both coexist)
- TypeScript strict mode passes with no `any` casts on new/updated declarations
- **Critical acceptance gate: no consumer WI (1200/1201/1202/1203) may implement its own mapper; all must import canonical builder from here**
- **Purity guarantee: `buildDecisionOutcomeFromDecisionV2()` is pure, deterministic, and side-effect free; same input always returns identical DecisionOutcome**
- **Cross-consumer determinism gate: using the same decision_v2 fixture, web, worker, and package imports produce byte-identical DecisionOutcome JSON**
- If any legacy/internal status token (for example `INVALID`) exists in type declarations, it must never be emitted by canonical builder output and validator must reject it
