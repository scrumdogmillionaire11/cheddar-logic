## **WI-1105: Analytics Schema Lift — Materialize Projection Accuracy Core Fields (Hardened)**

**ID**: WI-1105
**Goal**: Eliminate JSON-dependent projection accuracy reads by materializing a deterministic, immutable core field set required for grading, confidence scoring, and market trust evaluation.

---

## **Core Design Requirements (NON-NEGOTIABLE)**

### 1. Immutable Projection Capture

The following fields must be **materialized and immutable after insert**:

* `projection_raw`
* `synthetic_line`
* `synthetic_rule`
* `synthetic_direction`

**Invariant:**

* These fields must NEVER be recomputed or overwritten after initial insert.
* Any mismatch between recomputed and stored values must raise a failure flag.

---

### 2. Correct Synthetic Line Definition (FIXED)

Synthetic line must use **true nearest half-point**:

```js
synthetic_line = Math.round(projection_raw * 2) / 2
```

**NOT allowed:**

* `Math.round(projection_raw - 0.5) + 0.5`

**Invariant:**

* `projection_raw === synthetic_line` → `synthetic_direction = NO_EDGE`

---

### 3. Direction Strength Classification

Materialize:

* `edge_distance = abs(projection_raw - synthetic_line)`
* `direction_strength = WEAK | STRONG`

Rule:

* `< 0.15` → `WEAK` → must set flag `DIRECTION_TOO_WEAK`
* `>= 0.15` → `STRONG`

**Invariant:**

* WEAK rows must be excluded from directional W/L aggregates but included in error metrics.

---

### 4. Grading Outputs (Materialized)

Materialize:

* `actual`
* `graded_result` (`WIN | LOSS | PUSH`)
* `abs_error`
* `signed_error`

These must be written only during settlement.

---

### 5. Confidence Fields (REQUIRED ADDITION)

Materialize:

* `projection_confidence` (0–100 int)
* `confidence_band` (`LOW | WATCH | TRUST | STRONG`)

Confidence must be deterministic and include:

* edge distance
* historical bucket hit rate
* variance penalty
* **sample size weighting (NEW REQUIREMENT)**

**Invariant:**

* Same input row must always produce same confidence score.

---

### 6. Market Trust Surface (REQUIRED ADDITION)

Materialize or derive at query layer:

* `market_trust_status`:

  * `INSUFFICIENT_DATA`
  * `NOISE`
  * `WATCH`
  * `TRUSTED`
  * `SHARP`

**Required hard gate:**

If:

```text
win_rate(STRONG) <= win_rate(WEAK or WATCH)
```

→ must classify as `NOISE`

---

### 7. Multi-Line Evaluation Integrity

Ensure `projection_accuracy_line_evals`:

* is keyed by `(card_id, line)`
* is written once at grading
* is never recomputed

Lines must be fixed per market:

* MLB F5: `3.5, 4.5, 5.5`
* Pitcher Ks: `4.5, 5.5, 6.5, 7.5`
* NHL Shots: `1.5, 2.5, 3.5, 4.5`
* NHL Blocks: `0.5, 1.5, 2.5, 3.5`

---

### 8. JSON Fallback Rules (STRICT)

* Fallback is **read-only compatibility only**
* If JSON fallback is used:

  * must tag row with `FALLBACK_USED`
* New rows must NEVER depend on JSON path

---

## **Migration Requirements**

Migration must:

1. Create or extend table with:

* `projection_raw`
* `synthetic_line`
* `synthetic_rule`
* `synthetic_direction`
* `edge_distance`
* `direction_strength`
* `actual`
* `graded_result`
* `abs_error`
* `signed_error`
* `projection_confidence`
* `confidence_band`

2. Add indexes on:

* `(market_type, status)`
* `(confidence_band)`
* `(direction_strength)`

3. Enforce immutability:

* either via application guard or DB constraint trigger
