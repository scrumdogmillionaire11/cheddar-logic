# WI-0973: Build walk-forward validation and preseason shadow gate for NHL 1P

**ID**: WI-0973
**Goal**: Establish a walk-forward validation harness and shadow-release gate to approve NHL 1P model rollout for next season.

**Depends on**: WI-0972 (bucket thresholds live); WI-0969 (baseline bucket labels + hit-rate targets)

**Scope**:
- `apps/worker/src/audit/run_model_audit.js`
- `apps/worker/src/audit/scorecard.js`
- `apps/worker/src/jobs/report_telemetry_calibration.js`
- `apps/worker/src/jobs/__tests__/report_telemetry_calibration.test.js`
- `apps/worker/src/jobs/__tests__/run_nhl_model.test.js` _(regression guard only — no changes to run_nhl_model.js; guards WI-0505 Phase-2 gate and WI-0970 training exclusion contracts)_

**Out of scope**:
- Production deployment automation changes
- Discord or notification format changes
- Non-NHL shadow workflows

**Acceptance**:
- Walk-forward evaluation reports NHL 1P metrics by fixed bucket and side (OVER/UNDER) over sequential time windows (14-day non-overlapping windows; minimum 10 samples per window to report; at least 3 windows required for a promote verdict).
- Shadow gate defines explicit promote/hold criteria versus WI-0969 baseline: OVER hit-rate in `2.0-2.19` ≥ baseline; OVER hit-rate in `2.20+` ≥ baseline; any bucket with <10 samples is flagged but does not block promote.
- Scorecard output flags regression if OVER performance in `2.0-2.19` or `2.20+` fails target threshold.
- Documentation-level criteria are reflected in executable checks in audit scripts.

**Owner agent**: unassigned
**Time window**: 2026-05-20 -> 2026-05-31
**Coordination flag**: none _(removed needs-sync: no lockfiles, migrations, or public API surfaces in scope)_

**Tests to run**:
- `npm --prefix apps/worker run audit:nhl`
- `npm --prefix apps/worker run audit:scorecard`
- `npm --prefix apps/worker run test -- src/jobs/__tests__/report_telemetry_calibration.test.js --runInBand`

**Manual validation**:
- Run shadow evaluation on a recent NHL sample and confirm gate verdict includes baseline comparison, bucket-level side metrics, and clear promote/hold decision.
- Verify output explicitly reports fixed bucket names exactly as defined in WI-0969.
