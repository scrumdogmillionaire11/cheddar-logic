# WI-1243: Enforce Fail-Closed Job Health for Critical Pipeline Breaks

**ID**: WI-1243
**Goal**: Make worker job health reflect real operational failures by promoting critical partial-success conditions into degraded or failed job outcomes.

**Context**:

Several worker jobs currently continue past unmatched settlements, partial writes, or watchdog failures and still mark the run successful. That leaves the system operationally fail-open and makes stale or incomplete surfaces look healthy. This WI tightens health semantics without changing model thresholds.

Dependency rationale:
- WI-1250 also edits `apps/worker/src/jobs/check_pipeline_health.js`; this WI is explicitly serialized after WI-1250 to avoid parallel file ownership conflicts and merge ambiguity.
- WI-1242 remains a prerequisite so monitoring/readiness consumers receive aligned fail-closed health semantics across web readiness and worker pipeline health.

**Scope**:
- `apps/worker/src/jobs/check_pipeline_health.js`
- `packages/data/src/job-runtime.js`
- `apps/worker/src/jobs/settle_game_results.js`
- `apps/worker/src/jobs/settle_pending_cards.js`
- `apps/worker/src/__tests__/check-pipeline-health.fail-closed.test.js` (new)
- `apps/worker/src/__tests__/settlement-partial-failure-health.test.js` (new)
- `WORK_QUEUE/WI-1243.md`

**Out of scope**:
- Any change to betting decision thresholds or promotion rules
- Any change to readiness route behavior
- Any change to web read-path dedupe or visibility rules

**Acceptance**:
- Critical watchdog failures no longer preserve a fully successful job status.
- Partial settlement failures and unresolved critical mismatches can surface as degraded or failed job outcomes.
- Job status/exit code semantics reflect real operational health for critical failure classes.
- Regression coverage proves forced partial failures and watchdog breaches no longer pass silently.
- Critical failure class to outcome mapping is deterministic:
  - `WATCHDOG_CRITICAL_BREACH`: job status is `degraded` or `failed`; process exit code is non-zero.
  - `SETTLEMENT_PARTIAL_WRITE_CRITICAL`: job status is `degraded` or `failed`; process exit code is non-zero.
  - `SETTLEMENT_CRITICAL_MISMATCH_UNRESOLVED`: job status is `degraded` or `failed`; process exit code is non-zero.
- Non-critical informational warnings remain non-fatal:
  - `WATCHDOG_INFO` and `SETTLEMENT_INFO` classes do not force degraded/failed status and keep process exit code at zero.

- Force a partial settlement failure and confirm the run no longer appears fully successful.
- Force a non-critical watchdog/info warning and confirm the run remains successful with zero exit.

**Guard for WI closeout**:
- Do not convert non-critical informational warnings into hard failures without explicit classification.
- Keep the change focused on operational truthfulness, not business-rule tuning.
