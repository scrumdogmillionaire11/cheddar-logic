# WI-0821: MLB Offense Stack Collapse — Replace Four-Term Multiplicative Chain with Two-Term Composite

| Field | Value |
|---|---|
| **ID** | WI-0821 |
| **Goal** | Remove `ISO` and `hard_hit_pct` from the F5 run projection formula. Replace the four-term multiplicative offense adjustment with a single weighted composite of `wrc_plus_vs_hand` and `xwoba_vs_hand`, applied once to `adjustedRa9`. Apply one globally clamped output transform instead of sequential multipliers. |
| **Priority** | High |
| **Depends on** | WI-0820 (input gate must land first so NO_BET path is clean before changing projection math) |

## Background

`projectTeamF5RunsAgainstStarter` currently applies four correlated offense adjustments to `adjustedRa9` multiplicatively:

```js
adjustedRa9 *= (wrc_plus / 100);       // wRC+ term
adjustedRa9 *= (1 + iso_adj);          // ISO term
adjustedRa9 *= (1 - k_pct_adj);        // k% term (also applied in pitcher skill already)
adjustedRa9 *= (1 + bb_pct_adj);       // BB% term (also applied in pitcher skill already)
adjustedRa9 *= contactMult;            // xwOBA + hard_hit combo
```

The statistical audit found:
- xwOBA ↔ wRC+ r ≈ 0.81
- xwOBA ↔ ISO r ≈ 0.72
- hard_hit_pct has low independent variance after xwOBA is included

A correlated elite offense (e.g., LAD vs a poor pitcher) can receive a stacked multiplier of ~1.14 when the intended maximum adjustment is ~1.08–1.10. This inflates OVER projections for high-offense matchups.

Additionally, `bb_pct` is applied twice: once to `skillRa9` in `resolveStarterSkillProfile` and again via `adjustedRa9 *= bb_adj`. This double-counts walk rate.

## Formula Change

### Removed features
- `ISO` multiplier (`1 + (iso - MLB_F5_DEFAULT_TEAM_ISO) * 0.35`)
- `hard_hit_pct` component inside `contactMult`
- `bb_pct` multiplier in `projectTeamF5RunsAgainstStarter` (keep it only in `resolveStarterSkillProfile` where it belongs on pitcher quality)
- `k_pct` team offense multiplier in `projectTeamF5RunsAgainstStarter` (was double-counting — pitcher k-rate already captured in `resolveStarterSkillProfile`)
- `iso === null` null-guard condition in `resolveTeamSplitProfile` (iso no longer required for a valid matchup profile)

### New offense composite
```js
// Z-score helpers (season population means/stds from constants)
const MLB_LEAGUE_WRC_PLUS_MEAN = 100;
const MLB_LEAGUE_WRC_PLUS_SD = 14;   // approximate cross-team SD
const MLB_LEAGUE_XWOBA_MEAN = 0.320;
const MLB_LEAGUE_XWOBA_SD = 0.018;

function zScore(value, mean, sd) {
  return (value - mean) / sd;
}

/**
 * Composite offensive quality index.
 * Returns a multiplier centred at 1.0, clamped to [0.88, 1.14].
 * 60% wRC+ split / 40% xwOBA split.
 */
function resolveOffenseComposite(matchupProfile) {
  const zWrc = zScore(
    matchupProfile.wrc_plus,
    MLB_LEAGUE_WRC_PLUS_MEAN,
    MLB_LEAGUE_WRC_PLUS_SD,
  );
  const xwoba = matchupProfile.xwoba ?? MLB_F5_DEFAULT_TEAM_XWOBA;
  const zXwoba = zScore(xwoba, MLB_LEAGUE_XWOBA_MEAN, MLB_LEAGUE_XWOBA_SD);
  const composite = 0.60 * zWrc + 0.40 * zXwoba;
  // Scale: 1 SD of composite ≈ 0.06 run adjustment
  return clampValue(1.0 + composite * 0.06, 0.88, 1.14);
}
```

### Updated `projectTeamF5RunsAgainstStarter` application
```js
// Replace the four inline adjustedRa9 multiplications with:
const offenseMult = resolveOffenseComposite(matchupProfile);
adjustedRa9 *= offenseMult;

// k% team multiplier removed (double-counted — pitcher k-rate captured in resolveStarterSkillProfile)
// bb% multiplier removed from here (only in resolveStarterSkillProfile — pitcher quality)

// Keep park factor and weather factor as before
adjustedRa9 *= clampValue(parkFactor, 0.9, 1.12);
adjustedRa9 *= weatherFactor ?? 1.0;
adjustedRa9 *= starterLeashProfile.ttop_penalty_mult;
```

Return object — add alongside existing fields:
```js
offense_composite: offenseMult,   // new; replaces contact_mult (remove contact_mult from return)
```

### Rolling 14d wRC+ overlay (keep, tighten clamp)
The `rolling_14d_wrc_plus` overlay applies a secondary adjustment. Tighten clamp from `[0.95, 1.05]` to `[0.97, 1.03]` (±3% max) because the composite already captures medium-term form.
