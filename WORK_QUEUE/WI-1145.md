# WI-1145: Align Worker Projection Settlement To Canonical Confidence Tiers

**ID**: WI-1145
**Goal**: Ensure `settle_projections.js` resolves confidence bucket from the canonical chain before writing proxy eval rows, rather than reading `payload?.confidence_band` directly. Extend and reuse the canonical resolver in `projection_evaluator.js` so settlement supports top-level bands, legacy `drivers[0].confidence_band`, and score-based fallback without duplicating logic. Remove dead direct-field reads superseded by this resolution.

**Scope**:

- apps/worker/src/jobs/settle_projections.js
- apps/worker/src/jobs/__tests__/settle_projections.test.js
- apps/worker/src/audit/projection_evaluator.js

**Out of scope**:

- UI display logic
- Non-projection settlement pipelines
- Broad schema migrations unrelated to confidence normalization
- One-time backfill or remediation of historical rows
- Ingestion-time confidence assignment in `run_mlb_model.js` or `run_nhl_player_shots_model.js`

**Acceptance**:

- `resolveMoneylineConfidenceBucket` is exported from `projection_evaluator.js`, extended to include legacy `payload.drivers[0].confidence_band` fallback, and imported by `settle_projections.js`; no equivalent logic is duplicated inline.
- Settlement resolves `confidence_bucket` via the canonical chain before passing to `buildProjectionProxyMarketRows`: top-level band fields (`confidence_bucket`/`confidence_band`) -> `payload.projection_accuracy.confidence_band` -> `payload.drivers[0].confidence_band` -> score-based fallback (`confidence_score`).
- No null/unknown `confidence_bucket` reaches `projection_proxy_evals` for eligible rows where confidence data exists in any position of the payload.
- Legacy payload shapes (confidence only in `drivers[0].confidence_band`, no top-level field) produce a correct non-null `confidence_bucket`.
- Dead direct-field reads (`payload?.confidence_band` passed as bare `confidence_bucket`) in `settle_projections.js` that are superseded by the canonical resolver are removed.
- Tests cover all four resolution cases: top-level `confidence_band` present; top-level absent, `drivers[0].confidence_band` present; both band fields absent with `confidence_score` present; all absent (`LOW` default expected).
