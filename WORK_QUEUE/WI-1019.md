# WI-1019: NBA Phase 1A — NBA totals accuracy tracking foundation

**ID**: WI-1019
**Goal**: Wire NBA total-projection card types into the projection accuracy framework so every miss is measurable, with enough contextual fields that calibration (WI-1020), residual learning (WI-1024), and volatility empirics (WI-1023) are not working from a skeleton.

**Depends on**: none (parallel with WI-1018)

**Scope**:
- `packages/data/src/db/projection-accuracy.js`
- `packages/data/db/migrations/` (new migration for evaluation schema enrichment)
- `apps/worker/src/jobs/run_nba_model.js` (settlement write path — stamping projection_accuracy fields)
- `packages/data/__tests__/projection-accuracy-engine.test.js`
- `apps/worker/src/__tests__/run-nba-model.test.js`

**Out of scope**:
- Rolling bias computation and correction (WI-1020 depends on this)

- UI changes
- Other sport models

**Acceptance**:
- `TRACKED_PROJECTION_ACCURACY_CARD_TYPES` contains entries for `nba-totals-call` and `nba-total-projection` with `marketFamily: 'NBA_TOTAL'`, `actualKeys: ['total_score']`, `propType: 'game_total'`, `identity: 'game'`.
- `projectionKeys` resolves correctly for both card types against real `nba-totals-call` payloads.
- At card write time in `run_nba_model.js`, the settlement annotation (near the existing `sigma_source` stamp) writes `projection_accuracy.projection_raw` from `projection.total` or `odds_context.projection_comparison` — whichever is non-null.
- The following contextual fields are stamped on the card payload's `raw_data` at write time so they survive into the accuracy table:
  - `raw_data.market_total` — the market total line at snap time
  - `raw_data.pace_tier` — from the pace synergy result's `synergyType` field
  - `raw_data.vol_env` — LOW / MED / HIGH if derivable from game state
  - `raw_data.total_band` — `'<220'`, `'220-230'`, `'230-240'`, or `'240+'` derived from market_total
  - `raw_data.injury_cloud` — `'NONE'`, `'MODERATE'`, or `'SEVERE'` based on total point impact from availability gate (0 = NONE, 1-14 = MODERATE, 15+ = SEVERE)
  - `raw_data.driver_contributions` — array of `{ driver, weight, signal }` from active NBA_DRIVER_WEIGHTS, shape already exists in `driver_summary`
- After a game settles, `projection_accuracy_line_evals` rows exist for NBA total cards with non-null `projection_raw` and non-null `actual_value`.
- `job:projection-accuracy-health` does not crash when NBA rows are present.
- At least one unit test verifies projection key resolution returns a non-null projection from a representative `nba-totals-call` fixture payload.

```js
function deriveTotalBand(marketTotal) {
  if (!Number.isFinite(marketTotal)) return null;
  if (marketTotal < 220) return '<220';
  if (marketTotal < 230) return '220-230';
  if (marketTotal < 240) return '230-240';
  return '240+';
}
```
