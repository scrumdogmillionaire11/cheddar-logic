## Design

### Adjustment trust → goalie factor rule

```js
function applyGoalieAdj(savePct, adjustmentTrust) {
  if (adjustmentTrust === 'BLOCKED' || adjustmentTrust === 'NEUTRALIZED') {
    return 1.0; // no directional adjustment
  }
  const factor = goalieAdjFactor(savePct); // existing function
  if (adjustmentTrust === 'DEGRADED') {
    return 1.0 + (factor - 1.0) * 0.5; // half-weight
  }
  return factor; // FULL trust
}
```

### Output fields added to `predictNHLGame()` return value

```js
{
  // ... existing fields ...
  homeAdjustmentTrust: 'FULL' | 'DEGRADED' | 'NEUTRALIZED' | 'BLOCKED',
  awayAdjustmentTrust: 'FULL' | 'DEGRADED' | 'NEUTRALIZED' | 'BLOCKED',
  official_eligible: true | false,
  // home/awayGoalieState passed through as-is for downstream consumers
  homeGoalieState: CanonicalGoalieState | null,
  awayGoalieState: CanonicalGoalieState | null,
}
```

### Eligibility rule

```js
official_eligible = !(
  homeAdjustmentTrust === 'BLOCKED' ||
  awayAdjustmentTrust === 'BLOCKED'
);
```

A NEUTRALIZED state does not block eligibility — it means unknown goalie, which is allowed to produce an official total with reduced confidence. Only CONFLICTING (BLOCKED) blocks official eligibility at model time.

## Failure Modes & Guards

### Failure mode: NEUTRALIZED still produces directional goalie nudge
**Guard:** test explicitly asserts `awayGoals` and `homeGoals` are not affected by save-pct factor when trust is NEUTRALIZED.

### Failure mode: official_eligible missing from output
**Guard:** test asserts key is present in all return paths including null-input early returns.

### Failure mode: CONFLICTING state skips projection entirely
**Guard:** test asserts `predictNHLGame()` returns a non-null result even when both states are BLOCKED; only `official_eligible` distinguishes it from a normal result.
