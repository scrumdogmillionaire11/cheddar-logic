# WI-0619: Extract FPL scheduler to deadline-based schedulers/fpl.js

**ID**: WI-0619
**Goal**: Move FPL jobs out of `schedulers/main.js` into a dedicated `schedulers/fpl.js` with deadline-aware cron cadence instead of game-time rhythms.

**Background**:
FPL jobs are hardcoded into the game-time scheduler in `apps/worker/src/schedulers/main.js` with a `// TEMPORARY` comment (lines 21, 166). FPL uses deadline-based windows (GW deadline = when transfers lock), not game start times. Running FPL jobs on game-time rhythms produces stale or mis-timed model runs.

**Scope**:
- `apps/worker/src/schedulers/fpl.js` (new) — FPL-specific cron jobs keyed to GW deadline windows; import from `schedulers/main.js` entry point
- `apps/worker/src/schedulers/main.js` — remove FPL job registrations and `// TEMPORARY` block; import and call `schedulers/fpl.js`
- Document the GW deadline logic in the new file's header comment

**Out of scope**:
- Changes to the FPL model itself
- Auto-detection of GW deadlines from the FPL API (can be hardcoded per season for now)

**Acceptance**:
- FPL jobs no longer appear in `main.js`.
- `schedulers/fpl.js` exists with deadline-aware cron expressions.
- Worker starts without error: `node apps/worker/src/schedulers/main.js --dry-run`.
- No `// TEMPORARY` comments remain in `main.js`.
