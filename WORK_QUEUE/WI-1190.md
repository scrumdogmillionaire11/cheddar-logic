# WI-1190: Unify Decision Reader Authority Options (Publisher ↔ Web)

**ID**: WI-1190
**Goal**: Align worker publisher and web reader to use identical `readDecisionAuthority` options, eliminating divergence in canonical decision source strictness.

**Scope**:

- `apps/worker/src/utils/decision-publisher.js`
- `web/src/lib/runtime-decision-authority.ts`
- `apps/worker/src/utils/__tests__/decision-publisher.v2.test.js`

**Implementation notes**:

- Worker publisher currently calls `resolveCanonicalDecision()` with `strictSource: false` (line 215 in decision-publisher.js)
- Web reader calls the same function with `strictSource: true` (line 163 in runtime-decision-authority.ts)
- `strictSource: true` enforces canonical `decision_v2.canonical_envelope_v2` as the sole authority source
- `strictSource: false` permits fallback to alternative sources if canonical envelope is missing
- Convergence: both should use `strictSource: true` per ADR principle "canonical authority only"

- Both publisher and web reader now share identical option set: `{ fallbackToLegacy: false, strictSource: true }`
