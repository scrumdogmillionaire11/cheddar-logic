# WI-1103: Remove legacy WATCH/HOLD auto-void settlement logic

## Problem

MLB full-game moneyline cards with legacy `status: 'WATCH'` are automatically voided during settlement, even when the game is final and should settle normally.

**Root cause:** MLB model degrades when inputs are missing (e.g., bullpen data), sets `status: 'WATCH'` (legacy field), and settlement job auto-closes these cards as `NON_ACTIONABLE_FINAL_LEGACY_WATCH_HOLD`.

**Real-world case:** Royals ML card (2026-04-20) had degraded model, got legacy WATCH status, and was voided instead of settling.

## Critical constraint

**Cards only settle if classified as LEAN or PLAY.** PASS cards are skipped, WATCH cards are blocked. Removing the auto-void block only helps if the card has `decision_v2.official_status: 'LEAN'` or `'PLAY'`.

## Goal

Remove legacy WATCH/HOLD auto-void logic from settlement and ensure degraded MLB cards with a lean edge emit proper `decision_v2.official_status: 'LEAN'` so they settle normally.

- `apps/worker/src/models/mlb-model.js` — Add `decision_v2.official_status` to card payload (must be LEAN for degraded actionable cards)

## Implementation

### 1. Settlement job: Remove legacy WATCH/HOLD block

**File:** `apps/worker/src/jobs/settle_pending_cards.js` (lines 1013-1021)

Delete the entire conditional block that auto-closes cards with legacy WATCH/HOLD status:

```javascript
// DELETE THIS BLOCK:
if (
  !officialStatus &&
  (legacyStatus === 'WATCH' || legacyStatus === 'HOLD') &&
  isMlbFullGameTotalOrMoneylineRow(row, payloadData)
) {
  return {
    code: 'NON_ACTIONABLE_FINAL_LEGACY_WATCH_HOLD',
    message: 'Legacy MLB full-game WATCH/HOLD card is non-actionable',
    details: { legacyStatus, legacyFallback: true },
  };
}
```

Also delete the `isMlbFullGameTotalOrMoneylineRow()` helper function (line 934) if it's not used elsewhere.

### 2. MLB model: Emit decision_v2.official_status (CRITICAL)

**File:** `apps/worker/src/models/mlb-model.js` (lines 1520-1524)

Add `decision_v2` object to payload. **Key requirement:** For degraded MLB cards with a lean edge to settle, must emit `official_status: 'LEAN'`.

```javascript
decision_v2: {
  official_status: canFire ? 'PLAY' : (canLean ? 'LEAN' : 'PASS'),
  degraded: isDegraded,
  degradation_reason: isDegraded ? 'INSUFFICIENT_DATA' : null,
  watchdog_status: 'READY',
}
```

This ensures:
- Cards with a fire signal → `official_status: 'PLAY'` → settles
- Cards with a lean edge → `official_status: 'LEAN'` → settles (even if degraded)
- Cards with no edge → `official_status: 'PASS'` → skips settlement

Keep legacy `status/action/classification` for backward compatibility but mark as display-only in comments.

### 3. Decision resolution: Add deprecation comments

**File:** `packages/data/src/decision-status.js`

Mark `resolveLegacyDecisionStatusToken()` as deprecated. Add comment that settlement should use `decision_v2.official_status` instead.
