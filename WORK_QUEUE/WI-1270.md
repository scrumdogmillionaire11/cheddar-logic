# WI-1270: Gate Dotenv Loading in Worker Diagnostics

**ID**: WI-1270
**Goal**: Prevent operational worker diagnostics from auto-loading local `.env` in production-like runtime unless explicitly allowed.

**Scope**:
- `apps/worker/src/jobs/check_pipeline_health.js`
- `apps/worker/src/jobs/report_settlement_health.js`
- `apps/worker/src/__tests__/dotenv-loading-guard.test.js` (new)

**Out of scope**:
- Process manager configuration
- Deployment secrets management
- Non-diagnostic worker jobs unless they share the same dotenv helper

## Implementation contract

Create or reuse a single helper:

```js
function maybeLoadLocalDotenv(options = {}) {
  const env = process.env.NODE_ENV || "development";
  const isProductionLike = ["production", "staging", "preprod"].includes(env);
  const explicitlyAllowed = process.env.ALLOW_DOTENV_LOCAL === "true";

  if (isProductionLike && !explicitlyAllowed) {
    return { loaded: false, reason: "DOTENV_DISABLED_PRODUCTION_LIKE" };
  }

  // Load dotenv without overriding existing env vars.
  // dotenv.config({ override: false })
}
```

Both diagnostics must use this helper. No direct `dotenv.config()` calls may remain in:

- `check_pipeline_health.js`
- `report_settlement_health.js`

**Acceptance**:

- `.env` loading is skipped by default when `NODE_ENV` is `production`, `staging`, or `preprod`.
- In production-like runtime (`production|staging|preprod`), local dotenv loading is allowed only when `ALLOW_DOTENV_LOCAL=true`.
- In `development` and `test`, local dotenv loading remains allowed.
- Importing `check_pipeline_health.js` in production-like mode must not hydrate local `.env` through direct load or transitive import side effects from `report_settlement_health.js`.
- Explicit process env vars remain respected (dotenv must not override existing process values).
- No direct top-level `dotenv.config()` call remains in either diagnostic module.
- Any dotenv load path uses `override: false`.
- Tests prove production-like mode does not hydrate local `.env`.
- Tests prove local/dev mode still works when allowed.
