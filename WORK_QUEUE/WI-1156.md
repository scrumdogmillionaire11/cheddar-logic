**Acceptance**:
- Projection settlement is materially complete for all projection families intentionally surfaced in results.
- `1P O/U` projection settlements are persisted and exposed consistently, not just selectively represented by current proxy-eval paths.
- `/api/results/projection-settled` and `/api/results/projection-accuracy` agree on the supported projection families.
- Historical missing rows can be backfilled so the fix applies to past settled projection cards, not only future ones.
- Desktop projection results use a desktop-oriented table/layout with readable columns and expandable detail rows.
- Mobile behavior remains usable and unchanged in spirit.
- Dead or legacy projection-settlement/UI code in scoped files that becomes obsolete after the unified contract lands is removed rather than left as parallel legacy behavior.

## Must-haves (goal-backward)

**truths**:
- Projection cards for all supported surfaced families, including NHL `1P O/U`, move to deterministic settled outcomes instead of remaining selectively unresolved.
- Projection settlement and projection accuracy APIs expose the same supported-family contract and cannot silently diverge.
- Historical settled projection cards missing completeness rows can be backfilled with a reproducible in-scope path.
- `/results/projections` provides a desktop-readable projection table with expandable detail rows while preserving mobile usability.
- Superseded settlement and projection-results legacy branches in scoped files are removed or intentionally documented.

**artifacts**:
- `apps/worker/src/jobs/settle_projections.js` writes complete family coverage and deterministic `1P O/U` outcomes.
- `packages/data/src/db/cards.js` and `packages/data/src/db/projection-accuracy.js` expose a shared supported-family contract used by settlement writes and result reads.
- `web/src/app/api/results/projection-settled/route.ts` and `web/src/app/api/results/projection-accuracy/route.ts` read the shared family contract.
- `web/src/components/results/ProjectionResultsTable.tsx` and `web/src/app/results/projections/page.tsx` provide a desktop-first projection surface with expandable rows.

**key_links**:
- Worker settlement write path -> projection materialization rows for supported families.
- Materialized projection rows -> `/api/results/projection-settled` and `/api/results/projection-accuracy` using one family allowlist.
- Projection results APIs -> `/results/projections` desktop table and details rendering.
- Legacy branch removal list -> grep/test verification proving old parallel behavior is gone.
