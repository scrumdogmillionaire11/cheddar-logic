# WI-0664: DB migration — add public betting splits columns to odds_snapshots

**ID**: WI-0664
**Goal**: Add 8 nullable columns to `odds_snapshots` to store public bet %, handle %, ticket %, and splits source metadata from external providers.

**Background**:
No public betting percentage or handle data exists anywhere in the codebase. This migration is the prerequisite for the full splits pipeline (WI-0665, WI-0666, WI-0667). The `PASS_SHARP_MONEY_OPPOSITE` reason code already has display labels in Discord (`apps/worker/src/jobs/post_discord_cards.js`) and the web UI (`web/src/components/cards/game-card-helpers.tsx`) but has no data producer — this WI creates the DB foundation.

**Depends on**: none

**Scope**:
- `packages/data/db/migrations/052_add_public_splits_to_odds_snapshots.sql` (new file)

**Out of scope**:
- Any application code changes to read or write these columns (WI-0665+)
- Model or pipeline changes (WI-0667)
- UI changes

**Acceptance**:
- Migration file `052_add_public_splits_to_odds_snapshots.sql` adds all 8 columns via `ALTER TABLE odds_snapshots ADD COLUMN ... DEFAULT NULL`:
  - `public_bets_pct_home REAL` — % of bet tickets on home side (0–100)
  - `public_bets_pct_away REAL` — % of bet tickets on away side
  - `public_handle_pct_home REAL` — % of wagered dollars on home side (0–100)
  - `public_handle_pct_away REAL` — % of wagered dollars on away side
  - `public_tickets_pct_home REAL` — ticket count % distinct from dollar handle (0–100)
  - `public_tickets_pct_away REAL`
  - `splits_source TEXT` — `'action_network'` or `'pinnacle_proxy'`
  - `splits_captured_at TEXT` — ISO 8601 UTC timestamp of splits fetch
- Migration runs cleanly against the existing DB without modifying any existing rows
- `sqlite3 $CHEDDAR_DB_PATH ".schema odds_snapshots"` shows all 8 new columns
- All existing tests continue to pass
