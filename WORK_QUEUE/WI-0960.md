# WI-0960: Align MLB model expectations with persisted odds coverage

**ID**: WI-0960
**Goal**: Remove the systematic MLB pricing-state mismatch where the model expects executable price inputs that the odds ingest path does not persist for most games.

**Depends on**: none

**Scope**:
- `WORK_QUEUE/WI-0960.md`
- `apps/worker/src/jobs/pull_odds_hourly.js`
- `packages/odds/src/**/*.js`
- `packages/data/src/db/odds.js`
- `apps/worker/src/jobs/run_mlb_model.js`
- `apps/worker/src/jobs/__tests__/pull_odds_hourly.test.js`
- `apps/worker/src/jobs/__tests__/run_mlb_model.test.js`
- `apps/worker/src/__tests__/run-mlb-model.dual-run.test.js`

**Out of scope**:
- Retuning MLB thresholds, sigma, or directional model formulas
- Adding new sportsbooks beyond the current provider contract
- Web UI card rendering changes

**Acceptance**:
- The codebase explicitly documents, in code and tests, which MLB markets are required for `pricing_ready=true` and which are optional/degraded.
- If a supported market is required for execution, odds ingest fetches and persists the needed inputs for that market; otherwise the model stops advertising that input as a blocker for intentionally unsupported paths.
- The current dev-run pattern of `pricing_ready=3/11` with `MARKET_PRICE_MISSING|F5_ML_UNAVAILABLE` on every game is either materially reduced or explained by deterministic, tested fallback semantics.
- Regression tests cover both supported-market availability and degraded-market behavior so future ingest/model changes cannot silently reintroduce the mismatch.

**Execution plan (deferred):**
- Phase 1: Map current ingest contract for MLB markets and identify exactly which fields gate `pricing_ready` in `run_mlb_model`.
- Phase 2: Decide policy per market: required-for-execution vs optional/degraded, then encode policy comments near ingest + model checks.
- Phase 3: Align ingest persistence and model blocker logic so unsupported paths stop presenting as hard missing-price blockers.
- Phase 4: Add regression coverage for supported-market availability and deterministic degraded behavior.
- Phase 5: Run pull-odds + run-mlb-model manually and capture pricing-ready deltas for acceptance evidence.
