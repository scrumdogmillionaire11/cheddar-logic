# WI-0938-03: THE BOARD - Opportunities Tab Component

**ID**: WI-0938-03  
**Parent**: WI-0938  
**Goal**: Build the Opportunities Tab component that displays actionable edges with market error framing, action state (FIRE/WAIT/PASS), and trigger conditions.

**Depends on**: WI-0938-02 (skeleton must exist first)

**Scope**:

- `web/src/components/board/OpportunitiesTab.tsx` (main container)
- `web/src/components/board/OpportunityCard.tsx` (individual opportunity card)
- `web/src/components/board/ActionStateBadge.tsx` (renders FIRE/WAIT/PASS with styling)
- `web/src/components/board/TriggerConditionsList.tsx` (renders trigger details)
- Mock data integration (pull from `/api/board/opportunities` endpoint)
- Styling (follows existing Cheddar design system)

**Out of scope**:

- Edge reasoning enrichment (backend computes this in WI-0938-06)
- Real card payload data (uses mock until WI-0938-06)
- Discord formatting (that's WI-0938-07)
- Analytics/tracking

**Acceptance**:

- `OpportunitiesTab` component renders list of opportunities from `/api/board/opportunities`
- Each opportunity card displays:
  - Edge % + Quality Score (visible together, e.g., "EDGE: +1.8% | QUALITY: 8.2/10")
  - Action State badge (FIRE/WAIT/PASS with distinct colors: green/yellow/gray)
  - Market Error (1-2 sentence explanation, e.g., "Vegas anchored to outdated pace assumptions")
  - Edge Types (Primary: STALE_PRICE, Secondary: STRUCTURAL)
  - Trigger Conditions (formatted clearly, shows what needs to happen for state change)
  - Quality Drivers (checklist of positive ✓ and negative ✗ factors)
  - Pattern Context (if available: hit rate, ROI, trend arrow)
- Empty state renders when no opportunities exist
- Responsive design (mobile-friendly)
- TypeScript types match schema from WORK_QUEUE/THE_BOARD_EDGE_SCHEMA_SPEC.md
- Unit tests for OpportunityCard rendering
- Manual: Card displays correctly with mock data
