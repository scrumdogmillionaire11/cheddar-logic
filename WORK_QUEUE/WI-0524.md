**ID**: WI-0524
**Goal**: Add a production-safe Discord webhook snapshot job that posts scheduled `/cards`-equivalent non-PASS card output at 09:00, 12:00, and 18:00 ET with automatic multi-message chunking.

**Acceptance**:

- Scheduler queues a Discord snapshot job at fixed ET windows `09:00`, `12:00`, and `18:00` with deterministic job keys and idempotency behavior aligned with existing scheduler patterns.
- Webhook payload selection excludes PASS outcomes and includes slight-edge/non-PASS rows (including projection-only rows when present), with explicit sections for player props and 1P cards.
- Discord publishing handles message size constraints by sending multiple ordered chunks when content exceeds configured limit.
- Feature is production-gated by explicit env flags and no-op safe when webhook URL is unset.
- Runbook/env docs include setup, safety notes, and manual invocation command.
