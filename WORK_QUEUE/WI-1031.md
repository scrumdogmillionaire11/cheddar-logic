# WI-1031: Execution-gate — Audit and fix vig double-deduction

**ID**: WI-1031
**Goal**: Determine whether `execution-gate.js` is double-counting vig friction by subtracting `VIG_COST_STANDARD (4.5%) + SLIPPAGE_ESTIMATE (0.5%)` from a `rawEdge` that was itself already computed against an implied probability derived from a vig-inclusive book price. If double-counting is confirmed, remove the subtraction and replace with a documented noise-floor rationale.

**Background**:
`execution-gate.js` line 175:
```js
const netEdge = hasRawEdge ? rawEdge - vigCost - slippageCost : null;
```
`vigCost` defaults to `VIG_COST_STANDARD = 0.045` and `slippageCost = 0.005`.

The question is what `rawEdge` represents when callers invoke `evaluateExecution`:

- In `signal-engine.js` (POTD path — does NOT call execution-gate): `edgePct = modelWinProb - americanToImplied(price)` where `americanToImplied(price)` already includes vig in the book price. Devigging is implicit: the vig is "in the price" the bettor pays.

- In the regular model pipeline (MLB, NHL, NBA cards calling execution-gate): `rawEdge` is stamped on the card payload as `edge` or `edge_pct`. These come from model outputs in `run_mlb_model.js`, `run_nhl_model.js`, `run_nba_model.js`. The computation there is `modelWinProb - impliedProbFromPrice`.

If `rawEdge = modelWinProb - impliedProbFromPrice` and `impliedProbFromPrice` already captures the vig embedded in the book price, then subtracting `vigCost = 4.5%` again means the gate effectively requires `rawEdge > 5%` before a bet is allowed — suppressing any play with a 2–4% true edge against the market.

- If `rawEdge` is consistently `model_prob - implied_from_vigged_price`:
  - Remove `vigCost` and `slippageCost` parameters and the subtraction from `netEdge`.
  - Replace with: `netEdge = rawEdge` (the edge is already stated net of vig because implied price is vig-inclusive).

  - `VIG_COST_STANDARD` and `SLIPPAGE_ESTIMATE` constants are removed or demoted to documentation comments.

- If any call site does pass a devigged `rawEdge` (model-vs-fair-prob), document which one and why; those callers need fixing instead, not execution-gate.
