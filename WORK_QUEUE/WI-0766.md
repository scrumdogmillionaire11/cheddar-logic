# WI-0766: Define NFL data layer spec before enabling the NFL model stub

| Field              | Value |
|--------------------|-------|
| **ID**             | WI-0766 |
| **Goal**           | Either build the minimum NFL data layer (team stats pull, injury/depth chart, EPA/DVOA) needed to make `run_nfl_model.js` produce non-trivial cards, or formally remove the stub and document NFL as a future roadmap item |
| **Priority**       | Medium (disabled stub creates confusion; if enabled it emits cards from odds alone with no sport-specific context) |
| **Owner agent**    | Unassigned |
| **Time window**    | TBD |
| **Coordination flag** | `needs-sync` — modifies scheduler + adds pull jobs or removes them |

---

## Problem

The NFL model pipeline is a stub in three ways:

**1. Permanently disabled**
`ENABLE_NFL_MODEL=false` in `.env`. Scheduler registration in `apps/worker/src/schedulers/main.js` lines 126–129 checks this flag.

**2. No sport-specific data sources**
No `pull_nfl_*` jobs exist anywhere in `apps/worker/src/jobs/`. The model reads only generic `odds_snapshots` — no team stats, no player stats, no weather, no injury/lineup data.

**3. Writes to a dead sink**
`run_nfl_model.js` calls `insertModelOutput()` (line ~254) — which writes to `model_outputs`, a table with no consumers (see WI-0760).

**Impact:**
- If accidentally enabled, the NFL model would emit cards based solely on odds movement — no sport context, no injury gating, no EPA-based edge signal
- The stub misleads future agents into thinking an NFL pipeline is in progress

---

## Decision required

**Option A — Build minimum viable NFL data layer (3–4 WIs)**

Minimum required before enabling:
1. `pull_nfl_team_stats.js` — team-level EPA, DVOA, or equivalent (ESPN Stats & Info or Pro Football Reference)
2. `pull_nfl_injuries.js` — injury reports from ESPN or NFL.com
3. NFL totals model wiring — connect team O/D efficiency to totals projection in `run_nfl_model.js`
4. Enable gate: `ENABLE_NFL_MODEL=true` only after data layer tests pass

**Option B — Remove stub (preferred if NFL is not in scope this season)**

1. Remove `run_nfl_model.js` scheduler registration from `main.js`
2. Move `apps/worker/src/jobs/run_nfl_model.js` to `apps/worker/src/jobs/_archived/run_nfl_model.js`
3. Add `docs/decisions/ADR-NFL-removal.md` documenting the decision and what would be needed to reintroduce it
4. Remove `insertModelOutput()` call in the archived file so it does not count against WI-0760 scope
