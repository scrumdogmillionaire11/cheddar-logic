# WI-1191: Re-enable Guarded MLB Full-Game Fallback Merge for /api/games

**ID**: WI-1191
**Goal**: Fix payload wiring where active-run filtering drops available MLB full-game card rows, causing false missing-driver/missing-projection pass reasons.

**Scope**:

- `WORK_QUEUE/WI-1191.md`
- `web/src/lib/games/route-handler.ts`
- `web/src/__tests__/api-games-market-smoke.test.js`

**Out of scope**:

**Implementation notes**:

- `/api/games` currently scopes `card_payloads` by active run ids and only backfills missing game ids.
- MLB games often have `mlb-f5` in the active run while `mlb-full-game` / `mlb-full-game-ml` remain in older successful runs.
- `mergeMlbGameLineFallbackRows` already exists with strict guardrails (`ODDS_BACKED`, `EXECUTABLE`, official status/actionability checks, age and odds-capture tolerance) but is not invoked.
- Re-enable this guarded merge in the query path so existing valid data is wired into game cards.

**Acceptance**:

- Merge remains constrained by existing eligibility guardrails (no stale card injection).
