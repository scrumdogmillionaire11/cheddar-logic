# WI-1228: High-End Slight Edge Promotion

**ID**: WI-1228
**Goal**: Add a conservative high-end `LEAN -> PLAY` promotion path inside `packages/models/src/decision-pipeline-v2.js` for a small explicit market set, while keeping existing `PLAY` thresholds, `PASS` behavior, and external `LEAN -> SLIGHT_EDGE` mapping unchanged.

**Context**:

`decision-pipeline-v2` currently classifies wave-1 markets into `PLAY`, `LEAN`, or `PASS` using sport/market threshold profiles plus later demoters such as heavy-favorite caps and NBA-total quarantine. The system has no narrow path for a strong-edge `LEAN` to surface as a `PLAY` without lowering the base `PLAY` thresholds.

This WI adds that path without weakening the baseline decision model:

- existing `PLAY` thresholds remain unchanged
- only current internal `LEAN` candidates are eligible
- `PASS` candidates remain non-promotable
- promotion must reuse the pipeline's existing blocker/gate semantics
- promotion metadata must be additive and traceable

**Acceptance**:

- Clean scoped-market `LEAN` candidates above promotion thresholds can promote to `PLAY`.
- Existing `PLAY` thresholds remain unchanged.
- `PASS` candidates are never promoted.
- Promotion reuses canonical blocker evaluators and existing reason-code families.
- `decision_v2` carries additive promotion metadata:
  - `promoted_from: 'LEAN'`
  - `promotion_reason_code: 'HIGH_END_SLIGHT_EDGE_PROMOTION'`
- `HIGH_END_SLIGHT_EDGE_PROMOTION` is registered in canonical reason codes and appears in emitted reason arrays.
- If a promoted row remains final `PLAY`, its `primary_reason_code` is `HIGH_END_SLIGHT_EDGE_PROMOTION`.
- If a later demoter fires, the demoter remains primary and promotion metadata is retained.
- External canonical status mapping remains `PLAY | SLIGHT_EDGE | PASS`.
