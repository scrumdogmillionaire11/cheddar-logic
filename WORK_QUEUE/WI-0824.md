# WI-0824: Two-Layer Bet Execution Gate — Separate Model Fair Probability from Executable Edge

| Field | Value |
|---|---|
| **ID** | WI-0824 |
| **Goal** | Add a stateless execution gate that sits between model output and card payload creation. A FIRE/PLAY status from a model is a necessary but not sufficient condition to place a bet. The gate enforces: net edge after vig and slippage must be positive; market must not be stale; model status must be MODEL_OK; no missing critical upstream fields. |
| **Priority** | High |

## Background

Currently, a model that outputs `FIRE` with a positive `edge` field directly triggers a card payload write. The `edge` value is the model's raw fair-probability edge against the implied market probability, with no deduction for:
- The vig (juice) on the bet
- Market staleness (time since odds snapshot was taken)
- Slippage risk (thin markets, single-book pricing)

A model with a raw edge of +2.5% against -110 juice (~4.5% vig cost) is actually a -2% net bet. This WI creates `executionGate.js` that enforces net-positive edge before any bet signal propagates.

## Gate Function

```js
'use strict';

const VIG_COST_STANDARD = 0.045;   // -110 juice = ~4.5% cost on fair 50/50
const SLIPPAGE_ESTIMATE = 0.005;   // Conservative 0.5% round-trip slippage
const MAX_SNAPSHOT_AGE_MS = 5 * 60 * 1000;  // 5 minutes; stale = NO_BET

/**
 * Evaluate whether a model output is executable as a live bet.
 *
 * @param {object} params
 * @param {'MODEL_OK'|'DEGRADED'|'NO_BET'} params.modelStatus
 * @param {number|null} params.rawEdge         - model fairProb - impliedProb (0–1 scale)
 * @param {number|null} params.confidence      - model confidence (0–1)
 * @param {number|null} params.snapshotAgeMs   - ms since odds snapshot was captured
 * @param {number} [params.vigCost]            - override vig cost if known from book (default 0.045)
 * @param {number} [params.slippageCost]       - override slippage estimate (default 0.005)
 * @param {number} [params.minNetEdge]         - minimum acceptable net edge (default 0.025)
 * @param {number} [params.minConfidence]      - minimum confidence threshold (default 0.60)
 * @returns {{ shouldBet: boolean, reason: string, netEdge: number|null, blocked_by: string[] }}
 */
function evaluateExecution(params) {
  const {
    modelStatus,
    rawEdge,
    confidence,
    snapshotAgeMs,
    vigCost = VIG_COST_STANDARD,
    slippageCost = SLIPPAGE_ESTIMATE,
    minNetEdge = 0.025,
    minConfidence = 0.60,
  } = params;

  const blocked_by = [];

  if (modelStatus !== 'MODEL_OK') {
    blocked_by.push(`MODEL_STATUS_${modelStatus}`);
  }

  if (rawEdge === null || rawEdge === undefined) {
    blocked_by.push('NO_EDGE_COMPUTED');
  }

  const netEdge = rawEdge !== null ? rawEdge - vigCost - slippageCost : null;

  if (netEdge !== null && netEdge < minNetEdge) {
    blocked_by.push(`NET_EDGE_INSUFFICIENT:${netEdge.toFixed(4)}`);
  }

  if (confidence !== null && confidence < minConfidence) {
    blocked_by.push(`CONFIDENCE_BELOW_THRESHOLD:${confidence.toFixed(3)}`);
  }

  if (snapshotAgeMs !== null && snapshotAgeMs > MAX_SNAPSHOT_AGE_MS) {
    blocked_by.push(`STALE_SNAPSHOT:${Math.round(snapshotAgeMs / 1000)}s`);
  }

  const shouldBet = blocked_by.length === 0;
  const reason = shouldBet ? 'ALL_GATES_PASSED' : blocked_by[0];

  return { shouldBet, reason, netEdge, blocked_by };
}

module.exports = { evaluateExecution, VIG_COST_STANDARD, SLIPPAGE_ESTIMATE };
```

## Card Payload Addition

Every card payload that went through the gate should include:
```js
execution_gate: {
  evaluated: true,
  should_bet: true|false,
  net_edge: 0.031,
  blocked_by: [],
  model_status: 'MODEL_OK',
  snapshot_age_ms: 45000,
  evaluated_at: '2026-04-06T14:30:00Z',
}
```

Cards with `should_bet: false` are written with status `PASS` (not FIRE/PLAY) and flagged in the payload.
