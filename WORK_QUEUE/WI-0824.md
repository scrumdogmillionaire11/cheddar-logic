# WI-0824: Two-layer execution gate — block non-positive executable edge cards

**ID**: WI-0824
**Goal**: Enforce a strict two-layer execution gate so cards can only fire when model status is healthy and executable net edge is positive; blocked cards must persist as PASS with `execution_gate.should_bet=false` across NBA, MLB, and NHL.

**Decision lock (contract)**:

- Strict WI-0824 semantics apply for all sports: when execution gate blocks an otherwise executable candidate, persist as PASS with `execution_gate.should_bet=false`.
- NHL stale/mixed-book blocked branches must not emit `WATCH/HOLD/LEAN`.

**Must-haves (goal-backward)**:

Truths:

- A card with non-positive executable net edge never fires.
- Blocked executable candidates are persisted as PASS, not WATCH/HOLD.
- Persisted payloads include machine-readable `execution_gate` metadata explaining why firing was blocked.
- Gate behavior is consistent across NBA, MLB, and NHL.

Artifacts:

- `apps/worker/src/jobs/execution-gate.js` defines and exports gate evaluation contract.
- `apps/worker/src/jobs/run_nba_model.js` applies gate before card write and demotes blocked outcomes to PASS.
- `apps/worker/src/jobs/run_mlb_model.js` applies gate before card write and demotes blocked outcomes to PASS.
- `apps/worker/src/jobs/run_nhl_model.js` applies gate before card write and demotes blocked outcomes to PASS (including stale/mixed-book branch).
- Test suites under `apps/worker/src/jobs/__tests__/` assert gating behavior and persisted decision/status/action semantics.

Key links:

- `evaluateExecution(...)` output -> runner decision mapping (`should_bet=false` => PASS contract).
- Runner mapping -> DB payload write path includes `execution_gate` object.
- Snapshot timing (`captured_at` fallback `fetched_at`) -> stale detection branch -> PASS demotion path.

**Implementation plan (patching blockers)**:

1. Contract hardening and interface consistency

- Ensure `evaluateExecution` has a stable return shape consumed by all runners:
  - `should_bet`, `reason_code`, `net_edge`, `confidence_ok`, `price_fresh`, `model_ok`, `book_alignment_ok`, `snapshot_age_seconds`.
- Keep stale detection using `captured_at` with `fetched_at` fallback and include resolved age in gate metadata.

1. Runner enforcement (NBA/MLB/NHL)

- In each runner, evaluate gate before any card payload insert/update.
- If gate blocks an otherwise executable candidate, force:
  - `decision = PASS`
  - `action = PASS`
  - `status = PASS`
  - `execution_gate.should_bet = false`
- NHL-specific patch: replace stale/mixed-book `WATCH/HOLD/LEAN` branch with strict PASS contract.

1. Metadata persistence

- Ensure `execution_gate` is written on both evaluated path and early-exit path, including reason and timing data.
- Guarantee persisted payload fields are present for downstream diagnostics.

1. Regression coverage

- Add or update tests to assert:
  - Non-positive net edge yields `should_bet=false`.
  - NBA/MLB/NHL blocked candidates persist as PASS.
  - NHL stale/mixed-book blocked path no longer emits WATCH/HOLD.
  - Early-exit and normal execution paths both persist `execution_gate` metadata.

**Acceptance**:

- Any candidate with `net_edge <= 0` is not fired and is persisted with PASS contract.
- Any blocked executable candidate has `execution_gate.should_bet=false` in persisted payload.
- NBA, MLB, and NHL runners emit identical PASS semantics for blocked outcomes.
- NHL stale/mixed-book gate block no longer writes `action=HOLD` / `status=WATCH` / `classification=LEAN`.
- Worker test suites covering gate and runner semantics pass.
