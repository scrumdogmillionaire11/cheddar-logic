# WI-0895: Add PASS Visibility + Best-Odds Transparency in Main Cards View

**ID**: WI-0895
**Goal**: Make non-actionable outcomes visible on demand in game mode and guarantee odds-backed cards clearly surface the best available odds/line, including which sportsbook provides each value and the exact numbers shown to users.

**Depends on**: WI-0893

**Scope**:
- web/src/lib/game-card/filters.ts
- web/src/components/cards/shared.ts
- web/src/components/cards/CardsPageContext.tsx
- web/src/components/cards/CardsHeader.tsx
- web/src/components/cards/GameCardItem.tsx
- web/src/components/cards/game-card-helpers.tsx
- web/src/__tests__/filters-pass-play-main-view-regression.test.js
- web/src/__tests__/cards-best-odds-display.test.js

**Out of scope**:
- Changing default production stance to always show PASS in game mode
- Modifying model runner pass reason generation
- Props mode filtering defaults
- Re-ranking model confidence logic (this item is about presentation and source transparency of already-selected odds-backed plays)

**Locked decisions for this WI**:
- Main cards remain FIRE/WATCH by default; PASS remains opt-in via explicit UI control.
- Odds-backed cards must present best available values with sportsbook attribution.
- UI must show concrete numbers (line and American odds where applicable), not generic "best" labels.
- If input data is incomplete for a best-claim, UI must degrade safely and never present an unverified "best" label.

**Deterministic best-value policy (must implement exactly)**:
- Data eligibility:
  - A value is eligible only when numeric value and source book are both present.
  - Missing numeric or missing book disqualifies that candidate from "best" selection.
- Market ranking:
  - MONEYLINE: choose bestPrice only (most favorable American odds for chosen side).
  - SPREAD/PUCKLINE: choose bestLine first for chosen side; then attach paired price if available.
  - TOTAL: choose bestLine first for OVER/UNDER choice; then attach paired price if available.
- Tie-breakers (strict order):
  1. Better actionable value (market rule above)
  2. Better paired price when primary value ties
  3. Newer odds timestamp when available
  4. Stable lexical book key order as final deterministic fallback
- Split source handling:
  - If bestLine and bestPrice are from different books, preserve and display both books and both numbers.
- No-claim fallback:
  - When policy cannot produce an eligible best value, card must not display "best" claim text.

**Implementation plan**:
1. PASS visibility controls and diagnostics wiring
- Files: web/src/lib/game-card/filters.ts, web/src/components/cards/CardsPageContext.tsx, web/src/components/cards/CardsHeader.tsx, web/src/components/cards/shared.ts
- Add or confirm first-class include-PASS toggle state in main cards mode.
- Preserve default FIRE/WATCH filtering when toggle is off.
- When enabled, include PASS deterministically and expose reason buckets at header and diagnostic level.
- Ensure diagnostics separate odds-blocked, data-error, and projection-only outcomes.

2. Input contract validation + best-odds helper contract
- Files: web/src/components/cards/game-card-helpers.tsx, web/src/components/cards/shared.ts
- Add helper contract that resolves display payload for odds-backed cards:
  - bestPrice: numeric American odds + priceBook
  - bestLine: numeric line + lineBook (when market supports line shopping)
  - displayValues: exact values to render in card summary
  - eligibility flags describing why a best claim is allowed or suppressed
- Validate required fields before best-claim rendering:
  - numeric candidate exists
  - source book exists
  - market-side context is resolvable
- Implement deterministic policy and tie-breakers exactly as defined in this WI.

3. Card UI rendering for transparency
- Files: web/src/components/cards/GameCardItem.tsx
- Render explicit "Best" labels only when eligibility flags allow it.
- Render exact numeric values from helper payload with sportsbook attribution:
  - best price book
  - best line book (if different)
- For split-source outcomes, render both books and both numbers in one clear block.
- Avoid ambiguous copy; card must always display the actual number tied to any "best" claim.

4. Regression and behavior tests (required matrix)
- Files: web/src/__tests__/filters-pass-play-main-view-regression.test.js, web/src/__tests__/cards-best-odds-display.test.js
- Keep or extend FIRE/WATCH default exclusion regression.
- Add deterministic matrix for odds-backed rendering:
  - MONEYLINE HOME and AWAY: bestPrice + book shown
  - SPREAD/PUCKLINE HOME and AWAY: bestLine + paired price + book labels
  - TOTAL OVER and UNDER: bestLine + paired price + book labels
  - split-source case: bestLine and bestPrice from different books displayed clearly
  - tie cases: equal primary value resolved by paired price, then timestamp, then stable lexical fallback
  - incomplete input: no "best" claim when numeric or book data is missing

**Acceptance**:
- Game mode keeps FIRE/WATCH defaults but offers a first-class toggle to include PASS with reason buckets.
- Diagnostics identify odds-blocked vs data-error vs projection-only reasons.
- Existing regression for default FIRE/WATCH exclusion remains valid.
- Enabling PASS toggle includes PASS cards deterministically.
- Every odds-backed card renders the best currently available actionable number for its market context according to deterministic policy in this WI.
- Cards explicitly label sportsbook source(s) for displayed best values (book for price, and line-book if different).
- Cards show exact displayed numbers (line and American odds) without ambiguity, using consistent formatting and no hidden inference.
- If best-line and best-price originate from different books, UI states this clearly rather than collapsing to a single unlabeled value.
- If required input is missing, cards suppress best-claim language and do not imply verified best values.
