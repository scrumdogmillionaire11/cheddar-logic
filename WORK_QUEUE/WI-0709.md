# WI-0709: Derive nextGW ceiling/floor pts from FPL data to activate posture-aware transfer scoring

**ID:** WI-0709
**Goal:** Populate `nextGW_ceiling_pts` and `nextGW_floor_pts` on player candidate objects in the data pipeline so that the `transfer_ceiling_weight` / `transfer_floor_weight` blend in `transfer_advisor.py` (introduced in WI-0707) produces posture-differentiated transfer rankings in production.

**Problem:**
`transfer_advisor._score_candidate_for_strategy` blends `ceiling_pts` and `floor_pts` using posture config weights:
```python
ceiling_pts = float(getattr(candidate, 'nextGW_ceiling_pts', None) or ceiling)
floor_pts   = float(getattr(candidate, 'nextGW_floor_pts',  None) or floor)
effective_pts = posture_cfg.transfer_ceiling_weight * ceiling_pts + posture_cfg.transfer_floor_weight * floor_pts
```
Both attributes are never set anywhere in the pipeline — both `getattr` calls return `None` — so `effective_pts` always collapses to the same `next_pts` value regardless of posture. Conservative and Aggressive users receive identical transfer rankings.

**Proposed approach:**
Derive ceiling/floor at enrichment time using a volatility spread applied to `ep_next`:
- `ceiling_pts = ep_next * (1.0 + volatility_spread)`
- `floor_pts   = ep_next * (1.0 - volatility_spread)`

`volatility_spread` can be approximated from existing data:
- Use `form` variance across last N GWs if available
- Fall back to a per-position constant (GK: 0.10, DEF: 0.15, MID: 0.25, FWD: 0.30)
- Clamp floor to 0.0

Enrichment point: wherever player objects are constructed/enriched before being passed to `TransferAdvisor` (likely `result_transformer.py` ~line 900+ where player dicts are assembled, or the data enrichment layer in `src/cheddar_fpl_sage/`).

**Scope:**
- `src/cheddar_fpl_sage/analysis/decision_framework/transfer_advisor.py` — no change needed (already reads these attrs)
- Enrichment layer where player candidate objects are built (identify exact file during implementation)
- `tests/test_risk_posture_impact.py` — extend Layer 3 with 3 tests that set `nextGW_ceiling_pts` / `nextGW_floor_pts` to non-trivial values and assert posture-differentiated scores (AGGRESSIVE scores ceiling-heavy candidates higher; CONSERVATIVE scores floor-heavy candidates higher)

**Out of scope:**
- Live FPL variance model or ML uncertainty estimates
- Any changes to `RiskPostureConfig` weights
- UI changes

**Acceptance:**
1. `nextGW_ceiling_pts` and `nextGW_floor_pts` are set on all player candidate objects before scoring
2. With a player whose `ceiling_pts > floor_pts` by a meaningful delta (e.g., 4 pts), AGGRESSIVE scores that player measurably higher than CONSERVATIVE in unit tests
3. 3 new Layer 3 tests in `test_risk_posture_impact.py` pass
4. Full `pytest tests/` exits 0
