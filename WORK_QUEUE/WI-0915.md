# WI-0915: News Polling Orchestration + Target Resolution + Slate Context Attachment

**ID**: WI-0915
**Goal**: Implement deterministic news polling orchestration, canonical target resolution, and runner-facing `slate_context` attachment so model jobs consume structured news signals instead of parsing raw source payloads.

**Depends on**: WI-0913

**Scope**:

- `apps/worker/src/news/fetch-news-signals.js`
- `apps/worker/src/news/resolve-news-targets.js`
- `apps/worker/src/schedulers/main.js`
- `apps/worker/src/jobs/run_nhl_model.js`
- `apps/worker/src/jobs/run_nba_model.js`
- `apps/worker/src/jobs/run_nfl_model.js`
- `apps/worker/src/jobs/run_mlb_model.js`
- `apps/worker/src/news/__tests__/fetch-news-signals.test.js`
- `apps/worker/src/news/__tests__/resolve-news-targets.test.js`
- `apps/worker/src/jobs/__tests__/run_nhl_model.test.js`
- `apps/worker/src/jobs/__tests__/run_nba_model.test.js`
- `apps/worker/src/jobs/__tests__/run_nfl_model.test.js`
- `apps/worker/src/jobs/__tests__/run_mlb_model.test.js`
- `apps/worker/src/__tests__/scheduler-main-calibration.test.js`

**Out of scope**:

- Sport-specific confidence penalties, volatility math, or execution caps
- Watchdog invalidation, card-payload validation, or final status capping
- Free-text parsing inside workflows or models
- Whole-slate recompute behavior when a resolved `game_id` is available

**Acceptance**:

- Scheduler adds explicit per-sport news polling stages ahead of model windows instead of one undifferentiated global news job.
- Polling uses only registry-defined sources and emits normalized `NewsSignal` items through the shared contract from WI-0913.
- Target resolution maps source entities to canonical `game_id`, `team_id`, `player_id`, and affected market identifiers when resolvable; ambiguous mappings remain non-confirming warnings.
- Runner inputs attach normalized news under shared `slate_context` / `game.stats.news` ownership rather than ad hoc job payload fields.
- If a critical signal resolves to a specific game, the scheduler supports impacted-game-only refresh instead of full-slate recomputation.
- No workflow or model file in scope parses raw headline/body text directly.
