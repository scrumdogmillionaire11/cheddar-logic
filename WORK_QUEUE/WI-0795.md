# WI-0795: AUTH_SECRET placeholder — startup assertion + prod secret rotation

**ID**: WI-0795
**Goal**: Add a startup assertion in the web server that throws (or hard-warns) if `AUTH_SECRET` matches the placeholder string, and confirm the production Pi has the actual secret set.

**Scope**:

- `web/src/lib/db-init.ts` (or web server startup entry point — add assertion)
- `.env.production` (document that `AUTH_SECRET` must be replaced before deploy)

**Out of scope**:

- JWT auth implementation changes
- Token TTL or refresh logic

**Acceptance**:

- Starting the web server with `AUTH_SECRET=REPLACE_WITH_GENERATED_SECRET` throws a clear startup error (does not silently boot)
- `.env.production` has a comment noting the generation command (`openssl rand -base64 32`) and confirming the placeholder must not be used in prod
- Verified on Pi: `grep AUTH_SECRET /opt/cheddar-logic/.env.production` returns a non-placeholder value
