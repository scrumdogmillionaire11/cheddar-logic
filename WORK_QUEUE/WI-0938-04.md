# WI-0938-04: THE BOARD - Blocked Edges Tab Component

**ID**: WI-0938-04  
**Parent**: WI-0938  
**Goal**: Build the Blocked Edges Tab component that displays strong suppressed edges with exact unlock conditions. This tab builds trust by showing disciplined model restraint.

**Depends on**: WI-0938-02 (skeleton must exist first)

**Scope**:

- `web/src/components/board/BlockedEdgesTab.tsx` (main container)
- `web/src/components/board/BlockedEdgeCard.tsx` (individual blocked edge card)
- `web/src/components/board/UnlockConditionsList.tsx` (renders unlock triggers)
- Mock data integration (pull from `/api/board/blocked-edges` endpoint)
- Styling (follows existing Cheddar design system)

**Out of scope**:

- Edge reasoning enrichment (backend computes this in WI-0938-06)
- Real card payload data (uses mock until WI-0938-06)
- Discord formatting (that's WI-0938-07)

**Acceptance**:

- `BlockedEdgesTab` component renders list of blocked opportunities from `/api/board/blocked-edges`
- Each blocked edge card displays:
  - Game/Market info (e.g., "NBA: NYK @ BOS | Total 208.0")
  - Model edge (e.g., "Our model: 205.2 (+1.3% edge)")
  - State badge: "PASS (currently blocked)"
  - Why Blocked (1-2 sentences explaining suppression reason)
  - Unlock Conditions (explicitly formatted steps to transition PASS → FIRE or PASS → RECALCULATE)
  - Timeline hint if available (e.g., "Decision pending until 6:30 PM ET")
- Empty state renders when all edges are active (no blocked edges)
- Responsive design (mobile-friendly)
- TypeScript types match schema
- Unit tests for BlockedEdgeCard rendering
- Manual: Cards display correctly with mock data
