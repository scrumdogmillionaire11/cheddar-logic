# WI-1238: Formalize Undisplayed-Settled Diagnostics

**ID**: WI-1238
**Goal**: Formalize operator-facing diagnostics for settled rows that remain hidden from `/results` because they never enrolled into `card_display_log`, without changing live authority or adding repair writes from the web layer.

**Context**:

`card_display_log` remains the visibility bridge for surfaced results. When enrollment is missed, settled rows can exist in `card_results` but never appear in `/results`. The repo already has partial signals for this problem (`orphanedSettled` in `/results` meta and worker-side `DISPLAY_LOG_NOT_ENROLLED` checks), but those signals are not yet a single explicit operator contract shared across results diagnostics and worker health reporting. This WI tightens that diagnostic contract only.

**Scope**:

- `apps/worker/src/jobs/report_settlement_health.js`
- `apps/worker/src/jobs/check_pipeline_health.js`
- `web/src/lib/results/query-layer.ts`
- `apps/worker/src/__tests__/settlement-health-report.display-log-diagnostics.test.js` (new)
- `apps/worker/src/__tests__/check-pipeline-health.display-log-diagnostics.test.js` (new)
- `web/src/__tests__/results-orphaned-display-log-diagnostics.test.js` (new)
- `WORK_QUEUE/WI-1238.md`

**Out of scope**:

- Any automatic repair or backfill of `card_display_log`
- Any relaxation of `/results` display-proof semantics
- Any change to settlement-time enrollment behavior in `settle_pending_cards.js`
- Any changes to `/api/games` or `/api/cards` filtering

**Acceptance**:

- Diagnostics expose an explicit undisplayed-settled / `DISPLAY_LOG_NOT_ENROLLED` contract rather than relying on implicit meta math alone.
- Read paths expose undisplayed-settled diagnostics without mutating historical rows or widening official ledger inclusion.
- Worker health/reporting reuse the same missing-display-log bucket semantics for operator triage.
- No new web-side write path is introduced.
- Regression coverage proves orphaned settled rows are counted, sampled, and explainable while still excluded from the official ledger.
