# WI-1238: Add Card Display Log Diagnostics

**ID**: WI-1238
**Goal**: Expose settled cards that are hidden from results because they never enrolled into `card_display_log`, without changing live authority or adding repair writes from the web layer.

**Context**:

`card_display_log` remains the visibility bridge for surfaced results. When enrollment is missed, settled rows can exist in `card_results` but never appear in `/results`. Current reporting is not rich enough to explain these hidden rows quickly. This WI adds diagnostics only.

**Scope**:
- `packages/data/src/db/cards.js`
- `apps/worker/src/jobs/settle_pending_cards.js`
- `web/src/lib/results/query-layer.ts`
- `packages/data/__tests__/card-display-log-orphaned-diagnostics.test.js` (new)
- `apps/worker/src/__tests__/settle_pending_cards.display-log-diagnostics.test.js` (new)
- `web/src/__tests__/results-orphaned-display-log-diagnostics.test.js` (new)
- `WORK_QUEUE/WI-1238.md`

**Out of scope**:
- Any automatic repair or backfill of `card_display_log`
- Any relaxation of `/results` display-proof semantics
- Any changes to `/api/games` or `/api/cards` filtering

**Acceptance**:
- A diagnostic query/report can count settled cards that have no matching `card_display_log` enrollment.
- Read paths expose undisplayed-settled diagnostics without mutating historical rows.
- No new web-side write path is introduced.
- Regression coverage proves orphaned settled rows are counted and explainable.
