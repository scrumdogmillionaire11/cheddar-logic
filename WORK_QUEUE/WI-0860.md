# WI-0860: Schedule `run_calibration_report` as a nightly job

| Field | Value |
|---|---|
| **ID** | WI-0860 |
| **Goal** | `run_calibration_report` was built in WI-0825 but never wired into the scheduler. The kill switch has no auto-release path — once set it is permanently on until manually cleared. Schedule the report nightly at 04:00 ET so the kill switch auto-releases when model calibration improves. |
| **Priority** | Critical — NBA totals are currently dark in production |
| **Depends on** | WI-0825 (COMPLETE — job exists) |
| **Coordination flag** | solo |

---

## Context

`run_calibration_report.js` computes ECE per market and sets `kill_switch_active` in `calibration_reports`. It was designed as a nightly job but was never added to the scheduler. Because no scheduled path exists:

1. The kill switch fires once (manually or on first invocation) and **never rechecks**
2. After WI-0822 (NBA pace normalization), the model's `fair_prob` outputs changed — but the kill switch was set against pre-WI-0822 predictions and has not been refreshed
3. NBA totals are permanently dark unless someone SSH-es in and runs the report manually

The fix is purely additive: wire the job into the scheduler. No logic changes to `run_calibration_report.js` are needed.

---

## Scope

**Files to change:**
- `apps/worker/src/schedulers/main.js`
  - Import `runCalibrationReport` from `../jobs/run_calibration_report`
  - Add fixed-time trigger at `04:00` ET in the nightly reporting block (after `run_daily_performance_report` at 03:30)
  - Job key: `` `calibration_report|${nowEt.toISODate()}` ``
  - Reason: `'nightly calibration report + kill switch refresh'`
  - Guard: `process.env.ENABLE_SETTLEMENT !== 'false'` (same guard as CLV snapshot — both depend on resolved bet history)

**Out of scope:**
- Changes to `run_calibration_report.js` — correct as-is
- Changing ECE thresholds or `PERIOD_DAYS`
- Isotonic regression (WI-0831)

---

## Acceptance

1. `isFixedDue(nowEt, '04:00')` gates a `run_calibration_report` job in `computeDueJobs`
2. Job key is date-unique: `calibration_report|YYYY-MM-DD`
3. `npm --prefix apps/worker test -- --testPathPattern=main` passes
4. `DRY_RUN=true` at 04:00 ET logs `DRY_RUN would run calibration_report|...`
5. After deploying to prod and the 04:00 run fires: if post-WI-0822 ECE < 0.06 then `kill_switch_active` resets to 0 and NBA totals resume
