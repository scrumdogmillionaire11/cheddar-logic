# WI-0665: ActionNetwork adapter — fetch and normalize public betting splits

**ID**: WI-0665
**Goal**: Build a `packages/adapters/src/action-network.js` module that fetches public bet % and handle % from the ActionNetwork unofficial API and normalizes results to our canonical game shape, with fuzzy team-name matching to our `game_id`s.

**Background**:
ActionNetwork exposes public betting percentages (bet %, handle %, ticket %) via an unofficial endpoint (`https://api.actionnetwork.com/web/v1/game?league={sport}&date={YYYYMMDD}`) without authentication. This is the primary data source for the fade-the-public pipeline. A browser User-Agent header avoids 403s. The existing `packages/adapters/` package currently contains only `normalize.js` and `index.js` — this WI adds the ActionNetwork adapter alongside it.

**Acceptance**:
- `fetchSplitsForDate({ sport, date })` calls `https://api.actionnetwork.com/web/v1/game?league={sport}&date={date}` with `User-Agent: Mozilla/5.0`; returns raw response array
  - Supported sport keys: `NFL`, `NBA`, `NHL`, `MLB`
  - Date format: `YYYYMMDD`
  - Soft-fails (returns `[]` + logs warning) on non-200 responses; does not throw
- `normalizeSplitsResponse(raw, sport)` maps raw response to canonical shape per game:
  ```
  { actionNetworkGameId, homeTeam, awayTeam, commenceTime,
    publicBetsPctHome, publicBetsPctAway,
    publicHandlePctHome, publicHandlePctAway,
    publicTicketsPctHome, publicTicketsPctAway }
  ```
  - All pct fields are floats 0–100 or `null` if absent in the source
- `matchSplitsToGameId(normalizedSplits[], knownGames[])` fuzzy-matches on team name using the same `resolveTeamVariant` pattern used in `apps/worker/src/jobs/pull_odds_hourly.js`; returns `[{ gameId, splits }]`; unmatched games are logged as warnings
- Unit tests cover:
  - Successful normalization with all fields present
  - Missing pct fields → `null` (not `undefined` or `0`)
  - Empty API response → empty array, no throw
  - Team name mismatch → unmatched entry logged, not thrown
- `npm test --workspace=packages/adapters` passes
