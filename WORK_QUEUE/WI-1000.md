# WI-1000: Sunset EDGE_VERIFICATION_REQUIRED alias across all runtime paths

| Field               | Value |
|---------------------|-------|
| **ID**              | WI-1000 |
| **Goal**            | Remove all `EDGE_VERIFICATION_REQUIRED` alias handling from runtime logic, leaving only the canonical replacement codes introduced in WI-0998 |
| **Priority**        | High — dual-behavior risk increases every release the alias stays live |
| **Owner agent**     | Unassigned |
| **Time window**     | One release window after WI-0998 is merged and verified in staging |
| **Coordination flag** | `needs-sync` — touches shared decision-surface types and CardsPageContext |

---

## Background

WI-0998 introduced canonical blocker codes (`LINE_NOT_CONFIRMED`, `EDGE_RECHECK_PENDING`, etc.) and kept `EDGE_VERIFICATION_REQUIRED` as a fallback alias to avoid breaking in-flight payloads. That compatibility window now has a hard end date. Every branch that checks both old and new codes is dual-behavior risk and prevents clean analytics.
