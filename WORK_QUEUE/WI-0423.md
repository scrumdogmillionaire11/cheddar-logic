# WI-0423: Auto-close non-actionable final pending card results

**ID**: WI-0423
**Goal**: Prevent finished games from leaving non-actionable card_results stuck in pending by auto-closing final PASS/EVIDENCE rows during settlement.

**Scope**:

- `WORK_QUEUE/WI-0423.md`
- `apps/worker/src/jobs/settle_pending_cards.js`
- `apps/worker/src/__tests__/settle-coverage.test.js`

**Out of scope**:

- Changes to canonical grading rules for actionable locked markets
- Changes to game result ingestion/matching

- Settlement detects final-game pending rows that are non-actionable (`kind != PLAY` and/or PASS status) and auto-closes them as `status='error', result='void'` with explicit metadata reason.
- Actionable settlement path remains unchanged for valid displayed locked markets.
- Settlement logs include count of non-actionable final rows auto-closed.
- Coverage/telemetry in job response includes auto-closed non-actionable count.
- Tests cover at least one final PASS row auto-closing path.
