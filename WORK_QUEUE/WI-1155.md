**ID**: WI-1155
**Goal**: Eliminate dev/prod drift for NHL player shots cards by making the SOG lane enablement, tracked-player prerequisites, and failure states explicit and testable.

**Acceptance**:
- [WI-1155-CONTRACT-01] NHL SOG sync enablement is controlled by one canonical contract across scheduler paths; parallel or contradictory gates are removed.
- [WI-1155-PREREQ-02] If tracked players are missing and no fallback IDs exist, the worker emits an explicit, operator-visible prerequisite failure state instead of a silent no-card success path.
- [WI-1155-HEALTH-03] Health checks expose lane stage outcomes with explicit diagnostics for sync missing, logs missing, model skipped, cards suppressed, and cards published.
- [WI-1155-PARITY-04] Docs, env contract examples, and runtime behavior describe the same SOG enablement and fallback contract so dev/prod can be configured equivalently.
- [WI-1155-LEGACY-05] Dead or legacy NHL SOG enablement branches superseded by the canonical contract are removed (or explicitly documented as intentionally retained with reason).

## Must-haves (goal-backward)

**truths**:
- NHL SOG sync gating behaves identically across the NHL scheduler and player-props scheduler, with no hidden alternate path.
- Missing tracked-player prerequisites are observable as explicit failure semantics and are distinguishable from legitimate no-card suppression.
- Pipeline health output identifies exactly where NHL SOG failed in the chain, rather than collapsing all failures into generic freshness noise.
- Documentation and env examples match runtime defaults and enablement semantics for SOG sync and fallback IDs.
- Superseded legacy SOG enablement branches in scoped files are removed or intentionally retained with explicit rationale.

**artifacts**:
- `packages/data/src/feature-flags.js`, `apps/worker/src/schedulers/nhl.js`, and `apps/worker/src/schedulers/player-props.js` implement one canonical SOG sync contract.
- `apps/worker/src/jobs/pull_nhl_player_shots.js` and `apps/worker/src/jobs/run_nhl_player_shots_model.js` emit deterministic prerequisite-failure semantics when IDs/logs are missing.
- `apps/worker/src/jobs/check_pipeline_health.js` reports NHL SOG lane stage diagnostics covering sync/log/model/card states.
- `docs/NHL_PLAYER_SHOTS_PROP_MARKET.md` and `env.example` document the same contract used at runtime.

**key_links**:
- Canonical feature flag contract -> both scheduler paths (`nhl.js` and `player-props.js`) for `sync_nhl_sog_player_ids` enqueue behavior.
- Tracked-player prerequisite check -> explicit job result semantics -> pipeline health diagnostics and operator alert visibility.
- Pull job output/log freshness -> model-run preconditions -> card publish/suppress diagnostics.
- Runtime defaults -> docs table -> env example values for `ENABLE_NHL_SOG_PLAYER_SYNC` and fallback ID behavior.
- Legacy branch removals -> grep/test evidence proving only the canonical SOG enablement path remains.
