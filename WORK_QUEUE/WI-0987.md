# WI-0987: Centralize Betting Decision Status Normalization

**ID**: WI-0987
**Goal**: Replace duplicated betting decision status normalization logic with one shared contract for display-log eligibility, settlement backfill, and one-shot repair tooling.

**Depends on**: WI-0982

**Scope**:

- `packages/data/src/decision-status.js`
- `packages/data/src/index.js`
- `packages/data/src/db/cards.js`
- `packages/data/__tests__/decision-status.test.js`
- `packages/data/__tests__/card-display-log-play-selection.test.js`
- `apps/worker/src/jobs/settle_pending_cards.js`
- `apps/worker/src/jobs/repair_mlb_full_game_display_log.js`
- `apps/worker/src/jobs/__tests__/settle_pending_cards.market-contract.test.js`

**Out of scope**:

- Changing `/api/results` display-log scoping
- Changing which statuses are actionable
- Expanding MLB historical repair eligibility beyond the WI-0982 contract
- Adding new DB migrations or modifying production data
- Refactoring unrelated settlement markets or card filtering behavior

**Acceptance**:

- A shared data-package helper defines the strict decision-status contract used by producer display-log eligibility, settlement backfill logic, and MLB repair tooling.
- `decision_v2.official_status` remains authoritative when present, including treating unknown explicit values as non-actionable instead of falling back to legacy fields.
- Legacy fallback remains strict and unchanged: `PLAY`/`FIRE` -> `PLAY`, `LEAN` -> `LEAN`, and `PASS`/`WATCH`/`HOLD`/missing/unknown -> non-actionable.
- MLB repair eligibility continues to require legacy `LEAN` plus `execution_status=EXECUTABLE`.
- Existing WI-0982 behavior is preserved while removing duplicated local implementations in `cards.js`, `settle_pending_cards.js`, and `repair_mlb_full_game_display_log.js`.
- Tests cover the shared helper directly and verify all current display-log, settlement, and repair regression cases still pass.
