# WI-0811: Book-to-Book Mispricing Scanner

| Field | Value |
|---|---|
| **ID** | WI-0811 |
| **Goal** | Create a deterministic cross-book mispricing scanner that groups same-event same-market prices, builds a consensus reference, and emits normalized `MispricingCandidate` objects when a configurable threshold is breached — with no recommendation logic. |
| **Priority** | Medium |
| **Depends on** | Existing odds snapshot ingestion; canonical market normalization for spread/total/ML |
| **Coordination flag** | `needs-sync` if scheduler or shared DB/query layers are touched |

## Background

The codebase currently mixes model edge, market structure, and cross-book price disagreement in the same flows. A dedicated mispricing scanner separates those concerns, giving downstream workflows a clean, objective upstream primitive:

> **"Book A is materially off-market relative to the rest."**

This layer is reusable, independent of public-splits or projection logic, and must never produce bet recommendations.

## Core Algorithm

### Step 1 — Normalize comparable markets

Only compare like-for-like:
- **Spread**: same `game_id`, same side
- **Total**: same `game_id`, same over/under
- **ML**: same `game_id`, same side
- **Props**: only if canonical player + stat + line match exactly (excluded from v1 unless already solid)

### Step 2 — Build consensus reference

For each comparable market group:
- Exclude target book from reference pool
- Require minimum **2 distinct books** (configurable, default 2)
- Compute:
  - Median line across reference books
  - Median price across reference books
  - Optional: sharp-book-only subset (config flag, not required v1)

### Step 3 — Apply threshold rules

| Market | Watch | Trigger |
|--------|-------|---------|
| Spread | line delta ≥ 0.5 | line delta ≥ 1.0 |
| Total | line delta ≥ 0.5 | line delta ≥ 1.0 |
| ML (near even) | 10¢ implied spread | 20¢ implied spread |
| ML (big fav/dog) | implied edge ≥ 3% | implied edge ≥ 5% |
| Props | n/a v1 — skip | n/a v1 — skip |

### Step 4 — Classify

- `NONE` — below watch threshold
- `WATCH` — at or above watch, below trigger
- `TRIGGER` — at or above trigger

### Step 5 — Emit candidate object

No recommendation language. Output only:

```ts
type MispricingCandidate = {
  game_id: string
  sport: string
  market_type: "SPREAD" | "TOTAL" | "ML" | "PROP"
  selection: string
  source_book: string
  consensus_books: string[]
  source_line: number | null
  source_price: number | null
  consensus_line: number | null
  consensus_price: number | null
  edge_type: "LINE" | "PRICE" | "HYBRID"
  stale_delta: number | null
  implied_edge_pct: number | null
  threshold_class: "NONE" | "WATCH" | "TRIGGER"
  reason_codes: string[]
  captured_at: string
}
```

## Invariants

1. This flow detects **mispricing only**
2. It does **not** decide plays
3. It does **not** use public splits
4. It does **not** use team/player projections
5. It must require minimum book coverage (reject if < 2 comparison books)
6. It must refuse ambiguous market comparisons (mixed lines, stale snapshots outside recency window)

---

## Failure Guards

| Failure mode | Guard |
|---|---|
| Thin market / bad consensus | Require minimum distinct book count; use median not mean |
| Stale snapshots | Enforce recency window (configurable, e.g. 30 min); check timestamp skew |
| Mixed-market contamination | Strict canonical comparison key (game_id + market_type + selection + line) |
| Scanner mistaken for rec engine | Output schema uses no "play/bet" language; downstream must explicitly consume |
| Missing prices | Skip row, log warning, do not fabricate |
| Duplicate same-book rows | Deduplicate by (game_id, market_type, selection, book) before consensus |
| Malformed lines | Validate numeric; skip and log |
| Only one comparison book | Return no candidate (below minimum coverage) |

---

## Acceptance

- Scanner groups same-event same-market same-selection prices across books
- Scanner computes consensus from books excluding source book
- Scanner emits candidate only when configured threshold is breached
- Scanner supports spread line delta, total line delta, and moneyline price mispricing
- Scanner rejects: insufficient books, mixed lines, stale snapshots outside recency window
- Output includes source book, consensus reference, stale delta, implied edge, threshold class, reason codes
- Job logs candidate count grouped by sport and market type
- No consumer is forced to treat candidates as official plays

## Tests to Run

**Unit tests** (`packages/models`)

- Spread 0.5-point delta → `threshold_class: "WATCH"`
- Spread 1.0-point delta → `threshold_class: "TRIGGER"`
- ML 15¢ implied spread → triggers at near-even price
- Fewer than 2 comparison books → returns no candidate
- Snapshot outside recency window → ignored
- Mixed market types → not compared
- Consensus excludes target book

**Smoke tests**

- One event with 4 books → expected candidate set produced
- Single sport run → logs counts by market type with no crash
- Aligned market → no candidate emitted

**Failure cases**

- Missing prices → skipped, no crash
- Duplicate same-book rows → deduped correctly
- Malformed line value → skipped with warning
- Unsupported market type → rejected gracefully
- Inconsistent timestamps → stale delta computed or row excluded

```
npm --prefix packages/models run test -- --no-coverage
npm --prefix apps/worker run test -- --no-coverage
```

- Confirm no candidate object contains the words "bet", "play", or "recommend"
