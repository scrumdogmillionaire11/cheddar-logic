# WI-1132: Distributed Rate Limiter Migration

**ID**: WI-1132

**Goal**: Replace in-memory process-local rate limiting with distributed rate limiting suitable for multi-instance deployment.

**Requirements**:

- WI-1132-RL-03: Existing API-facing rate-limit headers remain stable.

**Implementation Notes** (for executor reference):
- **Abstraction pattern**: Create `DistributedRateLimiter` class with backend interface; in-memory and redis strategies implement interface
- **Backend selection**: Use `RATE_LIMITER_BACKEND` env var (default: 'memory' for dev, 'redis' for production)
- **Counter key format**: `{identifier}:{window_start_ms}` to enable distributed query
- **Fallback identity**: If X-Forwarded-For parsing fails, use User-Agent + timestamp hash (avoids global user-agent collisions)
- **Headers**: Return consistent `Retry-After`, `X-RateLimit-Limit`, `X-RateLimit-Remaining`, `X-RateLimit-Reset` across all backends
