# WI-1115: Projection Confidence Bucket Attribution Contract (Payload -> Eval -> API)

**ID**: WI-1115
**Goal**: Expose a deterministic, per-card attribution contract that maps confidence buckets to the payload-derived fields used by the projection confidence engine.

**Scope**:

- `packages/data/src/db/projection-accuracy.js`
- `packages/data/__tests__/projection-accuracy-engine.test.js`
- `web/src/app/api/results/projection-accuracy/route.ts`
- `web/src/lib/types/projection-accuracy.ts`
- `web/src/__tests__/api-results-projection-accuracy.test.js`

**Out of scope**:

- Any UI rendering changes on `/results`
- Changes to directional grading policy or confidence thresholds
- New DB migrations unless proven strictly required for backward-compatible read surface

**Acceptance**:

- API response includes per-row attribution fields needed to explain bucket placement, including at minimum: `projection_raw`, `synthetic_line`, `edge_distance`, `weak_direction_flag`, `direction_strength`, `projection_confidence`, and `confidence_band`.
- `edge_distance` is returned as a numeric absolute value derived from `abs(projection_raw - synthetic_line)` and is covered by API contract tests.
- Attribution contract is deterministic and sourced from materialized eval data (not ad-hoc web-side payload recomputation).
- Existing behavior that excludes weak directions from directional W/L but keeps MAE/bias auditing intact remains unchanged.
- Tests cover both data-layer and API-layer contract shape for the attribution fields.

**Manual validation**:

- Pick one sample card and verify bucket assignment from returned fields matches engine thresholds (LOW/WATCH/TRUST/STRONG).
- Confirm `edge_distance` equals `abs(projection_raw - synthetic_line)` for the sampled row.
- Confirm no web-side write calls are introduced (read-only DB contract preserved).
