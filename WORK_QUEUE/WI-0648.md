# WI-0648: MLB empirical recalibration gate (2026 season data)

**ID**: WI-0648
**Goal**: Add a `MIN_MLB_GAMES_FOR_RECAL` gate (default: 20 games per team) that triggers a recalibration of MLB run-line and totals sigma/mu via `computeSigmaFromHistory` once sufficient 2026 season data accumulates, replacing the current pre-season defaults.

**Background**:
`computeSigmaFromHistory` was wired for NBA (WI-0552 ✓) but MLB still uses hardcoded sigma defaults (`MLB_SIGMA_DEFAULT`). The 2026 MLB season started. After ~3 weeks (GW3–4), there will be enough game history to compute an empirical sigma per team that will significantly improve projection accuracy — especially for park-factor and bullpen-usage contexts that are calibrated off of 2025 data.

**Scope**:
- `apps/worker/src/jobs/run_mlb_model.js`
- `packages/data/src/sigma-defaults.js` (or equivalent — add `computeMLBSigmaFromHistory`)
- `apps/worker/src/__tests__/run-mlb-model.dual-run.test.js` (extend with recal-gate scenario)

**Out of scope**:
- Park-factor data ingestion (separate WI)
- F5 sigma recalibration (can follow in a subsequent WI once this gate is validated)
- Pitcher K model recalibration (different data source)

**Acceptance**:
- When game count per team in `game_results` for 2026 season >= `MIN_MLB_GAMES_FOR_RECAL`, `run_mlb_model.js` calls `computeSigmaFromHistory` and uses the result instead of `MLB_SIGMA_DEFAULT`
- When game count < threshold, falls back to default and logs `[MLB_SIGMA_PRESEASON_DEFAULT]`
- When computed sigma is used, logs `[MLB_SIGMA_EMPIRICAL] team=<team> sigma=<val> n=<count>`
- `npm --prefix apps/worker test -- --testPathPattern=run-mlb-model` passes including gate scenarios
