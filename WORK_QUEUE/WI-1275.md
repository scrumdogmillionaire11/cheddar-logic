# 🔧 **WI-1275: Market Pulse → Actionable Price Discrepancy Engine**

````md
# WI-1275: Upgrade Market Pulse to Actionable Price Discrepancy Engine

## ID
WI-1275

## Goal
Transform Market Pulse from a passive “price difference list” into an actionable edge-detection surface that highlights exploitable pricing inefficiencies across books.

---

## Problem

Current Market Pulse:
- shows price differences
- shows implied spread %
- does NOT tell user:
  - if it’s bettable
  - which side is the edge
  - if it’s stale vs real
  - if it’s noise vs opportunity

Result:
→ informational, not actionable
→ low user trust + low engagement

---

## Core Upgrade

Convert discrepancies into **classified opportunities**:

| Tier | Meaning |
|------|--------|
| 🔴 STRONG EDGE | clear exploitable pricing gap |
| 🟡 LEAN EDGE | small but real discrepancy |
| ⚪ NOISE | market inefficiency but not actionable |
| ⚠️ STALE | likely already gone / unreliable |

---

## Scope

- `web/src/components/market-pulse/*`
- `web/src/lib/market-pulse/*` (new)
- `apps/worker/src/jobs/market-pulse-aggregator.js` (or equivalent source)
- `web/src/__tests__/market-pulse.test.ts` (new)

---

## Out of Scope

- Live odds ingestion changes
- Arbitrage execution
- Notifications/alerts (future WI)
- User personalization

---

## Feature Enhancements

### 1. Edge Classification Layer

Replace:
> "3.2% implied spread"

With:

```js
edge_score = best_price_vs_worst_price_implied_prob_diff
````

Then classify:

```text
>= 3.5% → STRONG EDGE
2.0%–3.49% → LEAN EDGE
1.0%–1.99% → NOISE
<1.0% → HIDE (optional)
```

---

### 2. Directional Recommendation

Instead of:

> fanduel -110 vs dk -125

Show:

```text
BET: Over 7 @ FanDuel (-110)
Market: DraftKings (-125)
Edge: +3.2%
```

Must clearly answer:
👉 **where to bet**
👉 **why**

---

### 3. Book Sharpness Weighting (Phase 1 Light)

Add simple weighting:

* Pinnacle / Circa (if present) = anchor
* If soft book vs soft book → downgrade confidence

Output:

```text
Confidence: HIGH / MEDIUM / LOW
```

---

### 4. Staleness Guard

Use timestamp:

```text
if last_update > 5 min → STALE
if odds diverging rapidly → VOLATILE
```

UI:

```text
⚠️ Possibly stale — verify before betting
```

---

### 5. De-duplication Logic

Current issue:
Same market appears multiple times (over/under both sides)

Fix:

* group by:

  * game_id
  * market_type
  * line
* show best edge only per group

---

### 6. UI Improvements

Each card should show:

```text
[EDGE BADGE]

Market: Total 7.0
Play: OVER
Best Book: FanDuel (-110)
Worst Book: DraftKings (-125)

Edge: +3.2%
Confidence: Medium
Time: 7 min ago
```

Replace generic “PRICE” tag with:

* EDGE
* LEAN
* STALE

---

### 7. Filtering Controls

Add:

* Edge tier filter (Strong / Lean / All)
* Min edge % slider (future-ready)
* Toggle: Show stale / hide stale

---

## Acceptance

* Discrepancies are classified into tiers (STRONG / LEAN / NOISE)

  * recommended side
  * best book

* Duplicate market entries are reduced to best edge

* UI no longer shows raw “price difference” without interpretation
