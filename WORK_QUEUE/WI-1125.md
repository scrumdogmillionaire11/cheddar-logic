# WI-1125: Trusted Client IP And Token Route Hardening

**ID**: WI-1125
**Goal**: Prevent spoofed client identity by hardening client IP derivation and token-route allowlist checks.

**Scope**:
- web/src/lib/api-security/rate-limiter.ts
- web/src/app/api/auth/token/route.ts
- web/src/lib/api-security/audit-logger.ts
- web/src/lib/api-security/index.ts
- web/src/__tests__/rate-limiter-trusted-proxy.test.js
- web/src/__tests__/api-auth-token-allowlist.test.js
- web/src/__tests__/api-audit-log-redaction.test.js

**Out of scope**:
- Full distributed rate limiter migration
- CDN or edge platform networking configuration changes

**Acceptance**:
- Client IP extraction uses trusted proxy rules and rejects untrusted forwarded header chains.
- Token route allowlist logic cannot be bypassed via forged forwarding headers.
- Audit logs capture allowlist deny reasons without leaking sensitive values.
- Automated coverage asserts deny-path audit payloads redact sensitive token/header values while preserving deny reason classification.
