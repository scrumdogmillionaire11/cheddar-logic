# WI-0733: MLB Pitcher K distribution model rebuild

**ID**: WI-0733
**Goal**: Replace raw MLB pitcher strikeout projections with a batter-faced and opponent-adjusted K mean plus over-line probability distribution, while enforcing PASS semantics when inputs are missing or edge is insufficient.

**Scope**:
- `WORK_QUEUE/WI-0733.md`
- `apps/worker/src/models/mlb-model.js`
- `apps/worker/src/jobs/run_mlb_model.js`
- `apps/worker/src/jobs/pull_mlb_pitcher_stats.js`
- `apps/worker/src/jobs/__tests__/run_mlb_model.test.js`
- `apps/worker/src/__tests__/run-mlb-model.dual-run.test.js`
- `packages/data/src/validators/card-payload.js`
- `docs/decisions/ADR-0008-mlb-pitcher-k-distribution-contract.md`

**Out of scope**:
- MLB F5 model changes
- Web card rendering changes
- Promotion of MLB pitcher K cards from current rollout state to a broader live rollout
- Repo-wide formatting or unrelated cleanup

**Acceptance**:
- Pitcher K projections are derived from expected batters faced and adjusted K% versus opponent handedness splits, not raw K/9 alone.
- Pitcher K payloads include a probability distribution for at least `P(5+)`, `P(6+)`, and `P(7+)` plus the source and missing-input metadata needed for PASS gating.
- Synthetic or incomplete-input pitcher K projections cannot emit PLAY/LEAN semantics and remain visible only as PASS / projection-only outputs.
- No-edge pitcher K outcomes are not promoted into actionable cards when probability edge is below the configured threshold.
- ADR-0008 documents the pitcher K model contract, fallback policy, and rollout constraints.

**Owner agent**: unassigned
**Time window**: TBD -> TBD
**Coordination flag**: needs-sync

**Tests to run**:
- `npm --prefix apps/worker test -- --runInBand --testPathPattern=run_mlb_model.test.js`
- `npm --prefix apps/worker test -- --runInBand --testPathPattern=run-mlb-model.dual-run.test.js`

**Manual validation**:
- Inspect one odds-backed pitcher K payload and one projection-only fallback payload; confirm K mean, threshold probabilities, source, missing inputs, and PASS semantics match the contract.
- Confirm no synthetic fallback pitcher K card renders as PLAY/LEAN or executable.

**Guard for WI closeout**:

- Before marking complete, verify every test command above still exists and is runnable from repo root.
