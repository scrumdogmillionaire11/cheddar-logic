# WI-0531: Rolling PP rate splits (L10/L5) + role stability weighting

**ID**: WI-0531
**Goal**: Extend `pp_rate_per60` from season-level only to L10/L5 rolling splits and add PP role stability weighting so recent PP usage trends are reflected in projections.

**Owner agent**: unassigned
**Time window**: TBD
**Coordination flag**: solo
**Depends on**: WI-0530 (season-level pp_rate_per60 must be live)

---

## Context

WI-0530 wires season-level `pp_rate_per60` into the model. Season averages are a reasonable baseline but they miss:

1. **Trend**: A player who recently moved to PP1 will be undervalued by season averages
2. **Role instability**: A player who lost PP time due to injury/lineup change will be overvalued
3. **Recency**: Late-season PP deployment shifts matter more than October games

The same rolling window logic used for EV shot rates (`l5RatePer60`, `l10RatePer60`) should apply to PP rates.

---

## Scope

- `apps/worker/src/jobs/ingest_nst_pp_rates.js` — extend to store L10/L5 splits (requires game-log level NST data or rolling compute from stored logs)
- `apps/worker/src/models/nhl-player-shots.js` — extend `projectSogV2` PP rate input to use weighted blend of season + L10 + L5 (matching existing EV rate blend pattern)
- `apps/worker/src/jobs/run_nhl_player_shots_model.js` — surface `pp_l5_rate`, `pp_l10_rate`, `pp_season_rate` in drivers block
- DB: extend `player_pp_rates` schema with `pp_l5_shots_per60`, `pp_l10_shots_per60` columns

### Out of scope

- PP matchup adjustments (WI-0532)
- Changes to EV rate blending
- UI changes

---

## Acceptance

- [ ] `player_pp_rates` stores `pp_shots_per60`, `pp_l10_shots_per60`, `pp_l5_shots_per60`
- [ ] `projectSogV2` PP rate input uses weighted blend: season (0.4) + L10 (0.35) + L5 (0.25) — matching EV rate blend weights
- [ ] When L5/L10 are absent (new player, small sample), fall back to season rate
- [ ] `PP_SMALL_SAMPLE` flag pushed when fewer than 5 games of PP data available
- [ ] `drivers` block surfaces `pp_l5_rate`, `pp_l10_rate`, `pp_season_rate`, `pp_blend_rate`
- [ ] Tests: player with rising L5 PP rate gets higher projection than season-only; player with falling L5 gets lower
- [ ] Tests: missing L5 falls back gracefully, no crash
