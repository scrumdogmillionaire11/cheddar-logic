# WI-1101: POTD Near-Miss UI - Link Settled Metric to Play Log

**ID:** WI-1101

**Goal:** Add a clickable link on the Near-Miss Tracking "Settled" metric block that navigates users to the POTD play log section so settled counts are immediately explorable.

**Scope:**

- `web/src/components/play-of-the-day-client.tsx`
- `web/src/app/play-of-the-day/page.tsx` (only if needed for anchor target wiring)

**Out of scope:**

- Any changes to settlement/grading data logic
- Any schema/database changes
- Redesign of POTD page layout beyond minimal link affordance

**Acceptance:**

- The "Settled" metric block in Near-Miss Tracking is visibly interactive (link/button affordance).
- Clicking "Settled" takes the user to the POTD play log section (existing history/log view).
- Navigation works on desktop and mobile.
- No regression to existing POTD page behavior or API contracts.

**Owner agent:** copilot

**Time window:** 0.5 day

**Coordination flag:** none

**Tests to run:**

- `node web/src/__tests__/ui-potd-smoke.test.js`
- `node web/src/__tests__/api-potd.test.js`

**Manual validation:**

- Open `/play-of-the-day`.
- In Near-Miss Tracking, click the "Settled" block.
- Confirm viewport navigates to the POTD play log section.
- Confirm keyboard navigation (Tab + Enter) activates the link.

## Implementation Notes

- Preferred implementation: anchor link from the Settled metric to a stable id on the play log section.
- Keep existing card styling; add only minimal focus/hover states needed for accessibility.
- If no reliable in-page anchor exists, route to the closest existing POTD log route/section and document the fallback in PR notes.
