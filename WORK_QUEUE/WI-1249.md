# WI-1249: Guard Game-Surface Transform Against INFO Primary Plays

**ID**: WI-1249
**Goal**: Lock in the current `isRenderableGameSurfacePlay` hardening so the main game-card transform can never promote an `INFO` play into the primary game surface.

**Context**:

Recent NBA failures proved the backend can emit 4-5 plays for a game and still collapse to an `INFO`-only surface. The current transform-side fix appears to block that on the main surface, but the protection is implicit. This WI converts it into an explicit regression contract.

**Scope**:
- `web/src/__tests__/integration/games-pipeline-contract.test.ts`
- `web/src/lib/game-card/transform/index.ts`
- `WORK_QUEUE/WI-1249.md`

**Out of scope**:
- Adding new worker-side health checks
- Filtering `/api/games` server-side
- Changing model-generation logic for NBA or MLB

**Acceptance**:
- `games-pipeline-contract.test.ts` asserts that transformed main-surface cards never expose a primary play whose `market_type` is `INFO`.
- Any transform change is minimal and limited to preserving the existing `isRenderableGameSurfacePlay` contract.
- The regression fails if an `INFO`-only game can again surface as a primary game card.
