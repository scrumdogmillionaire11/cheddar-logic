# WI-1110: Fix NHL projection settlement — populate NHL gamecenter IDs in `game_id_map`

**ID**: WI-1110
**Goal**: `nhl-pace-1p` projection cards never settle because `resolveNhlGamecenterId` finds no matching rows in `game_id_map` (only ESPN IDs exist, queried provider is `'nhl'/'nhl_api'/'nhl_gamecenter'`), and the UUID game IDs fail the pure-numeric fallback. Add a job that fetches NHL API game IDs from the NHL schedule API and writes them to `game_id_map` with `provider='nhl_gamecenter'`, keyed by canonical `game_id`.

**Depends on**: none

**Scope**:
- `apps/worker/src/jobs/pull_nhl_game_ids.js` (new)

- `apps/worker/src/jobs/__tests__/pull_nhl_game_ids.test.js` (new)

**Out of scope**:

- NHL model or odds jobs

## Root cause

`fetchNhlSettlementSnapshot` calls `https://api-web.nhle.com/v1/gamecenter/{nhlGameId}` and requires a **numeric NHL API game ID** (e.g. `2025021234`).

`resolveNhlGamecenterId` queries:
```sql
SELECT external_game_id FROM game_id_map
WHERE sport = 'nhl' AND provider IN ('nhl', 'nhl_api', 'nhl_gamecenter')
AND game_id = ?
```

`game_id_map` only has 217 rows with `provider='espn'`. No job has ever written NHL API game IDs. The fallback (pure-numeric `game_id`) fails because canonical game IDs are UUIDs.

## Acceptance

### 1) New job: `pull_nhl_game_ids.js`

Fetch NHL schedule for a date range (default: today ± 7 days) from:
```
https://api-web.nhle.com/v1/schedule/{YYYY-MM-DD}
```

Response shape (NHL API v1):
```js
data.gameWeek[].games[].id          // numeric NHL game ID (e.g. 2025021234)
data.gameWeek[].games[].gameDate    // "YYYY-MM-DD"
data.gameWeek[].games[].homeTeam.abbrev
data.gameWeek[].games[].awayTeam.abbrev
data.gameWeek[].games[].season      // e.g. 20252026
```

For each game fetched, match it to a canonical `game_id` in the `games` table using:
```sql
SELECT game_id FROM games
WHERE sport = 'icehockey_nhl'
  AND DATE(game_time_utc) = ?   -- gameDate
  AND home_team LIKE ?           -- homeTeam.abbrev partial match or exact
  AND away_team LIKE ?
LIMIT 1
```

If a match is found, upsert into `game_id_map`:
```sql
INSERT OR REPLACE INTO game_id_map (sport, provider, external_game_id, game_id, match_method, match_confidence, matched_at)
VALUES ('nhl', 'nhl_gamecenter', ?, ?, 'schedule_date_teams', 1.0, datetime('now'))
```

### 2) Matching strategy

`games.home_team` stores full names (e.g. `"SAN JOSE SHARKS"`). NHL API provides abbreviations (e.g. `"SJS"`). Use a `LIKE` match against the last word of the full name, or maintain a small abbreviation→name map (same approach as `resolveMlbTeamAbbreviation` in `settle_mlb_f5.js`).

Alternatively, match on `game_time_utc` date + cross-reference with ESPN `game_id_map` rows for the same date/teams to find the canonical `game_id`, then write the NHL ID alongside it.

Recommended approach: query `game_id_map` for ESPN rows on the same date and match by team via `games` table join — avoids duplicating the abbreviation map.

```sql
SELECT gim.game_id
FROM game_id_map gim
JOIN games g ON g.game_id = gim.game_id
WHERE gim.sport = 'espn_nhl'
  AND gim.provider = 'espn'
  AND DATE(g.game_time_utc) = ?
  AND (g.home_team LIKE '%' || ? || '%' OR g.away_team LIKE '%' || ? || '%')
LIMIT 1
```

### 3) Job function signature

```js
async function pullNhlGameIds({ jobKey = null, dryRun = false, dateRange = 7 } = {})
```

Returns `{ success, inserted, updated, skipped, unmatched }`.

Logs:

### 4) Scheduler registration in `main.js`

Run daily at 06:00 ET (before settlement sweep):
