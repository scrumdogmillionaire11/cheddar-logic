# WI-0462: NHL Player Injury Stage 1 — Wire player_availability between pull job and model runner

**ID**: WI-0462
**Goal**: Connect the existing injury detection in the pull job to the model runner so that a player flagged as injured during data ingestion is never surfaced as an actionable card — without relying on the stale "fewer than 5 logs" proxy.

**Depends on**: migration 030 (`packages/data/db/migrations/030_add_player_availability.sql`) — table already exists.

**Scope**:
- `packages/data/src/db.js` — add `upsertPlayerAvailability`, `getPlayerAvailability`
- `packages/data/index.js` — export both new functions
- `apps/worker/src/jobs/pull_nhl_player_shots.js` — write to `player_availability` after injury check
- `apps/worker/src/jobs/run_nhl_player_shots_model.js` — read `player_availability` before model run per player
- `apps/worker/src/jobs/__tests__/pull_nhl_player_shots.test.js` — cover availability write behavior
- `apps/worker/src/jobs/__tests__/run_nhl_player_shots_model.test.js` — cover availability-based skip

**Out of scope**:
- Stage 2 (MoneyPuck injury wiring)
- Status tiering (OUT vs DTD vs QUESTIONABLE)
- Dedicated injury sync job (separate poll cadence)
- NHL team-level model changes

**Acceptance**:
1. `pull_nhl_player_shots` writes `status = 'INJURED'` (+ `status_reason`) to `player_availability` when `checkInjuryStatus` returns `skip: true`.
2. `pull_nhl_player_shots` writes `status = 'ACTIVE'` to `player_availability` for healthy players.
3. `run_nhl_player_shots_model` reads `player_availability` per player before the `< 5 logs` check; skips with `status=INJURED` when `checked_at` is within 24 hours.
4. A player flagged INJURED in `player_availability` does not generate a card even if they have 5+ historical logs.
5. The "fewer than 5 logs" proxy check is preserved as a secondary safeguard (runs after availability check).
6. No regression on existing tests.

## Implementation Log

### Stage 1a — DB layer
- Added `upsertPlayerAvailability({ playerId, sport, status, statusReason, checkedAt })` to `db.js`
- Added `getPlayerAvailability(playerId, sport)` to `db.js`
- Both exported from `packages/data/index.js`

### Stage 1b — Pull job writes availability
- After `checkInjuryStatus`, call `upsertPlayerAvailability`:
  - Injured: `status = 'INJURED'`, `status_reason = injuryCheck.reason`
  - Active: `status = 'ACTIVE'`, `status_reason = null`

### Stage 1c — Model runner reads availability
- Before the `< 5 logs` check, query `player_availability` for the player.
- If `status != 'ACTIVE'` and `checked_at > now - 24h`: skip with explicit log `status=<status>`.
- Fail-open: if no row in `player_availability`, proceed normally (pull job may not have run yet).
