# WI-0853: MLB F5 market token rename FIRST_PERIOD ‚Üí FIRST_5_INNINGS

## ID
WI-0853

## Goal
Replace the semantically incorrect `FIRST_PERIOD` market token used for MLB First 5 Innings cards with a new `FIRST_5_INNINGS` token. Maintain backward compatibility with existing DB rows that carry the old token. Keep NHL cards unchanged (they legitimately use `FIRST_PERIOD` for first-period markets).

## Scope
- `apps/worker/src/jobs/run_mlb_model.js` ‚Äî emit `FIRST_5_INNINGS` instead of `FIRST_PERIOD` for F5/F5ML cards
- `apps/worker/src/calibration/calibration-gate.js` ‚Äî accept both tokens in normalizePeriod + resolveCalibrationMarketKey MLB block
- `apps/worker/src/jobs/settle_pending_cards.js` ‚Äî accept both tokens in all period detection branches
- `apps/worker/src/jobs/backfill_period_token.js` ‚Äî accept both tokens
- `apps/worker/src/audit/audit_rules_config.js` ‚Äî map `FIRST_5_INNINGS` ‚Üí `'F5'` (new display label)
- `apps/worker/src/audit/performance_drift_report.js` ‚Äî accept both tokens
- `apps/worker/src/audit/projection_evaluator.js` ‚Äî accept both tokens
- `web/src/lib/types/game-card.ts` ‚Äî add `FIRST_5_INNINGS` to `CanonicalMarketType`
- `web/src/lib/games/market-inference.ts` ‚Äî `mlb-f5` infers `FIRST_5_INNINGS`; add to `MarketType` and `WAVE1_MARKETS`
- `web/src/lib/games/route-handler.ts` ‚Äî MLB `expectedPlayableMarkets` uses `FIRST_5_INNINGS`; add `normalizeLegacyMarketType` for DB read

## Out of scope
- No DB migration (backward compat via dual-token acceptance in readers)
- No rename of NHL `FIRST_PERIOD` usages in `run_nhl_model.js`, `models/index.js`, `nhl-player-shots.js`
- No changes to `packages/models/`
