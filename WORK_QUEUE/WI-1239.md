# WI-1239: Add `/results` Suppression Diagnostics

**ID**: WI-1239
**Goal**: Make suppressed `/results` rows explainable through diagnostics while preserving the current official `/results` contract.

**Context**:

`/results` intentionally excludes projection-only, PASS, INVALID, `potd-call`, and undisplayed rows from the main betting ledger. Today that suppression is mostly implicit: the query layer knows about display-log-backed settled rows, while transform-layer business rules decide which settled rows actually contribute to ledger and segment output. This WI adds diagnostics so operators can see why rows are missing without changing the official payload.

**Scope**:

- `web/src/app/api/results/route.ts`
- `web/src/lib/results/query-layer.ts`
- `web/src/lib/results/transform-layer.ts`
- `web/src/__tests__/api-results-suppression-diagnostics.test.js` (new)
- `WORK_QUEUE/WI-1239.md`

**Out of scope**:

- Any change to the main results ledger inclusion rules
- Any change to projection-only placement on `/results/projections`
- Any changes to `card_display_log` proof requirements

**Acceptance**:

- Diagnostics expose suppressed-row counts for PASS, INVALID, projection-only, `potd-call`, and undisplayed categories.
- Official ledger and segments remain unchanged.
- Diagnostics are clearly separated from the official results payload contract.
- Bucket counts are derived from the same transform/query-layer predicates that govern official `/results` inclusion, not a duplicated approximation.
- Regression coverage proves suppressed rows are classified into the correct buckets.
