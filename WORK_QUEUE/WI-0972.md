## Bucket Threshold Specification

Market 1P total buckets and per-side thresholds. OVER thresholds are tightened for
historically weak (low-scoring) buckets; UNDER thresholds are flat across all buckets.

> **Note**: `play` and `slightEdge` values below are initial calibration targets.
> Update from backtested OVER hit-rate data once available. Reason codes
> in `classifyNhlTotalsStatus` are stable audit strings and must not change.

```js
// In nhl-totals-status.js — above get1pBucketThresholds()
const BUCKET_THRESHOLDS = {
  // market_1p_total [1.0, 1.5)  — historically weakest OVER bucket
  '1.0-1.4': { over: { play: 1.5, slightEdge: 0.8 }, under: { play: 1.0, slightEdge: 0.5 } },
  // market_1p_total [1.5, 2.0)  — moderately weak OVER bucket
  '1.5-1.9': { over: { play: 1.2, slightEdge: 0.6 }, under: { play: 1.0, slightEdge: 0.5 } },
  // market_1p_total [2.0, 2.2)  — near-average; default thresholds
  '2.0-2.19': { over: { play: 1.0, slightEdge: 0.5 }, under: { play: 1.0, slightEdge: 0.5 } },
  // market_1p_total >= 2.2      — high-scoring; default thresholds
  '2.20+':    { over: { play: 1.0, slightEdge: 0.5 }, under: { play: 1.0, slightEdge: 0.5 } },
};
```

Bucket assignment is based on the **market** 1P total (`marketTotal` at the call site, i.e.
`market_1p_total` from the odds snapshot). Non-finite or missing market total → returns
`null` → caller omits `thresholds` from `classifyNhlTotalsStatus`, falling back to defaults.

## Implementation Spec

### New export in `nhl-totals-status.js`

```js
/**
 * Returns per-side threshold config for the given market 1P total bucket.
 * Returns null if marketTotal is not a finite number (caller omits thresholds → defaults apply).
 */
function get1pBucketThresholds(marketTotal) {
  if (!Number.isFinite(Number(marketTotal))) return null;
  const t = Number(marketTotal);
  if (t < 1.5)  return BUCKET_THRESHOLDS['1.0-1.4'];
  if (t < 2.0)  return BUCKET_THRESHOLDS['1.5-1.9'];
  if (t < 2.2)  return BUCKET_THRESHOLDS['2.0-2.19'];
  return BUCKET_THRESHOLDS['2.20+'];
}
module.exports = { classifyNhlTotalsStatus, computeNhl1pForecast, get1pBucketThresholds };
```

## Required New Tests in `nhl-1p-totals.test.js`

Add a new `describe('NHL 1P OVER guardrails — bucket-aware calibration', ...)` block.
**Do not modify existing tests.**

Import `get1pBucketThresholds` from `nhl-totals-status` in the test file header.

### `get1pBucketThresholds` contract (add 3 tests):
1. `get1pBucketThresholds(1.2)` → `over.play === 1.5` (bucket `1.0-1.4`)
2. `get1pBucketThresholds(1.7)` → `over.play === 1.2` (bucket `1.5-1.9`)
3. `get1pBucketThresholds(NaN)` → `null`

### OVER guardrail — blocked scenario (add 1):
- Input: `market_1p_total: 1.2`, OVER side, `modelTotal` producing `absDelta = 1.2`
  (would be `PLAY` at default `play: 1.0` but below `play: 1.5` in bucket `1.0-1.4`)
- Call `classifyNhlTotalsStatus` directly with `thresholds: get1pBucketThresholds(1.2)`
- Assert: `result.status !== 'PLAY'` (must be `'SLIGHT EDGE'` or `'PASS'`)

### OVER guardrail — allowed scenario (add 1):
- Input: `market_1p_total: 1.2`, OVER side, `modelTotal` producing `absDelta = 1.6`
  (clears `play: 1.5`)
- Assert: `result.status === 'PLAY'`

### UNDER not suppressed in weak bucket (add 1):
- Input: `market_1p_total: 1.2`, UNDER side, `absDelta = 1.0`
- Assert: `result.status === 'PLAY'` (UNDER threshold unchanged in `1.0-1.4` bucket)
