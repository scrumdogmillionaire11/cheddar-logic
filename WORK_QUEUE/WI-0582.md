# WI-0582 — Upgrade Silent `opponentFactor`/`paceFactor` Default to `console.warn` + Card Flag

**ID:** WI-0582
**Goal:** When `team_metrics_cache` is empty after a refresh failure and `opponentFactor`/`paceFactor` silently default to `1.0`, emit a `console.warn` and surface a flag in the card payload so operators are alerted that opponent/pace adjustment was skipped.
**Severity:** MEDIUM
**Owner agent:** gsd-executor
**CLAIM:** gsd-executor 2026-03-24T01:10:28Z
**Status:** OPEN
**Time window:** —
**Coordination flag:** none

## Problem

`run_nhl_player_shots_model.js` defaults `opponentFactor = 1.0` and `paceFactor = 1.0` at `console.debug` level when the team metrics cache returns empty. At `debug` level, this is invisible in production logs. The card is published with projections that assume league-average opponent defense and pace, but nothing in the card indicates this. The resulting projection bias is unquantified and untraceable in monitoring.

## Scope

- `apps/worker/src/jobs/run_nhl_player_shots_model.js` — change `console.debug` → `console.warn` at the fallback site for both `opponentFactor` and `paceFactor`.
- Add flag `OPPONENT_FACTOR_MISSING` (and `PACE_FACTOR_MISSING`) to card `flags` array when the fallback is used.
- `apps/worker/src/jobs/__tests__/run_nhl_player_shots_model.test.js` — add test that the flag is present when cache is empty.

## Out of Scope

- Changes to how `team_metrics_cache` is populated.
- Changes to the model math.

## Acceptance

- [ ] `console.warn` fires (with game/player context) when either factor defaults to `1.0` due to missing cache.
- [ ] `OPPONENT_FACTOR_MISSING` flag present in card `flags` array when `opponentFactor` falls back.
- [ ] `PACE_FACTOR_MISSING` flag present when `paceFactor` falls back.
- [ ] Test: empty cache → card flags contain the expected flags.
- [ ] All existing tests pass.
