# WI-1028: POTD — Replace single POTD_MIN_EDGE with sport-specific noise floors

**ID**: WI-1028
**Goal**: Replace the single global `POTD_MIN_EDGE = 2%` noise floor with a sport+market-type specific map so that the gate reflects actual model precision per sport rather than a one-size-fits-all cutoff that both over-suppresses tight-model sports and under-suppresses noisy-model sports.

**Background**:
Currently `gatherBestCandidate` in `run_potd_engine.js` applies a single `POTD_MIN_EDGE` constant to all sports and market types. A 2% floor treats an MLB moneyline (high noise, wide confidence interval) identically to an NHL moneyline (goalie-composite model, tighter signal). The correct fix is a per-sport noise floor that encodes how much gross edge the model must show before the signal is distinguishable from estimation error.

**Depends on**: none

**Scope**:
- `apps/worker/src/jobs/potd/signal-engine.js`
- `apps/worker/src/jobs/potd/run_potd_engine.js`

**Out of scope**:
- Changing `POTD_MIN_TOTAL_SCORE` (score/confidence gate is separate from noise floor)
- Any UI changes
- Execution-gate.js vig deduction (WI-1031)
- NBA model signal wiring (WI-1030)
- Any other sport model files

**Acceptance**:
- `signal-engine.js` exports a `resolveNoiseFloor(sport, marketType)` function that returns a number.
- Default noise floor values (each env-var overridable):
  - MLB MONEYLINE: `0.030` — env: `POTD_NOISE_FLOOR_MLB_ML` (default `0.03`)
  - MLB SPREAD: `0.025` — env: `POTD_NOISE_FLOOR_MLB_SPREAD` (default `0.025`)
  - NHL MONEYLINE: `0.020` — env: `POTD_NOISE_FLOOR_NHL_ML` (default `0.02`)
  - NHL SPREAD: `0.020` — env: `POTD_NOISE_FLOOR_NHL_SPREAD` (default `0.02`)
  - NBA MONEYLINE: `0.025` — env: `POTD_NOISE_FLOOR_NBA_ML` (default `0.025`)
  - NBA SPREAD: `0.020` — env: `POTD_NOISE_FLOOR_NBA_SPREAD` (default `0.02`)
  - NBA TOTAL: `0.020` — env: `POTD_NOISE_FLOOR_NBA_TOTAL` (default `0.02`)
  - Default fallback for all others: `POTD_MIN_EDGE` (backward-compatible)
- `gatherBestCandidate` in `run_potd_engine.js` replaces the fixed `POTD_MIN_EDGE` filter with `resolveNoiseFloor(c.sport, c.marketType)` per candidate.
- `selectTopPlaysFn` call in `gatherBestCandidate` no longer passes a single `minEdgePct` — instead, filtering by sport-specific floor happens in the `viableCandidates` pre-filter before `selectTopPlaysFn` is called, or `selectTopPlaysFn` is refactored to accept a per-candidate floor function.
- `writeShadowCandidates` persists `gap_to_min_edge` using the per-candidate noise floor (not the global `POTD_MIN_EDGE`).
- Existing `POTD_MIN_EDGE` env var continues to work as the fallback floor for any sport/market combination not explicitly mapped. This preserves backward compatibility.
- All existing tests in `apps/worker/src/jobs/potd/__tests__/` pass without modification.
- New unit test in `apps/worker/src/jobs/potd/__tests__/signal-engine.test.js` verifies `resolveNoiseFloor` returns correct values for each sport/market combination and falls back correctly.

**Implementation notes**:

In `signal-engine.js`, add near the top:

```js
// Sport+market noise floors. Values represent minimum gross edge required
// for a candidate to be distinguishable from model estimation error.
// Each threshold is independently env-var overridable.
const NOISE_FLOORS = {
  MLB: {
    MONEYLINE: Number(process.env.POTD_NOISE_FLOOR_MLB_ML || 0.03),
    SPREAD:    Number(process.env.POTD_NOISE_FLOOR_MLB_SPREAD || 0.025),
  },
  NHL: {
    MONEYLINE: Number(process.env.POTD_NOISE_FLOOR_NHL_ML || 0.02),
    SPREAD:    Number(process.env.POTD_NOISE_FLOOR_NHL_SPREAD || 0.02),
  },
  NBA: {
    MONEYLINE: Number(process.env.POTD_NOISE_FLOOR_NBA_ML || 0.025),
    SPREAD:    Number(process.env.POTD_NOISE_FLOOR_NBA_SPREAD || 0.02),
    TOTAL:     Number(process.env.POTD_NOISE_FLOOR_NBA_TOTAL || 0.02),
  },
};

function resolveNoiseFloor(sport, marketType, globalFallback = 0.02) {
  const sportKey = String(sport || '').toUpperCase().replace('BASEBALL_', '').replace('ICEHOCKEY_', '');
  const marketKey = String(marketType || '').toUpperCase();
  return NOISE_FLOORS[sportKey]?.[marketKey] ?? globalFallback;
}
```

In `run_potd_engine.js`, the `viableCandidates` filter in `gatherBestCandidate` becomes:

```js
const viableCandidates = scoredCandidates.filter(c =>
  isFiniteNumber(c.edgePct) &&
  c.edgePct > resolveNoiseFloor(c.sport, c.marketType, POTD_MIN_EDGE) &&
  isFiniteNumber(c.totalScore) &&
  c.totalScore >= POTD_MIN_TOTAL_SCORE,
);
```

The `selectTopPlaysFn` call should filter the `scoredCandidates` array to only those passing their per-sport noise floor before ranking:

```js
const noisePassed = scoredCandidates.filter(c =>
  isFiniteNumber(c.edgePct) &&
  c.edgePct > resolveNoiseFloor(c.sport, c.marketType, POTD_MIN_EDGE)
);
const fireableNominees = selectTopPlaysFn(noisePassed, {
  minConfidence: POTD_MIN_TOTAL_SCORE,
  minEdgePct: 0,   // floor already applied above per-candidate
  maxNominees: POTD_MAX_NOMINEES,
  requirePositiveEdge: true,
});
```
