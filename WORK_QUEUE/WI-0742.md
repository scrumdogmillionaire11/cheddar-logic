# WI-0742: Projection-Only Numeric Output Contract

**ID**: WI-0742
**Goal**: Every PROJECTION_ONLY card must emit a decision-ready projection package — full-model expected value, confidence band, market translation layer, play range, edge sensitivity and stability, model quality tier, risk profile, and key drivers — so the card tells the user not just what the model thinks but exactly how to attack every line they might see in the market.

**Scope**:
- `WORK_QUEUE/WI-0742.md`
- `apps/worker/src/jobs/run_nhl_model.js` — emit projection contract for NHL_1P_TOTAL
- `apps/worker/src/jobs/run_nhl_player_shots_model.js` — emit projection contract for NHL_PLAYER_SHOTS
- `apps/worker/src/jobs/run_mlb_model.js` — emit projection contract for MLB_PITCHER_K and MLB_F5_TOTAL
- `apps/worker/src/audit/audit_invariants.js` — add INV-007
- `apps/worker/src/audit/compare_audit_snapshot.js` — add tolerant numeric projection checks
- `apps/worker/src/audit/fixtures/nhl/nhl_1p_total_projection_01.json`
- `apps/worker/src/audit/fixtures/nhl/nhl_player_shots_projection_01.json`
- `apps/worker/src/audit/fixtures/mlb/mlb_pitcher_k_clean_01.json`
- `apps/worker/src/audit/fixtures/mlb/mlb_f5_total_clean_01.json`
- `apps/worker/src/audit/fixtures/mlb/mlb_f5_total_degraded_01.json` — if relevant
- `apps/worker/src/audit/__tests__/**` — update or add test coverage for INV-007

**Out of scope**:
- Web UI display changes (WI-0734 already surfaced pitcher-K; other families are separate)
- Adding a live odds lane to any PROJECTION_ONLY family
- Changing any `card_family_status` in `card-family-registry.json`
- Any LIVE family (NBA_TOTAL, NBA_SPREAD, NHL_TOTAL, NHL_ML) — they are not affected
- New sports or models

## Line Context by Family

Some families have known standard lines; others require the user to hunt the
posted line. Both cases use the same output contract, but `play_range` is
computed differently.

| Family | Standard line(s) | Line hunting? |
|--------|-----------------|---------------|
| NHL_1P_TOTAL | always **1.5** | No — direction computes automatically |
| MLB_F5_TOTAL | typically **3.5 or 4.5** | No — card computes edge vs both; user picks the live line |
| NHL_PLAYER_SHOTS | none fixed | Yes — `play_range` is absolute; compare against whatever line the book posts |
| MLB_PITCHER_K | none fixed | Yes — `play_range` is absolute; compare against whatever K line the book posts |

### Field definitions

| Field | Required | Type | Constraint |
|-------|----------|------|------------|
| `numeric_projection` | yes | `number` | full-model μ, finite positive — **never a defensive fallback** |
| `projection_unit` | yes | `string` | `"goals_1p"` / `"shots_on_goal"` / `"strikeouts"` / `"runs_f5"` |
| `projection_low` | yes | `number` | finite positive, < `numeric_projection` |
| `projection_high` | yes | `number` | finite positive, > `numeric_projection` |
| `recommended_direction` | yes | `"OVER"\|"UNDER"` | computed from projection vs applicable line |
| `confidence` | yes | `number` | 0 < confidence ≤ 1 |
| `model_quality` | yes | `string` | `"FULL_MODEL"` / `"DEGRADED_MODEL"` / `"FALLBACK"` |
| `model_consensus` | optional | `string` | `"ALIGNED"` / `"SPLIT"` — design-forward; default `"ALIGNED"` when single model |
| `play_range` | yes\* | `object` | see per-family format below; **null when `model_quality=FALLBACK`** |
| `market_translation` | K/shots only | `object` | map of posted line strings → edge label for absolute-line families |
| `market_grid` | F5 only | `object` | map of line strings → edge label covering full range incl. alt lines |
| `no_edge_zone` | yes\* | `[number, number]` | `[low, high]` — if posted line is inside, PASS; **null when `model_quality=FALLBACK`** |
| `edge_sensitivity` | yes | `"HIGH"\|"MEDIUM"\|"LOW"` | derived from band width relative to projection |
| `edge_stability` | yes | `"FRAGILE"\|"STABLE"` | how durable the edge is near the no-edge zone |
| `leash_class` | K only | `"TIGHT"\|"NORMAL"\|"EXTENDED"` | pitcher's expected pitch count regime; major K-prop risk driver |
| `drivers` | yes | `string[]` | 2–3 items max; prefix `+` for tailwinds, `-` for headwinds |
| `leverage_point` | yes | `string` | the single input that most swings the outcome |
| `failure_profile` | yes | `"BINARY"\|"GRADUAL"` | BINARY = one event kills bet; GRADUAL = distribution-based loss |
| `volatility` | F5 only | `"HIGH"\|"MEDIUM"\|"LOW"` | how fragile the F5 run environment is |
| `failure_modes` | F5 only | `string[]` | 1–2 items; how the bet loses |
| `actionable` | yes | `false` | hardcoded; enforced by INV-006 |
| `execution_status` | yes | `"PROJECTION_ONLY"` | enforced by INV-004 |

\* Required unless `model_quality === "FALLBACK"` — must be null in that case, not absent.

### `play_range` format by family

**MLB_PITCHER_K / NHL_PLAYER_SHOTS** (no fixed line — absolute thresholds):

```json
{
  "over_at_or_below": 6.5,
  "lean_over": 7.0,
  "pass_at_or_above": 7.5
}
```

Logic: `over_at_or_below = projection_low + 0.5 half-step below μ`,
`pass_at_or_above = projection_high - 0.5 half-step below top`, heuristic
refined per family. See implementation notes.

**MLB_F5_TOTAL** (lines expected at 3.5 and/or 4.5):

```json
{
  "vs_3_5": { "recommended_direction": "OVER", "edge": "STRONG" },
  "vs_4_5": { "recommended_direction": "UNDER", "edge": "LEAN" }
}
```

`edge` values: `"STRONG"` (line outside band) / `"LEAN"` (line near band edge) /
`"NONE"` (line inside no-edge zone).

**NHL_1P_TOTAL** (line always 1.5):

```json
{
  "vs_1_5": { "recommended_direction": "OVER", "edge": "STRONG" }
}
```

### `market_translation` (MLB_PITCHER_K / NHL_PLAYER_SHOTS)

Maps specific posted-line strings a user might see to an immediate action label.
Computed from `play_range` thresholds — not independently derived.

```json
{
  "5.5": "STRONG_OVER",
  "6.5": "LEAN_OVER",
  "7.5": "PASS",
  "8.5": "LEAN_UNDER"
}
```

Label values: `"STRONG_OVER"` / `"LEAN_OVER"` / `"PASS"` / `"LEAN_UNDER"` /
`"STRONG_UNDER"`. Labels map to `play_range` thresholds:

- line ≤ `over_at_or_below` → `STRONG_OVER`
- `over_at_or_below` < line ≤ `lean_over` → `LEAN_OVER`
- `lean_over` < line < `pass_at_or_above` → `PASS`
- line ≥ `pass_at_or_above` → `LEAN_UNDER` or `STRONG_UNDER`

Generate keys for every 0.5-increment line in `[numeric_projection - 1.5,
numeric_projection + 1.5]` rounded to nearest 0.5. At minimum include the
four lines bracketing `no_edge_zone`.

### `market_grid` (MLB_F5_TOTAL)

Covers real-world posted lines including 4.0 alt and full range. Maps line
strings to edge labels.

```json
{
  "3.5": "STRONG_OVER",
  "4.0": "LEAN_OVER",
  "4.5": "LEAN_UNDER",
  "5.0": "STRONG_UNDER"
}
```

Computed from the band against each line (same STRONG/LEAN/PASS/STRONG_UNDER
label logic as `market_translation`). Always include 3.5, 4.0, 4.5, 5.0 at
minimum. `play_range.vs_3_5` and `play_range.vs_4_5` remain for contract
compliance; `market_grid` adds 4.0 and 5.0 for real-world usability.

```text
band_width = projection_high - projection_low
sensitivity = HIGH   if band_width < 0.15 * numeric_projection
            = LOW    if band_width > 0.35 * numeric_projection
            = MEDIUM otherwise
```

### `edge_stability` derivation

`edge_stability` captures the second dimension of edge quality: how durable
the edge is when the market drifts slightly off the play point.

```text
stability = FRAGILE  if numeric_projection is within 0.3 of no_edge_zone boundary
                     OR model_quality == DEGRADED_MODEL
                     OR any single input contributes > 50% of projection weight
stability = STABLE   otherwise
```

For MLB_F5_TOTAL: FRAGILE when `volatility == HIGH` and `failure_profile == BINARY`.
For MLB_PITCHER_K: FRAGILE when `leash_class == TIGHT` OR `leverage_point ==
"Pitch count"` (because a hook changes the distribution step-change).

### `leash_class` (MLB_PITCHER_K only)

```text
leash_class = TIGHT    if innings_cap <= 5 or flagged short-leash
leash_class = EXTENDED if innings_cap >= 7 or flagged ace-start
leash_class = NORMAL   otherwise
```

Derive from the pitcher's `innings_cap` field in the model payload when
present. If absent, default to `NORMAL` and set `model_quality` to
`DEGRADED_MODEL`.

### `failure_profile`

| Value | Meaning | Primary use cases |
|-------|---------|-------------------|
| `BINARY` | One discrete event kills the bet (HR, ejection, early hook, walk spike) | MLB_F5_TOTAL, NHL_1P_TOTAL |
| `GRADUAL` | Loss emerges from distribution; no single event is decisive | MLB_PITCHER_K, NHL_PLAYER_SHOTS |

### FALLBACK hard rule

**When `model_quality === "FALLBACK"`, the following fields MUST be `null`:**

- `play_range`
- `market_translation`
- `market_grid`
- `no_edge_zone`
- `recommended_direction`

A FALLBACK card still emits `numeric_projection` (the floor/synthetic estimate),
`projection_low`, `projection_high`, `drivers`, `leverage_point`, and
`failure_profile` — but all actionable fields are nulled. The card surface must
display a loud warning and suppress any "play at" language.

INV-007 enforces this: if `model_quality === "FALLBACK"` and `play_range` is
non-null, emit INV-007-WARN-FALLBACK-MISMATCH.

### `model_consensus` (optional, design-forward)

```text
model_consensus = ALIGNED  if all sub-models agree on direction
model_consensus = SPLIT    if sub-models disagree
```

Default to `"ALIGNED"` when only one model is running. When a second model
or ensemble is added later, this field requires no contract change. SPLIT
cards should be treated as `no_edge_zone` candidates regardless of band width.

### `model_quality` tiers

| Value | Meaning | Action |
|-------|---------|--------|
| `FULL_MODEL` | All model inputs present | Use normally |
| `DEGRADED_MODEL` | 1–2 inputs missing or estimated | Use with caution, size down |
| `FALLBACK` | Synthetic / ERA-only / no real inputs | Do not use for betting |

`FALLBACK` replaces the former `projection_is_floor: true` flag. If `model_quality`
is `FALLBACK`, INV-007 records a WARN (not CRITICAL) AND enforces that
`play_range`, `market_translation`, `market_grid`, `no_edge_zone`, and
`recommended_direction` are all `null` (FALLBACK-MISMATCH WARN if not).
Cards can still publish but must display a loud visual warning and suppress
all "play at"/direction language.

## Implementation Notes

### 1. Model job changes

Each job populates the full contract before writing the card payload to the DB.
Use existing model computed values — do not add new inputs to the model here.

**`run_nhl_model.js` / NHL_1P_TOTAL**

- `numeric_projection` = 1P expected-goals from the F1P driver
- `projection_low` / `projection_high` = from F1P variance; if not available
  approximate as `μ ± 0.4`
- `model_quality` = `FULL_MODEL` if goalie certainty = CONFIRMED both sides,
  `DEGRADED_MODEL` if any goalie is EXPECTED, `FALLBACK` if UNKNOWN
- `play_range.vs_1_5.recommended_direction` = `OVER` if μ > 1.5 else `UNDER`
- `play_range.vs_1_5.edge` = `STRONG` / `LEAN` / `NONE` per line vs band
- `drivers` = top 2–3 of: goalie save pct differential, SOG-1P pace, home/away
  split
- `leverage_point` = the highest-weight input to the 1P model (e.g. goalie
  status if uncertain, SOG pace if goalies confirmed)

**`run_nhl_player_shots_model.js` / NHL_PLAYER_SHOTS**

- `numeric_projection` = season SOG avg adjusted for opponent allow rate,
  home/away split, pace factor
- `projection_low` = μ × 0.78 (approx 1 std dev below for SOG distribution)
- `projection_high` = μ × 1.28
- `model_quality` = `FULL_MODEL` if season sample ≥ 20 games and matchup data
  present; `DEGRADED_MODEL` otherwise
- `play_range.over_at_or_below` = max posted line still inside `[μ - 0.5, μ]`
  rounded to nearest 0.5 line increment
- `drivers` = opponent allow-SOG rank, player hot/cold streak, matchup pace
- `leverage_point` = whichever of opponent allow rate vs player streak has
  larger pull on the projection

**`run_mlb_model.js` / MLB_PITCHER_K**

- `numeric_projection` = `k_mean` (already computed; same value surfaced in
  WI-0734's web UI)
- `projection_low` / `projection_high` = from the K prop driver's confidence
  interval; if not present, approximate from innings cap and K% variance as
  `μ ± (0.15 * μ + 0.5)`
- `model_quality` = `FULL_MODEL` if season_starts ≥ 10 and opponent k_rate
  present; `DEGRADED_MODEL` if < 10 starts or opponent data estimated;
  `FALLBACK` if no starter profile
- `play_range.over_at_or_below` = floor to nearest 0.5 line at or below
  `projection_low + 0.3`
- `play_range.lean_over` = nearest 0.5 line to μ − 0.3
- `play_range.pass_at_or_above` = ceiling to nearest 0.5 line above
  `projection_high − 0.5`
- `no_edge_zone` = `[play_range.lean_over, play_range.pass_at_or_above]`
- `market_translation` = derived from `play_range` thresholds; generate keys
  for every 0.5-increment line in `[μ - 1.5, μ + 1.5]`; map to STRONG_OVER /
  LEAN_OVER / PASS / LEAN_UNDER using threshold logic from spec
- `leash_class` = derived from `innings_cap` in model payload: `TIGHT` if ≤ 5,
  `EXTENDED` if ≥ 7, `NORMAL` otherwise; default `NORMAL` + flag
  `model_quality = DEGRADED_MODEL` if no innings_cap present
- `edge_stability` = `FRAGILE` if `leash_class != NORMAL` or
  `leverage_point == "Pitch count"`; `STABLE` otherwise
- `failure_profile` = `GRADUAL` (K props are distribution-based, not
  single-event binary)
- `drivers` = opponent K% (vs hand), pitcher SwStr%, innings cap; prefix `+`/`-`
- `leverage_point` = innings cap if pinch-hit/count volatility is flagged, else
  opponent K% vs hand
- If `model_quality === FALLBACK`: set `play_range = null`, `market_translation
  = null`, `no_edge_zone = null`, `recommended_direction = null`

**`run_mlb_model.js` / MLB_F5_TOTAL**

- `numeric_projection` = full-model F5 expected runs (starter ERA/xFIP ×
  opponent wRC+F5 × weather × park factor). **Never `projectionFloorF5`** as
  primary value. If full model unavailable, set `model_quality = FALLBACK`.
- `projection_low` / `projection_high` = from model variance; if not available
  approximate as `μ ± 0.8`
- `model_quality` = `FULL_MODEL` if both starters have ≥ 5 starts and weather
  present; `DEGRADED_MODEL` if either has < 5 starts or weather estimated;
  `FALLBACK` if projectionFloorF5 fallback path fired
- `play_range.vs_3_5` and `play_range.vs_4_5`: compute edge for each standard
  line against the band; `STRONG` if line > 0.5 outside band, `LEAN` if within
  0.5 of band edge, `NONE` if inside `no_edge_zone`
- `market_grid` = compute all four keys: 3.5, 4.0, 4.5, 5.0; label each using
  band vs line logic (STRONG_OVER / LEAN_OVER / PASS / LEAN_UNDER / STRONG_UNDER)
- `no_edge_zone` = `[max(3.5, μ - 0.3), min(4.5, μ + 0.3)]`
- `edge_stability` = `FRAGILE` if `volatility == HIGH` and
  `failure_profile == BINARY`; `STABLE` otherwise
- `failure_profile` = `BINARY` (F5 totals are killed by single events: HR
  spike, strand rate spike, early hook)
- `volatility` = `HIGH` if starter WHIP > 1.35 or wind > 10mph or either team
  walk rate top-quartile; `LOW` if both starters ERA < 3.00 and calm conditions;
  `MEDIUM` otherwise
- `failure_modes`: max 2, chosen from:
  - "Strand rate spikes → Under"
  - "Early walks → Over"
  - "Groundball rate suppresses HR → Under"
  - "Early HR → Over"
  - "Double play rate → Under"
- `drivers` = top 3 from: SP ERA/xFIP differential, weather (wind speed/dir),
  opposing offense vs hand, park factor, walk rate
- If `model_quality === FALLBACK`: set `play_range = null`, `market_grid =
  null`, `no_edge_zone = null`, `recommended_direction = null`

### 2. INV-007 (new invariant)

```text
INV-007 — PROJECTION_ONLY_DECISION_CONTRACT
Severity: CRITICAL (missing required field) or WARN (FALLBACK violations)
Trigger: execution_status === 'PROJECTION_ONLY'

Required fields check (CRITICAL if missing):
  numeric_projection — finite positive number
  projection_unit — valid enum value
  projection_low — finite positive, < numeric_projection
  projection_high — finite positive, > numeric_projection
  confidence — number in (0, 1]
  model_quality — 'FULL_MODEL' | 'DEGRADED_MODEL' | 'FALLBACK'
  edge_sensitivity — 'HIGH' | 'MEDIUM' | 'LOW'
  edge_stability — 'FRAGILE' | 'STABLE'
  failure_profile — 'BINARY' | 'GRADUAL'
  drivers — non-empty array, length <= 3
  leverage_point — non-empty string

Conditional fields (CRITICAL if model_quality != 'FALLBACK' and missing):
  recommended_direction — 'OVER' or 'UNDER'
  play_range — non-null object with at least one keyed entry
  no_edge_zone — array of exactly two numbers [low, high], low < high

Conditional fields (CRITICAL if card_family ends with _PITCHER_K or _PLAYER_SHOTS and missing):
  market_translation — non-null object with at least one keyed entry
  leash_class (K only) — 'TIGHT' | 'NORMAL' | 'EXTENDED'

Conditional fields (CRITICAL if card_family === 'MLB_F5_TOTAL' and missing):
  market_grid — non-null object with keys 3.5, 4.0, 4.5, 5.0
  volatility — 'HIGH' | 'MEDIUM' | 'LOW'
  failure_modes — array, length <= 2

FALLBACK model_quality checks (WARN, not CRITICAL):
  INV-007-WARN: emit when model_quality === 'FALLBACK'
  INV-007-WARN-FALLBACK-MISMATCH: emit when model_quality === 'FALLBACK'
    AND any of play_range, market_translation, market_grid, no_edge_zone,
    recommended_direction is non-null
```

### 3. Comparator changes

  (null allowed and expected when `model_quality_expected === "FALLBACK"`)
