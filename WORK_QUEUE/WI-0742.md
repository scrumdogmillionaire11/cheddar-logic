## Line Context by Family

Some families have known standard lines; others require the user to hunt the
posted line. Both cases use the same output contract, but `play_range` is
computed differently.

| Family | Standard line(s) | Line hunting? |
|--------|-----------------|---------------|
| NHL_1P_TOTAL | always **1.5** | No — direction computes automatically |
| MLB_F5_TOTAL | typically **3.5 or 4.5** | No — card computes edge vs both; user picks the live line |
| NHL_PLAYER_SHOTS | none fixed | Yes — `play_range` is absolute; compare against whatever line the book posts |
| MLB_PITCHER_K | none fixed | Yes — `play_range` is absolute; compare against whatever K line the book posts |

## Required Output Contract

### Core (INV-007 fails if any required field is missing or malformed)

```json
{
  "numeric_projection": 6.8,
  "projection_unit": "strikeouts",
  "projection_low": 5.6,
  "projection_high": 7.9,
  "recommended_direction": "OVER",
  "confidence": 0.63,
  "model_quality": "DEGRADED_MODEL",
  "play_range": {
    "over_at_or_below": 6.5,
    "lean_over": 7.0,
    "pass_at_or_above": 7.5
  },
  "no_edge_zone": [6.5, 7.0],
  "edge_sensitivity": "HIGH",
  "drivers": [
    "+ Opponent K% elevated (23.8%)",
    "+ Strong SwStr% profile",
    "- Pitch count volatility"
  ],
  "leverage_point": "Pitch count",
  "actionable": false,
  "execution_status": "PROJECTION_ONLY"
}
```

F5 total also includes `volatility` and `failure_modes`:

```json
{
  "numeric_projection": 4.2,
  "projection_unit": "runs_f5",
  "projection_low": 3.5,
  "projection_high": 5.1,
  "model_quality": "FULL_MODEL",
  "play_range": {
    "vs_3_5": { "recommended_direction": "OVER", "edge": "STRONG" },
    "vs_4_5": { "recommended_direction": "UNDER", "edge": "LEAN" }
  },
  "no_edge_zone": [3.8, 4.2],
  "edge_sensitivity": "MEDIUM",
  "volatility": "HIGH",
  "drivers": [
    "+ Weak SP matchup both sides",
    "+ Warm weather boost",
    "- Below-average power lineups"
  ],
  "leverage_point": "Wind direction",
  "failure_modes": [
    "Strand rate spikes → Under",
    "Early HR → Over"
  ],
  "actionable": false,
  "execution_status": "PROJECTION_ONLY"
}
```

### Field definitions

| Field | Required | Type | Constraint |
|-------|----------|------|------------|
| `numeric_projection` | yes | `number` | full-model μ, finite positive — **never a defensive fallback** |
| `projection_unit` | yes | `string` | `"goals_1p"` / `"shots_on_goal"` / `"strikeouts"` / `"runs_f5"` |
| `projection_low` | yes | `number` | finite positive, < `numeric_projection` |
| `projection_high` | yes | `number` | finite positive, > `numeric_projection` |
| `recommended_direction` | yes | `"OVER"\|"UNDER"` | computed from projection vs applicable line |
| `confidence` | yes | `number` | 0 < confidence ≤ 1 |
| `model_quality` | yes | `string` | `"FULL_MODEL"` / `"DEGRADED_MODEL"` / `"FALLBACK"` |
| `play_range` | yes | `object` | see per-family format below |
| `no_edge_zone` | yes | `[number, number]` | `[low, high]` — if posted line is inside, PASS |
| `edge_sensitivity` | yes | `"HIGH"\|"MEDIUM"\|"LOW"` | derived from band width relative to projection |
| `drivers` | yes | `string[]` | 2–3 items max; prefix `+` for tailwinds, `-` for headwinds |
| `leverage_point` | yes | `string` | the single input that most swings the outcome |
| `volatility` | F5 only | `"HIGH"\|"MEDIUM"\|"LOW"` | how fragile the F5 run environment is |
| `failure_modes` | F5 only | `string[]` | 1–2 items; how the bet loses |
| `actionable` | yes | `false` | hardcoded; enforced by INV-006 |
| `execution_status` | yes | `"PROJECTION_ONLY"` | enforced by INV-004 |

### `play_range` format by family

**MLB_PITCHER_K / NHL_PLAYER_SHOTS** (no fixed line — absolute thresholds):

```json
{
  "over_at_or_below": 6.5,
  "lean_over": 7.0,
  "pass_at_or_above": 7.5
}
```

Logic: `over_at_or_below = projection_low + 0.5 half-step below μ`,
`pass_at_or_above = projection_high - 0.5 half-step below top`, heuristic
refined per family. See implementation notes.

**MLB_F5_TOTAL** (lines expected at 3.5 and/or 4.5):

```json
{
  "vs_3_5": { "recommended_direction": "OVER", "edge": "STRONG" },
  "vs_4_5": { "recommended_direction": "UNDER", "edge": "LEAN" }
}
```

`edge` values: `"STRONG"` (line outside band) / `"LEAN"` (line near band edge) /
`"NONE"` (line inside no-edge zone).

**NHL_1P_TOTAL** (line always 1.5):

```json
{
  "vs_1_5": { "recommended_direction": "OVER", "edge": "STRONG" }
}
```

### `edge_sensitivity` derivation

```
band_width = projection_high - projection_low
sensitivity = HIGH   if band_width < 0.15 * numeric_projection
            = LOW    if band_width > 0.35 * numeric_projection
            = MEDIUM otherwise
```

### `model_quality` tiers

| Value | Meaning | Action |
|-------|---------|--------|
| `FULL_MODEL` | All model inputs present | Use normally |
| `DEGRADED_MODEL` | 1–2 inputs missing or estimated | Use with caution, size down |
| `FALLBACK` | Synthetic / ERA-only / no real inputs | Do not use for betting |

`FALLBACK` replaces the former `projection_is_floor: true` flag. If `model_quality`
is `FALLBACK`, INV-007 records a WARN (not CRITICAL) — cards can still publish
but are visually marked as unreliable.

## Implementation Notes

### 1. Model job changes

Each job populates the full contract before writing the card payload to the DB.
Use existing model computed values — do not add new inputs to the model here.

**`run_nhl_model.js` / NHL_1P_TOTAL**

- `numeric_projection` = 1P expected-goals from the F1P driver
- `projection_low` / `projection_high` = from F1P variance; if not available
  approximate as `μ ± 0.4`
- `model_quality` = `FULL_MODEL` if goalie certainty = CONFIRMED both sides,
  `DEGRADED_MODEL` if any goalie is EXPECTED, `FALLBACK` if UNKNOWN
- `play_range.vs_1_5.recommended_direction` = `OVER` if μ > 1.5 else `UNDER`
- `play_range.vs_1_5.edge` = `STRONG` / `LEAN` / `NONE` per line vs band
- `drivers` = top 2–3 of: goalie save pct differential, SOG-1P pace, home/away
  split
- `leverage_point` = the highest-weight input to the 1P model (e.g. goalie
  status if uncertain, SOG pace if goalies confirmed)

**`run_nhl_player_shots_model.js` / NHL_PLAYER_SHOTS**

- `numeric_projection` = season SOG avg adjusted for opponent allow rate,
  home/away split, pace factor
- `projection_low` = μ × 0.78 (approx 1 std dev below for SOG distribution)
- `projection_high` = μ × 1.28
- `model_quality` = `FULL_MODEL` if season sample ≥ 20 games and matchup data
  present; `DEGRADED_MODEL` otherwise
- `play_range.over_at_or_below` = max posted line still inside `[μ - 0.5, μ]`
  rounded to nearest 0.5 line increment
- `drivers` = opponent allow-SOG rank, player hot/cold streak, matchup pace
- `leverage_point` = whichever of opponent allow rate vs player streak has
  larger pull on the projection

**`run_mlb_model.js` / MLB_PITCHER_K**

- `numeric_projection` = `k_mean` (already computed; same value surfaced in
  WI-0734's web UI)
- `projection_low` / `projection_high` = from the K prop driver's confidence
  interval; if not present, approximate from innings cap and K% variance as
  `μ ± (0.15 * μ + 0.5)`
- `model_quality` = `FULL_MODEL` if season_starts ≥ 10 and opponent k_rate
  present; `DEGRADED_MODEL` if < 10 starts or opponent data estimated;
  `FALLBACK` if no starter profile
- `play_range.over_at_or_below` = floor to nearest 0.5 line at or below
  `projection_low + 0.3`
- `play_range.lean_over` = nearest 0.5 line to μ − 0.3
- `play_range.pass_at_or_above` = ceiling to nearest 0.5 line above
  `projection_high − 0.5`
- `no_edge_zone` = `[play_range.lean_over, play_range.pass_at_or_above]`
- `drivers` = opponent K% (vs hand), pitcher SwStr%, innings cap; prefix `+`/`-`
- `leverage_point` = innings cap if pinch-hit/count volatility is flagged, else
  opponent K% vs hand

**`run_mlb_model.js` / MLB_F5_TOTAL**

- `numeric_projection` = full-model F5 expected runs (starter ERA/xFIP ×
  opponent wRC+F5 × weather × park factor). **Never `projectionFloorF5`** as
  primary value. If full model unavailable, set `model_quality = FALLBACK`.
- `projection_low` / `projection_high` = from model variance; if not available
  approximate as `μ ± 0.8`
- `model_quality` = `FULL_MODEL` if both starters have ≥ 5 starts and weather
  present; `DEGRADED_MODEL` if either has < 5 starts or weather estimated;
  `FALLBACK` if projectionFloorF5 fallback path fired
- `play_range.vs_3_5` and `play_range.vs_4_5`: compute edge for each standard
  line against the band; `STRONG` if line > 0.5 outside band, `LEAN` if within
  0.5 of band edge, `NONE` if inside `no_edge_zone`
- `no_edge_zone` = `[max(3.5, μ - 0.3), min(4.5, μ + 0.3)]`
- `volatility` = `HIGH` if starter WHIP > 1.35 or wind > 10mph or either team
  walk rate top-quartile; `LOW` if both starters ERA < 3.00 and calm conditions;
  `MEDIUM` otherwise
- `failure_modes`: max 2, chosen from:
  - "Strand rate spikes → Under"
  - "Early walks → Over"
  - "Groundball rate suppresses HR → Under"
  - "Early HR → Over"
  - "Double play rate → Under"
- `drivers` = top 3 from: SP ERA/xFIP differential, weather (wind speed/dir),
  opposing offense vs hand, park factor, walk rate

### 2. INV-007 (new invariant)

```
INV-007 — PROJECTION_ONLY_DECISION_CONTRACT
Severity: CRITICAL (missing required field) or WARN (FALLBACK model_quality)
Trigger: execution_status === 'PROJECTION_ONLY'

Required fields check (CRITICAL if missing):
  numeric_projection — finite positive number
  projection_unit — valid enum value
  projection_low — finite positive, < numeric_projection
  projection_high — finite positive, > numeric_projection
  recommended_direction — 'OVER' or 'UNDER'
  confidence — number in (0, 1]
  model_quality — 'FULL_MODEL' | 'DEGRADED_MODEL' | 'FALLBACK'
  play_range — non-null object with at least one keyed entry
  no_edge_zone — array of exactly two numbers [low, high], low < high
  edge_sensitivity — 'HIGH' | 'MEDIUM' | 'LOW'
  drivers — non-empty array, length <= 3
  leverage_point — non-empty string

F5-only fields check (CRITICAL if card_family === 'MLB_F5_TOTAL' and missing):
  volatility — 'HIGH' | 'MEDIUM' | 'LOW'
  failure_modes — array, length <= 2

FALLBACK model_quality check (WARN, not CRITICAL):
  if model_quality === 'FALLBACK': emit INV-007-WARN
```
