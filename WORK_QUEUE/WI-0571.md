# WI-0571: Market evaluator — projection comparator

**ID**: WI-0571
**Goal**: Add a projection comparison layer that computes edge vs consensus and edge vs best available separately, exposes both in card payloads, and produces a `playable_edge` verdict. This is the Type 1 misprice case: the model materially disagrees with what the market prices.

**Background**:
Currently cards bake a single `edgePoints` value computed at model run time using the line from the odds snapshot. After WI-0568, we'll have a `consensus_line` that is the correct reference for "does the model see a real edge?" — edge vs consensus = is there a structural disagreement with the market. We'll also have `best_available_line` = edge vs best execution = what the bettor can actually capture.

These two numbers tell different stories:
- "Edge vs consensus" = is the model's signal real?
- "Edge vs best available" = can we actually get paid on it?

The delta between them is execution alpha. Both should be visible.

**Scope**:
- `packages/odds/src/market_evaluator.js`
  - `compareProjection({ fairLine, consensusLine, bestLine, consensusPrice, bestPrice })` — returns:
    ```
    {
      fair_line_from_projection: number | null,
      edge_vs_consensus_pts: number | null,        // fairLine - consensusLine
      edge_vs_best_available_pts: number | null,   // fairLine - bestLine
      execution_alpha_pts: number | null,          // edge_vs_best - edge_vs_consensus
      fair_price_from_projection: number | null,   // American odds implied by fairLine
      edge_vs_consensus_pct: number | null,        // in probability terms
      edge_vs_best_available_pct: number | null,
      playable_edge: boolean                       // both consensus and best-available edges clear threshold
    }
    ```
  - Threshold for `playable_edge`: configurable constant, default 1.0 pts for spread/total
- `apps/worker/src/models/cross-market.js` (and sport-specific model runners)
  - After computing the model's fair spread/total, call `compareProjection` using the consensus line (from odds snapshot) and best-line
  - Store output in card payload `odds_context` block alongside existing edge fields
- `apps/worker/src/run_nba_model.js` (and NHL/NCAAM equivalents)
  - Pass `spread_consensus_line` / `total_consensus_line` from oddsSnapshot into the model's market decision flow
  - Log `edge_vs_consensus_pts` and `edge_vs_best_available_pts` per game in model run output
- `web/src/components/cards-page-client.tsx`
  - In the spread/total math detail section, show both edge numbers:
    - "Edge vs market: +1.8 pts" (consensus)
    - "Edge at best book: +2.6 pts (DraftKings)" (best available)
  - Replace current single `edgePoints` display with these two where available; fall back to current behavior when consensus data is absent (old snapshots)

**Out of scope**:
- Changing how the model generates its fair line (model architecture changes)
- Props and team totals (defer)
- Retroactive recomputation of old card payloads

**Acceptance**:
- For an NBA spread card: both `edge_vs_consensus_pts` and `edge_vs_best_available_pts` are computed and non-null when `spread_consensus_line` is populated.
- `execution_alpha_pts` = `edge_vs_best_available_pts - edge_vs_consensus_pts` and reflects how much extra edge the bettor gets from line shopping.
- `playable_edge = true` only when BOTH consensus edge and best-available edge clear the 1.0pt threshold — prevents a rogue outlier book from generating a fake signal.
- Card UI shows two edge lines when both are available; degrades gracefully to current behavior on old data.
- Model run log outputs "consensus edge" + "execution edge" per game.
- TypeScript compiles cleanly.
