# WI-0880: Add h2h market to NHL odds fetch

**ID**: WI-0880
**Goal**: Add the h2h (moneyline) market to the NHL odds fetch so the POTD signal engine can build and score NHL moneyline candidates. Currently NHL fetches only totals; buildMoneylineCandidates always receives an empty game.market.h2h array and produces zero NHL candidates -- making NHL invisible as a POTD source regardless of game count.
**Scope**:
- packages/odds/src/config.js

**Out of scope**:
- Changes to any consumer of SPORTS_CONFIG beyond the config file itself
- Signal-engine wiring (that is WI-0881)
- NHL totals candidates or other sport configs

**Acceptance**:
- SPORTS_CONFIG.NHL.markets contains both totals and h2h
- SPORTS_CONFIG.NHL.tokensPerFetch updated from 1 to 2
- Budget comment updated: NHL (main job): totals + h2h = 2 tokens; grand total 4 -> 5
- All existing tests pass; no other sport configs changed

**Owner agent**: unassigned
**Time window**: 2026-04-11 -> TBD
**Coordination flag**: needs-sync -- packages/odds/src/config.js is a serialized shared touchpoint

**Tests to run**:
- npm --prefix apps/worker test --no-coverage

**Manual validation**:
- Run POTD engine on a NHL game day; confirm [Odds] API log shows markets=totals,h2h for NHL
- With force=true outside publish window, confirm NHL games appear in scored output

**Guard for WI closeout**:
- node -e confirm SPORTS_CONFIG.NHL.markets includes h2h and tokensPerFetch is 2
- Budget comment must match before merge
