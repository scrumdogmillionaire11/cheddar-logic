# WI-1260-c: Add Conservative High-End LEAN/SLIGHT_EDGE Promotion

**ID**: WI-1260-c

**Goal**: Add a guarded promotion path in decision_v2 from LEAN or SLIGHT_EDGE to PLAY for high-end candidates that pass strict support and safety checks.

**Scope**:

- apps/worker/src/jobs/helpers/ensure-canonical-decision-v2.js
- `apps/worker/src/__tests__/ensure-canonical-decision-v2.test.js`

**Out of scope**:

- modifying POTD selection logic directly
- model health scheduler implementation
- broad model retraining or recalibration

**Acceptance**:

- promotion applies only when all guards pass:
- status is LEAN or SLIGHT_EDGE
- edge floor met
- support score floor met
- no hard blockers
- model health acceptable
- price verification passed
- promoted outputs include promoted_from
- promoted outputs include promotion_reason_code HIGH_END_EDGE_PROMOTION
- default NHL moneyline thresholds are configurable with initial defaults:
- edge_pct >= 0.10
- support_score >= 0.30
- non-qualifying high-edge but low-support examples are not promoted
