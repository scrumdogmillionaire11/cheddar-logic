# WI-1002: Consolidate stale-data vocabulary and expand terminal reason-family granularity

**ID**: WI-1002

**Goal**: Retire `BLOCK_STALE_DATA` in favor of the canonical `MARKET_DATA_STALE` code, and split the `LINE_NOT_CONFIRMED` terminal-reason-family bucket into granular sub-families so telemetry and reporting can distinguish line-verification failures from market-staleness and edge-drift events.

## Background

### Stale-data vocabulary (debt item 3)

Two codes describe the same concept:

| Code | Source |
| ---- | ------ |
| `MARKET_DATA_STALE` | Canonical code introduced in WI-0998 |
| `BLOCK_STALE_DATA` | Legacy code still emitted and still present in `reason-labels.ts` |

`decision-logic.ts` branches on both at line 1465â€“1466. `reason-labels.ts` has separate label entries for both (lines 6 and 23). This dual vocabulary inflates branching and makes analytics counts unreliable.

### Terminal reason-family granularity (debt item 4)

`deriveTerminalReasonFamilyForPayload` in `decision-publisher.js` (lines 62â€“95) collapses four distinct conditions into one family bucket `LINE_NOT_CONFIRMED`:

- `LINE_NOT_CONFIRMED` â€” line verification failed
- `EDGE_RECHECK_PENDING` â€” edge rechecking in flight
- `PRICE_SYNC_PENDING` â€” price sync not yet received
- `EDGE_VERIFICATION_REQUIRED` â€” legacy alias (removed by WI-1000)

This collapses actionable distinction. An `EDGE_RECHECK_PENDING` event is operationally different from a `PRICE_SYNC_PENDING` event, and both differ from a terminal `LINE_NOT_CONFIRMED`.

## Changes

### 1. Stale-data normalization

Add a normalization shim at the ingest boundary (model output assembly) that maps `BLOCK_STALE_DATA` â†’ `MARKET_DATA_STALE` before codes are stored or published. Remove `BLOCK_STALE_DATA` from `reason-labels.ts` after normalization is in place. Remove dual-branch in `decision-logic.ts`.

### 2. Terminal reason-family expansion

Replace the single `LINE_NOT_CONFIRMED` family bucket with three granular buckets:

| New family value | Triggered by |
| ---------------- | ------------ |
| `LINE_NOT_CONFIRMED` | `LINE_NOT_CONFIRMED` only |
| `MARKET_STALE_RECHECK` | `MARKET_DATA_STALE`, `PRICE_SYNC_PENDING` |
| `EDGE_MOVED` | `EDGE_RECHECK_PENDING`, `EDGE_NO_LONGER_CONFIRMED` |
