# WI-1153: POTD Near-Miss Best-Edge Dedup Per Market/Match

**ID**: WI-1153
**Goal**: Enforce one canonical near-miss candidate per market/match by keeping only the highest-edge side at write time and preserving that one-side contract in settled/read surfaces.

**Scope**:
- `apps/worker/src/jobs/potd/run_potd_engine.js` (shadow-candidate write-path canonicalization)
- `apps/worker/src/jobs/potd/__tests__/run-potd-engine.test.js`
- `apps/worker/src/jobs/potd/__tests__/shadow-settlement.test.js` (only if settlement identity expectations need updates)
- `web/src/lib/potd-server.ts` (settled/read-path defensive dedup for legacy rows)
- `web/src/app/play-of-the-day/settled/page.tsx` (only if render contract needs adjustment)
- `web/src/__tests__/api-potd.test.js`
- `web/src/__tests__/potd-near-miss-best-edge.test.js` (new; must be created in this WI)
- `WORK_QUEUE/WI-1153.md`

**Out of scope**:
- Any changes to unrelated POTD logic or UI
- Database schema changes/migrations

**Tasks**:

1. **Canonicalize opposing sides at the source write path (authoritative dedup)**
   - In `apps/worker/src/jobs/potd/run_potd_engine.js`, define market/match grouping using existing fields that are already persisted (`sport`, `market_type`, `game_id`, canonical home/away pairing, and normalized market context such as line where applicable).
   - Do **not** use non-existent fields (for example `market_target`) in the grouping contract.
   - Ensure only the highest-edge candidate survives per group for each `play_date`; opposing-side candidates in the same group must not both persist as near-miss shadow rows.
   - Keep deterministic tie-break behavior (higher edge first, then deterministic fallback) so reruns are stable.

2. **Preserve one-side contract in settled/read surfaces**
   - In `web/src/lib/potd-server.ts`, make `getPotdSettledHistoryData()` resilient to legacy duplicates by applying the same one-per-group best-edge selection when duplicate/opposing records exist.
   - Keep current API shape stable unless a field is strictly required for deterministic grouping; if added, update mapped types and tests in scope.
   - Only adjust `web/src/app/play-of-the-day/settled/page.tsx` if needed for the deduped list contract.

3. **Add explicit regression coverage across worker + web**
   - Extend `apps/worker/src/jobs/potd/__tests__/run-potd-engine.test.js` with fixtures where opposite sides of the same market/match are scored with different edges; assert only one shadow candidate persists and it is the highest-edge row.
   - Add `web/src/__tests__/potd-near-miss-best-edge.test.js` to validate settled/read-side suppression of opposing duplicates (including legacy duplicate fixture cases).
   - Update `web/src/__tests__/api-potd.test.js` only as needed to keep POTD response contract assertions passing.

**Acceptance**:
- For each play date and market/match group, only the highest-edge side is retained as near-miss in authoritative shadow candidate storage.
- Settled near-miss history never returns both opposing sides for the same market/match group.
- No reliance on fields absent from the current schema.
- Automated tests cover both writer and reader paths for opposing-side suppression.
