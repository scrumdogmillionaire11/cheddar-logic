# WI-0324: Race Condition Mitigation (DB + Worker)

**ID**: WI-0324
**Goal**: Eliminate high-risk race conditions in multi-process scheduler/worker usage against shared CHEDDAR_DB_PATH, with atomic job gating and safe write paths.

**Scope**:

- `packages/data/src/db.js`
- `packages/data/db/migrations/**` (only if new constraints or indexes are required)
- `apps/worker/src/schedulers/main.js`

- Job idempotency is atomic across processes (no read-then-write race for `job_runs`)

- Delete-then-insert write paths are protected by a transactional or lock-based boundary
- Multi-process DB write hazards are mitigated or explicitly guarded with clear errors
- New or updated tests cover at least one concurrency-safe path
