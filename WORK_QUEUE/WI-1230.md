# WI-1230: Fail-Closed NHL False Listing Cleanup

**ID**: WI-1230
**Goal**: Prevent transient bad NHL schedule rows from surfacing on the production cards board, remove stale `game_id_map` lookup drift in the NHL prop runner, and add a pipeline-health signal for future false-listing conditions.

**Context**:

Investigation on 2026-05-03 found a reported production board listing for `LOS ANGELES KINGS @ COLORADO AVALANCHE` at `May 3, 2026, 12:00 AM EDT` that did not match trusted external schedules. By the time the investigation ran, the row had disappeared from `/api/games`, but the code paths showed a plausible failure mode:

- `/api/games` pregame surfacing currently allows a row through when it has plays even if it has no odds.
- NHL prop generation can attach plays to any future `games` row that remains `scheduled`.
- `apps/worker/src/jobs/run_nhl_player_shots_model.js` still contains a stale `game_id_map` lookup using legacy column names, which silently degrades to fallback matching.

The cleanup must harden the authoritative surfacing path and improve detection without changing betting/model decision rules.

**Out of scope**:

**Acceptance**:

- `/api/games` no longer surfaces an NHL pregame matchup solely because prop plays exist when the game has no displayable odds and no ingest-failure marker
- `run_nhl_player_shots_model.js` resolves canonical game IDs against the current `game_id_map` schema instead of silently missing the explicit mapping path
- Pipeline health exposes a dedicated failed/degraded signal when future scheduled NHL rows appear likely stale or false-listing-prone
- Focused regression tests cover:
  - NHL prop-only row suppression on `/api/games` service-layer filtering
  - current-schema `game_id_map` canonical ID resolution in the NHL prop runner
  - pipeline-health detection/reporting of suspect scheduled NHL rows
