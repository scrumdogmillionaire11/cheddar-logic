# WI-0637: MLB without-odds mode — synthetic F5 projection_floor line

**ID**: WI-0637
**Goal**: When `ENABLE_WITHOUT_ODDS_MODE=true`, derive a synthetic F5 total projection_floor line from pitcher stats so `driversReady` passes and MLB generates PROJECTION_ONLY cards with null prices.

## Background

In without-odds mode the `odds_snapshots` rows have null market fields (`f5_total`, `ml_f5_home/away`, `total`).
`buildMlbMarketAvailability` checks `f5TotalContext.line !== null` for `f5LineOk`.
`driversReady = f5_line_ok || (expect_f5_ml && f5_ml_ok)` — both false → 0 cards.

NHL and NBA produce PROJECTION_ONLY cards by using `projection_floor` lines derived from model projections.
MLB needs the same path: a pitcher-stats-sourced F5 total floor that un-gates `driversReady` when odds are absent.

## Implementation Notes

- In `run_mlb_model.js`, when `withoutOddsMode === true` and `f5TotalContext.line === null`:
  - Compute `projectionFloorF5` from pitcher run projections already loaded in the model run
    (home SP `era`-derived expected runs over 5 innings + away SP, rounded to nearest 0.5).
    Fallback: use a configurable constant (default `8.5`) if pitcher stats are missing.
  - Inject `f5_line_ok = true` and `f5_total = projectionFloorF5` into the market availability
    object **only** when `withoutOddsMode === true`.
  - `f5_ml_ok` remains false (no ML card in without-odds mode — `expect_f5_ml` is already false for ESPN-direct games).
  - Mark generated card `without_odds_mode: true`, `projection_floor: true`, prices null.
- Do **not** change `buildMlbMarketAvailability` signature — accept an optional `{ withoutOddsMode, projectionFloorF5 }` options bag.
- The `blocking_reason_codes` array should drop `F5_TOTAL_UNAVAILABLE` when the floor line is used; keep `MARKET_PRICE_MISSING` and `F5_ML_UNAVAILABLE` (expected).
