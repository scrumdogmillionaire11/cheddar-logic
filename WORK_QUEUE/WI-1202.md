# WI-1202: Wire POTD to DecisionOutcome

**ID**: WI-1202
**Goal**: Refactor POTD engine to consume `DecisionOutcome` only, selecting strictly on `status='PLAY'`, eliminating fallback ghosts and drift from web/Discord.

**Scope**:
- `apps/worker/src/jobs/potd/run_potd_engine.js` (import canonical builder from `@cheddar-logic/data`, resolve `DecisionOutcome` from model payload decision_v2 inputs, filter model-contract candidates for `status='PLAY'` only, remove any local decision_v2 status reinterpretation that becomes dead)

**Out of scope**:
- Implementing POTD-specific decision mappers (use canonical builder from WI-1199 only)
- Changing worker decision_v2 production
- Comprehensive refactoring or reorganization of POTD utilities (covered in WI-1203)

- **In scope (not deferred to WI-1203):** Removing "easy hanging" dead code — functions that become unused after builder switch, have no external callers, and are not exported. This reduces maintenance burden immediately.
