# WI-0655: Screenshot parsing and parsed-squad normalization

**ID**: WI-0655
**Goal**: Add MVP screenshot parsing for official FPL mobile screenshots and normalize the result into a confidence-scored 15-man parsed squad.

**Depends on**:

- WI-0652

**Scope**:
- `WORK_QUEUE/WI-0655.md`
- `cheddar-fpl-sage/backend/models/screenshot_api_models.py`
- `cheddar-fpl-sage/backend/services/player_registry.py`
- `cheddar-fpl-sage/backend/services/screenshot_parser.py`
- `cheddar-fpl-sage/backend/routers/screenshot_parse.py`
- `cheddar-fpl-sage/tests/test_player_registry.py`
- `cheddar-fpl-sage/tests/test_screenshot_parser.py`
- `cheddar-fpl-sage/tests/fixtures/screenshots/`

**Out of scope**:
- Desktop screenshots
- Video parsing or scroll capture
- Audit/comparison scoring
- UI correction flow

**Acceptance**:
- `POST /api/v1/screenshot-parse` accepts 1-3 official FPL mobile screenshots in pitch or list view for MVP.
- Parsing pipeline classifies layout, reconstructs slots, matches against the current-season player registry, and emits slot-level plus overall confidence.
- Response normalizes to a parsed-squad object with starters, bench order, captain/vice, bank if available, and unresolved-slot metadata.
- Low-confidence slots are surfaced explicitly; the parser never returns a silent “complete” parse when ambiguity remains.
- Golden fixtures cover pitch view, list view, multi-image merges, and low-confidence fallbacks.
