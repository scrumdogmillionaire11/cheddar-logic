# WI-0577 — V2 Poisson Veto Gate for 1P FIRE on Odds-Backed Cards

**ID:** WI-0577
**Goal:** Add V2 Poisson edge veto in the 1P path so V1 cannot emit FIRE when V2's probability edge for the selected direction is negative on odds-backed 1P cards.
**Severity:** CRITICAL
**Owner agent:** bmad-master
**Status:** COMPLETE

## Problem

The 1P `firstPeriodDecision` comes exclusively from V1 (`derivePlayDecision` via edge tier + support/confidence). V2's `edge_over_pp` / `edge_under_pp` (Poisson probability edges) are computed once for the full-game run but never consulted as a gate in the 1P path. V1 can therefore emit `FIRE` for a 1P card while V2's probability edge for that exact direction is negative — a direct contradiction.

The full-game path is NOT affected: it uses `legacyDecisionFromProp` derived from `buildCanonicalPropDecision.verdict` (V2-driven), which already enforces `probEdge >= 0.06` for PLAY.

## Implementation

Added **Guard 3 (WI-0577)** in `run_nhl_player_shots_model.js` at the 1P decision block:

```js
if (realPropLine1p && firstPeriodDecision.action === 'FIRE') {
  const v2EdgeForDir1p =
    firstPeriodEdge.direction === 'UNDER' ? v2EdgeUnderPp : v2EdgeOverPp;
  if (typeof v2EdgeForDir1p === 'number' && Number.isFinite(v2EdgeForDir1p) && v2EdgeForDir1p < 0) {
    console.warn(`[...] [v2-veto-1p] Downgraded ${playerName} 1P FIRE→WATCH ...`);
    firstPeriodDecision = { action: 'HOLD', status: 'WATCH', classification: 'LEAN', officialStatus: 'LEAN' };
  }
}
```
