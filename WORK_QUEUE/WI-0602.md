# WI-0602: MLB DUAL_RUN market selection — elevate F5 Total as primary game market

**ID**: WI-0602
**Goal**: Add DUAL_RUN-style market selection to the MLB model so F5 Total is treated as the primary game market and Pitcher Strikeout cards are treated as additive props, matching the NHL/NBA pattern.

**Scope**:

- `apps/worker/src/jobs/run_mlb_model.js`
- `apps/worker/src/models/mlb-model.js` (scoring/selection helpers if needed)
- `apps/worker/src/__tests__/` (add DUAL_RUN selection tests for MLB)

**Out of scope**:

- Changes to F5 ML ingest or side-projection (see WI-0603)
- Changes to pipeline health watchdog differentiation (see WI-0604)
- Changing the F5 projection math in `projectF5TotalCard`
- Any NHL or NBA model changes

**Background**:
Currently `run_mlb_model.js` emits one card per qualifying driver: an F5 Total card and up to two Pitcher K cards (home/away) all compete as peers. There is no DUAL_RUN log, no `chosen_market` selection, and no explicit separation of game market vs player prop. This means the web display layer has no reliable signal for which MLB card is the "top-level play" for a game. The registry `docs/MARKET_REGISTRY.md` declares F5 Total as the primary MLB game market. This WI wires that intent.

**Acceptance**:

- `run_mlb_model.js` logs a `[MLB_DUAL_RUN]` line per game with `chosen_market`, `why_this_market`, and each scored market candidate (F5 only at this stage).
- F5 Total card is written to `card_payloads` with `card_type: 'mlb-f5'` and flagged as the primary game market.
- Pitcher K cards (home/away) are written separately with `card_type: 'mlb-pitcher-k'` and are not considered in the game-level market selection pass.
- If no F5 line is present the game logs `NO_F5_LINE` as a blocking reason (analogous to `FIRST_PERIOD_NO_PROJECTION`).
- Existing Pitcher K card output is unchanged in content; only the selection loop changes.
