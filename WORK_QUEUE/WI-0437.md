CLAIM: 


## Summary

WI-0437 should harden **soccer Ohio-scope Tier 1 markets only as the default success path**: **Player Shots, Team Totals, and TSOA**. Tier 2 should stay conditional and explicitly secondary. Do not spend this WI chasing draw logic, 3-way pricing, BTTS, legacy totals, or “cute” markets. That stuff is either deferred, banned, or outside the strongest current contract path. The system already points that way.   

## Why This Is Needed

Right now the biggest risk is not “missing one more market.” It is **shipping soccer cards with weak or placeholder projection context**, then letting downstream consumers pretend the payload is trustworthy. Your existing footie stack already narrows scope to Ohio-legal props and explicitly removes legacy 1X2/DNB/AH/match totals/BTTS from scope. The parser, router, normalizer, validator, and output template all reinforce that narrow lane. WI-0437 should lean into that instead of reopening the casino buffet.   

The cleanest path is:

* harden required context for **Tier 1**
* make missing context explicit, not silently defaulted
* keep Tier 2 optional and blocked unless filters pass
* document deferred projector gaps clearly, especially draw-market ingestion

That matches the existing watchdog philosophy: explicit PASS/reason over silent nonsense.

### Market priority

**Primary success markets for WI-0437**

1. **Player Shots**
2. **Team Totals** (`o0.5`, `o1.5`, `u2.5`)
3. **TSOA`

Those are the always-enabled Tier 1 markets in the router/model stack.

**Secondary, conditional only**

* Player SOT
* Anytime Scorer
* Team Corners

These remain optional and gated by stricter role, price, and data conditions. They are not the center of this WI.

**Explicitly not worth chasing here**

* Draw odds / 3-way settlement logic
* DNB / AH
* Match totals / BTTS
* Cards / fouls

Those are either removed from Ohio scope, banned, or explicitly deferred by your WI language. The line to hold here is “make the narrow lane real,” not “expand the surface area.”

### New invariants

`run_soccer_model.js` should emit soccer payloads that satisfy these invariants:

1. **Every emitted soccer card must declare a scoped market family**

   * `player_shots`
   * `team_totals`
   * `to_score_or_assist`
   * optionally `player_shots_on_target`, `anytime_goalscorer`, `team_corners`

These canonical keys already exist in the normalizer contract. 

2. **Tier 1 cards must include hardened projection context**
   At minimum, per card/market:

   * canonical market key
   * line
   * price
   * model projection or fair probability basis
   * edge / EV or equivalent model-support field
   * required gating fields used to justify eligibility
   * pass/block reason when not actionable

3. **Player markets must never rely on fake confidence**
   They must carry enough player eligibility context to prove why they were even considered:

   * starter/expected starter status
   * projected minutes
   * role tags
   * relevant per-90 hints for the market

The normalizer already defines deterministic role tags for shots, TSOA, anytime, and SOT. Use that instead of vibes and duct tape. 

4. **Missing critical context must become flags, not placeholders**
   The slate contract requires affected objects and/or global slate flags when data is missing or suspect. 

5. **No silent drift into banned or deprecated markets**
   Validator must reject or flag cards containing:

   * cards
   * fouls
   * 1X2
   * DNB
   * AH
   * match totals
   * BTTS

That is already the documented market scope. The validator should enforce it like a bouncer, not a greeter.

## Implementation Plan

### 1. Update the WI doc first

Amend `WORK_QUEUE/WI-0437.md` to make the intent painfully explicit:

* **Primary objective:** harden Tier 1 market payloads only
* **Non-goals:** no draw-market pricing, no legacy markets, no endpoint expansion
* **Definition of done:** Tier 1 payloads have non-placeholder projection context and explicit validation coverage
* **Follow-up WIs:** draw odds ingestion, broader projector rebuild, legacy market restoration if ever needed

That keeps future-you from “helpfully” scope-creeping this into a month-long swamp.

### 2. Harden `run_soccer_model.js` around a Tier 1-first pipeline

Refactor the job into explicit stages.

#### Stage A — normalize market scope

Before projection logic, map all inbound soccer candidates into canonical market groups:

* `player_shots`
* `team_totals`
* `to_score_or_assist`
* `player_shots_on_target`
* `anytime_goalscorer`
* `team_corners`

Drop anything else with an explicit blocked reason. Use the normalizer/router contract, not ad hoc string matching.

#### Stage B — build minimum required context

For each match, derive a conservative context packet:

* match id / league / teams
* context/style/form tags
* environment fields from pace/consistency layer if available
* warnings/fatal markers when not

The consistency contract says footie workflows must attach consistency results and footie validation; missing them is a `GAME_FATAL` condition at watchdog time.

#### Stage C — build Tier 1 market packets

**Player Shots**

* require player identity
* `PRIMARY_VOLUME_SHOOTER` or equivalent evidence
* projected minutes / starter signal
* shots per 90 or team-relative shot share support
* line / price
* projection or fair win probability
* edge classification support

**Team Totals**

* require side/team identity
* team lambda or equivalent projection basis
* allowed lines only: `o0.5`, `o1.5`, `u2.5`
* line / price
* edge basis

**TSOA**

* require player identity
* one of `TERMINAL_NODE | PRIMARY_CREATOR | SET_PIECE_ROLE`
* projected minutes / starter signal
* xG/xA support
* line / price
* fair probability / edge basis

These requirements are already implied by the footie model and validation rules; WI-0437 just needs to stop the worker from faking its way past them.

#### Stage D — keep Tier 2 in the passenger seat

Allow Tier 2 outputs only when fully qualified:

* SOT only when shots is unavailable or poor-priced and price cap passes
* Anytime only for terminal nodes at `> +180`
* Corners only with extreme mismatch plus data

Otherwise emit PASS or do not surface them. No honorary medals for almost-data.

#### Stage E — explicit degradation behavior

When core projection context is missing:

* do **not** backfill with generic defaults pretending to be real
* emit structured warnings / flags
* downgrade to PASS or skip output for that market
* still allow the card payload to be structurally valid if the consumer expects pass-through with reason

This matches both `slate_context` flagging rules and watchdog “no pick without reason.”

### 3. Tighten `packages/data/src/validators/card-payload.js`

This is the real teeth.

Add a `soccer_ohio_scope` validator block that asserts:

#### Scope validation

Allowed market keys only:

* `player_shots`
* `team_totals`
* `to_score_or_assist`
* `player_shots_on_target`
* `anytime_goalscorer`
* `team_corners`

Reject / flag all others.

#### Required-field validation by market type

**Player Shots**

* `player_name` present
* `line` present
* `price` present
* `proj_minutes` present
* role tag evidence present
* projection support present

**Team Totals**

* `team` present
* allowed line set only
* `price` present
* team projection basis present

**TSOA**

* `player_name`
* `price`
* `proj_minutes`
* qualifying role tag
* fair probability / edge basis

#### Guardrail validation

* shots price must not be worse than `-150`
* TSOA must not be worse than `-140`
* SOT must not be worse than `-130`
* anytime must be `> +180`
* player markets require starter + minutes floor
* Tier 2 warnings/tags when applicable

Those rules are already in the consistency/output docs. Validator should mirror them so drift gets caught at the data layer, not after the UI has already put on makeup.

#### Placeholder detection

Add explicit checks that reject fields like:

* `"unknown"`
* `"tbd"`
* `0` used as fake projection where not semantically valid
* empty arrays where a qualifying role tag is required
* generic fallback text masquerading as notes

The WI’s whole point is reducing placeholder/default usage. Make that a real contract, not a motivational poster.
