## Context

The admin page's "Current Snapshot" section only shows `pipeline_health` table rows (schedule, odds, cards, settlement phases). Dr. Claire's model performance data (47.7% NBA vs 53.0% NHL hit rate, −12u NBA ROI) is visible only by running a CLI command — there is no persistent record and no UI surface.

This WI closes the gap so any stakeholder can load `/admin` and see model health without running a terminal command.

## Implementation Sketch

**Migration:**

```sql
CREATE TABLE model_health_snapshots (
  id INTEGER PRIMARY KEY AUTOINCREMENT,
  sport TEXT NOT NULL,
  run_at TEXT NOT NULL,
  hit_rate REAL,
  roi_units REAL,
  roi_pct REAL,
  total_unique INTEGER NOT NULL DEFAULT 0,
  wins INTEGER NOT NULL DEFAULT 0,
  losses INTEGER NOT NULL DEFAULT 0,
  streak TEXT,
  last10_hit_rate REAL,
  status TEXT NOT NULL,
  signals_json TEXT,
  lookback_days INTEGER NOT NULL DEFAULT 30,
  created_at TEXT NOT NULL DEFAULT CURRENT_TIMESTAMP
);
CREATE INDEX idx_mhs_sport_run ON model_health_snapshots(sport, run_at DESC);
```

**dr_claire_health_report.js --persist addition** (write section, worker-side DB):

```javascript
if (opts.persist) {
  const writerDb = getDatabase(); // writer instance
  const stmt = writerDb.prepare(`
    INSERT INTO model_health_snapshots
      (sport, run_at, hit_rate, roi_units, roi_pct, total_unique, wins, losses,
       streak, last10_hit_rate, status, signals_json, lookback_days)
    VALUES (?, ?, ?, ?, ?, ?, ?, ?, ?, ?, ?, ?, ?)
  `);
  for (const [sport, s] of Object.entries(report.sports)) {
    stmt.run(sport, report.generatedAt, s.hitRate, s.netUnits, s.roiPct,
      s.totalPredictions, s.wins, s.losses, s.streak, s.last10HitRate,
      s.status, JSON.stringify(s.degradationSignals), lookbackDays);
  }
  closeDatabaseInstance(writerDb);
}
```
