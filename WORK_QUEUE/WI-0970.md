**Acceptance**:
- NHL 1P pipeline emits four explicit playoff regime features onto `card.payloadData.raw_data` (see **Playoff Regime Feature Spec** below). All four must be non-null for confirmed playoff games (`isPlayoffGame()` returns `true`); `null` is acceptable for regular-season games.
- Training row construction in `run_nhl_model.js` filters malformed input and stamps `raw_data.training_row_excluded = true` + `raw_data.training_exclusion_reason = '<REASON_TAG>'` on excluded cards. Reason tags: `MALFORMED_INPUT`, `GOALIE_UNCERTAIN`, `INSUFFICIENT_DATA`.
- PASS/HOLD/WATCH cards are excluded from directional training targets by checking `payloadData.officialStatus` (or `payloadData.status`) before including a card in any training dataset export ŌĆö verified by test.
- Tests validate: (a) all four regime features are present on a mocked playoff payload, (b) each training exclusion reason tag is emitted for the correct condition, (c) PASS/HOLD/WATCH cards do not appear in the post-filter training candidate set.

**Playoff Regime Feature Spec**:

All four features are stamped onto `card.payloadData.raw_data` in `run_nhl_model.js` in the same section that stamps `rest_days_home` / `rest_days_away` (lines ~3488 and ~3610).

| Feature key | Type | Source | Derivation |
|---|---|---|---|
| `playoff_series_game_num` | `number \| null` | `oddsSnapshot.raw_data?.seriesStatus?.seriesGameNumber` | Read directly; `null` if field absent or non-playoff |
| `playoff_elimination_pressure` | `boolean \| null` | `oddsSnapshot.raw_data?.seriesStatus` | `true` when either team's series wins = 3 (one win from elimination in best-of-7); `null` if non-playoff |
| `playoff_rest_compression` | `boolean` | existing `rest_days_home`, `rest_days_away` on payload | `true` when either team's rest days Ōēż 1; derive in-place from already-stamped values |
| `playoff_goalie_stability` | `'FULL' \| 'DEGRADED' \| 'NEUTRALIZED' \| 'BLOCKED' \| null` | `makeCanonicalGoalieState()` result | Read `adjustment_trust` from the home or away goalie state object (use whichever is the starting goalie; if both confirmed, use home); `null` if non-playoff |

ESPN field availability note: `seriesStatus` was not found in currently indexed code paths. At implementation time, verify its presence by inspecting a live playoff snapshot (e.g., `SELECT raw_data FROM odds_snapshots LIMIT 1` on a known playoff game). If `seriesStatus` is absent from ESPN, use the fallback: `playoff_series_game_num = null` and `playoff_elimination_pressure = null`, and add a `TODO(WI-0970)` comment noting the missing upstream field.

**Training Row Exclusion Contract**:

Exclusion is applied in `run_nhl_model.js` after card assembly, before any training export path. The stamp is:

```js
card.payloadData.raw_data.training_row_excluded = true;
card.payloadData.raw_data.training_exclusion_reason = 'MALFORMED_INPUT'; // or GOALIE_UNCERTAIN | INSUFFICIENT_DATA
```

Cards are excluded when:
- `MALFORMED_INPUT`: `oddsSnapshot` is null, missing required fields (`total`, `h2h_home`, `h2h_away`), or contains NaN/Infinity in any numeric model input.
- `GOALIE_UNCERTAIN`: either home or away `makeCanonicalGoalieState()` returns `adjustment_trust === 'NEUTRALIZED'` or `'BLOCKED'`.
- `INSUFFICIENT_DATA`: `sigma_games_sampled` (already stamped on `raw_data`) is below threshold (< 5 games).

Cards with `officialStatus === 'PASS'` or `status === 'PASS'` or `status === 'HOLD'` or `status === 'WATCH'` are excluded from directional training targets ŌĆö separate from the `training_row_excluded` flag above. These are never training-eligible for side direction regardless of data quality.
