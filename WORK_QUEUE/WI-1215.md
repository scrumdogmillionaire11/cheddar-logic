# WI-1215: Split Projection Accuracy vs Calibration Reporting for MLB F5 Proxy

**ID**: WI-1215

**Goal**:
Separate official proxy hit-rate reporting from calibration tracking so gray-zone records inform model quality without inflating betting accuracy metrics.

**Scope**:

- `web/src/app/api/results/projection-accuracy/route.ts`
- `web/src/lib/types/projection-accuracy.ts`
- `web/src/__tests__/api-results-projection-accuracy.test.js`
- `apps/worker/src/jobs/projection_accuracy_health.js`
- `apps/worker/src/jobs/__tests__/projection_accuracy_health.test.js`

**Out of scope**:

- UI redesign of results surfaces
- Non-MLB F5 family threshold policy changes
- Settlement authority changes for displayed/non-displayed cards

**Acceptance**:

- API/reporting surfaces two distinct result groups:
  - Official proxy performance (`OFFICIAL` rows only)
  - Calibration tracking (includes gray-zone `TRACK_ONLY` rows)
- Gray-zone records do not contribute to official over/under win rate denominators.
- Calibration output includes bucket-level actual-runs diagnostics for gray and clear buckets.
- Health/report job emits explicit no-call/track-only rates for monitoring drift.
- Tests verify denominator isolation and report-shape compatibility.
