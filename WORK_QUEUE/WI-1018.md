# WI-1018: NBA Phase 1A — Pace adjustment cap + driver weight rebalancing

**ID**: WI-1018
**Goal**: Stop additive compounding errors in the NBA totals engine by capping pace synergy output and increasing base projection weight relative to add-on drivers.

**Depends on**: none

**Scope**:
- `apps/worker/src/models/nba-pace-synergy.js`
- `apps/worker/src/jobs/run_nba_model.js`
- `apps/worker/src/models/__tests__/nba-pace-normalization.test.js`

**Out of scope**:
- Rolling bias calibration (WI-1020)
- Dynamic league baselines (WI-1021)
- Any UI changes
- Other sport models

**Acceptance**:
- `paceAdjustment` in all synergy result objects returned by `analyzePaceSynergy` is clamped to `[-1.5, +1.5]`.
- `NBA_DRIVER_WEIGHTS` updated: `baseProjection` 0.35 → **0.55**, `restAdvantage` 0.15 → 0.10, `welcomeHomeV2` 0.10 → 0.07, `matchupStyle` 0.20 → 0.14, `blowoutRisk` 0.07 → 0.05, `totalProjection` 0.13 → 0.09. Sum remains exactly 1.00.
- All existing nba-pace-normalization tests pass.
- No regressions in `run-nba-model.test.js`.

**Attribution contract (required — do not skip)**:

Because this WI makes two distinct changes to the model, both must be independently measurable at validation time. Before merging, the PR must include a comment or test output showing:

1. **Pace cap attribution**: pace adjustment distribution before/after cap. Run a sample of 5+ real game snapshots through `analyzePaceSynergy` with old constants and new constants. Record `paceAdjustment` distribution (min, max, mean). If VERY_FAST×VERY_FAST games are not present in current data, note that explicitly.

2. **Weight rebalance attribution**: per-driver contribution distribution before/after rebalance. For the same game snapshots, compute `driver.signal * driver.weight` for each driver. Record how much `baseProjection` contribution increased relative to `matchupStyle` and `paceAdjustment` stacked total.

3. **Bias by total band**: group game projections into `<220`, `220-230`, `230-240`, `240+` bands. Record mean projected total vs mean actual total (from `projection_accuracy_line_evals` if available). If insufficient settled data, use the last 20 settled NBA games from the current season.

If both changes are made in one PR, the validation output must break them apart. If that is not feasible, split into two PRs with WI-1018A (pace cap) and WI-1018B (driver rebalance). The choice is at the implementer's discretion — but the measurement cannot be skipped.
