# WI-1245: Build a Unified Market Health Report

**ID**: WI-1245
**Goal**: Create one report per market/family that joins card production, surface rate, win rate, CLV, projection accuracy, and POTD shadow performance into a single operational view.

**Context**:

The codebase already has separate calibration, drift, projection-accuracy, and POTD health assets. They answer adjacent questions, but not from one source of operational truth. This WI creates the cross-source synthesis layer needed before any promotion or confidence tuning.

**Scope**:
- `apps/worker/src/jobs/report_telemetry_calibration.js`
- `apps/worker/src/audit/performance_drift_report.js`
- `apps/worker/src/jobs/projection_accuracy_health.js`
- `web/src/app/api/admin/model-health/route.ts`
- `web/src/app/api/admin/pipeline-health/route.ts`
- `apps/worker/src/__tests__/unified-market-health-report.test.js` (new)
- `web/src/__tests__/api-admin-unified-market-health.test.js` (new)
- `WORK_QUEUE/WI-1245.md`

**Out of scope**:
- Any threshold or promotion changes
- Any changes to public betting/results contracts
- Any FPL route changes

**Acceptance**:
- One market-health report exists per sport/market/family combining production volume, surface rate, settled performance, CLV where applicable, projection accuracy where applicable, and POTD shadow outcomes where applicable.
- The report reconciles cross-source counts instead of presenting split analytics views.
- Admin health reads can expose the unified report or a directly derived summary.
- Regression coverage proves cross-source reconciliation for representative markets.
