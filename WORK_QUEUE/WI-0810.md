# WI-0810: Fix remaining raw-implied residuals from WI-0805 devig rollout

| Field | Value |
|---|---|
| **ID** | WI-0810 |
| **Goal** | Three residual devig issues were identified after WI-0805 landed. Fix the live card payload inflation in the BLK multi-line loop, audit/remove the legacy `calculateMoneylineEdge` raw path, and clean up a stale comment. |
| **Priority** | Medium — item 1 is a correctness bug (card payloads show inflated implied prob); items 2–3 are housekeeping |

## Root Cause

WI-0805 fixed `projectBlkV1` and `projectSogV2` in `nhl-player-shots.js`, but the job runner that *calls* those functions has its own inline edge/implied computation for multi-line BLK cards that was not in scope.

---

## Scope

### Bug 1 — `run_nhl_player_shots_model.js`: BLK multi-line loop uses raw `americanToImplied` (correctness)

Two call sites in the multi-line BLK card builder still use the local raw `americanToImplied`:

**Primary card** (~line 3149–3157):
```js
impliedOverProb: blkMarket.over_price != null
  ? americanToImplied(blkMarket.over_price) : null,
impliedUnderProb: blkMarket.under_price != null
  ? americanToImplied(blkMarket.under_price) : null,
```
These are payload display fields — inflated by ~2.4pp at -110/-110.

**Multi-line extra loop** (~line 3344–3379):
```js
const implOvr = americanToImplied(extraCandidate.over_price);
extraEdgeOverPp = extraFairOverProb - implOvr;   // BUG: raw edge
// ...
impliedOverProb: ... ? americanToImplied(extraCandidate.over_price) : null,
impliedUnderProb: ... ? americanToImplied(extraCandidate.under_price) : null,
```
`extraEdgeOverPp` / `extraEdgeUnderPp` feed into `buildCanonicalPropDecision` and affect the `verdict` for extra-line BLK cards.

**Fix:** Import `twoSidedFairProb` from `@cheddar-logic/models` (same pattern as `nhl-player-shots.js` after WI-0805) and replace both `americanToImplied` edge computations with `twoSidedFairProb(over, under) ?? americanToImplied(over)`. Update payload `impliedOverProb`/`impliedUnderProb` fields to use the devigged value.

---

### Item 2 — `card-model.js`: `calculateMoneylineEdge` / `oddsToProbability` legacy raw path (housekeeping)

`packages/models/src/card-model.js` exports `calculateMoneylineEdge` which uses `oddsToProbability` (raw, no devig). This predates `edgeCalculator.computeMoneylineEdge`.

**Action:** Grep all callers of `calculateMoneylineEdge` across the monorepo. If no live callers exist, delete the function and remove it from `module.exports`. If callers exist, migrate them to `edgeCalculator.computeMoneylineEdge` (which already does devig when `priceOpposite` is supplied).

---

### Item 3 — `mlb-model.js`: stale comment (housekeeping)

Line ~664:
```js
// American odds → implied probability (includes vig)
function mlToImplied(ml) { ... }
```
After WI-0805, `mlToImplied` is used only as an intermediate raw conversion step before normalization. The comment "includes vig" is misleading — update it to reflect its current role as a raw helper feeding the two-sided normalization.

**Fix:** Change comment to `// Raw implied probability — intermediate only; normalized via two-sided devig below`.
