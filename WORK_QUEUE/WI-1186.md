# WI-1186: Retire PM2 from Production — Migrate to systemd-only Process Management

**ID:** WI-1186
**Owner agent:** unassigned
**CLAIM:** unassigned 2026-04-26T00:00:00Z

---

## Goal

Remove PM2 from the Pi entirely and make systemd the sole process manager for `cheddar-worker` and `cheddar-web`. Eliminate the dual-management footgun where `pm2 stop all` during deploy leaves PM2 processes offline for the remainder of the session even after systemd restarts them.

## Why

The deploy workflow calls `pm2 stop all` before migrations and only restarts via `systemctl`. PM2 never gets told to restart — so `pm2 list` shows both processes offline until someone manually runs `pm2 restart`. This is confusing, error-prone, and unnecessary: systemd already owns both services via `cheddar-worker.service` and `cheddar-web.service` with `Restart=always` and watchdog timers. PM2 provides zero value here and adds noise.

The cleanest fix is not to keep patching the deploy to also call `pm2 restart` — it's to remove PM2 from the equation entirely so there's one source of truth.

## Scope

**Files to modify:**
- `.github/workflows/deploy-production.yml`
- `.github/workflows/deploy-branch.yml` (if PM2 references exist)
- `scripts/start-all.sh` (if it starts processes via PM2)
- `deploy/` — any PM2 ecosystem files (e.g. `ecosystem.config.js`, `ecosystem.config.cjs`)

**Out of scope:**
- Any changes to systemd service unit files (already correct)
- cheddar-fpl-sage PM2 config (only if it is also systemd-managed; otherwise leave it)
- Local dev scripts (`scripts/dev-mode-local.sh`, etc.)

## Acceptance

1. `pm2 list` on the Pi returns an empty table (or PM2 is uninstalled) after deploy
2. `systemctl status cheddar-worker cheddar-web` both show `active (running)` immediately after deploy
3. Deploy workflow contains no `pm2 stop`, `pm2 start`, or `pm2 restart` calls
4. Pi survives a reboot: `cheddar-worker` and `cheddar-web` auto-start via systemd without any `pm2 resurrect`
5. Existing smoke tests in the deploy workflow pass unchanged

## Implementation Steps

### Step 1 — On the Pi (one-time manual step before merging)

SSH into the Pi and drain PM2:

```bash
# Stop and delete all PM2-managed processes
pm2 delete all
pm2 save --force
# Disable PM2 startup hook so it does not resurrect on reboot
pm2 unstartup
# Optionally uninstall PM2 globally
npm uninstall -g pm2
```

Verify systemd takes over:
```bash
sudo systemctl status cheddar-worker cheddar-web
```

### Step 2 — Deploy workflow cleanup

Remove the PM2 stop/restart blocks from `.github/workflows/deploy-production.yml`:

**Remove (pre-migration stop block, ~line 294):**
```bash
PM2_BIN=$(find /home -name pm2 -path "*/bin/pm2" 2>/dev/null | head -1 || true)
if [ -n "$PM2_BIN" ]; then
  "$PM2_BIN" stop all || true
fi
```

**Remove (post-deploy restart block added by WI-1186 interim fix):**
```bash
# Restart PM2-managed processes if PM2 is active (keeps pm2 list in sync after deploy)
if [ -n "$PM2_BIN" ]; then
  "$PM2_BIN" restart cheddar-worker || true
  "$PM2_BIN" restart cheddar-web || true
  echo "PM2 processes restarted"
fi
```

Also remove the `PM2_BIN` variable assignment if it is no longer referenced anywhere in the file.

Repeat for `deploy-branch.yml` if it has the same pattern.

### Step 3 — Remove PM2 ecosystem files

```bash
find /opt/cheddar-logic -name "ecosystem.config.*" | xargs rm -f
```

Remove from repo if checked in.

### Step 4 — Update `scripts/start-all.sh`

Replace any `pm2 start` calls with a comment pointing to systemd:

```bash
# Processes are managed by systemd on the Pi.
# To start: sudo systemctl start cheddar-worker cheddar-web
# To check: sudo systemctl status cheddar-worker cheddar-web
```
