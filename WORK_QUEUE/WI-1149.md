# WI-1149: Independent Controls And Polish For Each Results View

**ID**: WI-1149
**Goal**: Give each view its own purpose-fit controls and refine the layout now that they are fully separated.

**Scope**:
- web/src/app/results/page.tsx
- web/src/app/results/projections/page.tsx
- web/src/components/results/ (filter/control components as needed)

**Out of scope**:
- New projection model families or accuracy algorithms
- Changes to API route response shapes
- Shared/global filter state lifted to layout (each page owns its own filters)

**Acceptance**:

Betting view (`/results`):
- Existing filters (sport, card category, market, 60%+ confidence toggle) remain and apply only to the betting view.
- Decision tiers and ledger respond to filter changes as before.

Projections view (`/results/projections`):
- Card family selector lets user toggle between All / NHL 1P / MLB F5 (or any active projection families).
- Selecting a family scopes the settlement cards and accuracy metrics to that family.
- Date range or season selector (if data supports it) to scope projection accuracy window.
- Layout is optimized for reading accuracy metrics: MAE, bias, directional accuracy, and Brier score are clearly labeled with context (e.g. what "good" looks like for each metric).

Both views:
- No layout regressions on mobile viewports.
- Page titles and metadata (`<title>`) are distinct: "Betting Results | Cheddar" vs "Projection Accuracy | Cheddar".
