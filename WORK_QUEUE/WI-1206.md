# WI-1206: Legacy Alignment & Consolidation (Post-WI-1205 Drift Cleanup)

**ID**: WI-1206
**Goal**: Eliminate drift introduced by WI-1205 shared helper pattern; ensure all model layers use single truth builder for decision_v2 production and narrow public API surface.

---

## Context

WI-1205 extracted `ensureCanonicalDecisionV2` into a shared worker helper and wired it into MLB, NHL, and NBA model runners. Post-implementation scan identified three cleanup items:

1. **NHL player shots** still writes `decision_v2` without calling shared helper (no `source` stamping)
2. **NBA** has two pre-existing `syncCanonicalDecisionEnvelope` call sites that may conflict with or be redundant with post-write enforcement
3. **Helper API** exports internals only used by tests; should tighten to single public symbol

This WI consolidates those loose ends to complete the contract enforcement universalization.

---

## High-Priority Items

### 1. Align NHL Player Shots to Shared Helper

**File**: `apps/worker/src/jobs/run_nhl_player_shots_model.js`

**Current State**:
- Writes four card types with inline `decision_v2` blocks (fullDecisionV2, firstPeriodDecisionV2, blkDecisionV2, extraDecisionV2)
- `decision_v2.source` NOT stamped on any card type (grep confirms zero matches)
- Already at 100% coverage (WI-1205 scope was 0% → ~100% projection cards; player shots was already high, just skipped source alignment)

**Required Changes**:
1. Add import at top of file:
   ```javascript
   const { ensureCanonicalDecisionV2 } = require('./helpers/ensure-canonical-decision-v2.js');
   ```

2. Locate all `insertCardPayload()` calls for shot card types

3. Before each write, call:
   ```javascript
   ensureCanonicalDecisionV2(cardPayloadData);
   ```

**Acceptance**:
- Post-run SQL shows 100% of new NHL player shot/BLK cards have `decision_v2.source = 'decision_authority'`
- All four card variants (full, 1P, BLK, extra) consistent
- Tests still pass

### 2. Audit & Consolidate NBA Envelope Rewrites

**File**: `apps/worker/src/jobs/run_nba_model.js`

**Current State**:
- Line 1692: `syncCanonicalDecisionEnvelope(payload, {...})` called during card generation (project card context)
- Line 3095: `syncCanonicalDecisionEnvelope(card.payloadData, {...})` called during market call card generation (withoutOddsMode override)
- Line 3252: `ensureCanonicalDecisionV2(entry.card.payloadData)` called post-transaction before write

**Investigation Task**:
1. Read both `syncCanonicalDecisionEnvelope` call sites with surrounding context (20 lines each)
2. Determine purpose: are they fixing edge cases that shared helper handles, or do they serve a distinct purpose?
3. Check if shared helper call at 3252 supersedes or conflicts with 1692/3095

**Decision Tree**:
- **If redundant**: Mark for removal; add comment explaining why they are now handled by `ensureCanonicalDecisionV2`
- **If complementary**: Document the reason in code (e.g., withoutOddsMode requires override AFTER helper runs) and add assertion to verify no conflict
- **If conflicting**: Consolidate into single pre-write strategy; verify no envelope shape corruption

**Acceptance**:
- No envelope shape mismatches in post-run SQL
- All three paths (1692, 3095, 3252) are either removed, consolidated, or explicitly documented as complementary
- Tests pass; no regression in decision quality
