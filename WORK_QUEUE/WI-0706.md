# WI-0706: Persist manager profile across sessions — onboarding state is lost on reload

**ID**: WI-0706
**Goal**: A manager profile created through the onboarding flow must survive page reloads and tab switches. Currently the `FPLOnboarding` component submits answers to the backend but the result is never stored client-side, so the profile is gone the moment the user navigates away. The product shell has no awareness of whether onboarding has been completed.

**Depends on**: none (backend profile API exists at `/api/v1/profiles` per WI-0653's scope, even if gated by a feature flag)

**Scope**:
- `WORK_QUEUE/WI-0706.md`
- `web/src/components/fpl-product-shell.tsx` — load profile on mount, pass to child sections, gate onboarding completion state
- `web/src/components/fpl-onboarding.tsx` — emit completed profile via `onComplete` callback; do not reload page
- `web/src/lib/fpl-api.ts` — add `getManagerProfile(userId)` and confirm `createManagerProfile` exists

**Out of scope**:
- Auth / multi-user session management
- Profile editing after onboarding (PATCH endpoint wiring)
- Any backend changes

**Acceptance**:
- After completing onboarding, the profile is stored (backend call + local state in shell).
- Navigating to another tab and back to Profile tab shows the completed profile, not the onboarding form.
- If the backend call fails, the error is surfaced inline — onboarding does not silently swallow failures.
- Profile tab shows a "Profile set" summary view when profile exists; shows onboarding form when it does not.

**Owner agent**: unassigned
**Time window**: TBD
**Coordination flag**: solo
**LOE**: S

**Root cause**:
`FPLProductShell` renders `<FPLOnboarding userId="demo" mock />` unconditionally with `mock` mode. No profile is fetched on mount. No `onComplete` handler stores the result. The `mock` prop likely bypasses the real API entirely, so even if onboarding is submitted nothing is persisted.

**Tests to run**:
- `npm --prefix web run build`

**Manual validation**:
- Complete onboarding in Profile tab
- Switch to Build Lab tab then back to Profile tab
- Profile summary is shown, not the onboarding form again
- Reload the page — profile summary loads from backend, not blank form
