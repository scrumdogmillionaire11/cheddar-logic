# WI-1253: Formalize Projection-Only MLB Pitcher-K Contract

**ID**: WI-1253
**Goal**: Make MLB pitcher-K output intentionally projection-only, honest, and useful without any live odds dependency.

**Context**:

Current behavior already runs in projection-only mode, but docs/tests/work-item language still imply ODDS_BACKED execution. This creates product-contract confusion and misleading expectations.

**Scope**:
- `apps/worker/src/models/mlb-model.js`
- `apps/worker/src/__tests__/mlb-k-market-edge.test.js`
- `apps/worker/src/__tests__/run-mlb-model.dual-run.test.js`
- `docs/pitcher_ks/05market.md`
- `docs/pitcher_ks/07output.md`
- `WORK_QUEUE/WI-1253.md`

**Out of scope**:
- Live line ingestion or odds API integration
- ODDS_BACKED execution or market freshness gating
- Actionable betting publication without user-provided line/price

**Acceptance**:
- `mode='PROJECTION_ONLY'` is treated as intentional contract behavior, not a bug.
- Runtime/tests/docs remove or disable misleading ODDS_BACKED expectations for pitcher-K card generation.
- Pitcher-K output surface is limited to projection intelligence only:
  - projected strikeouts
  - fair ladder probabilities
  - posture labels: `UNDER_CANDIDATE`, `OVER_CANDIDATE`, `NO_EDGE_ZONE`, `TRAP_FLAGGED`, `DATA_UNTRUSTED`, `UNDER_LEAN_ONLY`
- System never emits `PLAY` for pitcher-K without a user-provided line/price context.
- Regression tests assert projection-only reason-code invariants and no hidden upgrade to executable edge states.
- Posture assignment must be derived from deterministic multi-input projection logic:
  - opponent K factor
  - projected innings (leash bucket)
  - pitcher K baseline
- Posture cannot be assigned from a single input or a fallback-only path.
- Tests must validate that posture changes when each component is varied independently.
