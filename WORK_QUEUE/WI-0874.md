# WI-0874: POTD MLB model integration — wire MLB model edge into POTD candidate scoring

**ID**: WI-0874
**Goal**: When the POTD engine processes an MLB game, enrich each market candidate with the MLB model's win probability and edge (from `computeMLBDriverCards()`), so the POTD scoring reflects model edge rather than only consensus-line devig.

**Scope**:
- `apps/worker/src/jobs/potd/run_potd_engine.js`

**Out of scope**:
- Changes to signal-engine.js (consensus scoring logic stays untouched)
- Changes to NHL/NBA candidate building
- New DB tables

**Depends on**: WI-0872, WI-0873 (needs full driver card support), WI-0871 (F5 ML on full model path)

**Acceptance**:
1. `buildMLBModelCandidates(game)` helper added in `run_potd_engine.js`; calls `computeMLBDriverCards(game.gameId, game)` and translates each driver card to a POTD candidate with `modelWinProb`, `edgePct`, `sport: 'MLB'`, and `marketType` matching the driver card market
2. In `gatherBestCandidate()`, when a game's `sport === 'MLB'`, MLB model candidates are built and appended to `scoredCandidates` alongside consensus candidates
3. `scoreCandidate()` handles MLB model candidates correctly: when `candidate.modelWinProb` is explicitly populated, uses it directly rather than recomputing from devig
4. Graceful degradation: when `computeMLBDriverCards()` returns no driver cards with `ev_threshold_passed: true`, returns empty array — consensus candidates still score
5. NHL and NBA candidate scoring unchanged (no regressions)
6. ≥ 2 new unit/integration tests: one confirming MLB model candidate appears when pitcher data present, one confirming empty array when no pitcher data

- `npm --prefix apps/worker test -- --testPathPattern=mlb-model`

**Manual validation**:
- Run POTD engine dry-run with MLB game in odds feed and pitcher data in snapshot; confirm MLB model candidate appears in scored list with `modelWinProb` populated

**Guard for WI closeout**:
- Before marking complete: both test patterns pass and no NHL/NBA test regressions
