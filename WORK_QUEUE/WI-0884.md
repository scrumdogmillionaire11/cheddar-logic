# WI-0884: POTD multi-day sanity tracking — potd_daily_stats table and diagnostic script

**ID**: WI-0884
**Goal**: Persist a lightweight stats row for every POTD engine run (fired or not) so the system can self-report fire rate, average edge, and average stake — making miscalibrated thresholds visible before they cost money.

**Scope**:
- `packages/data/db/migrations/*.sql` — one new migration creating `potd_daily_stats` table
- `apps/worker/src/jobs/potd/run_potd_engine.js`
- `apps/worker/src/jobs/potd/__tests__/run-potd-engine.test.js`
- `apps/worker/src/scripts/potd-sanity-check.js` (new file)
- `apps/worker/package.json` — add `"stats:potd"` script entry

**Out of scope**:
- Web API or UI surface for sanity stats — this is a worker-side diagnostic only
- Settlement outcomes in stats (those live in `potd_plays.result` already)
- Retroactive backfill of historical runs before this migration

**Acceptance**:
1. New migration creates `potd_daily_stats` table with columns: `play_date TEXT PRIMARY KEY`, `potd_fired INTEGER NOT NULL` (0 or 1), `candidate_count INTEGER NOT NULL`, `viable_count INTEGER NOT NULL`, `top_edge_pct REAL`, `top_score REAL`, `selected_edge_pct REAL` (null when no play), `selected_score REAL` (null when no play), `stake_pct_of_bankroll REAL` (null when no play), `created_at TEXT NOT NULL DEFAULT CURRENT_TIMESTAMP`
2. `runPotdEngine` writes a `potd_daily_stats` row on every successful run — regardless of whether a play fires; row is upserted (`INSERT OR REPLACE`) so re-runs on the same date overwrite
3. `candidate_count` = total scored candidates before threshold filter; `viable_count` = candidates that passed minConfidence + minEdgePct filter (i.e. pool size from WI-0882, or `selectBestPlay` viable count if WI-0882 not yet merged)
4. New file `apps/worker/src/scripts/potd-sanity-check.js` queries last 30 days from `potd_daily_stats` and prints a summary table: columns `play_date | fired | viable | top_edge | stake_pct`; final row shows `fire_rate`, `avg_edge_when_fired`, `avg_stake_when_fired`; exits 0
5. `npm --prefix apps/worker run stats:potd` added to `apps/worker/package.json` and runs `potd-sanity-check.js`
6. Test: engine run with a valid candidate writes a stats row with `potd_fired = 1` and non-null `selected_edge_pct`
7. Test: engine run with no viable candidates writes a stats row with `potd_fired = 0` and null `selected_edge_pct`
8. Test: second run on same `play_date` upserts (row count stays 1, values updated)
9. All pre-existing `run-potd-engine` tests pass
