# WI-0628: Surface edge_vs_consensus and edge_vs_best_available in cards UI

**ID**: WI-0628
**Goal**: Display the two projection-comparison edge numbers (`edge_vs_consensus_pts`, `edge_vs_best_available_pts`) computed by `compareProjection` in the cards UI, replacing the single stale `edgePoints` display for cards that have consensus data.

**Background**:
WI-0571 built `compareProjection` in `packages/odds/src/market_evaluator.js` and wired it into `apps/worker/src/models/cross-market.js` for NHL and NBA. The values are computed at model run time and stored in card payload `odds_context`. However, `web/src/components/cards-page-client.tsx` still shows only the single legacy `edgePoints` field. The WI-0571 scope included updating the UI to show both numbers with a fallback, but this was not implemented.

**Scope**:
- `web/src/components/cards-page-client.tsx` — in the spread/total math detail section, read `displayPlay.odds_context?.projection_comparison` and display:
  - "Edge vs market: +X.X pts" (edge_vs_consensus_pts)
  - "Edge at best book: +X.X pts (BookName)" (edge_vs_best_available_pts)
  - Fall back to existing `edgePoints` display when `projection_comparison` is absent (old snapshots)
- `web/src/lib/game-card/transform.ts` — pass `odds_context.projection_comparison` fields through to `displayPlay` if not already present

**Out of scope**:
- Changes to how `compareProjection` is called in the worker
- Props and team totals (WI-0571 deferred these)

**Acceptance**:
- Cards with `odds_context.projection_comparison` show both edge numbers in the UI.
- Cards without `projection_comparison` fall back to the existing `edgePoints` display unchanged.
- `npx tsc --noEmit --project web/tsconfig.json` passes.
- `npm --prefix web run lint` passes.
