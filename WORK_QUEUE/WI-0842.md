# WI-0842: Settlement Expiry Sweep — Void Unresolvable Pending card_results

| Field | Value |

| **Goal** | 6,944 `card_results` rows have `result IS NULL` with the oldest dating back to 2026-03-13 (~24 days). `settle_pending_cards` only settles rows where a matching `game_results` entry exists, so cancelled/postponed games and ended seasons (NCAAM March Madness post-championship) will never resolve. ## Context

Root cause: `settle_pending_cards` joins `card_results` to `game_results` on `game_id`. For any game where `game_results` was never inserted (cancelled game, NCAAM bracket game with no score feed, FPL ended season), the card_result row stays `result=NULL` indefinitely.

**Boundary with `settle_pending_cards.js` `nonActionableAutoClosed`:** That path auto-voids non-actionable pending rows but requires a `game_results` row to exist for the join to fire. This sweep handles the complementary case where `game_results` is permanently absent — the two paths do not overlap.

The `settle_game_results` job fetches ESPN scores into `game_results` and is not related to this fix. The backlog growth is not a job failure, it is a structural coverage gap.

**Run mode:** Run as a one-off CLI script on first deploy (`node apps/worker/src/jobs/sweep_stale_settlements.js`) to clear the existing 6,944 backlog. After verifying results, add a weekly entry in `ecosystem.config.js` so future zombie rows do not accumulate.

```sql
-- Scope of affected rows (as of 2026-04-06):
SELECT COUNT(*) FROM card_results
WHERE result IS NULL
  AND created_at < datetime('now', '-48 hours');
-- Result: ~6,944
```

## Implementation Notes

1. Create `sweep_stale_settlements.js` as a standalone job following the pattern of `backfill_card_results.js`:

```javascript
function sweepUnresolvable(db) {
  const rows = db.prepare(`
    SELECT cr.id, cr.game_id, LOWER(g.status) AS game_status
    FROM card_results cr
    JOIN games g ON g.game_id = cr.game_id
    LEFT JOIN game_results gr ON gr.game_id = cr.game_id
    WHERE cr.result IS NULL
      AND cr.created_at < datetime('now', '-48 hours')
      AND LOWER(g.status) IN ('final', 'ft', 'completed', 'cancelled', 'postponed')
      AND gr.game_id IS NULL
  `).all();

  if (rows.length === 0) return 0;

  const stmt = db.prepare(`
    UPDATE card_results
    SET result = 'void', status = 'settled', pnl_units = 0,
        settled_at = datetime('now'), updated_at = datetime('now')
    WHERE id = ?
  `);
  for (const row of rows) stmt.run(row.id);
  return rows.length;
}
```

2. Gate writes behind `--dry-run` flag at top of `main()`, following the pattern of `backfill_card_results.js`.
3. Log result to console and include count in the `job_runs` metadata payload.
