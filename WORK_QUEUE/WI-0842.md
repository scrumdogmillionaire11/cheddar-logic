# WI-0842: Settlement Expiry Sweep — Void Unresolvable Pending card_results

| Field | Value |
|---|---|
| **ID** | WI-0842 |
| **Goal** | 6,944 `card_results` rows have `result IS NULL` with the oldest dating back to 2026-03-13 (~24 days). `settle_game_results` only settles rows where a matching `game_results` entry exists, so cancelled/postponed games and ended seasons (NCAAM March Madness post-championship) will never resolve. These zombie rows inflate the pending backlog forever and prevent accurate settlement reporting. Implement a targeted sweep that voids card_results whose underlying game is confirmed final (or status `cancelled`/`postponed`) with no `game_results` record after a 48h grace period, and marks them `result='void'`, `status='settled'`, `pnl_units=0`. |

## Context

Root cause: `settle_game_results` joins `card_results` → `game_results` on `game_id`. For any game where `game_results` was never inserted (cancelled game, NCAAM bracket game with no score feed, FPL ended season), the card_result row stays `result=NULL` indefinitely.

The `settle_game_results` job runs successfully every cycle (green in `job_runs`) — the backlog growth is not a job failure, it's a coverage gap.

```sql
-- Scope of affected rows (as of 2026-04-06):
SELECT COUNT(*) FROM card_results
WHERE result IS NULL
  AND created_at < datetime('now', '-48 hours');
-- Result: ~6,944
```

## Implementation Notes

1. Add a sweep function after the main settlement pass:
```javascript
function sweepUnresolvable(db) {
  const rows = db.prepare(`
    SELECT cr.id, cr.game_id, g.status as game_status
    FROM card_results cr
    JOIN games g ON g.game_id = cr.game_id
    LEFT JOIN game_results gr ON gr.game_id = cr.game_id
    WHERE cr.result IS NULL
      AND cr.created_at < datetime('now', '-48 hours')
      AND g.status IN ('final', 'cancelled', 'postponed')
      AND gr.game_id IS NULL
  `).all();

  if (rows.length === 0) return 0;

  const stmt = db.prepare(`
    UPDATE card_results
    SET result = 'void', status = 'settled', pnl_units = 0,
        settled_at = datetime('now'), updated_at = datetime('now')
    WHERE id = ?
  `);
  for (const row of rows) stmt.run(row.id);
  return rows.length;
}
```

2. Gate behind `--dry-run` or default-run in `settle_game_results.js` near end of main loop.
3. Log result to console and include in the `job_runs` metadata.
