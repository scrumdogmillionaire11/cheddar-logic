## Core Principle (Non-Negotiable)

Truth is created once at the model layer and never reconstructed downstream.

- `decision_v2` MUST be produced inside model jobs (`run_*_model.js`)
- No downstream system (web, transform, results, POTD) may construct, infer, or repair `decision_v2`
- All consumers are strictly read-only with respect to `decision_v2`
- If `decision_v2` is missing at model output → treat as INVALID, not PASS

## Implementation

### Step 1: Understand the Existing `ensureCanonicalDecisionV2` Pattern

MLB has a working pattern at `run_mlb_model.js:727`. Read it before modifying NHL/NBA. The function:

1. Initializes `payload.decision_v2 = {}` if missing
2. Normalizes `official_status` from legacy fields (`payload.status`, `payload.action`, `payload.classification`) as a **fallback only** (not as a replacement for a real decision)
3. Writes `canonical_decision` envelope with lifecycle
4. Sets `decision_v2.source = 'decision_authority'`
5. Sets `decision_v2.canonical_envelope_v2` with full resolution fields

**For projection cards specifically:** The official_status will typically be `PASS` (projection cards are not betting recommendations). The `ensureCanonicalDecisionV2` call guarantees the field exists and has a valid envelope, not that the card becomes actionable.

### Step 2: INVALID Emitting for Failure Cases

When `ensureCanonicalDecisionV2` is invoked but the payload has no valid status signal at all (no `action`, no `status`, no `classification`, no existing `decision_v2.official_status`), the function must emit `official_status = 'INVALID'` with a structured reason code instead of defaulting to `PASS`.

Modify `ensureCanonicalDecisionV2` in `run_mlb_model.js` to handle this case:

```javascript
// CURRENT (run_mlb_model.js ~line 751):
const officialStatus = normalizeOfficialStatus(
  payload.decision_v2.official_status ||
    payload.status ||
    payload.action ||
    payload.classification,
);

// AFTER: If all sources are absent, emit INVALID instead of defaulting PASS
const rawStatusSource =
  payload.decision_v2.official_status ||
  payload.status ||
  payload.action ||
  payload.classification;

const officialStatus = rawStatusSource
  ? normalizeOfficialStatus(rawStatusSource)
  : 'INVALID';

const invalidReasonCode = !rawStatusSource
  ? (payload.pass_reason_code || 'MISSING_DECISION_INPUTS')
  : null;

if (officialStatus === 'INVALID') {
  payload.decision_v2.official_status = 'INVALID';
  payload.decision_v2.primary_reason_code =
    invalidReasonCode || payload.decision_v2.primary_reason_code || 'MISSING_DECISION_INPUTS';
  payload.decision_v2.source = 'decision_authority';
  payload.canonical_decision = {
    official_status: 'INVALID',
    is_actionable: false,
    tier: 'INVALID',
    reason_code: payload.decision_v2.primary_reason_code,
    source: 'decision_authority',
    lifecycle: [{
      stage: 'publisher',
      status: 'INVALID',
      reason_code: payload.decision_v2.primary_reason_code,
    }],
  };
  return; // Don't continue writing partial envelope
}
```

**Then port this shared logic** into NHL and NBA models so all three models call the same function from a shared location — or duplicate it explicitly (do not import from `web/`).
