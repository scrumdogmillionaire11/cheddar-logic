# WI-1025: NBA Phase 3B — Regime detection with measurable triggers

**ID**: WI-1025
**Goal**: Tag each NBA game with a regime label (PLAYOFF_PUSH, TANK_MODE, INJURY_ROTATION, REST_HEAVY) using only objective, measurable triggers — no story tags.

**Depends on**: WI-1021 (dynamic baselines), WI-1022 (lineup-aware impact providing totalPointImpact), WI-1023 (vol_env sigma active)

**Scope**:
- `apps/worker/src/utils/nba-regime-detection.js` (new file)
- `apps/worker/src/jobs/run_nba_model.js`
- `apps/worker/src/__tests__/run-nba-model.test.js`

**Out of scope**:
- External game-state APIs (detection uses only existing team_metrics_cache and schedule data)
- Market intelligence wiring (WI-1026)
- NHL/MLB regime detection
- UI changes

**Regime trigger definitions (all must be objective and testable)**:

`PLAYOFF_PUSH`:
- Trigger: either team has a winning percentage ≥ 0.500 AND the game date is after March 1 AND the team's position is within 3 games of the 10th seed (last play-in spot)
- Data source: `team_metrics_cache.wins`, `team_metrics_cache.losses`, `game_time_utc`, and either a stored `playoff_seed_delta` field or a query against all NBA team win percentages in `team_metrics_cache`
- If `playoff_seed_delta` is not available and computing it would require a query against all 30 teams, skip this tag rather than approximate
- Effect: `{ paceMultiplier: 1.00, sigmaMultiplier: 0.95, blowoutRiskBoost: -0.10 }`

`TANK_MODE`:
- Trigger: either team has `wins_in_last_10 <= 2` AND the game date is after February 1 (late-season elimination scenario)
- Data source: `team_metrics_cache.recent_form` array — count wins in last 10 games
- Must be computed from actual game results, not assumed. If `recent_form` is null, tag is skipped.
- Effect: `{ paceMultiplier: 0.97, sigmaMultiplier: 1.10, blowoutRiskBoost: 0.15 }`

`INJURY_ROTATION`:
- Trigger: `availabilityGate.totalPointImpact >= 15` (from WI-1022 output — at least one significant impact player is out)
- Data source: directly from `availabilityGate` result, no additional query
- Effect: `{ paceMultiplier: 1.03, sigmaMultiplier: 1.15, blowoutRiskBoost: 0.10 }`

`REST_HEAVY`:
- Trigger: BOTH teams have `restDays >= 3`
- Data source: `computeRestDays` utility (already imported in `run_nba_model.js`)
- Effect: `{ paceMultiplier: 0.98, sigmaMultiplier: 1.05, blowoutRiskBoost: 0.0 }`

**Regime priority** (highest wins when multiple match): `INJURY_ROTATION > PLAYOFF_PUSH > TANK_MODE > REST_HEAVY > STANDARD`

**Acceptance**:
- `detectNbaRegime({ homeTeam, awayTeam, restDaysHome, restDaysAway, availabilityGate, teamMetricsHome, teamMetricsAway, gameDate })` exports from `nba-regime-detection.js`.
- Returns `{ regime, tags, modifiers }` where modifiers contains the paceMultiplier, sigmaMultiplier, and blowoutRiskBoost for the dominant regime.
- All four trigger paths have explicit boolean tests (not heuristics) that a different implementer could reproduce.
- When detection data is null (e.g., `recent_form` missing), the corresponding tag is silently skipped with no error — fail-open.
- `raw_data.nba_regime` stamped: `{ regime, tags, modifiers }`.
- Unit tests: `wins_in_last_10 = 1`, `game_date = March 15` → `TANK_MODE`; `restDaysHome = 4`, `restDaysAway = 3` → `REST_HEAVY`; null `recent_form` → `STANDARD`; combination: `INJURY_ROTATION + TANK_MODE` → `INJURY_ROTATION` wins.

**Modifier application in run_nba_model.js**:

After vol_env sigma multiplier (WI-1023):

```js
const regime = detectNbaRegime({ ... });
const regimeSigmaAdjusted = applyVolEnvSigmaMultiplier(
  effectiveSigma,
  oddsSnapshot.raw_data?.vol_env ?? null,
  volEnvMultipliers,
);
// Apply regime sigma on top
effectiveSigma = {
  ...regimeSigmaAdjusted,
  total: regimeSigmaAdjusted.total * regime.modifiers.sigmaMultiplier,
  margin: regimeSigmaAdjusted.margin * regime.modifiers.sigmaMultiplier,
};
```

Pace multiplier applied to `paceAnchorTotal` before blend. Clamped so total sigma multiplier chain (playoff × vol_env × regime) cannot exceed 2.0× or fall below 0.6×.
