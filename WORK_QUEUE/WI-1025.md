## Acceptance

### 1) New regime detection utility

Add:

```js id="nba5r1"
detectNbaRegime({
  homeTeam,
  awayTeam,
  restDaysHome,
  restDaysAway,
  availabilityGate,
  teamMetricsHome,
  teamMetricsAway,
  gameDate,
})
```

Returns:

```js id="nba5r2"
{
  regime,     // 'INJURY_ROTATION' | 'PLAYOFF_PUSH' | 'TANK_MODE' | 'REST_HEAVY' | 'STANDARD'
  tags,       // string[]
  modifiers: {
    paceMultiplier,
    sigmaMultiplier,
    blowoutRiskBoost,
  }
}
```

### 2) Trigger definitions (objective only)

All trigger paths must be explicit boolean tests with no approximation or narrative logic.

#### `PLAYOFF_PUSH`

Trigger when **either team** satisfies all of:

* winning percentage `>= 0.500`
* `gameDate` is after March 1
* team is within **3 games of the 10th seed**

Data source:

* `team_metrics_cache.wins`
* `team_metrics_cache.losses`
* `game_time_utc` / `gameDate`
* either:

  * stored `playoff_seed_delta`, or
  * a query against all NBA team records in `team_metrics_cache`

Rules:

* If `playoff_seed_delta` is unavailable and computing it would require a full standings query not already supported in the runtime, **skip this tag**
* Do not approximate standings from partial data

Effect:

```js id="nba5r3"
{ paceMultiplier: 1.00, sigmaMultiplier: 0.95, blowoutRiskBoost: -0.10 }
```

#### `TANK_MODE`

Trigger when **either team** satisfies all of:

* `wins_in_last_10 <= 2`
* `gameDate` is after February 1

Data source:

* `team_metrics_cache.recent_form`

Rules:

* Compute `wins_in_last_10` from actual recent game results
* If `recent_form` is null or malformed, **skip this tag**
* Do not infer tanking from season record alone

Effect:

```js id="nba5r4"
{ paceMultiplier: 0.97, sigmaMultiplier: 1.10, blowoutRiskBoost: 0.15 }
```

#### `INJURY_ROTATION`

Trigger when:

* `availabilityGate.totalPointImpact >= 15`

Data source:

* directly from WI-1022 availability output

Effect:

```js id="nba5r5"
{ paceMultiplier: 1.03, sigmaMultiplier: 1.15, blowoutRiskBoost: 0.10 }
```

#### `REST_HEAVY`

Trigger when:

* `restDaysHome >= 3`
* `restDaysAway >= 3`

Data source:

* `computeRestDays` utility already used in `run_nba_model.js`

Effect:

```js id="nba5r6"
{ paceMultiplier: 0.98, sigmaMultiplier: 1.05, blowoutRiskBoost: 0.0 }
```

#### `STANDARD`

If no regime triggers fire:

```js id="nba5r7"
{ paceMultiplier: 1.00, sigmaMultiplier: 1.00, blowoutRiskBoost: 0.0 }
```

### 3) Priority resolution

When multiple regime tags match, dominant regime is chosen by strict priority:

```txt id="nba5r8"
INJURY_ROTATION > PLAYOFF_PUSH > TANK_MODE > REST_HEAVY > STANDARD
```

Rules:

* `tags` should include all matched objective tags
* `regime` should be the single highest-priority tag
* `modifiers` should come from the dominant regime only

### 4) Fail-open behavior

If a trigger cannot be evaluated because required data is null or missing:

* silently skip that trigger
* do not throw
* do not invent substitutes
* continue evaluating remaining triggers

Example:

* `recent_form = null` → `TANK_MODE` skipped
* no `playoff_seed_delta` and no supported standings lookup → `PLAYOFF_PUSH` skipped

### 5) Raw data stamping

Stamp on card payload:

```js id="nba5r9"
raw_data.nba_regime = {
  regime,
  tags,
  modifiers,
}
```

### 6) Modifier application in `run_nba_model.js`

#### Sigma chain

Apply regime sigma after WI-1023 vol_env sigma:

```js id="nba5r10"
const regime = detectNbaRegime({ ... });

const sigmaAfterVolEnv = applyVolEnvSigmaMultiplier(
  priorSigma,
  oddsSnapshot.raw_data?.vol_env ?? null,
  volEnvMultipliers,
);

effectiveSigma = {
  ...sigmaAfterVolEnv,
  total: sigmaAfterVolEnv.total * regime.modifiers.sigmaMultiplier,
  margin: sigmaAfterVolEnv.margin * regime.modifiers.sigmaMultiplier,
};
```

#### Pace adjustment point

Apply `paceMultiplier` to `paceAnchorTotal` before blend:

```js id="nba5r11"
paceAnchorTotal = paceAnchorTotal * regime.modifiers.paceMultiplier
```

### 7) Sigma chain clamp

The full sigma multiplier chain must be bounded:

* minimum total chain multiplier: `0.60x`
* maximum total chain multiplier: `2.00x`

This applies to the combined effect of:

* playoff adjustment
* vol_env multiplier
* regime multiplier

Implementation requirement:

* clamp the final sigma-adjusted total and margin values, or clamp the final chain multiplier before applying it
* do not allow runaway expansion or compression

### 8) Unit tests

Cover:

1. `TANK_MODE`

   * `wins_in_last_10 = 1`
   * `gameDate = March 15`
   * result: `TANK_MODE`

2. `REST_HEAVY`

   * `restDaysHome = 4`
   * `restDaysAway = 3`
   * result: `REST_HEAVY`

3. Null `recent_form`

   * `recent_form = null`
   * result: `STANDARD` if no other triggers

4. `INJURY_ROTATION`

   * `availabilityGate.totalPointImpact >= 15`
   * result: `INJURY_ROTATION`

5. Priority resolution

   * `INJURY_ROTATION + TANK_MODE`
   * dominant regime = `INJURY_ROTATION`
   * `tags` include both

6. `PLAYOFF_PUSH`

   * valid win percentage
   * post–March 1 date
   * within 3 games of 10th
   * result: `PLAYOFF_PUSH`

7. Missing playoff delta / standings support

   * trigger skipped cleanly
   * no approximation

8. Sigma clamp

   * combined multiplier chain > 2.0 → clamped
   * combined multiplier chain < 0.6 → clamped

9. Raw data stamp

   * `raw_data.nba_regime` present with regime, tags, modifiers
