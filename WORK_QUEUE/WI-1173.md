# WI-1173: Prune Only Truly Unused MLB Pitcher Game-Log Fields

**ID**: WI-1173
**Goal**: Remove storage/write-path handling for MLB pitcher game-log fields that are truly unused, while preserving the fields that current MLB logic still consumes.

**Scope**:

- `apps/worker/src/jobs/pull_mlb_pitcher_stats.js`
- `apps/worker/src/jobs/__tests__/pull_mlb_pitcher_stats.test.js`

**Out of scope**:

- Removing `walks`, `batters_faced`, `home_away`, `number_of_pitches`, `innings_pitched`, or `strikeouts`

- Remove parsing/upsert handling for those fields from the pull job only if no downstream readers remain. Leave schema-drop work to a later serialized DB WI if desired.
- Add a narrow regression test that proves the pull job still persists the active K/BB/split fields required by MLB model consumers.

**Acceptance**:

- `pull_mlb_pitcher_stats.js` no longer parses or writes `hits` and `earned_runs` if repo-wide verification confirms no active readers.
- Active MLB consumers for `walks`, `batters_faced`, `home_away`, `number_of_pitches`, `innings_pitched`, and `strikeouts` remain intact.
- A narrow pull-job test covers the persisted field set and passes.
