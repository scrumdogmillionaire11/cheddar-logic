# WI-1109: Fix MLB projection settlement â€” team name mismatch in `settle_projections.js`

**ID**: WI-1109
**Goal**: Fix `mlb-f5` and `mlb-f5-ml` cards never settling because `settle_projections.js` builds `mlb_game_pk_map` lookup keys using full team names (e.g. `"SAN FRANCISCO GIANTS"`) while the table stores abbreviation keys (e.g. `"SF"`). Replace the ad-hoc key construction with `resolveMlbGamePk` / `buildGamePkLookupCandidates` from `settle_mlb_f5.js`, which already handles abbreviation normalization and tries 5 key candidates.

**Depends on**: none

**Scope**:
- `apps/worker/src/jobs/settle_projections.js`
- `apps/worker/src/jobs/settle_mlb_f5.js`
- `apps/worker/src/jobs/__tests__/settle_projections.test.js`

**Out of scope**:
- NHL settlement

- Any other card types

## Root cause

`settle_projections.js` has three identical broken lookup blocks (for `mlb-f5`, `mlb-f5-ml`, and `mlb-pitcher-k`):

```js
const gamePkKey =
  gameDate && homeTeam && awayTeam
    ? `${gameDate}|${homeTeam}|${awayTeam}`
    : null;

const pkRow = gamePkKey
  ? db.prepare('SELECT game_pk FROM mlb_game_pk_map WHERE game_pk_key = ?').get(gamePkKey)
  : null;
```

`games.home_team` stores full names like `"SAN FRANCISCO GIANTS"`, but `mlb_game_pk_map` keys use abbreviations like `"SF"`. No match is ever found â†’ every MLB card logs `no gamePk in mlb_game_pk_map` and skips.

`settle_mlb_f5.js` already solves this correctly via `resolveMlbGamePk(db, card)` which:
1. Calls `resolveMlbTeamAbbreviation(fullName)` to normalize team names
2. Tries 5 candidate key formats including both full-name and abbreviation variants
3. Also checks `mlb_probable_starter_map` as a fallback

## Acceptance

### 1) Export `resolveMlbGamePk` and `buildGamePkLookupCandidates` from `settle_mlb_f5.js`

These two functions are not currently exported. Add them to `module.exports`.

### 2) Replace all three broken lookup blocks in `settle_projections.js`

Locations:
- `mlb-f5` handler (~line 261)
- `mlb-f5-ml` handler (~line 327)
- `mlb-pitcher-k` handler (~line 553)

Replace each instance of:
```js
const homeTeam = card.home_team;
const awayTeam = card.away_team;
const gamePkKey =
  gameDate && homeTeam && awayTeam
    ? `${gameDate}|${homeTeam}|${awayTeam}`
    : null;

const pkRow = gamePkKey
  ? db
      .prepare('SELECT game_pk FROM mlb_game_pk_map WHERE game_pk_key = ?')
      .get(gamePkKey)
  : null;

if (!pkRow?.game_pk) {
  console.warn(`  [${JOB_NAME}] mlb ${card.game_id}: no gamePk in mlb_game_pk_map for key=${gamePkKey ?? ...}`);
  skipped++;
  continue;
}
```

With:
```js
const gamePk = resolveMlbGamePk(db, card);

if (!gamePk) {
  console.warn(`  [${JOB_NAME}] mlb ${card.game_id}: no gamePk resolved`);
  skipped++;
  continue;
}
```

Then replace all subsequent uses of `pkRow.game_pk` with `gamePk`.
