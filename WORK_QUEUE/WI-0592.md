# WI-0592: NHL shots props breakout-usage overlay

**ID**: WI-0592
**Goal**: Add a bounded breakout-usage overlay to the NHL shots props pipeline so clean OVER candidates with rising usage are no longer flattened by the current conservative projection stack, while preserving all existing pricing/anomaly vetoes.

**Scope**:
- `apps/worker/src/jobs/run_nhl_player_shots_model.js`
- `apps/worker/src/jobs/__tests__/run_nhl_player_shots_model.test.js`
- `apps/worker/src/jobs/report_telemetry_calibration.js`
- `apps/worker/src/jobs/__tests__/report_telemetry_calibration.test.js`
- `docs/NHL_PLAYER_SHOTS_PROP_MARKET.md`

**Out of scope**:
- New external lineup or PP-unit feeds
- Position-specific matchup data or opponent shot-allowance-by-position modeling
- DB migrations
- Changes to verdict names or the canonical prop-decision contract

**Background**:
The current NHL SOG props model uses a conservative projection stack with high-volume regression, PP contribution caps, anomaly guards, and V2 priced-edge vetoes. That keeps bad overs out, but it also risks flattening legitimate breakout-over candidates when a player's tonight usage should exceed baseline. This WI adds a bounded breakout-usage overlay using only current repo data: recent shot logs, season `shotsPer60`, projected TOI, PP TOI, team-level matchup factors, and MoneyPuck skater context.

**Acceptance**:
- The runner computes a breakout context for NHL player shots props with:
- `baseline_toi`
- `baseline_shots60`
- `baseline_sog_mu`
- `tonight_toi_proj`
- `adjusted_shots60`
- `breakout_sog_mu`
- `delta_mu`
- `breakout_score`
- The runner expands the player-log lookback from L5-only logic to use up to the last 10 games for breakout baseline/trend calculations.
- Breakout candidate thresholds are implemented as:
- `TOI_TREND_UP`: `last3_toi_avg - baseline_toi >= 1.0`
- `TOI_TREND_STRONG`: `last3_toi_avg - baseline_toi >= 2.0`
- `PP_ROLE_ELEVATED`: `ppToi >= 1.8`
- `SHOTS60_TREND_UP`: recent shots/60 at least `15%` above baseline
- candidate gate: `delta_mu >= 0.45` and `breakout_sog_mu >= 2.4`
- The breakout overlay is bounded and OVER-only:
- may raise projected TOI by at most `+2.5`
- may raise EV shot rate by at most `+15%`
- applies only to real, odds-backed, non-anomalous, non-role-blocked candidates
- The overlay never bypasses or overrides:
- `PROJECTION_CONFLICT`
- `PROJECTION_ANOMALY`
- `ROLE_IN_FLUX`
- negative priced-edge vetoes
- NHL shots payloads gain an optional `breakout` object with:
- `baseline_toi`
- `baseline_shots60`
- `baseline_sog_mu`
- `tonight_toi_proj`
- `adjusted_shots60`
- `breakout_sog_mu`
- `delta_mu`
- `score`
- `flags`
- `eligible`
- The runner emits breakout flags:
- `BREAKOUT_CANDIDATE`
- `TOI_TREND_UP`
- `TOI_TREND_STRONG`
- `PP_ROLE_ELEVATED`
- `SHOTS60_TREND_UP`
- `ENV_BOOST`
- `PRICE_TOO_JUICED`
- `BREAKOUT_BLOCKED_ANOMALY`
- `BREAKOUT_BLOCKED_ROLE`
- `BREAKOUT_BLOCKED_NO_REAL_LINE`
- Telemetry/reporting gains breakout segmentation so calibration output can compare breakout-tagged vs non-breakout-tagged NHL shots props for OVER hit rate, ROI, and CLV.
- Position-specific matchup remains explicitly neutral in this WI via a reserved `position_env_factor = 1.0` style placeholder; no position claims are introduced.
