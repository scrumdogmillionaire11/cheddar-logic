# WI-1182: Harden Pipeline Health for Odds Freshness + NHL SOG Sync Recency

**ID**: WI-1182
**Goal**: Stop repeated false-noisy pipeline alerts by making odds freshness checks self-healing and NHL SOG sync freshness checks feature-aware + retry-resilient.

**Context**:

- Current Discord alerts show repeated failures for:
  - `odds / freshness` with stale near-term games despite duplicate-game dedupe.
  - `nhl / sog_sync_freshness` when `sync_nhl_sog_player_ids` has no success in the last 24h.
- `check_pipeline_health` currently marks `sog_sync_freshness` as failed whenever no recent successful run exists and upcoming NHL games are present.
- NHL SOG sync execution is currently sparse (daily fixed window), so a single missed run can create a full-day alert streak.
- Odds freshness currently fails immediately for stale games inside the alert window without a targeted remediation pass.

**Scope**:

- `apps/worker/src/jobs/check_pipeline_health.js`
- `apps/worker/src/schedulers/player-props.js`

**Out of scope**:

- Changes to POTD selection, bankroll sizing, or publish logic.
- Broad scheduler cadence rewrites outside NHL SOG sync and freshness remediation.
- DB schema migrations.

**Acceptance**:

- `nhl / sog_sync_freshness` health check is feature-aware:
  - If NHL SOG sync feature is disabled, check is skipped or downgraded to non-failed status with explicit reason.
  - If enabled, failures reflect true stale-sync risk only.
- NHL SOG sync receives an additional deterministic retry cadence (or equivalent catch-up trigger) so one missed daily run does not leave a full 24h blind spot.
- `odds / freshness` check performs a bounded remediation pass for stale games inside the alert window before final fail classification.
- Final fail status for odds freshness occurs only when stale near-term games remain after remediation attempt; reason string includes attempted remediation summary.
- Existing duplicate-matchup dedupe behavior remains intact (no regression in duplicate-row handling).
- Test coverage explicitly exercises:
  - feature-disabled SOG sync check behavior,
  - feature-enabled stale SOG sync behavior,
  - odds freshness remediation-success and remediation-still-stale paths.
