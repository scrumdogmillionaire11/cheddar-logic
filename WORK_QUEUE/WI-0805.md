# WI-0805: Fix raw implied probability ‚Äî add two-sided vig removal across all sport models

| Field | Value |
|---|---|
| **ID** | WI-0805 |
| **Goal** | All sport models compute edge as `fairProb - rawImpliedProb`. The raw implied probability includes the bookmaker's vig (~4-7pp overround), so every apparent edge is inflated by roughly half the overround. The PLAY threshold (6pp) is measured against vig-inflated odds, leaving only ~1-2pp of real alpha requirement ‚Äî indistinguishable from noise. Fix all three sport paths to use two-sided devigged implied probability before computing edge. |
| **Priority** | High ‚Äî correctness of every PLAY/WATCH verdict depends on this |
| **Coordination flag** | needs-sync: `packages/models/src/edge-calculator.js` is a shared touchpoint |

---

## Root Cause

Three independent devig bugs, same class of error:

### Bug 1 ‚Äî NHL Player Props (SOG, BLK, SOG 1P)
`packages/models/src/nhl-player-shots.js` and `apps/worker/src/jobs/run_nhl_player_shots_model.js` both define a local `americanToImplied` with the comment **"raw, no vig removal"**. Edge is computed as:
```js
edge_over_pp = fairOverProb - americanToImplied(market_price_over);  // inflated by vig
```
Both `over_price` AND `under_price` are available at the call site. The fix is to normalize both sides before subtracting: `fairProb = rawOver / (rawOver + rawUnder)`.

### Bug 2 ‚Äî NHL Game Totals / Spread / ML + NBA Totals / Spreads
`run_nhl_model.js` and `run_nba_model.js` both call `edgeCalculator.computeEdge*` from `@cheddar-logic/models`. The edge calculator already has devig logic gated on a `priceOpposite` param ‚Äî but **no caller ever passes it**, so it falls through to raw implied and sets `VIG_REMOVAL_SKIPPED: true` on every result. Fix: pass the opposite side price in all `computeEdge*` call sites.

### Bug 3 ‚Äî MLB F5 Totals (ML) + K Props
`mlb-model.js` defines `mlToImplied` with comment **"includes vig"** and uses it in F5 ML edge: `homeEdge = winProbHome - mlToImplied(mlF5Home)`. Both `mlF5Home` and `mlF5Away` are already in scope ‚Äî devig inline using `raw / (rawHome + rawAway)`. MLB K props go through the same `edgeCalculator` path without `priceOpposite`.

---

## Threshold Impact

After fixing devig, the current PLAY threshold (`probEdge >= 0.06`) will require 6pp above fair odds instead of 6pp above vig-inflated odds. This is stricter ‚Äî expect fewer PLAY verdicts. **Threshold recalibration is out of scope for this WI** ‚Äî a separate WI should decide whether to lower the threshold based on live accuracy data from the `/results` page.
