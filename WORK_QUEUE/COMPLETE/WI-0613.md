# WI-0613: Delete committed scratch, debug, and backup files

**ID**: WI-0613
**Goal**: Remove development-time artifacts that were committed to the repo and add gitignore rules to prevent recurrence.

**Background**:
The following files have no place in the main branch:
- `Claude-research/` directory — research scratch dir containing `.db` binary, patch files, and `.py` scripts
- `scripts/debug/debug-card-insert.js` — one-off debug script
- `scripts/tmp_lineup_reason_check.py` — temporary analysis script
- `cheddar-fpl-sage/frontend/tailwind.config.js.backup` — editor backup file

The `.db` file inflates git history. The patch files in `Claude-research/` create confusion with the production `packages/models/src/decision-pipeline-v2.patch.js`. The debug scripts may be mistakenly run by operators.

**Scope**:
- Delete: `Claude-research/` (entire directory)
- Delete: `scripts/debug/debug-card-insert.js`
- Delete: `scripts/tmp_lineup_reason_check.py`
- Delete: `cheddar-fpl-sage/frontend/tailwind.config.js.backup`
- `.gitignore` — add `*.backup`, `Claude-research/`, `scripts/tmp_*`, `scripts/debug/`

**Out of scope**:
- Any changes to production code
- `git filter-branch` or history rewriting (content is gone from HEAD; history rewrite is a separate decision)

**Acceptance**:
- None of the listed files exist in the working tree after the commit.
- `.gitignore` prevents re-committing files matching those patterns.
- `git status` is clean after the commit.
