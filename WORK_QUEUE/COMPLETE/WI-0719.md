## Background

The NBA runner currently:
1. Computes market decisions (`computeNBAMarketDecisions`)
2. Runs `publishDecisionForCard` — which sets `decision_v2.official_status`
3. Runs semantic decision finalization inside `publishDecisionForCard`
4. **Then rewrites `classification`, `action`, `status`, `pass_reason_code`, and `decision_v2.official_status` again** for without-odds mode cards tagged `no_odds_mode`

Step 4 is the problem. It makes the DECISION_OUTPUT stage and PUBLISH_OUTPUT stage non-deterministic relative to the decision layer. Any audit snapshot taken after step 3 is silently invalidated by step 4.

There is a second issue: `applyUiActionFields` is not actually display-only today. It rebuilds `decision_v2` and rewrites semantic decision fields, which means any caller that runs it after publish can silently mutate the audit contract again.

## Implementation Plan

1. **Add explicit execution mode to card payload before decisioning** (not after):

   ```js
   // Set BEFORE publishDecisionForCard, not after
   card.payloadData.execution_status = withoutOddsMode ? 'PROJECTION_ONLY' : 'EXECUTABLE';
   ```

2. **Add `execution_status` field** to NBA market call card payloads in `generateNBAMarketCallCards`:
   - Without-odds mode cards: `execution_status: 'PROJECTION_ONLY'`
   - Odds-backed cards with pricing: `execution_status: 'EXECUTABLE'`
   - Cards where `canPriceCard` returns false: `execution_status: 'BLOCKED'`

3. **Remove the post-decision classification rewrite block** in `runNBAModel` (the block that currently runs after `applyUiActionFields` for `no_odds_mode` tagged cards):

   ```js
   // DELETE this block — it must no longer exist:
   // if (withoutOddsMode && Array.isArray(card.payloadData.tags) && card.payloadData.tags.includes('no_odds_mode')) {
   //   card.payloadData.classification = 'LEAN';
   //   card.payloadData.action = 'HOLD';
   //   card.payloadData.status = 'WATCH';
   //   ...
   // }
   ```

4. **Thread without-odds mode into `generateNBAMarketCallCards`** so that cards built in that function already carry the correct `execution_status`, `classification`, and `status` from construction — not as a post-hoc override.
   - Without-odds total call: `status='LEAN'`, `classification='LEAN'`, `execution_status='PROJECTION_ONLY'`
   - This is already partially correct — the `status` and `confidence` are set correctly. The issue is the `classification` / `decision_v2.official_status` rewrite happening after decisioning.

5. **Split semantic finalization from post-publish UI decoration**:

   - `publishDecisionForCard` becomes the only place that finalizes strict decision fields
   - `applyUiActionFields` becomes additive and display-only
   - Add `ui_display_status` mapping that can never surface `PLAY` for `PROJECTION_ONLY` or `BLOCKED`

6. **Add assertion guard** (dev/test mode only) after the publish step:

   ```js
   // After publishDecisionForCard, assert nothing downstream changes official_status
   const frozenStatus = card.payloadData.decision_v2?.official_status;
   // store frozenStatus for invariant check in tests
   ```

7. **Add `pricing_ready` → `execution_status` invariant**:
   - After `pendingCards` is fully built, assert: for every card where `execution_status === 'EXECUTABLE'`, `canPriceCard(card)` must be true.
   - If violated, log a warning and downgrade to `execution_status: 'BLOCKED'`.

8. **Add `ui_display_status`** to keep UI fields separate from semantic status:
   - `ui_display_status: 'PLAY' | 'WATCH' | 'PASS'`
   - Derived from `execution_status` and `status` — purely display, never used in downstream logic
