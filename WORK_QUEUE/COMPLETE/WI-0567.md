# WI-0567: Surface 1P vs full-game distinction on /results page

**ID**: WI-0567
**Goal**: NHL 1P total results are currently indistinguishable from full-game totals on the /results page. Return `market_period_token` from the API and use it to visually label rows and separate segment stats so 1P totals are clearly identifiable.

**Scope**:
- `web/src/app/api/results/route.ts`
  - Add `market_period_token` to the ledger SELECT (lines 744-795) so it is returned in the API response
  - Add to the ledger row transformation object (lines 797-1010)
- `web/src/app/results/page.tsx`
  - Add `market_period_token` (or derived `is1P: boolean`) to the `LedgerRow` type (lines 41-70)
  - Render a "1P" badge/pill on ledger rows where `market_period_token === '1P'` — desktop (lines 684-752) and mobile (lines 755-851)
  - In the Decision Tier segment table, ensure "nhl-pace-1p" and "nhl-pace-totals" groupings display distinct human-readable labels (e.g. "NHL 1P Totals" vs "NHL Totals") rather than raw card type strings
- `web/src/__tests__/ui-results-smoke.test.js`
  - Assert the results route exposes `market_period_token`
  - Assert the results page references `marketPeriodToken` and renders a `1P` badge path
- `web/src/__tests__/api-results-decision-segmentation.test.js`
  - Assert ledger rows include `marketPeriodToken` while preserving existing CLV coverage

**Out of scope**:
- Changing settlement logic or grading
- Adding a new filter/tab UI for 1P vs full-game
- Retroactive backfill of `market_period_token` in DB rows (it is computed at query time)

**Background**:
`market_period_token` is already computed correctly in the API's deduplication CTE (`results/route.ts:397-404`) and drives dedup partitioning. It is never returned to the frontend. As a result:
- A user looking at the /results ledger cannot tell if a row is a 1P total or a full-game total
- The segment stats table groups 1P and full-game cards under the same visual block if card types aren't inspected manually
- The only indirect signal is the `projection1p` sub-text in the Pick column, which is ambiguous

**Acceptance**:
- Every ledger row with `market_period_token === '1P'` renders a visible "1P" badge adjacent to the market/sport label — desktop and mobile.
- Rows with `market_period_token === 'FULL_GAME'` render no badge (no change to current appearance for full-game rows).
- The Decision Tier segment table shows a human-readable period label so "nhl-pace-1p" reads as "NHL 1P Totals" (or equivalent) rather than the raw card type slug.
- No visual regression to full-game NHL, NBA, or NCAAM rows.
- The `LedgerRow` type is updated and TypeScript compiles cleanly.
