# WI-0832: Backtest Closing-Line Substitution Removal — Enforce Pre-Game Odds Only

| Field | Value |
|---|---|
| **ID** | WI-0832 |
| **Goal** | Identify and remove every place in the backtest/audit simulation path where closing-line odds substitute for open or mid-market odds. Enforce that simulated bets use only prices that were available at `bet_placed_at`, not the final market price. Closing-line substitution biases backtest results in favour of the model because closing lines embed the wisdom of the broader market — the model appears to agree with sharp money that it never actually saw. |
| **Priority** | High |
| **Depends on** | WI-0827 (feature timestamp audit; same data availability principle applied to prices rather than features) |
| **Coordination flag** | None — audit and backtest tooling only; no production execution path changes |

---

## Background

WI-0827 guards against feature leakage at runtime. This WI addresses the equivalent problem in backtesting:

**Closing-line substitution** occurs when a backtest, lacking the price at bet placement time, uses the market's final price. This is problematic because:

1. **Closing lines are sharper** — they reflect late sharp action, syndicate moves, and injury news the model didn't have.
2. **Edge appears larger** — if the model called OVER at 52% and the market closed at 54%, the backtest records +2pp edge. The real edge at bet placement may have been -1pp.
3. **Win rate appears higher** — the model looks like it agrees with closing prices far more than it actually would have.

The CLV ledger (`ENABLE_CLV_LEDGER=true`) correctly tracks open vs close. The problem is in the separate audit/backtest scripts that use historical data for model validation.

## Scope

**Files to audit and change:**
- `apps/worker/src/audit/` — all audit scripts; identify any that read `market_price` without asserting it is the pre-game price
- Any backtest or simulation script that joins on `game_id + market_type` without a `snapshot_time < event_start` filter
- `docs/models/` — update any backtest methodology docs to document the enforcement

**Files to create:**
- `apps/worker/src/audit/validate_no_closing_line_sub.js` — static analysis / data check: scans the `odds_snapshots` table for any backtest run where the snapshot timestamp is after `event_start`; emits a report

**Out of scope:**
- Changing what prices are stored in the odds ingest pipeline
- Changing the production CLV ledger logic (already correct)
- Changing `risk_analysis.ipynb` — that is WI-0834

---

## Enforcement Rule

Every simulated bet in audit/backtest must satisfy:

```
snapshot_time <= event_start - PRICE_BUFFER_MINUTES
```

Default `PRICE_BUFFER_MINUTES = 60` (configurable). Snapshot timestamps that fall within the buffer or after `event_start` must be excluded from backtest edge calculations with a log warning.

Any backtest that cannot source a qualifying snapshot for ≥20% of events must fail with an explicit error rather than silently using closing prices.
