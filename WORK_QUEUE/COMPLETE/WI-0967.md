# WI-0967: Replace projection settlement with proxy-line grading on /results

**ID**: WI-0967
**Goal**: Remove simple HIT/MISS settlement logic and replace with proxy-line grading from `projection_proxy_evals` table, exposing direction (OVER/UNDER), tier (PLAY/LEAN/STRONG), edge, and full diagnostic data on `/results` page.

**Scope**:

- `web/src/app/api/results/projection-settled/route.ts`
- `web/src/components/results/ProjectionResultsTable.tsx`
- `web/src/app/results/page.tsx`
- `web/src/app/api/results/projection-metrics.ts`

**Out of scope**:
- Changes to worker settlement logic (settle_projections.js)
- Changes to projection_proxy_evals table schema
- Changes to main /results ledger display (ODDS_BACKED bets)
- Baseball/basketball projection handling

**Acceptance**:

- `/api/results/projection-settled` returns data from `projection_proxy_evals` instead of computing binary HIT/MISS.
- Endpoint exposes: `recommendedSide` (OVER/UNDER/PASS), `tier` (PLAY/LEAN/STRONG), `edgeVsLine`, `gradedResult` (WIN/LOSS/NO_BET), `tierScore`.
- `ProjectionResultsTable` displays: Date, Matchup, Projection, Direction badge, Tier badge, Edge, Outcome.
- All removed computation functions (resolveOutcome, resolveModelProjection, resolveDirection) are deleted.
- ProjectionSettledRow type is replaced with ProjectionProxyRow type.
- `/results` page loads and renders projection proxy data without errors.
- Projection metrics resolve settled prop actuals from `card_payloads.actual_result` fallback so NHL shots/blocks and MLB pitcher K rows do not remain stuck on "Awaiting settled outcome data" when game metadata is sparse.
- Projection Models table only renders families with settled actuals to avoid persistent placeholder rows.
