## Problem

Two NHL block data jobs exist but are not registered in the scheduler:

**`pull_nhl_player_blk.js`**
- Fetches per-game block data from the NHL API
- Writes to `player_blk_logs`
- Zero references in `apps/worker/src/schedulers/main.js`
- Must be run manually; data goes stale between manual runs

**`ingest_nst_blk_rates.js`**
- Reads a CSV from Natural Stat Trick and populates `player_blk_rates`
- Zero references in scheduler
- Requires downloading a CSV manually and pointing the script at it
- No documented cadence or runbook exists

**Downstream consumer:** `run_nhl_player_shots_model.js` reads `player_blk_rates`.

**`ingest_nst_blk_rates.js` requires three env vars that are undocumented:**

- `NHL_BLK_NST_SEASON_CSV_URL`, `NHL_BLK_NST_L10_CSV_URL`, `NHL_BLK_NST_L5_CSV_URL`
- These don't appear in `env.example` — they are silently unset in any new deployment
- The job throws immediately when they're absent, writing nothing to `player_blk_rates`
- `run_nhl_player_shots_model.js` then silently reads a null `blkRateRow` for every player, passes `null` to `weightedRateBlend`, and produces `blk_mu = 0` for all projections

**Impact:**

- Block-shot inputs silently go stale without manual intervention
- There is no alerting or watchdog when block rates are > N days old
- Model continues to run with stale or absent block rates, producing zero projections with no warning
- Any new deployment without the NST URL env vars will have broken BLK projections from day one

## Scope

| File | Change |
| ---- | ------ |
| `env.example` | Document `NHL_BLK_NST_SEASON_CSV_URL`, `NHL_BLK_NST_L10_CSV_URL`, `NHL_BLK_NST_L5_CSV_URL` in the `# NHL player blocked shots ingest` block with a comment explaining required cadence |
| `apps/worker/src/jobs/ingest_nst_blk_rates.js` | Replace hard throw on missing URLs with a structured warn-and-return: `{ inserted: 0, skipped: 0, error: 'missing_urls' }` — so a misconfigured deployment is observable in logs rather than exploding the scheduler chain |
| `apps/worker/src/schedulers/main.js` | Register `pull_nhl_player_blk` in the nightly NHL window (after `pull_nhl_player_shots`, before model run); add `maxAgeHours` guard if pattern exists |
| `apps/worker/src/jobs/pull_nst_blk_rates.js` | **New job** — automated HTTP fetch from Natural Stat Trick export URL (or documented API equivalent); parse CSV in-process; upsert into `player_blk_rates` |
| `apps/worker/src/schedulers/main.js` | Register `pull_nst_blk_rates` on weekly cadence (NST data updates weekly) |
| `apps/worker/src/jobs/run_nhl_player_shots_model.js` | (a) Add per-player `WARN` log when `blkRateRow` is null (e.g. `[run-nhl-player-shots-model] WARN: no player_blk_rates row for player ${playerId} season ${nhlSeasonKey} — BLK mu will be 0`); (b) add card-level staleness check: if `MAX(updated_at) FROM player_blk_rates` is > 8 days old, emit `missing_inputs: ['block_rates_stale']` on affected cards |
| `docs/runbooks/nst-blk-rates-manual-ingest.md` | If automated fetch is not feasible (NST blocks scraping): formal runbook documenting exact steps, required frequency, and how to trigger ingest manually |

## Out of Scope

- Refactoring `ingest_nst_blk_rates.js` CSV parsing internals beyond the error-handling fix
- Changes to `player_blk_logs` schema
- NHL shot model projection logic changes
- Setting production env vars directly (deployment config is out of scope; env.example documentation is in scope)
