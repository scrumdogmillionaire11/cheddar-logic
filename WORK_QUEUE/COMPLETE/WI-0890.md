# WI-0890: True-Play Authority Contract - Single Runtime Source of Truth

| Field | Value |
|---|---|
| **ID** | WI-0890 |
| **Goal** | Establish and enforce one authoritative source for surfaced true play selection so live cards cannot diverge across worker settlement backfill, API selection, and UI transform layers. |
| **Priority** | Critical |
| **Depends on** | none |
| **Coordination flag** | needs-sync |

---

## Background

Current behavior allows multiple layers to appear authoritative for true play selection:

- Settlement backfill can mutate display-adjacent fields after model output is produced.
- API selection logic can apply ambiguous fallback precedence.
- UI/transform can infer rank rationale without a single canonical contract.

This WI is the root dependency for WI-0891 and WI-0892 because those items assume a pre-defined authority contract. Without this contract, follow-on work can still produce split-brain behavior under race and replay scenarios.

## Must-haves (goal-backward)

### Truths (observable)
- A single contract defines the live true-play authority chain and tie-break order.
- /api/games uses the contract deterministically and emits stable authority metadata.
- Settlement job role is explicitly constrained (historical-only or live-authoritative) with no dual-authority overlap.
- Regression tests fail if authority source or precedence changes without contract updates.

### Artifacts
- docs/decisions/ADR-0003-true-play-authority.md defines canonical authority and tie-breakers.
- web/src/lib/games/route-handler.ts implements contract-based source selection and authority marker emission.
- apps/worker/src/jobs/settle_pending_cards.js enforces non-overlapping role relative to live authority.
- web/src/__tests__/game-card-decision-authority.test.ts asserts authority precedence and mutation resistance.

### Key links
- ADR contract -> API selection: route handler references contract constants/comments and applies contract order.
- ADR contract -> worker settlement: settlement logic documents/enforces historical-only boundaries on authority fields.
- API authority marker -> test assertions: test validates selected source and rank rationale remain stable before/after settlement sweep.
