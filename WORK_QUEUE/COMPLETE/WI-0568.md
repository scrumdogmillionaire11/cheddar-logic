# WI-0568: Market evaluator — consensus layer

**ID**: WI-0568
**Goal**: Build a `market_evaluator.js` module that computes a consensus view of each market (spread, total, moneyline) across all bookmakers. Consensus line and price become the canonical reference for edge calculation, replacing the current practice of using a single book's line (or the baked snapshot line).

**Scope**:
- `packages/odds/src/market_evaluator.js` — new module
  - `buildConsensus(entries, side?)` — given an array of bookmaker entries for a market, compute:
    - `consensus_line` — median of all available lines
    - `consensus_price_home`, `consensus_price_away` (or `over`/`under`) — median prices
    - `source_book_count` — how many books contributed
    - `dispersion_stddev` — stddev of lines across books
    - `consensus_confidence: "high" | "medium" | "low"` — derived from book count + stddev
      - high: ≥4 books, stddev ≤ 0.5
      - medium: ≥2 books OR stddev ≤ 1.0
      - low: everything else
  - Works for all three market types: `spread`, `total`, `h2h`
- `packages/odds/src/normalize.js`
  - Replace the current `spreads[0]` single-book pattern (already changed to best-line in prior WI) with calls to `buildConsensus` for each market
  - Add consensus outputs alongside the existing best-line outputs in the `odds` object:
    - `spreadConsensusLine`, `spreadConsensusConfidence`, `spreadDispersionStddev`, `spreadSourceBookCount`
    - `totalConsensusLine`, `totalConsensusConfidence`, `totalDispersionStddev`, `totalSourceBookCount`
    - `h2hConsensusHome`, `h2hConsensusAway`, `h2hConsensusConfidence`
- `packages/data/db/migrations/038_add_consensus_to_odds_snapshots.sql`
  - Add columns: `spread_consensus_line REAL`, `spread_consensus_confidence TEXT`, `spread_dispersion_stddev REAL`, `spread_source_book_count INTEGER`
  - Add columns: `total_consensus_line REAL`, `total_consensus_confidence TEXT`, `total_dispersion_stddev REAL`, `total_source_book_count INTEGER`
  - Add columns: `h2h_consensus_home REAL`, `h2h_consensus_away REAL`, `h2h_consensus_confidence TEXT`
- `packages/data/src/db.js` — `insertOddsSnapshot`
  - Add new consensus fields to INSERT
- `apps/worker/src/jobs/pull_odds_hourly.js` + `refresh_stale_odds.js`
  - Pass consensus fields through to `insertOddsSnapshot`
- `web/src/app/api/games/route.ts`
  - Expose consensus fields in the `odds` block of the API response
- `web/src/components/cards-page-client.tsx`
  - Update `GameData.odds` interface with consensus fields
  - For spread cards: use `spreadConsensusLine` (not best-line) as the reference for edge delta calculation
  - Display consensus confidence ("high/med/low") alongside market line if space permits
- `apps/worker/src/__tests__/market-evaluator-consensus.test.js`
  - Cover evaluator math and normalize output contract for consensus fields

**Out of scope**:
- Misprice detection (WI-0570)
- Projection comparison (WI-0571)
- Retroactive backfill of existing snapshots

**Background**:
Today, `normalize.js` takes `spreads[0]` (first bookmaker) or now the best-line (after the fix in this sprint). Both are wrong as a reference for edge math. Best-line overstates edge because it uses the most favorable number, not what the market actually prices the event at. Consensus median is the correct reference for "what does the market believe?" — separating that from "where do I get the best price?" is the core correctness fix.

- `consensus_confidence` = "high" when ≥4 books and stddev ≤ 0.5; "low" when only 1-2 books or stddev > 1.5.
