## Per-Case Assertion Contract

Every scenario must assert all of:

```
canonical_starter_state     ('CONFIRMED'|'EXPECTED'|'UNKNOWN'|'CONFLICTING')
evidence_flags              (string[] — specific flags expected)
adjustment_trust            ('FULL'|'DEGRADED'|'NEUTRALIZED'|'BLOCKED')
model_descriptive_output    (present or absent)
official_eligible           (true or false)
consistency_vol_env         ('VOLATILE'|'STABLE')
consistency_total_bias      ('OK'|'INSUFFICIENT_DATA')
watchdog_reason_codes       (string[] — exact codes expected or empty)
wrapper_action              ('FIRE'|'HOLD'|'PASS')
```

## Failure Modes & Guards

### Failure mode: harness exists but is not actually blocking downstream WIs
**Guard:** WI-0381, WI-0382, and WI-0383 each carry `Blocked by: WI-0384` in their dependency field. PRs for those WIs must reference WI-0384.

### Failure mode: stubs mask real failures until never
**Guard:** Stubs must be annotated `// TODO: replace stub with real implementation after WI-####`. Any stub that remains after its target WI merges is a test failure.

### Failure mode: cases only validate model math, not end-to-end semantics
**Guard:** Every case asserts `wrapper_action` and `watchdog_reason_codes`, not just `model_descriptive_output`.
