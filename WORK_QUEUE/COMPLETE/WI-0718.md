# WI-0718 — Freeze Cross-Sport Audit Contract

**ID:** WI-0718
**Goal:** Create the written contract for what gets audited, at which pipeline stages, and which fields are strict vs tolerant. Without this, every drift argument becomes folklore.
**Owner agent:** agent/copilot
**Time window:** one session
**Coordination flag:** none — docs only, no shared touchpoints

## Implementation Plan

1. **Define five audit stages** in `model_audit_framework.md`:
   - `INPUT` — raw odds snapshot as loaded from DB
   - `ENRICHED_INPUT` — after ESPN enrichment and normalization
   - `MODEL_OUTPUT` — after driver cards / market decisions computed
   - `DECISION_OUTPUT` — after `publishDecisionForCard` runs
   - `PUBLISH_OUTPUT` — final emitted card payload written to DB

2. **For each sport (NBA, NHL, MLB)** document in `model_audit_framework.md`:
   - Required inputs
   - Optional inputs
   - Canonical derived outputs
   - Blocking rules (when pipeline must stop)
   - Publish/no-publish rules

3. **Define comparison classes** in `model_audit_framework.md`:
   - **strict**: labels (`classification`, `execution_status`), reason codes, gating flags, `official_status`, card type
   - **tolerant**: probabilities, projections, totals, fair prices (tolerance map per market family — not global)
   - **ignored**: timestamps, UUIDs, log-only metadata, `generated_at`, `run_id`

4. **Define canonical drift categories** in `model_audit_reason_codes.md`:
   - `SPEC_DRIFT` — field shape or type changed unexpectedly
   - `INPUT_SHAPE_DRIFT` — enrichment changed a required field
   - `MODEL_DRIFT` — model output changed vs baseline
   - `DECISION_DRIFT` — decision layer changed classification vs baseline
   - `PUBLISH_DRIFT` — final card payload diverged from post-decision snapshot
   - `PERFORMANCE_DRIFT` — outcome quality metrics degraded over rolling window

5. **Add explicit non-negotiable rules** in `model_audit_framework.md`:
   - No code may mutate `classification`, `action`, or `status` after `publishDecisionForCard` returns
   - `pricing_ready=false` implies `execution_status !== EXECUTABLE`
   - Projection-floor outputs must carry `execution_status=PROJECTION_ONLY`; they must never carry `execution_status=EXECUTABLE`
   - `watchdog` and `consistency` fields must be present before publishable classification proceeds
   - No workflow may surface a PLAY when no qualified edge exists

6. **Add one concrete example per sport** in `model_audit_baselines.md` showing all five stages with actual field shapes (can use sanitized/synthetic values — no live data).

7. **Add one negative example** in `model_audit_baselines.md` showing an illegal post-decision classification rewrite and marking it `PUBLISH_DRIFT / SPEC_DRIFT`.
