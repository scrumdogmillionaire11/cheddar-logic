# WI-0874: POTD MLB Integration -- Wire MLB Model Edge into POTD Candidate Scoring

**ID**: WI-0874
**Goal**: Make POTD candidate scoring for MLB games use model-derived win probability and edge from computeMLBDriverCards(), so MLB plays compete on model signal rather than consensus-odds-only scoring.

**Scope**:
- apps/worker/src/jobs/potd/signal-engine.js (scoreCandidate: add MLB model override path)
- apps/worker/src/models/mlb-model.js (expose resolveMLBModelSignal helper)
- apps/worker/src/jobs/potd/run_potd_engine.js (verify MLB games flow through gatherBestCandidate unchanged; no structural change)

**Out of scope**: POTD bankroll/ledger, Discord formatting, NBA/NHL signal changes, new DB tables

**Background**:
POTD scores all candidates via consensus odds only (scoreCandidate fair-pair de-vig).
For MLB the worker has a full model (F5 total, F5 ML, full-game total/ML via WI-0872/0873)
but POTD never calls it. Result: MLB plays fire on line value vs consensus, ignoring
pitcher quality, park, weather, and offense. data/mlb-research.md is explicit: the
projection layer must feed into the execution gate, not just be logged.

**Implementation notes**:
resolveMLBModelSignal(game): in mlb-model.js
  - Calls computeMLBDriverCards(game.gameId, game.oddsSnapshot)
  - Picks best FIRE driver card; returns { modelWinProb, edge, projection_source } or null
  - null when: no pitcher data, NO_BET, SYNTHETIC_FALLBACK, or POTD MLB feature disabled

In scoreCandidate(): signal-engine.js
  - If game.sport === MLB and resolveMLBModelSignal returns non-null:
    replace modelFairProbability with modelWinProb from driver card
    replace edgePct with model edge
  - Else: current consensus path unchanged (backward compatible)

**Acceptance**:
1. ENABLE_MLB_MODEL=true + valid pitcher data: POTD candidate has modelWinProb from computeMLBDriverCards
2. Missing pitcher data: falls back to consensus fair-pair; no crash; no silent wrong signal
3. Test: mock game with elite F5 edge passes through scoreCandidate with correct edgePct from model
4. Test: mock game with no MLB model data produces same scoreCandidate result as today
5. POTD dry-run with MLB fixture selects MLB play when MLB edge is highest among active sports
6. npm --prefix apps/worker test --no-coverage passes, zero regressions
