# WI-0761: Model Health Dashboard ‚Äî replace Admin page stub

| Field              | Value |
|--------------------|-------|
| **ID**             | WI-0761 |
| **Goal**           | Replace the non-functional `/admin` stub (fake auth form, 4 disabled placeholder cards) with a live **Model Health Dashboard** that surfaces `pipeline_health` and `model_outputs` data. This makes health check results observable without querying the DB directly. |
| **Priority**       | Medium (ops visibility ‚Äî `pipeline_health` table is a silent dead sink; `/admin` page is fully non-functional) |
| **Owner agent**    | Unassigned |
| **Time window**    | TBD |
| **Coordination flag** | needs-sync (`packages/data/src/db/index.js`, `web/src/app/api/`) |
| **Depends on**     | WI-0760 ‚úì COMPLETE (`/api/model-outputs` route exists, commit `a5b3bd1`) |

---

## Problem

`check_pipeline_health.js` runs on schedule and writes rows to `pipeline_health`. Zero consumers exist ‚Äî no `FROM pipeline_health` anywhere in `web/src/`, `apps/`, or `packages/`. Health check results are computed and stored but never seen.

The `/admin` page (`web/src/app/admin/page.tsx`, 105 lines) is entirely non-functional: a fake auth form (no API call), 4 `opacity-50` placeholder cards, no data fetching. It renders nothing useful.

WI-0760 (complete) wired `model_outputs` to `GET /api/model-outputs`. That surface point now needs to appear somewhere in the UI.

---

## Decision

**Option A ‚Äî Web route + replace Admin page** ‚Üê chosen.

- Add `GET /api/admin/pipeline-health` route
- Add `getPipelineHealth(limit)` getter in `packages/data/src/db/`
- Gut and rewrite `web/src/app/admin/page.tsx` as a live Model Health Dashboard showing pipeline health status + recent model outputs
- Discord emit deferred to a future WI
