# WI-0555: Unify spread threshold and enable MARKET_THRESHOLDS_V2

**ID**: WI-0555
**Goal**: Eliminate the dual-threshold conflict where NBA spread call cards pass a 2% gate in run_nba_model.js but are subsequently blocked by the 3.5% pipeline-v2 gate, and enable ENABLE_MARKET_THRESHOLDS_V2 in production.

**Scope**:
- `apps/worker/src/jobs/run_nba_model.js`
- `packages/models/src/decision-pipeline-v2.patch.js`
- `packages/models/src/flags.js`
- `.env` / `env.example` (if ENABLE_MARKET_THRESHOLDS_V2 is set there)

**Out of scope**:
- Recalibrating the threshold values themselves (0.035 for lean, 0.07 for play) — accept current v2 values as-is
- Other sports' threshold profiles
- Any UI or results page changes

**Background**:
`run_nba_model.js` defines a local `const SPREAD_EDGE_MIN = 0.02` (line ~497) used to gate which spread call cards are generated. The `decision-pipeline-v2.patch.js` profile for `NBA:SPREAD` uses `lean_edge_min: 0.035`. A spread card with 2.5% edge passes the call-card generator gate, gets written to the DB, but is then blocked by the downstream pipeline-v2 decision layer. This produces published call cards that silently never become actionable plays. `ENABLE_MARKET_THRESHOLDS_V2` is defined in flags.js but is off by default, meaning the stricter per-market thresholds aren't active in production.

**Acceptance**:
- The local `SPREAD_EDGE_MIN = 0.02` constant is removed from `run_nba_model.js`.
- The call card spread gate in `generateNBAMarketCallCards` uses the canonical threshold from `resolveThresholdProfile({ sport: 'NBA', marketType: 'SPREAD' })`.
- `ENABLE_MARKET_THRESHOLDS_V2` defaults to `true` in `flags.js` (or is set in env.example with documentation).
- After the change, no spread call card is generated with edge below `lean_edge_min` from the NBA:SPREAD profile.
- Existing tests pass. Add a test asserting that a spread card at 2.5% edge does NOT appear in call card output for NBA.
