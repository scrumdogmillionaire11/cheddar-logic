# WI-0979: Add migration safety contract to deployment workflow

**ID**: WI-0979
**Goal**: Enforce forward-compatible migration checks so deploys do not leave production in a partial or incompatible state.

**Depends on**: WI-0974

**Scope**:
- `.github/workflows/deploy-production.yml`
- `scripts/run-migrations.js`
- `scripts/migrate-dev-to-prod-safe.sh`
- `docs/runbooks/migration-safety.md`

**Out of scope**:
- Net-new schema redesign
- Data model feature additions

**Acceptance**:
- Deployment performs migration preflight and post-deploy migration status verification.
- Workflow fails if migration status is incompatible with target release.
- Runbook documents rollback-safe migration requirements.

**Time window**: 2026-04-16 -> TBD
**Coordination flag**: needs-sync

**Tests to run**:
- `npm --prefix web run lint`
- `npm --prefix web run test:ci:smoke`

**Manual validation**:
- Run migration preflight in a staging-like environment and verify pass/fail behavior.
- Confirm deployment halts when migration verification reports mismatch.
