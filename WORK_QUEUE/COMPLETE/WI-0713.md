# WI-0713: Fix getActiveSports() season date logic — MLB incorrectly active in offseason

**ID**: WI-0713
**Goal**: Fix the OR season-date check in `getActiveSports()` so MLB is not marked active in January–February when it is out of season.

**Problem:**
`packages/odds/src/config.js#getActiveSports()` uses OR to test whether today is in-season:

```js
const inSeason = mmdd >= cfg.season.start || mmdd <= cfg.season.end;
```

This OR is correct for year-spanning seasons (NFL: Sep–Feb, `start='09-01' > end='02-15'`)
because those seasons cross the January boundary. But for seasons that stay within one
calendar year (MLB: Mar–Nov, `start='03-20' < end='11-01'`) the OR incorrectly fires
during the off-season months of January and February:

| Date    | `mmdd >= '03-20'` | `mmdd <= '11-01'` | OR result | Expected |
|---------|-------------------|-------------------|-----------|----------|
| Jan 15  | false             | true              | **true**  | false    |
| Feb 20  | false             | true              | **true**  | false    |
| Apr 1   | true              | true              | true      | true ✓   |
| Nov 15  | true              | false             | true      | false ✗  |

This means MLB would be fetched (and tokens spent) for the entire January–February offseason,
and also in November after the World Series. No current production impact (April, MLB is
in season) but will waste tokens when the season ends.

**Root cause:**
The OR logic is intentional for wrap-around seasons (NFL) but wrong for within-year seasons.
The fix is to detect the wrap-around case (`start > end`) and apply OR only then, using AND
for within-year seasons.

**Proposed fix** (`packages/odds/src/config.js`):
```js
const wrapsYear = cfg.season.start > cfg.season.end; // e.g. NFL: '09-01' > '02-15'
const inSeason = wrapsYear
  ? (mmdd >= cfg.season.start || mmdd <= cfg.season.end)  // straddles Jan 1
  : (mmdd >= cfg.season.start && mmdd <= cfg.season.end); // stays within year
```

Apply the same fix to `isInSeason()` which duplicates the same logic (lines 148–156).

**Scope**:
- `packages/odds/src/config.js` — `getActiveSports()` and `isInSeason()`

**Out of scope**:
- `SPORTS_CONFIG` active flags — those are separate opt-in gates
- Any scheduler or job logic that reads the result of `getActiveSports()`

**Acceptance**:
- `getActiveSports()` returns MLB for April, October (in-season) but NOT January, February, December (off-season)
- `getActiveSports()` returns NHL for January (in-season) and July (off-season correctly excluded)
- `getActiveSports()` returns NFL for January (in-season, wraps year) and April (off-season)
- Unit tests cover the boundary dates for all four sports
