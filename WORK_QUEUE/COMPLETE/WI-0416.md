**Acceptance**:
- NHL full-game totals with `kind=PLAY` enroll into `card_display_log` even when `decision_v2.official_status=PASS`.
- NHL 1P totals enroll only when actionable (`PLAY`/`LEAN` via official status + legacy fallback).
- NHL moneyline rows enroll only when actionable (`PLAY`/`LEAN` via official status + legacy fallback).
- `deriveLockedMarketContext()` does not short-circuit only because `recommendation.type=PASS` when payload is explicitly actionable with market fields.
- 1P lock derivation supports `odds_context.total_price_over_1p` / `odds_context.total_price_under_1p` when period is `1P`.
- `applyNhlSettlementMarketContext()` emits `kind=PLAY` for `nhl-pace-1p` only when both side and 1P price are present.
- `settle_pending_cards` telemetry includes `NHL_MONEYLINE` bucket and backfill enrollment logic aligns with the same NHL enrollment rules.
