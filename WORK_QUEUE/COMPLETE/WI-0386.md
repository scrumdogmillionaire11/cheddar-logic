# WI-0386: Daily Settlement for NBA Totals + NHL Full/1P Totals

**ID**: WI-0386
**Goal**: Ensure NBA full-game totals and NHL full-game + 1P totals are tracked and settled every day with correct market-specific grading logic.

**Scope**:
- `apps/worker/src/jobs/settle_game_results.js`
- `apps/worker/src/jobs/settle_pending_cards.js`
- `apps/worker/src/jobs/run_nba_model.js`
- `apps/worker/src/jobs/run_nhl_model.js`
- `packages/data/src/db.js`
- `packages/data/src/market-contract.js`
- `apps/worker/src/__tests__/settlement-pipeline-integration.test.js`
- `apps/worker/src/__tests__/settle-coverage.test.js`
- `WORK_QUEUE/WI-0386.md`

**Out of scope**:
- Regrading historical seasons beyond a bounded backfill window explicitly defined during implementation
- Changes to odds providers or scoreboard providers
- UI redesign beyond surfacing existing settled status fields

**Acceptance**:
- NBA totals cards are persisted with market metadata sufficient for deterministic settlement (`market_type=TOTAL`, side, line, price, source).
- NHL full-game totals cards are persisted with deterministic settlement metadata.
- NHL 1P totals cards are persisted with deterministic settlement metadata that distinguishes 1P grading from full-game totals.
- Settlement pipeline grades NBA full totals using final game score only.
- Settlement pipeline grades NHL full totals using final game score only.
- Settlement pipeline grades NHL 1P totals using first-period score only.
- Daily settlement run settles all eligible NBA/NHL totals cards and writes `card_results` rows with `WON`/`LOST`/`PUSH`/`VOID` where appropriate.
- Settlement tracking includes per-market daily counts (pending/settled/failed) for: `NBA_TOTAL`, `NHL_TOTAL`, `NHL_1P_TOTAL`.
- Integration tests cover at least: NBA full total over/under, NHL full total over/under, NHL 1P over/under, push cases, and missing-score guardrails.
