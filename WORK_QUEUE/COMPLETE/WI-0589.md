# WI-0589: Confidence tier correction layer for PLAY vs LEAN

**ID**: WI-0589
**Goal**: Redefine `PLAY` vs `LEAN` so `PLAY` is rarer, cleaner, and measurably stronger than `LEAN`, with explicit cleanliness/contradiction gates and a recent-sample regrade report.

**Out of scope**:

**Background**:
Recent production performance shows `PLAY` underperforming `SLIGHT EDGE`, which means the system is mis-ranking confidence. The correction needed here is structural: `PLAY` should require both stronger edge and cleaner conditions than `LEAN`. Missing inputs, stale market data, proxy dependence, or contradictory drivers should block `PLAY` even if raw edge clears a threshold.

**Closeout note (2026-03-27):** Implementation and regression coverage are complete, and the new `decision_tier_audit` report section is live. The current 14-day production-style sample still shows `PLAY` underperforming `LEAN`, so the original operational target in Acceptance is not yet satisfied by available live evidence.

**Terminology note**: The internal tier names `PLAY` and `LEAN` map to front-end labels **"Play"** and **"Slight Edge"** respectively. This work changes internal tier assignment logic only â€” the display labels and their mapping to internal names are frozen and must not be touched.

**Acceptance**:
- `PLAY` requirements are stricter than `LEAN` on both edge and support/cleanliness criteria.
- A row with missing inputs, stale market data, proxy usage, or meaningful driver contradiction cannot resolve to `PLAY`.
- `report_telemetry_calibration` (or equivalent report output in scope) includes a decision-tier section summarizing recent settled performance for `PLAY` vs `LEAN`.
- The regraded recent sample shows fewer `PLAY` decisions than the current ruleset.
- The regraded recent sample satisfies the operational target: `PLAY` win rate > `LEAN` win rate and `PLAY` ROI > `LEAN` ROI, or the WI remains open until the rule set is tightened enough to achieve that.
