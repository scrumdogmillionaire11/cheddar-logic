# WI-0918: Unify Card-Type Contracts and MLB Market Coverage in Web Pipeline

**Goal**: Eliminate contract drift between games route and transform classification by centralizing sport card-type contracts and adding full MLB market coverage for newly emitted game cards.

**Depends on**: none

- One canonical sport card-type contract definition is used by both `/api/games` routing and game-card transform classification.
- MLB contract includes `mlb-full-game` and `mlb-full-game-ml` as play-producer card types where applicable.
- MLB expected playable markets include FULL_GAME pathways in addition to existing first-5/prop pathways.
- Existing NBA/NHL behavior remains unchanged for current card types.
