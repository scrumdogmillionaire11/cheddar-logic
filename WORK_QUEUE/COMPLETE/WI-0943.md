# WI-0943: Fix EVIDENCE Cards Bleeding into Discord Play/Lean Sections

**ID**: WI-0943
**Goal**: Stop `kind=EVIDENCE` driver cards (nhl-base-projection, nhl-goalie-certainty, nhl-rest-advantage, nhl-model-output) from appearing as independent play or lean rows in the Discord snapshot. The play call and its context drivers must be separated: only cards with `kind=PLAY` and a real selection+price may populate the `ЁЯЯв PLAY` or `ЁЯЯб Slight Edge (Lean)` sections.

**Depends on**: none

**Root cause (confirmed via DB inspection 2026-04-14)**:

- EVIDENCE cards are written with `webhook_eligible=1` and `webhook_bucket=official` (or `lean`) by the NHL model writer even when `kind=EVIDENCE`.
- `classifyDecisionBucketLegacy()` in `post_discord_cards.js` does not gate on `kind`: it routes cards by `action`/`classification` alone, so a FIRE-action EVIDENCE card lands in `official`.
- Result: WSH@CBJ shows `ЁЯЯв PLAY` (nhl-model-output, EVIDENCE, NEUTRAL, no real bet) and `ЁЯЯб Slight Edge (Lean)` (nhl-base-projection AWAY, nhl-goalie-certainty NEUTRAL, nhl-rest-advantage NEUTRAL) тАФ none of which are placed bets. The actual bet calls (nhl-moneyline-call, nhl-totals-call, nhl-pace-totals) are all correctly in `pass_blocked`.

**Scope**:

- `apps/worker/src/jobs/post_discord_cards.js`
- `apps/worker/src/jobs/__tests__/post_discord_cards.test.js`
- `apps/worker/src/jobs/run_nhl_model.js` тАФ stop setting `webhook_eligible=true` / `webhook_bucket` on EVIDENCE rows

**Out of scope**:

- Model threshold or edge formula changes
- NBA or MLB card writers
- Web UI or API contract changes
- Changing what constitutes a PLAY vs LEAN decision (policy is unchanged; only surfacing discipline)

**Changes required**:

1. **`classifyDecisionBucketLegacy()` guard**: if `payload.kind === 'EVIDENCE'`, return `'pass_blocked'` (suppressed) immediately тАФ EVIDENCE cards are never standalone bet rows.
2. **`isDisplayableWebhookCard()` guard**: EVIDENCE cards with no real selection/price should return `false` from the webhook eligibility check.
3. **NHL model writer (`run_nhl_model.js`)**: when writing EVIDENCE-kind card payloads, set `webhook_eligible: false` and omit `webhook_bucket` so the explicit flags don't override the formatter gate.
4. **Tests**: add a case where an EVIDENCE card with `action=FIRE` is present alongside a real `kind=PLAY` pass-blocked card for the same game тАФ confirm the EVIDENCE card does not appear in any rendered section, and the game is either shown only if a real bet row qualifies or suppressed entirely.

**Acceptance**:

- A game with only `kind=EVIDENCE` cards marked `action=FIRE` produces **no rendered Discord message** for that game.
- A game with one real `kind=PLAY` FIRE card and multiple `kind=EVIDENCE` FIRE cards renders only the PLAY card under `ЁЯЯв PLAY`; the EVIDENCE cards do not appear as separate rows.
- A game with all bet calls in `pass_blocked` and only EVIDENCE cards remaining does not emit a spurious `ЁЯЯв PLAY` or `ЁЯЯб Slight Edge (Lean)` section.
- `resolveLeanSectionTitle()` / `sectionLines()` output for the WSH@CBJ pattern (all bet calls PASS, only EVIDENCE cards FIRE) is empty / game skipped.
- Worker tests pass: `npm --prefix apps/worker run test -- --runInBand src/jobs/__tests__/post_discord_cards.test.js`
- Full worker test suite passes: `npm --prefix apps/worker run test`
