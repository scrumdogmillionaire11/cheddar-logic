# WI-0853: MLB F5 Market Token — FIRST_PERIOD → FIRST_5_INNINGS

| Field | Value |
|---|---|
| **ID** | WI-0853 |
| **Goal** | Replace the semantically incorrect `FIRST_PERIOD` token for MLB F5 (first 5 innings) cards with the unambiguous `FIRST_5_INNINGS` token across the full worker-to-web stack. Maintain backward compatibility for existing DB rows that carry the legacy token. |
| **Scope** | `apps/worker/src/jobs/run_mlb_model.js`, `apps/worker/src/calibration/calibration-gate.js`, `apps/worker/src/jobs/settle_pending_cards.js`, `apps/worker/src/jobs/backfill_period_token.js`, `apps/worker/src/audit/audit_rules_config.js`, `apps/worker/src/audit/performance_drift_report.js`, `apps/worker/src/audit/projection_evaluator.js`, `web/src/lib/types/game-card.ts`, `web/src/lib/games/market-inference.ts`, `web/src/lib/games/normalizers.ts`, `web/src/lib/games/route-handler.ts` |
| **Out of scope** | NHL model files (`run_nhl_model.js`, `nhl-player-shots.js`, `models/index.js NHL paths`), DB migrations, any worker write path changes beyond token enum expansion |
| **Acceptance** | (1) `run_mlb_model.js` emits `market_type='FIRST_5_INNINGS'` for all F5 cards. (2) All consumer functions (calibration, settlement, audit, web normalizers) accept both `FIRST_5_INNINGS` and legacy `FIRST_PERIOD`. (3) NHL cards continue to use `FIRST_PERIOD` unchanged. (4) Web `CanonicalMarketType` includes `FIRST_5_INNINGS`; `inferMarketFromCardType('mlb-f5')` returns `FIRST_5_INNINGS`. (5) `/api/games` diagnostic shows `MLB: ['FIRST_5_INNINGS']` for F5 slots, not `FIRST_PERIOD`. (6) Legacy DB rows with `FIRST_PERIOD` from MLB games are remapped at the web read layer. (7) Worker test suite 0 new failures; web `npx tsc --noEmit` exits 0. |
