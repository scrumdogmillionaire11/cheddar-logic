# WI-0862: Fix `parseSnapshotPayload` shape mismatch — scanner sees 0 books for all prod snapshots

| Field | Value |
|---|---|
| **ID** | WI-0862 |
| **Goal** | `scanLineDiscrepancies` and `scanOddsDiscrepancies` always return `[]` on production data because `parseSnapshotPayload` looks for `payload.markets` but the DB stores `normalized.market` directly as the raw blob (i.e., `{ spreads, totals, h2h }` at the top level, no `markets` wrapper). Fix the fallback in `parseSnapshotPayload` and add a regression test using the stored shape. |
| **Priority** | High |
| **Depends on** | WI-0855 (scanner refactor) — COMPLETE |
| **Coordination flag** | none |
| **Owner agent** | unassigned |
| **Time window** | 1 session |

---

## Background

`pull_odds_hourly.js` stores `rawData: normalized.market` (line 514), which produces a blob shaped:
```json
{ "spreads": [...], "totals": [...], "h2h": [...] }
```

`parseSnapshotPayload` (mispricing-scanner.js line 325) reads:
```js
markets: payload && payload.markets ? payload.markets : {}
```

`payload.markets` is always `undefined` on prod data → scanner always returns `{}` → 0 books → 0 discrepancies for every scan.

Confirmed via live prod run: 180 snapshots (90 games), 0 games with ≥ 3 books, 0 gaps found.

---

## Scope

| File | Change |
|---|---|
| `packages/models/src/mispricing-scanner.js` | Fix `parseSnapshotPayload` fallback: when `payload.markets` is absent, fall back to using the payload itself as the markets object when it has `spreads`, `totals`, or `h2h` keys |
| `packages/models/src/__tests__/mispricing-scanner.test.js` | Add regression test: pass snapshot in prod-stored shape `{ spreads, totals, h2h }` (no `markets` wrapper) → verify scanner finds expected discrepancy |

**Out of scope:** Changing the storage shape in `pull_odds_hourly.js` or `normalize.js`. The scanner must handle both shapes for backward compatibility.

---

## Acceptance Criteria

1. `parseSnapshotPayload` returns the correct `markets` object when `raw_data` is `{ spreads, totals, h2h }` (no wrapper).
2. `parseSnapshotPayload` still works when `raw_data` is `{ markets: { spreads, totals, h2h } }` (with wrapper, as in existing unit tests).
3. Regression test passes: snapshot with `raw_data = { spreads: [...with 3 books...] }` → `scanLineDiscrepancies` returns at least 1 result.
4. All existing mispricing-scanner tests still pass (42 assertions).
5. Live prod run returns results: running the one-liner against prod DB finds LineGaps or OddsGaps.
