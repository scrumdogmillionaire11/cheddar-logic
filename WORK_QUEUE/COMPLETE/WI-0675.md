# WI-0675: Normalize chip analyzer output to ChipRecommendation literals

**ID**: WI-0675
**Goal**: Ensure `ChipAnalyzer.analyze_chip_decision()` emits full chip names accepted by `ChipRecommendation` instead of short enum codes like `FH`.

**Background**:
After fixing `WI-0674`, the full `cheddar-fpl-sage` suite exposed `tests/tests_new/test_chip_window_edge_cases.py::TestEmptyChipWindows::test_empty_windows_returns_recommendation`. `analyze_chip_decision()` currently uses `guidance.selected_chip.value`, but the `ChipType` enum stores abbreviations (`WC`, `FH`, `BB`, `TC`) while the Pydantic `ChipRecommendation.chip` field only accepts full labels (`Wildcard`, `Free Hit`, `Bench Boost`, `Triple Captain`, `None`).
