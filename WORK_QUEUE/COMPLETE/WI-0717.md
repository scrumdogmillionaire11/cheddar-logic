**Acceptance**:
- MLB prop odds pulls are triggered only by `run_mlb_prop_pipeline`, never directly from per-game scheduler fanout
- Candidate generation uses non-odds MLB inputs only and emits market-aware `pitcher_strikeouts` `UNDER` candidates with `priority_score`, `confidence`, and `reason_codes`
- Empty candidate sets result in zero MLB prop odds API calls
- Budget controller enforces per-slate, per-game, daily, dedupe, and burn-rate caps with structured skip reasons
- Scoped event mapping is stored persistently; stale/missing mappings skip without any full-slate fallback
- `run_mlb_model` supports `gameIds` narrowing and does not publish executable pitcher-K cards when scoped odds are stale or absent
- Top-level invariant tests prove that no prop execution path can trigger a full-slate odds request
