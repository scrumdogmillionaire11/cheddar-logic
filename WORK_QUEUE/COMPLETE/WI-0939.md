# WI-0939: Restore MLB Full-Game Surfaces and Retire Legacy Decision Debt

**ID**: WI-0939
**Goal**: Fix audited MLB non-surfacing market gaps by restoring full-game total emission, improving full-game moneyline surfacing reliability, and removing legacy or redundant decision paths that create silent drops, contradictory states, or maintenance debt.

**Depends on**: none

**Architecture constraints (must hold)**:

- ADR-0004 principles govern decision ownership for this WI: worker remains decision authority; downstream layers consume canonical decision payloads and must not recompute verdicts.
- ADR-0013 policy applies for MLB game-lines: `mlb-full-game` and `mlb-full-game-ml` are active odds-backed lanes; F5 and pitcher-K remain separate lanes.
- DATA_CONTRACTS invariants apply: one candidate, one deterministic terminal reason family; no silent drops between decision and persisted card payload.

**Scope**:

- `apps/worker/src/jobs/run_mlb_model.js`
- `apps/worker/src/models/mlb-model.js`
- `apps/worker/src/jobs/check_pipeline_health.js`
- `apps/worker/src/jobs/__tests__/run_mlb_model.test.js`

- `apps/worker/src/__tests__/check-pipeline-health.mlb.test.js`
- `docs/audits/mlb-market-surface-audit.md`

**Out of scope**:

- Rewriting core MLB projection formulas
- Retuning NBA totals quarantine policy (`QUARANTINE_NBA_TOTAL`)
- Redesigning web cards UX defaults (FIRE/WATCH vs PASS toggle behavior)

**Enumerated tech debt to address in this WI**:

- TD-01: Full-game total key drift (`total_line` hydration vs `full_game_line` consumption) causing suppressed `full_game_total` generation.
- TD-02: Legacy selector migration residue from old winner-take-all flow (comments/helpers/aliases) not aligned with independent market evaluation path.
- TD-03: Multiple status surfaces (`status`, `action`, `classification`, `decision_v2.official_status`) with potential divergence in MLB game-market writes.
- TD-04: Diagnostics gap where blocked MLB candidates are not consistently grouped into deterministic reason families in health output.
- TD-05: Stale documentation/comments describing deprecated behavior after market-eval migration.

**Acceptance**:

- MLB full-game totals can be emitted when full-game total odds exist.
- Full-game total field contract mismatch is removed with one canonical key path from worker hydration to model consumption.
- End-to-end tests prove at least one `full_game_total` path inserts `mlb-full-game` when total line is present and edge passes.
- MLB full-game ML no longer silently disappears under stale/edge gates: each rejected candidate is recorded with explicit reason codes and counted in diagnostics.
- `check_pipeline_health` (or equivalent pipeline audit output) reports per-market reject counts for MLB `f5_total`, `full_game_total`, and `full_game_ml` with deterministic reason-family buckets.
- No behavioral regression is introduced for `mlb-f5`, `mlb-f5-ml`, and `mlb-pitcher-k` publication paths.
- Tech-debt closeout is objective and complete:
  - TD-01: canonical key contract test added and passing.
  - TD-02: explicit list of removed legacy helper(s)/branch(es)/alias(es) recorded in audit doc; each removed symbol has an `rg` absence proof line and at least one fixture-based regression test proving the independent market path still emits expected lanes.
  - TD-03: parity assertions prove persisted terminal status fields are internally consistent for MLB game-market rows.
  - TD-04: health diagnostics include non-zero/zero counters for each reason family and no uncategorized blocker bucket.
  - TD-05: outdated comments/runbook statements removed or rewritten; debt ledger includes retained intentional debt with rationale and owner.
