# WI-0617: Remove initDb() no-op callers across codebase

**ID**: WI-0617
**Status**: COMPLETE
**Goal**: Find and remove all call sites of `initDb()` across the codebase, then remove the no-op export from `db.js`.

**Background**:
`packages/data/src/db.js` exports `initDb()` as an async no-op preserved "for caller back-compat." Every caller invokes an unnecessary async call; the back-compat promise has no expiry, so removal never happens organically. The fix is to grep all callers, remove the calls, then remove the export.

**Scope**:
- `packages/data/src/db.js` — remove `initDb` export (lines 466–471, 4170)
- All callers found via `grep -rn "initDb(" --include="*.js" --include="*.ts"` — remove the call site

**Out of scope**:
- Any other DB initialization logic

**Acceptance**:
- `initDb` does not appear in `db.js` or any caller file.
- All worker job entry points still start correctly (spot-check: `node apps/worker/src/jobs/pull_odds_hourly.js --dry-run`).
