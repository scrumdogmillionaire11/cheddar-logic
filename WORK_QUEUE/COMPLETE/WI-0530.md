# WI-0530: NST PP shot rate ingestion + wire into projectSogV2

**ID**: WI-0530
**Goal**: Pull season-level PP shots + PP TOI from Natural Stat Trick, compute `pp_rate_per60`, and wire it into `projectSogV2` so PP-heavy players project correctly.

**Owner agent**: quick-55
**Time window**: 2026-03-20
**Coordination flag**: solo

---

## Context

`toi_proj_pp` now flows correctly (WI-0528), but `pp_rate` in `projectSogV2` is still 0 because there is no source for PP shots per 60. The NHL Stats API does not cleanly expose PP-specific shot counts at the player level — they must be derived from an external source.

Natural Stat Trick (NST) provides individual PP shots, PP TOI, and shot rates by situation. Season-level data is downloadable as CSV from their skater report (individual stats, strength = PP, current season).

Current state:

```
PP contribution = pp_rate * toi_proj_pp / 60
                = 0 * toi_proj_pp / 60
                = 0  ❌
```

After this WI:

```
PP contribution = pp_rate_per60 * toi_proj_pp / 60
                = (real value) * toi_proj_pp / 60  ✅
```
