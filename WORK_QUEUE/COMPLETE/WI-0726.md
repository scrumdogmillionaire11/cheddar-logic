## Implementation Plan

### 1. Scripts and artifact layout

Add to `apps/worker/package.json`:
```json
{
  "audit:nba": "node src/audit/run_model_audit.js --sport NBA",
  "audit:nhl": "node src/audit/run_model_audit.js --sport NHL",
  "audit:mlb": "node src/audit/run_model_audit.js --sport MLB",
  "audit:all": "node src/audit/run_model_audit.js --all",
  "audit:performance": "node src/audit/performance_drift_report.js --all",
  "audit:scorecard": "node src/audit/scorecard.js --all"
}
```

All workflow outputs are written under `apps/worker/audit-output/<run-scope>/`.

### 2. `run_model_audit.js`

Promote the runner into the CI gate source of truth:
- emit a machine-readable summary JSON with `fixture_count`, `passed`, `failed`, `critical_count`, `high_severity_count`, `warn_count`, failing fixtures, drift categories, and exact field diffs
- fail on any `CRITICAL` invariant breach, any `HIGH` audit drift, empty fixture set, expired `_baseline_change_note`, or touched fixture still using `RECOMPUTE_ON_FIRST_RUN`
- grandfather existing untouched bootstrap fixtures on push/schedule runs
- print a condensed summary suitable for CI log output

### 3. `fixture_loader.js`

Add `_baseline_change_note` validation:
```json
"_baseline_change_note": {
  "changed_by": "WI-0730",
  "reason": "NBA sigma recalibrated after 2026 playoff data integrated",
  "approved_at": "2026-04-15",
  "expires_after_runs": 3
}
```

Rules:
- `changed_by`, `reason`, `approved_at` required
- `expires_after_runs` defaults to `3` for newly supplied notes
- malformed notes fail validation
- expiry is interpreted as weekly audit cadence, not literal rerun count

### 4. `scorecard.js`

Combine audit output + performance drift output into governance artifacts:
- `scorecard.json`
- `audit-scorecard.md`

Per `sport + card_family`, include:
- `risk`
- `reasons`
- `trend`
- `performance_alerts`
- optional `model_decay=true`

Risk mapping:
- `LOW` = no HIGH/CRITICAL audit drift and no performance alerts
- `MEDIUM` = WARN-only audit drift or WARN performance alert
- `HIGH` = any HIGH audit drift, HIGH performance alert, or repeated alert pattern
- `CRITICAL` = invariant breach / CRITICAL audit breach or CRITICAL sustained performance alert

Model decay rule:
- if performance alerts exist for a family and audit shows no HIGH/CRITICAL drift, mark `model_decay=true`

### 5. `.github/workflows/audit.yml`

Create a dedicated workflow aligned with current repo standards:
- `actions/checkout@v5`
- `actions/setup-node@v5`
- Node `24`
- PR, push to `main`, and weekly schedule triggers
- artifact name `audit-scorecard-${{ github.run_id }}`
- CI logs must print the condensed audit diff summary before exiting

- `npm --prefix apps/worker run audit:nba` runs and exits 0 on clean fixtures

- CI log output contains failing fixture, drift category, and exact field diff when the audit gate fails
- Touched fixtures using `RECOMPUTE_ON_FIRST_RUN` fail the gate; untouched historical fixtures do not fail push/schedule runs solely for that state
- Worker tests pass
