# WI-0679: Honor chip-specific policy windows in chip analyzer selection

**ID**: WI-0679
**Goal**: Ensure `ChipAnalyzer.analyze_chip_guidance()` only evaluates chips allowed by `chip_policy["chip_windows"]` when explicit chip names are configured, so recommendations respect the active or future policy window instead of selecting an unrelated chip.

**Background**:
After fixing `WI-0678`, the full `cheddar-fpl-sage` suite exposed `tests/tests_new/test_stabilization_integration.py::TestChipAnalyzerIntegration::test_chip_recommendation_with_valid_window`. The analyzer currently evaluates every available chip whenever any windows exist, even if the policy window explicitly targets `Bench Boost`; this allows `Free Hit` to win selection despite the configured window.

**Scope**:
- `cheddar-fpl-sage/src/cheddar_fpl_sage/analysis/decision_framework/chip_analyzer.py`
- `cheddar-fpl-sage/tests/tests_new/test_stabilization_integration.py`

**Out of scope**:
- Deterministic chip-engine evaluator formulas
- Chip recommendation serialization contracts
- Non-chip-analyzer tests

**Acceptance**:

- `cd cheddar-fpl-sage && python -m pytest tests/tests_new/test_stabilization_integration.py -v` exits `0`
