**Data model note — pre-aggregated odds**:
`GameData['odds']` contains pre-aggregated best values already selected upstream by the `pull-odds` job. There is exactly one `h2hHomeBook`, one `spreadHomeBook`, one `totalLineOverBook`, etc. `resolveOddsDisplayPayload` reads these fields directly — it does **not** perform multi-candidate selection or tie-breaking. The function's job is eligibility validation, field assembly, and split-source detection. The tie-breaker policy below governs the upstream `pull-odds` job; it is reproduced here for transparency and for test fixture documentation, but is **not** implemented in `resolveOddsDisplayPayload`.

**Deterministic best-value policy (upstream, documented for context)**:
- Data eligibility:
  - A value is eligible only when numeric value and source book are both present.
  - Missing numeric or missing book disqualifies that candidate from "best" selection.
- Market ranking:
  - MONEYLINE: choose bestPrice only (most favorable American odds for chosen side).
  - SPREAD/PUCKLINE: choose bestLine first for chosen side; then attach paired price if available.
  - TOTAL: choose bestLine first for OVER/UNDER choice; then attach paired price if available.
- Split source handling:
  - If bestLine and bestPrice are from different books, preserve and display both books and both numbers.
- No-claim fallback:
  - When policy cannot produce an eligible best value, card must not display "best" claim text.

**WI-0893 reason code → display bucket mapping (authoritative)**:

Use this mapping in `classifyPassReasonBucket`. Apply in order — first match wins.

Access paths: `card.play?.execution_status`, `card.play?.reason_codes` (flat array on play), `card.play?.decision_v2?.price_reason_codes` (nested under `decision_v2`), `card.play?.transform_meta?.quality`.

| Bucket | Matching condition |
|--------|-------------------|
| `projection-only` | `card.play?.execution_status === 'PROJECTION_ONLY'` or `card.play?.reason_codes` includes `'PROJECTION_ONLY_EXCLUSION'` |
| `odds-blocked` | `card.play?.execution_status === 'BLOCKED'` or `card.play?.decision_v2?.price_reason_codes` includes `'PROXY_EDGE_BLOCKED'` or `'PROXY_EDGE_CAPPED'` or any entry in `card.play?.reason_codes` starts with `'PASS_EXECUTION_GATE_'` |
| `data-error` | `card.play?.transform_meta?.quality === 'BROKEN'` or `card.play?.reason_codes` includes any of: `PASS_DATA_ERROR`, `PASS_MISSING_KIND`, `PASS_MISSING_SELECTION`, `PASS_MISSING_MARKET_TYPE`, `PASS_MISSING_LINE`, `PASS_NO_MARKET_PRICE`, `PASS_TOTAL_INSUFFICIENT_DATA`, `PASS_MISSING_DRIVER_INPUTS`, `PASS_NO_ACTIONABLE_PLAY` |
| `odds-blocked` (default) | Any PASS card not matching the above (e.g., `PASS_DRIVER_SUPPORT_WEAK`, `PASS_NO_EDGE`, `PASS_DRIVER_CONFLICT`) falls into `odds-blocked` |

**OddsDisplayPayload contract (canonical type for this WI)**:

The following interface must be exported from `web/src/components/cards/game-card-helpers.tsx`. All Steps 2–4 reference it by this name.

```typescript
export interface OddsDisplayPayload {
  market: 'MONEYLINE' | 'SPREAD' | 'PUCKLINE' | 'TOTAL';
  bestPrice: number | null;       // American odds for chosen side (from odds object)
  priceBook: string | null;       // book supplying bestPrice
  bestLine: number | null;        // null for MONEYLINE
  lineBook: string | null;        // null for MONEYLINE; may differ from priceBook
  isSplitSource: boolean;         // true when lineBook !== priceBook and both non-null
  hasVerifiedBest: boolean;       // true ONLY when bestPrice is non-null AND priceBook is non-null
}

export function resolveOddsDisplayPayload(
  marketType: string | undefined,
  selectionSide: string | undefined,
  odds: GameData['odds'],
): OddsDisplayPayload
```
