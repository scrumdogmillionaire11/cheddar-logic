# WI-0878: POTD signal-engine sport-fairness fixes

**ID**: WI-0878
**Goal**: Remove structural scoring bias that favors MLB/NHL over NBA in POTD candidate selection.

**Scope**:
- `apps/worker/src/jobs/potd/signal-engine.js`
- `apps/worker/src/jobs/potd/__tests__/signal-engine.test.js`

**Out of scope**:
- `apps/worker/src/jobs/potd/run_potd_engine.js` (no changes to orchestration)
- Any DB schema, migrations, or other jobs

**Acceptance**:
1. Fixed-line markets (MLB runline ±1.5, NHL puck line ±1.5) no longer receive automatic `lineScore = 1.0`; they fall back to price-only scoring like MONEYLINE
2. `lineDelta` in `scoreCandidate` is normalized by the market's consensus line magnitude so a 1-point NBA spread delta is not valued the same as a 1-point runline delta
3. `selectBestPlay` picks the best candidate per sport first, then compares sport winners — prevents MLB volume (14+ games/day) from dominating by sheer candidate count
4. All pre-existing signal-engine tests pass
5. Three new tests validate: (a) fixed-line fallback to price-only scoring, (b) per-sport pool selection, (c) line-delta normalization
