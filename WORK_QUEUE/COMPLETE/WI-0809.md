# WI-0809: Add rolling cohort windows to decision tier calibration report

| Field | Value |
|---|---|
| **ID** | WI-0809 |
| **Goal** | `buildDecisionTierAudit` in `report_telemetry_calibration.js` should report PLAY/LEAN win rate and total PnL for 14, 30, 60, and 90-day trailing windows so that edge decay or regime shifts are visible without running a manual SQL query. |
| **Priority** | Medium — non-blocking; addresses the "single-regime risk" finding in the quant audit |
| **Depends on** | None |
| **Coordination flag** | solo |

---

## Context

The quant audit found three weeks of data with strong headline numbers (PLAY 70.9% WR) but no way to detect whether performance is degrading. A rolling window view makes regime changes visible in the standard calibration report output.

Current `buildDecisionTierAudit` SQL filters on `status = 'settled'` across all time with no date constraint. The function runs once and produces a flat table for each sport × market_type × tier combination.

Target `--json` output structure after this WI:

```json
{
  "decisionTierAudit": {
    "windows": [
      {
        "days": 14,
        "rows": [
          { "sport": "NBA", "market_type": "MONEYLINE", "tier": "PLAY", "n": 12, "win_rate": 0.75, "total_pnl": 8.4 },
          ...
        ]
      },
      { "days": 30, "rows": [...] },
      { "days": 60, "rows": [...] },
      { "days": 90, "rows": [...] }
    ],
    "allTime": [ ... ]   // existing flat table preserved for backward compatibility
  }
}
```

## Scope

**Files to change:**
- `apps/worker/src/jobs/report_telemetry_calibration.js`
  - `buildDecisionTierAudit(db)` → accept optional `daysBack` param; when provided, add `AND settled_at >= datetime('now', '-N days')` to the WHERE clause
  - Add a `buildDecisionTierAuditWindows(db)` wrapper that calls `buildDecisionTierAudit` for each window (14, 30, 60, 90) plus `null` (all-time) and merges results
  - Update `formatTelemetryCalibrationReport` to include the windowed result in both `--json` and human-readable output

**Out of scope:**
- Adding new sports or market types to the audit scope (remains NBA + NHL, same constant list)
- Changing Brier score, log-loss, or NHL ML sigma sections
- CLI flag changes (runs the same way, just richer output)
- Web-facing API routes

## Implementation Notes

The settled_at column on `card_results` is an ISO8601 string. SQLite `datetime('now', '-14 days')` works correctly for this comparison.

Minimum viable implementation — parameterize the existing query:

```js
function buildDecisionTierAudit(db, { daysBack = null } = {}) {
  const dateFilter = daysBack
    ? `AND cr.settled_at >= datetime('now', '-${daysBack} days')`
    : '';
  const sql = `
    SELECT ...
    FROM card_results cr
    WHERE cr.status = 'settled'
      ${dateFilter}
      AND ...
    GROUP BY ...
  `;
  ...
}
```

Human-readable output format (ASCII table per window, show n=0 windows as "(no data)"):

```
Decision Tier Audit — 14-day window
sport  market_type  tier  n    win_rate  total_pnl
NBA    MONEYLINE    PLAY  12   75.0%     +8.4
...

Decision Tier Audit — All-time
...
```
