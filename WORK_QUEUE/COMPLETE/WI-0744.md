## Problem

Every pitcher K card surfaces:

```
Missing core input: opponent contact profile
```

This fires from `resolveOpponentPitcherKProfile()` in `apps/worker/src/models/mlb-model.js` at L1097–1101:

```js
if (oppObp == null && oppXwoba == null && oppHardHit == null) {
  missingInputs.push('opponent_contact_profile');
}
```

These three fields are read directly from the matchup object:

```js
const oppObp     = matchup?.opp_obp;
const oppXwoba   = matchup?.opp_xwoba;
const oppHardHit = matchup?.opp_hard_hit_pct;
```

The matchup object is built by `buildMatchup()` in `run_mlb_model.js` (called at L975 and L1719). **Neither call site populates these fields.** There is an explicit deferral comment confirming this:

```js
// apps/worker/src/jobs/run_mlb_model.js L1148
// Statcast — null until pull_mlb_statcast is added
swstr_pct: row.season_swstr_pct ?? null,
```

## Root Cause

No MLB team batting contact stats table exists in the DB:
- `team_stats` — NHL only (PP%, penalties)
- `team_metrics_cache` — generic JSON blob cache (sport/team/date), not structured for contact metrics
- No `mlb_team_batting_stats` or equivalent table

`enrichMlbPitcherData()` in `run_mlb_model.js` never queries opposing-team batting data because there is nothing to query.

## Secondary Gap (lower priority, same milestone)

`resolveOpponentPitcherKProfile()` also reads:
- `matchup.opp_k_pct_vs_handedness_l30`
- `matchup.opp_k_pct_vs_handedness_season`
- `matchup.opp_k_pct_vs_handedness_l30_pa` / `_season_pa`

These require per-handedness team batting split data (team K% vs RHP/LHP) — a separate ingest. **Do not block the primary contact profile fix on this.**

Also in scope as a piggyback: `swstr_pct` for pitchers (same Statcast deferral comment, same milestone).

## Implementation Notes

### Preferred approach: new `mlb_team_batting_stats` table (Option A)

Structured table preferred over `team_metrics_cache` JSON blob — enables SQL joins, indexing, and schema evolution.

### Data source priority

1. **FanGraphs Team Batting** (free, daily) — OBP, K%, xwOBA available at team level
2. **Baseball Reference** — OBP, K% (no xwOBA)
3. **MLB Stats API** (`/v1/teams/stats`) — OBP available; hard_hit_pct requires Statcast endpoint

If `xwoba` and `hard_hit_pct` are unavailable in a single source, populate what's available; `resolveOpponentPitcherKProfile()` already handles partial data gracefully (falls back per-field).

### Wiring point in `enrichMlbPitcherData`

After the pitcher rows are resolved, add (K mode only):

```js
// Resolve opposing-team contact profile for K engine
if (forKEngine) {
  const oppTeamForHome = awayTeam;  // home pitcher faces away batters
  const oppTeamForAway = homeTeam;
  const oppContactHome = queryTeamContactStats(db, oppTeamForHome, currentSeason);
  const oppContactAway = queryTeamContactStats(db, oppTeamForAway, currentSeason);
  mlb.opp_contact_home = oppContactHome;  // { opp_obp, opp_xwoba, opp_hard_hit_pct }
  mlb.opp_contact_away = oppContactAway;
}
```

Then `buildMatchup` (or the K engine input builder) attaches these as `matchup.opp_obp` etc.
