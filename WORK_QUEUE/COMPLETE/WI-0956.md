**Diagnosis notes**:

- At runtime every MLB game and every NHL game taking the TOTAL market path crashes inside the per-game try/catch with message: `fallback_chain_executed is not defined`.
- `fallback_chain_executed` is a *property* on the object returned by `resolveSnapshotAge()` (internal camelCase var `fallbackChainExecuted`, snake_case in return value at `packages/data/src/db/odds.js`). It is never a standalone declared variable in the visible source of either model file, yet a ReferenceError fires at runtime.
- Likely cause: a destructuring pattern such as `const { ..., fallback_chain_executed } = resolution` was added in a recent edit but the destructured binding is referenced somewhere outside the scope where it was destructured, or the variable name in the destructuring doesn't match the property key being accessed elsewhere.
- Affected output pattern (MLB): `[MLB_PIPELINE_STATE] ÔÇŽ card_ready=true` immediately followed by `ÔŁî <gameId>: fallback_chain_executed is not defined`.
- Affected output pattern (NHL): `[err] <gameId>: fallback_chain_executed is not defined` for any game entering the TOTAL market path.
- Two NHL games succeed (KRAKEN @ GOLDEN KNIGHTS, SHARKS @ BLACKHAWKS) ÔÇö both take the SPREAD market path, bypassing the crash site. All TOTAL-path games fail.
