# WI-0756: Projection settled view — display resolved projection cards with actual vs projected comparison

**ID**: WI-0756
**Goal**: Build a dedicated section on the Results page (or a new `/projections/results` route) that shows settled projection cards with actual outcome vs model projection, stratified by card type (`nhl-pace-1p` binary hit/miss, `mlb-f5` MAE delta).

**Background**:
Once WI-0757 (actual result ingestion) is complete, `card_payloads` rows for `nhl-pace-1p` and `mlb-f5` will have an `actual_result` field. This WI builds the display layer:

- `nhl-pace-1p`: binary outcome — show model projected total, fixed 1.5 line, actual 1P goals, HIT (✓) or MISS (✗)
- `mlb-f5`: continuous outcome — show projected F5 runs, actual F5 runs, absolute error delta (e.g., "|4.2 - 3.0| = 1.2")

The Results page currently has tabs/sections for game lines, props, and model health. A new "Projections" section should be additive — it must not disturb the odds-backed settlement tables.

**Scope**:
- `web/src/app/results/page.tsx` — add Projections section (or tab)
- `web/src/app/api/results/route.ts` (or adjacent) — add projection settled query
- `web/src/app/api/results/projection-metrics.ts` — add `PROJECTION_FAMILY_LABELS` entries if needed
- `web/src/components/results/` — new `ProjectionResultsTable.tsx` component (or inline)

**Out of scope**:
- Odds-backed settlement tables — must not be modified
- Actual result ingestion (WI-0757) — this WI reads from whatever field WI-0757 establishes
- Retroactive backfill of historical records

**Acceptance**:
- Results page shows a "Projections" section listing settled cards for `nhl-pace-1p` and `mlb-f5`
- Each row shows: game, card type, model projected value, actual value, delta/outcome
- `nhl-pace-1p` rows show: `Projected X.X | Line 1.5 | Actual Y | HIT/MISS`
- `mlb-f5` rows shows: `Projected X.X | Actual Y | Error: |X.X - Y|`
- Section is hidden / shows empty state when no settled projection records exist
- `npm --prefix web run lint` passes
