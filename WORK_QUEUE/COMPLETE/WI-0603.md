# WI-0603: MLB F5 ML ingest + side-projection layer

**ID**: WI-0603
**Goal**: Ingest F5 Moneyline odds (`ml_f5`) from the odds API and add a side-projection layer to the MLB model so F5 ML can become a playable card market.

**Scope**:

- `apps/worker/src/jobs/pull_odds_hourly.js` (add `ml_f5` key mapping)
- `packages/data/db/migrations/` (add `ml_f5` column to `odds_snapshots` if not present)
- `apps/worker/src/models/mlb-model.js` (add `projectF5ML` side-projection function)
- `apps/worker/src/jobs/run_mlb_model.js` (wire F5 ML into output if signal clears threshold)
- `docs/MARKET_REGISTRY.md` (update F5 ML row from ⚙️ to ✅ on completion)

**Out of scope**:

- DUAL_RUN market selection logic (see WI-0602)
- Pipeline health watchdog differentiation (see WI-0604)
- Full-game ML or any non-F5 ML market
- Any NHL/NBA changes

**Background**:
F5 ML is a natural complement to F5 Total — the same pitcher-quality projection that generates an F5 Total signal also carries implied side information (the weaker starting pitcher's team is more likely to give up early runs). `ml_f5` odds exist on The Odds API under the `h2h` market scoped to first 5 innings, but the current `pull_odds_hourly.js` does not map this key. Without the price the model cannot compute edge. `docs/MARKET_REGISTRY.md` lists this as ⚙️ (partially wired).

**Acceptance**:

- `pull_odds_hourly.js` pulls and stores `ml_f5_home` and `ml_f5_away` into the `odds_snapshots` row for each MLB game.
- `mlb-model.js` exposes a `projectF5ML(homePitcher, awayPitcher, mlF5Home, mlF5Away)` function that returns a side pick (HOME/AWAY/PASS) with edge and confidence.
- F5 ML cards are only emitted when `ml_f5` price is present AND edge clears `lean_edge_min`.
- A game with no `ml_f5` price logs `NO_F5_ML_LINE` as a blocking reason.
- `run_mlb_model.js` enables the dormant WI-0604 `expectF5Ml` health path so missing F5 ML can surface as `F5_ML_UNAVAILABLE` once ingest is live.
- A migration is added if `odds_snapshots` table is missing the F5 ML columns.
