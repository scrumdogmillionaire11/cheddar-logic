# WI-0450: FPL Polling Stops On Failed Analysis Status

**ID**: WI-0450
**Goal**: Ensure FPL projections polling stops immediately when backend returns `ANALYSIS_NOT_READY` with a failed analysis state, instead of continuing to poll as if still running.

**Scope**:
- `WORK_QUEUE/WI-0450.md`
- `web/src/lib/fpl-api.ts`
- `web/src/__tests__/fpl-api-analysis-failure-polling.test.js`

**Acceptance**:
- `pollForDetailedProjections()` stops retrying when projections endpoint returns `425` with failed-analysis detail.
- User-facing error preserves backend failure context when available (e.g., `Analysis is failed, not complete`).
- `425`/`202` continue to retry for queued/running/analyzing states.
