# WI-0453: NHL Injury Status Filtering — Player Props + Goalie Confirmation

**ID**: WI-0453
**Goal**: Integrate NHL injury/availability status checks across all NHL models so injured or unavailable players are automatically excluded from shot log pulls, card generation, and goalie confidence scoring — replacing the current manual `NHL_SOG_EXCLUDE_PLAYER_IDS` workaround and hardening the main NHL model's goalie status surface.

**Context**: Two surfaces are affected:

1. **Player prop model**: `pull_nhl_player_shots.js` and `run_nhl_player_shots_model.js` use a static `NHL_SOG_EXCLUDE_PLAYER_IDS` env var. This requires manual maintenance and will silently produce bad cards if the list falls out of date.
2. **Main NHL model**: `run_nhl_model.js` uses goalie status (`goalie.home.status`, `goalie.away.status`) sourced from odds snapshots/scrapers. If the confirmed starting goalie is injured or a late scratch, the model may be scoring with stale goalie data. Whether this is already handled should be verified.

**Scope**:

- `apps/worker/src/jobs/pull_nhl_player_shots.js` — check NHL API injury/roster status per player; skip players marked injured, IR, or inactive before fetching shot logs
- `apps/worker/src/jobs/run_nhl_player_shots_model.js` — secondary guard: skip card generation for players with no valid recent logs due to injury
- `apps/worker/src/jobs/run_nhl_model.js` — audit goalie status handling: confirm that a confirmed-starter check exists and that an unknown/injured goalie degrades confidence score rather than silently using a stale value
- `packages/data/db/migrations/030_add_player_injury_status.sql` — add `injury_status` and `checked_at` to `player_shot_logs` or a new `player_availability` table
- `docs/NHL_PLAYER_SHOTS_PROP_MARKET.md` — update process doc with injury check step

**Out of scope**:

- Injury tracking for non-NHL sports in this WI
- Push notifications or UI indicators for injured players
- Historical injury data / retroactive log cleanup
- Full lineup confirmation pipeline for skaters (not goalies)

**Acceptance**:

- Players with NHL API status `injured`, `IR`, or inactive are skipped in `pull_nhl_player_shots` and logged by name + reason
- `run_nhl_player_shots_model` does not generate cards for players with no valid recent logs
- `NHL_SOG_EXCLUDE_PLAYER_IDS` continues to work as a manual override
- `run_nhl_model.js` goalie status audit documented: either confirm existing degradation path is sound, or add a confidence penalty for unknown/unconfirmed goalie
- Process doc updated to describe injury check step and fallback behavior
