## Problem Statement

A projection for **Colorado Rockies @ Houston Astros (Apr 16)** shows:

- **Status:** PASS
- **Label:** "No edge at current price"
- **But underlying model data visible:** finalEdge +0.90, drivers=[SP_MISMATCH|OFFENSE_MATCHUP_EDGE]

This creates confusion: PASS projections should not be surfaced as actionable, and PASS cards must not expose internal edge math.

**Incident Context:**

- WI-0967 excluded PASS rows from over/under record tallying but did not address rendering/surfacing.
- This suggests the filter boundary is incomplete (metrics path patched, card/display path still leaking PASS internals).
