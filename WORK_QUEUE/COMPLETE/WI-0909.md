# WI-0909: NHL Player Shots (Full-Game) Settlement Source Unification

**ID**: WI-0909
**Goal**: Unify full-game NHL player-shots settlement so projection actual_result and card_results grading use a single consistent stat source and reconciliation rules, and ensure full-game NHL player-shots plays reliably surface in both dev and production under the same publish/display contract.

**Depends on**: none

**Acceptance**:

- Full-game nhl-player-shots uses one declared source-of-truth path for settlement values, with deterministic fallback behavior when data is missing.
- actual_result.shots and card_results result cannot silently disagree for the same card without explicit error/reason metadata.
- Settlement errors for source mismatch are observable and test-covered.
- Tests include at least one mismatch fixture that would have previously diverged.
- Full-game `nhl-player-shots` plays that meet publish/display eligibility surface consistently in dev and production APIs (same lifecycle and gating behavior, no environment-only suppression).
- If a play is not surfaced, the reason is explicit via deterministic status/reason metadata or telemetry (not silent drop).
