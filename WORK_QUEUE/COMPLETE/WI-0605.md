# WI-0605: MLB pitcher stats ingest — populate team join key for model enrichment

**ID**: WI-0605
**Goal**: Fix MLB pitcher-stats ingest so `mlb_pitcher_stats.team` is populated with a value that matches `run_mlb_model.js` enrichment lookups.

**Scope**:

- `apps/worker/src/jobs/pull_mlb_pitcher_stats.js`
- `apps/worker/src/jobs/run_mlb_model.js` (only if a defensive fallback is needed after ingest fix)
- `apps/worker/src/jobs/__tests__/` or `apps/worker/src/__tests__/` (MLB ingest/enrichment regression tests)

**Out of scope**:

- F5 total / F5 ML market ingest
- MLB projection-threshold tuning
- NHL or NBA pipeline issues

**Background**:
The first post-WI-0604 MLB refresh showed `pull_mlb_pitcher_stats` reporting successful upserts, but every sampled `mlb_pitcher_stats` row had `team = null`. `run_mlb_model.js` enriches by `team = ?`, so all pitchers were treated as missing and MLB emitted only `F5_TOTAL_UNAVAILABLE` / `NO_F5_LINE` no-play states. This is an upstream ingest-key bug, not a watchdog regression.

**Acceptance**:

- Newly upserted `mlb_pitcher_stats` rows have non-null `team` values for probable starters.
- `run_mlb_model.js` can find same-day probable pitchers for MLB games using the stored team value.
- A same-day MLB refresh no longer logs blanket `PITCHER_DATA_MISSING` for games whose probable pitchers were just upserted.
- Regression tests cover the team-source path that previously produced null.
