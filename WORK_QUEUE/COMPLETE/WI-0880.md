# WI-0880: Add h2h market to NHL odds fetch

**ID**: WI-0880
**Goal**: Add the h2h (moneyline) market to the NHL odds fetch so the POTD signal engine can build and score NHL moneyline candidates. Currently NHL fetches only totals; buildMoneylineCandidates always receives an empty game.market.h2h array and produces zero NHL candidates -- making NHL invisible as a POTD source regardless of game count.

**Scope**:
- packages/odds/src/config.js

**Out of scope**:
- Changes to any consumer of SPORTS_CONFIG beyond the config file itself
- Signal-engine wiring (WI-0881)
- NHL totals candidates or any other sport config

**Acceptance**:
1. SPORTS_CONFIG.NHL.markets contains both "totals" and "h2h"
2. SPORTS_CONFIG.NHL.tokensPerFetch updated from 1 to 2
3. Budget comment updated: NHL (main job): totals + h2h = 2 tokens; grand total 4 -> 5
4. No other sport markets or tokensPerFetch values changed
5. npm --prefix apps/worker test --no-coverage passes

**Guard for WI closeout**:
- Run: node -e "const c=require('./packages/odds/src/config'); console.log(c.SPORTS_CONFIG.NHL.markets, c.SPORTS_CONFIG.NHL.tokensPerFetch)"
  Expected output: [ 'totals', 'h2h' ] 2

**Owner agent**: unassigned
**Time window**: 2026-04-11 -> TBD
**Coordination flag**: needs-sync -- packages/odds/src/config.js is a serialized shared touchpoint

**Tests to run**:
- npm --prefix apps/worker test --no-coverage

**Manual validation**:
- Run POTD engine on a day with NHL games; confirm [Odds] API log line for NHL shows markets=totals,h2h in URL
- With force=true outside publish window, confirm NHL games appear in the scored candidate output
