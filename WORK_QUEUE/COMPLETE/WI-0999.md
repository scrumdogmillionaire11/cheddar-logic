# WI-0999: POTD Shadow Settlement Reporting

**ID**: WI-0999
**Goal**: Extend the POTD sanity report to show settled shadow-candidate performance, especially dynamic-only candidates, while remaining read-only.

**Scope**:
- `apps/worker/src/scripts/potd-sanity-check.js`
- `apps/worker/src/jobs/potd/__tests__/potd-sanity-check.test.js`
- `docs/ops-runbook.md` only if operator instructions are added

**Out of scope**:
- Official POTD firing-rule changes
- Writes to any database table
- Shadow settlement job implementation
- Web/API/UI changes
- Discord publishing

**Acceptance**:
- Report includes daily comparison rows with: date, strict count, soft count, dynamic count, official POTD fired, best strict edge, best dynamic-only edge, and best dynamic-only score.
- Report includes settlement performance for dynamic-only candidates: settled count, wins, losses, pushes, virtual P&L units, and ROI.
- Report includes sport breakdown for strict-qualified, soft-qualified, dynamic-qualified, and dynamic-only candidates.
- Report includes edge buckets: `<0%`, `0-0.5%`, `0.5-1.0%`, `1.0-1.5%`, `1.5-2.0%`, and `2.0%+`.
- Report includes high-score buckets: `0.70-0.72`, `0.72-0.75`, and `0.75+`.
- Dynamic-only candidate rows include sport, play/selection, edge, score, dynamic floor, result when settled, virtual P&L when settled, and nominee rank when applicable.
- Tests cover bucket boundaries, sport breakdown counts, settled dynamic-only aggregation, and read-only formatting behavior.

**Owner agent**: unassigned
**Time window**: TBD
**Coordination flag**: solo

**Tests to run**:
- `npm --prefix apps/worker test -- --runTestsByPath src/jobs/potd/__tests__/potd-sanity-check.test.js`

**Manual validation**:
- Run `npm --prefix apps/worker run stats:potd` against a DB containing settled `potd_shadow_results` rows and confirm the output can answer whether dynamic-only qualifiers are high-score positive-edge candidates or low-quality noise.

**Dependencies**:
- WI-0998

**Notes**:
- This WI is reporting-only. It should not change selector thresholds, publish gates, settlement behavior, or official bankroll accounting.
