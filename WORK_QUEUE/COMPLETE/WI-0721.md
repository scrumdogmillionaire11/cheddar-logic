## Background

The NHL totals model is detailed enough that drift can hide in multiple sub-layers:
- `homeGoalieCertainty` / `awayGoalieCertainty` (CONFIRMED | EXPECTED | UNKNOWN)
- `homeAdjustmentTrust` / `awayAdjustmentTrust` (FULL | DEGRADED | BLOCKED | NEUTRALIZED)
- Modifier breakdown and cap application
- 1P classification which has its own separate confidence cap and gating rules

Without explicit snapshots at these sub-layers, a change to any constant (goalie certainty threshold, trust degradation rate, modifier cap value) is invisible until a card flips classification — and by then the cause is lost.

## Implementation Plan

1. **Identify call sites**: search codebase for `predictNHLGame` and any wrapper that calls it from the scheduler/runner. Confirm the exact file that is the NHL runner (analogous to `run_nba_model.js`).

2. **Add audit snapshot emission** at the NHL runner layer. Use a shared `buildNhlModelSnapshot({ paceResult, payload, sigmaTotal })` helper to validate fields, build the snapshot, freeze it, and attach it to the pace-derived payloads:

   ```js
   card.payloadData._model_snapshot = {
     homeExpected: result.homeExpected,
     awayExpected: result.awayExpected,
     expectedTotal: result.expectedTotal,
     rawTotalModel: result.rawTotalModel,
     regressedTotalModel: result.regressedTotalModel,
     sigma_total: sigmaTotal,
     modifierBreakdown: result.modifierBreakdown,
     homeGoalieCertainty: result.homeGoalieCertainty,
     awayGoalieCertainty: result.awayGoalieCertainty,
     homeAdjustmentTrust: result.homeAdjustmentTrust,
     awayAdjustmentTrust: result.awayAdjustmentTrust,
     official_eligible: result.official_eligible,
     first_period_model: result.first_period_model,
     consistency: {
       pace_tier: payload.consistency?.pace_tier,
       event_env: payload.consistency?.event_env,
       total_bias: payload.consistency?.total_bias,
     },
   };
   ```

   Attach `_model_snapshot` and `nhl_goalie_certainty_pair` to:
   - `nhl-pace-totals`
   - `nhl-pace-1p`
   - `nhl-totals-call`

3. **Add `execution_status` field** to NHL card payloads (same pattern as NBA/MLB):
   - `execution_status: 'EXECUTABLE' | 'PROJECTION_ONLY' | 'BLOCKED'`
   - Keep the existing `publishDecisionForCard` / `applyUiActionFields` path
   - NHL-specific demotion: pace-derived cards and totals calls with any `UNKNOWN` goalie certainty must not remain `EXECUTABLE`

4. **Add invariants** to the NHL model wrapper (throw/log if violated):
   - **Invariant 1**: If either `homeGoalieCertainty === 'UNKNOWN'` or `awayGoalieCertainty === 'UNKNOWN'`, final confidence must be capped (verify the cap is applied, log if not)
   - **Invariant 2**: If goalie certainty is `UNKNOWN`, the same side must not have `AdjustmentTrust === 'FULL'`
   - **Invariant 3**: If `modifierBreakdown.modifier_cap_applied === true`, the capped modifier total must be the applied result and must stay within the modifier cap
   - **Invariant 4**: regression must move `rawTotalModel` toward the NHL baseline, never farther away
   - **Invariant 5**: 1P classification must be `PASS` when goalie uncertainty rule applies
   - **Invariant 6**: 1P cannot remain actionable when full-game execution is blocked by the same unknown-goalie gating state
   - **Invariant 7**: if `first_period_model.classification !== 'PASS'`, `first_period_model.reason_codes` must be non-empty
   - Mixed severity:
     - `CRITICAL` for logic corruption / impossible state
     - `WARNING` for missing but non-blocking audit metadata

5. **Add `nhl_goalie_certainty_pair`** to card payload for quick filtering in audits:
   ```js
   card.payloadData.nhl_goalie_certainty_pair =
     `${homeGoalieCertainty}/${awayGoalieCertainty}`;
   // e.g. 'CONFIRMED/CONFIRMED', 'EXPECTED/UNKNOWN', etc.
   ```
