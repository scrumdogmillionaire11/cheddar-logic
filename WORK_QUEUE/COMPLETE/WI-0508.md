### 1. ✓ Dependency Drift Documentation

**Current State Documented**:
- Python baseline: 3.12.10 with 191 packages
- soccerdata==1.8.8 installed (no new deps; version pinning risk only)
- Related packages: pandas==2.3.3, numpy==2.4.0, scipy==1.16.3, lxml==6.0.2, requests==2.32.3, beautifulsoup4==4.13.4
- Risk: Version updates via pip could affect concurrent non-soccer workflows
- Location: Section 1 of isolation guide

### 2. ✓ Isolation Strategy Proposed

**PRIMARY**: pyenv Virtual Environment
- Recommended for production: survives OS upgrades, supports rollback, operator-friendly
- Implementation: `.soccer-python-isolation/venv` directory with dedicated Python 3.12.10
- Activation: via `SOCCER_XG_PYTHON_BIN` environment variable

**FALLBACK**: Docker Containerization
- Recommended for deployment CI/CD: complete host isolation, reproducible
- Implementation: Dockerfile with pinned soccerdata==1.8.8 + deps
- Fallback used if pyenv unavailable

Location: Section 2 of isolation guide

### 3. ✓ Operator-Safe Commands Defined

**Setup** (one-time):
```bash
mkdir -p .soccer-python-isolation/venv
python -m venv .soccer-python-isolation/venv
source .soccer-python-isolation/venv/bin/activate
pip install soccerdata==1.8.8
deactivate
export SOCCER_XG_PYTHON_BIN="/path/to/.soccer-python-isolation/venv/bin/python3"
```

**Runtime**:
```bash
# xG job uses isolated Python (SOCCER_XG_PYTHON_BIN configured)
ENABLE_SOCCER_XG_MODEL=true npm --prefix apps/worker run job:pull-soccer-xg-stats
```

**Integration**: Worker respects `SOCCER_XG_PYTHON_BIN` env var; falls back to global `python3` if unset
- File: `apps/worker/src/jobs/pull_soccer_xg_stats.js` line 16
- Backward compatible: existing deployments unaffected

Location: Section 3 of isolation guide

### 4. ✓ Rollback & Cleanup Steps Defined

**4-Step Rollback**:
1. Unset `SOCCER_XG_PYTHON_BIN` and remove from shell rc files
2. Delete `.soccer-python-isolation/` directory
3. Verify soccerdata removed from global Python (import should fail)
4. Confirm non-soccer Python workflows still function

**Evidence of Success**:
- `$SOCCER_XG_PYTHON_BIN` returns empty
- `.soccer-python-isolation/` removed
- `python3 -c "import soccerdata"` → ModuleNotFoundError
- `python3 -c "import pandas; import numpy"` → succeeds (global untouched)
