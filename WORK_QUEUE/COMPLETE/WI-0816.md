# WI-0816: Add 5xx retry to Odds API fetch in `packages/odds/src/index.js`

| Field | Value |
|---|---|
| **ID** | WI-0816 |
| **Goal** | Wrap the single `axios.get` call in `fetchFromOddsAPI` with a 2-attempt exponential-backoff retry loop for transient errors (5xx, network timeout, ECONNRESET). Permanent errors (401/402/403) remain non-retried. Prevents a transient network blip from silently dropping all odds for a sport for an entire job cycle. |
| **Priority** | Medium |
| **Depends on** | None |
| **Coordination flag** | None |

## Background

`fetchFromOddsAPI` in `packages/odds/src/index.js` makes a single `axios.get` with a 10-second timeout. Any transient error (5xx, ECONNRESET, ETIMEDOUT) propagates immediately to `pull_odds_hourly.js`, which logs the error, records an ingest failure, and skips the sport entirely for the full cycle (~30–60 min).

```js
// Current — no retry
const response = await axios.get(url, { params, timeout: 10000 });
```

A transient 502/503 at The Odds API (common during peak hours) silently stales odds until the next scheduled run. For T-minus windows this means the model runs on odds that may be 60–90 minutes old during the most latency-sensitive window.

The existing backup-key fallback handles 401/402/429 (quota/auth), not 5xx. This WI adds 5xx retry orthogonally to that mechanism.

## Detailed Implementation

### Task 1 — Extract `axiosGetWithRetry` helper

**File:** `packages/odds/src/index.js`

Add a small private helper above `fetchFromOddsAPI`:

```js
/**
 * axios.get with exponential-backoff retry for transient errors.
 * Permanent errors (401, 402, 403) are rethrown immediately.
 * @param {string} url
 * @param {object} config - axios config (params, timeout, etc.)
 * @param {number} maxRetries - default 2 (total attempts = 3)
 * @returns {Promise<import('axios').AxiosResponse>}
 */
async function axiosGetWithRetry(url, config, maxRetries = 2) {
  for (let attempt = 0; attempt <= maxRetries; attempt++) {
    try {
      return await axios.get(url, config);
    } catch (err) {
      const status = err?.response?.status;
      const isPermanent = status === 401 || status === 402 || status === 403;

      if (isPermanent) {
        throw err; // Propagate immediately — backup-key fallback handles these
      }

      if (attempt === maxRetries) {
        throw err; // Out of retries
      }

      const delayMs = Math.pow(2, attempt) * 1000; // 1s, 2s
      console.warn(
        `[Odds] Transient error (HTTP ${status ?? err.code}) on attempt ${attempt + 1}/${maxRetries + 1} — retrying in ${delayMs}ms`,
        { url, sport: config?.params?.sport },
      );
      await new Promise((r) => setTimeout(r, delayMs));
    }
  }
}
```

### Task 2 — Replace `axios.get` calls in `fetchFromOddsAPI`

Replace every `await axios.get(url, ...)` in `fetchFromOddsAPI` (single-league and multi-league paths) with `await axiosGetWithRetry(url, ...)`.

There are exactly two call sites in the single-league path and two in the multi-league path (regular + soccer spread fallback). Replace all four.

The rest of `fetchFromOddsAPI` is unchanged — `response.data`, `response.headers`, and error handling remain exactly as-is.
