# WI-0558: Stabilize live API smoke/contract tests when no local server is running

**ID**: WI-0558
**Goal**: Make API/UI smoke tests deterministic by either (a) requiring an explicit reachable `CARDS_API_BASE_URL` preflight or (b) providing source/DB fallback behavior, instead of failing with opaque `fetch failed` errors.

**Scope**:
- `web/src/__tests__/ui-results-smoke.test.js`
- `web/src/__tests__/api-games-market-smoke.test.js`
- `web/src/__tests__/api-games-repair-budget.test.js`
- `web/src/__tests__/api-nhl-payload-contract.test.js`
- `web/src/__tests__/api-results-decision-segmentation.test.js`
- `web/package.json` (optional: add a dedicated script variant for explicit live endpoint runs)
- `.github/workflows/ci.yml`

**Out of scope**:
- Business logic changes to `/api/games`, `/api/cards`, or `/api/results`
- Rewriting these tests to a full integration harness with auto-booted Next server
- Any production runtime behavior

**Background**:
Current smoke/contract scripts default to `http://localhost:3000` and hard-fail with `fetch failed` when a local server is absent. Reproduced failures from repo root:
- `npm --prefix web run test:ui:results`
- `npm --prefix web run test:api:games:market`
- `npm --prefix web run test:api:games:repair-budget`
- `npm --prefix web run test:api:nhl-contract`
- `node web/src/__tests__/api-results-decision-segmentation.test.js`

This creates noisy false negatives and unclear operator guidance.

**Acceptance**:
- Each scoped test script performs a reachable-endpoint preflight before fetch assertions.
- On missing endpoint, tests either:
  - fail fast with explicit setup message (including expected env var and sample command), or
  - run a deterministic fallback path (source/contract assertions only) and log that fallback mode was used.
- No scoped test exits with an unqualified bare `fetch failed` message.
- `web/package.json` scripts and/or test file headers document when a live API base URL is required.
