# WI-0577: Prove and harden the V2 FIRE veto for NHL props

**ID**: WI-0577
**Goal**: Prove with runner-level regression coverage that odds-backed NHL prop cards never land as `FIRE` / `PLAY` when the V2 priced edge for the projected side is non-playable or loses on projection conflict, and harden the runner only if that proof fails.

**Scope**:
- `WORK_QUEUE/WI-0577.md`
- `apps/worker/src/jobs/run_nhl_player_shots_model.js`
- `apps/worker/src/jobs/__tests__/run_nhl_player_shots_model.test.js`

**Out of scope**:
- `web/**` display or transform logic
- Payload shape changes, field renames, or schema changes
- Queue/state reconciliation for `WI-0573` through `WI-0576`
- Non-runner NHL prop fixes (`WI-0578+`)

**Background**:
The March 23 sprint reprioritization flagged a critical defect: V1/legacy direction could still make an odds-backed NHL prop look playable even when the V2 priced edge for that same projected side was not playable or the priced side lost on projection conflict. This WI proves the current runner contract with regression coverage and applies the smallest runner-side fix only if the proof fails.

**Acceptance**:
- `WORK_QUEUE/WI-0577.md` exists and is claimed on this branch.
- A regression test covers a real priced prop where legacy/V1 seeds an `OVER`/`HOT` path but V2 pricing makes the projected side non-playable; the stored payload must end as:
  - `prop_decision.verdict = 'NO_PLAY'`
  - `play.action = 'PASS'`
  - `play.status = 'PASS'`
  - `decision_v2.official_status = 'PASS'`
- A second regression test covers explicit projection conflict (projection leans one side, priced edge supports the other); the stored payload remains non-playable on the projected side and includes `PROJECTION_CONFLICT`.
- If either regression fails, the fix is applied in `buildCanonicalPropDecision()` within `run_nhl_player_shots_model.js`.
- No public API, schema, or payload-field changes are introduced.
