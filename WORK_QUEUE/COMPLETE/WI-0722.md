### 2. `build_audit_snapshot.js`

Core function: `buildAuditSnapshot(fixture, options)` → snapshot object

Steps:
1. Deep-clone input (never mutate fixture)
2. Compute `input_hash` from normalized input
3. Run enrichment pipeline
4. Compute `enriched_hash`
5. Deep-clone enriched state before model runs
6. Run model layer
7. Deep-clone model output before decision runs
8. Run decision layer (`publishDecisionForCard` equivalent)
9. Deep-clone decision output before publish/UI runs
10. Run publish/UI layer
11. Snapshot final cards
12. Collect violations from invariants check (import from WI-0723 when available; stub until then)
13. Return full snapshot object

**Hash normalization rules** (applied before every hash):
- Sort all object keys recursively
- Remove volatile fields: `generated_at`, `run_id`, `runId`, `captured_at`, `job_run_id`, `created_at`, `updated_at`
- Normalize `null` and `undefined` → `null`
- Normalize empty array `[]` and missing field → `[]`

Snapshot output shape:
```json
{
  "fixture_id": "...",
  "sport": "NBA",
  "model_version": "",
  "run_at": "",
  "stage_hashes": {
    "input": "",
    "enriched": "",
    "model": "",
    "decision": "",
    "publish": ""
  },
  "model_snapshot": {},
  "decision_snapshot": {},
  "publish_snapshot": {},
  "final_cards": [],
  "violations": []
}
```

### 3. `run_model_audit.js`

CLI entrypoint:

```
# Single fixture
node apps/worker/src/audit/run_model_audit.js --sport NBA --fixture nba_2026_04_01_total_01

# All fixtures for sport
node apps/worker/src/audit/run_model_audit.js --sport NBA

# All sports
node apps/worker/src/audit/run_model_audit.js --all
```

Output summary:
```json
{
  "total": 12,
  "passed": 10,
  "failed": 2,
  "drift_categories": ["MODEL_DRIFT", "DECISION_DRIFT"],
  "violations": [...]
}
```

Exit code: 0 if all passed, 1 if any failed or any high-severity violation.

## Acceptance

- `loadFixture` throws hard (not returns null) for malformed fixture
- Hash is stable: same input object always produces same hash after normalization
- Volatile fields are excluded from hash path
- Deep clone is performed before each stage so in-place mutation does not contaminate snapshots
- `run_model_audit.js` runs in CLI mode and suite mode
- Worker tests pass
