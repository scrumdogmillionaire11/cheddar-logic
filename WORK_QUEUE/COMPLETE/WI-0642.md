# WI-0642: Remove Soccer and NCAAM — full extraction from worker, web, and packages

**ID**: WI-0642
**Goal**: Delete all Soccer and NCAAM code from the codebase — jobs, scheduler hooks, models, web route branches, transform logic, package references, and tests. Neither sport will be supported going forward; dead code should not be maintained or tested.
**Owner agent**: codex
**Time window**: 2026-03-28
**Coordination flag**: needs-sync
**CLAIM**: codex 2026-03-28T19:59:20Z

**Background**:

Soccer and NCAAM are fully wired into the system but neither is actively generating value. The code adds scheduler complexity, inflates the web transform and route, seeds false card types into the DB contract tests, and creates ongoing maintenance surface. NCAAM was already soft-disabled (`ENABLE_NCAAM_FT_REFRESH` defaults off as of 2026-03-24 comment in `main.js`). Soccer has a custom T-45 lineup checkpoint, xG ingest, and player props pipeline that all run live. Clean removal is safer than leaving disabled stubs that diverge from real behaviour silently.
