# WI-0469: Quick Filter Overhaul

**ID**: WI-0469
**Goal**: Create a better quick-filter experience with fast sort/sport/market controls, and ensure Player Props only shows and applies market-relevant filters.

**Scope**:

- `WORK_QUEUE/WI-0469.md`
- `web/src/components/filter-panel.tsx`
- `web/src/components/cards-page-client.tsx`
- `web/src/lib/game-card/filters.ts`
- `web/src/lib/game-card/presets.ts`

**Out of scope**:

- API payload/schema changes under `web/src/app/api/**`
- Worker/model generation logic under `apps/worker/**`
- DB schema/migrations under `packages/data/**`
- New pages/routes outside the existing cards page

**Acceptance**:

1. Cards page exposes always-visible quick controls for Sort, Sports, and Markets.
2. Game Lines mode uses game markets (`ML`, `SPREAD`, `TOTAL`) in quick filters and applies them through the game filter pipeline.
3. Player Props mode does **not** use parity; it only shows/apply relevant prop-market filters (prop stat groups) and hides irrelevant game-market controls.
4. Props filtering is functional for relevant controls (sport, time window, actionability, prop market group, search target/query, and sort).
5. Quick Presets active-state matching is reliable (order-insensitive array matching), and dynamic `watch_next_4h` behaves as a rolling 4-hour preset when applied.
6. Focused web tests and lint pass.
