# WI-0385: NHL 1P Model Rebuild (Pass-First, De-biased)

**ID**: WI-0385

**Goal**:
Replace the current NHL 1P projection logic (fixed 30% transform + hard clamp + raw delta vs 1.5) with a calibrated first-period layer that has independent environment modifiers, pass-first classification bands, and explicit guardrails against structural over bias.

## Summary

Current 1P logic is too primitive to be production-reliable:

```text
expected1pTotal = clamp((full_game_total) * 0.30, 1.4, 2.4)
edge = expected1pTotal - 1.5
```

This guarantees three failure behaviors:

1. 1P is a shadow of full-game total
2. clamp compresses outcomes into synthetic over territory
3. edge vs 1.5 is projection distance, not pricing edge

Observed slate symptoms (latest run) support this:
- average projection: ~`1.975`
- min projection: ~`1.611`
- average edge vs 1.5: ~`+0.475`

Interpretation: the current structure is biased to prefer overs.

## Why This Is Needed

### Problem 1 — Fixed 30% transform

Using:

```text
1P = full_game_total * 0.30
```

means 1P is derived, not modeled. This is only acceptable if full-game totals are perfectly calibrated and first-period share is stable across game states, which is not true.

### Problem 2 — Hard clamp [1.4, 2.4]

Clamp behavior suppresses both tails:
- low-event games cannot express true under environments
- high-event games flatten into narrow ceiling bands

### Problem 3 — “Edge” is not edge

```text
edge = proj_1p - 1.5
```

is projection delta, not a priced market edge. Raw delta alone should not drive play status.

## New Interface Contract

Implement/emit 1P fields following this shape:

```ts
type NhlFirstPeriodModel = {
	game_id: string;

	full_game_total_model: number;

	first_period: {
		projection_raw: number;
		projection_final: number;

		pace_1p: number;
		suppressor_1p: number;
		accelerant_1p: number;

		goalie_confidence: "HIGH" | "MEDIUM" | "LOW";
		environment_tag: "UNDER_1P" | "NEUTRAL_1P" | "OVER_1P";

		fair_over_1_5_prob: number | null;
		fair_under_1_5_prob: number | null;

		market_line_ref: 1.5;
		market_price_over?: number | null;
		market_price_under?: number | null;

		classification:
			| "BEST_OVER"
			| "PLAY_OVER"
			| "LEAN_OVER"
			| "PASS"
			| "LEAN_UNDER"
			| "PLAY_UNDER"
			| "BEST_UNDER";
		reason_codes: string[];
	};
};
```

## Invariants

The rebuilt 1P layer must obey:
- 1P cannot be a pure percentage transform
- 1P must include first-period environment components
- no fixed clamp that structurally forces over bias
- projection delta alone cannot decide playable status
- dead zone around 1.5 is mandatory
- goalie uncertainty reduces confidence and caps actionability
- when no meaningful signal exists, return PASS

## Implementation Plan

### Step 1 — Replace fixed 30%

Remove:

```text
rawExpected1pTotal = (homeGoals + awayGoals) * 0.30
```

Use starter production formula:

```text
raw_1p = 0.18 + 0.275 * full_game_total_model + first_period_env_adjustment
```

Where:

```text
first_period_env_adjustment = pace_1p + special_teams_1p + goalie_1p + rest_1p
```

Component caps:

```text
pace_1p:          ±0.12
special_teams_1p: ±0.08
goalie_1p:        ±0.10
rest_1p:          ±0.05
sum_adjustments:  ±0.18
```

### Step 2 — Replace clamp regime

Remove:

```text
clamp(rawExpected1pTotal, 1.4, 2.4)
```

Use safety rail only:

```text
projection_final = clamp(raw_1p, 1.20, 2.25)
```

Emit flags when rail triggers:
- `NHL_1P_CLAMP_LOW`
- `NHL_1P_CLAMP_HIGH`

### Step 3 — Build explicit 1P environment model

Include dedicated first-period components:
- base_1p from full-game total (weaker transform)
- early pace modifier
- suppressor stack
- accelerant stack

Explicitly exclude non-1P dynamics:
- empty-net effects
- late-game tails
- comeback/endgame dynamics

### Step 4 — Add goalie confidence scaling

Use certainty multiplier:

```text
CONFIRMED = 1.0
EXPECTED  = 0.6
UNKNOWN   = 0.0
```

Apply:

```text
applied_goalie_1p_delta = raw_goalie_1p_delta * certainty_multiplier
```

Policy:
- if either goalie is `UNKNOWN`, classification max = `PASS` or `LEAN`
- if both confirmed, normal banding allowed

### Step 5 — Replace raw-delta status with classification bands

Keep delta as telemetry only:

```text
projection_delta = proj_1p - 1.5
```

Use exact production table:

| Projection  | Classification |
| ----------- | -------------- |
| `<= 1.42`   | BEST_UNDER     |
| `1.43–1.50` | PLAY_UNDER     |
| `1.51–1.58` | LEAN_UNDER     |
| `1.59–1.99` | PASS           |
| `2.00–2.14` | LEAN_OVER      |
| `2.15–2.24` | PLAY_OVER      |
| `>= 2.25`   | BEST_OVER      |

### Step 6 — Optional pricing phase (Phase 2)

Phase 1 ships projection + classification only.

Phase 2 adds fair probabilities:

```text
mu = projection_final
sigma_1p = 1.22 to 1.30

P(over 1.5)  = 1 - CDF(1.5, mu, sigma_1p)
P(under 1.5) = CDF(1.5, mu, sigma_1p)
```

### Step 7 — Pass-first UI + reason codes

Add and wire reason codes:
- `NHL_1P_PASS_DEAD_ZONE`
- `NHL_1P_OVER_LEAN`
- `NHL_1P_OVER_PLAY`
- `NHL_1P_OVER_BEST`
- `NHL_1P_UNDER_LEAN`
- `NHL_1P_UNDER_PLAY`
- `NHL_1P_UNDER_BEST`
- `NHL_1P_GOALIE_UNCERTAIN`
- `NHL_1P_CLAMP_LOW`
- `NHL_1P_CLAMP_HIGH`
- `NHL_1P_MODEL_HOT_CAP`

If classification is PASS, UI/action path must show PASS (no implied lean language).

### Step 8 — Exact starter formula block (production-safe)

```text
base_1p = 0.18 + 0.275 * full_game_total_model

pace_1p_adj =
	clamp(first_period_pace_score, -0.12, 0.12)

special_teams_1p_adj =
	clamp(first_period_penalty_pressure_score, -0.08, 0.08)

goalie_1p_adj =
	clamp(raw_goalie_1p_delta * goalie_certainty_multiplier, -0.10, 0.10)

rest_1p_adj =
	clamp(first_period_rest_delta, -0.05, 0.05)

total_adj =
	clamp(
		pace_1p_adj + special_teams_1p_adj + goalie_1p_adj + rest_1p_adj,
		-0.18,
		0.18
	)

raw_1p_projection =
	base_1p + total_adj

final_1p_projection =
	clamp(raw_1p_projection, 1.20, 2.25)
```

Classification policy:

```text
if either goalie unknown:
	max classification = PASS

else if final_1p_projection <= 1.42:
	BEST_UNDER
else if final_1p_projection <= 1.50:
	PLAY_UNDER
else if final_1p_projection <= 1.58:
	LEAN_UNDER
else if final_1p_projection < 2.00:
	PASS
else if final_1p_projection < 2.15:
	LEAN_OVER
else if final_1p_projection < 2.25:
	PLAY_OVER
else:
	BEST_OVER
```

## Failure Modes & Guards

### Failure mode: too many overs still

Cause:
- full-game totals too hot
- 1P modifiers too generous

Guard:
- apply `NHL_1P_MODEL_HOT_CAP` when full-game model is materially above market
- require band-qualified classifications independent of raw delta

### Failure mode: no unders ever

Cause:
- base too high
- floor too high
- uncertainty bias leaking positive

Guard:
- keep floor at `1.20`
- keep unknown goalie multiplier at `0.0`
- monitor 100+ game distribution for under-band occupancy

### Failure mode: everything is PASS

Cause:
- coefficients too conservative

Guard:
- tune `0.275` gradually with backtest monitoring
