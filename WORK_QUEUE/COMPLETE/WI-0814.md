# WI-0814: Sigma fallback safety gate — downgrade PLAY to LEAN when sigma_source=fallback

| Field | Value |
|---|---|
| **ID** | WI-0814 |
| **Goal** | Prevent false PLAY classifications at season start or after a DB reset by downgrading every qualifying card from PLAY to LEAN whenever `computeSigmaFromHistory` returns `sigma_source='fallback'`. Emit a `SIGMA_FALLBACK_DEGRADED` reason code and a startup warning so the condition is observable. |
| **Priority** | High at season boundaries (first ~20 settled games); Medium in steady state |
| **Depends on** | None |
| **Coordination flag** | None |

---

## Background

`computeSigmaFromHistory` in `packages/models/src/edge-calculator.js` returns `sigma_source: 'fallback'` when fewer than 20 settled games exist for the sport:

```js
if (!rows || rows.length < 20) {
  return fallback;  // { margin: 12, total: 14, sigma_source: 'fallback' }
}
```

The fallback constants are [explicitly documented as uncalibrated](packages/models/src/edge-calculator.js):

```js
// NBA margin=12 (set ~2024, uncalibrated — no lineage in codebase)
// NBA total=14  (set ~2024, uncalibrated — no lineage in codebase)
```

A 1-point sigma error (~8%) shifts spread edge by ~1.5% at typical lines (±6 points), which can push a card from LEAN (edge=0.04) to PLAY (edge≥0.07) entirely due to bad sigma. At season start every game runs on fallback sigma, so every PLAY generated in the first week may be a false positive.

**Safe behavior:** Under fallback sigma, treat all qualifying output as LEAN (not PLAY) until 20+ settled games accumulate. Once sigma is empirically computed this gate releases automatically.

## Detailed Implementation

### Task 1 — Add `sigmaSource` parameter to `resolveThresholdProfile`

**File:** `packages/models/src/decision-pipeline-v2-edge-config.js`

```js
// BEFORE
function resolveThresholdProfile({ sport, marketType }) {
  ...
  return profile;
}

// AFTER
function resolveThresholdProfile({ sport, marketType, sigmaSource = null }) {
  ...
  // Sigma fallback gate: if sigma is uncalibrated, treat every card as LEAN
  // by raising play_edge_min to match lean_edge_min.
  if (sigmaSource === 'fallback') {
    profile.edge = {
      ...profile.edge,
      play_edge_min: profile.edge.lean_edge_min,
    };
    profile.meta = {
      ...(profile.meta ?? {}),
      sigma_degraded: true,
      sigma_degraded_reason: 'SIGMA_FALLBACK_DEGRADED',
    };
  }

  return profile;
}
```

All existing call sites pass no `sigmaSource` → default `null` → behavior unchanged.

### Task 2 — Pass `sigmaSource` from model runner context

**File:** `apps/worker/src/jobs/run_nba_model.js`

The job already computes:

```js
const computedSigma = edgeCalculator.computeSigmaFromHistory({
  sport: 'NBA',
  db: getDatabase(),
});
console.log('[run_nba_model] sigma:', JSON.stringify(computedSigma));
```

Add a startup warning when fallback:

```js
if (computedSigma.sigma_source === 'fallback') {
  console.warn(
    '[run_nba_model] [SIGMA_FALLBACK] Fewer than 20 settled games — using uncalibrated sigma defaults. ' +
    'All PLAY cards will be downgraded to LEAN until empirical sigma is available.',
  );
}
```

Pass `sigmaSource` through to the decision pipeline context wherever `buildDecisionV2` or `resolveThresholdProfile` is called for this run. The exact wiring depends on how `sigmaOverride` is currently threaded (see `publishDecisionForCard` options). Add `sigmaSource: computedSigma.sigma_source` to that context object.

Apply the same change to `run_nhl_model.js` and `run_mlb_model.js`.

### Task 3 — Emit `SIGMA_FALLBACK_DEGRADED` reason code on affected cards

**File:** `packages/models/src/decision-pipeline-v2.js`

When `sigma_degraded: true` is present in the resolved threshold profile and the computed official_status would have been `PLAY`, emit `SIGMA_FALLBACK_DEGRADED` as an additional entry in `reason_codes`. This makes the condition visible in card payloads and queryable in the DB.

## Acceptance Criteria

- [ ] When `sigma_source='fallback'`, `resolveThresholdProfile` returns a profile where `play_edge_min === lean_edge_min`
- [ ] When `sigma_source='computed'` (or null), `resolveThresholdProfile` behavior is identical to before this WI
- [ ] A card with edge=0.07 (above normal PLAY threshold) generated under fallback sigma produces `official_status='LEAN'` not `'PLAY'`
- [ ] That same card's `reason_codes` includes `'SIGMA_FALLBACK_DEGRADED'`
- [ ] A `console.warn` with tag `[SIGMA_FALLBACK]` appears in model runner output when sigma_source=fallback
- [ ] With ≥20 settled games (sigma_source=computed), PLAY classification is unaffected
- [ ] All existing decision-pipeline and threshold tests pass
