# WI-0490: Precise no-play failure taxonomy for root-cause diagnosis

**ID**: WI-0490
**Goal**: Replace overly generic no-play/missing-data failures with precise root-cause labels so operators can distinguish team mapping, projection input, driver, and true no-edge outcomes.

**Acceptance**:

1. `/api/games` exposes enough additive metadata to distinguish missing team mapping from missing projection inputs when a game has zero playable cards.
2. `web/src/lib/game-card/transform.ts` no longer collapses those states into generic `MISSING_DATA_NO_PLAYS` / `PASS_NO_ACTIONABLE_PLAY` when a more specific root cause is available.
3. Cards diagnostics buckets group missing team mapping separately from missing projection inputs, preserving existing no-odds/driver-failure behavior.
4. Existing evidence-only/no-play contract remains intact for true no-edge/no-actionable cases.
