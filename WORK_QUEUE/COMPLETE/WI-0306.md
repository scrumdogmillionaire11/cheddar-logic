# WI-0306: Implement Card Display Ledger for Settlement Accuracy

CLAIM: github-copilot 2026-03-05T16:00:00Z

**ID**: WI-0306

**Goal**: Ensure results settlement only includes plays that were actually displayed on `/cards`, eliminating phantom settlements.

**Scope**:

- `packages/data/db/migrations/019_create_card_display_log.sql` (new table)
- `packages/data/src/db-multi.js` (logCardDisplay, getDisplayedPickIds, getSettlementLedger)
- `apps/worker/src/jobs/settle_pending_cards.js` (rewrite settlement logic)
- `web/src/app/api/cards/route.ts` (log displayed picks after SELECT)
- `web/src/app/api/games/route.ts` (log displayed picks after transform)
- `web/src/app/api/results/route.ts` (filter by displayed ledger)

**Out of scope**:

- Props settlement (separate future WI)
- Audit logging (display_log is RO SoT, not audit trail)
- Historical backfill (start fresh; past orphaned plays stay as-is)
- Multi-sport handling (single table, all sports)

**Acceptance**:

- `card_display_log` table exists: columns `id, pick_id, run_id, game_id, sport, market_type, selection, line, odds, odds_book, confidence_pct, displayed_at, api_endpoint`.
- Every returned play from `/api/cards` writes a row to `card_display_log` before response sent.
- Every returned play from `/api/games` writes a row to `card_display_log` before response sent.
- Settlement reads ONLY from `card_display_log` (not orphaned logic); exactly one row per settled pick.
- Results API ledger rows match displayed ledger; no "most confident per game" suppression.
- No phantom settlements: `COUNT(card_results WHERE result IS NOT NULL) <= COUNT(card_display_log)` for any date range.

**Owner agent**: github-copilot

**Time window**: 2026-03-05T18:00:00Z - 2026-03-06T14:00:00Z

**Coordination flag**: needs-sync (settlement job outputs, `/api/results`)

**Tests to run**:

- `npm --prefix apps/worker run test -- __tests__/settle_pending_cards.phase2.test.js`
- New test: `npm --prefix apps/worker run test -- __tests__/settle_display_ledger.test.js` (see WI-0306-TEST-SPIKE.md)
- Manual: seed 20 displayed plays via mock `/api/games` response, settle, verify exactly 20 rows settled.

**Manual validation**:

- Display log populated: `SELECT COUNT(*) FROM card_display_log WHERE displayed_at >= datetime('now', '-1 day');` (should be > 0).
- Settlement count <= display count: `SELECT COUNT(*) FROM card_results WHERE result IS NOT NULL;` vs display log count.
- Results page filters work: `GET /api/results?sport=NBA` returns only NBA rows from display ledger.
- Full Slate (PASS included): PASS plays show in results as "graded" (settled status), not "orphaned."
