# WI-0526: NHL Props Price Wiring + Blocked Shots Market Expansion

**ID:** WI-0526
**Goal:** Wire real SOG prop prices into NHL shots decisioning (Phase 1), then add blocked-shots ingestion (Phase 2).
**Owner agent:** copilot

**Branch:** agent/copilot/WI-0526-nhl-props-price-wiring

## Context

`pull_nhl_player_shots_props.js` already fetches and stores `over_price`/`under_price` for `player_shots_on_goal` from DraftKings, FanDuel, and BetMGM. `getPlayerPropLine()` in the data package already SELECTs them. But `run_nhl_player_shots_model.js` only reads `.line` — prices are never read, never passed to `projectSogV2`, and therefore never used in decisioning.

`projectSogV2` (WI-0525) is fully implemented and tested (25 tests green) but is not imported in the model job at all. Its Stage 2 pricing layer — `edge_over_pp`, `ev_over`, `opportunity_score` — is dormant.

Current state:

| Market        | Pulled | Priced | Used by model |
| ------------- | ------:| ------:| -------------:|
| Shots on goal |    Yes |    Yes |        **No** |
| Blocked shots |     No |     No |            No |

## Phase 1: Wire SOG Prices

### Scope

- `apps/worker/src/jobs/run_nhl_player_shots_model.js`
- `apps/worker/src/models/__tests__/nhl-sog-price-wiring.test.js` (new)

### Out of scope

- `player_blocked_shots` market (Phase 2)
- Changes to `pull_nhl_player_shots_props.js`
- Changes to `nhl-player-shots.js` model functions
- Decisioning threshold changes (FIRE/HOLD/PASS gate values unchanged)

### Acceptance (Phase 1)

- [x] `run_nhl_player_shots_model.js` imports and calls `projectSogV2`
- [x] `over_price` and `under_price` are read from `realPropLine` (already returned by `getPlayerPropLine`)
- [x] `projectSogV2` is called with those prices and the existing shot stats
- [x] Card payload `decision` object includes `v2.edge_over_pp`, `v2.ev_over`, `v2.opportunity_score`, `v2.flags`
- [x] `odds_backed: true` set when both prices present; `odds_backed: false` when prices missing
- [x] `opportunity_score` surfaced at `decision.opportunity_score` for ranking
- [x] `MISSING_PRICE` flag propagated into the card when line exists but prices absent
- [x] Tests prove priced SOG props have non-null `opportunity_score`; projection-only props have `opportunity_score: null`
