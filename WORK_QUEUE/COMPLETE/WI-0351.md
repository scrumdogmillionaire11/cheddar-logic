## Implementation Summary

### Phase 1: Daily Team Metrics Cache (COMPLETED)

**Commit**: `f4a8388` - "WI-0351: Add daily team metrics cache with DB persistence"

#### What was implemented

1. **Database Migration** (025_create_team_metrics_cache.sql):
   - Table: `team_metrics_cache` with columns: `id`, `sport`, `team_name`, `cache_date`, `status`, `metrics`, `team_info`, `recent_games`, `resolution`, `fetched_at`, `created_at`
   - Indexes: Composite unique on (sport, team_name, cache_date), plus indexes on cache_date and status
   - Sport CHECK constraint aligned with existing patterns

2. **DB Helpers** (packages/data/src/db.js):
   - `getTeamMetricsCache(sport, teamName, cacheDate)` - Read cache entry
   - `upsertTeamMetricsCache(cacheEntry)` - Write/update cache entry
   - `deleteStaleTeamMetricsCache(beforeDate)` - Clean up old entries
   - All functions exported via module.exports

3. **Cache Integration** (packages/data/src/team-metrics.js):
   - Modified `getTeamMetricsWithGames()` to check cache first (skipCache option for testing)
   - Best-effort cache read: Falls through to live fetch on cache error
   - Best-effort cache write: Logs warning on write failure but returns successful fetch result
   - Only caches 'ok' status (failed fetches retry on next model run)

4. **Prewarm Job** (apps/worker/src/jobs/refresh_team_metrics_daily.js):
   - Fetches ESPN metrics for all active teams: NBA (30), NHL (32), NCAAM (40)
   - Scheduled at 09:00 ET (before first model run)
   - Job key: `refresh_team_metrics|{cache_date}` (daily idempotency)
   - Cleanup: Deletes cache entries older than 7 days
   - CLI: `node src/jobs/refresh_team_metrics_daily.js [--dry-run] [--sport=NBA]`

5. **Scheduler Integration** (apps/worker/src/schedulers/main.js):
   - Added job to computeDueJobs at 09:00 ET fixed window (section 2.5, before models section 3)
   - Environment variable: `ENABLE_TEAM_METRICS_CACHE` (default: true)

#### Cache Strategy

- **Key**: sport + team_name + cache_date (ET timezone)
- **TTL**: 1 day (overwritten at next 09:00 ET refresh)
- **Retry**: Only cache 'ok' status; failed fetches retry on next model run
- **Cleanup**: Entries older than 7 days are deleted
- **Performance**: Reduces ESPN API calls during model runs (~102 teams × multiple runs/day)
