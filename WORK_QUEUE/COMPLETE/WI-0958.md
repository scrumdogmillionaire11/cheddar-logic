# WI-0958: Fix `Missing inputs: [object Object]` Rendering Bug in NBA Cards

**ID**: WI-0958
**Goal**: Ensure `missing_inputs` entries on NBA (and any sport) cards render as human-readable strings in the UI, not as the browser default `[object Object]` stringification.

**Depends on**: none

**Scope**:

- `web/src/lib/game-card/transform/index.ts`
- `web/src/components/cards/shared.ts`
- `web/src/__tests__/game-card-decision-authority.test.ts`
- `WORK_QUEUE/WI-0958.md`

**Out of scope**:

- Changes to the worker model or card payload shape in `apps/worker/`
- DB schema changes
- Changes to the `missing_inputs` array structure emitted by the worker
- NBA projection model formula changes

**Acceptance**:

- NBA cards with non-empty `missing_inputs` display each entry as a readable label (e.g., `missing data team mapping`, `team_metrics_cache absent`) rather than `[object Object]`.
- If `missing_inputs` entries are objects, the transform/display layer extracts a consistent string representation (e.g., `.reason`, `.code`, `.label`, or `JSON.stringify` as a last resort) rather than implicitly calling `.toString()`.
- String-valued `missing_inputs` entries are unaffected and continue to render correctly.
- No regression to non-NBA sports with existing missing_inputs rendering.

**Root cause observed**:

- The NBA card for `GOLDEN STATE WARRIORS @ LOS ANGELES CLIPPERS` displayed `Missing inputs: [object Object]` in production.
- Worker emits `missing_inputs` as an array. Recent changes to team-metrics enrichment (WI-0949 area) may have introduced object-shaped entries. The web transform passes the array directly to the template which calls `.toString()` on each element, producing `[object Object]` for non-string entries.
