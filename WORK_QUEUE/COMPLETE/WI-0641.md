# WI-0641: Player-props scheduler — extract NHL SOG/BLK and MLB pitcher-K prop jobs into dedicated scheduler at 2× daily + T-60

**ID**: WI-0641
**Goal**: Move all player-prop ingest and model jobs for NHL (SOG, BLK) and MLB (pitcher Ks) out of `main.js` into a dedicated `schedulers/player-props.js` that fires at most twice daily (09:00 ET, 18:00 ET) plus T-60 per game — eliminating redundant runs at T-120, T-90, T-30 and the 12:00 ET fixed window.

**Background**:

`main.js` queues player-prop jobs at every fixed window and every T-minus window per game:

- **NHL**: `pull_nhl_player_shots_props` + `run_nhl_player_shots_model` at 09:00/12:00/18:00 ET and T-120/T-90/T-60/T-30 per game
- **MLB**: `pull_mlb_pitcher_stats` + `pull_mlb_weather` + `pull_mlb_pitcher_strikeout_props` + `run_mlb_model` at T-120/T-90/T-60/T-30 per game

Player prop lines don't move on that cadence. NST rate data is daily, pitcher stats are daily, weather settles hours before first pitch, and prop lines are stable until ~60 min before game time. Running the full chain at T-120/T-90/T-30 is pure waste and burns Odds API quota. The NHL BLK ingest jobs (`sync_nhl_blk_player_ids`, `pull_nhl_player_blk`, `ingest_nst_blk_rates`) aren't wired in at all yet — they need a home and `main.js` is the wrong one.

**New cadence**:

| Window | NHL jobs | MLB jobs |
| --- | --- | --- |
| 09:00 ET daily | `sync_nhl_sog_player_ids`, `sync_nhl_blk_player_ids`, `pull_nhl_player_blk`, `ingest_nst_blk_rates`, `pull_nhl_player_shots_props`, `run_nhl_player_shots_model` | `pull_mlb_pitcher_stats`, `pull_mlb_weather`, `pull_mlb_pitcher_strikeout_props`, `run_mlb_model` |
| 18:00 ET daily | `pull_nhl_player_shots_props`, `run_nhl_player_shots_model` | `pull_mlb_pitcher_strikeout_props`, `run_mlb_model` |
| T-60 per game | `pull_nhl_player_shots_props`, `run_nhl_player_shots_model` | `pull_mlb_pitcher_strikeout_props`, `run_mlb_model` |

Heavy ingest (NST rates, BLK logs, pitcher stats, weather, player sync) only at 09:00 — none of it changes intraday. The 18:00 and T-60 windows refresh prop lines and regenerate cards with latest odds only.

**Scope**:

- `apps/worker/src/schedulers/player-props.js` (new) — standalone scheduler owning all prop ingest + model jobs for NHL and MLB; own tick loop, own fixed-time and T-60 windows; follows `fpl.js` pattern
- `apps/worker/src/schedulers/main.js` — remove `queueNhlShotsPropIngestBeforeModel`, `queueMlbPitcherStatsBeforeModel`, `queueMlbWeatherBeforeModel`, `queueMlbPitcherKPropIngestBeforeModel` calls and their imports; remove `syncNhlSogPlayerIds` call from the 04:00 daily block (moved to player-props 09:00)
- `apps/worker/src/schedulers/__tests__/player-props.test.js` (new) — unit tests for window due-logic, job queuing at each window per sport, idempotency key format, and feature-flag suppression

**Env vars** (new, all opt-out defaults):

- `ENABLE_PLAYER_PROPS_SCHEDULER` — set `false` to disable entirely (default: `true`)
- `ENABLE_NHL_BLK_INGEST` — set `false` to skip the three BLK ingest jobs at 09:00 (default: `true`)
- `PLAYER_PROPS_FIXED_TIMES_ET` — override fixed-time windows as comma-separated HH:MM (default: `09:00,18:00`)

**Key format**:

```
player_props|nhl|fixed|<YYYY-MM-DD>|<HH:MM>        # NHL fixed windows
player_props|nhl|tminus|<game_id>|T60               # NHL T-60 per game
player_props|nhl_blk_ingest|daily|<YYYY-MM-DD>      # BLK ingest (09:00 only)
player_props|mlb|fixed|<YYYY-MM-DD>|<HH:MM>        # MLB fixed windows
player_props|mlb|tminus|<game_id>|T60               # MLB T-60 per game
```

**Out of scope**:

- SOG 1P prop scheduling (gated behind `NHL_SOG_1P_CARDS_ENABLED`, low-priority)
- Soccer props (already in `main.js` via `queueSoccerPropIngestBeforeModel` — separate WI if needed)
- Any changes to model logic or card generation

**Acceptance**:

- `player-props.js` starts without error and registers its tick loop.
- At 09:00 ET, all NHL ingest jobs queue in order: `sync_nhl_sog_player_ids`, `sync_nhl_blk_player_ids`, `pull_nhl_player_blk`, `ingest_nst_blk_rates`, `pull_nhl_player_shots_props`, `run_nhl_player_shots_model`. MLB ingest jobs also queue: `pull_mlb_pitcher_stats`, `pull_mlb_weather`, `pull_mlb_pitcher_strikeout_props`, `run_mlb_model`.
- At 18:00 ET, only prop-line + model jobs queue (no heavy ingest) for both sports.
- At T-60 per game, only prop-line + model jobs queue. T-120, T-90, T-30 produce no prop jobs for either sport.
- `main.js` no longer imports or queues any of the prop jobs moved here.
- `ENABLE_NHL_BLK_INGEST=false` suppresses the three BLK ingest jobs without affecting SOG prop pull or model run.
- `ENABLE_PLAYER_PROPS_SCHEDULER=false` suppresses the entire scheduler; `main.js` prop behaviour is already removed so this is a hard disable.
- Idempotency keys are deterministic per window — re-running within the same window skips already-succeeded jobs.
- All new tests pass; existing `run_nhl_player_shots_model.test.js` and `run_mlb_model.test.js` pass without regression.
