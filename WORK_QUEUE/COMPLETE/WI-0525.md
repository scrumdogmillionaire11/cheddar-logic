# WI-0525: NHL player shots two-stage model (SOG_mu projection + market value layer)

**ID**: WI-0525
**Goal**: Refactor the NHL player shots model into a clean two-stage system: Stage 1 projects `SOG_mu` from EV/PP shot rates and a clamped multiplier chain; Stage 2 compares that projection against the book using Poisson fair probabilities, edge in probability points, and EV. Trend and value stay separated — trend adjusts `SOG_mu` only; it does not manufacture edge.

**Scope**:

- `apps/worker/src/models/nhl-player-shots.js` — add `projectSogV2(inputs)` export alongside existing exports (do not break existing callers)
- `apps/worker/src/models/__tests__/nhl-player-shots-two-stage.test.js` (new) — unit + smoke tests per spec
- `docs/NHL_PLAYER_SHOTS_PROP_MARKET.md` — append two-stage architecture section (existing content untouched)

**Out of scope**:

- Changes to existing `projectSog()` or any callers of the current shots model
- DB schema or migration changes
- UI card rendering changes (output contract documented only; wiring is a follow-on)
- Pace model or projections orchestration changes unless required to expose `shot_env_factor` inputs

**Acceptance**:

- `projectSogV2(inputs)` returns a `NhlShotsProjection`-shaped object with `sog_mu`, `role_stability`, `trend_score`, `fair_over_prob_by_line`, `fair_under_prob_by_line`, `fair_price_over_by_line`, `fair_price_under_by_line`, and optional edge/EV fields when market price is present.
- Stage 1 computes `sog_mu` via weighted EV+PP shot rate blend (season/L10/L5 at 0.35/0.35/0.30) multiplied through a clamped chain: `shot_env_factor` (0.92–1.08), `opponent_suppression_factor` (0.90–1.10), `goalie_rebound_factor` (0.97–1.03), `trailing_script_factor` (0.95–1.08), `trend_factor` (0.93–1.07, forced to 1.0 when `role_stability` is LOW).
- Stage 2 computes Poisson fair probabilities per available line and derives American fair prices, `edge_over_pp`, `edge_under_pp`, `ev_over`, `ev_under` only when a market price is present.
- Gating: no edge/EV fields emitted when price is missing; `role_stability: LOW` suppresses trend effect; missing goalie / major lineup uncertainty sets a flag and should cap or block output.
- `opportunity_score` uses weighting: 0.45 edge + 0.20 EV + 0.20 (projection−line) + 0.10 trend_score + 0.05 pace_env. Only computed when Gate A (projection quality) and Gate B (price quality) both pass.
- All unit tests and smoke tests listed below pass.

**Unit tests required**:

- `sog_mu` increases when `toi_proj` increases (all else equal)
- `sog_mu` increases when `shot_env_factor` increases (all else equal)
- `trend_factor` cannot exceed 1.07 or go below 0.93
- `trend_factor` equals 1.0 when `role_stability` is LOW (regardless of recent rate)
- Fair over probability is monotonically decreasing as line increases
- Edge and EV calculations are correct for a fixed set of inputs

**Smoke tests required**:

- Stable L1/PP1 player with a qualifying market price → object has `edge_over_pp`, `ev_over`, `opportunity_score`
- Stable player with strong `sog_mu` but price missing → object has `fair_over_prob_by_line` but no `edge_over_pp` or `opportunity_score`
- Hot recent trend with `role_stability: LOW` → `trend_factor` equals 1.0, player does not rank ahead of stable players
- `market_price_over` absent → `edge_over_pp` and `ev_over` are undefined/absent

**Reference spec** (design decisions locked):

```
NhlShotsProjection shape:
  player_id, game_id, sog_mu, sog_sigma?, toi_proj,
  shot_rate_ev_per60, shot_rate_pp_per60, shot_env_factor,
  role_stability: "HIGH"|"MEDIUM"|"LOW", trend_score,
  fair_over_prob_by_line: Record<string,number>,
  fair_under_prob_by_line: Record<string,number>,
  fair_price_over_by_line: Record<string,number>,
  fair_price_under_by_line: Record<string,number>,
  market_line?, market_price_over?, market_price_under?,
  edge_over_pp?, edge_under_pp?, ev_over?, ev_under?,
  opportunity_score?, flags: string[]

Invariants:
- Projection and price are separate layers
- trend_factor adjusts SOG_mu only; it must not directly manufacture edge
- No edge/EV output if price is missing
- No official play (opportunity_score) if role_stability is LOW
- Unknown goalie / major lineup uncertainty → flag + cap or block
```
