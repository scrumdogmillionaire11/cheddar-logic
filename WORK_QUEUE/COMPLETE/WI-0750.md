# WI-0750: Always emit mlb-f5 PROJECTION_ONLY card — populate Game Props tab

**ID:** WI-0750
**Owner agent:** copilot
**Time window:** 2026-04-04
**Coordination flag:** none

## Goal

Write an `mlb-f5` card with `execution_status=PROJECTION_ONLY` for every MLB game
where a projection can be computed, regardless of whether live F5 odds are available.
This is the MLB equivalent of `nhl-pace-1p` — a guaranteed Game Props projection surface.

Currently the `projectionFloorDriver` is only synthesized when `withoutOddsMode=true`.
Since `ENABLE_WITHOUT_ODDS_MODE` is `false` in normal operation, no `mlb-f5` card is
ever written to `card_payloads`, leaving the Game Props tab empty for all MLB games.

## Root cause

In `run_mlb_model.js` (~line 1590–1640), the F5 projection floor logic is:

```javascript
const projectionFloorF5 = (withoutOddsMode && f5TotalContextForFloor.line === null)
  ? computeProjectionFloorF5(gameOddsSnapshot)
  : null;
```

And later (~line 1640), the `projectionFloorDriver` is gated on `withoutOddsMode`:

```javascript
const projectionFloorDriver = (withoutOddsMode && marketAvailability.projection_floor && projectionFloorF5 !== null)
  ? { market: 'f5_total', ... }
  : null;
```

When odds ARE present (normal mode), `withoutOddsMode=false`, so `projectionFloorF5=null`
and `projectionFloorDriver=null`. No `mlb-f5` card is ever written. The `candidateDrivers`
array never includes an F5 projection entry, `qualified` stays empty for the F5 market, and
`insertCardPayload` is never called with `card_type='mlb-f5'`.

## Fix

Two changes in `run_mlb_model.js`:

**Change 1** — compute `projectionFloorF5` unconditionally when no live F5 line exists:

```javascript
// Before
const projectionFloorF5 = (withoutOddsMode && f5TotalContextForFloor.line === null)
  ? computeProjectionFloorF5(gameOddsSnapshot)
  : null;

// After
// Always compute projection floor F5 when no live line is available
// so the Game Props tab has content regardless of odds mode.
const projectionFloorF5 = f5TotalContextForFloor.line === null
  ? computeProjectionFloorF5(gameOddsSnapshot)
  : null;
```

**Change 2** — synthesize the `projectionFloorDriver` unconditionally when floor exists:

```javascript
// Before
const projectionFloorDriver = (withoutOddsMode && marketAvailability.projection_floor && projectionFloorF5 !== null)

// After
// Emit projection-only F5 card whenever a floor projection exists, even in normal
// odds mode — this is the Game Props tab's data source (same as nhl-pace-1p).
const projectionFloorDriver = (projectionFloorF5 !== null)
```

The `projectionFloorDriver` object already sets `without_odds_mode: true` and
`projection_source: 'SYNTHETIC_FALLBACK'` — ensure those remain only when
`withoutOddsMode` is true. When the driver is emitted in normal mode (odds present but
no F5 line), use `projection_source: 'PROJECTION_ONLY'` and omit `without_odds_mode`.

**Change 3** — `buildMlbMarketAvailability` currently only sets `projection_floor: true`
when `withoutOddsMode && useFloor`. Add a distinct flag so the pipeline state correctly
reflects the projection-only path when it's triggered outside of `withoutOddsMode`:

No change needed here — the `candidateDrivers` array already includes
`projectionFloorDriver` unconditionally; the flag is informational only.

## Scope

- `apps/worker/src/jobs/run_mlb_model.js` — the two change points above

## Out of scope

- `withoutOddsMode` mode behaviour (no change when `ENABLE_WITHOUT_ODDS_MODE=true`)
- MLB full-game total or moneyline cards — only `mlb-f5` projection floor
- Web/route-handler changes (Game Props tab already handles PROJECTION_ONLY via the
  route-handler fix in commit `abd1a64`)
- Any DB migration

## Acceptance

1. `CHEDDAR_DB_ALLOW_MULTI_PROCESS=true node apps/worker/src/jobs/run_mlb_model.js`
   exits 0 and logs `card_type=mlb-f5` writes for games where F5 odds are absent.
2. DB query confirms `mlb-f5` rows in `card_payloads` after the run (should return > 0):

   ```sql
   SELECT COUNT(*) FROM card_payloads WHERE card_type='mlb-f5'
     AND created_at >= datetime('now', '-1 hour');
   ```

3. `GET /api/games` response includes games with `plays` containing
   `cardType='nhl-pace-1p'` AND equivalent `mlb-f5` plays.
4. Game Props tab in the UI renders MLB F5 projection cards alongside NHL 1P cards.
5. `execution_status` on the emitted cards is `PROJECTION_ONLY`.
6. `projection_source` is `PROJECTION_ONLY` (not `SYNTHETIC_FALLBACK`) when odds
   ARE present but just lack an F5 line.
