## Problem

`refresh_team_metrics_daily.js` fetches and caches per-team `avgPoints`, `avgPointsAllowed`, and `netRating` for every NBA team from ESPN daily. The NBA model (`run_nba_model.js`) has **zero references** to `team_metrics_cache`.

The NBA model's only sport-specific DB query touches the `games` table for road-trip sequence logic. It does not read team offensive or defensive context.

**What the model is missing:**
- Scoring offense and defense per team (raw pace and efficiency)
- Home/away scoring differential context
- Any efficiency-based total adjustment (current: total is derived from line movement only)

**Contract violation:** `refresh_team_metrics_daily.js` runs, writes data, proves trustworthy ‚Äî and the model contract says: if data is fetched and trustworthy, the current model must either use it or surface why it is ignored. Neither happens.

---

## Input proof

`team_metrics_cache` schema (JSON blob per sport/team/date):

```
avgPoints        ‚Äî team's rolling offensive scoring average
avgPointsAllowed ‚Äî team's rolling defensive scoring average
netRating        ‚Äî offensive rating minus defensive rating
```

Available via `getTeamMetricsWithGames(sport, teamId)` in `packages/data/src/db/team-metrics.js`.

---

## Fix spec

In `apps/worker/src/jobs/run_nba_model.js` game loop:

1. After game record is loaded, call `getTeamMetricsWithGames('nba', homeTeamId)` and `getTeamMetricsWithGames('nba', awayTeamId)` for both sides
2. Compute `impliedTotal = (homeAvgPoints + awayAvgPointsAllowed + awayAvgPoints + homeAvgPointsAllowed) / 2` as a pace-anchor total
3. Blend `impliedTotal` with the current odds-derived `marketTotal` using a configurable weight (e.g. `teamContextWeight = 0.25`)
4. If both teams' cache entries are missing or stale (> 3 days), emit `missing_inputs: ['nba_team_context']` and cap classification at `LEAN` (not `EXECUTABLE`)

**Output proof:** The adjusted total appears in `raw_data.pace_anchor_total` and is reflected in the `proj_total` used by edge calculation.

**Failure proof:** When `team_metrics_cache` entries are absent, `missing_inputs` is populated and classification is downgraded ‚Äî no silent fallback to odds-only.
