# WI-0871: Fix F5 ML Formula

**ID**: WI-0871
**Goal**: Replace ERA-arithmetic in projectF5ML() with per-team run projections from projectTeamF5RunsAgainstStarter(), aligning F5 ML edge with F5 total edge.

**Scope**:
- apps/worker/src/models/mlb-model.js (projectF5ML function only)
- apps/worker/src/models/__tests__/mlb-model.test.js (4 new tests)
- apps/worker/src/jobs/run_mlb_model.js (line ~1950: pass homeOffenseProfile + awayOffenseProfile from mlb.home_offense_profile / mlb.away_offense_profile; no structural change)

**Out of scope**: full-game ML (WI-0873), full-game total (WI-0872), DB migrations, new card types

**Background**:
projectF5ML() computes expected F5 runs as (era+4.5)/2*(5/9) -- ignoring park, weather, offense, leash, TTO. F5 total and F5 ML use incompatible math. data/mlb-research.md recommends a unified run distribution path.

**Implementation notes**:
New signature: `projectF5ML(homePitcher, awayPitcher, mlF5Home, mlF5Away, homeOffenseProfile, awayOffenseProfile, context)`

- `homeOffenseProfile`, `awayOffenseProfile`, `context` are **optional** (default null).
- When either offenseProfile is null: falls back to ERA formula, sets `projection_source: 'F5_ML_FALLBACK_ERA'` and `reason_codes: ['F5_ML_FALLBACK_ERA']`.
- When both offenseProfiles are present: calls `projectTeamF5RunsAgainstStarter(awayPitcher, homeOffenseProfile, context)` for homeRunsResult and `projectTeamF5RunsAgainstStarter(homePitcher, awayOffenseProfile, context)` for awayRunsResult. If either returns `f5_runs: null`, falls back to ERA formula.
- Confidence derivation (new path): use 7 as the fixed starting confidence; subtract 1 for each side whose result has non-empty `degraded_inputs` (floor 5). Do not inherit `proj.confidence` from the ERA path -- the new code path never calls `projectF5Total`. ERA fallback path keeps its existing confidence derivation unchanged.
- The existing 4-arg call in run_mlb_model.js continues to work unchanged because new params default to null (ERA fallback fires). Update the call site to also pass `mlb.home_offense_profile ?? null`, `mlb.away_offense_profile ?? null`, and `{ park_run_factor: mlb.park_run_factor ?? null, weather: mlb.weather ?? null }` so the full model path is reachable in production.
- Existing 10+ tests in `run_mlb_model.test.js` (lines 1629–1700) use the 4-arg signature and must continue to pass without modification (backward compat via null defaults).

**Acceptance**:
1. projectF5ML() reads f5_runs from projectTeamF5RunsAgainstStarter() for both sides when offenseProfiles are provided
2. winProbHome logistic coefficient (0.8) preserved; only run mean inputs change
3. f5_runs null for either side (or offenseProfile null): falls back to ERA formula + `projection_source: 'F5_ML_FALLBACK_ERA'`
4. Test: same pitcher/offense inputs yield identical f5_runs in F5 total and F5 ML paths
5. Test: elite-offense vs poor pitcher yields higher f5_runs than avg-offense vs avg pitcher
6. Existing run_mlb_model.test.js describe block `projectF5ML — F5 ML side projection (WI-0603)` (10 tests, 4-arg signature) passes without modification
7. Confidence is 7 when both offenseProfiles present and degraded_inputs is empty; drops to ≤7 when degraded_inputs non-empty; ERA-fallback path confidence unchanged
8. npm --prefix apps/worker test --no-coverage passes, zero regressions
