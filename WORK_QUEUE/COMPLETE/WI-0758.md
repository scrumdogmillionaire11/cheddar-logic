# WI-0758: Actual result ingestion for player prop cards (nhl-player-shots, nhl-player-blk, mlb-pitcher-k)

**ID**: WI-0758
**Goal**: After games complete, write per-player actual stat results back into `card_payloads` for the three existing player prop projection card types (`nhl-player-shots`, `nhl-player-shots-1p`, `nhl-player-blk`, `mlb-pitcher-k`) so projection accuracy can be tracked.

**Background**:
WI-0757 introduced the `actual_result` column and settlement infrastructure for game-level projection cards (`nhl-pace-1p`, `mlb-f5`). This WI extends that same mechanism to player-level prop cards using the same column тАФ no new migration is needed.

Card types and expected `actual_result` shapes:
- `nhl-player-shots`: full-game shots on goal for the player тЖТ `{"shots": N}`
- `nhl-player-shots-1p`: 1st-period shots on goal for the player тЖТ `{"shots_1p": N}`
- `nhl-player-blk`: full-game blocked shots for the player тЖТ `{"blocks": N}`
- `mlb-pitcher-k`: game strikeout total for the starting pitcher тЖТ `{"pitcher_ks": N}`

The player ID is already stored in each card's `payload_data` (as `player_id` or equivalent). The actual stats are available from the same NHL/MLB stat sources used by `pull_nhl_player_shots.js`, `pull_nhl_player_blk.js`, and the MLB score ingestion path in `run_mlb_model.js`.

**Acceptance**:
- `getUnsettledProjectionCards()` returns cards where `card_type IN ('nhl-pace-1p', 'mlb-f5', 'nhl-player-shots', 'nhl-player-shots-1p', 'nhl-player-blk', 'mlb-pitcher-k')` and `actual_result IS NULL` and game is finalized
- After an NHL game completes, `actual_result` is set to `{"shots": N}` for matching `nhl-player-shots` cards and `{"shots_1p": N}` for `nhl-player-shots-1p` cards
- After an NHL game completes, `actual_result` is set to `{"blocks": N}` for matching `nhl-player-blk` cards
- After an MLB game completes, `actual_result` is set to `{"pitcher_ks": N}` for matching `mlb-pitcher-k` cards
- Player ID from `payload_data` is used to look up the correct player's stat line тАФ no cross-player contamination
- Single-writer contract maintained: only the worker writes `actual_result`
