# WI-0754: nhl-pace-1p projection card — render fixed 1.5 line with OVER/UNDER direction badge

**ID**: WI-0754
**Goal**: Replace the blank "no odds" zone on `nhl-pace-1p` cards with a dedicated projection display showing the fixed 1.5 line, the model's projected 1P total, and a derived OVER/UNDER direction badge.

**Background**:
`nhl-pace-1p` cards always have execution_status=PROJECTION_ONLY and a fixed line of 1.5 — the NHL 1P total market always settles on whether combined 1P goals exceed 1.5. The model emits a projected 1P total (e.g., "1.8 goals"). The card currently shows a blank/empty zone because `hasVisibleBetOdds` is false and no affirmative projection render fills the gap.

Direction is always derivable: if `projectedTotal > 1.5` → OVER, else UNDER. This is informational — not a bet recommendation — so no action/verdict badge should be shown.

**Scope**:
- `web/src/components/cards/GameCardItem.tsx` — add `isNhlPace1p` branch in the card body render to show projection block
- `web/src/components/cards/game-card-helpers.tsx` — add `formatPace1pProjectionBlock` helper if needed

**Out of scope**:
- Any changes to the worker / model / card_payloads
- Results tracking or settlement (WI-0756, WI-0757)
- mlb-f5 display (WI-0755)
- Changing the `isProjectionOnlyCard` flag logic at line 608 (do not alter odds-backed suppression)

**Acceptance**:
- A card with `cardType === 'nhl-pace-1p'` renders a projection block containing:
  - Label: `1P Total — Projection`
  - Line display: `Line: 1.5`
  - Model value: `Projected: X.X goals`
  - Direction badge: `OVER` (amber) or `UNDER` (blue) based on `projected > 1.5`
  - No bet odds field, no action/verdict badge (`FIRE`/`WATCH`/`PASS`)
- The projection block does NOT appear on odds-backed cards (guard: `cardType === 'nhl-pace-1p'` only)
- No TypeScript errors — `npm --prefix web run lint` passes on changed files
