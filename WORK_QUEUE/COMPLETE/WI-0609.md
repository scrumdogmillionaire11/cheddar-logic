# WI-0609: Token route IP whitelist / endpoint hardening

**ID**: WI-0609
**Goal**: Enforce access control on the token issuance endpoint (`/api/auth/token`) so it cannot be called from untrusted origins in production.

**Background**:
`web/src/app/api/auth/token/route.ts` line 37 contains a comment: "Remove this in production or add IP whitelist." The endpoint is currently open without that guard. If reachable publicly, it represents an unauthenticated token issuance surface.

**Scope**:
- `web/src/app/api/auth/token/route.ts` — add middleware or inline IP/origin check; return 403 when caller is not in the allowed set
- `web/src/lib/api-security/validation.ts` — register `/api/auth/token` query params with the central allowlist
- `env.example` — document `TOKEN_ROUTE_ALLOWED_IPS` env var

**Out of scope**:
- Full OAuth/OIDC integration
- Changes to token payload or expiry

**Acceptance**:
- A request from an unlisted IP/origin to `/api/auth/token` returns 403.
- Requests from the configured allowed set succeed as before.
- The allowed set is configured via env var (not hardcoded).
- TypeScript compiles cleanly.
