# WI-0446: FPL Sage Mobile Decision-First Layout

**ID**: WI-0446
**Goal**: Replace the current desktop-stacked mobile experience with a decision-first FPL layout that surfaces transfers, captaincy, pitch, and bench within the first screenful.

**Scope**:
- `WORK_QUEUE/WI-0446.md`
- `web/src/components/fpl-dashboard.tsx`
- `web/src/components/fpl-lineup-view.tsx`
- Mobile-only supporting components/styles directly required for FPL layout modules and collapsible sections

**Out of scope**:
- Changes to transfer optimization logic, captain scoring logic, or lineup solver outputs
- Non-FPL pages/routes
- API payload contract/schema changes
- Repo-wide style refactors

**Acceptance**:
- Mobile breakpoint `< 768px` uses a single-column, decision-first module stack, ordered: Transfers, Captaincy, Pitch, Bench, Strategy Notes, Advanced Data.
- Top summary module shows GW, recommended action/chip status, transfer count, captain/vice, and projected points near the top of mobile render.
- Transfer recommendation is first high-emphasis card with OUT/IN and expected gain/confidence.
- Pitch view expands to available mobile width with readable player points, tightened spacing, and scaled player cards; bench renders immediately below pitch.
- Strategy Notes is collapsible and collapsed by default on mobile.
- Advanced analytics sections (fixture planner, detailed projection tables, risk/model transparency blocks) are collapsed by default on mobile.
- Responsive behavior is explicitly defined for three modes:
  - Desktop: `> 1200px` multi-column dashboard
  - Tablet: `768px - 1200px` two-column layout
  - Mobile: `< 768px` single-column decision-first layout
- Mobile interaction ergonomics are enforced: 16px card padding and 44px minimum touch targets for interactive controls.
- Sticky mobile header remains visible while scrolling and preserves quick context/navigation (GW/team/transfers).
- Initial mobile render defers heavy analytics (fixture matrix/large projection tables) until user expands related sections.
- Existing desktop behavior remains intact aside from responsive adjustments required for breakpoint coherence.
