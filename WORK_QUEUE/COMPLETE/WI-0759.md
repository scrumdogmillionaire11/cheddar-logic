# WI-0759: ActionNetwork public splits — browser fixture capture and parser spec

**ID**: WI-0759
**Goal**: Capture real browser JSON responses from the ActionNetwork public splits
endpoint for NBA, MLB, and NHL; store them as preserved fixtures; validate (or
correct) the field-alias assumptions in the v2 adapter; and produce a parser spec.

**Depends on**: WI-0665

**Scope**:
- `WORK_QUEUE/WI-0759.md`
- `packages/adapters/fixtures/action_network/` (new directory)
- `packages/adapters/fixtures/action_network/*.raw.json` (one per sport/date captured)
- `docs/parsers/action_network_public_splits.md` (new parser spec)
- `packages/adapters/src/action-network.js` — update `MARKET_KEY_MAP` and
  field-alias comments if real response differs from hypothesis
- `packages/adapters/src/__tests__/action-network.test.js` — add fixture-driven
  tests once real shape is confirmed

**Out of scope**:
- Wiring splits into `slate_context` or downstream models (separate WI)
- Changes to any adapter other than `action-network.js`
- Production deployment of the adapter
- Building a browser extension or proxy

**Background**:
The ActionNetwork API endpoint
  `https://api.actionnetwork.com/web/v1/game?league={SPORT}&date={YYYYMMDD}`
is blocked from server/datacenter IPs via Cloudfront (returns 404 regardless
of User-Agent). It is accessible from real browser sessions. The v2 adapter in
`packages/adapters/src/action-network.js` carries this header:

  "STATUS: schema hypothesis — runtime blocked"

Until real fixture JSON is captured, the field aliases (`home_bets`, `away_handle`,
`over_bets`, `total`, `spread`, etc.) must be treated as provisional.

**Acceptance**:
1. At least one `.raw.json` fixture per sport (NBA, MLB, NHL) stored verbatim in
   `packages/adapters/fixtures/action_network/`. No transformation, no redaction.
2. `docs/parsers/action_network_public_splits.md` documents:
   - Exact top-level response shape
   - Each field in `bets[]` entries with type and observed values
   - Any discrepancies vs. the v2 hypothesis in `action-network.js`
   - Confirmed `bet_type` key strings present in real responses
   - Confirmed over/under field names for `TOTAL` entries
3. `MARKET_KEY_MAP` in `action-network.js` updated to match confirmed `bet_type`
   strings; field-alias arrays in `parseMarketEntry` updated where needed.
4. Fixture-driven test added: loads a `.raw.json` file, runs
   `normalizeSplitsResponse`, asserts all markets valid and key fields non-null.
5. Header comment in `action-network.js` updated from "schema hypothesis" to
   confirmed status with fixture date.
6. All 44 existing tests continue to pass.
