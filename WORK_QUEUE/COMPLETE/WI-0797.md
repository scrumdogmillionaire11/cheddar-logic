# WI-0797: Wire pipeline health watchdog Discord alerts

**ID**: WI-0797
**Goal**: When `ENABLE_PIPELINE_HEALTH_WATCHDOG=true`, emit a Discord notification to `DISCORD_CARD_WEBHOOK_URL` whenever any `pipeline_health` check records `status='failed'`, so failures surface without manual dashboard polling.

**Scope**:

- `apps/worker/src/jobs/check_pipeline_health.js`
- `apps/worker/src/jobs/post_discord_cards.js` (or extract a shared Discord util)
- new Discord jobs. We currently send to StatGeeks Discord. I want to add our play output to
  - cheddar stack: https://discord.com/api/webhooks/1490770264550674534/8_0Hx7KU0itlN4v1oL2j8WbRsG-DGcHcCd2Ay6yHZNseYkSlYJcqGY6fMPtWS0LFeJLM
- New model health alert hook: https://discord.com/api/webhooks/1457539311372206110/Acf_boakxQEQAsIoKCRMYt7p2gOXMQNniArWTP-A5x-cZzaHaXn06ZZoAu7QWsg53NfM

**Out of scope**:

- Changes to the pipeline health check logic or phase definitions
- `ENABLE_ODDS_HEALTH_WATCHDOG` alerting (separate concern, can follow same pattern)
- Dashboard UI changes (WI-0792)

**Acceptance**:

- When `check_pipeline_health.js` records at least one `status='failed'` row in a run, it sends a Discord message listing each failed phase/check and its reason
- Message is gated behind `ENABLE_PIPELINE_HEALTH_WATCHDOG=true` — disabled = no message
- `status='warning'` rows do not trigger Discord (logged only)
- Duplicate suppression: if the same phase/check has been `failed` for N consecutive checks (configurable, default 3), only send one Discord message per cooldown window (default 30 min) rather than on every tick
- `DISCORD_CARD_WEBHOOK_URL` is reused — no new env var needed for the URL
- Dry-run mode skips the Discord send
