# WI-0556: Track line movement delta in model output

**ID**: WI-0556
**Goal**: Record the line at first model run vs line at decision time so the pipeline can detect when movement has consumed the edge buffer and downgrade or block affected cards.

**Scope**:
- `packages/data/src/db.js` (add `odds_snapshots` or `line_history` query)
- `packages/models/src/decision-pipeline-v2.js` (consume line delta in edge computation)
- `packages/models/src/decision-gate.js` (wire line delta into `shouldFlip` context)
- `apps/worker/src/jobs/run_nba_model.js` (pass line history context at job run)
- `packages/models/src/__tests__/decision-pipeline-v2-line-move.test.js` (unit coverage for adverse/favorable/zero delta)

**Out of scope**:
- UI display of line movement
- Retroactive backfill of historical line movement data
- Line movement alerts or notifications

**Background**:
The pipeline captures `captured_at` timestamps and checks staleness (STALE_MARKET_INPUT watchdog), but it does not track *direction* of line movement. A spread card that showed 6% edge when the line was -3 may show 3% edge after the line moved to -4.5. The pipeline cannot detect this. `decision-gate.js` already has `lineMoved/lineDelta` fields in its context shape, but these must be provided externally — the pipeline does not compute them. This means the gate's LINE_MOVE logic is only reachable if callers explicitly construct the context, which the current job runners do not do.

**Acceptance**:
- The worker has access to at least two odds snapshots per game (opener/first-run and current) from the `odds_snapshots` table or equivalent.
- A `computeLineDelta({ sport, gameId, marketType, db })` utility returns `{ opener_line, current_line, delta, delta_pct }` for the given game/market.
- `run_nba_model.js` (and NHL if applicable) calls `computeLineDelta` and passes the result as `lineContext` into the decision pipeline.
- `decision-pipeline-v2.js` uses `lineContext.delta` to: (a) reduce edge by the line delta when movement is adverse to the model's side, and (b) emit a `LINE_MOVE_ADVERSE` price reason code when delta > 1 point against position.
- A card whose edge drops below `lean_edge_min` after line adjustment is downgraded to PASS.
- Unit tests cover: adverse move consuming edge (LEAN→PASS), favorable move increasing edge (no change to verdict), zero delta (no effect).
