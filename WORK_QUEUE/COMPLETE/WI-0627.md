# WI-0627: Wire check_odds_health into scheduler as automated watchdog

**ID**: WI-0627
**Goal**: Add `check_odds_health.js` to `schedulers/main.js` so stale odds (>90 min with no successful `pull_odds_hourly`) is detected and logged automatically on every tick rather than requiring manual CLI invocation.

**Background**:
`apps/worker/src/jobs/check_odds_health.js` exists and works correctly when run manually, but it is not imported or called in `schedulers/main.js`. If `pull_odds_hourly` silently fails overnight, the NHL/NBA/MLB models run against stale odds with no alert. The job is read-only and lightweight — safe to run every tick or on a fixed cadence (e.g., every 30 min).

**Scope**:
- `apps/worker/src/schedulers/main.js` — import `checkOddsHealth` and register it on a ~30-min cadence (or every scheduler tick behind a time-window key like `health|odds|${nowUtc.toISO().slice(0, 13)}-${Math.floor(nowUtc.minute / 30)}`)

**Out of scope**:
- Changes to `check_odds_health.js` itself
- Adding Discord/PagerDuty alerting (separate concern)

**Acceptance**:
- `check_odds_health` is imported and scheduled in `schedulers/main.js`.
- Job runs at most once per 30-min window (idempotent via jobKey).
- `node apps/worker/src/schedulers/main.js --dry-run` starts without error.
- `npm --prefix apps/worker test` passes.
