# WI-0749: Canonical Reporting Identity — Results Page

**ID:** WI-0749
**Owner agent:** copilot
**Time window:** 2026-04-03
**Coordination flag:** none

## Goal

Fix the results page grouping bug where `NHL Totals` and `nhl-totals-call` appear as separate
rows in the Betting Decision Tiers table. Root cause: segments are keyed on raw `card_type`
(an opaque string) instead of a canonical market-family key.

Add three derived identity fields (`cardFamily`, `modelFamily`, `modelVersion`) computed at
query time from `sport` + `card_type`. Re-key the segment accumulator on `cardFamily`. Update
the results UI to show **Family** and **Engine** columns instead of raw **Type** and **Market**.

## Scope

- `web/src/app/api/results/projection-metrics.ts` — add `deriveCardFamily`, `deriveModelFamily`, `deriveModelVersion` exports
- `web/src/app/api/results/route.ts` — use those derivations in segment accumulator key and output; add fields to ledger rows
- `web/src/app/results/page.tsx` — update `SegmentRow` + `LedgerRow` types; rename table columns; swap display values

## Out of scope

- DB migration (no new columns on `card_payloads` in this WI)
- Worker changes
- Any files outside the three listed above

## Acceptance

- `cd web && npm run build` exits 0 with no TypeScript errors
- `/results` Betting Decision Tiers table shows one `NHL_TOTAL` row per tier, not separate `NHL Totals` + `nhl-totals-call` rows
- Table header columns read **Family** and **Engine** (not **Type** / **Market**)
