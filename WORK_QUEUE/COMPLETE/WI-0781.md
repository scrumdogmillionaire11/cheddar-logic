# WI-0781: Adopt job-runtime.js createJob wrapper across all worker jobs

| Field              | Value |
|--------------------|-------|
| **ID**             | WI-0781 |
| **Goal**           | Wire the existing `job-runtime.js` helper from `@cheddar-logic/data` into all jobs under `apps/worker/src/jobs/` to eliminate per-job boilerplate (CLI arg parsing, dry-run flag, logging preamble, error/exit handling) and enforce a consistent contract |
| **Priority**       | Medium |
| **Owner agent**    | Unassigned |
| **Time window**    | TBD |
| **Coordination flag** | None — each job file is a self-contained change; no shared-touchpoint files modified |

---

## Problem

`packages/data/src/job-runtime.js` exists and exports job lifecycle helpers but most jobs do not use it. Each of the 40+ files under `apps/worker/src/jobs/` re-implements:

- `DRY_RUN` env-var check (`process.env.DRY_RUN === 'true'`)
- `console.log('[job-name] Starting...')` / `console.log('[job-name] Complete')` preamble
- `process.exit(0)` / `process.exit(1)` on success/failure
- Individual `try/catch` with varying error logging formats

This means ~5–15 lines of near-identical boilerplate per job, error messages are inconsistently formatted, and there's no single place to add cross-cutting concerns (e.g. runtime duration logging, Slack error notification, job-run DB record).

---

## Approach

Audit `packages/data/src/job-runtime.js` API first. If it doesn't already expose a `createJob({ name, dryRun, run })` style wrapper:

1. Add `createJob` to `job-runtime.js`
2. Convert jobs incrementally — start with the five smallest jobs as a pilot
3. Expand to remaining jobs in a follow-on WI if the pilot is clean

If `job-runtime.js` already has the wrapper, skip step 1 and go straight to adoption.

## Out of Scope

- Changing job logic or output beyond the lifecycle wrapper
- Remaining 35+ jobs (deferred to a follow-on WI pending pilot review)
- Scheduler changes

---

## Acceptance

- Pilot jobs run identically to before (`DRY_RUN=true node src/jobs/check_odds_health.js` produces same output)
- `createJob` is exported from `@cheddar-logic/data`
- No per-job `process.exit()` calls in the five pilot files (handled by `createJob`)
- Existing tests pass for the five pilot jobs
