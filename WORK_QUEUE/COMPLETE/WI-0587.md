# WI-0587: Remove NCAAM matchup-style as an actionable betting source

**ID**: WI-0587
**Goal**: Stop `ncaam-matchup-style` from producing actionable production bets. Keep it as context/evidence only, or suppress it entirely, until a future study proves it has standalone edge.

**Scope**:
- `apps/worker/src/models/index.js`
- `apps/worker/src/jobs/run_ncaam_model.js`
- `packages/models/src/card-factory.js` (if needed to emit evidence-only rows cleanly)
- `apps/worker/src/models/__tests__/ncaam-drivers.test.js`
- `web/src/__tests__/api-results-decision-segmentation.test.js` (if decision-tier/result enrollment behavior changes)

**Out of scope**:
- Retuning `ncaam-base-projection`
- Retuning NCAAM spread or moneyline thresholds
- NCAAM sigma calibration work (track separately)

**Background**:
Production results show `ncaam-matchup-style` at `0-5` and `-5.00u` across PLAY + SLIGHT EDGE. The current architecture already treats `ncaam-matchup-style` as `CONTEXT` in the web driver registry, but the worker still emits direct NCAAM cards that can become actionable spread/moneyline bets. That is a pipeline-integrity problem, not just a bad run.

**Acceptance**:
- `ncaam-matchup-style` no longer inserts actionable spread or moneyline cards in production model runs.
- If `ncaam-matchup-style` is retained, it is emitted only as non-actionable context/evidence and cannot enroll into decision-tier results reporting.
- `ncaam-base-projection`, `ncaam-rest-advantage`, and FT-driven NCAAM cards continue to run unchanged.
- Regression coverage proves `ncaam-matchup-style` cannot be the sole source of a PLAY/LEAN outcome.
