# WI-0707: Canonical Risk Posture Config + Scoring Integration

## Context

Risk posture currently lives as isolated magic numbers scattered across:
- `get_volatility_multiplier()` in constants.py (1.25/1.0/0.8 hardcoded)
- `get_risk_multipliers()` in risk_aware_filter.py (dicts duplicating the same concept)
- Inline weights in transfer_advisor and captain_selector

There is no single config object. That means:
- Adding a new posture-aware behavior requires finding and updating multiple files
- The scoring layers (floor/ceiling weighting) are not connected to posture at all
- Hit ROI logic uses fixed thresholds with no posture awareness
- Captain differential bias does not exist

## Goal

One frozen `RiskPostureConfig` dataclass is the sole source of truth. Everything else reads from it.

## Implementation Notes

### Step 1: Create `backend/config/risk_posture.py`

```python
from dataclasses import dataclass
from typing import Literal

PostureName = Literal["CONSERVATIVE", "BALANCED", "AGGRESSIVE"]
PostureSignal = Literal["CHASE", "HOLD", "PROTECT"]

@dataclass(frozen=True)
class RiskPostureConfig:
    name: PostureName
    hit_threshold_net_pts: float      # min net pts to approve a hit
    hit_short_weight: float            # weight applied to delta_next2 in ROI calc
    hit_mid_weight: float              # weight applied to delta_next6 in ROI calc
    transfer_ceiling_weight: float     # weight on P75 / ceiling projection
    transfer_floor_weight: float       # weight on P25 / floor projection
    volatility_penalty_weight: float   # multiplier on volatility term in scoring
    fragility_penalty_weight: float    # multiplier on fragility term in scoring
    ownership_bias_weight: float       # +ve = reward template, -ve = penalise template
    diff_captain_bias: float           # scales differential_bonus in captain scoring
    bench_boost_threshold_pts: float   # bench points needed to recommend BB
    bench_fragility_tolerance: float   # max acceptable bench fragility score for BB
    wildcard_weakness_threshold: int   # min squad weaknesses to trigger WC suggestion
    captain_template_tiebreak: bool    # if True, prefer highest-owned on ties
    posture_signal_default: PostureSignal

def get_posture_config(posture: str) -> RiskPostureConfig:
    normalized = posture.strip().upper()
    if normalized not in RISK_POSTURE_CONFIGS:
        return RISK_POSTURE_CONFIGS["BALANCED"]
    return RISK_POSTURE_CONFIGS[normalized]
```

### Step 2: Add `posture_hit_allowed` to `risk_aware_filter.py`

```python
def posture_hit_allowed(
    cfg: RiskPostureConfig,
    delta_next2: float,
    delta_next6: float,
    hit_cost: float,  # positive number, e.g. 4 for a -4 hit
) -> tuple[bool, float]:
    weighted_gain = (
        cfg.hit_short_weight * delta_next2
        + cfg.hit_mid_weight * delta_next6
    ) - hit_cost
    return weighted_gain >= cfg.hit_threshold_net_pts, weighted_gain
```

### Step 3: Use ceiling/floor weights in TransferAdvisor

In `_score_candidate_for_strategy`, use `cfg.transfer_ceiling_weight` and `cfg.transfer_floor_weight` when `nextGW_ceiling_pts` / `nextGW_floor_pts` are available on the candidate:

```python
# If projection has floor/ceiling, use posture-weighted blend; else fall back to next_pts
if hasattr(candidate, 'nextGW_ceiling_pts') and hasattr(candidate, 'nextGW_floor_pts'):
    effective_pts = (
        cfg.transfer_ceiling_weight * float(candidate.nextGW_ceiling_pts or next_pts)
        + cfg.transfer_floor_weight * float(candidate.nextGW_floor_pts or next_pts)
    )
else:
    effective_pts = next_pts
```

`effective_pts` replaces `next_pts` in all four strategy branches.

### Step 4: Apply diff_captain_bias in CaptainSelector

```python
from backend.config.risk_posture import get_posture_config
cfg = get_posture_config(self.risk_posture)

```

Apply this after the strategy branch calculates base_score, before horizon_adj.
