# WI-0949: NBA ESPN Null Metrics Alert Noise Reduction and Stability Gate

**ID**: WI-0949
**Goal**: Prevent noisy NBA ESPN null-metrics alerts from transient single-run `NO_GAMES` fetch gaps while preserving fast detection of persistent ESPN data outages.

**Scope**:

- `apps/worker/src/jobs/run_nba_model.js`
- `apps/worker/src/jobs/__tests__/run_nba_model.test.js`
- `packages/data/src/team-metrics.js`
- `WORK_QUEUE/WI-0949.md`

**Out of scope**:

- NBA projection model formula changes
- Odds scheduling cadence changes (handled separately)
- Discord webhook infrastructure changes
- NHL/MLB null-metric alert policy changes

**Acceptance**:

- NBA ESPN null alerting requires persistence criteria (for example, consecutive-run or short-window recurrence) before sending Discord alerts for `NO_GAMES` team reasons.
- Alerting still fires for persistent ESPN failures and includes deterministic team+reason output.
- Single-run transient nulls no longer emit alerts when subsequent run resolves metrics for the same teams.
- Team metrics fallback behavior remains fail-open for model execution (no hard crash) and keeps explicit null reason codes for diagnostics.
- Legacy ESPN-null handling branches are consolidated where possible, and any retained legacy behavior is documented with clear deprecation intent.
- Automated tests cover both suppression and send paths: below persistence threshold, at threshold, and cooldown behavior.
