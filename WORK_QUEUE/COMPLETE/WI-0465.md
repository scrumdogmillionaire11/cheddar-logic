# WI-0465 — Remaining injury infrastructure (deferred scope)

## Status
BACKLOG

## Goal
Close the remaining gaps in the NHL injury pipeline that were explicitly
deferred from WI-0462 / WI-0463.

---

## Item A — Dedicated injury sync job (hourly poll)

**Problem**: Player availability is currently only updated when
`pull_nhl_player_shots` runs (fetches L5 game logs).  That job may run
infrequently, leaving stale injury status in `player_availability`.

**Proposed work**:
- New job `sync_nhl_player_availability.js` that polls NHL API `/player/{id}/landing`
  for all configured player IDs **without** writing shot logs.
- Runs on its own cron (e.g., every 60 min) independently of the shot-log pull.
- Reuses `checkInjuryStatus()` and `upsertPlayerAvailability()`.

---

## Item B — DTD / Questionable status tiering in card payloads

**Problem**: Day-to-day and questionable statuses are currently treated as
"not injured" (fail-open).  The model doesn't differentiate a GTD star from
a healthy scratch.

**Proposed work**:
- Extend `checkInjuryStatus()` to return `{ skip: false, tier: 'DTD' | 'ACTIVE' }`.
- In `run_nhl_player_shots_model.js`, include `tier` in the output card payload
  so downstream consumers can surface "key player questionable" flags.
- Store tiered status in `player_availability.status` (e.g., `'DTD'`).

---

## Item C — Defense-side skater injury adjustment

**Problem**: WI-0463 only adjusts `homeOffRating` / `awayOffRating`.  A team
missing a top defensive pairing will also suppress their opponent's expected
goals via reduced shot suppression, but this is not modeled.

**Proposed work**:
- Add `homeSkaterDefInjuryFactor` / `awaySkaterDefInjuryFactor` to
  `predictNHLGame()`.
- Applied to `homeDefRating` / `awayDefRating` — but this affects *opponent*
  goals, so semantics need careful analysis (missing defense → opponent scores
  more → increase opponent off-rating or reduce your def-rating).
- Requires player positional data to distinguish D from F.

---

## Item D — Injury card emission in `computeNHLDriverCards`

**Problem**: There is no dedicated "lineup" card emitted when key players are
out.  The totals model is adjusted, but bettors don't see an explicit injury
signal card.

**Proposed work**:
- Emit a `nhl-lineup` driver card when `homeSkaterInjuryFactor` or
  `awaySkaterInjuryFactor` is < 0.95 (meaningful reduction).
- Card includes list of confirmed-out players, estimated goal suppression,
  and resulting direction signal (typically UNDER).

---

## Dependencies
- A depends on WI-0462 (`player_availability` schema, `upsertPlayerAvailability`).
- B depends on WI-0462 (status tiers need to be stored).
- C depends on WI-0463 (pace model params already in place).
- D depends on WI-0463 (factor computed before card emission block).
