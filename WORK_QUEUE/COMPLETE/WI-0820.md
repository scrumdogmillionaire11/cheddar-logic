# WI-0820: Core Input Gate — NO_BET / DEGRADED / MODEL_OK

| Field | Value |
|---|---|
| **ID** | WI-0820 |
| **Goal** | Replace the current silent synthetic-fallback path with three hard model states: `MODEL_OK`, `DEGRADED`, and `NO_BET`. Missing core inputs for any sport must short-circuit to `NO_BET` before any projection arithmetic runs. |
| **Priority** | Critical |
| **Depends on** | None — standalone hardening |
| **Coordination flag** | `needs-sync` — touches `mlb-model.js`, `nhl-pace-model.js`, `projections.js`, `cross-market.js`, and any callers that read `projection_source === 'SYNTHETIC_FALLBACK'` |

---

## Background

The current codebase allows projection to proceed with missing inputs by routing to `SYNTHETIC_FALLBACK`. This creates a false sense that models are live when the data pipeline has failed. A SYNTHETIC_FALLBACK "PLAY" is indistinguishable from a data-backed "PLAY" in card payloads and monitoring.

The fix is a gating utility that runs before any projection math and returns a standardized status envelope that callers must check.

## Implementation

### `input-gate.js`
```js
'use strict';

/**
 * Classify model inputs into MODEL_OK / DEGRADED / NO_BET.
 *
 * @param {object} featureMap  - flat object of feature name → value
 * @param {string[]} required  - keys that must be non-null, non-NaN
 * @param {string[]} [optional] - keys that degrade confidence when missing
 * @returns {{ status: 'MODEL_OK'|'DEGRADED'|'NO_BET', missingCritical: string[], missingOptional: string[] }}
 */
function classifyModelStatus(featureMap, required, optional = []) {
  const missingCritical = required.filter(
    (k) => featureMap[k] == null || (typeof featureMap[k] === 'number' && Number.isNaN(featureMap[k]))
  );
  const missingOptional = optional.filter(
    (k) => featureMap[k] == null || (typeof featureMap[k] === 'number' && Number.isNaN(featureMap[k]))
  );

  const status =
    missingCritical.length > 0
      ? 'NO_BET'
      : missingOptional.length > 0
        ? 'DEGRADED'
        : 'MODEL_OK';

  return { status, missingCritical, missingOptional };
}

/**
 * Build a standard NO_BET return envelope.
 */
function buildNoBetResult(missingCritical, context = {}) {
  return {
    status: 'NO_BET',
    reason: 'MISSING_CORE_INPUTS',
    missingCritical,
    prediction: null,
    confidence: 0,
    projection: null,
    ...context,
  };
}

module.exports = { classifyModelStatus, buildNoBetResult };
```

### Usage pattern in model functions
```js
const { classifyModelStatus, buildNoBetResult } = require('./input-gate');

function projectF5Total(homePitcher, awayPitcher, context = {}) {
  const features = {
    starter_skill_ra9_home: resolveStarterSkillProfile(awayPitcher).starter_skill_ra9,
    starter_skill_ra9_away: resolveStarterSkillProfile(homePitcher).starter_skill_ra9,
    wrc_plus_vs_hand_home: resolveTeamSplitProfile(context?.home_offense_profile, awayPitcher?.handedness)?.wrc_plus ?? null,
    wrc_plus_vs_hand_away: resolveTeamSplitProfile(context?.away_offense_profile, homePitcher?.handedness)?.wrc_plus ?? null,
    park_run_factor: context?.park_run_factor ?? null,
  };

  const gate = classifyModelStatus(
    features,
    ['starter_skill_ra9_home', 'starter_skill_ra9_away', 'wrc_plus_vs_hand_home', 'wrc_plus_vs_hand_away', 'park_run_factor'],
    ['temp_f', 'wind_mph'],
  );

  if (gate.status === 'NO_BET') {
    return buildNoBetResult(gate.missingCritical, { projection_source: 'NO_BET' });
  }

  // ... existing projection logic follows, with gate.status propagated into result
}
```
