# WI-0915: NHL BLK Context Wiring and Factor Observability

**ID**: WI-0915
**Goal**: Wire BLK defensive-zone and underdog-script factors into runtime, make factor computation deterministic, and expose active-vs-defaulted context in card payloads.

**Depends on**: WI-0909

**Scope**:

- `apps/worker/src/jobs/run_nhl_player_shots_model.js`
- `apps/worker/src/jobs/__tests__/run_nhl_player_shots_model.test.js`

**Scope guardrails (overlap control)**:

- Execute only after WI-0909 is merged/closed to avoid shared-file ownership conflict.
- If WI-0911 or WI-0914 becomes claimed before execution, rescope or re-sequence before code changes.

**Out of scope**:

- Changing `projectBlkV1` math or clamp bounds in model layer
- New external data providers or schema migrations
- SOG, points, or goals market refactors

**Acceptance**:

- `projectBlkV1` call in `run_nhl_player_shots_model` passes all four BLK context factors:
  - `opponent_attempt_factor`
  - `defensive_zone_factor`
  - `underdog_script_factor`
  - `playoff_tightening_factor`
- BLK runtime no longer silently defaults `defensive_zone_factor` and `underdog_script_factor` to 1.0 when computable context exists.
- BLK payload includes deterministic observability fields for factor values and source status (`computed` vs `defaulted`).
- BLK default/fallback paths are explicit in decision flags and test-covered.
- At least one test proves BLK projection path is sensitive to underdog/DZ factor movement (not cosmetic wiring).

**Implementation spec**:

- **Underdog script factor (deterministic):**
  - Source signal: game moneyline-implied win probability for player team, when available from existing odds snapshot path in runner.
  - Mapping:
    - `winProb < 0.35 => 1.10`
    - `0.35 <= winProb < 0.40 => 1.08`
    - `0.40 <= winProb < 0.45 => 1.05`
    - `winProb >= 0.45 => 1.00`
  - Fallback: if win probability unavailable, set `underdog_script_factor = 1.00` and add `BLK_UNDERDOG_FACTOR_MISSING`.
  - Clamp remains model-owned in `projectBlkV1`.

- **Defensive-zone factor (deterministic proxy):**
  - Source signal: opponent shot-volume pressure proxy from existing `team_metrics_cache` lookup.
  - Formula:
    - `rawDz = opponent_shots_against_pg / league_avg_shots_against_pg`
    - `defensive_zone_factor = rawDz` (model clamps to [0.95, 1.08])
  - Fallback: if matchup metrics unavailable, set `defensive_zone_factor = 1.00` and add `BLK_DZ_FACTOR_MISSING`.

- **Double-counting guardrail:**
  - Before calling `projectBlkV1`, cap the multiplicative context contribution from dynamic context factors:
    - `combinedDynamic = opponent_attempt_factor * defensive_zone_factor * underdog_script_factor`
    - if `combinedDynamic > 1.20`, rescale `defensive_zone_factor` and `underdog_script_factor` proportionally so product is 1.20 while preserving `opponent_attempt_factor`.
  - Add flag `BLK_CONTEXT_CAP_APPLIED` when this guardrail triggers.

- **Card observability fields:**
  - Add under BLK payload drivers:
    - `blk_factor_inputs`: `{ opponent_attempt_factor, defensive_zone_factor, underdog_script_factor, playoff_tightening_factor }`
    - `blk_factor_source`: `{ opponent_attempt_factor: computed|defaulted, defensive_zone_factor: computed|defaulted, underdog_script_factor: computed|defaulted, playoff_tightening_factor: computed|defaulted }`
    - `blk_context_tag`: one of `FAVORITE_LOW_BLOCK`, `UNDERDOG_HIGH_PRESSURE`, `NEUTRAL`
  - Tag rules:
    - `UNDERDOG_HIGH_PRESSURE` when `underdog_script_factor > 1.0` and `defensive_zone_factor > 1.0`
    - `FAVORITE_LOW_BLOCK` when `underdog_script_factor === 1.0` and `defensive_zone_factor < 1.0`
    - else `NEUTRAL`

- **Decision flags:**
  - Ensure BLK decision/prop flags include any of:
    - `BLK_DZ_FACTOR_MISSING`
    - `BLK_UNDERDOG_FACTOR_MISSING`
    - `BLK_CONTEXT_CAP_APPLIED`

**Tests to run**:

**Test additions required**:

- `projectBlkV1` invocation includes non-default `defensive_zone_factor` and `underdog_script_factor` when data exists.
- Missing context sets default values and emits missing-factor flags.
- Context cap path emits `BLK_CONTEXT_CAP_APPLIED`.
- Classification impact test: for fixed player/rates/TOI, flipping team from favorite-context to underdog/high-pressure context changes BLK decision tier or verdict in expected direction.
- Payload observability test: `drivers.blk_factor_inputs`, `drivers.blk_factor_source`, and `drivers.blk_context_tag` are present and correct.

- Run one path with missing matchup/odds context and verify defaulted factors plus missing-factor flags.
- Compare two otherwise-identical scenarios (favorite vs underdog) and confirm BLK projection/decision shift is visible in card payload.
