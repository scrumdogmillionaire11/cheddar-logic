# WI-0591: Wire empirical sigma overrides into NBA and NCAAM decisioning

**ID**: WI-0591
**Goal**: Complete the empirical sigma rollout so basketball decisioning actually consumes computed sigma values instead of silently falling back to static defaults, and extend the same path to NCAAM before any further college-basketball tuning.

**Scope**:
- `packages/models/src/decision-pipeline-v2.js`
- `packages/models/src/edge-calculator.js`
- `packages/models/src/__tests__/edge-calculator.test.js`
- `apps/worker/src/jobs/run_nba_model.js`
- `apps/worker/src/jobs/run_ncaam_model.js`
- `apps/worker/src/jobs/__tests__/run_nba_model.test.js`
- `apps/worker/src/jobs/__tests__/run_ncaam_model.test.js` (new if needed)

**Out of scope**:
- Changing sport-specific edge thresholds
- Retuning NCAAM base-projection logic
- Any NHL sigma work

**Background**:
`computeSigmaFromHistory()` exists and NBA logs a computed sigma object at job start, but current canonical decisioning still reads `getSigmaDefaults(sport)` directly. That means the pipeline can appear calibrated while continuing to price edges off fallback constants. This WI makes sigma overrides real for NBA and gives NCAAM the same path so future college-basketball work is not built on static assumptions.

**Acceptance**:
- NBA decisioning consumes an explicit computed sigma override when available; it no longer just logs the value and then prices off fallback defaults.
- NCAAM model runs compute and pass empirical sigma through the same override path when enough history exists.
- When history is insufficient, the pipeline falls back cleanly and marks the sigma source as `fallback`.
- Decision outputs or logs expose whether sigma came from `computed` or `fallback`.
- Regression coverage proves the override path changes the sigma used by edge math for NBA and NCAAM when computed history is available.
