# WI-0777: Enable npm workspaces at the repo root

| Field              | Value |
|--------------------|-------|
| **ID**             | WI-0777 |
| **Goal**           | Declare `workspaces` in the root `package.json` so that `packages/data`, `packages/adapters`, `packages/odds`, `packages/models`, `apps/worker`, and `web` are managed as first-class workspace members â€” collapsing duplicate `node_modules` hoisting and removing the need for path-aliased symlinks |
| **Priority**       | Medium |
| **Owner agent**    | Unassigned |
| **Time window**    | TBD |
| **Coordination flag** | `needs-sync` â€” modifies root `package.json` and may alter `package-lock.json`; coordinate with any agent currently touching lockfiles |

---

## Problem

The root `package.json` has no `workspaces` field despite the repo having multiple publishable packages under `packages/` and two apps under `apps/` and `web/`. This means:

- Each app manages its own copies of `node_modules` independently
- `apps/worker/packages/data/` is an orphaned local copy with no source files (only contains `cheddar.db` and a `package-lock.json`), suggesting the workspace link was never wired
- Dependency deduplication is manual; version drift between `apps/worker` and `web` for shared packages like `luxon`, `zod`, etc. is undetected
- Cross-package `require('@cheddar-logic/data')` relies on `node_modules` symlinks created during individual `npm install` steps rather than workspace resolution

---

## Scope

| File | Change |
|------|--------|
| `package.json` | Add `"workspaces": ["packages/*", "apps/*", "web"]` |
| `apps/worker/packages/data/` | Delete the orphaned stub directory (contains only `cheddar.db` and `package-lock.json`, no `src/`) |
| `apps/worker/package.json` | Verify `@cheddar-logic/*` dependencies resolve via workspace symlinks after root `npm install`; remove any `file:` path overrides that are superseded by workspace resolution |

## Out of Scope

- Changing package `name` fields or exports in `packages/*`
- Migrating from CJS to ESM (separate concern â€” see existing CONCERNS.md)
- Modifying `web/package.json` dependencies
- Changing CI workflow install steps (deferred to a follow-on once workspace install is verified locally)

---

## Acceptance

- `npm install` at repo root completes without errors
- `node -e "require('@cheddar-logic/data')"` succeeds from `apps/worker/`
- `apps/worker/packages/data/` directory no longer exists
- `npm ls --workspaces` lists all six workspace members without errors
- Existing tests pass: `cd apps/worker && npm test`
