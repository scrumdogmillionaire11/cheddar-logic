## Implementation Plan

### Stage A — normalize market scope in `run_soccer_model.js`

Map all inbound soccer candidates into canonical market groups using the normalizer/router contract. Drop anything outside the soccer allowlist with an explicit blocked reason. No ad hoc string matching.

Canonical groups: `player_shots`, `team_totals`, `to_score_or_assist`, `player_shots_on_target`, `anytime_goalscorer`, `team_corners`.

### Stage B — build minimum required context per match

Derive conservative context packet: match id / league / teams, context/style/form tags, environment fields when available, warnings/fatal markers when not. Missing footie consistency results → `GAME_FATAL` at watchdog time.

### Stage C — build Tier 1 market packets

**Player Shots**: player identity, `PRIMARY_VOLUME_SHOOTER` evidence, projected minutes / starter signal, shots/90 or team-relative shot share, line, price, projection, edge classification.

**Team Totals**: team identity, team lambda or equivalent projection basis, allowed lines only (`o0.5`, `o1.5`, `u2.5`), price, edge basis.

**TSOA**: player identity, one of `TERMINAL_NODE | PRIMARY_CREATOR | SET_PIECE_ROLE`, projected minutes / starter signal, xG/xA support, line, price, fair probability / edge basis.

### Stage D — keep Tier 2 in the passenger seat

SOT only when Shots is unavailable or poor-priced and price cap passes. Anytime only for terminal nodes at `> +180`. Corners only with extreme mismatch plus data. Otherwise emit PASS — no honorary medals for almost-data.

### Stage E — explicit degradation behavior

When core projection context is missing: do not backfill with generic defaults, emit structured warnings/flags, downgrade to PASS or skip, still allow structurally valid card with reason. Matches slate_context flagging rules and watchdog "no pick without reason."

### Stage F — tighten validator (`card-payload.js`)

Add `soccer_scope` block:

- Scope allowlist: reject any market key outside the six canonical keys.
- Required-field validation per market type (see Acceptance criteria #2–#3).
- Price caps: shots ≤ -150, TSOA ≤ -140, SOT ≤ -130, anytime > +180.
- Placeholder detection: reject `"unknown"`, `"tbd"`, `0` as fake projection, empty required arrays, generic fallback text.

### Stage G — update worker tests

Add deterministic happy-path cases (Shots, Team Totals, TSOA), rejection cases (banned market, insufficient eligibility, price cap failure, draw odds synthesized from nothing).

### Stage H — update web decision test

Assert: soccer payload with insufficient context cannot be promoted to a real play; PASS with explicit reason is acceptable; missing context must not render as actionable edge.

## Appendix A - Canonical Soccer Market Logic Stack

### System Flow (Parse -> Normalize -> Environment -> Project -> Classify -> Validate/Watchdog -> Emit)

1. Parse soccer-scope soccer market candidates only: `TEAM_TOTALS`, `PLAYER_SHOTS`, `PLAYER_SOT`, `TSOA`, `ANYTIME_SCORER`, `TEAM_CORNERS`.
2. Normalize source candidates into canonical keys: `player_shots`, `team_totals`, `to_score_or_assist`, `player_shots_on_target`, `anytime_goalscorer`, `team_corners`.
3. Build per-match environment packet (`xg_env`, `event_env`, `vol_env`, context tags) before market modeling.
4. Build market-level projections (projected mean and/or fair probability) and market-aware edge/EV.
5. Classify using canonical thresholds:
   - Base: EV >= 3.0%
   - Lean: EV 1.0% to 2.9%
   - Pass: EV < 1.0% or vetoed
6. Run scope validator + watchdog gates; downgrade to PASS with explicit reason when blocked.
7. Emit actionable market packet with line, price, fair probability/projection basis, edge/EV, tier, variance class, warning tags/unit caps where required.

### Shared Cross-Market Requirements

- Player market eligibility requires: confirmed starter, projected minutes >= 60, deterministic role tags, and market-relevant stat hints.
- Every market packet must include: line, price, implied probability, fair probability/fair line, and edge/EV.
- Context inputs must be explicit (team attack/defense profile, xG/xGA context, and environment tags).
- Empty official slate is valid: output PASS with reason when no edge exists; do not force a pick.
- `footie_validation` consistency payload is required for watchdog compatibility.

### Market-by-Market Logic

| Canonical market | Required inputs | Eligibility gates | Projection/model requirement | PASS reasons |
|---|---|---|---|---|
| `player_shots` | player shots market snapshot, player id/team, shots/90, projected minutes, starter status, role tags, line/price, environment | `PRIMARY_VOLUME_SHOOTER`, starter=true, minutes>=60, price >= -150 | projected shots mean -> fair prob/fair price -> edge/EV | `PASS_NOT_A_STARTER`, `PASS_PRICE_TOO_JUICED`, `PASS_NOT_ELIGIBLE_ROLE`, `PASS_NO_EDGE` |
| `team_totals` | team totals market snapshot, team side, team attack profile, opponent defense profile, lambda basis, line/price | line in allowlist (`o0.5`, `o1.5`, `u2.5`), lambda present, lambda range gates | team lambda -> fair probabilities for allowed lines -> edge/EV | `PASS_LAMBDA_OUT_OF_RANGE`, `PASS_NO_EDGE` |
| `to_score_or_assist` | TSOA market snapshot, xG/90, xA/90, starter/minutes, role tags, line/price, team attack context | role in (`TERMINAL_NODE`, `PRIMARY_CREATOR`, `SET_PIECE_ROLE`), starter=true, minutes>=60, price >= -140 | score prob + assist prob + overlap handling -> combined fair prob -> edge/EV | `PASS_NOT_ELIGIBLE_ROLE`, `PASS_PRICE_TOO_JUICED`, `PASS_NOT_A_STARTER`, `PASS_NO_EDGE` |
| `player_shots_on_target` | SOT snapshot + shots inputs + conversion context + keeper/opponent suppression + line/price | `PRIMARY_VOLUME_SHOOTER`, starter=true, minutes>=60, price >= -130, shots market unavailable or poor-priced, Tier2 enabled | projected shots + SOT conversion -> fair prob -> edge/EV | `PASS_SHOTS_PREFERRED`, `PASS_PRICE_TOO_JUICED`, `PASS_NOT_ELIGIBLE_ROLE`, `PASS_NOT_A_STARTER`, `PASS_NO_EDGE` |
| `anytime_goalscorer` | anytime snapshot, xG/90, projected minutes, terminal role, team goal context, line/price | `TERMINAL_NODE`, price > +180, Tier2 enabled | player goal probability model -> fair odds -> edge/EV | `PASS_NOT_TERMINAL_NODE`, `PASS_PRICE_TOO_JUICED`, `PASS_NO_EDGE` |
| `team_corners` | corners snapshot, corners rates/proxy, style mismatch tags, line/price | corners enabled for state/book, corners data present, `EXTREME_WIDE_PRESSURE_MISMATCH`, Tier2 enabled | projected corners expectation -> fair probability -> edge/EV | `PASS_NO_CORNERS_DATA`, `PASS_NO_MISMATCH_TAG`, `PASS_NO_EDGE` |

### Runtime Guardrails

- Scope hard allowlist: only six canonical keys survive.
- Keep banned/out-of-scope markets blocked (`cards`, `fouls`, `1x2`, `draw_no_bet`, `asian_handicap`, `match_total`, `btts`).
- Price caps:
  - shots >= -150
  - TSOA >= -140
  - SOT >= -130
  - anytime > +180
- Starter/minutes floor for player markets: starter=true and projected_minutes >= 60.
- Tier2 enforcement: allowed only when Tier2 enabled and gating passes; warning tags + unit controls must be present where required.
- No-edge guard: PASS with explicit reason; no forced best bet.
