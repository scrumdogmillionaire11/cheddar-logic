# WI-0730: Card family registry with operational status field

**ID**: WI-0730
**Goal**: Create an explicit, machine-readable card family registry that assigns every current family a `card_family_status` of `LIVE`, `PROJECTION_ONLY`, or `QUARANTINED`. This registry is the hard guard against projection-only families silently becoming pseudo-live.

**Scope**:
- `WORK_QUEUE/WI-0730.md`
- `apps/worker/src/audit/card-family-registry.json` — new file
- `apps/worker/src/audit/README.md` — update to reference the registry

**Out of scope**:
- Runtime enforcement in model jobs (that is a follow-on WI if needed)
- Web UI changes
- Any changes to fixture or baseline files

**Acceptance**:
- `apps/worker/src/audit/card-family-registry.json` exists and contains an entry for every current card family with at minimum:
  ```json
  {
    "family": "NBA_TOTAL",
    "card_family_status": "LIVE",
    "status_set_date": "2026-04-02",
    "status_rationale": "<one sentence>",
    "odds_backed": true,
    "executable": true
  }
  ```
- Day-one registry matches the approved split:
  - `LIVE`: `NBA_TOTAL`, `NBA_SPREAD`, `NHL_TOTAL`, `NHL_ML`
  - `PROJECTION_ONLY`: `NHL_1P_TOTAL`, `NHL_PLAYER_SHOTS`, `MLB_PITCHER_K`, `MLB_F5_TOTAL`
- No family is missing from the registry.
- The audit `README.md` links to the registry and explains how to promote or quarantine a family.
- Changing a family's status requires a `status_rationale` update and a `status_set_date` bump — not a silent overwrite.

**Owner agent**: unassigned
**Time window**: 2026-04-02 → TBD
**Coordination flag**: solo

**Tests to run**:
- Schema lint: confirm every entry has `family`, `card_family_status`, `status_set_date`, `status_rationale`, `odds_backed`, `executable` fields — a small validation script or `jq` check is sufficient.
- `jq '[.[] | select(.card_family_status == "LIVE")] | length' apps/worker/src/audit/card-family-registry.json` returns `4`.
- `jq '[.[] | select(.card_family_status == "PROJECTION_ONLY")] | length' ...` returns `4`.

**Manual validation**:
- Read each registry entry and confirm the rationale is not a placeholder.
- Confirm no family that is PROJECTION_ONLY has `"executable": true`.
- Confirm the README update makes the promotion/quarantine process clear.

**Failure modes to guard**:
- Do not leave any active card family out of the registry.
- Do not set `card_family_status: LIVE` for any projection-only family without also updating `odds_backed: true` and a written rationale.
