# WI-0876: Suppress duplicate-game false positives in odds freshness watchdog

| Field | Value |
|---|---|
| **ID** | WI-0876 |
| **Goal** | Stop `odds / freshness` Discord alerts from counting duplicate ESPN-derived game rows as separate stale games when a fresh canonical odds-backed row already exists, and reduce future duplicate row creation from ESPN schedule/direct ingest paths. |
| **Scope** | `WORK_QUEUE/WI-0876.md`, `apps/worker/src/jobs/check_pipeline_health.js`, `apps/worker/src/jobs/__tests__/check_pipeline_health.test.js`, `apps/worker/src/jobs/pull_espn_games_direct.js`, `apps/worker/src/jobs/pull_schedule_nba.js`, `apps/worker/src/jobs/pull_schedule_nhl.js`, `apps/worker/src/jobs/__tests__/pull_espn_games_direct.test.js`, `apps/worker/src/jobs/__tests__/pull_schedule_espn_canonicalization.test.js` |
| **Out of scope** | DB migrations, retroactive cleanup/deletion of existing duplicate rows in production, web/API dedupe changes, card/model storage rewrites, non-ESPN ingest sources |
| **Acceptance** | (1) `checkOddsFreshness()` counts one real matchup once even when `games` contains ESPN numeric, `espndirect_*`, and canonical odds-backed IDs for the same matchup/date. (2) If any duplicate row for a matchup has a fresh odds snapshot, the matchup is not reported stale. (3) `pull_espn_games_direct` skips seeding a duplicate `espndirect_*` game/snapshot for active-odds sports when a matching game row already exists for the same matchup/time window. (4) `pull_schedule_nba` and `pull_schedule_nhl` avoid upserting a duplicate ESPN numeric `games` row when a canonical odds-backed match already exists, while still writing `game_id_map`. (5) Focused worker tests cover the watchdog dedupe path and the ESPN ingest duplicate-suppression path. |

## Context

Production `pipeline_health` showed `odds / freshness` failures like `6/11 games within T-2h have stale odds (>75m old)`. Database inspection showed those six stale rows were really three NBA matchups represented by:

- a bare ESPN numeric `game_id`
- an `espndirect_<sport>_<eventId>` `game_id`
- a fresh canonical odds-backed `game_id`

The watchdog currently evaluates raw `games` rows by exact `game_id`, so duplicate identity rows inflate stale counts and trigger false Discord alerts even when `pull_odds_hourly` is healthy.
