## Context

Six constants in `mlb-model.js` are 2024-era values with comments like `// update seasonally` and no automation. The worst offender is `LEAGUE_AVG_K_PCT = 0.225` at line 903, used as a denominator in relative K% scoring (`effectiveStarterKPct * oppKPctVsHand / LEAGUE_AVG_K_PCT`). If the 2026 league K% shifts materially (it increased from ~22% in 2021 to ~23% in 2024), this ratio produces systematically wrong relative edge. `MLB_F5_TEAM_OFFENSE_SPLITS` has no `bb_pct`/`xwoba`/`hard_hit_pct` — so every team that falls back to these defaults uses stale 2024 numbers.

## Data Source

- `kPct`: compute from current-season `pitcher_stats` rows (`strikeouts / batters_faced` average).
- `xwoba`, `hardHitPct`, `bbPct`, `wrcPlus`: compute from `team_offense_profiles` rows if available (populated by the Statcast pull job); else retain static fallback.
- `xfip`: compute from current-season `pitcher_stats` average xFIP if available.

## Dependencies

- No blocking dependencies. Uses existing DB tables.
