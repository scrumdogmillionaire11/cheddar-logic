# WI-0807: Create clv_ledger table and populate CLV on settlement

| Field | Value |
|---|---|
| **ID** | WI-0807 |
| **Goal** | When any card settles, look up the closing line in `odds_snapshots` and write `(card_id, open_price, close_price, clv_pct)` to a new `clv_ledger` table so that `report_telemetry_calibration.js` can measure whether the system beats the closing line. |
| **Priority** | High — CLV is the gold-standard predictor of sustainable edge; table is currently missing entirely |
| **Depends on** | WI-0806 (prop cards must settle to get CLV coverage on props; game markets can be implemented immediately regardless) |
| **Coordination flag** | needs-sync: adds migration `packages/data/db/migrations/061_create_clv_ledger.sql` |

---

## Context

`report_telemetry_calibration.js` already contains a CLV ledger section. It probes for the table and returns `TABLE_MISSING` when the table doesn't exist. The settlement job (`settle_pending_cards.js`) already has a check at line 284:

```js
SELECT name FROM sqlite_master WHERE type = 'table' AND name = 'clv_ledger'
```

...and a query at line 308 reading from it. This implies the CLV capture was planned but the migration and write path were never completed.

`odds_snapshots` stores market prices keyed by `game_id` + `market_type` + `side`. The closing-line snapshot is the row with the latest `scraped_at` timestamp before the game's start time (or the last available snapshot if no post-start row exists).

## Migration DDL (target)

```sql
-- 061_create_clv_ledger.sql
CREATE TABLE IF NOT EXISTS clv_ledger (
  id          INTEGER PRIMARY KEY AUTOINCREMENT,
  card_id     TEXT    NOT NULL UNIQUE,   -- FK to card_results.id
  market_type TEXT    NOT NULL,
  side        TEXT    NOT NULL,          -- 'over','under','home','away'
  open_price  REAL,                      -- American odds at card creation time
  close_price REAL,                      -- Last odds_snapshot price before game start
  clv_pct     REAL,                      -- (devigged close - devigged open) / devigged open * 100
  settled_at  TEXT    NOT NULL DEFAULT (datetime('now'))
);
CREATE INDEX IF NOT EXISTS clv_ledger_market_type ON clv_ledger(market_type);
CREATE INDEX IF NOT EXISTS clv_ledger_settled_at  ON clv_ledger(settled_at);
```

---

## CLV Calculation

```
clv_pct = (fairClose - fairOpen) / fairOpen * 100
```

Where `fairOpen` and `fairClose` are two-sided devigged implied probabilities (same helper added by WI-0805: `twoSidedFairProb(priceA, priceB)`). A positive `clv_pct` means the system found a price before the market moved against it — evidence of genuine edge.

If no closing snapshot exists for a card (scraper missed it), write `NULL` for `close_price` and `clv_pct` rather than failing settlement.
