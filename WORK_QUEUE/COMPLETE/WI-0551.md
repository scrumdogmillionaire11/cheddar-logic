# WI-0551: Remove vig from implied probability calculation

**ID**: WI-0551
**Goal**: Strip bookmaker overround from implied probability before edge comparison so all edge estimates are computed against fair-price baselines rather than biased raw probabilities.

**Scope**:
- `packages/models/src/edge-calculator.js`
- `packages/models/src/__tests__/edge-calculator.test.js` (create if absent)

**Out of scope**:
- Threshold recalibration (separate concern)
- Changes to decision-pipeline-v2.js thresholds or verdict logic
- Any non-edge-calculator consumers until vig removal is validated

**Background**:
At standard -110/-110 pricing, raw implied probability is ~52.4% per side. The overround is ~4.8% total, meaning ~2.4% per side is systematic bias against every edge estimate. The pipeline currently calls `impliedProbFromAmerican` on raw prices, so the effective edge it measures is always ~2.4% lower than the true edge available. This affects every market type (moneyline, spread, total) and all sports.

**Acceptance**:
- A `noVigImplied(priceHome, priceAway)` function exists in `edge-calculator.js` that returns `{ home: number, away: number }` using the standard two-sided vig-removal formula: `p_home_nv = p_home_raw / (p_home_raw + p_away_raw)`.
- `computeMoneylineEdge`, `computeSpreadEdge`, and `computeTotalEdge` all use vig-removed implied probabilities when both sides' prices are available.
- When only one side's price is available, behavior falls back to raw implied probability and logs a `VIG_REMOVAL_SKIPPED` flag in the return object.
- Unit tests confirm: at -110/-110, `noVigImplied` returns `{ home: 0.5, away: 0.5 }` (not 0.524/0.524).
- Unit tests confirm: at -150/+130, `noVigImplied` returns mathematically correct no-vig probabilities.
- Existing tests pass with updated edge values (expected values may need updating to reflect corrected baseline).
