# WI-0650: OWNERSHIP.md contracts for decomposed db.js modules + CI import graph validation

**ID**: WI-0650
**Goal**: Add `OWNERSHIP.md` entries for each domain module produced by WI-0620 (db.js decomposition), and add a CI step that validates no module outside the declared owner's scope imports internal db domain modules directly.

**Background**:
WI-0620 (decompose `db.js` into domain modules) is COMPLETE, but the resulting modules have no ownership contracts in `OWNERSHIP.md`. Without these contracts the decomposition is structurally present but organizationally invisible — new code will gradually re-centralize imports back into the old pattern and the decomposition will erode. The CI import graph check is the enforcement mechanism.

**Scope**:
- `OWNERSHIP.md` (add entries for each new domain db module)
- `.github/workflows/ci.yml` (add import-graph lint step — `needs-sync` flag required)
- `scripts/check-db-imports.js` (new script — validates import boundaries using `grep` or `madge`)

**Out of scope**:
- Changes to any db domain module logic
- Adding new db domain modules (scope creep)

**Acceptance**:
- `OWNERSHIP.md` has a section `## DB Domain Modules` listing each domain module, its owner, and its permitted consumers
- `scripts/check-db-imports.js` exits non-zero if a file outside the permitted consumer list imports a non-public db domain module directly
- CI step `check-db-import-boundaries` runs `node scripts/check-db-imports.js` and fails the build on violation
- `node scripts/check-db-imports.js` exits 0 against the current codebase (no existing violations)

**Coordination flag**: `needs-sync` (`.github/workflows/ci.yml`)
