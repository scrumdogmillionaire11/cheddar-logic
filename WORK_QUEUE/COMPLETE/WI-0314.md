## Diagnostic Findings

### 1. ESPN Enrichment Status
**All ESPN metrics are MISSING from raw_data storage** — no games in any sport have `espn_metrics` in raw_data:
```
NBA:    291 total games,   0 with ESPN metrics (0%)
NHL:    374 total games,   0 with ESPN metrics (0%)
NCAAM: 1436 total games,   0 with ESPN metrics (0%)
```

Despite enrichment being called in the model jobs, ESPN metrics are **not persisted in the database**. The enrichment runs but the enriched data is not saved to `odds_snapshots.raw_data`.

### 2. Driver Card Generation
**Cards ARE being generated successfully** — coverage is nearly 100%:
```
Lowercase sport normalization issue:
- nba:    6 games, 38 cards   (lower case)
- NBA:   20 games,  0 cards   (upper case)
- ncaam: 40 games, 119 cards  (lower case)
- NCAAM: 112 games, 0 cards   (upper case)
- nhl:   13 games, 24 cards   (lower case)
- NHL:   20 games,  0 cards   (upper case)

Card types generated successfully:
- NBA: base-projection(19), rest-advantage(8), matchup-style(18), spread-call(12), total-projection(19), totals-call(16), blowout-risk(9)
- NHL: base-projection(17), model-output(20), pace-totals(11), rest-advantage(7), generic(43), totals-call(14)
- NCAAM: base-projection(224), generic(119), matchup-style(118), rest-advantage(16)
```

**Root cause of "missing drivers" UI: Sport name case mismatch in card_payloads storage.**

### 3. Log Analysis
Model runs show NO enrichment errors or failures:
- **NBA job**: 7 games processed, 38 cards generated, 0 failed
- **NHL job**: 8 games processed, 24 cards generated, 0 failed  
- **NCAAM job**: Team metrics resolution mostly working via scoreboard fallback (ESPN API v2 fallback)

NCAAM team metrics report 4 unresolved teams (Gardner-Webb, CSU Fullerton, Cal Baptist, Long Beach St) but these DO NOT block card generation — drivers gracefully degrade with neutral metrics.

### 4. Team Name Mapping
Team metrics resolution is **working correctly**:
- Resolved teams use scoreboard fallback pattern when ESPN lookup fails
- Example: "Hawai'i Rainbow Warriors" → resolved via NCAAM scoreboard (id: 62)
- Unknown teams (Gardner-Webb Bulldogs, CSU Fullerton Titans, etc.) log warnings but don't block cards

**No systematic normalization failures** — team name mapping is not the issue.

### 5. Root Cause Determination
**Multi-part root cause identified:**

**Primary Issue: Sport Name Case Normalization Bug**
- Games stored in `odds_snapshots.sport` use lowercase ('nba', 'nhl', 'ncaam')
- Cards stored in `card_payloads.sport` use uppercase ('NBA', 'NHL', 'NCAAM')
- UI filters cards by sport match: uppercase sports get no card matches
- Impact: **~80% of games appear to have "drivers missing"** (only lowercase sports have visible cards)

**Secondary Issue: ESPN Enrichment Not Persisted**
- `enrichOddsSnapshotWithEspnMetrics()` is called in model jobs
- Enrichment succeeds and adds `espn_metrics` to raw_data object
- **Enriched data is NOT saved back to database** — odds_snapshots.raw_data never contains ESPN metrics
- Models work around this with fallback fields (raw.pace_home, raw.avg_points_home)
- Impact: Driver models run blind without ESPN context; future features relying on ESPN data will fail

### 6. Recommended Fix

**Priority 1: Fix sport name normalization → [WI-0315](WI-0315.md)**
- Normalize all sport storage to consistent case (uppercase recommended)
- Update card writers and API query to use normalized values
- Migrate existing mixed-case cards to single canonical form
- Expected impact: Restore visible driver cards to UI (~80% of games)

**Priority 2: Persist ESPN enrichment to database → [WI-0316](WI-0316.md)**
- Save enriched ESPN metrics from enrichOddsSnapshotWithEspnMetrics() to odds_snapshots.raw_data
- Currently enrichment runs successfully but return value is discarded
- Ensures models can access ESPN context; currently work around with fallbacks
- Expected impact: Enable ESPN-dependent features (pace analysis, advanced rest calculations)

**Priority 3: Add schema validation → [WI-0317](WI-0317.md)**
- Add CHECK constraint on card_payloads.sport to enforce valid values
- Prevent future case mismatches and invalid sport codes
- Should be completed after WI-0315 migration finishes
- Expected impact: Data integrity; prevents regression
