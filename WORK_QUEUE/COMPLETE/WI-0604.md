# WI-0604: MLB pipeline health — differentiate F5 vs full-game watchdog blocking

**ID**: WI-0604
**Goal**: Make `WATCHDOG_MARKET_UNAVAILABLE` distinguish between a missing full-game total and a missing F5 total so MLB pipeline health reporting is actionable.

**Scope**:

- `apps/worker/src/jobs/run_mlb_model.js` (pipeline state + blocking reason codes)
- `apps/worker/src/jobs/check_pipeline_health.js` (if F5 health check needs wiring)
- `apps/worker/src/__tests__/` (pipeline state tests for MLB)

**Out of scope**:

- Changing the F5 Total projection model (see WI-0602)
- F5 ML ingest (see WI-0603)
- NHL or NBA watchdog changes
- Any UI rendering of pipeline health

**Background**:
All MLB games currently report `WATCHDOG_MARKET_UNAVAILABLE` as a generic blocking reason when running in no-odds mode (`ENABLE_WITHOUT_ODDS_MODE=true`) or when odds haven't been pulled. This is the same code emitted for NHL/NBA. For MLB, this is ambiguous — it could mean the full-game total is missing (irrelevant, we don't use it) or the F5 line is missing (the actual signal we care about). Operators monitoring pipeline health cannot tell if the F5 market is simply unavailable pre-game or if ingest is broken. See `docs/MARKET_REGISTRY.md` MLB gap #3.

**Acceptance**:

- Games missing `total_f5` emit `F5_TOTAL_UNAVAILABLE` as a blocking reason instead of (or alongside) `WATCHDOG_MARKET_UNAVAILABLE`.
- Games missing `ml_f5` (after WI-0600) emit `F5_ML_UNAVAILABLE` as a distinct blocking reason.
- `check_pipeline_health.js` or its MLB equivalent reports F5 availability separately from full-game market availability.
- Existing `WATCHDOG_MARKET_UNAVAILABLE` code is not removed for full-game markets — only augmented for F5.
- Pipeline state logged per-game includes `f5_line_ok: true/false`.
