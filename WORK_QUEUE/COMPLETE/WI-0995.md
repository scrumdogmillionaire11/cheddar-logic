# WI-0995: Restore POTD nominee visibility and no-pick diagnostics

**ID**: WI-0995

**Goal**: Ensure Play of the Day records displayable nominees even when no candidate clears the positive-edge firing gate, so operators can distinguish "no qualified pick" from "POTD did not run."

**Scope**:

- `apps/worker/src/jobs/potd/run_potd_engine.js`

- `web/src/__tests__/api-potd.test.js`

**Acceptance**:

- POTD still fires only when a candidate has positive edge and passes score/stake gates.
- POTD persists a ranked nominee list for no-pick days when candidates exist but none clear the firing gate.
- Nominee listing is deterministic and does not let a negative-edge candidate become the winner.
- The web POTD endpoint can return the no-pick nominee list for the current play date.
- Tests cover positive-edge winner selection and negative-edge nominee persistence.
