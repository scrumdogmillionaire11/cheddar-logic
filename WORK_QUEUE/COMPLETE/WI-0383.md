## Design

### New WATCHDOG_REASONS entries

```js
// In decision-pipeline-v2.js WATCHDOG_REASONS:
GOALIE_UNCONFIRMED: 'GOALIE_UNCONFIRMED',
GOALIE_CONFLICTING: 'GOALIE_CONFLICTING',
```

### Watchdog goalie check logic

```js
// After existing checks, before final action derivation:
const homeStarterState = card.homeGoalieState?.starter_state;
const awayStarterState = card.awayGoalieState?.starter_state;

if (homeStarterState === 'CONFLICTING' || awayStarterState === 'CONFLICTING') {
  reasonCodes.push(WATCHDOG_REASONS.GOALIE_CONFLICTING);
  return { action: 'PASS', reason_codes: reasonCodes };
}

if (homeStarterState === 'UNKNOWN' || awayStarterState === 'UNKNOWN') {
  reasonCodes.push(WATCHDOG_REASONS.GOALIE_UNCONFIRMED);
  // GOALIE_UNCONFIRMED does not force PASS by itself — wrapper action may still be HOLD
  // unless confidence is already below threshold
}
```

### Wrapper eligibility gate

```js
// In decision-publisher.js, NHL totals action derivation:
if (card.official_eligible === false) {
  return 'PASS'; // never FIRE or HOLD when model has marked ineligible
}
```

### Migration invariant for legacy boolean proxies

After this WI, any read of `homeGoalieConfirmed` or `awayGoalieConfirmed` as an authoritative input in the NHL totals path must carry:

```js
// DEPRECATED: not authoritative for NHL totals after WI-0383
// Use homeGoalieState.starter_state instead
```

When WI-0384 FC-7 runs, if the legacy boolean path bypasses the canonical state and produces a non-PASS result, that is a test failure.

