# WI-0411: Eliminate False PASS_MARKET_PRICE_MISSING from Oddsless Pregame Rows

**ID**: WI-0411
**Goal**: Stop emitting misleading `PASS_MARKET_PRICE_MISSING` for pregame rows that have neither odds snapshots nor actionable plays by tightening API row eligibility and using explicit missing-data reason taxonomy.

**Scope**:

- `WORK_QUEUE/WI-0411.md`
- `web/src/app/api/games/route.ts`
- `web/src/lib/game-card/transform.ts`
- `web/src/components/cards-page-client.tsx`

**Out of scope**:

- Worker ingestion contract changes (`TEAM_MAPPING_UNMAPPED`, odds pulls)
- Team variant map expansion in data layer
- Decision threshold tuning
- DB schema changes/migrations

**Acceptance**:

- Pregame API response excludes rows with neither odds nor plays to avoid synthetic no-data cards.
- Transform maps no-odds/no-plays and no-plays outcomes to explicit missing-data taxonomy (`MISSING_DATA_NO_ODDS`, `MISSING_DATA_NO_PLAYS`) instead of `PASS_MARKET_PRICE_MISSING`.
- Diagnostics bucket logic continues to classify missing-data outcomes correctly.
- Existing edge/price gating behavior remains unchanged for rows that do have odds or plays.
