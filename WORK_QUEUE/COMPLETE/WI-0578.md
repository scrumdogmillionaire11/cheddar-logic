# WI-0578 — Fix PP_RATE_MISSING Silent PP-Component Collapse

**ID:** WI-0578
**Goal:** When `PP_RATE_MISSING` is flagged (`ppRatePer60 === null && ppToi > 0`), use a conservative league-average PP shot rate fallback instead of passing null to `projectSogV2`, which silently collapses the PP component to 0 and under-projects by 0.3–0.5 SOG for top PP players.
**Severity:** HIGH
**Owner agent:** bmad-master
**Status:** COMPLETE

## Problem

In `run_nhl_player_shots_model.js`, `v2ProjectionInputs.pp_shots_season_per60` is set to `ppRatePer60`, which is null when NST data is unavailable. `weightedRateBlendPP(null, null, null)` returns `0`, so `pp_component = 0` even for players with meaningful `ppToi` (e.g., 2–3 min/game). This silently under-projects by ~0.3–0.5 SOG vs. a realistic estimate.

The `PP_RATE_MISSING` flag is already added to flags by `buildSharedPropFlags` — the issue is the projection itself, not the signaling.

## Scope

- `apps/worker/src/jobs/run_nhl_player_shots_model.js` — `v2ProjectionInputs` block
- `apps/worker/src/jobs/__tests__/run_nhl_player_shots_model.test.js` — new test

## Out of Scope

- NST ingestion pipeline (WI-0530)
- Changes to `weightedRateBlendPP` itself
- Rolling split ingestion (WI-0531)
