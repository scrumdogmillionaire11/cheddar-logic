### ⏳ Pending (User Actions)

#### Phase 1: Tailscale Org Setup (5–10 min)

1. **Go to ACL Policy**: [https://login.tailscale.com/admin/acls](https://login.tailscale.com/admin/acls)

2. **Add/verify this ACL policy**:

   ```acl
   acls = [
     // Allow GitHub Actions (tag:ci) to deploy to production (tag:servers)
     {
       "action": "accept",
       "src": ["tag:ci"],
       "dst": ["tag:servers:*"],
     },
   ]
   ```

3. **Go to OAuth Credentials**: [https://login.tailscale.com/admin/settings/trust-credentials](https://login.tailscale.com/admin/settings/trust-credentials)

4. **Create OAuth Client**:
   - Click **Credential** → **OAuth**
   - **Scope**: `auth_keys` (writable)
   - **Tags**: Check `tag:ci`
   - Click **Generate credential**
   - **Copy and save** both **Client ID** and **Client Secret** (you won't see the secret again)

5. **Go to Machines**: [https://login.tailscale.com/admin/machines](https://login.tailscale.com/admin/machines)

6. **Tag your Pi**:
   - Find `cheddarpi` (100.82.80.89)
   - Click **Machine settings**
   - Under **Custom routes / tags**, add `tag:servers`
   - Save

#### Phase 2: GitHub Secrets Setup (5 min)

1. **Go to your GitHub repo**: Settings → Secrets and variables → Actions

2. **Add these secrets**:

   | Secret Name | Value |
   | --- | --- |
   | `TS_OAUTH_CLIENT_ID` | `tskey-client-...` (from Phase 1, step 4) |
   | `TS_OAUTH_SECRET` | `tskey-secret-...` (from Phase 1, step 4) |
   | `PI_HOST` | `100.71.1.87` (or `cheddarpi.tail419697.ts.net`) |
   | `PI_USER` | `babycheeses11` (or your Pi SSH user) |
   | `PI_SSH_KEY` | Your existing SSH private key |

3. **Verify all 5 secrets exist** (no typos in names)
