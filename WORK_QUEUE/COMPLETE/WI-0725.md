## Background

The existing pipeline tracks `card_payloads` and `card_results` but has no rolling performance view across sport / market family / execution status / model version. Without this layer:
- A model that fires 80% of the time (executable rate spike) looks healthy in fixture tests
- A model where all FIRE outputs are losing vs closing line value is invisible until manual review
- Calibration decay (predicted 60% win rate actually hitting at 40%) is invisible

This report builds a read-only rolling view against existing DB tables. It does not write to any table.

### 3. Alerts

Alert triggers:
- `executable_rate > 0.6` over any window of 50+ samples â†’ `EXECUTABLE_RATE_SPIKE`
- `pass_rate < 0.2` over any window of 50+ samples â†’ `PASS_RATE_COLLAPSE`
- `calibration` shows adjacent bins diverging by > 0.15 hit rate over same sample window â†’ `CALIBRATION_DIVERGENCE`
- Block rate for any single reason code increases by > 0.1 (absolute) vs prior window â†’ `BLOCK_RATE_SHIFT`

Alert shape:
```json
{
  "alert_type": "EXECUTABLE_RATE_SPIKE",
  "sport": "NBA",
  "market_family": "totals",
  "window": "last_50",
  "value": 0.72,
  "threshold": 0.60,
  "severity": "WARN"
}
```

### 4. Segmentation rule

**Critical invariant**: never mix `execution_status=PROJECTION_ONLY` with `execution_status=EXECUTABLE` in the same metrics bucket. Every bucket must be keyed by `execution_status`.

### 5. DB access rule

This module must use read-only DB access patterns only:
- `getDatabase()` â†’ read-only queries only
- No `exec()`, `stmt.run()`, `closeDatabase()` â€” read-only selects only
- Must respect ADR-0002 single-writer contract
