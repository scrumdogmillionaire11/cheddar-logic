## Root Cause

The per-player processing block has two sibling `if` branches:

```js
if (hasSogLookback) {
  // Shots-on-goal processing (~line 1810ŌĆō3024)
  let blkOppAttemptFactor = 1.0;   // ŌåÉ declared INSIDE hasSogLookback block
  // ... factorRow query assigns blkOppAttemptFactor ...
}

if (blkEnabled && hasBlkLookback) {
  // BLK (blocked shots) processing (~line 3025+)
  // uses blkOppAttemptFactor ŌåÉ ReferenceError for players who skipped the shots block
}
```

A defenseman who has fewer than 5 shot game logs (`hasSogLookback = false`) but has BLK lookback data (`hasBlkLookback = true`) skips the SOG block entirely. `blkOppAttemptFactor` is never declared, so the BLK block throws `ReferenceError: blkOppAttemptFactor is not defined`.

In the current production run this drops all 5 defensemen per game (Ian Cole, Connor Murphy, MacKenzie Weegar, Darnell Nurse, Mikhail Sergachev, etc.).

## Fix

Move `blkOppAttemptFactor` (and the co-located `opponentFactor`, `paceFactor`, `opponentFactorMissing`, `paceFactorMissing`) declarations to the enclosing shared scope ŌĆö BEFORE the `if (hasSogLookback)` guard:

```js
// Before (inside if (hasSogLookback)):
if (hasSogLookback) {
  ...
  let opponentFactor = 1.0;
  let paceFactor = 1.0;
  let blkOppAttemptFactor = 1.0;
  ...
}

// After (hoisted to shared scope ~line 1809, before both if blocks):
let opponentFactor = 1.0;
let paceFactor = 1.0;
let blkOppAttemptFactor = 1.0;
let opponentFactorMissing = false;
let paceFactorMissing = false;
// ... factorRow query block moves here or remains inside hasSogLookback (still sets the variables) ...

if (hasSogLookback) {
  // shots processing ŌĆö factorRow assignment still happens here
}

if (blkEnabled && hasBlkLookback) {
  // blkOppAttemptFactor now in scope
}
```

The factorRow DB query block can remain inside `hasSogLookback` since it's correctly setting the hoisted variables ŌĆö or alternatively, refactor it into a shared lookup that runs once before both branches.

## Notes

- With `NHL_BLK_CARDS_ENABLED=true` this is a complete blind spot ŌĆö no defenseman BLK cards are ever written.
- `opponentFactor` and `paceFactor` are used in the SOG path only and may not need hoisting, but moving them avoids confusion. Only `blkOppAttemptFactor` causes a crash ŌĆö prioritize that.
- Check for the same pattern in any other variables set inside `if (hasSogLookback)` and consumed in `if (blkEnabled && hasBlkLookback)`.
