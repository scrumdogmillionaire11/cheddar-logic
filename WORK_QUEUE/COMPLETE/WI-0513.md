# WI-0513: FPL transfer engine value-first logic refinement plan

**ID**: WI-0513  
**Goal**: Define and lock an implementation-ready transfer recommendation framework that prioritizes value (not just start probability + short-form points), with explicit scoring factors, guardrails, and test requirements.

**Scope**:

- `WORK_QUEUE/WI-0513.md`
- `docs/fpl_transfer_engine_framework.html`
- `docs/fpl_transfer_engine_framework.md` (new, canonical markdown summary for engineering handoff)

**Out of scope**:

- Direct changes to recommendation runtime code in `cheddar-fpl-sage/src/**`
- API contract changes in `web/src/lib/fpl-api.ts`
- UI redesign of transfer/captain screens

**Acceptance**:

- Framework defines must-have vs nice-to-have ranking factors for transfer recommendations.
- Framework includes concrete scoring formula candidates for:
  - expected points (short + medium horizon)
  - value (`xPts / price` style metric)
  - minutes/security risk
  - fixture quality (including blank/DGW handling)
  - hit economics (net gain after cost)
- Framework explicitly replaces hard 75% start-probability gating with a softer risk treatment proposal.
- Framework provides guardrails (hard exclusions), fallback behavior for missing data, and explainability output requirements.
- Framework includes a phased implementation plan (MVP -> v2 -> v3) with acceptance checks.
- Framework includes a verification plan (offline backtest KPIs + online A/B metrics).
