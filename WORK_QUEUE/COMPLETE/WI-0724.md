## Fixture schema (required for all)

```json
{
  "fixture_id": "nba_total_clean_01",
  "sport": "NBA",
  "card_family": "NBA_TOTAL",
  "card_mode": "ODDS_BACKED",
  "description": "Clean total edge ‚Äî full odds, FIRE status, executable",
  "input_contract": "ODDS_SNAPSHOT",
  "match_key": {
    "game_id": "nba_test_game_001",
    "market_type": "TOTAL",
    "selection": "OVER"
  },
  "input": {
    "game_id": "nba_test_game_001",
    "sport": "NBA",
    "home_team": "Boston Celtics",
    "away_team": "Golden State Warriors",
    "game_time_utc": "2026-04-02T00:00:00Z",
    "total": 224.5,
    "total_price_over": -110,
    "total_price_under": -110,
    "spread_home": -7.5,
    "spread_away": 7.5,
    "spread_price_home": -112,
    "spread_price_away": -108,
    "h2h_home": -320,
    "h2h_away": 260,
    "captured_at": "2026-04-01T18:00:00Z"
  },
  "expected": {
    "input_hash": "RECOMPUTE_ON_FIRST_RUN",
    "execution_status": "EXECUTABLE",
    "classification": "PLAY",
    "market_type": "TOTAL",
    "reason_codes_must_not_include": ["PASS_NO_EDGE", "BLOCKED_MISSING_PRICING"]
  },
  "baseline_reviewed": false
}
```

Notes:
- `card_family` tells the comparator which rule set to apply. `card_mode` tells the audit whether price fields are required. `match_key` tells the comparator how to pair expected vs actual cards. All three are required ‚Äî a fixture missing any of them fails schema validation hard.
- `input_hash` should be set to `"RECOMPUTE_ON_FIRST_RUN"` initially. After the first snapshot run, replace with the actual computed hash **and** set `baseline_reviewed: true`. This two-step process prevents first-run output being silently promoted as an approved baseline.
- `expected` fields are the minimum required; additional fields can be added for stricter comparison.

### Canonical `card_family` values

| card_family | card_mode | Sport / lane |
|---|---|---|
| `NBA_TOTAL` | `ODDS_BACKED` | NBA game total |
| `NBA_SPREAD` | `ODDS_BACKED` | NBA spread |
| `NHL_TOTAL` | `ODDS_BACKED` | NHL full-game total |
| `NHL_ML` | `ODDS_BACKED` | NHL moneyline |
| `NHL_1P_TOTAL` | `PROJECTION_ONLY` | NHL first-period total |
| `NHL_PLAYER_SHOTS` | `PROJECTION_ONLY` | NHL player shots-on-goal prop |
| `MLB_PITCHER_K` | `PROJECTION_ONLY` | MLB pitcher strikeout prop |
| `MLB_F5_TOTAL` | `PROJECTION_ONLY` | MLB first-five-innings total |

Rule: `sport` selects the adapter; `card_family` selects the rule set; `card_mode` determines whether price fields are required.

## Acceptance

- All 19 fixtures exist as valid JSON files (5 NBA + 7 NHL + 6 MLB + 1 multi-card)
- All fixtures pass `validateFixtureSchema` (from `fixture_loader.js`)
- Every fixture has `card_family`, `card_mode`, and `match_key` ‚Äî missing any one of these three fields is a hard schema failure, not a warning
- Every fixture has `baseline_reviewed` field (initially `false`)
- `ODDS_BACKED` fixtures include line/price inputs; `PROJECTION_ONLY` fixtures are allowed to omit price fields ‚Äî the schema enforces this rule-set per `card_mode`
- No fixture uses `sport` as the sole routing key (i.e., no fixture may omit `card_family`)
- `loadFixturesForSport('NBA')` returns all 5 NBA fixtures
- `loadFixturesForSport('NHL')` returns all 7 NHL fixtures (including `nhl_multi_card_same_game_01`)
- `loadFixturesForSport('MLB')` returns all 6 MLB fixtures
- No fixture contains live API keys, real user data, or production game IDs
- All `input_hash` fields set to `"RECOMPUTE_ON_FIRST_RUN"` initially; `baseline_reviewed` must remain `false` until hash is replaced after manual review

## Failure Modes & Guards

The agent must explicitly guard against these at schema-validation time:

| Failure mode | Guard |
|---|---|
| Fixture uses `sport` only and omits `card_family` | Hard schema error ‚Äî validator must reject, not warn |
| `PROJECTION_ONLY` fixture includes price fields labeled as required | Warn; do not silently accept blurred semantics |
| `ODDS_BACKED` fixture omits price fields | Hard schema error |
| `baseline_reviewed=true` on any fixture that still has `input_hash=RECOMPUTE_ON_FIRST_RUN` | Hard schema error ‚Äî reviewed flag must wait for a real hash |
| MLB executable game card added to this fixture library | Loader must reject; note that MLB is projection-only in current scope |
| Comparator receives multiple cards from one game with no `match_key` | Must surface a no-match error, not silently drop cards |
| First-run hash auto-promoted as baseline without `baseline_reviewed=true` | CI must block ‚Äî never merge a fixture where hash was auto-set and flag was not manually reviewed |
