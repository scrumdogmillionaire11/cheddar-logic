# WI-0863: Decouple MLB projection-only scheduling from seed freshness

| Field | Value |
|---|---|
| **ID** | WI-0863 |
| **Goal** | Keep `run_mlb_model` schedulable in April projection-only mode when `MLB.active === false`, while reporting MLB seed freshness separately from model freshness and preserving non-actionable projection-only semantics. |
| **Scope** | `apps/worker/src/schedulers/main.js`, `apps/worker/src/schedulers/mlb.js`, `apps/worker/src/jobs/run_mlb_model.js`, `apps/worker/src/jobs/check_pipeline_health.js`, `apps/worker/src/__tests__/scheduler-main-calibration.test.js`, `apps/worker/src/__tests__/check-pipeline-health.mlb.test.js`, `apps/worker/src/jobs/__tests__/run_mlb_model.test.js` |
| **Out of scope** | Re-enabling live MLB odds, changing `/api/cards` filtering, adding actionable MLB totals/moneyline logic, changing FTP/trap-line promotion rules |
| **Acceptance** | (1) Projection-only MLB `run_mlb_model` jobs queue and run without the scheduler enforcing fresh upstream inputs. (2) MLB scheduled jobs carry explicit projection-only metadata (`requireFreshInputs`, `freshnessSourceJobs`, `runMode`). (3) `run_mlb_model` logs and persists structured projection-only runtime context including seed freshness state. (4) Projection-only MLB outputs carry conservative non-actionable / no-market-trust flags. (5) `check_pipeline_health` adds `mlb / seed_freshness` and keeps `mlb / model_freshness` independent. (6) When there are no MLB games within T-6h, seed freshness writes an `ok` row with skipped reason. (7) Focused scheduler, health, and MLB runner tests cover the new behavior. |
| **Owner agent** | codex |
| **Time window** | 2026-04-10 |
| **Coordination flag** | solo |
| **Tests to run** | `npm --prefix apps/worker test -- --runInBand --testPathPattern=\"scheduler-main-calibration|check-pipeline-health.mlb|run_mlb_model.test\"` |
| **Manual validation** | Run the worker against a DB with upcoming MLB games while `MLB.active=false`; confirm `run_mlb_model` and `pull_espn_games_direct` appear independently in `job_runs`, and confirm pipeline health reports separate MLB `seed_freshness` and `model_freshness` rows. |
