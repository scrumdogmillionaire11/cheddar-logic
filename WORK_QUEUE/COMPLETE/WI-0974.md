# WI-0974: Enforce immutable artifact promotion for production deploys

**ID**: WI-0974
**Goal**: Ensure production deploys promote a single immutable artifact built once per commit, eliminating rebuild drift.

**Depends on**: none

**Scope**:
- `.github/workflows/ci.yml`
- `.github/workflows/deploy-production.yml`
- `scripts/deployment-verify.js`
- `docs/processes/release-promotion.md`

**Out of scope**:
- Runtime application feature changes in `web/src/**`
- Database schema migrations

**Acceptance**:
- CI produces one versioned artifact per commit and records commit SHA + artifact digest.
- Production deploy workflow consumes that existing artifact (no production rebuild step).
- Deploy logs include intended commit SHA and artifact digest for auditability.

**Owner agent**: unassigned
**Time window**: 2026-04-16 -> TBD
**Coordination flag**: needs-sync

**Tests to run**:
- `npm --prefix web run lint`
- `npm --prefix web run test:ci:smoke`

**Manual validation**:
- Trigger a deploy from a known commit and confirm production deploy references the exact artifact digest built in CI.
- Confirm deploy job fails if artifact digest lookup is missing.
