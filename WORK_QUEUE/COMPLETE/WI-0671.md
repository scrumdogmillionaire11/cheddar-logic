# WI-0671: Embed fallback BMAD core files in bmad_gsd package

**ID**: WI-0671
**Goal**: Make the `bmad_gsd` package resolve agent/task definition files even when repo-root `.bmad-core` is absent by shipping an embedded fallback core tree inside the package.

**Background**:
After fixing `WI-0670`, the full `cheddar-fpl-sage` suite exposed `tests/test_bmad_gsd_package.py::test_file_access`. The current package hardcodes `BMAD_CORE_ROOT = PACKAGE_ROOT.parent / ".bmad-core"`, but this checkout has no `.bmad-core` directory. The package already ships agent/task registries in Python, so the missing piece is a packaged markdown fallback for file-path resolution.

**Scope**:
- `cheddar-fpl-sage/bmad_gsd/__init__.py`
- `cheddar-fpl-sage/bmad_gsd/_bmad_core/agents/*.md`
- `cheddar-fpl-sage/bmad_gsd/_bmad_core/tasks/*.md`

**Out of scope**:
- Changes to registry metadata in `bmad_gsd/agents.py` or `bmad_gsd/tasks.py`
- Changes to non-BMAD package tests
- Repo-root `.bmad-core` restoration

**Acceptance**:
- `BMAD_CORE_ROOT.exists()` is true in this checkout
- `get_agent_path("gsd-builder")` and `get_task_path("gsd-quick-build")` return existing markdown files
- `list_agents()` and `list_tasks()` return non-empty filesystem-backed registries consistent with package expectations
- `cd cheddar-fpl-sage && python -m pytest tests/test_bmad_gsd_package.py -v` exits `0`
