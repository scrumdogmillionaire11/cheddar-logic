## Problem

The MLB K engine explicitly references two Statcast-derived fields:
- `season_avg_velo` ‚Äî fastball velocity average
- `season_swstr_pct` (`swstr_pct`) ‚Äî swinging strike / whiff rate

Both are stored as `null` unconditionally. The pull job (`pull_mlb_pitcher_stats.js` line 487) contains an explicit comment: "season_swstr_pct / season_avg_velo require Statcast ‚Äî stored null until pull_mlb_statcast is added."

**Current behavior:**
- `swstr_pct` fallback: hardcoded `0.12` (line 1512) ‚Äî league average proxy, flagged via `starter_whiff_proxy`
- `season_avg_velo`: `null` ‚Äî no fallback, silently ignored in the K formula

**Why `0.12` hardcoded is wrong:**
- A high-whiff pitcher (0.18 swstr%) and an average pitcher (0.12) are treated identically
- The `starter_whiff_proxy` missing-input flag fires on every single K card without exception
- This is not a "partial signal" problem ‚Äî it is a "no signal" problem wearing a guard flag costume

**Data source:** Baseball Savant CSV endpoint (`https://baseballsavant.mlb.com/statcast_search/csv?group_by=name&...`) ‚Äî no auth required, same source referenced in WI-0596.

---

## Input proof

`mlb_pitcher_stats` table already has schema columns:
- `season_avg_velo REAL` (NULL in all rows currently)
- `season_swstr_pct REAL` (NULL in all rows currently, alias `swstr_pct`)

Pull job upsert is already structured ‚Äî only the fetch/computation is missing.

---

## Fix spec

**Step 1 ‚Äî Build `pull_mlb_statcast.js`:**
- Fetch current-season pitcher averages from Baseball Savant `/statcast_search/csv` endpoint
- Parse: `player_id`, `pitcher_name`, `avg_velocity` (fastball), `whiff_percent`
- Upsert into `mlb_pitcher_stats`: `season_avg_velo = avg_velocity`, `season_swstr_pct = whiff_percent`
- Schedule: daily (Savant updates after each game day)

**Step 2 ‚Äî Update K engine gate:**
- Remove hardcoded `0.12` fallback for `swstr_pct`
- When `swstr_pct` is non-null: use it directly in K rate formula
- When `swstr_pct` IS null: emit `missing_inputs: ['statcast_swstr']`; cap classification at `LEAN` (not EXECUTABLE for a pure-K card)
- When `season_avg_velo` is non-null: incorporate as a velocity-tier modifier (high velo ‚â• 95mph: small K boost; low velo < 90mph: small K penalty)
- When `season_avg_velo` IS null: no modifier applied, add to `missing_inputs: ['statcast_velo']` but do not block

**Output proof:**
- `raw_data.statcast_inputs.swstr_pct` populated from DB (not hardcoded)
- `raw_data.statcast_inputs.season_avg_velo` populated from DB
- Edge value differs from hardcoded-fallback baseline

**Failure proof:**
- `swstr_pct` null ‚Üí `missing_inputs` populated, card cannot be `EXECUTABLE`
- `season_avg_velo` null ‚Üí noted in `missing_inputs` but does not block by itself
