**ID**: WI-0307

**Goal**: Eliminate contradictory value verdicts ("good value" + "price too steep") by centralizing edge/EV logic in one module.

**Scope**:

- `apps/worker/src/models/edge-checker.js` (new module)
- `apps/worker/src/models/cross-market.js` (integrate edge-checker into decision output)
- `packages/models/src/index.js` (export new value verdict types)
- `web/src/lib/types/game-card.ts` (add ValueStatus enum)
- `web/src/app/api/games/route.ts` (pass through value_verdict, value_reason)
- `web/src/components/game-card.tsx` (render verdict only; no mixed signals)

**Out of scope**:

- UI styling/design (just ensure one verdict renders, not multiple)
- Coinflip tier deletion (confidence tier stays; coinflip is label, not verdict)
- Multi-model handling (separate P2 WI)
- Legacy field cleanup (keep recommendation for backward compat)

**Acceptance**:

- `edge-checker.js` module exports `computeValueVerdict(fair_prob, best_odds, edge_threshold, min_prob_floor, market_stale?) => {verdict, reason, edge, implied_prob}`.
- Verdict is exactly one of: `VALUE_OK`, `NO_VALUE_PRICE_TOO_STEEP`, `NO_VALUE_INSUFFICIENT_EDGE`, `NO_VALUE_INSUFFICIENT_DATA`, `NO_VALUE_MARKET_STALE`.
- Cross-market.js calls edge-checker after building decision; sets verdict + reason on final play output.
- No card has two contradictory verdicts in payload (test via schema validation).
- Rules deterministic: same (fair_prob, odds, thresholds) always yields same verdict.
- Edge math exposed in reason string: "edge 2.3%, implied 54%, fair 56%".

**Manual validation**:

- Sample 20 cards from `/api/games`: each has one verdict (no dupes).
- Verify edge math: `fair_prob - implied_prob == edge` in at least 5 samples.
- Coinflip plays with edge > min threshold: verdict is `VALUE_OK`, not `NO_VALUE_*`.
- Price too steep: plays with `fair_prob < implied_prob` are `NO_VALUE_PRICE_TOO_STEEP`.
