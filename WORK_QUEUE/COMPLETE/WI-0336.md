# WI-0336: Missing-Data Taxonomy + Diagnostics Buckets

**ID**: WI-0336
**Goal**: Separate missing-data vs no-edge outcomes and surface per-sport diagnostics buckets so degraded states are explicit and actionable.

**Scope**:

- web/src/app/api/games/route.ts
- web/src/lib/game-card/transform.ts
- web/src/components/cards-page-client.tsx

**Out of scope**:

- Model logic changes in apps/worker
- Odds ingestion/provider changes
- Database schema changes

**Acceptance**:

- Cards never show "No clear edge found" when drivers are missing.
- Missing-data reason codes are separated from no-edge reasons and are visible in UI.
- Cards view includes a per-sport diagnostics summary with buckets (missing mapping, driver load failed, no odds, no projection).
- Bucket counts are clickable or filterable to show blocked plays.
- The empty main view still shows a compact summary of excluded games.
