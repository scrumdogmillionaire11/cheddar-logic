# WI-0960: Align MLB model expectations with persisted odds coverage

**ID**: WI-0960
**Goal**: Remove the systematic MLB pricing-state mismatch where the model expects executable price inputs that the odds ingest path does not persist for most games.

**Depends on**: WI-0965

**Architecture constraints (must hold)**:

- Execute this WI only after WI-0965 is merged to avoid concurrent edits on `run_mlb_model` execution paths.
- This WI is limited to pricing-readiness contract alignment (ingest coverage vs model blocker semantics); do not change MLB execution-gate threshold policy owned by WI-0965.
- Worker-owned reason/status fields remain source-of-truth for blocker semantics; this WI may refine when reasons are emitted, but must not introduce UI-only blocker meaning.

**Acceptance**:
- The codebase explicitly documents, in code and tests, which MLB markets are required for `pricing_ready=true` and which are optional/degraded.
- If a supported market is required for execution, odds ingest fetches and persists the needed inputs for that market; if unsupported by provider contract, model logic does not emit it as a hard blocker.
- Manual evidence includes before/after `pricing_ready` and blocker-reason snapshots for the same run window. Pass condition:
  - `pricing_ready=true` game count increases from baseline; or
  - if count does not increase, blocker reasons are limited to explicitly documented degraded/unsupported semantics covered by regression tests.
- Regression tests cover both supported-market availability and degraded-market behavior so future ingest/model changes cannot silently reintroduce mismatch semantics.
