# WI-0747: MLB K Pipeline Hardening — Explicit Input Contract + Deterministic Quality Classification

**ID**: WI-0747
**Goal**: Eliminate silent proxy substitution in the MLB pitcher-K model by installing
an explicit four-stage input contract, a deterministic `classifyMlbPitcherKQuality`
function, per-pitcher pre-model completeness logging, and card-level flag deduplication.
Cards that cannot meet FULL_MODEL criteria must emit FALLBACK, not quietly downgrade
to DEGRADED_MODEL while publishing unconstrained projections.

- `WORK_QUEUE/WI-0747.md`

- `apps/worker/src/jobs/run_mlb_model.js` — pre-model audit block + classifier wiring + flag dedup
- `apps/worker/src/audit/audit_invariants.js` — add INV-007
- `apps/worker/src/audit/compare_audit_snapshot.js` — tolerant numeric projection checks

**Acceptance**:
- `classifyMlbPitcherKQuality(inputs)` exists in its own module; accepts structured
  StarterSkillInput + OpponentContactInput + LeashInput; returns `{model_quality, hardMissing, proxies, degraded}`
- `starter_whiff_proxy` present → result is never `FULL_MODEL`
- `IP_PROXY` present → result is never `FULL_MODEL`
- `contact_pct_vs_hand` missing → result is never `FULL_MODEL`
- Pre-model audit block in `run_mlb_model.js` logs, per pitcher,
  `{pitcher, starter_skill_status, opponent_contact_status, leash_status, missing_fields, proxy_fields, quality_before_projection}` before any `k_mean` calculation
- `flags` array in emitted card payload contains only unique values (Set dedup)
- INV-007 in `audit_invariants.js` asserts `model_quality` in `["FULL_MODEL","DEGRADED_MODEL","FALLBACK"]`
  and enforces proxy-exclusion rules
- `mlb_pitcher_k_clean_01.json` fixture passes INV-007 with `model_quality: "FULL_MODEL"`
- `mlb_pitcher_k_proxy_degraded_01.json` fixture passes INV-007 with `model_quality: "FALLBACK"`
- All unit tests pass: `npm --prefix apps/worker test -- --runInBand src/jobs/__tests__/mlb-k-input-classifier.test.js`
- All existing audit tests continue to pass
