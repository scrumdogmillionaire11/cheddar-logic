# WI-0374: Sync Odds Across Game Card Header and Play Section

**ID**: WI-0374
**Goal**: Eliminate the stale-odds lag where game card headers show current odds but individual play sections (especially moneyline plays) show old embedded odds from when the card_payload was created. All odds rendered within a card must come from the same source-of-truth: the game's latest odds snapshot.

**Root Cause**:

- Game card **headers** render odds from `originalGame.odds` (latest `capturedAt` snapshot, polled every 60s).
- Individual **play sections** render `displayPlay.price` / `displayPlay.bet?.as_of_iso` — values embedded in `card_payloads` at model-run time, potentially hours stale.
- Result: header may show ML -140/+120, play section shows ML -160/+135 from an earlier snapshot.

**Scope**:

- `WORK_QUEUE/WI-0374.md`
- `web/src/components/cards-page-client.tsx`

**Out of scope**:

- `web/src/app/api/games/route.ts` (no API shape change needed — game-level odds already returned alongside plays)
- `apps/worker/**` (odds ingest pipeline is not the bug)
- DB migrations
- Polling frequency changes

**Acceptance**:

1. Moneyline odds shown in a play's price/implied-prob display match the header moneyline for that game.
2. The "odds updated" timestamp shown in or near a play reflects `originalGame.odds.capturedAt`, not `displayPlay.bet?.as_of_iso`.
3. When the 60-second poll fires and odds move, both header and play sections update together on the next render cycle.
4. No regression: plays that have no corresponding game-level market (e.g., props, TEAM_TOTAL) fall back gracefully.
5. TypeScript compiles clean: `npx tsc --noEmit --skipLibCheck` passes.

**Implementation notes**:

**A. `displayOddsTimestamp`** — currently:

```ts
const displayOddsTimestamp = displayPlay.bet?.as_of_iso
  ? formatDate(displayPlay.bet.as_of_iso)
  : updatedTime;
```

Change to prefer `originalGame.odds?.capturedAt`:

```ts
const displayOddsTimestamp =
  originalGame.odds?.capturedAt
    ? formatDate(originalGame.odds.capturedAt)
    : displayPlay.bet?.as_of_iso
      ? formatDate(displayPlay.bet.as_of_iso)
      : updatedTime;
```

**B. Play-level price display** — when rendering the moneyline price for a play (the odds number shown inside the play strip, e.g. "-140"), resolve from game-level odds first:

```ts
function resolvePlayPrice(
  play: DisplayPlay,
  marketType: string | undefined,
  selection: { side?: string } | undefined,
  gameOdds: GameData['odds'],
): number | undefined {
  // Prefer live game-level odds for standard markets
  if (gameOdds) {
    if (marketType === 'MONEYLINE') {
      const side = selection?.side ?? play.selection?.side;
      if (side === 'HOME' && gameOdds.h2hHome != null) return gameOdds.h2hHome;
      if (side === 'AWAY' && gameOdds.h2hAway != null) return gameOdds.h2hAway;
    }
    if (marketType === 'SPREAD') {
      const side = selection?.side ?? play.selection?.side;
      if (side === 'HOME' && gameOdds.spreadPriceHome != null) return gameOdds.spreadPriceHome;
      if (side === 'AWAY' && gameOdds.spreadPriceAway != null) return gameOdds.spreadPriceAway;
    }
    if (marketType === 'TOTAL' || marketType === 'PUCKLINE') {
      const side = selection?.side ?? play.selection?.side;
      if (side === 'OVER' && gameOdds.totalPriceOver != null) return gameOdds.totalPriceOver;
      if (side === 'UNDER' && gameOdds.totalPriceUnder != null) return gameOdds.totalPriceUnder;
    }
  }
  // Fallback: embedded play price
  return typeof play.price === 'number' ? play.price : undefined;
}
```

Use this helper wherever the current code reads `displayPlay.price` for display purposes (implied prob chip, odds badge, edge display).

**C. Implied probability** — if `resolvedImpliedProb` is being derived from embedded play odds, prefer recalculating from game-level price:

```ts
const livePrice = resolvePlayPrice(displayPlay, marketType, displayPlay.selection, originalGame.odds);
const resolvedImpliedProb =
  typeof displayPlay.impliedProb === 'number'
    ? displayPlay.impliedProb
    : typeof resolvedDecisionV2?.implied_prob === 'number'
      ? resolvedDecisionV2.implied_prob
      : livePrice != null
        ? impliedProbFromAmericanOdds(livePrice)
        : undefined;
```

(The `impliedProbFromAmericanOdds` helper already exists in `route.ts` — replicate or inline the formula in the client.)
