## Context

As of 2026-04-06 21:34 UTC, the Dr. Claire report showed all 5 sports as 🚨 STALE because `card_payloads.created_at` for the latest cards was ~11 hours old (cards were generated at ~14:40 UTC for evening games). The `pipeline_health` table simultaneously confirmed NBA model ran successfully within last 4h. The signals contradicted each other, making the report misleading.

## Implementation Sketch

In `runSportHealthQuery()`, after querying for `latestCard`, also query:

```javascript
const modelHealthRow = db.prepare(`
  SELECT status, created_at FROM pipeline_health
  WHERE phase = ? AND check_name = 'model_freshness'
  ORDER BY created_at DESC LIMIT 1
`).get(sport);

const modelFreshMs = modelHealthRow
  ? new Date(modelHealthRow.created_at).getTime()
  : null;
const modelIsOk = modelHealthRow?.status === 'ok'
  && modelFreshMs
  && (Date.now() - modelFreshMs) < 4 * 60 * 60 * 1000; // within 4h
```

Then pass `modelIsOk` into `assignStatus()` as a third argument and skip the stale check when `modelIsOk === true`.
