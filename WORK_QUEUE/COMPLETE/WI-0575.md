# WI-0575 — Direction-Aware opportunity_score in NHL Player Shots Model

**ID:** WI-0575
**Goal:** Fix `opportunity_score` always computed using OVER direction regardless of V1 play direction; an UNDER call incorrectly shows a positive OVER opportunity_score, contradicting the bet.
**Severity:** CRITICAL
**Owner agent:** bmad-master
**Status:** COMPLETE

## Problem

`projectSogV1` (and related scoring in `nhl-player-shots.js`) always computed `opportunity_score` using the OVER-direction formula:
```
opportunity_score = 0.45 * edge_over_pp + 0.20 * ev_over + 0.20 * (sog_mu - market_line) + ...
```
When the V1 decision was actually an UNDER play, `sog_mu < market_line` makes `(sog_mu - market_line)` negative, and `edge_over_pp` / `ev_over` have the wrong sign — producing a contradicting positive score for an UNDER call.

## Scope

- `apps/worker/src/models/nhl-player-shots.js` — `projectSogV1` and `projectBlkV1` opportunity_score blocks

## Out of Scope

- V1/V2 model replacement  
- Any other fields in the model

## Acceptance

- [x] `opportunity_score` uses `edge_under_pp`/`ev_under`/`(market_line - sog_mu)` when `play_direction === 'UNDER'`
- [x] `opportunity_score` uses `edge_over_pp`/`ev_over`/`(sog_mu - market_line)` for OVER (default)
- [x] Both full-game (`projectSogV1`) and BLK model (`projectBlkV1`) updated
- [x] Comment labels updated: `// ---- OpportunityScore (direction-aware, WI-0575) ----`

## Implementation

Both `opportunity_score` blocks in [apps/worker/src/models/nhl-player-shots.js](../apps/worker/src/models/nhl-player-shots.js) now branch on `play_direction`:

```js
if (play_direction === 'UNDER') {
  opportunity_score = 0.45 * edge_under_pp + 0.20 * ev_under + 0.20 * (market_line - sog_mu) + ...
} else {
  opportunity_score = 0.45 * edge_over_pp + 0.20 * ev_over + 0.20 * (sog_mu - market_line) + ...
}
```
