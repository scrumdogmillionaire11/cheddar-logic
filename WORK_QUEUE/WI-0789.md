## Context

### Part A — Jest structural migration

nba-availability-gate.test.js and nhl-goalie-state.test.js both use a custom self-contained
assertion loop with zero Jest test()/it() blocks. Internal logic passes
(nba-availability-gate: 6 passed, 0 failed) but Jest marks the suite failed.
Migration is purely structural — no assertion logic changes.

### Part B — NHL injury availability gate (new)

run_nba_model.js has buildNbaAvailabilityGate which gates tier output on key-player
availability (OUT → LEAN cap, DTD → uncertain flag, no rows → missing_inputs).
run_nhl_model.js has no equivalent gate. sync_nhl_player_availability.js already
populates player_availability rows (ACTIVE / INJURED / DTD) with proper Jest tests.

Add buildNhlAvailabilityGate to run_nhl_model.js that:
- Reads player_availability rows for home/away NHL teams via getPlayerAvailabilityByTeam
- Flags INJURED key players → caps tier at LEAN, adds 'key_player_out' to missing_inputs
- Flags DTD key players → adds 'key_player_uncertain' to missing_inputs, does NOT cap tier
- No availability rows for either team → adds 'nhl_availability_unresolved' to missing_inputs
- Missing data handled gracefully (gate skipped, model continues — never throws)

Define NHL_IMPACT_PLAYERS inline (parallel to NBA_IMPACT_PLAYERS): a curated set of
top-impact skaters and starting goaltenders whose absence materially affects model output.
