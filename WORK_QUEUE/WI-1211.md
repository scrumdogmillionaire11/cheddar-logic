# WI-1211: Add Regression Guard for "Settle Only Surfaced Plays" Policy Exceptions

**ID**: WI-1211
**Goal**: Add explicit regression coverage that enforces "only surfaced plays settle" policy boundaries, including market-specific exceptions, to prevent false or duplicate settlement on non-displayed plays.

**Scope**:

- `apps/worker/src/jobs/settle_pending_cards.js`
- `apps/worker/src/jobs/__tests__/settle_pending_cards.phase2.test.js`
- `apps/worker/src/jobs/__tests__/` (new targeted settlement policy test file(s), if needed)

**Out of scope**:

- Changing display-log ingestion architecture
- Refactoring projection settlement jobs outside test-targeted fixes
- Reworking `/api/results` display filtering behavior

**Acceptance**:

- New regression tests cover non-displayed settlement policy with clear expected outcomes:
  - non-displayed non-exception markets do not settle
  - displayed markets settle when final results exist and grading contract is valid
  - exception pathways (if intentional, e.g. `mlb-pitcher-k`) are explicitly tested and documented
- Tests assert no duplicate settlement writes for the same logical market row in the same run path
- Tests assert no false settlement on rows that were never surfaced and are not in an approved exception class
- Any intentional exception remains explicit in test names and assertions (no implicit behavior)
- Existing settlement tests continue to pass
