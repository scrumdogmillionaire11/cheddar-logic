# WI-1022: NBA Phase 2B — Lineup-aware injury impact (role-class + redistribution)

**ID**: WI-1022
**Goal**: Replace the flat `key_player_out` flag in the NBA availability gate with a role-class-aware, redistribution-informed point impact estimate so the model correctly adjusts total projections based on who is missing, not just that someone is missing.

**Depends on**: WI-1020 (bias loop should be live before adding more complexity to the projection path)

**Scope**:
- `apps/worker/src/jobs/run_nba_model.js` (`buildNbaAvailabilityGate`, `applyNbaImpactGateToCard`)
- `apps/worker/src/__tests__/nba-availability-gate.test.js`
- `apps/worker/src/__tests__/nba-team-context.test.js`
- `apps/worker/src/__tests__/run-nba-model.test.js`

**Out of scope**:
- External player-minutes data sources (no new APIs)
- Full usage-redistribution modeling requiring shot chart data
- NHL or MLB availability models
- UI changes

**Acceptance**:
- `buildNbaAvailabilityGate` classifies each impact player into a role class based on available signals:
  - `OFFENSIVE_HUB`: high assists or primary ball handler (proxy: `assists_last5 > 4` or `usage_pct > 28` if available, else `avg_points_last5 > 22`)
  - `RIM_PROTECTOR`: defensive big (proxy: `blocks_last5 > 1.5` if available, else `avg_points_last5 < 12 && starts_last5 >= 3`)
  - `REBOUNDER`: high rebound share (proxy: `rebounds_last5 > 8` if available)
  - `SPACER`: perimeter role (everyone else who starts)
  - `BENCH`: `starts_last5 == 0`
- Role-class impact multipliers applied to `avg_points_last5`:
  - `OFFENSIVE_HUB`: 1.20 (pace often increases, scoring efficiency drops → total goes down more)
  - `RIM_PROTECTOR`: 0.60 (minimal scoring impact, but opponent gets easier looks → small total bump; net effect modeled as slight reduction to avoid over-correction)
  - `REBOUNDER`: 0.70
  - `SPACER`: 0.85
  - `BENCH`: 0.20
- `startRatio = Math.min(1, (starts_last5 ?? 0) / 5)` applied on top of role multiplier.
- `pointImpact = avg_points_last5 * roleMultiplier * startRatio`
- Redistribution adjustment: when an `OFFENSIVE_HUB` is out, apply a `paceSurgeOffset` of 0.5 points against that team's raw injury impact because increased pace can partially offset lost scoring efficiency. Compute `teamImpactAfterRedistribution = Math.max(0, rawTeamImpact - 0.5)`. Apply only when a hub is missing AND at least 2 active starters remain.
- Team-level cap after the offset: `cappedTeamImpact = Math.min(teamImpactAfterRedistribution, 12)`.
- Net total reduction applied to `projection.total`: `reduction = (cappedHomeImpact + cappedAwayImpact) * 0.5`.
- `raw_data.injury_projection_reduction` stamped with full breakdown: `{ home_impact, away_impact, reduction_applied, role_classes }`.
- Tier downgrade from FIRE/WATCH → LEAN for `key_player_out` is preserved.
- Unit tests cover all role classes, redistribution path, and the team cap.

**Role classification with fallback chain**:

If player metadata fields beyond `avg_points_last5` and `starts_last5` are unavailable (which is likely for most of the current season), the classifier falls back to a points-only heuristic:

- `avg_points_last5 > 22` → treat as `OFFENSIVE_HUB`
- `avg_points_last5 >= 10` → treat as `SPACER`
- `avg_points_last5 < 10 && starts_last5 >= 3` → treat as `RIM_PROTECTOR`
- else → `BENCH`

The role_class assigned must be logged in `raw_data.injury_projection_reduction.role_classes` so it is auditable.

**What this WI explicitly does NOT do**:

Usage redistribution to surviving teammates is not modeled here. That requires minutes data per player per game, which is not currently stored in `team_metrics_cache`. If that data becomes available, a follow-on WI can add it. The current implementation models the point impact of the missing player only, with role-class multipliers as a proxy for redistribution effects.
