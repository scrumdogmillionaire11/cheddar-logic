# WI-0668: Natural Language Intent Translation Layer

**ID**: WI-0668
**Goal**: Build the core interaction layer for the Draft Lab. Free-text input must resolve to one or more typed intents, return a confidence signal, and never silently succeed on ambiguous input. This is not a helper feature — it is the surface through which users modify their build. Treat it as core infrastructure.

## Required behavior

### 1. Always return exactly one of three states

Every translation response must be one of:

- `mapped` — one or more intents resolved with high confidence; execute immediately
- `partial` — intent(s) identified but confidence is medium/low; surface confirmation before executing
- `rejected` — no recognizable intent class; return structured error with suggestions

No silent guessing. No silent no-ops. No fallback to partial execution on ambiguity.

### 2. Multi-intent parsing (required, not optional)

A single input may resolve to multiple intents. All must be extracted and returned as an array.

Example input: "keep Salah but make the rest safer and improve bench"

Expected output:

    {
      "status": "mapped",
      "intents": [
        { "type": "lock_player",       "params": { "player": "Salah" }, "confidence": "high" },
        { "type": "risk_down",         "params": {},                    "confidence": "high" },
        { "type": "bench_priority_up", "params": {},                    "confidence": "high" }
      ]
    }

Each intent in the array is independently executable. The caller executes them in order.

### 3. Confidence scoring per intent

Every resolved intent carries a confidence level:

- `high` — clear match; execute immediately
- `medium` — probable match; surface confirmation UI before executing
- `low` — weak match; surface clarification prompt; do not execute without confirmation

If any intent in the array is `medium` or `low`, the overall `status` becomes `partial`.

### 4. Reversibility (in this WI)

`DELETE /api/v1/draft-lab/{session_id}/intents/last` removes the most recently applied intent and returns the pre-undo session state. Returns 404 if no intents have been applied to the session.

## Intent mapping coverage (v1 minimum)

| Phrase class | Resolved intent |
| --- | --- |
| "make this safer", "too risky", "lower risk", "de-risk" | `risk_down` |
| "go more aggressive", "need upside", "attack mode" | `risk_up` |
| "I don't trust X", "drop X", "ban X", "avoid X" | `ban_player` |
| "keep X", "lock X in", "must start X", "don't change X" | `lock_player` |
| "stronger bench", "bench coverage", "more bench depth" | `bench_priority_up` |
| "lighter bench", "sacrifice bench" | `bench_priority_down` |
| "double Arsenal", "too many X", "exposure concerns" | `ban_team_combo` |
| Combined phrases | multi-intent array |
| No recognizable phrase | `rejected` with suggestions |

## API contract shapes

`POST /api/v1/draft-lab/translate` accepts `{ "session_id": "<uuid>", "input": "<free text>" }`.

Mapped response:

    { "status": "mapped", "intents": [{ "type": "<intent>", "params": {}, "confidence": "high" }] }

Partial response:

    { "status": "partial", "intents": [{ "type": "<intent>", "params": {}, "confidence": "medium" }], "confirmation_prompt": "Did you mean: lower your risk profile?" }

Rejected response:

    { "status": "rejected", "message": "No supported intent recognized.", "suggestions": ["Try: 'make this safer', 'lock Salah', 'stronger bench'"] }

---

**Acceptance**:

- Translate endpoint returns `mapped`, `partial`, or `rejected` — nothing else.
- Multi-intent input produces an array of all resolved intents, not just the first match.
- `partial` is returned when any intent in the array has confidence `medium` or `low`.
- `rejected` always includes non-empty `suggestions`.
- `DELETE /draft-lab/{session_id}/intents/last` returns pre-undo state and 404 if no intents applied.
- No LLM calls, no external API dependencies.
- `pytest tests/test_intent_translator.py` passes covering: all 7 phrase classes, multi-intent sentence, partial confidence path, rejected path, undo endpoint, case insensitivity, empty input string. Minimum 20 test cases.
