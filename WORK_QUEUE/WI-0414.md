# WI-0414: NCAAM FT-Trend Decision Semantics (PLAY/LEAN vs PASS)

**ID**: WI-0414
**Goal**: Redefine decision/status behavior for `ncaam-ft-trend` spread cards so qualifying FT-trend plays present as actionable (`PLAY` or `LEAN`) instead of defaulting to `PASS`, while preserving risk controls and avoiding stale lock contradictions.

**Migration note**: Renumbered from WI-0396 due to ID collision with COMPLETE.

**Out of scope**:

- Broad UI redesign outside FT-trend status presentation

**Acceptance**:
- For `ncaam-ft-trend` spread cards that meet rule criteria, canonical decision status is no longer forced to `PASS` by default pricing/edge flow.
- A deterministic mapping is documented and implemented for FT-trend cards:
  - qualifying signal -> `PLAY` or `LEAN`
  - blocked/risk-invalid signal -> `PASS` with explicit reason code
- `decision_v2.direction` and recommendation side remain aligned with FT-advantaged team after publish/gating.
- Cards page shows status/badge behavior that matches persisted decision status for FT-trend cards.
- Regression tests cover at least:
  - FT-trend qualifying case resulting in non-PASS status
  - FT-trend blocked case remaining PASS with reason code
  - No regression to legacy `ncaam-ft-spread` compatibility
