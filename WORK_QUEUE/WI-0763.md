# WI-0763: Wire fetched-but-unused columns from `mlb_pitcher_game_logs` into the K engine

|--------------------|-------|
| **ID**             | WI-0763 |
| **Goal**           | Wire `walks` and `batters_faced` from `mlb_pitcher_game_logs` into a BB% weighting factor in the MLB K engine, and wire `home_away` into a home/road split adjustment — eliminating three wasted data fetches and closing a known model signal gap |
| **Priority**       | Medium (API quota waste + missed model signals) |
| **Owner agent**    | Unassigned |

## Problem

`pull_mlb_pitcher_stats.js` fetches and stores 5 columns that the MLB K engine never reads:

| Column | Stored by | Read by model | Signal value |
|--------|-----------|---------------|--------------|
| `walks` | `pull_mlb_pitcher_stats.js` | Never | BB% denominator |
| `batters_faced` | same | Never | BB% denominator |
| `earned_runs` | same | Never | Low — ERA proxy |
| `home_away` | same | Never | Home/road split |
| `hits` | same | Never | Low — H/9 proxy |

The MLB model's game log read query (around `run_mlb_model.js` line 833):

```sql
SELECT season, game_date, strikeouts, number_of_pitches, innings_pitched
FROM mlb_pitcher_game_logs
```

`walks`, `batters_faced`, `home_away`, `hits`, and `earned_runs` are all absent.

**Impact:**
- MLB Stats API quota used to fetch data that contributes zero signal
- `walks/batters_faced` → BB% is a known K rate predictor (high BB% pitchers have lower true K rates per batter)
- `home_away` → home/road K splits exist for most starters and are ignored

## Scope

| File | Change |
|------|--------|
| `apps/worker/src/jobs/run_mlb_model.js` | Expand game log SELECT to include `walks`, `batters_faced`, `home_away`; compute `bb_pct = walks / batters_faced` per start; apply BB% adjustment in K rate projection (down-weight when bb_pct > threshold, e.g. 0.10) |
| `apps/worker/src/jobs/run_mlb_model.js` | Add `home_away` split: compute weighted K rate separately for H and A starts when `>= 3` home and `>= 3` away starts exist in the window; blend with overall K rate using configurable weight |
| `apps/worker/src/jobs/run_mlb_model.js` | Add inline comment explaining why `hits` and `earned_runs` remain unread (low K rate signal value — document intentionally excluded) |
| `apps/worker/src/__tests__/` | Unit test: `bb_pct` adjustment reduces K projection when walk rate is high; home/road split is applied when sample is sufficient |

## Out of Scope

- `hits` and `earned_runs` — document as intentionally excluded; do not wire
- Changes to `pull_mlb_pitcher_stats.js` fetch logic — columns are already stored correctly
- Schema migrations — no new columns needed
