# WI-0839: NHL 1P Sigma Static Gate — Label or Suppress 1P Cards When sigma1p Is Uncalibrated

| Field | Value |
|---|---|
| **ID** | WI-0839 |
| **Goal** | The NHL first-period model uses `sigma1p = 1.26` (hardcoded in `nhl-pace-model.js:288`). Unlike game totals, 1P sigma has no empirical derivation, no seasonal update path, and no provenance label on the card. Gate 1P PLAY cards to LEAN when fewer than 40 settled first-period game results exist in `game_results`, and add `sigma_1p_source: 'static'` to all 1P card payloads until a live-computed alternative exists. |
| **Scope** | `apps/worker/src/models/nhl-pace-model.js`, `apps/worker/src/jobs/run_nhl_model.js` |
| **Out of scope** | Computing empirical sigma for 1P (future work requiring `first_period_score_home` / `first_period_score_away` in `game_results`), other period models |
| **Acceptance** | (1) All 1P card payloads carry `sigma_1p_source: 'static'` in `payloadData`. (2) When `game_results` has fewer than 40 rows with non-null `first_period_score_home`, any 1P card with `kind === 'PLAY'` is downgraded to `kind === 'LEAN'` and tagged `reason_codes: ['SIGMA_1P_INSUFFICIENT_HISTORY']`. (3) `[SIGMA_1P]` log line emitted once per NHL model run with settled 1P count and source. (4) Test: assert PLAY → LEAN downgrade when settled count < 40. |
| **Owner agent** | unassigned |
| **Time window** | unassigned |
| **Coordination flag** | none |
| **Tests to run** | `npm --prefix apps/worker test -- --testPathPattern=run_nhl_model`; `npm --prefix apps/worker test -- --testPathPattern=nhl-pace-model` |
| **Manual validation** | On a fresh DB run (empty game_results), confirm no 1P PLAY cards emitted; confirm they carry `sigma_1p_source` field |

## Context

`normalCDF(1.5, projection, sigma1p)` at [apps/worker/src/models/nhl-pace-model.js](apps/worker/src/models/nhl-pace-model.js#L658) uses `sigma1p = 1.26` — a constant with no citation, no calibration history, and no season-based validation. The outer game sigma now has empirical derivation via `computeSigmaFromHistory` (WI-0773), but 1P was not addressed. A miscalibrated sigma directly inflates implied probability coverage, producing false edge on 1P markets.

## Dependencies

- WI-0773 (done) — established the computeSigmaFromHistory pattern for game totals.
- WI-0814 (sigma gate, game sigma) — complements; can run in parallel.
