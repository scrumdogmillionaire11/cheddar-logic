# WI-0669: Final Recommendation Terminal Output

**ID**: WI-0669
**Goal**: Every draft workbench flow must terminate with an unambiguous "Final Recommendation" screen driven entirely by backend output. User leaves with a named best team, the primary reasons it was selected for their profile, and 1–2 enumerated alternatives with explicit tradeoffs. No open questions, no "here are some options to consider."

**Depends on**:
- WI-0656 (draft audit scoring + comparison API must exist)
- WI-0660 (draft workbench shell must exist)

**Scope**:
- `WORK_QUEUE/WI-0669.md`
- `cheddar-fpl-sage/backend/routers/draft_sessions.py` (add `POST /draft-sessions/{id}/recommend` endpoint)
- `cheddar-fpl-sage/backend/services/recommendation_service.py`
- `cheddar-fpl-sage/backend/models/recommendation_models.py`
- `cheddar-fpl-sage/tests/test_recommendation_service.py`
- `web/src/components/fpl-final-recommendation.tsx`
- `web/src/__tests__/fpl-final-recommendation-contract.test.js`

**Out of scope**:
- Transfer/chip recommendations (post-GW1 scope)
- Price change predictions
- Any multi-gameweek planning surface
- Changes to the audit or comparison endpoints

**Backend contract — `POST /draft-sessions/{id}/recommend` response shape**:

```json
{
  "recommendation": {
    "label": "Your best team",
    "session_id": "<uuid>",
    "profile_archetype": "<archetype>",
    "verdict": "<1–2 sentence plain-English summary>",
    "primary_reasons": ["<reason 1>", "<reason 2>", "<reason 3>"],
    "alternatives": [
      {
        "label": "Alternative A",
        "session_id": "<uuid or null>",
        "tradeoffs": {
          "improves": ["<axis>: <delta>"],
          "worsens": ["<axis>: <delta>"]
        }
      }
    ]
  }
}
```

All fields set by backend. Frontend renders, does not compute.

**UI requirements**:
- Rendered as the final step in the `FPLProductShell` flow — non-skippable once audit is complete.
- Primary reasons rendered as a short list (max 3).
- Each alternative shows what it gains and what it costs in plain terms — no raw numbers without labels.
- A "Lock This Team" CTA that records the selection (fires a `decision_receipt` via WI-0658 contract). Stub is acceptable if WI-0658 is not yet wired in this session.
- Mobile-width must be usable without horizontal scroll.

**Acceptance**:
- `POST /draft-sessions/{id}/recommend` returns the contract above; returns 400 if session has no completed audit.
- `fpl-final-recommendation.tsx` renders recommendation, primary reasons, and alternatives from props only — no fetch logic inside the component.
- "Lock This Team" renders and does not crash (can be a stub no-op in v1).
- `pytest tests/test_recommendation_service.py` passes (min 12 tests: archetype-aware reasoning, alternatives enumeration, missing-audit 400 path).
- `node web/src/__tests__/fpl-final-recommendation-contract.test.js` passes.
