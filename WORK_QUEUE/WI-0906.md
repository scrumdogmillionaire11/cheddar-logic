**Must-haves (goal-backward)**:

**Truths**:

- Every surfaced stake can be traced to a concrete edge measurement and confidence value.
- Any path marked non-actionable, downgraded, or fragile-confidence yields zero stake unless policy override is explicit and logged.
- POTD sizing and non-POTD sizing have parity for continuity guard intent, or differences are documented and justified.
- The continuity map is reproducible from source files and test evidence.

**Artifacts**:

- `docs/audits/calibration-staking-continuity.md` contains end-to-end lineage table and break taxonomy.
- `risk_analysis.ipynb` contains a reproducible notebook section validating one downgraded/non-actionable sample path.
- Worker test evidence references continuity guard assertions for both POTD and non-POTD flows.

**Key links**:

- Edge computation output -> confidence calibration input.
- Confidence output -> stake sizing input.
- Execution quality/eligibility gate -> final stake amount.
- POTD signal score/tier -> POTD stake sizing decision.

**Acceptance**:

- Audit map identifies where edge is measured, assumed, converted to confidence, and converted to size. (WI-0906-REQ-01)
- Continuity breaks are listed where size can be non-zero despite failed qualification quality. (WI-0906-REQ-02)
- POTD and non-POTD paths are compared for continuity parity. (WI-0906-REQ-03)
- Remediation list includes guards for zero-size on non-actionable or fragile-confidence paths. (WI-0906-REQ-04)
- At least one deterministic regression check exists for each break class in remediation list.
