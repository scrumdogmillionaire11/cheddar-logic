## Background

The evaluator and DB layer will exist after WI-0864 and WI-0865. This WI wires them into production: every time a projection is settled, the proxy-line grades are computed and stored immediately. The accuracy report job provides a scheduled read surface so operators can see whether the model is directionally accurate without touching the DB manually.

`settle_projections.js` already loops over unsettled cards and writes `actual_result`. This WI adds a post-write call for families that have proxy lines.

### After each successful `actual_result` write

After the existing line that calls `setProjectionActualResult(card.card_id, actualResultObj)` (signature: cardId string, result object â€” no db arg), add (define `actualResultObj` using the same value passed to `setProjectionActualResult`, e.g. `{ goals_1p: goals1p }` for NHL 1P):

```js
// Persist proxy-line grades if this card family has proxy lines.
// card_type is the DB column (e.g. 'mlb-f5'); CARD_TYPE_TO_FAMILY maps it to
// the evaluator family key (e.g. 'MLB_F5_TOTAL').
// model_projection lives inside payload_data.projected_total, not as a top-level card field.
// actual_result is the JSON object we just wrote â€” pass as JSON string.
const proxyRows = buildProjectionProxyMarketRows({
  card_id: card.card_id,
  game_id: card.game_id,
  game_date: card.game_time_utc?.slice(0, 10),   // YYYY-MM-DD; game_time_utc is the actual field
  sport: card.sport,
  card_family: CARD_TYPE_TO_FAMILY[card.card_type],  // e.g. 'MLB_F5_TOTAL'
  model_projection: payload?.projected_total ?? null, // parsed from payload_data above the loop
  actual_result: JSON.stringify(actualResultObj),     // object built per card_type in the handler above
});
if (proxyRows.length > 0) {
  try {
    batchInsertProjectionProxyEvals(db, proxyRows);
  } catch (proxyErr) {
    // Non-fatal â€” log and continue. Do not re-throw.
    console.error('[settle_projections] proxy eval insert failed', card.id, proxyErr?.message);
  }
}
```

The proxy eval insert is non-fatal. If it fails, the primary `actual_result` write already succeeded. Log and continue.

### Implementation notes

- Only call this for `MLB_F5_TOTAL` and `NHL_1P_TOTAL` card families. `buildProjectionProxyMarketRows` already returns `[]` for unknown families, so the `if (proxyRows.length > 0)` guard is sufficient.
- Do NOT wrap the proxy eval insert in the same transaction as `actual_result` â€” proxy evals are derived data, not source of truth.
- `payload` is already parsed at the top of each card iteration: `const payload = typeof card.payload_data === 'string' ? JSON.parse(card.payload_data) : card.payload_data;` â€” use `payload.projected_total` for `model_projection`.
- `actualResultObj` is the plain object passed to `setProjectionActualResult` (e.g. `{ goals_1p: 3 }`). Do not re-parse from DB; pass it directly as `JSON.stringify(actualResultObj)` to `buildProjectionProxyMarketRows`.
- The `game_date` field must be a `YYYY-MM-DD` string. Check how `game_date` is available on `card` in the loop; derive it from `game_time_utc`: `card.game_time_utc?.slice(0, 10)`.

## Part 2: Add `run_projection_accuracy_report.js`

### Purpose

Reads the last 90 days of settled proxy eval rows, computes a per-family summary, and emits structured JSON to stdout/console. Single-purpose, no DB writes, no side effects.

### File: `apps/worker/src/jobs/run_projection_accuracy_report.js`

```js
'use strict';

/**
 * run_projection_accuracy_report
 *
 * Generates a proxy-line accuracy report for settled MLB F5 and NHL 1P
 * projection cards. Reads from projection_proxy_evals, logs a structured
 * summary per card family, and returns the summary object.
 *
 * Not scheduled by default. Run on-demand:
 *   node -e "require('./apps/worker/src/jobs/run_projection_accuracy_report').run()"
 *
 * Or call from report_settlement_health.js as a follow-on.
 */

const { openDatabase } = require('@cheddar-logic/data');
const { getProjectionAccuracySummary } = require('@cheddar-logic/data/db/projection-accuracy');

const REPORT_FAMILIES = ['MLB_F5_TOTAL', 'NHL_1P_TOTAL'];
const DEFAULT_LOOKBACK_DAYS = 90;

async function run({ lookbackDays = DEFAULT_LOOKBACK_DAYS } = {}) {
  const db = openDatabase(process.env.CHEDDAR_DB_PATH);

  const gameDateGte = new Date(Date.now() - lookbackDays * 24 * 60 * 60 * 1000)
    .toISOString()
    .split('T')[0];

  const reports = [];

  for (const cardFamily of REPORT_FAMILIES) {
    const summary = getProjectionAccuracySummary(db, { cardFamily, gameDateGte });
    reports.push(summary);

    console.log(
      JSON.stringify({
        event: 'PROJECTION_ACCURACY_REPORT',
        card_family: cardFamily,
        lookback_days: lookbackDays,
        game_date_gte: gameDateGte,
        ...summary,
      })
    );
  }

  return reports;
}

module.exports = { run };

if (require.main === module) {
  run().catch((err) => {
    console.error('[run_projection_accuracy_report] fatal', err);
    process.exit(1);
  });
}
```
