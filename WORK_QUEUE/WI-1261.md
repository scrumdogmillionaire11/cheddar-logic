# WI-1261: Align Cards Surface With Decision States

**ID**: WI-1261

**Goal**: Make `/cards` the canonical opportunity surface for all card-backed decision states so official opportunities, monitored non-play opportunities, and diagnostic states are visible without redefining POTD.

**Context**:

The current web surface still behaves as if only official `PLAY` outcomes matter, even though card payloads already carry canonical `decision_v2` and `decision_outcome` data. This creates a mismatch between the worker's decision system and what users can actually see on `/cards`.

`/cards` should surface all card-backed opportunities with clear labels:

- `PLAY` = official actionable opportunity
- `LEAN` / `SLIGHT_EDGE` = monitored opportunity when present in raw source or canonical decision metadata
- `PASS` = non-actionable, diagnostic-only, or collapsed presentation

This work item is about parity and visibility only. It must not change POTD selection rules, fallback logic, staking behavior, or the shared decision-outcome contract.

**Scope**:

- `web/src/app/api/cards/route.ts`
- `web/src/lib/cards/query.ts`
- `web/src/lib/game-card/decision.ts`
- `web/src/components/cards-page-client.tsx`
- `web/src/components/cards/CardsList.tsx`
- `web/src/components/cards/CardsPageContext.tsx`
- `web/src/__tests__/cards-decision-outcome-parity.test.ts`
- `web/src/__tests__/api-cards-parity-snapshot.test.js`
- `web/src/__tests__/ui-cards-smoke.test.js`

**Out of scope**:

- POTD promotion rules or fallback selection logic
- Changes to `potd_plays` write semantics
- Bankroll, staking, or wager-sizing behavior
- Reclassifying `PASS` rows as actionable plays
- Expanding shared decision-outcome contracts to add first-class `LEAN` rendering

**Acceptance**:

- `/cards` includes decision-state metadata for every card-backed opportunity returned by the cards API
- `/cards` clearly distinguishes official `PLAY` opportunities from monitored non-play opportunities and PASS/diagnostic states
- If raw source data contains `LEAN`, that raw status is preserved as metadata when available, but the UI may bucket `LEAN` and `SLIGHT_EDGE` under `Monitored` unless the shared contract is expanded in a separate WI
- `PASS` rows do not present as actionable picks
- Existing cards filtering/sorting continues to work with the expanded decision-state visibility
- POTD metadata may attach when applicable, but `/cards` does not redefine official POTD semantics
