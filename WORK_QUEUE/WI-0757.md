# WI-0757: Actual result ingestion for projection cards (nhl-pace-1p and mlb-f5)

**ID**: WI-0757
**Goal**: After games complete, write the actual 1P goals (NHL) and actual F5 combined runs (MLB) back into `card_payloads` so projection settlement can be computed.

**Background**:
Projection-only cards (`nhl-pace-1p`, `mlb-f5`) cannot settle against odds because no market line was scraped. Accuracy is measured against the actual game output:

- `nhl-pace-1p`: actual combined 1st period goals → compared against fixed line 1.5 (binary HIT/MISS) and model projection (for MAE tracking)
- `mlb-f5`: actual combined F5 runs → compared against model projection (MAE only)

The worker already has access to final score data for full-game settlement. This WI extends that to sub-game segments. The actual result should be stored in a new `actual_result` JSON field (or column) on the `card_payloads` row, set to `null` until the game completes.

**Scope**:
- `packages/data/src/db/migrations/` — add migration to add `actual_result` column (TEXT/NULL) to `card_payloads` if not already present
- `packages/data/src/db/card-payloads.js` — add `setProjectionActualResult(cardId, actualResult)` write function; add `getUnsettledProjectionCards()` query
- `packages/data/index.js` — export new functions
- `apps/worker/src/jobs/` — extend score ingestion job (or add `settle_projections.js`) to: (a) detect completed games, (b) pull 1P score segments (NHL) and F5 line scores (MLB) from the existing score source, (c) call `setProjectionActualResult` for matching cards
- `WORK_QUEUE/WI-0757.md` — this file

**Out of scope**:

**Acceptance**:
- `card_payloads` has an `actual_result` column (nullable TEXT, JSON-serialized)
- After an NHL game with a `nhl-pace-1p` card completes, `actual_result` is set to `{"goals_1p": N}` where N is actual combined 1P goals
- After an MLB game with an `mlb-f5` card completes, `actual_result` is set to `{"runs_f5": N}` where N is actual combined F5 runs (innings 1–5)
- `getUnsettledProjectionCards()` returns cards where `card_type IN ('nhl-pace-1p', 'mlb-f5')` and `actual_result IS NULL` and game is finalized
- Single-writer contract is maintained: only the worker writes `actual_result` — web server is read-only
