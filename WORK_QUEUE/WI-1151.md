# WI-1151: Fix POTD MLB Model Signal — Read From card_payloads Not Raw Odds Snapshot

**ID**: WI-1151
**Goal**: Enable MLB moneyline plays to compete for POTD by adding `getLatestMlbModelOutput` and wiring it into the POTD engine's game hydration step, mirroring the NHL/NBA pattern.

**Scope**:
- `packages/data/src/db/cards.js`
- `packages/data/index.js`
- `apps/worker/src/jobs/potd/run_potd_engine.js`
- `apps/worker/src/jobs/potd/signal-engine.js`

**Out of scope**:
- Changes to `run_mlb_model.js` or `mlb-model.js`
- Changes to `enrichMlbPitcherData`
- Changes to `computeMLBDriverCards` internals
- Changes to `resolveMLBModelSignal` internals (the existing function is correct once given a properly hydrated game)

**Root cause**:
`resolveMLBModelSignal` in `signal-engine.js` calls `computeMLBDriverCards(game.gameId, game.oddsSnapshot)` where `game.oddsSnapshot = getLatestOdds(gameId)`. The raw `odds_snapshot` has no pitcher data in `raw_data.mlb` — those fields are populated only at model runtime by `enrichMlbPitcherData` inside `run_mlb_model.js` and stored in `card_payloads` (card_type `mlb-full-game`). So `computeMLBDriverCards` always finds null pitchers and returns no FIRE cards, causing `resolveMLBModelSignal` to return null for every game.

The fix mirrors the NBA pattern: add a `getLatestMlbModelOutput` function that reads the already-computed model signal from `card_payloads`, then inject it into the game object in `gatherBestCandidate` before candidates are scored.

**Implementation**:

**Task 1 — `packages/data/src/db/cards.js` — add `getLatestMlbModelOutput`**

Add after `getLatestNbaModelOutput` (line ~1285):

```js
function getLatestMlbModelOutput(gameId) {
  const db = getDatabase();
  const row = db.prepare(`
    SELECT payload_data FROM card_payloads
    WHERE game_id = ? AND card_type = 'mlb-full-game'
    ORDER BY created_at DESC LIMIT 1
  `).get(gameId);
  if (!row) return null;
  const rd = JSON.parse(row.payload_data);
  // Extract the moneyline model signal from the stored driver card.
  // mlb-full-game cards store the best FIRE card's driver at drivers[0].
  const driver = Array.isArray(rd.drivers) ? rd.drivers[0] : null;
  if (!driver) return null;
  const { win_prob_home, edge, side } = driver;
  if (!Number.isFinite(win_prob_home) || !Number.isFinite(edge)) return null;
  if (side !== 'HOME' && side !== 'AWAY') return null;
  return {
    modelWinProbHome: win_prob_home,
    edge,
    side,
    projection_source: rd.projection_source ?? 'MLB_FULL_GAME_MODEL',
  };
}
```

Also add `getLatestMlbModelOutput` to the `module.exports` at the bottom of the file.

**Task 4 — `apps/worker/src/jobs/potd/signal-engine.js` — consume `mlbSnapshot`**

Currently `resolveMLBModelSignal` re-runs `computeMLBDriverCards` from raw odds data. With the new approach, the POTD engine should use the pre-computed snapshot instead — mirroring how `resolveNHLModelSignal` and `resolveNBAModelSignal` work.

Add a `resolveMLBSnapshotSignal` function in `signal-engine.js`:

```js
function resolveMLBSnapshotSignal(game) {
  const snap = game?.mlbSnapshot;
  if (!snap) return null;
  if (!Number.isFinite(snap.modelWinProbHome) || !Number.isFinite(snap.edge)) return null;
  return {
    modelWinProbHome: snap.modelWinProbHome,
    edge: snap.edge,
    side: snap.side,
    projection_source: snap.projection_source ?? 'MLB_FULL_GAME_MODEL',
  };
}
```

In `buildCandidates`, add a MLB model signal block parallel to the NHL block. After the existing NHL block and before the final `return candidates`:

```js
// MLB model signal block: inject mlbSnapshot onto MONEYLINE candidates only
if (isMlbSport(game.sport) && game.mlbSnapshot) {
  const mlbSnapshotSignal = resolveMLBSnapshotSignal(game);
  if (mlbSnapshotSignal) {
    return candidates.map((candidate) =>
      candidate.marketType === 'MONEYLINE'
        ? { ...candidate, mlbSnapshotSignal }
        : candidate
    );
  }
}
```

In `scoreCandidate`, update the MLB model override block to first try the snapshot signal (which is already computed and clean), falling back to the existing `resolveMLBModelSignal` path for backward compat:

const mlbSnapSignal = candidate.mlbSnapshotSignal ?? null;
const useMlbSnapshotSignal =

if (useMlbSnapshotSignal) {
