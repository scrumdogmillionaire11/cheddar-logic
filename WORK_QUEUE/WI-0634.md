# WI-0634: Wire report_settlement_health into scheduler as diagnostic job

**ID**: WI-0634
**Goal**: Add `report_settlement_health.js` to `schedulers/main.js` on a low-frequency cadence (e.g., daily) so persistent settlement failures and stuck pending rows are surfaced automatically in job run history.

**Background**:
`apps/worker/src/jobs/report_settlement_health.js` is a read-only diagnostic that reports unsettled rows, blocked pending cards, and recent settlement job failures. It is not imported in `schedulers/main.js` and requires manual invocation for triage. Running it automatically once daily (e.g., 08:00 ET) means anomalies show up in `job_runs` without requiring SSH access.

Lower priority than WI-0627 (`check_odds_health`) since settlement health is less time-sensitive, but the pattern is identical.

**Scope**:
- `apps/worker/src/schedulers/main.js` — import `runSettlementHealthReport` (or equivalent export name) and register it on a daily cadence keyed to `report_settlement_health|${cacheDate}`

**Out of scope**:
- Changes to `report_settlement_health.js` itself
- Discord/alert integration

**Acceptance**:
- `report_settlement_health` is imported and scheduled in `schedulers/main.js`.
- Job key is date-based so it runs at most once per day.
- `node apps/worker/src/schedulers/main.js --dry-run` starts without error.
- `npm --prefix apps/worker test` passes.
