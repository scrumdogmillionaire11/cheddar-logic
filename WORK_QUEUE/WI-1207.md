# WI-1207: Wire POTD to DecisionOutcome

**ID**: WI-1207
**Goal**: Refactor POTD engine to consume `DecisionOutcome` only, selecting strictly on `status='PLAY'`, eliminating fallback ghosts and drift from web/Discord.

**Scope**:
- `apps/worker/src/jobs/publish_potd_selection.js` (import canonical builder from @cheddar-logic/data, apply to decision_v2 list, filter outcomes for status='PLAY' only; remove all locally-defined decision_v2 → status mappers that are no longer called)
- `apps/worker/src/jobs/__tests__/publish_potd_selection.test.js` (add anti-ghost tests: no plays selected if no PLAY status outcomes; remove test cases for old mapper functions if they exist)
- `apps/worker/src/utils/potd-selector.js` (refactor to accept DecisionOutcome list; filter status='PLAY' only, no mapping logic; delete easy hanging dead code)
- `apps/worker/src/utils/__tests__/potd-selector.test.js` (new/refactored: test DecisionOutcome input only; remove tests for deleted mapper functions)
- Audit `apps/worker/src/utils/` for POTD-specific decision helpers that are no longer called after builder switch; delete easy hanging dead code (functions with no external callers, not exported publicly)

**Out of scope**:
- Implementing POTD-specific decision mappers (use canonical builder from WI-1199 only)
- Changing worker decision_v2 production
- Comprehensive refactoring or reorganization of POTD utilities (covered in WI-1203)
- POTD ranking/scoring logic changes
- **In scope (not deferred to WI-1203):** Removing "easy hanging" dead code — functions that become unused after builder switch, have no external callers, and are not exported. This reduces maintenance burden immediately.

**Acceptance**:
- `publish_potd_selection.js` calls `buildDecisionOutcomeFromDecisionV2()` from canonical builder (imported from @cheddar-logic/data); does NOT implement its own mapper
- `potd-selector` accepts list of `DecisionOutcome` objects only (no decision_v2 input)
- Selector filters for `status='PLAY'` only; ignores SLIGHT_EDGE and PASS
- If no PLAY outcomes exist, POTD is empty (no fallback to SLIGHT_EDGE or consensus logic)
- If multiple PLAY outcomes exist, selector picks highest `confidence` (or first if tied)
- Anti-ghost test: 100 fixture decision lists with 0 PLAY outcomes produce 0 POTD selections
- POTD selection count ≤ web PLAY count (never exceeds it); invariant enforced
- All existing POTD tests pass
- No POTD-specific mapper code exists (audit confirms removal + dead code deleted)
- Easy hanging dead functions removed (git diff shows deleted lines from old mapper logic)
- No orphaned test cases for deleted functions
