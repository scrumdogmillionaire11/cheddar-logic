# WI-1207: Enforce Strict PLAY-Only POTD (Remove SLIGHT_EDGE Fallback Activation)

**ID**: WI-1207
**Goal**: Harden the current POTD engine path so it remains strictly `DecisionOutcome.status='PLAY'` end-to-end by removing runtime activation of the SLIGHT_EDGE fallback tier in `apps/worker/src/jobs/potd/run_potd_engine.js`.

**Why this is still needed**:

- Current code already builds `DecisionOutcome` via `buildDecisionOutcomeFromDecisionV2()` and rejects non-PLAY during assembly.
- But the engine still accumulates a SLIGHT_EDGE scored pool and can activate it later when no PLAY candidates pass gates.
- This leaves a live path where POTD can fire from non-PLAY status, which contradicts strict PLAY-only behavior.

**Scope**:

- `apps/worker/src/jobs/potd/run_potd_engine.js`
  - Remove the SLIGHT_EDGE accumulation/activation path after non-PLAY rejection.
  - Delete or neutralize `usedFallbackTier` behavior so selection cannot fire on `SLIGHT_EDGE`.
  - Keep existing PLAY candidate scoring/ranking logic unchanged.
  - Keep existing non-PLAY rejection diagnostics (or equivalent) for observability.
- `apps/worker/src/jobs/potd/__tests__/run-potd-engine.test.js`
  - Replace fallback-positive expectations with strict PLAY-only expectations.
  - Add/keep anti-ghost tests proving no POTD is emitted when all candidates are non-PLAY (including strong-edge SLIGHT_EDGE cases).
  - Assert no `usedFallbackTier='SLIGHT_EDGE'` outcome can occur.

**Out of scope**:

- Changes to decision production (`decision_v2`) upstream.
- Changes to general POTD ranking math, scoring formula, Kelly sizing, or shadow-group strategy.
- Consensus-vs-model edge-source contract work in `signal-engine.js`.
- Broader legacy cleanup covered by WI-1203.

**Acceptance**:

- `run_potd_engine.js` still uses canonical `buildDecisionOutcomeFromDecisionV2()` for model-contract payload interpretation.
- Non-PLAY outcomes (`SLIGHT_EDGE`, `PASS`, missing/invalid) never enter a fallback selector pool that can fire a POTD.
- The fallback block that can set `usedFallbackTier = 'SLIGHT_EDGE'` is removed from executable code (no latent/unreachable fallback branch retained).
- If no PLAY candidates pass gates, run result is no-pick (`noPlay=true`) and no row is written to `potd_plays`.
- Existing PLAY-only behavior remains intact: PLAY candidates that satisfy gates can still fire.
- All targeted POTD tests pass with updated expectations.
