## Background

The existing `projection_evaluator.js` computes MAE, bias, and directional accuracy over raw projection vs actual deltas. It does not grade projections against canonical proxy lines — i.e. it cannot answer "did the model correctly favor Over 3.5 when the actual was 5?".

For MLB F5 the canonical proxy lines are **3.5 and 4.5** (the two most common market anchors). For NHL 1P the canonical line is always **1.5** (a fixed number). This WI creates the schema and data layer for storing per-game-per-line graded rows so subsequent WIs can write and the web API can read.

This WI is purely additive. No existing table is modified.

## Migration: `069_projection_proxy_evals.sql`

```sql
CREATE TABLE IF NOT EXISTS projection_proxy_evals (
  id                INTEGER PRIMARY KEY AUTOINCREMENT,

  -- Source card
  card_id           TEXT NOT NULL,
  game_id           TEXT NOT NULL,
  game_date         TEXT NOT NULL,            -- YYYY-MM-DD
  sport             TEXT NOT NULL,            -- 'baseball_mlb' | 'icehockey_nhl'
  card_family       TEXT NOT NULL,            -- 'MLB_F5_TOTAL' | 'NHL_1P_TOTAL'

  -- Projection and actual
  proj_value        REAL NOT NULL,            -- model's projected total
  actual_value      INTEGER NOT NULL,         -- settled actual (runs or goals)

  -- Proxy market fields
  proxy_line        REAL NOT NULL,            -- 3.5 | 4.5 | 1.5
  edge_vs_line      REAL NOT NULL,            -- proj_value - proxy_line (signed)
  recommended_side  TEXT NOT NULL,            -- 'OVER' | 'UNDER' | 'PASS'
  tier              TEXT NOT NULL,            -- 'PASS' | 'LEAN' | 'PLAY' | 'STRONG'
  confidence_bucket TEXT NOT NULL,            -- 'MICRO' | 'SMALL' | 'MEDIUM' | 'LARGE'

  -- Agreement fields (populated after all lines for a game are computed)
  agreement_group   TEXT NOT NULL DEFAULT '', -- 'CONSENSUS_OVER' | 'CONSENSUS_UNDER' | 'SPLIT' | 'PASS_ONLY'

  -- Graded result
  graded_result     TEXT NOT NULL,            -- 'WIN' | 'LOSS' | 'NO_BET'
  hit_flag          INTEGER NOT NULL,         -- 1 for WIN, 0 for LOSS/NO_BET

  -- Scoring
  tier_score        REAL NOT NULL DEFAULT 0,  -- signed float per tier weight rules
  consensus_bonus   REAL NOT NULL DEFAULT 0,  -- +/- 1.0 when both lines agree + same result

  created_at        TEXT NOT NULL DEFAULT (datetime('now')),

  UNIQUE(card_id, proxy_line)
);

CREATE INDEX IF NOT EXISTS idx_proxy_evals_game_date
  ON projection_proxy_evals (game_date, card_family);

CREATE INDEX IF NOT EXISTS idx_proxy_evals_card_id
  ON projection_proxy_evals (card_id);
```

## Data Access Module: `packages/data/src/db/projection-accuracy.js`

Export these functions. All accept `db` as first argument (the sql.js database instance, consistent with other modules in this package).

### `insertProjectionProxyEval(db, row)`

Inserts one row. `row` shape matches the table exactly (all non-default fields required). Uses `INSERT OR REPLACE` on `(card_id, proxy_line)`.

### `batchInsertProjectionProxyEvals(db, rows)`

Inserts an array of rows in a single transaction. Calls `insertProjectionProxyEval` for each row inside `db.exec('BEGIN')` / `db.exec('COMMIT')`. Returns count of rows written.

### `getProjectionProxyEvals(db, { cardFamily, gameDateGte, gameDateLte, agreementGroup, tier, limit })`

Reads rows with optional filters. All filter params are optional. Default limit 500. Returns array of plain objects matching table columns.

### `getProjectionAccuracySummary(db, { cardFamily, gameDateGte, gameDateLte })`

Returns an aggregated summary object:

```js
{
  card_family: string,
  game_date_range: { gte: string, lte: string },
  total_games: number,
  total_proxy_decisions: number,   // rows where graded_result != 'NO_BET'
  wins: number,
  losses: number,
  no_bets: number,
  proxy_hit_rate: number | null,   // wins / (wins + losses); null if no decisions
  consensus_games: number,         // rows where agreement_group in ('CONSENSUS_OVER','CONSENSUS_UNDER')
  consensus_wins: number,
  consensus_hit_rate: number | null,
  split_zone_games: number,        // rows where agreement_group = 'SPLIT'
  avg_tier_score: number | null,
  total_score: number,
  by_tier: {                       // keyed by tier string
    LEAN:   { decisions: number, wins: number, losses: number, hit_rate: number | null },
    PLAY:   { decisions: number, wins: number, losses: number, hit_rate: number | null },
    STRONG: { decisions: number, wins: number, losses: number, hit_rate: number | null },
  },
  by_proxy_line: {                 // keyed by proxy_line as string e.g. '3.5'
    [line]: { decisions: number, wins: number, losses: number, hit_rate: number | null },
  },
}
```

SQL: use `COUNT`, `SUM(hit_flag)`, `AVG(tier_score)`, grouped by `tier` and `proxy_line`.
