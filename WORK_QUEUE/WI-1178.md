# WI-1178: Normalize POTD Edge Calculation and Scoring Across Sports

**ID**: WI-1178
**Goal**: Eliminate structural bias toward NBA totals in POTD selection by replacing the uncalibrated `/20` linear edge heuristic with a sigma-based probability model, equalizing noise floors, and adding normalized edge weight to the scoring formula.

**Context**:
- POTD currently selects the winner by `totalScore = (lineValue * 0.625) + (marketConsensus * 0.375)` — edge is not in this formula and only acts as a tiebreaker.
- NBA totals use `modelOverProb = 0.5 + (projection - line) / 20` — a linear heuristic with no calibration lineage. `edge-calculator.js` itself comments: "NBA total=14 (set ~2024, uncalibrated — no lineage in codebase)".
- MLB and NHL use probability-based model outputs. NBA edges and MLB/NHL edges are therefore on incompatible scales.
- Totals candidates score on both `lineComponent` and `priceComponent`; moneyline candidates score on `priceComponent` only. This gives totals a structural ceiling advantage over all MLB/NHL moneyline plays.
- NBA TOTAL noise floor is 2%; MLB MONEYLINE is 3%. NBA has an easier qualification bar.
- The combined effect: POTD is selecting "best-looking market opportunity" rather than "best +EV bet."

**Acceptance**:
- NBA total edge is computed via `computeTotalEdge({ projection, line, sigmaTotal, impliedProb })` from `edge-calculator.js`, not the `/20` shortcut. `sigmaTotal` defaults to `getSigmaDefaults('NBA').total` (14).
- `modelWinProb` and `edgePct` returned for NBA TOTAL candidates are derived from the sigma-based path and remain finite numbers.
- The `totalScore` formula includes a normalized edge component: `(lineValue * 0.45) + (marketConsensus * 0.30) + (edgeComponent * 0.25)` where `edgeComponent = clamp(edgePct / EDGE_NORMALIZATION_CAP, 0, 1)` and `EDGE_NORMALIZATION_CAP = 0.12`. Zero or negative edge contributes 0 to the score; +12pp edge contributes the full 0.25 weight.
- `POTD_NOISE_FLOOR_NBA_TOTAL` default is raised from `0.02` to `0.03` in signal-engine.js.
- `confidenceLabel` still uses the updated `totalScore` so that the HIGH/ELITE threshold check in run_potd_engine is unaffected.
- All existing signal-engine tests pass. New tests cover: NBA sigma path produces smaller edge than `/20` for a 2-point gap; edgeComponent clamps correctly at 0 and 1; noise floor default reads as 0.03.
- No change to `EDGE_SOURCE_CONTRACT`, `edgeSourceTag`, or `isModelBackedCandidate` logic.
