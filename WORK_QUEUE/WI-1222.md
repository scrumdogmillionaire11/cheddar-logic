# WI-1222: Document NHL model_outputs asymmetry with ADR + contract test (R-08)

**ID**: WI-1222
**Goal**: Make the NHL `card_payloads`-only persistence behavior explicit and safe by adding an ADR plus a regression test that codifies the expected divergence from MLB/NFL.

**Context**:

The NHL model intentionally writes betting outputs to `card_payloads` and does not insert into `model_outputs`. MLB/NFL jobs do write to `model_outputs`. This is currently discoverable only via comments and implementation details, so API consumers can incorrectly assume `/api/model-outputs` should include NHL.

Without an explicit architecture decision + test contract, future refactors may accidentally write NHL rows into `model_outputs` or change web expectations without migration.

**Scope**:

- `docs/decisions/ADR-0018-nhl-card-payloads-only-no-model-outputs.md` (new)
- `apps/worker/src/__tests__/run-nhl-model.test.js` — add test proving NHL run writes `card_payloads` and does not write `model_outputs`
- `web/src/app/api/model-outputs/route.ts` — add explicit comment (or response metadata) clarifying NHL omission is expected by design
- `.planning/STATE.md`
- `WORK_QUEUE/WI-1222.md`

**Out of scope**:

- Changing NHL to write `model_outputs`
- Any schema migration for `model_outputs`
- Changes to MLB/NFL model jobs

**Acceptance**:

- ADR exists and states: rationale, trade-offs, consequences, and rollback plan if unification is later required
- NHL model test asserts:
  - `card_payloads` insert occurs for NHL run
  - `insertModelOutput()` (or equivalent model_outputs write path) is not called for NHL run
- `/api/model-outputs` endpoint source includes explicit note that NHL absence is expected under ADR-0018
- No change to runtime behavior beyond comments/tests/docs
