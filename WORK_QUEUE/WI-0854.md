## Context

### Issue 1 — settlement-mirror env leak

`apps/worker/src/jobs/potd/run_potd_engine.js` calls `require('dotenv').config()` at module-load time. When Jest requires it during the settlement-mirror test, it loads the root `.env`, which contains `POTD_STARTING_BANKROLL=100`. This overrides the `DEFAULT_BANKROLL` constant's fallback of `10`, so `ensureInitialBankroll` seeds the potd_bankroll table with 100, not 10.

The test's `beforeAll` block sets `process.env.CHEDDAR_DB_PATH` and similar keys, but never overrides `POTD_STARTING_BANKROLL`. The assertion `amount_before: 10` therefore fails with `amount_before: 100`.

This was introduced before this repository's sprint cycle began. The failing test (1 of 1359) has been masking itself as a pre-existing skip in every full suite run.

**Root cause trace:**
```
.env: POTD_STARTING_BANKROLL=100
  → require('dotenv').config() in run_potd_engine.js
    → DEFAULT_BANKROLL = Number(process.env.POTD_STARTING_BANKROLL || 10) = 100
      → ensureInitialBankroll seeds amount_after=100
        → settlement-mirror ledger row: amount_before=100
          → test expects amount_before=10 → FAIL
```

**Fix:** In `settlement-mirror.test.js` `beforeAll`, pin `process.env.POTD_STARTING_BANKROLL = '10'` before requiring the potd engine module so the test is isolated from `.env`. Restore in `afterAll`.

---

### Issue 2 — dead code in report_telemetry_calibration.js

WI-0809 refactored `buildDecisionTierAudit` from returning a `{ status, sampleWindow, tiers[] }` shaped object to returning a sparse `[{ sport, market_type, tier, n, win_rate, total_pnl }]` array. In doing so, `buildEmptyDecisionTierBucket` (line 336) and `finalizeDecisionTierBucket` (line 350) in `report_telemetry_calibration.js` lost all callers and became unreachable dead code.

They are not exported in `module.exports.__private` so nothing external relies on them.

**Fix:** Delete both functions from `report_telemetry_calibration.js`.
