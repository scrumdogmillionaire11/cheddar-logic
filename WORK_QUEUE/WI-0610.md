# WI-0610: moneypuck.js test suite — HTTP fetch, parse, normalization coverage

**ID**: WI-0610
**Goal**: Add a test file for `apps/worker/src/moneypuck.js` covering HTTP fetch, response parsing, and data normalization so upstream API shape changes are caught before they produce silent bad data.

**Background**:
`moneypuck.js` (1,339 lines) is the MoneyPuck integration module responsible for HTTP fetch, response parsing, and normalization of shot metrics used by the NHL model. It has no test file. Upstream API shape changes will silently produce bad data (null fields, wrong units) that only surfaces when model outputs visibly degrade — often after cards are already published.

**Scope**:
- `apps/worker/src/__tests__/moneypuck.test.js` (new) — mock HTTP responses with known fixture JSON; assert parsed fields match expected types and values; assert null-handling when fields are missing; assert normalization calculations (per-60 conversions, rate computations)
- No changes to `moneypuck.js` itself unless bugs are found during test authoring

**Out of scope**:
- Integration tests hitting the live MoneyPuck API
- Changes to the NHL model or downstream consumers

**Acceptance**:
- Test file exists and `npm --prefix apps/worker test -- moneypuck` passes.
- Tests cover: successful parse of a well-formed response, graceful handling of a missing field, correct per-60 computation on a known input.
- No changes to production behavior.
