# WI-0976: Harden protected deployment policy gates

**ID**: WI-0976
**Goal**: Require branch protections and required checks before any production deployment can proceed.

**Depends on**: WI-0975

**Scope**:
- `.github/workflows/ci.yml`
- `.github/workflows/deploy-production.yml`
- `docs/processes/deployment-policy.md`

**Out of scope**:
- Deploy artifact internals
- Application business logic changes

**Acceptance**:
- Production workflow enforces required upstream checks before execution.
- Deployment path from unprotected branches or failing checks is blocked.
- Policy document clearly lists required checks and approval requirements.

**Owner agent**: unassigned
**Time window**: 2026-04-16 -> TBD
**Coordination flag**: needs-sync

**Tests to run**:
- `npm --prefix web run lint`
- `npm --prefix web run test:ci:smoke`

**Manual validation**:
- Attempt to deploy with a failing check and confirm it is blocked.
- Attempt to deploy from a non-approved branch and confirm it is rejected.
