# WI-0566: Player props settlement framework — generalize beyond NHL shots on goal

**ID**: WI-0566
**Goal**: Player prop settlement is currently NHL shots-on-goal only. Build a generalizable player prop settlement layer that can support soccer player props (shots, goals/assists) and lay the groundwork for NBA props. This is the foundational plumbing WI; sport-specific expansions build on top of it.

**Scope**:
- `apps/worker/src/jobs/settle_pending_cards.js`
  - Refactor `isNhlShotsOnGoalCard()`, `resolveNhlShotsSettlementContext()`, `resolvePlayerShotsActualValue()`, `gradeNhlPlayerShotsMarket()` into a general player prop settlement interface
  - Add soccer player prop routing: `soccer_player_shots`, `to_score_or_assist`
  - Wire soccer prop resolution to actual values stored in `game_results.metadata.playerStats` (define schema below)
- `apps/worker/src/jobs/settle_game_results.js`
  - Persist soccer player stat actuals (shots, goals, assists) into `game_results.metadata.playerStats` after games finalize
- Schema contract: define `game_results.metadata.playerStats` shape (keyed by playerId, with `shots`, `goals`, `assists`, `period` sub-keys)
- `apps/worker/src/__tests__/settle_pending_cards.market-contract.test.js` — add soccer prop contract cases
- New test: `apps/worker/src/__tests__/settlement-player-props-integration.test.js`

**Out of scope**:
- NBA player props settlement (separate WI; this lays the interface, NBA wires in next)
- Player identification / name disambiguation overhaul (tracked separately if needed)
- Live/in-game prop settlement

**Background**:
NHL shots settlement was purpose-built and is tightly coupled to NHL-specific data paths. Soccer model already generates `soccer_player_shots` and `to_score_or_assist` cards (`pull_soccer_player_props.js`), but `settle_pending_cards.js` has no routing for them — they stay PENDING forever. The right approach is to:
1. Generalize the NHL prop settlement pattern into a reusable `resolvePlayerPropSettlementContext()` / `gradePlayerPropMarket()` interface.
2. Add a `soccer` implementation that reads from `game_results.metadata.playerStats`.
3. Keep NHL shots wired through the same interface (no functional change, refactor only).

Player identification strategy: prefer playerId match; fall back to `normalizePlayerName()` (already exists at line 744).

**Acceptance**:
- NHL shots on goal settlement behavior is unchanged after refactor (existing tests continue passing).
- Soccer `soccer_player_shots` cards settle WIN/LOSS/PUSH when player shot actuals are available.
- Soccer `to_score_or_assist` cards settle WIN/LOSS/PUSH when goal/assist actuals are available.
- Cards produce `MISSING_PLAYER_STAT` failure code (deterministic, consistent with `MISSING_PERIOD_SCORE` pattern) when actuals are absent.
- `game_results.metadata.playerStats` schema is documented in a comment or adjacent `// @typedef` block.
- Market contract tests cover: soccer shots over (win), under (win), push; goal/assist hit, miss.
- No regression to any existing NHL or NBA settlement path.
