# WI-0353: Legacy Removal (Worker/Web/Settlement)

**ID**: WI-0353
**Goal**: Remove legacy parsing and inference paths across worker, web, and settlement so UI and decisions rely solely on `decision_v2` and structured projection fields, per NOTES legacy removal direction.

**Scope**:

- `WORK_QUEUE/WI-0353.md`
- `apps/worker/src/utils/decision-publisher.js`
- `web/src/lib/game-card/transform.ts`
- `web/src/lib/game-card/tags.ts`
- `web/src/lib/types/game-card.ts`
- `web/src/lib/decision.ts`
- `web/src/lib/canonical-decision.ts`
- `web/src/lib/filters.ts`
- `web/src/app/api/games/route.ts`
- `apps/worker/src/jobs/run_nba_model.js`
- `apps/worker/src/jobs/run_nhl_model.js`
- `apps/worker/src/jobs/run_ncaam_model.js`
- `apps/worker/src/models/projections.js`
- `apps/worker/src/jobs/settle_pending_cards.js`
- `apps/worker/src/jobs/resettle_historical_cards.js`
- `docs/DATA_CONTRACTS.md`


**Out of scope**:

- New projection systems
- Schema migrations
- Historical backfills


**Acceptance**:

- Legacy status/classification and market inference fallbacks are removed; `decision_v2` is the sole source for wave-1 verdicts.
- UI no longer parses note/title text for market, projection, or risk tags.
- Settlement no longer attempts legacy fallback resettlement; only locked market keys are supported.
- Old cards may degrade in display; no legacy rendering is preserved.


**Tests to run**:

- `npm --prefix web run test:ui:cards`
- `npm --prefix apps/worker test -- src/utils/__tests__/decision-publisher.v2.test.js`


**Manual validation**:

- Open `/cards` and confirm projection context renders only when structured projection fields are present.
- Confirm legacy cards may show reduced context without fallback parsing.
