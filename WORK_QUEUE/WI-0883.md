# WI-0883: POTD confidence-weighted sizing — multiply Kelly stake by confidence multiplier

**ID**: WI-0883
**Goal**: Scale the POTD Kelly wager by the candidate's confidence tier so that an ELITE pick bets proportionally more than a HIGH pick with the same edge, and a HIGH pick bets more than a MEDIUM pick — expressing genuine belief strength in stake size.

**Scope**:
- `apps/worker/src/jobs/potd/signal-engine.js`
- `apps/worker/src/jobs/potd/run_potd_engine.js`
- `apps/worker/src/jobs/potd/__tests__/signal-engine.test.js`
- `apps/worker/src/jobs/potd/__tests__/run-potd-engine.test.js`
- `packages/data/db/migrations/*.sql` — one new migration adding nullable `confidence_multiplier REAL` column to `potd_plays`

**Out of scope**:
- Changes to `kellySize` inputs (edgePct, impliedProb) — multiplier is applied after Kelly, not inside it
- `maxWagerPct` cap changes — cap still enforced inside `kellySize` before multiplier is applied
- Web UI display of confidence multiplier (read `confidence_multiplier` from existing `score_breakdown` or `kelly_fraction` field)
- Any non-POTD sizing paths

**Acceptance**:
1. `confidenceMultiplier(label)` function exported from `signal-engine.js`; returns `{ ELITE: 1.0, HIGH: 0.85, MEDIUM: 0.65, LOW: 0.40 }[label] ?? 0.85`
2. `runPotdEngine` computes `wagerAmount = kellySize(...)` then `adjustedWager = round(wagerAmount * confidenceMultiplier(candidate.confidenceLabel), 2)`; the adjusted value is what persists to `potd_plays.wager_amount` and `potd-call` card payload
3. New migration adds `confidence_multiplier REAL` column to `potd_plays` (nullable, default null for pre-existing rows); `buildPotdPlayRow` and `buildCardPayloadData` write `confidence_multiplier` on insert
4. Test: ELITE candidate (same edge, bankroll, impliedProb as HIGH) produces wager ≈ 1.18× the HIGH wager (1.0 / 0.85)
5. Test: MEDIUM candidate wager ≤ HIGH candidate wager at identical edge inputs
6. Test: `confidenceMultiplier` with unknown label returns `0.85` (safe default)
7. The Kelly cap (`maxWagerPct = 0.2`) still applies at the `kellySize` level before multiplier scaling; multiplier never pushes total above 20% of bankroll
8. All pre-existing `kellySize`, `selectBestPlay`, `getTopNCandidates` (WI-0882), and `run-potd-engine` tests pass
