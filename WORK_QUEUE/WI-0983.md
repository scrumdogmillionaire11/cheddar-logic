# WI-0983: Simplify Cards Filtering Ownership and Predicate Contract

**ID**: WI-0983
**Goal**: Collapse duplicated cards filtering rules into a single canonical predicate contract so cards page filtering, projection visibility, and diagnostics all evaluate the same truth.

**Depends on**: none

**Scope**:

- web/src/lib/game-card/filters.ts
- web/src/components/cards/CardsPageContext.tsx
- web/src/components/cards/CardsList.tsx
- web/src/components/cards/shared.ts
- web/src/components/cards/types.ts
- web/src/__tests__/game-card-filters.test.js
- web/src/__tests__/filters-pass-play-main-view-regression.test.js
- web/src/__tests__/cards-filter-predicate-contract.test.js

**Out of scope**:

- Visual redesign of the cards page filter panel
- Preset UX/layout changes
- Results/API filtering changes outside cards page state
- Worker/model payload generation changes

**Acceptance**:

- Cards page runtime filtering, projection list inclusion, and diagnostic drop-reason reporting are driven by the same predicate ownership model rather than parallel ad hoc checks.
- The projections path no longer re-applies a divergent actionable-play filter in a second component after the main predicate has already run.
- Mode-specific PASS/FIRE/WATCH handling is defined once and reused across list rendering and diagnostics.
- Filter state types no longer rely on unnecessary mode-unsafe casts in the core cards filtering path.
- Existing filter regressions continue to pass with no behavior drift for default game-mode FIRE/WATCH handling.
