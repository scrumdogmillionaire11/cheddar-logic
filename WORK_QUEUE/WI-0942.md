# WI-0942: Decision-Layer Simplification (Holistic Blocking Fix)

**ID**: WI-0942
**Goal**: Reduce contradictory multi-layer decisioning by introducing one canonical block/decision contract used consistently from model output through worker publication to web surfacing.

**Depends on**: WI-0940, WI-0941

**Scope**:

- `packages/models/src/decision-pipeline-v2.js`
- `apps/worker/src/utils/decision-publisher.js`
- `apps/worker/src/jobs/run_nhl_model.js`
- `apps/worker/src/jobs/run_nba_model.js`
- `web/src/lib/games/route-handler.ts`
- `web/src/lib/game-card/transform/index.ts`
- `web/src/lib/game-card/filters.ts`
- `docs/decisions/ADR-00XX-decision-layer-simplification.md`
- `docs/audits/decision-layer-parity-audit.md`

**Out of scope**:

- Sport-specific projection math changes
- Odds ingest provider changes
- Product-level ranking or UI theme changes

**Acceptance**:

- A single canonical decision envelope defines final status, block family, primary reason, and visibility eligibility.
- Duplicate demotion logic across model, publisher, and web transform is removed or delegated to canonical helpers.
- Cross-layer parity tests guarantee no disagreement between worker persisted status and web surfaced status for equivalent payloads.
- Blocking outcomes are deterministic and traceable: one candidate ends with exactly one terminal reason family.
- ADR documents migration rules and compatibility strategy for existing payload consumers.
