### Root Cause Analysis

#### Issue #1: ESPN Enrichment NOT Persisted (WI-0316 scope)
**Finding**: 0 odds snapshots have ESPN metrics in raw_data:
```
SELECT COUNT(*) FROM odds_snapshots WHERE json_extract(raw_data, '$.espn_metrics') IS NOT NULL;
Result: 0
```

**Impact**: NCAAM model runs blind without ESPN team metrics. Driver logic uses fallback fields (raw.avg_points_home, raw.rest_days) instead of enriched metrics, reducing prediction accuracy.

**Code Flow**:
1. `run_ncaam_model.js` line 220: calls `enrichOddsSnapshotWithEspnMetrics(oddsSnapshot)`
2. Enrichment succeeds and adds `espn_metrics` to `raw_data` object
3. **Enriched data is DISCARDED** — never saved back to DB
4. Models see odds without enrichment context

#### Issue #2: Sport Case Normalization OK (WI-0315 scope)
**Finding**: Goods table uses lowercase 'ncaam', cards use uppercase 'NCAAM'. This creates an API join problem IF the web interface naively filters on sport without case conversion.

**Verification**:
- games table: 117 ncaam (lowercase) ✓
- card_payloads: 116 NCAAM (uppercase) + 67 NBA + 55 NHL ✓
- Direct API query bypasses game table: Returns NCAAM cards OK ✓

**Impact on UI**: If UI joins games to cards via `WHERE games.sport = card_payloads.sport`, 106 upcoming games would find 0 cards (case mismatch). This EXPLAINS the "73 missing" symptom if UI loads all upcoming games (106) but shows as "drivers missing" due to join failure.

#### Issue #3: API Gating Works
**Finding**: run_state has current NCAAM run_id:
```
ncaam|job-ncaam-model-2026-03-07T03:01:41-456b5bb2
```

**Impact**: API `/api/cards?sport=ncaam` correctly filters and returns 105+ cards ✓
