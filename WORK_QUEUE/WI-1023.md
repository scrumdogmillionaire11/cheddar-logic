# WI-1023: NBA Phase 2C — Empirically-validated volatility-aware sigma

**ID**: WI-1023
**Goal**: Wire `vol_env` (LOW / MED / HIGH) to sigma multipliers — but only after empirically validating from the accuracy table that HIGH vol games actually miss wider and LOW vol games actually miss tighter. Multipliers are derived from data, not assumed.

**Depends on**: WI-1018, WI-1027 (schema hardening must be active and vol_env must be recorded in accuracy rows before this WI can validate multipliers)

**Scope**:
- `apps/worker/src/jobs/run_nba_model.js`
- `apps/worker/src/jobs/__tests__/run_nba_model.test.js`

**Out of scope**:
- How `vol_env` is determined upstream (not changed here)
- NHL/MLB sigma
- UI changes

**Acceptance**:
- New function `computeVolEnvSigmaMultipliers({ db })` queries `projection_accuracy_line_evals` for NBA total games grouped by `vol_env` and computes empirical RMSE per bucket:

```sql
SELECT vol_env, SQRT(AVG((raw_total - actual_total) * (raw_total - actual_total))) AS rmse, COUNT(*) AS n
FROM projection_accuracy_line_evals
WHERE sport = 'nba'
  AND market_family = 'NBA_TOTAL'
  AND vol_env IS NOT NULL
  AND raw_total IS NOT NULL
  AND actual_total IS NOT NULL
  AND settled_at < datetime('now')
GROUP BY vol_env
HAVING COUNT(*) >= 15
```

- If any `vol_env` bucket has < 15 samples, that bucket falls back to multiplier = 1.0.
- Computed multipliers are derived as: `multiplier = empirical_rmse(bucket) / empirical_rmse('MED')` — normalized to MED, which is the reference.
- If `MED` bucket has < 15 samples, all multipliers fall back to hardcoded defaults (HIGH: 1.25, LOW: 0.85).
- Multipliers are logged at job start: `[NBAModel] [VOL_ENV_SIGMA] HIGH=1.3x(n=23) MED=1.0x(n=45) LOW=0.82x(n=18)` or `[NBAModel] [VOL_ENV_SIGMA] using hardcoded defaults — insufficient MED samples (N)`.
- Multipliers are applied after playoff adjustment and before vol_env stamp: `effectiveSigma = applyVolEnvSigmaMultiplier(priorSigma, volEnv, multipliers)`.
- Multipliers are clamped to [0.75, 1.50] regardless of what the data shows (prevents runaway from sparse samples).
- `raw_data.vol_env_sigma_multiplier` stamped on card payload.
- Unit tests: `computeVolEnvSigmaMultipliers` returns correct ratios from mock db; returns hardcoded defaults when MED bucket is sparse; clamping is enforced.

**Bootstrap period behavior**:

For the first season (before sufficient vol_env data accumulates), this WI ships with hardcoded defaults as the fallback:

- `HIGH`: 1.25
- `MED`: 1.0 (identity)
- `LOW`: 0.85

Once the accuracy table has 15+ games per bucket with `vol_env` populated (which requires WI-1027 to have been running long enough), the empirical path takes over automatically. No code change needed — the query handles it.

**Phase 2 exit gate**:

Phase 2 (WI-1021, WI-1022, WI-1023) is not complete until the following are true over 30+ settled games post Phase-2 deployment:

- Phase 1 exit criteria are still met (regression check)
- RMSE is differentiated by vol_env bucket (HIGH RMSE > MED RMSE > LOW RMSE) — confirms vol_env is meaningful
- Lineup-adjusted projections show lower RMSE on games with SEVERE injury_cloud vs WI-1020 baseline
- Dynamic baselines (WI-1021) are computing from live data (not falling back to hardcoded values)
- All three WIs (1021, 1022, 1023) showing `[computed]` or `[empirical]` source in logs, not `[fallback]`
