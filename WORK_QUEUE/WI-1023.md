# WI-1023: NBA Phase 2C — Empirically-validated volatility-aware sigma

**ID**: WI-1023
**Goal**: Wire `vol_env` (LOW / MED / HIGH) to sigma multipliers derived from empirical RMSE by bucket, normalized to MED. Use hardcoded defaults only when data is insufficient, and apply safe fallbacks per bucket. Multipliers are applied as a projection variance modifier, not a decision signal.

**Depends on**: WI-1018, WI-1027 (accuracy table must include `vol_env` and be populated with settled NBA total evaluations)

**Scope**:

* `apps/worker/src/jobs/run_nba_model.js`
* `apps/worker/src/jobs/__tests__/run_nba_model.test.js`

**Out of scope**:

* How `vol_env` is computed upstream
* NHL/MLB sigma
* UI changes

## Acceptance

### 1) Empirical multiplier computation

Add a function:

```js
computeVolEnvSigmaMultipliers({ db })
```

Query `projection_accuracy_line_evals` for NBA totals grouped by `vol_env`:

```sql
SELECT vol_env,
       SQRT(AVG((raw_total - actual_total) * (raw_total - actual_total))) AS rmse,
       COUNT(*) AS n
FROM projection_accuracy_line_evals
WHERE sport = 'nba'
  AND market_family = 'NBA_TOTAL'
  AND vol_env IS NOT NULL
  AND raw_total IS NOT NULL
  AND actual_total IS NOT NULL
  AND settled_at < datetime('now')
GROUP BY vol_env
HAVING COUNT(*) >= 15
```

### 2) Multiplier derivation rules

* Build a map of `{ vol_env → { rmse, n } }` from query results.
* If `MED` bucket is **missing** or has `< 15` samples:

  * Return **hardcoded defaults for all buckets**:

    * `HIGH = 1.25`
    * `MED = 1.0`
    * `LOW = 0.85`
* If `MED` bucket is valid (`n ≥ 15`):

  * `MED = 1.0`
  * `HIGH = clamp(rmseHigh / rmseMed, 0.75, 1.50)` if `HIGH` exists, else fallback to `1.25`
  * `LOW = clamp(rmseLow / rmseMed, 0.75, 1.50)` if `LOW` exists, else fallback to `0.85`

### 3) Clamping

* All multipliers must be clamped to `[0.75, 1.50]` regardless of empirical values.

### 4) Logging

At job start, log one of the following:

**Empirical (all buckets present):**

```txt
[NBAModel] [VOL_ENV_SIGMA] HIGH=1.30x(n=23,empirical) MED=1.00x(n=45,empirical) LOW=0.82x(n=18,empirical)
```

**Mixed empirical + fallback:**

```txt
[NBAModel] [VOL_ENV_SIGMA] HIGH=1.25x(fallback) MED=1.00x(n=45,empirical) LOW=0.85x(fallback)
```

**Full fallback (MED insufficient):**

```txt
[NBAModel] [VOL_ENV_SIGMA] using hardcoded defaults — insufficient MED samples (n=9)
```

### 5) Application point

Apply multiplier after playoff adjustment and before vol_env stamp:

```js
effectiveSigma = applyVolEnvSigmaMultiplier(priorSigma, volEnv, multipliers)
```

* `MED` is identity (1.0)
* `HIGH` increases sigma
* `LOW` decreases sigma
* Unknown or null `vol_env` defaults to `MED` behavior

### 6) Raw data stamping

Stamp on card payload:

```js
raw_data.vol_env_sigma_multiplier
raw_data.vol_env_sigma_source // 'empirical' or 'fallback'
```

### 7) Unit tests

Cover:

* Correct RMSE ratio normalization vs MED
* MED sparse → full fallback to hardcoded defaults
* MED valid but HIGH missing → HIGH uses fallback (1.25)
* MED valid but LOW missing → LOW uses fallback (0.85)
* Clamping:

  * ratio > 1.5 → 1.5
  * ratio < 0.75 → 0.75
* `applyVolEnvSigmaMultiplier`:

  * HIGH increases sigma
  * LOW decreases sigma
  * MED unchanged
* Raw data stamping:

  * multiplier present
  * source present

## Bootstrap period behavior

Until at least 15+ settled games exist per bucket:

* Use hardcoded defaults:

  * `HIGH`: 1.25
  * `MED`: 1.0
  * `LOW`: 0.85

Once `MED` and other buckets cross the threshold, empirical computation automatically activates.

## Phase 2 exit gate

Phase 2 (WI-1021, WI-1022, WI-1023) is complete only when, over 30+ settled games post deployment:

* Phase 1 criteria still hold (no regression)
* RMSE separation is observed:

  * `HIGH RMSE > MED RMSE > LOW RMSE`
* Lineup-aware injury logic (WI-1022) reduces RMSE in severe injury-cloud games vs WI-1020 baseline
* Dynamic baselines (WI-1021) are sourced from live data, not fallback
* Logs show `[empirical]` or `[computed]`, not `[fallback]`
