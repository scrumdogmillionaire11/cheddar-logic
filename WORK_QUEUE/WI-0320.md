# WI-0320: Dual-DB Runtime Contract (Record vs Results Isolation)

**ID**: WI-0320
**Goal**: Enforce explicit runtime modes so card generation data can be shared while results data remains environment-isolated, eliminating web/worker DB drift.

**Scope**:

- `QUICKSTART.md`
- `scripts/start-scheduler.sh`
- `scripts/manage-scheduler.sh`
- `scripts/db-context.sh`
- `scripts/dev-mode-local.sh`
- `scripts/dev-mode-snapshot.sh`

**Out of scope**:

- Model logic changes
- Odds ingestion/provider changes
- UI redesign
- Cross-repo or infra-only updates

**Acceptance**:

- Quickstart defines two explicit modes: `local` and `snapshot`.
- Local mode starts web + worker/scheduler against one canonical `CHEDDAR_DB_PATH`.
- Snapshot mode is explicit and does not silently alter local run mode.
- Scheduler tooling resolves expected path from canonical `CHEDDAR_DB_PATH` contract.
- `scripts/db-context.sh` reports effective DB source/path and key table counts.
