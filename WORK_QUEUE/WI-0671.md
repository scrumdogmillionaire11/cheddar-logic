# WI-0671: Post-Draft Season Loop Stub

**ID**: WI-0671
**Goal**: Stub the season-long loop so that after a team is locked the product has a re-entry point. This WI does not build a full weekly tracker — it builds the minimum viable surface: a locked-team record, a "come back for GW2" entry point, and a skeleton "Track my team" view that will be extended by future WIs. Without this, the product has no retention hook after draft day.

**Depends on**:
- WI-0669 (Final Recommendation lock action must exist)
- WI-0652 (ProductStore must exist)

**Scope**:
- `WORK_QUEUE/WI-0671.md`
- `cheddar-fpl-sage/backend/routers/season.py` (new — `POST /season/lock-team`, `GET /season/status`)
- `cheddar-fpl-sage/backend/models/season_models.py` (new — `LockedTeam`, `SeasonStatus`)
- `cheddar-fpl-sage/backend/services/season_service.py` (new — lock team, get status, next GW entry point)
- `cheddar-fpl-sage/tests/test_season_service.py`
- `web/src/components/fpl-season-hub.tsx` (new — post-lock landing: locked team summary + "GW2 opens [date]" banner + track CTA)

**Out of scope**:
- Actual GW-by-GW tracking, transfer suggestions, or captain advisories (future WIs)
- Performance scoring or "was Sage right?" retrospective analysis (future WIs)
- Push notifications or email reminders
- Multi-user support

**Backend behavior**:

`POST /season/lock-team` — accepts `{ "session_id": "<uuid>", "user_id": "<id>" }`. Stores the locked team state to ProductStore. Returns `{ "locked": true, "team_id": "<uuid>", "gw_next": <int>, "gw_next_deadline": "<ISO8601>" }`. Returns 409 if a team is already locked for this user.

`GET /season/status?user_id=<id>` — returns `{ "team_locked": <bool>, "team_id": "<uuid|null>", "gw_current": <int>, "gw_next": <int>, "gw_next_deadline": "<ISO8601|null>", "entry_available": <bool> }`. `entry_available` is true when GW deadline is within 72 hours or GW has started.

**UI requirements for `fpl-season-hub`**:
- Renders only after a team has been locked (shell gates on `team_locked: true` from season status).
- Shows a compact locked-team summary (formation, captain, key players — from the stored session).
- Shows "GW{N} opens {relative date}" — e.g., "GW2 opens in 5 days". Deadline from API.
- Shows a "Prepare for GW2" CTA that is active only when `entry_available: true`; greyed out otherwise.
- Mobile-width must be readable without scrolling past the fold.

**Acceptance**:
- `POST /season/lock-team` persists the locked session to ProductStore and returns expected contract.
- 409 is returned on duplicate lock attempt for same user.
- `GET /season/status` returns correct `entry_available` flag based on deadline proximity.
- `fpl-season-hub.tsx` renders locked state from props, shows correct GW banner text, and CTA state matches `entry_available`.
- `pytest tests/test_season_service.py` passes (min 10 tests: lock, duplicate-lock 409, status fields, deadline proximity logic).
- `node web/src/__tests__/fpl-season-hub-contract.test.js` passes.
