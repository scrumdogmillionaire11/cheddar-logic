# WI-1256: Replace Placeholder Quality Inputs With Real Feature Completeness

**ID**: WI-1256
**Goal**: Make quality/confidence labels auditable by removing placeholder classifier inputs and using real per-card features.

**Context**:

Hardcoded placeholder values can overstate quality and undermine the projection-only honesty contract.

**Scope**:
- `apps/worker/src/jobs/run_mlb_model.js`
- `apps/worker/src/jobs/mlb-k-input-classifier.js`
- `apps/worker/src/jobs/__tests__/mlb-k-input-classifier.test.js`
- `apps/worker/src/jobs/__tests__/run_mlb_model.test.js`
- `WORK_QUEUE/WI-1256.md`

**Out of scope**:
- New model features
- Live market integration
- Portfolio controls

**Acceptance**:
- Classifier uses real values for starter profile, opponent profile, leash confidence, and projection diagnostics.
- Card payload includes a machine-readable completeness matrix for every quality label.
- Missing critical features cannot produce upgraded quality/confidence states.
- Tests verify weak-input cards cannot be mislabeled as full quality.
