# WI-1256: Replace Placeholder Quality Inputs With Real Feature Completeness

**ID**: WI-1256
**Goal**: Make quality/confidence labels auditable by removing placeholder classifier inputs and using real per-card features.

**Context**:

Hardcoded placeholder values can overstate quality and undermine the projection-only honesty contract.

**Scope**:
- `apps/worker/src/jobs/run_mlb_model.js`
- `apps/worker/src/jobs/mlb-k-input-classifier.js`
- `apps/worker/src/jobs/__tests__/mlb-k-input-classifier.test.js`
- `apps/worker/src/jobs/__tests__/run_mlb_model.test.js`
- `WORK_QUEUE/WI-1256.md`

**Out of scope**:
- New model features
- Live market integration
- Portfolio controls

**Acceptance**:
- Classifier input contract MUST use real per-card values (not synthetic placeholders/sentinels) for:
  - `starter_profile`: `k_pct`, `swstr_pct|csw_pct`, `pitch_count_avg|ip_avg`
  - `opponent_profile`: `k_pct_vs_hand`, `contact_pct_vs_hand`, `projected_lineup_status`
  - `leash_confidence`: derived from actual card inputs used by scoring path
  - `projection_diagnostics`: projection-source and missing/degraded input diagnostics from current card payload
- Card payload MUST include a machine-readable completeness matrix under `input_completeness` with deterministic boolean leaves:
  - `input_completeness.starter_profile.*`
  - `input_completeness.opponent_profile.*`
  - `input_completeness.leash_profile.*`
  - matrix keys are stable and present for every pitcher-K card
- Quality/confidence downgrade rules MUST be deterministic:
  - any hard-missing critical field cannot resolve to `FULL_MODEL`
  - proxy-substituted critical fields cannot resolve to upgraded quality/confidence states
  - reason codes are emitted for hard-missing and proxy-substitution paths
- Tests MUST verify all of the following:
  - completeness matrix schema and key presence on emitted cards
  - weak-input cards cannot be labeled `FULL_MODEL`
  - placeholder/sentinel classifier inputs are rejected and do not upgrade quality
  - downgrade reason-code emission for each negative-path scenario
