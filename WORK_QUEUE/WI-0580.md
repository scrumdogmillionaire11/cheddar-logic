# WI-0580 — PROP Cards Not Wave-1 Eligible; V1 Wins Unconditionally

**ID:** WI-0580
**Goal:** Add `PROP` to the `WAVE1_MARKETS` set in `web/src/app/api/games/route.ts` so that `decision_v2.official_status` can override V1 `action`/`status` for NHL player prop cards. Currently V1 classification wins unconditionally for PROP cards.
**Severity:** HIGH
**Owner agent:** unassigned
**Status:** OPEN
**Time window:** —
**Coordination flag:** none

## Problem

`wave1Markets` (or equivalent gate in `route.ts`) determines whether `decision_v2.official_status` is allowed to override V1. PROP cards are not in that set, so even when V2 resolves `PLAY`, the card surfaces as `WATCH` or `PROJECTION_ONLY` because V1 retains control. This means the V2 Poisson veto and FIRE gate (WI-0577) cannot promote a card from V1's lower-confidence status.

## Scope

- `web/src/app/api/games/route.ts` — include `'PROP'` in the wave-1 market eligibility check.
- `web/src/app/api/games/__tests__/` (if exists) or add a unit test for the override path.

## Out of Scope

- Changes to V1 model logic.
- Changes to the V2 model runner.

## Acceptance

- [ ] `PROP` market key is included in the wave-1 override gate.
- [ ] A PROP card with `decision_v2.official_status = 'PLAY'` surfaces as PLAY in the API response.
- [ ] A PROP card with `decision_v2.official_status = 'WATCH'` surfaces as WATCH even if V1 says PLAY.
- [ ] Regression: existing non-PROP markets unaffected.
- [ ] All existing tests pass.
