# WI-1231: NHL Totals Shadow-Eval Calibration Infrastructure

**ID**: WI-1231
**Formerly**: WI-1224 (renumbered 2026-05-03 to resolve ID collision with COMPLETE/WI-1224 MLB F5 Settlement)
**Goal**: Emit shadow-eval records for NHL totals when raw-delta authority is active; block promotion to production if PLAY/LEAN rate delta or ECE delta exceeds tolerance over a 200-sample settled window.

**Context**:

This is the calibration guardrail from the original WI-1183 design spec, separated because it requires distinct DB/logging scope and cannot run until WI-1183 classifier changes are in place and producing real data.

Shadow-eval records must be written by the worker (single-writer contract — see ADR-0002). The web server must not write them.

**Tolerance constants (locked for implementation):**

```js
MAX_PLAY_RATE_DELTA_PCT = 0.05   // ±5 percentage point change in PLAY rate
MAX_LEAN_RATE_DELTA_PCT = 0.08   // ±8 percentage point change in LEAN/SLIGHT EDGE rate
MAX_ECE_DELTA           = 0.02   // max increase in Expected Calibration Error
```

Sign-off owner: product owner (aj). Status: approved for implementation in this WI. Any future tuning requires a new calibration WI.

**Scope**:

- `apps/worker/src/jobs/run_nhl_model.js` (shadow-eval record emission after classification)
- `apps/worker/src/models/nhl-totals-status.js` (wire legacy/raw status pairing fields and invoke shadow-eval helper)
- New file: `apps/worker/src/utils/nhl-totals-shadow-eval.js` (shadow record write + rolling window check)
- `packages/data/db/migrations/*_create_nhl_totals_shadow_eval.js` (create `nhl_totals_shadow_eval` table in worker-owned DB path only)
- `apps/worker/src/jobs/__tests__/nhl-totals-shadow-eval.test.js`
- `apps/worker/src/models/__tests__/nhl-totals-status.test.js` (integration assertions for legacy vs raw status pairing)
- `WORK_QUEUE/WI-1224.md`
- `.planning/STATE.md`

**Out of scope**:

- Classifier logic changes (→ WI-1183)
- Any web-server DB writes
- Automated promotion trigger — this WI only blocks promotion; actual promotion is a manual operator step

**Acceptance**:

- For each settled NHL totals game where `NHL_TOTALS_RAW_DELTA_AUTHORITY=true`, worker emits a shadow-eval record containing `raw_delta`, `raw_side`, `legacy_side`, `raw_status`, `legacy_status`, and `settled_result`.
- Rolling 14-day window check gate: if fewer than 200 settled records exist, `shadowEvalPromotionReady()` returns `false` with reason `INSUFFICIENT_SAMPLE`.
- If `rawPlayRate - baselinePlayRate > MAX_PLAY_RATE_DELTA_PCT`, `shadowEvalPromotionReady()` returns `false` with reason `PLAY_RATE_EXCEEDED`.
- If `rawLeanRate - baselineLeanRate > MAX_LEAN_RATE_DELTA_PCT`, returns `false` with reason `LEAN_RATE_EXCEEDED`.
- If `rawEce - baselineEce > MAX_ECE_DELTA`, returns `false` with reason `ECE_EXCEEDED`.
- Unit tests cover all four gate conditions.
