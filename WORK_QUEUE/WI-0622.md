# WI-0622: Decompose transform.ts into split transform modules

**ID**: WI-0622
**Goal**: Split `web/src/lib/game-card/transform.ts` (3,602 lines) into focused modules so each transform concern is independently testable and sport-specific additions don't require touching a single file.

**Background**:
`transform.ts` handles legacy repair, reason-code injection, title inference, market normalization, and LEGACY compatibility shims all in one file. Every transform concern is entangled; adding a new sport or market type requires touching this file.

**Target structure**:
```
web/src/lib/game-card/transform/
  legacy-repair.ts        — REPAIRED_LEGACY_CARD, PASS_UNREPAIRABLE_LEGACY paths
  market-normalize.ts     — market type normalization, period inference
  reason-codes.ts         — reason code injection and validation
  title-inference.ts      — card title construction logic
  index.ts                — composes and re-exports transform() (backward compat)
```

**Out of scope**:
- Removing legacy reason codes (tracked in WI-0624)
- Changes to transform output contracts

**Acceptance**:
- `transform/index.ts` re-exports the same `transform()` function signature.
- Each sub-module can be unit tested independently.
- `npx tsc --noEmit --project web/tsconfig.json` passes.
- `npm --prefix web run build` passes.
