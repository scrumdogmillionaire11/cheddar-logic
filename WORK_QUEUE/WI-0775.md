# WI-0775: UI market signal pills — surface public splits + sharp signals on game cards

**ID**: WI-0775
**Goal**: Render structured market signal pills on `GameCardItem` that translate raw public splits data and computed pipeline signal codes into human-readable conditions — "Public Heavy (74%)", "Sharp Divergence", "Contrarian Edge" — without conflating signal with decision. Decision (PLAY/LEAN/WATCH/PASS) remains the top layer; pills are secondary context only.

**Background**:
The three preceding WIs deliver the backend foundation:
- WI-0664: DB schema — `public_bets_pct_home/away`, `public_handle_pct_home/away`, `splits_source` columns on `odds_snapshots`
- WI-0666: Worker job — `pull_public_splits` writes live splits data into those columns on a 60-min cadence
- WI-0667: Pipeline gate — `computePublicSplitsGate` produces `PASS_SHARP_MONEY_OPPOSITE` reason code and `FADE_PUBLIC_POSITIVE` tag in card payloads when thresholds are crossed

The web layer currently does NOT pass splits percentages through to the game card API response (`public_bets_pct_home/away` are absent from `route-handler.ts` optional columns). `PASS_SHARP_MONEY_OPPOSITE` already has a dead display label in `game-card-helpers.tsx:245` but is never surfaced as a pill. This WI wires the full display layer.

**Design philosophy** (per product decision, 2026-04-03):
- Pills describe **market conditions**, never replace the decision
- Never show a single "Sharp Play" or "Fade the Public" label
- "Sharp Divergence" = the market signal (who is moving the line), not a bet recommendation
- Pills are conditional: render only when signal data is present; render nothing gracefully if splits are absent

**Pill signal mapping**:

| Pill | Color | Condition |
|------|-------|-----------|
| `Sharp Divergence` | blue | `primary_reason_code === 'PASS_SHARP_MONEY_OPPOSITE'` OR `FADE_PUBLIC_POSITIVE` in card tags |
| `Public Heavy (X%)` | amber | recommended-side `publicBetsPct > 65` (show rounded %) |
| `Contrarian Edge` | green | `FADE_PUBLIC_POSITIVE` in card tags AND recommended-side `publicBetsPct < 40` |
| `Consensus` | slate | `spreadConsensusConfidence === 'HIGH'` AND no sharp divergence signal |

When no splits data is present (`splitsSource === null`), render no pills — not a zero-state placeholder.

**Depends on**: WI-0666, WI-0667

**Scope**:
- `web/src/lib/games/route-handler.ts` — add `public_bets_pct_home`, `public_bets_pct_away`, `public_handle_pct_home`, `public_handle_pct_away`, `splits_source` to optional odds snapshot columns (same `buildOptionalOddsSelect` pattern as `spread_consensus_line`); expose as `publicBetsPctHome`, `publicBetsPctAway`, `publicHandlePctHome`, `publicHandlePctAway`, `splitsSource` on `GameResponse`
- `web/src/lib/types/game-card.ts` — add optional `marketSignals?: MarketSignalData` to `GameCard`; define `MarketSignalData` interface with the 5 fields above
- `web/src/lib/game-card/market-signals.ts` (new) — pure function `deriveMarketSignals(card: GameCard): MarketSignalPill[]`; export `MarketSignalPill` type `{ label: string; color: 'blue' | 'amber' | 'green' | 'slate' }`
- `web/src/components/cards/MarketSignalPills.tsx` (new) — renders array of pills as small inline badges below the decision block; returns `null` when `pills` is empty
- `web/src/components/cards/GameCardItem.tsx` — call `deriveMarketSignals(card)` and render `<MarketSignalPills>` in the card body, below the PLAY/LEAN badge and above the drivers list; no structural changes to existing layout
