# WI-0621: Decompose games/route.ts into web/src/lib/games/ helpers

**ID**: WI-0621
**Goal**: Extract the 30+ private helper functions inlined in `web/src/app/api/games/route.ts` (3,951 lines) into testable modules under `web/src/lib/games/`, leaving the route as a thin orchestrator.

**Background**:
The games API route contains normalization, market inference, stage counters, projection validation, and card-type contracts all inlined in a single file. Helper logic is untestable, cognitive load is enormous, and circular concerns between query-building and payload-shaping make changes risky.

**Target structure**:
```
web/src/lib/games/
  normalizers.ts      — response field normalization helpers
  market-inference.ts — market type detection, period inference
  stage-counters.ts   — card stage counting logic
  validators.ts       — projection and payload validation
  query-builder.ts    — buildOptionalOddsSelect and query composition helpers
```

**Out of scope**:
- Changes to the public API response shape
- Changes to the DB query (only extraction of helper functions)

**Acceptance**:
- Route file is under 500 lines after extraction.
- Each extracted module has at least a basic smoke test.
- `npx tsc --noEmit --project web/tsconfig.json` passes.
- `npm --prefix web run build` passes.
