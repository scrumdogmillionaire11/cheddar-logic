# WI-0850: POTD Worker Publish Path — Settlement-Compatible potd-call + Settlement Mirror

| Field | Value |
|---|---|
| **ID** | WI-0850 |
| **Goal** | Build the worker-side POTD publish pipeline that seeds bankroll, fetches active-sport odds, selects one daily play, writes `potd_plays` + `potd_bankroll`, publishes a real `potd-call` card through the existing lock/settlement contract, posts Discord, and mirrors settled `card_results` back into POTD tables. |
| **Scope** | `apps/worker/src/jobs/potd/run_potd_engine.js`, `apps/worker/src/jobs/potd/format-discord.js`, `apps/worker/src/jobs/potd/settlement-mirror.js`, `apps/worker/src/jobs/potd/__tests__/run-potd-engine.test.js`, `apps/worker/src/jobs/potd/__tests__/settlement-mirror.test.js` |
| **Out of scope** | Scheduler registration, web routes/pages, migration files already covered by WI-0848 |
| **Acceptance** | (1) `run_potd_engine` uses active odds-backed sports from `@cheddar-logic/odds`, scores candidates, and exits cleanly on no-play days. (2) The selected play is written to `potd_plays` and `potd_bankroll`, and a settlement-compatible `potd-call` is created via `insertCardPayload` rather than manual `card_payloads` SQL. (3) First run seeds bankroll to `$10.00`; `play_posted` is a zero-change ledger event. (4) Discord publishing uses `DISCORD_POTD_WEBHOOK_URL` and Discord failure is non-fatal. (5) `settlement-mirror` reads settled `card_results` for `potd-call`, updates `potd_plays`, and appends exactly one `result_settled` bankroll event per play. |
| **Owner agent** | unassigned |
| **Time window** | unassigned |
| **Coordination flag** | none |
| **Tests to run** | `npm --prefix apps/worker run test -- --runInBand src/jobs/potd/__tests__/run-potd-engine.test.js`<br>`npm --prefix apps/worker run test -- --runInBand src/jobs/potd/__tests__/settlement-mirror.test.js` |
| **Manual validation** | Run the publish job in dry-run and live test-DB modes. Confirm one `potd-call` appears in `card_payloads/card_results`, the Discord payload is formatted correctly, and a later mirrored settlement updates the POTD tables exactly once. |
