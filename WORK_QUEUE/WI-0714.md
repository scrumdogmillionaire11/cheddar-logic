# WI-0714: Fix 422 errors — split featured and period market bulk fetches

**ID:** WI-0714
**Goal:** Fix NHL and MLB odds fetches returning HTTP 422 by separating featured markets (`h2h`, `totals`, `spreads`) from period markets (`totals_p1`, `totals_1st_5_innings`) into distinct bulk API calls — one call per market group for all games, never per-game.

**Problem:**
The Odds API main `/v4/sports/{sport}/odds` endpoint only accepts featured markets (`h2h`, `spreads`, `totals`, `outrights`). Period/alternate markets (`totals_p1` for NHL 1st-period totals, `totals_1st_5_innings` for MLB F5 totals) must be fetched as a separate bulk call. Currently `SPORTS_CONFIG` mixes them into one `markets` array, and `fetchFromOddsAPI` joins them into a single request → 422 Unprocessable Entity.

NBA is unaffected — `totals` + `spreads` are both featured markets.

**Market requirements per sport (must all be fetched and applied to cards):**
- NHL: `h2h`, `totals` (featured) + `totals_p1` (period, separate call). Prop job covers `player_shots_on_goal` separately.
- MLB: `h2h` (featured) + `totals_1st_5_innings` (period, separate call). Prop job covers `pitcher_strikeouts` separately.
- NBA: `totals`, `spreads` (both featured, one call — unchanged).

**Scope:**
- `packages/odds/src/config.js`
- `packages/odds/src/index.js`

**Out of scope:**

**Acceptance:**
1. `packages/odds/src/config.js` — NHL has `markets: ['h2h', 'totals']` + `periodMarkets: ['totals_p1']`, `tokensPerFetch: 3`; MLB has `markets: ['h2h']` + `periodMarkets: ['totals_1st_5_innings']`, `tokensPerFetch: 2`; `propMarkets` and `propTokensPerRun` fields removed (never read by any consumer; prop jobs own their constants)
2. `packages/odds/src/index.js` `fetchFromOddsAPI` single-league path — if `config.periodMarkets` exists, makes a second standalone bulk call and merges period market bookmaker data into the corresponding featured game objects by `game.id` before `transformAPIResponse` runs

4. Worker logs for MLB show two `[Odds] API call` lines (`markets=h2h` then `markets=totals_1st_5_innings`), both returning 200
5. `odds_snapshots` rows for NHL games contain non-null `h2h_home`, `total`, and `total_1p_line`; MLB rows contain non-null `h2h_home` and `total_f5_line`
6. NBA fetch unchanged — one call, `markets=totals,spreads`, no errors
7. `.env` and `.env.vercel`: `ODDS_FETCH_SLOT_MINUTES=30`, `ODDS_FETCH_START_HOUR=6`
