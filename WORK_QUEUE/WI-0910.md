# WI-0910: NHL Player Shots (1P) Settlement Contract and Completeness Rules

**ID**: WI-0910
**Goal**: Define and enforce first-period player-shots settlement semantics so 1P cards settle only when first-period completeness is confirmed and grading inputs are consistent.

**Depends on**: WI-0909

**Scope**:

- apps/worker/src/jobs/settle_projections.js
- apps/worker/src/jobs/settle_pending_cards.js
- apps/worker/src/jobs/settle_game_results.js
- `apps/worker/src/jobs/__tests__/settle_projections.test.js`
- `apps/worker/src/jobs/__tests__/settle_pending_cards.phase2.test.js`

**Out of scope**:

- Full-game player-shots settlement
- NHL totals and moneyline settlement logic
- Scheduler cadence redesign

**Acceptance**:

- nhl-player-shots-1p settlement requires explicit first-period completeness before writing actual_result or card_results terminal outcomes.
- 1P grading uses period-correct values only and never falls back to full-game shots implicitly.
- Missing first-period data produces deterministic skip/error behavior with reason codes.
- Tests cover complete, incomplete, and missing-player scenarios for 1P.
