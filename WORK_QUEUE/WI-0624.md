# WI-0624: Audit and remove legacy reason codes

**ID**: WI-0624
**Goal**: Audit which cards still hit legacy reason code paths (`PROXY_LEGACY_MARKET_INFERRED`, `LEGACY_REPAIR`, `LEGACY_TITLE_INFERENCE_USED`, `PASS_UNREPAIRABLE_LEGACY`, `REPAIRED_LEGACY_CARD`), harden upstream payloads to eliminate them, then remove the shim branches.

**Background**:
These reason codes are backward-compat shims that should have been removed after the market-contract migration. They are still emitted in production via `transform.ts`, `decision-pipeline-v2.js`, and referenced in type files. New engineers add handling for them without knowing they are shims. The `canonical-play.ts` LEGACY COMPATIBILITY block (lines 268–278) preserves legacy fields with "TO BE DEPRECATED" but no enforcement date.

**Scope**:
- Audit: `grep -rn "PROXY_LEGACY_MARKET_INFERRED\|LEGACY_REPAIR\|LEGACY_TITLE_INFERENCE_USED\|PASS_UNREPAIRABLE_LEGACY\|REPAIRED_LEGACY_CARD"` across the entire codebase
- For each hit: determine if the upstream payload can be hardened to prevent the code path from being reached
- After eliminating all live occurrences: remove the shim branches from `transform.ts` and `decision-pipeline-v2.js`
- Remove legacy field block from `canonical-play.ts` (lines 268–278) once all call sites are migrated
- Remove the type union entries from `game-card.ts`

**Out of scope**:
- Changes to card display behavior
- Changes to settlement logic

**Acceptance**:
- None of the 5 legacy reason codes appear in production card payloads (verify via DB query after deploy).
- Shim branches are removed from `transform.ts` and `decision-pipeline-v2.js`.
- `npx tsc --noEmit --project web/tsconfig.json` passes.
- `npm --prefix apps/worker test` passes.
