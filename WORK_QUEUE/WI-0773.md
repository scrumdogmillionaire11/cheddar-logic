## Problem

The NHL totals model uses `getSigmaDefaults('NHL')` — a static lookup — to set outcome variance (sigma). This is called out explicitly in the source:

```
// apps/worker/src/jobs/run_nhl_model.js
// "NHL does not yet use computeSigmaFromHistory — use getSigmaDefaults as base"
```

The NBA model already solved this problem correctly:

```
// apps/worker/src/jobs/run_nba_model.js line ~858
edgeCalculator.computeSigmaFromHistory({...})
```

**Why static sigma is wrong:**
- Sigma determines how wide the confidence interval is around the projected total
- If the model's historical projections are consistently 0.3 goals off, static sigma will misprice edge
- The NHL model has been producing settled cards in `card_results` — that data exists and is unused for calibration
- Static sigma means a model with improving accuracy still uses the same variance estimate as one that is newly launched

**Contract violation:** The `computeSigmaFromHistory` function exists in `@cheddar-logic/models` and is already used by NBA. The NHL comment explicitly acknowledges the gap but was never closed.

---

## Input proof

`card_results` table contains settled NHL card records with:
- `sport = 'nhl'`
- `proj_total` — model's projected total
- `actual_total` — settled game total
- `card_type` — filter to `nhl-pace-totals`
- `created_at` — for recency windowing

`computeSigmaFromHistory` accepts `{ sport, cardType, minSamples }` and returns a calibrated sigma. Sufficient settled NHL data should exist; confirm with `SELECT COUNT(*) FROM card_results WHERE sport = 'nhl' AND card_type = 'nhl-pace-totals'`.

---

## Fix spec

In `apps/worker/src/jobs/run_nhl_model.js`:

1. At model initialization (before game loop), call `computeSigmaFromHistory({ sport: 'nhl', cardType: 'nhl-pace-totals', minSamples: 30 })`
2. If result is non-null and sufficient samples exist: use calibrated sigma in edge calculation
3. If insufficient history (< 30 settled cards): fall back to `getSigmaDefaults('NHL')` with a log line `[NHL] insufficient history for sigma calibration — using defaults`
4. Log calibrated sigma value at startup: `[NHL] sigma calibrated from N samples: {sigma}`

**Output proof:** Card payload includes `raw_data.sigma_source: 'calibrated'` or `'default'`; when calibrated, `sigma` value differs from the static default.

**Failure proof:** When `card_results` has < 30 NHL totals records, fallback to defaults is explicit (logged) — not silent.
