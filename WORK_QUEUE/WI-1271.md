# WI-1271: Freshness-Scope Scheduled Game Card Coverage

**ID**: WI-1271
**Goal**: Prevent scheduled_game_card_coverage from counting stale historical `card_payloads` rows as current coverage.

**Scope**:
- `apps/worker/src/jobs/check_pipeline_health.js`
- `apps/worker/src/jobs/__tests__/check-pipeline-health.scheduled-coverage.test.js`

**Out of scope**:
- Schema changes
- Run ID introduction
- Canonical ID reconciliation
- Web display filtering

**Contract**:
- Coverage must require freshness-qualified card rows.
- Default freshness: `card_payloads.created_at >= now - 6 hours`, same sport, same canonical game ID, publishability criteria from WI-1251 if available.

**Acceptance**:
- Historical card rows outside freshness window do not count.
- Fresh card rows count.
- Missing fresh rows produce warning/failed status based on existing thresholds.
- Reason text states fresh coverage count, not all-time coverage count.
- Tests cover stale historical row, fresh row, and mixed stale+fresh cases.

**Owner agent**: unassigned
**Time window**: 2-4 hours
**Coordination flag**: needs-sync
**Depends on**: WI-1250

**Tests to run**:
- `npm --prefix apps/worker test -- --testNamePattern="scheduled.*fresh"`

**Manual validation**:
- Execute coverage diagnostics against fixture data with stale + fresh rows and verify only fresh rows affect coverage percentage.

**Guard for WI closeout**:
- Keep existing threshold policy unchanged while updating freshness qualification.
