# WI-1172: Collapse Dead MLB SIERA/xERA Fallback Weighting to the Active Runtime Contract

**ID**: WI-1172
**Goal**: Remove the dead null-weighting branches for `siera` and `x_era` from MLB starter-skill fallback logic and align the runtime/documented contract with the signals that actually exist today.

**Scope**:

- `apps/worker/src/jobs/run_mlb_model.js`
- `apps/worker/src/jobs/__tests__/run_mlb_model.test.js`
- `docs/decisions/ADR-0007-mlb-f5-full-model-projection-contract.md`

**Out of scope**:

- Reintroducing true SIERA or xERA computation into the data pipeline
- Broad MLB model formula redesign outside the starter-skill fallback slice

**Implementation notes**:

- `getPitcherEraFromDb()` and `computeProjectionFloorF5()` still describe and implement a normalized three-signal blend (`siera`/`x_fip`/`x_era`), but current repo evidence shows the runtime path effectively rides on `x_fip` because the other signals remain absent in practice.
- Replace the null-weighting blend logic with the smallest honest contract for current data availability. Either use `x_fip` directly, or keep a multi-signal path only if this WI first proves those columns are materially populated by the active ingest path.
- Update ADR-0007 so the documented fallback contract matches the live implementation instead of describing a dormant future blend as if it were active.
- Add or update regression coverage for: populated `x_fip`, missing starter-skill data fallback, and raw-data-vs-DB fallback parity.

**Acceptance**:

- `run_mlb_model.js` no longer carries dead `siera`/`x_era` weighting branches in the fallback projection path unless backed by active populated data proven in-repo.
- The runtime contract and ADR-0007 describe the same active starter-skill fallback behavior.
- MLB model regression tests cover the simplified fallback behavior and pass.
