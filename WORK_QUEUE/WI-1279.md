# WI-1279: Harden /api/potd Error Contract to Opaque Responses

**ID**: WI-1279
**Goal**: Prevent internal exception leakage from `/api/potd` by returning opaque public errors with correlation IDs while preserving server-side diagnostics.

**Scope**:
- `web/src/app/api/potd/route.ts`
- `web/src/lib/api-security/index.ts`
- `web/src/__tests__/api-potd-error-contract.test.ts` (new)

**Out of scope**:
- POTD selection/scoring logic changes
- POTD data model changes
- Rate limit policy changes

**Acceptance**:
- `/api/potd` no longer returns raw exception strings in response bodies.
- Error responses follow the same opaque contract style used by other core APIs.
- Correlation ID is present for support/debug workflows.
- New tests assert error body shape and absence of internal stack/message leakage.

**Owner agent**: unassigned
**Time window**: 2-3 hours
**Coordination flag**: needs-sync
**Depends on**: none

**Tests to run**:
- `npm --prefix web exec tsx src/__tests__/api-potd-error-contract.test.ts`

**Manual validation**:
- Force an internal POTD exception and confirm client receives opaque error only.
- Confirm server logs still include root-cause detail and correlation ID.

**Failure modes**:
- Over-opaque responses remove useful client signal (for example retryability hints).
- Correlation ID not propagated consistently.
- Inconsistent error envelope versus other API routes.
