# WI-0433: Settlement Health Reporting for Prod Triage

**ID**: WI-0433
**Goal**: Add a read-only worker settlement health report so ops can identify unsettled plays and evaluate failed settlements in production without mutating settlement state.

**Scope**:
- `WORK_QUEUE/WI-0433.md`
- `apps/worker/src/jobs/report_settlement_health.js`
- `apps/worker/package.json`
- `apps/worker/src/__tests__/settlement-health-report.test.js`
- `docs/ops-runbook.md`

**Out of scope**:
- Changing settlement grading/write behavior in `apps/worker/src/jobs/settle_pending_cards.js`
- Web/API surface additions
- Historical backfill/resettlement behavior
- Repo-wide documentation cleanup

**Acceptance**:
- Ops can run a worker command that reports whether unsettled plays exist, including displayed/final settlement gaps and blocked reasons.
- Ops can see failed settlement totals, failure-code breakdown, sample errored rows, and recent settlement job failures from `job_runs`.
- The report runs read-only against the canonical DB and supports machine-readable JSON output.
- Worker test coverage validates coverage counts, error breakdowns, and recent job-run reporting.
- Ops runbook documents the new command and how to use it before/after settlement reruns.
