# WI-0670: Comparison as Core Behavior + Inline Tradeoff Surface

**ID**: WI-0670
**Goal**: Elevate comparison from a discrete "compare button" feature to a persistent behavior throughout the draft workbench. After onboarding, after each Draft Lab edit, and after screenshot parse — the user always sees relative position, not just absolute output. Every recommendation surface also shows explicit tradeoff deltas (what improves, what worsens).

**Depends on**:
- WI-0656 (comparison API must exist)
- WI-0660 (draft workbench components must exist)
- WI-0669 (final recommendation screen — tradeoff rendering shared)

**Scope**:
- `WORK_QUEUE/WI-0670.md`
- `cheddar-fpl-sage/backend/routers/draft_sessions.py` (add `GET /draft-sessions/{id}/delta` — change delta vs previous version)
- `cheddar-fpl-sage/backend/services/comparison_service.py` (extend with delta computation)
- `cheddar-fpl-sage/backend/models/comparison_models.py` (add `DeltaResult`, `TradeoffDelta`)

**Out of scope**:
- Multi-draft history tracking beyond immediate previous state
- Comparing more than two versions simultaneously
- Visual diff of individual player slots (out of scope for this WI)
- Changes to the existing two-candidate compare endpoint

**Behavior by surface**:

After onboarding → show 2–3 profile-matched builds side-by-side using existing `/draft-sessions/compare` endpoint. Shell passes them as initial comparison; this is not a user-triggered action.

After any Draft Lab intent → call `GET /draft-sessions/{id}/delta` to get axis-level delta vs the pre-edit state. `fpl-tradeoff-chip` renders inline below the change confirmation.

After screenshot parse → backend comparison of parsed team vs profile-optimal reference. Surface uses the same delta contract.

**`GET /draft-sessions/{id}/delta` response shape**:

```json
{
  "delta": {
    "session_id": "<uuid>",
    "baseline_label": "<string>",
    "current_label": "<string>",
    "axes": [
      { "name": "upside",      "direction": "+", "magnitude": 3, "unit": "pts" },
      { "name": "flexibility", "direction": "-", "magnitude": 2, "unit": "options" },
      { "name": "volatility",  "direction": "+", "magnitude": 1, "unit": "risk" }
    ]
  }
}
```

**`fpl-tradeoff-chip` rendering rule**: Show only axes where `magnitude > 0`. Max 3 chips per edit. Green chip for positive direction, amber for negative, neutral grey for zero. Labels and magnitudes come from API — no frontend math.

**Acceptance**:
- `GET /draft-sessions/{id}/delta` returns the delta contract or 404 if no baseline exists yet.
- `fpl-draft-compare.tsx` supports a `mode: "versus-previous"` prop in addition to existing two-candidate mode.
- After each Draft Lab edit confirmation, tradeoff chips render inline without requiring a page navigation.
- After onboarding completion, the shell shows profile-matched build comparison before entering the lab.
- `pytest tests/test_comparison_delta.py` passes (min 10 tests: positive/negative/zero axes, no-baseline 404, multi-axis ordering).
- `node web/src/__tests__/fpl-tradeoff-chip-contract.test.js` passes.
