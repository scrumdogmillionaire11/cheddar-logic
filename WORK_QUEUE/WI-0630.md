# WI-0630: Wire pull_nhl_team_stats into scheduler

**ID**: WI-0630
**Goal**: Add `pull_nhl_team_stats.js` to `schedulers/main.js` on a daily cadence so NHL team stats stay fresh without manual invocation.

**Background**:
`apps/worker/src/jobs/pull_nhl_team_stats.js` exists with `JOB_NAME = 'pull_nhl_team_stats'` and a full `withDb` / `shouldRunJobKey` / `insertJobRun` pattern — identical to other scheduled jobs. However, it is not imported or referenced in `schedulers/main.js`. The NHL model and team-metrics pipeline depend on this data. If the table goes stale, model quality degrades silently.

The job fetches from `https://api.nhle.com/stats/rest/en/team` and is self-contained. A daily early-morning run (e.g., 06:00 ET) is appropriate — team stats do not change intra-day.

**Scope**:
- `apps/worker/src/schedulers/main.js` — import `pullNhlTeamStats` and register it on a daily cadence keyed to `pull_nhl_team_stats|${cacheDate}` (same pattern as `refresh_team_metrics_daily`)

**Out of scope**:
- Changes to `pull_nhl_team_stats.js` itself
- Pull frequency changes beyond initial wiring

**Acceptance**:
- `pull_nhl_team_stats` is imported and scheduled in `schedulers/main.js`.
- Job key is date-based so it runs at most once per day.
- `node apps/worker/src/schedulers/main.js --dry-run` starts without error.
- `npm --prefix apps/worker test` passes.
