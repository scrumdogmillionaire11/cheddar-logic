# WI-0813: Fix cross-book vig removal in `market_evaluator.js` — use same-book two-sided prices

| Field | Value |
|---|---|
| **ID** | WI-0813 |
| **Goal** | Correct `selectBestExecution` so vig is removed using the same book's two-sided market rather than independently cherry-picked best-over from Book A and best-under from Book B. This eliminates ~1–2.5% systematic implied-probability understatement that makes every total and spread edge appear inflated. |
| **Priority** | High — directly inflates edge on the most-bet markets; every PLAY/LEAN card is affected |
| **Depends on** | None |
| **Coordination flag** | None — only touches the `@cheddar-logic/odds` package internals |

---

## Background

`selectBestExecution` in `packages/odds/src/market_evaluator.js` independently picks the best over-price from any book and the best under-price from any book:

```js
// Current behavior in selectTotalExecution():
const bestPriceOver  = pickBestValue(entries, 'over');   // e.g., -102 at DraftKings
const bestPriceUnder = pickBestValue(entries, 'under');  // e.g., -104 at FanDuel
```

Downstream, `computeTotalEdge` removes vig using both prices together:

```js
const noVig = noVigImplied(totalPriceOver, totalPriceUnder);
// → noVigImplied(-102, -104)
```

These prices are from different books. Their combined implied probability is ~1.96% lower than the actual vig at any single book (~4.5% at -110/-110). This makes `p_implied` systematically too small, so `edge = p_fair - p_implied` is inflated by ~1–2.5% on every total and spread market-call card.

The correct approach: when removing vig, use the **same book's** both sides. If we are pricing the OVER at Book A, remove vig using Book A's own UNDER price. Only fall back to raw implied probability (no vig removal) if the same book does not have a two-sided quote.

The same problem exists in `selectSpreadExecution` via independent `best_price_home` (Book A) / `best_price_away` (Book B) picks.

## Detailed Implementation

### Task 1 — Add same-book pairing to `selectTotalExecution`

**File:** `packages/odds/src/market_evaluator.js`

**Current:**
```js
function selectTotalExecution(entries) {
  const bestLineOver  = pickBestValue(entries, 'line', (c, cur) => c < cur);
  const bestLineUnder = pickBestValue(entries, 'line');
  const bestPriceOver = pickBestValue(entries, 'over');
  const bestPriceUnder = pickBestValue(entries, 'under');
  return {
    best_line_over: bestLineOver.value,
    best_line_over_book: bestLineOver.book,
    best_line_under: bestLineUnder.value,
    best_line_under_book: bestLineUnder.book,
    best_price_over: bestPriceOver.value,
    best_price_over_book: bestPriceOver.book,
    best_price_under: bestPriceUnder.value,
    best_price_under_book: bestPriceUnder.book,
  };
}
```

**After:**
```js
function selectTotalExecution(entries) {
  const bestLineOver  = pickBestValue(entries, 'line', (c, cur) => c < cur);
  const bestLineUnder = pickBestValue(entries, 'line');
  const bestPriceOver = pickBestValue(entries, 'over');
  const bestPriceUnder = pickBestValue(entries, 'under');

  // Same-book counterpart for vig removal
  const overBookEntry = (Array.isArray(entries) ? entries : []).find(
    e => e?.book === bestPriceOver.book && Number.isFinite(e?.under),
  );
  const underBookEntry = (Array.isArray(entries) ? entries : []).find(
    e => e?.book === bestPriceUnder.book && Number.isFinite(e?.over),
  );

  return {
    best_line_over: bestLineOver.value,
    best_line_over_book: bestLineOver.book,
    best_line_under: bestLineUnder.value,
    best_line_under_book: bestLineUnder.book,
    best_price_over: bestPriceOver.value,
    best_price_over_book: bestPriceOver.book,
    best_price_under: bestPriceUnder.value,
    best_price_under_book: bestPriceUnder.book,
    // Same-book pairs for single-book vig removal:
    same_book_under_for_over: overBookEntry?.under ?? null,
    same_book_over_for_under: underBookEntry?.over ?? null,
  };
}
```

Apply the same pattern to `selectSpreadExecution`: add `same_book_away_for_home` (away price from the book that gave the best home price) and `same_book_home_for_away`.

### Task 2 — Thread same-book fields through `normalizeGame`

**File:** `packages/odds/src/normalize.js`

Add `total_same_book_under_for_over`, `total_same_book_over_for_under`, `spread_same_book_away_for_home`, `spread_same_book_home_for_away` to the normalized game object's `odds` section, sourced from the execution block fields added in Task 1.

These fields flow into `insertOddsSnapshot` and land in the `odds_snapshots` table — or, if the snapshot schema does not include them, they flow through `raw_data` JSON. Either way they must be available to the model runner.

### Task 3 — Use same-book fields in edge calculator calls

**File:** `apps/worker/src/jobs/run_nba_model.js` (and NHL/MLB equivalents, plus `packages/models/src/decision-pipeline-v2.js`)

Where `computeTotalEdge` is called, change:

```js
// BEFORE
computeTotalEdge({
  totalPriceOver:  oddsSnapshot.total_price_over,
  totalPriceUnder: oddsSnapshot.total_price_under,
  ...
})

// AFTER: prefer same-book pair; fall back to existing fields
computeTotalEdge({
  totalPriceOver:  oddsSnapshot.total_price_over,
  totalPriceUnder: oddsSnapshot.total_same_book_under_for_over
                   ?? oddsSnapshot.total_price_under,
  ...
})
```

Apply same pattern to spread edge calls.

**Note:** Because the same-book field may be null for single-book data, the fallback to existing behavior is automatic. No existing cards break if the field is absent.
