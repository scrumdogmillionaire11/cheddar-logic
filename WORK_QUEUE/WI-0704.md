# WI-0704: Fix FPLParseReview crash on unresolved_slots undefined

**ID**: WI-0704
**Goal**: Guard `parsed_squad.unresolved_slots` and `parsed_squad.parse_warnings` against `undefined`/`null` in `FPLParseReview` so uploading any image to the Squad Audit tab never throws a runtime TypeError.

**Depends on**: none

**Scope**:
- `WORK_QUEUE/WI-0704.md`
- `web/src/components/fpl-parse-review.tsx`

**Out of scope**:
- Backend parse logic changes
- Changes to `ScreenshotParseResponse` type shape

**Acceptance**:
- Uploading an image to Squad Audit no longer throws `Cannot read properties of undefined (reading 'unresolved_slots')`.
- `unresolved_slots` and `parse_warnings` fall back to `[]` when absent in the API response.
- Existing parse-review rendering is unchanged when both fields are populated.

**Owner agent**: unassigned
**Time window**: TBD
**Coordination flag**: solo
**LOE**: XS

**Root cause**:
`fpl-parse-review.tsx` line 16 does `parsed_squad.unresolved_slots.map(...)` with no null guard. The real backend may return `null` or omit the field entirely. Same issue for `parse_warnings` at the warnings render.

**Fix**:
```tsx
// line 15-16
const unresolvedIndexes = new Set(
  (parsed_squad.unresolved_slots ?? []).map((s) => s.slot_index),
);
// and in JSX:
{(parsed_squad.parse_warnings ?? []).length > 0 && ( ... )}
```

**Tests to run**:
- `npm --prefix web run build`
- Manual: upload any image in Squad Audit tab — no crash

**Manual validation**:
- Navigate to FPL Sage → Squad Audit → upload an image
- Page renders Parse Review without error even if backend returns no `unresolved_slots`
