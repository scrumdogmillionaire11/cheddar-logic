# WI-0784: Add GitHub Actions CI workflow to run tests on every PR

| Field              | Value |
|--------------------|-------|
| **ID**             | WI-0784 |
| **Goal**           | Add a `.github/workflows/ci.yml` workflow that runs `npm test` for both `apps/worker` and `web` on every pull request to `main`, so regressions are caught before merge |
| **Priority**       | High |
| **Owner agent**    | Unassigned |
| **Time window**    | TBD |
| **Coordination flag** | `needs-sync` — adds a new file under `.github/workflows/` which is a serialized shared touchpoint |

---

## Problem

No CI test workflow exists. The only active GitHub Actions workflow is `deploy-production.yml` (production deploy, triggered on push to `main`) and `audit.yml`. There is no workflow that automatically runs the test suite on pull requests.

**Impact:**
- Regressions merged to `main` are only discovered during deployment or in production
- The existing `deployment:verify` script and test suites go completely unrun in CI on PR merges
- Contributors have no automated signal about whether their change breaks anything before review

---

## Scope

| File | Change |
|------|--------|
| `.github/workflows/ci.yml` | New workflow: trigger on `pull_request` targeting `main`; run `apps/worker` tests and `web` build+tests; use Node 20; cache `node_modules` via `actions/cache` or `actions/setup-node` cache option |

## Workflow Design

```yaml
name: CI
on:
  pull_request:
    branches: [main]

jobs:
  worker-tests:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - uses: actions/setup-node@v4
        with:
          node-version: '20'
          cache: 'npm'
          cache-dependency-path: apps/worker/package-lock.json
      - run: cd apps/worker && npm ci
      - run: cd apps/worker && npm test

  web-build:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - uses: actions/setup-node@v4
        with:
          node-version: '20'
          cache: 'npm'
          cache-dependency-path: web/package-lock.json
      - run: cd web && npm ci
      - run: cd web && npm run build
      - run: cd web && npm test
```

## Out of Scope

- Running the scheduler or any job in CI (requires Pi infrastructure)
- Running audit workflows in CI
- Adding coverage reporting (follow-on)
- Changing any test files

---

## Acceptance

- Opening a PR to `main` triggers the CI workflow
- Both `worker-tests` and `web-build` jobs appear in the PR checks
- A PR with a failing test shows a failing check and blocks merge (requires branch protection rule — document in PR description if not already set)
- Existing tests pass in the CI environment

N/A — this WI adds the CI infrastructure that runs the tests.
