# WI-0882: Fix getActivePotdSports to respect active:false in SPORTS_CONFIG

**ID**: WI-0882
**Goal**: getActivePotdSports() in run_potd_engine.js does not check
ODDS_SPORTS_CONFIG[sport].active -- it only checks the object is truthy and the
env var is not "false". Sports with active:false sneak into the POTD pipeline.
Today all active sports have active:true so this does not bite, but toggling a
sport off at season end will have no effect without this fix.

**Scope**:
- apps/worker/src/jobs/potd/run_potd_engine.js
- apps/worker/src/jobs/potd/__tests__/run-potd-engine.test.js (if test covers getActivePotdSports)

**Out of scope**:
- Changing SPORTS_CONFIG values for any sport
- Any other logic in run_potd_engine.js

**Change**:
In getActivePotdSports(), change the filter from:
  .filter(([sport, envKey]) => ODDS_SPORTS_CONFIG[sport] && process.env[envKey] !== "false")
to:
  .filter(([sport, envKey]) => ODDS_SPORTS_CONFIG[sport] && ODDS_SPORTS_CONFIG[sport].active !== false && process.env[envKey] !== "false")

**Acceptance**:
1. getActivePotdSports() excludes sports with ODDS_SPORTS_CONFIG[sport].active === false
2. Sports with active:true (or no active field) continue to be included
3. Env var "false" override continues to work
4. All existing tests pass
