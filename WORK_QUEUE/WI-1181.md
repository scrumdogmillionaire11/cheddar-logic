# WI-1181: Emit Actionable NHL model_signal Payload for POTD

**ID**: WI-1181
**Goal**: Update NHL model/card producer output so actionable NHL moneyline model decisions are emitted in a normalized `model_signal` object consumable by POTD.

**Context**:

- Current NHL model rows can be PASS/evidence-only with null edge/probability fields, which are not actionable for POTD.
- POTD requires finite model probability + edge context to qualify contract-`MODEL` NHL moneyline candidates.

**Scope**:

- `apps/worker/src/jobs/run_nhl_model.js`
- `apps/worker/src/models/nhl-pace-model.js` (only if required for payload assembly)

**Out of scope**:

- Changes to POTD runner orchestration/posting behavior.
- Changes to Kelly sizing, bankroll logic, or scheduler.
- Any cross-sport schema changes outside NHL producer payload.

**Acceptance**:

- NHL producer emits normalized `model_signal` for actionable rows with fields: `eligible_for_potd`, `market_type`, `selection_side`, `selection_team`, `model_prob`, `book_price`, `implied_prob`, `edge_pct`, `fair_price`, `edge_available`, `source`, and `blockers`.
- For non-actionable NHL rows, producer emits `eligible_for_potd=false`, `edge_available=false`, and explicit blockers (for example `NO_MARKET_LINE`, `GOALIE_CONTEXT_MISSING`) instead of ambiguous null-only payloads.
- Existing PASS/evidence semantics remain available for operator visibility, but actionable rows include model signal fields needed by POTD.
- NHL producer tests cover actionable and non-actionable payload variants.
