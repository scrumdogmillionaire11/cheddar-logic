# WI-1181: Emit Actionable NHL model_signal Payload for POTD

**ID**: WI-1181
**Goal**: Update NHL model/card producer output so actionable NHL moneyline model decisions are emitted in a normalized `model_signal` object consumable by POTD, with explicit producer-consumer contract semantics.

**Context**:

- Current NHL model rows can be PASS/evidence-only with null edge/probability fields, which are not actionable for POTD.
- POTD requires finite model probability + edge context to qualify contract-`MODEL` NHL moneyline candidates.
- Contract boundary must be explicit:
  - `payload.actionable` = execution/market availability state (producer-side operational flag)
  - `model_signal.eligible_for_potd` = POTD eligibility state (consumer-side selection gate)
  - `model_signal.selection_side` = producer-selected NHL moneyline direction for payload-backed MODEL rows
  - These are related but not identical, and POTD must gate on `model_signal.eligible_for_potd` and respect `model_signal.selection_side` for payload-backed MODEL candidates.

**Scope**:

- `apps/worker/src/jobs/run_nhl_model.js`

- `apps/worker/src/jobs/potd/signal-engine.js` (NHL consumer-side contract enforcement only)
- `apps/worker/src/jobs/__tests__/run_nhl_model*.test.js` (or equivalent NHL producer test files)

**Out of scope**:

- Any cross-sport schema changes outside NHL producer payload.

**Acceptance**:

- NHL producer emits normalized `model_signal` for actionable rows with fields: `eligible_for_potd`, `market_type`, `selection_side`, `selection_team`, `model_prob`, `book_price`, `implied_prob`, `edge_pct`, `fair_price`, `edge_available`, `source`, and `blockers`.
- For non-actionable NHL rows, producer emits `eligible_for_potd=false`, `edge_available=false`, and explicit blockers (for example `NO_MARKET_LINE`, `GOALIE_CONTEXT_MISSING`) instead of ambiguous null-only payloads.
- POTD consumption enforces `model_signal.eligible_for_potd` for NHL contract-`MODEL` candidates; POTD must not rely on generic `payload.actionable` as the acceptance gate.
- For payload-backed NHL contract-`MODEL` candidates, POTD enforces side-binding to `model_signal.selection_side`; opposite-side candidates fail closed with `MODEL_SIGNAL_INCOMPLETE` diagnostics including `SELECTION_SIDE_MISMATCH`.
- Existing PASS/evidence semantics remain available for operator visibility, but actionable rows include model signal fields needed by POTD.
- Missing or incomplete NHL `model_signal` for payload-present contract-`MODEL` candidates fails closed (`MODEL_SIGNAL_INCOMPLETE` behavior), not consensus fallback acceptance.
- NHL producer tests cover actionable and non-actionable payload variants.
- POTD contract tests cover:
  - `actionable=true` but `eligible_for_potd=false` => rejected
  - `eligible_for_potd=false` includes blockers in rejection diagnostics
  - `eligible_for_potd=true` with finite model fields => accepted
  - `selection_side=AWAY` with HOME candidate => rejected with `MODEL_SIGNAL_INCOMPLETE` + `SELECTION_SIDE_MISMATCH`
  - missing `model_signal` => fail closed
