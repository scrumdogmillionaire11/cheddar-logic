## Implementation Plan

### 1. npm scripts in `apps/worker/package.json`

Add to `"scripts"`:
```json
{
  "audit:nba": "node src/audit/run_model_audit.js --sport NBA",
  "audit:nhl": "node src/audit/run_model_audit.js --sport NHL",
  "audit:mlb": "node src/audit/run_model_audit.js --sport MLB",
  "audit:all": "node src/audit/run_model_audit.js --all",
  "audit:scorecard": "node src/audit/scorecard.js --all"
}
```

### 2. `scorecard.js`

Weekly scorecard formatter. Inputs: audit results JSON + performance drift report JSON. Outputs: JSON scorecard + markdown summary.

Scorecard JSON shape:
```json
{
  "generated_at": "2026-04-01T00:00:00Z",
  "period": "week_of_2026_03_31",
  "fixtures": {
    "total": 18,
    "passed": 16,
    "failed": 2,
    "by_sport": {
      "NBA": { "total": 6, "passed": 6, "failed": 0 },
      "NHL": { "total": 6, "passed": 5, "failed": 1 },
      "MLB": { "total": 6, "passed": 5, "failed": 1 }
    }
  },
  "top_drift_categories": ["MODEL_DRIFT", "DECISION_DRIFT"],
  "performance_alerts": [],
  "market_family_risk": {
    "NBA.totals": "LOW",
    "NHL.totals": "MEDIUM",
    "MLB.spread": "LOW"
  }
}
```

Markdown summary: one-page human-readable summary of the JSON above. Written to `audit-scorecard.md` in the audit output directory.

### 3. `.github/workflows/audit.yml`

CI behavior:
```yaml
name: Model Audit
on:
  push:
    branches: [working-branch, main]
  pull_request:
    branches: [main]
  schedule:
    - cron: '0 9 * * 1'  # Weekly Monday 09:00 UTC

jobs:
  audit:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - uses: actions/setup-node@v4
        with:
          node-version: '20'
      - name: Install dependencies
        run: npm --prefix apps/worker install
      - name: Run model audit — NBA
        run: npm --prefix apps/worker run audit:nba
      - name: Run model audit — NHL
        run: npm --prefix apps/worker run audit:nhl
      - name: Run model audit — MLB
        run: npm --prefix apps/worker run audit:mlb
      - name: Generate scorecard
        run: npm --prefix apps/worker run audit:scorecard
      - name: Upload scorecard
        uses: actions/upload-artifact@v4
        with:
          name: audit-scorecard
          path: apps/worker/audit-output/
```

CI exit behavior:
- **Fail (exit 1)** on: any `HIGH` severity invariant violation, any strict field drift vs approved baseline
- **Warn only** on: tolerance drift, performance drift threshold exceeded
- Weekly scheduled run generates full scorecard artifact

### 4. Baseline update protocol

When a baseline change is approved:
- The fixture's `expected` block must be updated with the new values
- A `_baseline_change_note` field must be added to the fixture:
  ```json
  "_baseline_change_note": {
    "changed_by": "WI-0730",
    "reason": "NBA sigma recalibrated after 2026 playoff data integrated",
    "expected_downstream_effect": "confidence values shift by ~0.02 for totals cards",
    "approved_at": "2026-04-15"
  }
  ```
- PR must include the fixture update, the change note, and the WI that drove the change
- Comparator treats any fixture with `_baseline_change_note` as having an approved baseline until the note is cleared
