# WI-0879: POTD reasoning narrative — persist and display pick rationale

**ID**: WI-0879
**Goal**: Generate a short deterministic reasoning string for the published Play of the Day and surface it in the POTD API and page alongside the pick.

**Scope**:
- `packages/data/db/migrations/*.sql` (one new migration to add nullable `reasoning` to `potd_plays`)
- `apps/worker/src/jobs/potd/signal-engine.js`
- `apps/worker/src/jobs/potd/run_potd_engine.js`
- `apps/worker/src/jobs/potd/__tests__/signal-engine.test.js`
- `apps/worker/src/jobs/potd/__tests__/run-potd-engine.test.js`
- `web/src/lib/potd-server.ts`
- `web/src/components/play-of-the-day-client.tsx`
- `web/src/__tests__/api-potd.test.js`
- `web/src/__tests__/ui-potd-smoke.test.js`

**Out of scope**:
- Discord formatting changes in `apps/worker/src/jobs/potd/format-discord.js`
- Non-POTD web surfaces such as `/cards`, `/results`, or game-card transforms
- LLM-authored or narrative-heavy copy; reasoning must be deterministic and derived from existing candidate inputs
- Historical backfill beyond allowing pre-migration rows to read as `null`

**Acceptance**:
- Worker scoring/publish path produces a concise reasoning string for the selected POTD using existing candidate facts: market type, line or price value, edge, and model/consensus source
- The published play persists the same reasoning string to both `potd_plays.reasoning` and `potd-call` `payloadData.reasoning`
- `/api/potd` returns `reasoning: string | null` on `today` and history play objects while preserving the current read-only web DB contract
- `/play-of-the-day` renders the reasoning beneath the pick when present and hides the block cleanly when reasoning is null or empty
- Focused worker and web POTD tests pass without unrelated contract changes

**Owner agent**: unassigned
**Time window**: 2026-04-11 -> TBD
**Coordination flag**: needs-sync

- `npm --prefix apps/worker test -- --testPathPattern="signal-engine|run-potd-engine"`
- `node web/src/__tests__/api-potd.test.js`
- `node web/src/__tests__/ui-potd-smoke.test.js`

**Manual validation**:
- Publish or fixture-drive one POTD row, hit `/api/potd`, and confirm `today.reasoning` matches the stored `potd-call` payload reasoning
- Open `/play-of-the-day` and confirm the reasoning block is visible for a current play and absent for a row with `reasoning=null`
