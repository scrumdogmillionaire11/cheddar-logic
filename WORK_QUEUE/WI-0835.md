# WI-0835: Sigma Provenance — Expose sigma_source on Card Payload and Logs

| Field | Value |
|---|---|
| **ID** | WI-0835 |
| **Goal** | Every card payload must carry the sigma provenance used to compute its edge. Add `sigma_source` (`'computed'` | `'fallback'`) and `sigma_games_sampled` (integer or null) to each card's `payloadData.raw_data`. Emit a structured `[SIGMA_SOURCE]` log line per model run. |
| **Scope** | `apps/worker/src/jobs/run_nba_model.js`, `apps/worker/src/jobs/run_nhl_model.js`, `apps/worker/src/jobs/run_mlb_model.js` |
| **Out of scope** | Changing sigma computation logic (WI-0814), changing edge thresholds, any UI changes, `packages/data/src/validators/card-payload.js` (raw_data is freeform — validator does not reject unknown fields; no schema change needed) |
| **Acceptance** | (1) Every PLAY/LEAN card payload has `payloadData.raw_data.sigma_source` (`'computed'` or `'fallback'`) and `payloadData.raw_data.sigma_games_sampled` (number or null). (2) `[SIGMA_SOURCE]` log emitted once per job run with sport, source, games_sampled. (3) Existing card-payload test suite still passes (confirmed by test suite — no code change to validator required). |
| **Owner agent** | unassigned |
| **Time window** | unassigned |
| **Coordination flag** | none |
| **Tests to run** | `npm --prefix apps/worker test -- --testPathPattern="run_nba_model|run_nhl_model|run_mlb_model" --no-coverage --forceExit` — in each test file assert that a generated PLAY/LEAN card has `payloadData.raw_data.sigma_source` equal to `'computed'` or `'fallback'`, and `payloadData.raw_data.sigma_games_sampled` is a number or null |
| **Manual validation** | Run `npm --prefix apps/worker run job:run-nba-model` locally; inspect card payload JSON for `payloadData.raw_data.sigma_source` and `payloadData.raw_data.sigma_games_sampled` fields |

## Context

`computeSigmaFromHistory()` already returns `sigma_source: 'computed' | 'fallback'` and optionally `games_sampled`. WI-0814 gates execution when source is `'fallback'`. This WI ensures the provenance is surfaced on each emitted card for observability, audit, and downstream consumer transparency regardless of tier.

**NHL pre-existing annotation (WI-0773):** `run_nhl_model.js` lines ~2015-2025 already annotates `card.payloadData.raw_data.sigma_source` using values `'calibrated'`/`'default'` (WI-0773 naming). This WI-0835 **supersedes** that annotation — update to use `_sigmaSource` directly and add `sigma_games_sampled`.

## Implementation Notes

**Placement:** All three sports write into `payloadData.raw_data` (consistent with the NHL WI-0773 precedent). Do not write to top-level `payloadData`.

**Field names:** Use `sigma_source` with values `'computed'`/`'fallback'` (from `computeSigmaFromHistory()` result; do not remap). Use `sigma_games_sampled` (renamed from source object's `games_sampled` key; integer or null).

**NBA** (`run_nba_model.js`): `computedSigma` is in scope at line 1130. After `attachRunId`, spread `{ sigma_source: computedSigma.sigma_source, sigma_games_sampled: computedSigma.games_sampled ?? null }` into `card.payloadData.raw_data` for every pending card. Emit `[SIGMA_SOURCE] sport=NBA ...` once after sigma resolves.

**NHL** (`run_nhl_model.js`): Update the **existing WI-0773 annotations** at lines ~2015-2025 (two spread sites). Change `sigma_source: _sigmaSource === 'computed' ? 'calibrated' : 'default'` to `sigma_source: _sigmaSource`. Add `sigma_games_sampled: _computedSigma.games_sampled ?? null` to each spread. Emit `[SIGMA_SOURCE] sport=NHL ...` once after line 1697.

**MLB** (`run_mlb_model.js`): `mlbSigma` is in scope at line 1513. Before `insertCardPayload`, spread `{ sigma_source: mlbSigma.sigma_source, sigma_games_sampled: mlbSigma.games_sampled ?? null }` into `card.payloadData.raw_data`. Emit `[SIGMA_SOURCE] sport=MLB ...` once alongside existing `[MLB_SIGMA_EMPIRICAL]` log.

**Validator:** No change required. `card-payload.js` does not validate `raw_data` field contents — adding keys to `raw_data` is a no-op. If any test fails citing `sigma_source`, add both fields as optional to the `raw_data` schema in the validator.
