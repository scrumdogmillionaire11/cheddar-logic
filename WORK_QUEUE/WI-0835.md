# WI-0835: Sigma Provenance — Expose sigma_source on Card Payload and Logs

| Field | Value |
|---|---|
| **ID** | WI-0835 |
| **Goal** | Every card payload must carry the sigma provenance used to compute its edge. Add `sigma_source` (`'computed'` | `'fallback'`) and `sigma_games_sampled` (integer or null) to each card's `payloadData`. Emit a structured `[SIGMA_SOURCE]` log line per model run. |
| **Scope** | `apps/worker/src/jobs/run_nba_model.js`, `apps/worker/src/jobs/run_nhl_model.js`, `apps/worker/src/jobs/run_mlb_model.js`, `packages/data/src/validators/card-payload.js` |
| **Out of scope** | Changing sigma computation logic (WI-0814), changing edge thresholds, any UI changes |
| **Acceptance** | (1) Every PLAY/LEAN card payload contains `sigma_source` and `sigma_games_sampled`. (2) `[SIGMA_SOURCE]` log emitted once per job run with sport, source, games_sampled. (3) Existing card-payload validator accepts the new fields without failing. |
| **Owner agent** | unassigned |
| **Time window** | unassigned |
| **Coordination flag** | none |
| **Tests to run** | `npm --prefix apps/worker test -- --testPathPattern=run_nba_model` and NHL/MLB equivalents; snapshot test for card payload shape |
| **Manual validation** | Run `npm --prefix apps/worker run job:run-nba-model` locally; inspect card payload JSON for `sigma_source` and `sigma_games_sampled` fields |

## Context

`computeSigmaFromHistory()` already returns `sigma_source: 'computed' | 'fallback'` and optionally `games_sampled`. WI-0814 gates execution when source is `'fallback'`. This WI ensures the provenance is surfaced on each emitted card for observability, audit, and downstream consumer transparency regardless of tier.

## Implementation Notes

- NBA: `computedSigma` is already in scope at job start (`run_nba_model.js:1129`). Thread `sigma_source` and `games_sampled` into each card's `payloadData` at the point of `insertCardPayload`.
- NHL: same pattern — `_sigmaSource` is already computed at line 1696.
- MLB: `mlbSigma` is already computed at line 1512.
- Add two optional fields to `card-payload.js` validator: `sigma_source` (string, optional) and `sigma_games_sampled` (integer | null, optional).
- Log format: `[SIGMA_SOURCE] sport=NBA source=fallback games_sampled=null` once per run.

## Dependencies

- WI-0814 (sigma gate) — can be done independently but complements it.
