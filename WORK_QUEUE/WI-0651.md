# WI-0651: Edge sort and filter-by-strength in cards UI

**ID**: WI-0651
**Goal**: Add an "Edge %" sort option and a minimum-edge-strength filter to the cards page so users can surface highest-edge plays without manually scanning all cards.

**Background**:
`edge_vs_consensus` and `edge_vs_best_available` were wired through to the UI in WI-0628 ✓. The data is displayed on each card but there is no way to rank or filter by it. The CLV ledger and market evaluator data are most actionable when users can prioritize by edge magnitude rather than recency order. This is the missing UX layer.

**Scope**:
- `web/src/app/cards/cards-page-client.tsx` (add edge sort option + min-edge filter pill)
- `web/src/lib/games/route-handler.ts` (add `sortByEdge` and `minEdge` query param support)
- `web/src/app/api/games/route.ts` (pass through new query params)
- `web/src/lib/types/card-filters.ts` (extend filter type if present)

**Out of scope**:
- Leaderboard page (separate surface — would be a new route)
- Edge history/trend charting
- Changes to how edge values are computed

**Acceptance**:
- Cards page has a "Sort: Edge %" option in the sort dropdown (alongside existing sort options)
- When selected, cards are ordered descending by `max(edge_vs_consensus, edge_vs_best_available)` where non-null
- A filter pill "Min Edge ≥ X%" with values [1%, 2%, 3%, 5%] is available in the filter panel
- Cards with null edge values are shown last when edge sort is active
- `npx tsc --noEmit --project web/tsconfig.json` passes
- `npm --prefix web run lint` exits 0
- Existing card filter tests continue to pass
