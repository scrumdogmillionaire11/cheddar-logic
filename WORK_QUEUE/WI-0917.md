# WI-0917: News Staleness Enforcement + Watchdog + Card Payload Contract

**ID**: WI-0917
**Goal**: Enforce news freshness, contradiction, and blocking semantics at the decision pipeline, watchdog, and payload-validator layers so stale or unresolved critical news cannot leak through as actionable plays.

**Depends on**: WI-0916

**Scope**:

- `apps/worker/src/jobs/check_pipeline_health.js`
- `apps/worker/src/utils/decision-publisher.js`
- `apps/worker/src/utils/__tests__/decision-publisher.v2.test.js`
- `packages/models/src/decision-pipeline-v2.js`
- `packages/models/src/decision-pipeline-v2-edge-config.js`
- `packages/models/src/__tests__/decision-pipeline-v2-news-watchdog.test.js`
- `packages/data/src/validators/card-payload.js`
- `packages/data/src/__tests__/validators/card-payload.news.test.js`
- `apps/worker/src/jobs/__tests__/check_pipeline_health.watchdog.test.js`

**Out of scope**:

- Polling cadence or source-adapter implementation
- New model feature engineering unrelated to news gating
- UI/dashboard changes for displaying news metadata
- Playoff overlay implementation work (separate WI)

**Acceptance**:

- `check_pipeline_health.js` checks per-sport source freshness, missing critical coverage, unresolved contradictions, and stale actionable outputs relative to latest critical news timestamps.
- Decision-pipeline/watchdog logic uses canonical reason codes and status caps; it does not introduce new public recommendation statuses.
- If `decision_ts` predates the latest critical news item for the same target, actionable output is invalidated or downgraded conservatively.
- Card payload validation requires news metadata fields needed for deterministic replay and rejects malformed news-blocking metadata.
- Actionable recommendations must carry `decision_ts`, `input_snapshot_ts`, `news_latest_item_ts`, `news_reason_codes`, and `news_blocking=false` semantics.
- Contradictory or unresolved critical news cannot surface as actionable output through silent success paths.
