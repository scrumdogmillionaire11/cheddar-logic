# WI-0464 — Player quality weighting for skater injury factors

## Status
BACKLOG

## Goal
Weight the skater injury factor by each missing player's on-ice impact (TOI,
points-per-game, xGF contribution) rather than treating all absences equally.

## Background
WI-0463 introduced a flat 3.5% reduction per confirmed-out skater.  A 4th-line
winger and a top-line centre both count as one "out" today.  A team losing
Connor McDavid should incur a much larger penalty than losing a checking forward.

## Proposed approach
1. **Data source**: MoneyPuck player-level stats CSV
   (`/moneypuck/playerData/seasonSummary/{season}/regular/skaters.csv`) already
   cached in `fetchMoneyPuckSnapshot`.  Fields available: `icetime`, `I_F_points`,
   `onIce_xGoalsFor`.
2. **Impact score per player**: `impact = clamp(toi_per_game / league_avg_toi, 0.5, 2.5)`.
3. **Weighted factor**: Replace flat count with `Σ(impact_i) × FACTOR_PER_IMPACT_UNIT`
   capped at the same `SKATER_INJURY_FACTOR_MIN = 0.88`.

## Dependencies
- WI-0463 must be deployed first (flat-factor baseline).
- MoneyPuck skater CSV parsing needs to be added to `fetchMoneyPuckSnapshot`.
- Skater name matching (MoneyPuck injury name → player stats lookup) requires
  a fuzzy-match or canonical-name mapping layer.

## Out of scope
- Defense-side weighting (skater injury factor currently applies to off-rating only).
- Real-time TOI (only season-average is used).

## Acceptance criteria
- [ ] Star player out (e.g., top-line centre, ≥22 min TOI) → factor at least 0.94
- [ ] 4th-liner out (≤11 min TOI) → factor no lower than 0.98
- [ ] Missing player not found in CSV → falls back to flat 3.5% weight
- [ ] All existing WI-0463 tests still pass
