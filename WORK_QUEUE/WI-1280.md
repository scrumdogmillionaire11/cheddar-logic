# WI-1280: Add Minimal API Performance Instrumentation for Core Read Routes

**ID**: WI-1280
**Goal**: Add low-overhead, route-level performance measurements for `/api/results`, `/api/cards`, and `/api/potd` to detect latency regressions and cache behavior issues early.

**Scope**:
- `web/src/app/api/results/route.ts`
- `web/src/app/api/cards/route.ts`
- `web/src/app/api/cards/[gameId]/route.ts`
- `web/src/app/api/potd/route.ts`
- `web/src/lib/results/cache.ts`
- `web/src/lib/api-security/perf-observability.ts` (new)
- `web/src/__tests__/api-perf-observability.test.ts` (new)

**Out of scope**:
- Third-party APM integration
- Sampling infrastructure outside existing logging
- Any API contract changes

**Acceptance**:
- Each scoped route emits duration logs with route name, status code, and row/count metadata when available.
- `/api/results` emits cache hit/miss signals tied to cache key path.
- Slow-route warnings are emitted when thresholds are exceeded.
- Tests validate log payload schema and ensure no sensitive data is emitted.

**Owner agent**: unassigned
**Time window**: 3-5 hours
**Coordination flag**: needs-sync
**Depends on**: none

**Tests to run**:
- `npm --prefix web exec tsx src/__tests__/api-perf-observability.test.ts`
- `npm --prefix web exec tsx src/__tests__/integration/results-behavioral-contract.test.ts`

**Manual validation**:
- Hit each route once and confirm expected duration + metadata logs.
- Trigger a cache hit and miss for `/api/results` and verify instrumentation output.

**Failure modes**:
- Logging overhead increases tail latency.
- Inconsistent instrumentation fields across routes reduce comparability.
- Sensitive payload details leak into logs.
