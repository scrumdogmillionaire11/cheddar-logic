# WI-0938-05: THE BOARD - Edge Type Tracker Component

**ID**: WI-0938-05  
**Parent**: WI-0938  
**Goal**: Build the Edge Type Tracker Tab component that shows historical performance by edge type. This enables users to identify repeatable patterns and learn independently.

**Depends on**: WI-0938-02 (skeleton must exist first)

**Scope**:

- `web/src/components/board/EdgeTypeTrackerTab.tsx` (main container)
- `web/src/components/board/EdgeTypeCard.tsx` (individual edge type performance card)
- Mock data integration (pull from `/api/board/edge-types` endpoint)
- Styling (follows existing Cheddar design system)
- Optional: chart/visualization for trends (bar chart, sparkline, or text-only acceptable)

**Out of scope**:

- Real historical data aggregation (backend computes this in WI-0938-06)
- Settlement grading changes (just displays what backend provides)
- Advanced charting/analytics
- Discord formatting (that's WI-0938-07)

**Acceptance**:

- `EdgeTypeTrackerTab` renders performance data for 5 edge types:
  - STALE_PRICE
  - INFORMATION
  - STRUCTURAL
  - BEHAVIORAL
  - TIMING
- Each edge type card displays:
  - Edge type name + emoji icon
  - Sample size (number of opportunities of this type)
  - Hit rate (% wins)
  - ROI (positive/negative return)
  - Trend indicator (↑ improving, → stable, ↓ deteriorating)
  - Lookback period (e.g., "last 90 days" or "2024-25 season")
  - Optional: brief explanation of what this edge type means
- Cards are sortable or filterable (optional, but nice-to-have)
- Data structure matches `/api/board/edge-types` response
- Responsive design
- TypeScript types match schema
- Unit tests for EdgeTypeCard rendering
- Manual: Cards display correctly with mock data
