# WI-1221: Enforce ENABLE_API_AUTH in Production and Add Auth Wall Smoke Test (R-01)

**ID**: WI-1221
**Goal**: Confirm `ENABLE_API_AUTH=true` is set in the production systemd environment file and add a CI smoke test that asserts non-public API routes return 401 when auth is enabled, preventing the flag from being accidentally omitted in future deploys.

**Context**:

`web/src/lib/api-security/config.ts` reads `process.env.ENABLE_API_AUTH` and defaults to `false` if unset. The auth middleware (`middleware.ts`) is a no-op unless this flag is true. All non-public API routes (`PUBLIC_ROUTES = {'/wedge', '/fpl', '/api/healthz', '/api/readyz'}`) are unprotected if the flag is missing or false.

If `ENABLE_API_AUTH` is absent from `deploy/systemd/cheddar-web.service` or the production `.env.production`, the auth wall is silently off. There is currently no test asserting auth enforcement.

This is a `needs-sync` item because it touches both `.github/workflows/ci.yml` (shared) and a new test file.

**Scope**:

- `deploy/systemd/cheddar-web.service` ŌĆö confirm or add `ENABLE_API_AUTH=true` in `[Service]` `Environment=` block (inspect current file; add if missing)
- `web/src/__tests__/api-auth-enforcement.test.ts` (new) ŌĆö integration-style test that:
  - Imports the auth middleware
  - Creates a mock NextRequest for a non-public route (e.g., `/api/cards`)
  - Asserts the middleware returns 401 when `ENABLE_API_AUTH=true`
  - Asserts the middleware returns 200/pass-through when the route is in `PUBLIC_ROUTES`
- `web/package.json` ŌĆö add `"test:api-auth": "node --import tsx/esm src/__tests__/api-auth-enforcement.test.ts"` to scripts
- `.github/workflows/ci.yml` ŌĆö add step running `npm --prefix web run test:api-auth` with `ENABLE_API_AUTH=true` in env
- `WORK_QUEUE/WI-1221.md`
- `.planning/STATE.md`

**Out of scope**:

- Implementing new auth methods or changing the auth logic itself
- Changes to `web/src/middleware.ts` beyond what is needed to make the test work
- Changes to the worker process

**Acceptance**:

- `deploy/systemd/cheddar-web.service` contains `Environment=ENABLE_API_AUTH=true` (or `EnvironmentFile=` references a file containing it)
- `web/src/__tests__/api-auth-enforcement.test.ts` exists with at least: one test asserting 401 for a protected route when `ENABLE_API_AUTH=true`; one test asserting pass-through for `/api/healthz`
- `npm --prefix web run test:api-auth` passes in CI with `ENABLE_API_AUTH=true`
- CI `validate` job runs the test; no flakiness
