# WI-0808: Add NCAAM tournament-window gate — block PLAY/WATCH cards during March Madness

| Field | Value |
|---|---|
| **ID** | WI-0808 |
| **Goal** | Prevent the NCAAM model from emitting PLAY or WATCH tier cards during the NCAA Tournament window (approx. March 15 – April 7 each year); the quant audit found NCAAM PLAY averaged −0.705/card (~16.5% implied win rate) during this period, indicating a catastrophic model inversion when tournament bracket seeding replaces regular-season matchups. |
| **Priority** | High — model emitted negative-expectation PLAY cards throughout the tournament; gate must be in place before next March |

## Context

The quant audit results:

| Tier | N | Avg PnL/card |
|---|---|---|
| NCAAM PLAY | 59 | −0.705 |
| NCAAM WATCH | 61 | −0.944 |

This is a ~16.5% implied win rate on PLAY — worse than random. The model's confidence ordering (PLAY > WATCH > LEAN) is inverted during the tournament, which suggests the regular-season projection model (point spreads, pace, efficiency margins) breaks down on single-elimination neutral-site games.

1. Is `ENABLE_NCAAM_MODEL` (or equivalent) already `false` in production `.env`? Check:

2. What dates did the 59 PLAY cards span? Run on Pi:
   ```sql
   SELECT DATE(created_at), COUNT(*), AVG(pnl)
   FROM card_results
   WHERE status = 'settled'
     AND json_extract(payload, '$.sport') = 'NCAAM'
     AND json_extract(payload, '$.decision') = 'PLAY'
   GROUP BY 1 ORDER BY 1;
   ```
3. Does the NCAAM model file have any existing season-gate logic?
   ```bash
   grep -n "tournament\|march\|MARCH\|gate\|season_end\|end_date" \
   ```

If `ENABLE_NCAAM_MODEL=false` in prod and dates confirm it was temporarily enabled during the tournament accidentally, the fix is a config guard + documentation. If the model was legitimately running, add a date-range gate.

## Scope

**Files to change (after diagnosis):**

Option A — Date-range gate in model runner (preferred if env flag doesn't exist):
- `apps/worker/src/jobs/run_ncaam_model.js` — add tournament window check at job entry:
  ```js
  const TOURNAMENT_START = { month: 3, day: 15 }; // Mar 15
  const TOURNAMENT_END   = { month: 4, day: 7  }; // Apr  7
  function inTournamentWindow(now = new Date()) {
    const m = now.getMonth() + 1, d = now.getDate();
    const afterStart  = m >  TOURNAMENT_START.month || (m === TOURNAMENT_START.month && d >= TOURNAMENT_START.day);
    const beforeEnd   = m <  TOURNAMENT_END.month   || (m === TOURNAMENT_END.month   && d <= TOURNAMENT_END.day);
    return afterStart && beforeEnd;
  }
  // At job entry:
  if (inTournamentWindow()) {
    logger.info('NCAAM model skipped — tournament window active (Mar 15 – Apr 7)');
    return { skipped: true, reason: 'TOURNAMENT_WINDOW' };
  }
  ```

Option B — Env flag guard (if `ENABLE_NCAAM_MODEL` already exists):
- Update prod `.env.example` and deployment docs to note flag must be `false` from Mar 15–Apr 7
- Add startup assertion that logs a warning if the flag is `true` during the tournament window

**Out of scope:**
- Building a tournament-specific NCAAM model (out of scope indefinitely)
- Changing NCAAM card generation for regular season
- Any NBA / NHL / MLB / NFL model changes
- Modifying the web-facing results pages
