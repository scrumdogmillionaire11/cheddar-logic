# WI-0836: Rest-Days Pipeline — Compute and Pass homeRest/awayRest into Live Models

| Field | Value |
|---|---|
| **ID** | WI-0836 |
| **Goal** | Compute `homeRest` and `awayRest` (days since each team's last game) from the `games` table and thread those values into `projectNBA()` and `projectNHL()` calls. The back-to-back penalty (−4 pts home / back-to-back exhaustion) and rest bonus (+2 pts) are silently zeroed in production because `homeRest`/`awayRest` are never passed. |
| **Scope** | `apps/worker/src/jobs/run_nba_model.js`, `apps/worker/src/jobs/run_nhl_model.js`, `packages/data/src/db/games.js`, `apps/worker/src/models/projections.js` |

(2) When a prior game is not found (season opener, new team), defaults to `restAdjustment(1)` (no adjustment) with a `rest_source: 'default'` flag on the card. (3) Test: mock games table with back-to-back; assert resulting projection is lower than baseline. |

## Context

`projectNBA()` in [apps/worker/src/models/projections.js](apps/worker/src/models/projections.js#L158) accepts `homeRest`/`awayRest` parameters. Live call sites in `run_nba_model.js` never pass these values — they fall to `undefined`, then `NaN`, then `0` via the final `else` branch of `restAdjustment()`. Backtests that used schedule data had the signal; production does not. The asymmetry silently removes a real signal from edge calculations.

## Implementation Notes

1. **Add `getTeamLastGameDate(db, teamName, sport, beforeDate)`** to `packages/data/src/db/games.js` — returns the most recent `game_date` for the team before `beforeDate`, or `null` if no prior game found.
2. **Compute rest in `run_nba_model.js`** before calling the projection: `homeRest = homeLastGameDate ? daysBetween(homeLastGameDate, gameDate) : 1`. Cap at `3` (well-rested plateau).
3. **Pass into projection call** the computed values.
4. **Add `rest_days_home` / `rest_days_away`** to card `payloadData` for observability.
5. Repeat for `run_nhl_model.js`.
