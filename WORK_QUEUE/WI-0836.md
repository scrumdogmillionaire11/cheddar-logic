## Context

**NBA:** `computeNBAMarketDecisions` starts at line 820. Its rest read block (~lines 855-859) has **no** `raw?.rest_days_home` fallback:

```js
const restDaysHome = toNumber(raw?.espn_metrics?.home?.metrics?.restDays ?? null);
const restDaysAway = toNumber(raw?.espn_metrics?.away?.metrics?.restDays ?? null);
```

Enriching `oddsSnapshot` in the job is not sufficient alone — `cross-market.js` must also be patched to add the fallback, otherwise the injected `rest_days_home`/`rest_days_away` fields are silently ignored. Activation requires both step 2 (job enrichment) and step 2b (cross-market.js fallback).

**NHL:** `computeNHLMarketDecisions` starts at line 260. Its rest read block (~lines 288-292) already has the fallback:

```js
const restDaysHome = toNumber(raw?.espn_metrics?.home?.metrics?.restDays ?? raw?.rest_days_home ?? null);
const restDaysAway = toNumber(raw?.espn_metrics?.away?.metrics?.restDays ?? raw?.rest_days_away ?? null);
```

Enriching the NHL snapshot in `run_nhl_model.js` WILL work without any additional `cross-market.js` patch. No step 2b equivalent is needed for NHL.

**NHL (projections):** `projectNHL()` (projections.js line 292) signature ends at `awayGoalieConfirmed` — no rest params exist. Rest adjustment logic must be added to match `projectNBA()`'s pattern.

**Existing pattern:** `getHomeTeamRecentRoadTrip()` (run_nba_model.js line ~517) queries `games` directly using `getDatabase()` and the `game_time_utc` column (NOT `game_date` — that column does not exist). The new rest-days helper follows this exact pattern as a local `apps/worker` utility, avoiding the package export chain entirely.

## Implementation Notes

1. **Create `apps/worker/src/utils/rest-days.js`** with three exports:

   - `getTeamLastGameTimeUtc(teamName, sport, beforeUtc)` — queries `games` table using `getDatabase()`. SQL: `SELECT game_time_utc FROM games WHERE LOWER(sport) = ? AND (UPPER(home_team) = UPPER(?) OR UPPER(away_team) = UPPER(?)) AND game_time_utc < ? AND status IN ('final', 'STATUS_FINAL') ORDER BY game_time_utc DESC LIMIT 1`. Returns string or `null`.

   - `daysBetween(earlierUtcIso, laterUtcIso)` — `Math.min(3, Math.max(0, Math.floor((new Date(laterUtcIso) - new Date(earlierUtcIso)) / 86400000)))`. Cap=3 (well-rested plateau), floor=0 (back-to-back).

   - `computeRestDays(teamName, sport, gameTimeUtc)` — calls `getTeamLastGameTimeUtc`, returns `{ restDays, restSource }`. If no prior game found: `{ restDays: 1, restSource: 'default' }`.

2. **Enrich NBA snapshot** in `run_nba_model.js`, immediately before `computeNBAMarketDecisions` (~line 1313). Add require at the top of the file. Replace the existing call:

   ```js
   const { computeRestDays } = require('../utils/rest-days');
   const homeRestResult = computeRestDays(oddsSnapshot.home_team, 'nba', oddsSnapshot.game_time_utc);
   const awayRestResult = computeRestDays(oddsSnapshot.away_team, 'nba', oddsSnapshot.game_time_utc);
   const enrichedSnapshot = {
     ...oddsSnapshot,
     rest_days_home: homeRestResult.restDays,
     rest_days_away: awayRestResult.restDays,
   };
   const nbaMarketDecisions = computeNBAMarketDecisions(enrichedSnapshot);
   ```

   Use `enrichedSnapshot` in the `computeNBAMarketDecisions` call **only** — not `computeNBADriverCards` (that function does not consume rest). Add `rest_days_home`, `rest_days_away`, `rest_source_home`, `rest_source_away` to any card `payloadData` writes for observability.

2b. **Add `raw?.rest_days_home` fallback to NBA rest read in `cross-market.js`** inside `computeNBAMarketDecisions` (~lines 855-859). Update the two rest-read lines to match the NHL pattern already in place at lines 288-292:

   ```js
   const restDaysHome = toNumber(raw?.espn_metrics?.home?.metrics?.restDays ?? raw?.rest_days_home ?? null);
   const restDaysAway = toNumber(raw?.espn_metrics?.away?.metrics?.restDays ?? raw?.rest_days_away ?? null);
   ```

   Without this, step 2 enrichment is silently ignored by `computeNBAMarketDecisions`.

3. **Add rest params to `projectNHL`** in `projections.js`. Append `homeRest = 1, awayRest = 1` as params 7 and 8 (defaults = 1 preserves existing call sites). Copy the `restAdjustment` helper from `projectNBA` (lines 214-221). Apply after the goalie penalty block, before confidence:

   ```js
   const restAdjustment = (rest) => { /* same formula as projectNBA */ };
   homeProjected += restAdjustment(homeRest);
   awayProjected += restAdjustment(awayRest);
   ```

   Add `homeRestAdj` and `awayRestAdj` to the return object for testability.

4. **Activate NHL rest signal — two-part change:**

   **4a. Enrich NHL snapshot** in `run_nhl_model.js`, immediately before `computeNHLMarketDecisions` (~line 1915). Add require at the top of the file. Apply the same `enrichedSnapshot` pattern as step 2 (sport `'nhl'`):

   ```js
   const { computeRestDays } = require('../utils/rest-days');
   const homeRestResult = computeRestDays(oddsSnapshot.home_team, 'nhl', oddsSnapshot.game_time_utc);
   const awayRestResult = computeRestDays(oddsSnapshot.away_team, 'nhl', oddsSnapshot.game_time_utc);
   const enrichedSnapshot = {
     ...oddsSnapshot,
     rest_days_home: homeRestResult.restDays,
     rest_days_away: awayRestResult.restDays,
   };
   const nhlMarketDecisions = computeNHLMarketDecisions(enrichedSnapshot);
   ```

   Add `rest_days_home`, `rest_days_away`, `rest_source_home`, `rest_source_away` to NHL card `payloadData` for observability.

   **4b. Wire rest values into `projectNHL` call** in `cross-market.js` `computeNHLMarketDecisions` (~line 351). `restDaysHome` and `restDaysAway` are already read from the enriched snapshot at lines 289-292 — pass them as args 7 and 8:

   ```js
   const nhlResult = projectNHL(
     homeGoalsFor, homeGoalsAgainst,
     awayGoalsFor, awayGoalsAgainst,
     homeGoalieConfirmed, awayGoalieConfirmed,
     restDaysHome ?? 1, restDaysAway ?? 1   // <-- add these
   );
   ```

5. **Write unit tests** in `apps/worker/src/utils/__tests__/rest-days.test.js`: mock `getDatabase` returning a prepared-statement stub. Assert: `restDays=0` for back-to-back (0 days apart), `restDays=2` for 50h gap, `restDays=3` for 96h gap (cap hit), `{ restDays: 1, restSource: 'default' }` when no prior game.
