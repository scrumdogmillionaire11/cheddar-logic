# WI-1021: NBA Phase 2A — Dynamic league baselines for pace synergy

**ID**: WI-1021
**Goal**: Replace the hardcoded 2025-26 season pace range constants in nba-pace-synergy.js with dynamically computed values from the live team_metrics_cache, so the model adapts to the actual current-season environment instead of freezing at October values.

**Depends on**: WI-1018 (pace cap must be in place before widening the dynamic range)

**Scope**:
- `apps/worker/src/models/nba-pace-synergy.js`
- `apps/worker/src/jobs/run_nba_model.js` (caller update to pass db for baseline computation)
- `apps/worker/src/models/__tests__/nba-pace-normalization.test.js`

**Out of scope**:
- Rolling bias (WI-1020)
- Lineup/injury modeling (WI-1022)
- NHL, MLB, NFL models
- Any UI changes

**Acceptance**:
- `PACE_MIN` and `PACE_MAX` are no longer hardcoded constants at module level; they are derived at runtime.
- New exported function `computeNbaLeagueBaselines({ db })` queries `team_metrics_cache` for NBA teams with `updated_at` within the last 14 days, computes p5/p95 of `pace` values, and returns `{ paceMin, paceMax, medianOffEff, gamesUsed }`.
- When fewer than 10 teams are available, `computeNbaLeagueBaselines` returns a safe fallback using the original hardcoded values and logs `[NBA_BASELINES] insufficient data — using hardcoded fallback`.
- `analyzePaceSynergy` signature gains an optional `{ paceMin, paceMax }` parameter; callers that do not pass it get the hardcoded fallback.
- `run_nba_model.js` calls `computeNbaLeagueBaselines({ db })` once at job start (before per-game loop) and passes result into `analyzePaceSynergy` calls.
- `LEAGUE_MEDIAN_OFF_EFF` (113.0) is similarly derived from the median `offensive_rating` in `team_metrics_cache`, with hardcoded 113.0 as fallback.
- All existing pace normalization tests still pass (update fixtures if needed to pass explicit baselines).

**Implementation notes**:

Query for baselines:

```sql
SELECT pace, offensive_rating
FROM team_metrics_cache
WHERE LOWER(sport) = 'nba'
  AND pace IS NOT NULL
  AND updated_at >= datetime('now', '-14 days')
ORDER BY pace ASC
```

Compute p5/p95:

```js
const sorted = rows.map(r => r.pace).sort((a,b) => a-b);
const p5idx = Math.floor(sorted.length * 0.05);
const p95idx = Math.ceil(sorted.length * 0.95) - 1;
const paceMin = sorted[Math.max(0, p5idx)];
const paceMax = sorted[Math.min(sorted.length - 1, p95idx)];
```

Pass through to `paceToPct(pace, { paceMin, paceMax })` instead of module-level constants.
