## Context

Four constants in `mlb-model.js` are 2024-era values with no automation. The highest-impact offender is **`LEAGUE_AVG_K_PCT = 0.225`** (line 933), used as a normalization denominator in relative K% scoring: `(effectiveStarterKPct * oppKPctVsHand) / LEAGUE_AVG_K_PCT` (line 1175). League K% has drifted from 22% (2021) to 23%+ (2024+); a stale denominator shifts relative-edge scores systematically for every game. The second constant is **`MLB_F5_DEFAULT_TEAM_K_PCT = 0.225`** (line 21), used as team-offense fallback when no split profile exists (line 112: `skillRa9 *= clampValue(1 - (kPct - MLB_F5_DEFAULT_TEAM_K_PCT) * 0.35, 0.88, 1.12)`). Both represent the same concept — league-average K% — and should be replaced by a single computed value. `MLB_F5_DEFAULT_XFIP` and `MLB_F5_DEFAULT_TEAM_BB_PCT` are lower-stakes but also computable from `mlb_pitcher_stats.x_fip` and `mlb_pitcher_stats.bb_pct`.

## Injection Mechanism

The computed constants are used in sub-functions (`resolveStarterSkillProfile`, `resolveOpponentPitcherKProfile`, `calculateProjectionK`), not inside `runMLBModel` directly. Shadow consts in `runMLBModel` scope cannot reach these call sites. Use a **module-level mutable + setter** pattern instead — no sub-function signature changes required.

1. `packages/data/src/db/mlb-stats.js` exports `computeMLBLeagueAverages(db)`.
2. `packages/data/src/db/index.js` re-exports `computeMLBLeagueAverages`.
3. In `mlb-model.js`, add three module-level mutables initialized to the existing static fallbacks, plus a setter:
   ```js
   // module-level mutables — replaced at job start by computeMLBLeagueAverages
   let _leagueAvgKPct = LEAGUE_AVG_K_PCT;        // covers both LEAGUE_AVG_K_PCT and MLB_F5_DEFAULT_TEAM_K_PCT
   let _defaultXfip   = MLB_F5_DEFAULT_XFIP;
   let _defaultBbPct  = MLB_F5_DEFAULT_TEAM_BB_PCT;

   function setLeagueConstants({ kPct, xfip, bbPct } = {}) {
     _leagueAvgKPct = kPct  ?? LEAGUE_AVG_K_PCT;
     _defaultXfip   = xfip  ?? MLB_F5_DEFAULT_XFIP;
     _defaultBbPct  = bbPct ?? MLB_F5_DEFAULT_TEAM_BB_PCT;
   }
   module.exports.setLeagueConstants = setLeagueConstants;
   ```
4. Replace the four constant references in their respective sub-functions:
   - Line 112 in `resolveStarterSkillProfile`: `MLB_F5_DEFAULT_TEAM_K_PCT` → `_leagueAvgKPct`
   - Line 1075 in `resolveOpponentPitcherKProfile`: `LEAGUE_AVG_K_PCT` → `_leagueAvgKPct`
   - Lines 1147, 1154, 1175 in `calculateProjectionK`: `LEAGUE_AVG_K_PCT` → `_leagueAvgKPct`
   - `MLB_F5_DEFAULT_XFIP` and `MLB_F5_DEFAULT_TEAM_BB_PCT` references → `_defaultXfip` / `_defaultBbPct` at their respective call sites
5. `run_mlb_model.js` imports both `computeMLBLeagueAverages` and `setLeagueConstants`, calls them once before the per-game loop:
   ```js
   const leagueConstants = computeMLBLeagueAverages(db);
   setLeagueConstants(leagueConstants);
   ```
