# WI-1035: Add time-proximity guard to `getProbableStarterMapRow` to prevent cross-game pitcher bleed

**ID**: WI-1035
**Goal**: Prevent `getProbableStarterMapRow` from silently binding a probable starter row from a *different* game on the same calendar date. When two COL @ LAD games share `game_date = 2026-04-19`, the closest-match ordering picks the only available row even when the `scheduled_start_utc` delta is ~19 hours, causing today's confirmed starters (Sheehan / Feltner) to appear on tomorrow's card.

**Depends on**: none

**Out of scope**:

**Root cause (confirmed)**:

`mlb_probable_starter_map` has one entry for `game_date = 2026-04-19` (COL @ LAD 00:10 UTC — tonight's game). The query in `getProbableStarterMapRow` matches on `(game_date, home_team_abbr, away_team_abbr)` and orders by `ABS(time_diff)` ascending. Because there is only one row, it is returned for the 19:11 UTC game too (tomorrow's 3:11 PM EDT game), binding Sheehan/Feltner to a game they will not start.

**Fix**:

Add a post-query proximity guard: if the returned row's `scheduled_start_utc` differs from `oddsSnapshot.game_time_utc` by more than 4 hours, return `null` instead of the mismatched row.

4 hours is chosen because:
- Double-headers on the same date are typically separated by 3–4 hours at minimum
- The misbinding scenario involves an 18–19 hour delta
- It leaves headroom for clock drift / minor schedule shifts without false negatives

```js
// Inside getProbableStarterMapRow, after `const row = rows[0] ?? null;`:
if (row && scheduledStartUtc) {
  const rowMs = Date.parse(row.scheduled_start_utc);
  const snapshotMs = Date.parse(scheduledStartUtc);
  const deltaHours = Math.abs(rowMs - snapshotMs) / 3_600_000;
  if (deltaHours > 4) return null;
}
return row;
```

**Acceptance**:
- `getProbableStarterMapRow` returns `null` when the closest matching starter-map row differs from the snapshot's `game_time_utc` by more than 4 hours
- `getProbableStarterMapRow` still returns the row when the time delta is within 4 hours
- The LAD @ COL 19:11 UTC card no longer shows Sheehan/Feltner in its pitcher projections when only a 00:10 UTC starter-map row exists for that date
- Existing unit tests in `run_mlb_model.test.js` pass without modification
