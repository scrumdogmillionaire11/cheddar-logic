# WI-1035: Require confirmed starter-map match before binding pitcher projections

**ID**: WI-1035
**Goal**: `getProbableStarterMapRow` must only return a starter-map row that was **confirmed for the specific game being modeled**. The current query matches on `(game_date, home_team_abbr, away_team_abbr)` alone — when two games share the same calendar date and teams (e.g. a scheduled doubleheader or back-to-back series), the "closest match" ordering returns the wrong game's confirmed starters. If no row is confirmed for the target game, the function must return `null` so no pitcher projection appears rather than silently binding starters from a different game.

**Root cause (confirmed)**:

`mlb_probable_starter_map` stores starters keyed by `(game_date, home_team_abbr, away_team_abbr, scheduled_start_utc)`. The lookup in `getProbableStarterMapRow` queries by date + team only and picks the row with the smallest UTC time delta. On Apr 18 there was exactly one COL @ LAD entry (`scheduled_start_utc = 2026-04-19T00:10Z`). The Apr 19 19:11 UTC game (a different, later game) matched the same date+team query and received Sheehan/Feltner — today's confirmed starters — on tomorrow's card.

**Fix**:

Add a post-query proximity guard: if the best-matching row's `scheduled_start_utc` is more than **4 hours** from `oddsSnapshot.game_time_utc`, return `null`.

Rationale for 4 hours:
- MLB same-day doubleheaders separate games by 3–4 hours at minimum
- The misbinding scenario involves an 18–19 hour delta — safely rejected
- Leaves headroom for minor schedule shifts without false negatives

```js
// Inside getProbableStarterMapRow, replace the final `return rows[0] ?? null;`:
const row = rows[0] ?? null;
if (row && scheduledStartUtc) {
  const rowMs = Date.parse(row.scheduled_start_utc);
  const snapshotMs = Date.parse(scheduledStartUtc);
  if (Number.isFinite(rowMs) && Number.isFinite(snapshotMs)) {
    const deltaHours = Math.abs(rowMs - snapshotMs) / 3_600_000;
    if (deltaHours > 4) return null;
  }
}
return row;
```

When `null` is returned, `getProbableStarterIdentity` returns `null` for both sides, and `enrichMlbPitcherData` falls through to `getPitcherRow` returning `null`. The pitcher-K card for that game will correctly show "Recent form unavailable" / "No line applied" rather than projections from the wrong game's starters.

**Acceptance**:
- `getProbableStarterMapRow` returns `null` when the nearest starter-map row differs from `game_time_utc` by more than 4 hours
- `getProbableStarterMapRow` returns the row when the delta is ≤ 4 hours (today's confirmed game is unaffected)
- The COL @ LAD 19:11 UTC card shows no pitcher projection data when only a 00:10 UTC starter-map row exists for that date
- The COL @ LAD 00:10 UTC card (tonight) correctly shows Sheehan/Feltner
