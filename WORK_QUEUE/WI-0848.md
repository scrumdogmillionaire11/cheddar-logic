# WI-0848: POTD DB Contract — potd_plays + potd_bankroll Schema

| Field | Value |
|---|---|
| **ID** | WI-0848 |
| **Goal** | Add the dedicated POTD tables and indexes needed for daily play history, settlement mirroring, and bankroll ledgering without violating the single-writer DB contract. |
| **Scope** | `packages/data/db/migrations/063_create_potd_plays.sql`, `packages/data/db/migrations/064_create_potd_bankroll.sql` |
| **Out of scope** | Worker job logic, scheduler changes, web routes/components, odds scoring logic |
| **Acceptance** | (1) `potd_plays` exists with `play_date UNIQUE`, `card_id UNIQUE`, game/pick/price/confidence/score fields, and settlement fields (`result`, `settled_at`, `pnl_dollars`). (2) `potd_bankroll` exists as an append-only ledger with `event_type`, `play_id`, `card_id`, `amount_before`, `amount_change`, `amount_after`. (3) Both migrations use `IF NOT EXISTS` patterns consistent with the repo. (4) The migration suite applies cleanly on a fresh DB and the new tables are queryable afterward. |
| **Owner agent** | unassigned |
| **Time window** | unassigned |
| **Coordination flag** | none |
| **Tests to run** | `node packages/data/src/migrate.js` |
| **Manual validation** | Open the resulting DB and confirm `potd_plays` and `potd_bankroll` exist with the expected columns, unique constraints, and indexes. |

## Notes

- Bankroll seeding is runtime behavior, not migration behavior.
- No web write paths are introduced here.
