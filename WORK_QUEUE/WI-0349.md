# WI-0349: Per-Game Pipeline State Contract (Wave-1)

**ID**: WI-0349
**Goal**: Add a canonical per-game pipeline state structure with stage checkpoints and explicit blocking reason codes for missing-data diagnosis, emitted in worker outputs and logs.

**Scope**:

- `WORK_QUEUE/WI-0349.md`
- `WORK_QUEUE/WI-0372.md`
- `apps/worker/src/jobs/run_nba_model.js`
- `apps/worker/src/jobs/run_nhl_model.js`
- `apps/worker/src/jobs/run_ncaam_model.js`
- `apps/worker/src/models/cross-market.js`
- `apps/worker/src/utils/decision-publisher.js`
- `packages/models/src/decision-pipeline-v2.js`
- `docs/DATA_CONTRACTS.md`


**Out of scope**:

- Team alias normalization changes
- Projection formula changes
- Driver model logic changes
- Schema migrations
- Downstream web/API removal of legacy compatibility fields


**Acceptance**:

- Each game emits a `pipeline_state` with stages: `ingested`, `team_mapping_ok`, `odds_ok`, `market_lines_ok`, `projection_ready`, `drivers_ready`, `pricing_ready`, `card_ready`, plus `blocking_reason_codes`.
- Pipeline reasons reuse existing `decision_v2` reason vocab when applicable; no new generic reason placeholders are introduced.
- `pipeline_state` is emitted alongside model job summaries and per-game decisions, without partial publish mid-run.
- `decision_publisher` passes through pipeline metadata without altering `decision_v2` verdicts.
- Changes are additive and non-breaking: existing verdict/action/classification fields continue to be emitted unchanged.
- Follow-up tech debt created from this work is logged in a new work item.


**Tests to run**:

- `npm --prefix apps/worker test -- src/utils/__tests__/decision-publisher.v2.test.js`
- `npm --prefix apps/worker test -- src/models/__tests__/cross-market.test.js`


**Manual validation**:
