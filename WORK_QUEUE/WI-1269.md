# WI-1269: Quarantine Historical Display-Enrollment Debt

## ID
WI-1269

## Goal
Separate legacy display-enrollment residue from current-path regressions in visibility and settlement audits.

## Scope
- `apps/worker/src/jobs/report_settlement_health.js`
- `apps/worker/src/__tests__/settlement-health-report.test.js`
- `packages/data/src/db/cards.js`
- `docs/audits/card-visibility-integrity-2026-05-04.md` read-only baseline
- `docs/audits/card-visibility-integrity-quarantine-addendum-YYYY-MM-DD.md` new

## Out of scope
- Rewriting historical card payloads
- Settlement result recalculation
- User-facing API changes
- Editing the original audit snapshot except to reference the addendum, if absolutely necessary
- Schema migration unless proven unavoidable

## Historical Quarantine Contract

Rows may be classified into:

| Bucket | Meaning |
|---|---|
| `LEGACY_QUARANTINED` | Known pre-fix display-enrollment residue |
| `CURRENT_PATH_DEFECT` | New/current write path defect that should count against regression health |
| `UNKNOWN_UNCLASSIFIED` | Not enough evidence to safely classify |

Classification must be non-destructive:
- no mutation of card payload content
- no settlement recalculation
- no hard deletion
- no backdating

## Implementation requirements

- Add quarantine classification logic to settlement-health diagnostics.
- Preserve the dated 2026-05-04 audit as baseline evidence.
- Write a new addendum documenting the quarantine split and classification rules.
- Prefer runtime/report classification over schema changes.
- If a migration is deemed necessary, stop and split that into a separate WI.

## Acceptance
- Audit output separates:
  - `LEGACY_QUARANTINED`
  - `CURRENT_PATH_DEFECT`
  - `UNKNOWN_UNCLASSIFIED`
- Current-path regression counts exclude `LEGACY_QUARANTINED`.
- Unknown rows are not silently treated as clean.
- Deterministic test fixture proves legacy rows do not pollute current defect counts.
- Deterministic test fixture proves current-path defects still count.
- Original audit snapshot remains preserved as baseline evidence.
- No historical payload content is mutated.

* Run settlement-health diagnostics on a fixture with:

  * known current-path defect rows
  * unknown/unclassified rows
* Confirm output separates all three buckets.
* Confirm current-path defect count excludes quarantined legacy rows.
* Confirm unknown rows are visible and not hidden.
