# WI-0843: Fix card_results Duplication — Deduplicate at Write or Add Primary Flag

| Field | Value |
|---|---|
| **ID** | WI-0843 |
| **Goal** | `card_results` contains 1.7–2.1× duplicate rows per unique market (same `game_id + card_type + recommended_bet_type`). This inflates raw hit rate queries, misleads settlement reporting, and causes the admin odds/model-output tables to show phantom activity. The correct fix is to either (a) enforce a unique constraint at insertion time to deduplicate upstream, or (b) add an `is_primary BOOLEAN DEFAULT 1` flag where only the latest card per market is primary, and all downstream queries filter on `is_primary = 1`. |
| **Scope** | `apps/worker/src/jobs/settle_game_results.js`, whichever job writes `card_results` rows (likely `run_nba_model.js` / `run_nhl_model.js` / `run_mlb_model.js`), `packages/data/db/migrations/` for schema change |
| **Out of scope** | Frontend changes, changing card_payloads table, historic backfill beyond setting `is_primary` flag on existing rows |
| **Acceptance** | (1) Identify the write path that creates duplicate card_results rows. (2) Either: add `UNIQUE(game_id, card_type, recommended_bet_type)` constraint with `ON CONFLICT REPLACE`, OR add `is_primary INTEGER NOT NULL DEFAULT 1` column and a migration that sets `is_primary=0` on all but the latest row per `(game_id, card_type, recommended_bet_type)`. (3) `dr_claire_health_report.js` `dupRatio` drops to ≤1.1× for all sports. (4) `settle_game_results.js` settlement counts match Dr. Claire's deduplicated counts. (5) Tests pass with no regressions in model output jobs. |
| **Owner agent** | unassigned |

## Context

Dr. Claire deduplicates with `GROUP BY game_id, card_type, recommended_bet_type` but no constraint prevents the writes. As of 2026-04-06:

- NBA: 234 raw rows → 109 unique markets (2.1× dup)
- NHL: 266 raw rows → 151 unique markets (1.8× dup)
- NCAAM: 63 raw rows → 38 unique markets (1.7× dup)

This also means NBA's true 30-day hit rate is **47.7%** (Dr. Claire deduplicated) vs the **53.0%** raw count — a 5.3 point gap that matters for betting decisions.

## Investigation Starting Point

Check which job inserts into `card_results`. Look for `INSERT INTO card_results` or equivalent prepare statements across the worker jobs. The model jobs likely create one card_result row per card generated, and cards can be regenerated multiple times per game without deduplication.
