## Context

Dr. Claire deduplicates with `GROUP BY game_id, card_type, recommended_bet_type` but no constraint prevents the writes. As of 2026-04-06:

- NBA: 234 raw rows â†’ 109 unique markets (2.1x dup)
- NHL: 266 raw rows â†’ 151 unique markets (1.8x dup)
- NCAAM: 63 raw rows â†’ 38 unique markets (1.7x dup)

This means NBA's true 30-day hit rate is **47.7%** (Dr. Claire deduplicated) vs the **53.0%** raw count â€” a 5.3 point gap that matters for betting decisions.

**Confirmed write path:** `backfill_card_results.js` is the production inserter â€” it creates one `card_results` row for every `card_payloads` row via `insertCardResult`. Since model runner jobs (e.g. `run_nba_model.js`) can insert new driver `card_payloads` rows for the same game+market on each 30-min run, `backfill_card_results.js` then creates a new `card_results` row for each. Migration 062 added a partial UNIQUE INDEX only for `%-call` card_payloads; driver cards were explicitly excluded. Verify with: `SELECT card_type, COUNT(*) FROM card_payloads GROUP BY game_id, card_type HAVING COUNT(*) > 1` â€” driver card types will show the duplication.

**Prior migration context:** Migration 007 already sets `card_id TEXT NOT NULL UNIQUE` on `card_results`, so each individual `card_payloads` row can have at most one `card_results` row. The duplication is at the `card_payloads` level for driver cards, not within `card_results` itself. Migration 062 deduped `%-call` card_payloads and enforced uniqueness; migration 068 addresses the analytics-layer consequence for driver cards by marking only the primary row visible to reporting queries.

## Implementation Notes

Migration 068 template:

```sql
-- Migration 068: Add is_primary flag to card_results
-- Marks only the most-recently-created row per (game_id, card_type, recommended_bet_type)
-- as is_primary=1. All downstream reporting queries must filter on is_primary=1.

ALTER TABLE card_results ADD COLUMN is_primary INTEGER NOT NULL DEFAULT 1;

-- Backfill: set is_primary=0 on all non-latest rows per market
UPDATE card_results
SET is_primary = 0
WHERE id NOT IN (
  SELECT id FROM card_results cr2
  WHERE cr2.game_id = card_results.game_id
    AND cr2.card_type = card_results.card_type
    AND cr2.recommended_bet_type = card_results.recommended_bet_type
  ORDER BY cr2.created_at DESC
  LIMIT 1
);
```
