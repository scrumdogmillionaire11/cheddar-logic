# WI-1034-a

## Goal
Formalize the verification contract in the vegas-agent doc layer. Define `GATE_CHECK` and `STANDARD_AUDIT` as named audit levels with explicit boundaries — `STANDARD_AUDIT` is the existing 8-step `## Audit Sequence` in `bet_review.md`, renamed and anchored. Define every blocker code, action type, severity, promotion rule, and expiry rule. Update the `<verdict_contract>` to clarify that `LEAN` (not `HOLD`) is the user-facing verdict for pending verification states.

## Depends on
None. Doc-layer only — no worker or pipeline code changes.
