# WI-1100: POTD Near-Miss Tracking — Automate Shadow Settlement & Connect W/L Record

**Goal:** Wire near-miss POTD candidates into the settlement pipeline so they are automatically graded into a connected W/L record, completing the virtual 1u tracking feature already scaffolded in the codebase.

**Scope:**
- `apps/worker/src/schedulers/main.js` — add scheduler rule for shadow settlement job
- `apps/worker/src/schedulers/settlement.js` — integrate shadow settlement into settlement chain
- `apps/worker/src/jobs/potd/settle-shadow-candidates.js` — no code changes (already correct)
- `web/src/lib/potd-server.ts` — no changes needed (already queries shadow results correctly)

**Out of Scope:**
- Changing candidate selection logic (WI-1028 handles noise floors)
- UI redesign or new components
- Historical backfill of pre-WI-1037 near-miss data

## Current Behavior

**Near-miss tracking infrastructure exists but is disconnected:**
1. `potd_shadow_candidates` table: 328+ records tracked (near-miss nominees that weren't the official POTD)
2. `potd_shadow_results` table: 0 settled (settlement job never runs)
3. UI dashboard shows: **328 tracked | 0 settled | N/A win rate**
   - Expected: W/L record once games settle (e.g., "328 tracked | 42 settled | 61.9% win rate")

**Why it's broken:**
- Settlement job `job:potd-shadow-settlement` exists in package.json but **is not scheduled**
- Manual trigger works: `npm --prefix apps/worker run job:potd-shadow-settlement`
- Automatic trigger missing: no scheduler rule fires this job

## Solution

**Integrate shadow settlement into scheduler's settlement chain:**

1. **Add scheduler window rule** in `schedulers/windows.js` and `schedulers/settlement.js`:
   - Key: `keyShadowSettlement()`
   - Timing: After main card settlement jobs complete (e.g., `job:settle-cards`)
   - Frequency: Once daily (nightly sweep is ideal to batch all settled games per day)

2. **Wire into main scheduler** (`schedulers/main.js`):
   - Check if shadow settlement is due (window predicate)
   - Invoke settlement job
   - Log outcome to heartbeat

3. **Verify data flow**:
   - Shadow candidates are already inserted during POTD engine run ✓
   - Game results are populated via settlement job ✓
   - Settlement grader exists and joins correctly ✓
   - Web API query handles LEFT JOIN with proper null handling ✓

## Implementation Details

### `schedulers/windows.js` additions

```javascript
function keyShadowSettlement() {
  return 'settlement:shadow_potd_candidates';
}

// Add alongside other settlement window predicates
function isShadowSettlementDue(now = new Date()) {
  // Run once per day at ~03:00 ET (after nightly settlement jobs complete)
  // Batch all games settled in the last 24h into one settlement job
  return isNightlySettlementWindow(now) && shouldRunJobKey(keyShadowSettlement());
}

module.exports = {
  // ... existing exports
  keyShadowSettlement,
  isShadowSettlementDue,
};
```

### `schedulers/settlement.js` additions

Integrate into the settlement chain export so main.js can invoke it:

```javascript
async function runShadowSettlement() {
  const { settleShadowCandidates } = require('../jobs/potd/settle-shadow-candidates');
  const result = await settleShadowCandidates({
    jobKey: 'settlement:shadow_potd_candidates',
    dryRun: false,
  });
  console.log('[settlement-chain] Shadow POTD settlement:', result);
  return result;
}

module.exports = {
  // ... existing exports
  runShadowSettlement,
};
```

### `schedulers/main.js` additions

In the nightly settlement block (after card settlement, before final sweep):

```javascript
if (isShadowSettlementDue(nowUtc)) {
  console.log('[scheduler] Running shadow POTD settlement...');
  const shadowResult = await runShadowSettlement();
  if (!shadowResult.success) {
    console.error('[scheduler] Shadow settlement failed:', shadowResult);
    // Non-fatal; don't block other jobs
  }
}
```
