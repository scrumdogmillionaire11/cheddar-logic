**Canonical Moneyline Suppression Taxonomy** (Pre-Defined):

All suppressions MUST use one reason from this table. No aliases, no ad-hoc codes.

| Layer | Suppression Reason | Canonical ID | Meaning | Response |
|-------|-------------------|--------------|---------|----------|
| **MODEL** | inferred_no_edge | MODEL_NO_EDGE | Model confidence threshold failed | Do not retry |
| MODEL | inferred_low_confidence | MODEL_LOW_CONFIDENCE | Model edge < min threshold | Do not retry |
| MODEL | inferred_integrity_fail | MODEL_INTEGRITY_FAIL | Data validation failed (missing pitcher, weather, etc.) | Do not retry |
| MODEL | inferred_no_recent_data | MODEL_NO_RECENT_DATA | Insufficient recent game history | Do not retry |
| **GATE** | stale_snapshot_expired | GATE_STALE_EXPIRED | Snapshot age > hardMax (WI-0950) | Retry (WI-0952) |
| GATE | stale_snapshot_valid | GATE_STALE_VALID | Snapshot age within grace window (WI-0950) | No retry (too recent, allowed by contract) |
| GATE | integrity_fail | GATE_INTEGRITY_FAIL | Snapshot missing required fields | Do not retry |
| GATE | no_applicable_contract | GATE_NO_CONTRACT | Sport not in WI-0950 contract | Do not retry |
| **TRANSFORM** | status_remapped_legacy | TRANSFORM_STATUS_REMAP | Legacy code remapped status (WI-0954 cleanup) | Emit audit log |
| TRANSFORM | insufficient_data_fallback | TRANSFORM_INSUFFICIENT_DATA | Fell back from EXECUTABLE to PROJECTION_ONLY | Emit audit log |
| **FILTER** | user_filtered_out | FILTER_USER_CHOICE | User disabled market type | Expected |
| FILTER | default_hidden_market | FILTER_DEFAULT_HIDDEN | Market hidden by default settings | Expected |

**market-eval.js Scope Clarification**:

market-eval.js ranks cards but does NOT emit suppression reasons. It receives cards with already-set status and reason from upstream (model, gate).

- **Does NOT do:** Assign suppression reason from scratch
- **Does do:** Pass through card metadata (reason, layer, etc.) as-is OR log it for ranking context
- **If market-eval.js filters/reorders:** Emit `TRANSFORM_STATUS_REMAP` only if the card's final published status changed from what gate returned

**Reconciliation Scope + Methodology**:

Reconciliation validates: "For a given time window, do card counts reconcile across layer boundaries without double-counting?"

Window: **24 hours, rolling, ending at NOW()**
- Why 24h? Captures one full scheduler cycle with multiple model runs per sport
- Why rolling? Demonstrates continuous correctness, not just snapshots
- Why not 72h? (original default) 72h can obscure recent regressions if card retention < 72h

Reconciliation query (pseudocode):

```
SELECT
  layer,
  suppression_reason,
  COUNT(*) as count
FROM moneyline_cards
WHERE created_at >= NOW() - 24h
  AND sport IN ('mlb', 'nhl')
  AND market = 'moneyline'
GROUP BY layer, suppression_reason
ORDER BY layer, suppression_reason

VALIDATE:
  - Sum of MODEL layer = total cards evaluated (baseline)
  - Sum of GATE layer suppressions ≤ MODEL baseline (gate is downstream)
  - Sum of TRANSFORM layer suppressions ≤ previous layer
  - Sum of FILTER layer suppressions ≤ previous layer
  - No reason code appears that is NOT in canonical taxonomy
  - No reason code is empty/null
```

Example valid output:

```
MODEL,        MODEL_NO_EDGE,            1200
MODEL,        MODEL_LOW_CONFIDENCE,      850
MODEL,        MODEL_INTEGRITY_FAIL,      120
GATE,         GATE_STALE_EXPIRED,        200
GATE,         GATE_INTEGRITY_FAIL,        50
TRANSFORM,    TRANSFORM_STATUS_REMAP,     10
FILTER,       FILTER_USER_CHOICE,         80
---
Total: P1 = 2170, P2 ≤ P1, P3 ≤ P2, P4 ≤ P3 (NO DOUBLE COUNT)
```

**Manual Validation Methodology**:

**Window justification:** 24-hour rolling window (not 72h) ensures detection of regressions within current scheduler cycle. If card retention is < 72h, using 72h window can hide suppressions that aged out.

**Steps:**

1. Run reconciliation query above over last 24h
2. Verify each layer's total ≤ upstream layer (no double-count)
3. Verify zero legacy reason codes in output (all codes in canonical taxonomy)
4. Spot-check 20 random cards with each suppression reason; manually verify reason is accurate
5. Inject test suppression into DB; run queries; confirm detection

**Success criteria:**

- Reconciliation query produces output with zero double-counts
- Zero legacy/unmapped codes appear
- 100% of sampled cards have accurate reason for their suppression
