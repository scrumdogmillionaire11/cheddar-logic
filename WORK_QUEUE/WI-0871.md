# WI-0871: Fix F5 ML Formula

**ID**: WI-0871
**Goal**: Replace ERA-arithmetic in projectF5ML() with per-team run projections from projectTeamF5RunsAgainstStarter(), aligning F5 ML edge with F5 total edge.

**Scope**:
- apps/worker/src/models/mlb-model.js (projectF5ML only)
- apps/worker/src/models/__tests__/mlb-model.test.js (4 new tests)

**Out of scope**: full-game ML (WI-0873), full-game total (WI-0872), DB/job changes

**Background**:
projectF5ML() computes expected F5 runs as (pitcher.era + 4.5)/2*(5/9) -- ignoring park, weather, offense quality, leash, and TTO. F5 total and F5 ML are therefore driven by incompatible math. data/mlb-research.md recommends a unified run distribution path for both markets.

**Acceptance**:
1. projectF5ML() reads f5_runs from projectTeamF5RunsAgainstStarter() for both sides
2. winProbHome logistic coefficient (0.8) preserved; only the run mean inputs change
3. When projectTeamF5RunsAgainstStarter() returns null f5_runs, fall back to ERA formula and emit reason code F5_ML_FALLBACK_ERA
4. Test: same pitcher inputs produce identical f5_runs in F5 total and F5 ML paths
5. Test: elite-offense team vs poor pitcher produces higher home f5_runs than avg-offense vs avg pitcher
6. npm --prefix apps/worker test --no-coverage passes, zero regressions
