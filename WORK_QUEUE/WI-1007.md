# WI-1007: Settle MLB F5 ML picks from inning-5 score (win/loss/push)

**ID**: WI-1007
**Goal**: Grade pending `mlb-f5-ml` projection-only picks as win/loss/push from first-5 score outcomes without requiring odds or line values.

**Depends on**: WI-1006
**Release classification**: optional phase-2 hardening after projection generation is live

**Scope**:

- `apps/worker/src/jobs/settle_mlb_f5.js`
- `apps/worker/src/jobs/__tests__/settle_mlb_f5.test.js`

**Out of scope**:

- Changing generic `settle_pending_cards` ownership for F5 rows

**Acceptance**:

- `settle_mlb_f5` recognizes F5 ML cards and grades from first-5 home/away runs instead of total-vs-line logic.
- F5 ML grading rules are deterministic: selected side leading after 5 innings = win, trailing = loss, tied after 5 = push.
- Settlement does not require `line` or odds fields for F5 ML cards.
- Existing F5 totals settlement behavior remains unchanged, with explicit test coverage proving no regression in totals grading.
- Integration tests cover HOME win, AWAY win, and tie-push grading paths.
