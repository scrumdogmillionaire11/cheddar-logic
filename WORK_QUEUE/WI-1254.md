# WI-1254: Add Automated Trap-Scan Diagnostics for Projection-Only Pitcher-K

**ID**: WI-1254
**Goal**: Add zero-manual trap diagnostics using currently available automated inputs and make missing inputs explicit.

**Context**:

Trap logic exists but can silently under-fire when fields are absent. The projection-only contract still needs robust automated trap visibility.

**Scope**:
- `apps/worker/src/models/mlb-model.js`
- `apps/worker/src/jobs/run_mlb_model.js`
- `apps/worker/src/jobs/__tests__/run_mlb_model.test.js`
- `apps/worker/src/models/__tests__/mlb-model.test.js`
- `docs/pitcher_ks/06trap.md`
- `docs/pitcher_ks/07output.md`
- `WORK_QUEUE/WI-1254.md`

**Out of scope**:
- Live market move ingestion
- Manual data-entry workflows
- Odds-backed execution decisions

**Acceptance**:
- Every pitcher-K card emits:
  - `trap_inputs_present` (array<string>, unique, sorted, deterministic order)
  - `trap_inputs_missing` (array<string>, unique, sorted, deterministic order)
  - `trap_flags` (array<string>, unique, sorted, deterministic order)
  - `confidence_cap_reason` (string|null; MUST be `null` in WI-1254 when no cap has been applied; WI-1255 populates cap-rule values)
- Trap diagnostics include automated checks for:
  - opponent K bucket (`opp_k_bucket` in `{LOW_K, MID_K, HIGH_K, UNKNOWN}`)
  - pitcher leash bucket (`leash_bucket` in `{SHORT, STANDARD, LONG, UNKNOWN}`)
  - pitcher popularity/name-risk proxy (`name_risk_proxy` in `{CLEAR, AMBIGUOUS, UNKNOWN}`)
  - projection vs tiered static K-line band (no live feed required):
    - LOW tier: 2.5-4.5 Ks (short leash / weak K arms)
    - MID tier: 4.5-6.5 Ks (standard starters)
    - HIGH tier: 6.5-8.5 Ks (elite strikeout arms)
  - recent opponent K volatility (`opp_k_volatility` in `{LOW, MID, HIGH, UNKNOWN}`)
  - stale/static opponent profile detection (`opp_profile_staleness` in `{FRESH, STALE, STATIC_FALLBACK, UNKNOWN}`)
- Unavailable ump/public/market-move inputs are explicitly marked unavailable using `UNAVAILABLE_*` trap flags (not omitted):
  - `UNAVAILABLE_UMP`
  - `UNAVAILABLE_PUBLIC`
  - `UNAVAILABLE_MARKET_MOVE`
- `trap_inputs_missing` MUST include unavailable optional feeds as explicit keys (`ump_context`, `public_betting`, `market_move`) when absent.
- Tests verify:
  - payload shape for all required trap diagnostic fields
  - deterministic complete vs missing behavior
  - unavailable optional feeds generate explicit `UNAVAILABLE_*` flags
  - `confidence_cap_reason === null` for WI-1254 baseline behavior
