# WI-0919: Remove Web Legacy Decision and Market Fallback Paths

**ID**: WI-0919
**Goal**: Simplify web card filtering/decision/rendering to canonical fields only (`market_type`, `selection`, `decision_v2`/canonical action) and remove legacy fallback branches that keep stale semantics alive.

**Depends on**: WI-0918

**Scope**:
- `web/src/lib/game-card/filters.ts`
- `web/src/lib/game-card/decision.ts`
- `web/src/lib/game-card/transform/market-normalize.ts`
- `web/src/lib/game-card/canonical-decision.ts`
- `web/src/lib/types/game-card.ts`
- `web/src/lib/types/canonical-play.ts`
- `web/src/__tests__/game-card-transform-market-contract.test.js`
- `web/src/__tests__/filters-pass-play-main-view-regression.test.js`
- `web/src/lib/game-card/transform/market-normalize.test.ts`

**Out of scope**:
- Card-type contract changes already covered by WI-0918
- Worker payload-writing changes
- Deprecating DB aliases and env path resolution

**Acceptance**:
- Market filtering relies on canonical `market_type` mapping only; legacy `play.market` and driver-based fallback market checks are removed or reduced to non-authoritative diagnostics.
- Display action derivation no longer depends on legacy `status`-only fallback when canonical decision fields exist.
- Legacy compatibility types in `game-card.ts` are narrowed to only fields still required for historical row rendering, with explicit migration comments.
- Existing PASS/FIRE/WATCH behavior remains stable for canonical rows.
- Regression tests cover canonical-only behavior and pass.
