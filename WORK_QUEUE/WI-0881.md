# WI-0881: Wire NHL goalie composite into POTD signal-engine moneyline scoring

**ID**: WI-0881
**Goal**: Add a goalie-composite model signal to POTD NHL moneyline scoring,
analogous to the MLB pitcher-quality signal. NHL h2h markets are too tight for
cross-book consensus vig-removal alone to reliably produce edgePct > 0. Adding a
goalie quality differential bias (via resolveGoalieComposite from WI-0823) gives
NHL candidates genuine model-backed edge scores.

**Scope**:
- apps/worker/src/jobs/potd/signal-engine.js
- apps/worker/src/jobs/potd/run_potd_engine.js
- packages/data/src/db/cards.js   (add getLatestNhlModelOutput helper)
- packages/data/index.js          (re-export getLatestNhlModelOutput)
- apps/worker/src/jobs/potd/__tests__/signal-engine.test.js

**Out of scope**:
- Adding h2h to NHL odds config (WI-0880)
- Changing the NHL main model pipeline (run_nhl_model.js, cross-market.js)
- Totals or spread candidate scoring for NHL
- New DB tables or migrations

**Background**:
resolveGoalieComposite(savePct, gsax) is exported from
apps/worker/src/models/nhl-pace-model.js (WI-0823, shipped). card_payloads
raw_data JSON for NHL cards stores goalie data as:
  goalie_home_gsax, goalie_home_save_pct (primary keys)
  goalie.home.gsax, goalie.home.save_pct (legacy fallback)
  goalie_away_gsax, goalie_away_save_pct (primary keys)
  goalie.away.gsax, goalie.away.save_pct (legacy fallback)
Note: DB field is save_pct; resolveGoalieComposite param is savePct -- map on read.

**DB shape for getLatestNhlModelOutput(gameId)**:
  SQL: SELECT raw_data FROM card_payloads
       WHERE game_id = ? AND sport = "icehockey_nhl"
       ORDER BY created_at DESC LIMIT 1
  Parse: const rd = JSON.parse(row.raw_data)
  Return: {
    homeGoalie: {
      savePct: rd.goalie_home_save_pct ?? rd.goalie?.home?.save_pct ?? null,
      gsax:    rd.goalie_home_gsax    ?? rd.goalie?.home?.gsax    ?? null,
    },
    awayGoalie: {
      savePct: rd.goalie_away_save_pct ?? rd.goalie?.away?.save_pct ?? null,
      gsax:    rd.goalie_away_gsax    ?? rd.goalie?.away?.gsax    ?? null,
    },
  }
  Returns null if no row found.

**Implementation notes**:
1. Add getLatestNhlModelOutput(gameId) to packages/data/src/db/cards.js.
   Uses getDatabase() (single-writer contract). Re-export from packages/data/index.js.

2. In run_potd_engine.js gatherBestCandidate, attach nhlSnapshot to NHL games:
   const candidateGame =
     sport === "MLB" && game?.gameId ? { ...game, oddsSnapshot: getLatestOdds(game.gameId) || null }
     : sport === "NHL" && game?.gameId ? { ...game, nhlSnapshot: getLatestNhlModelOutput(game.gameId) || null }
     : game;

3. In signal-engine.js:
   a. Add isNhlSport(sport) = sport === "NHL" || sport === "icehockey_nhl"
   b. Add resolveNHLModelSignal(game) that:
      - Returns null if !game.nhlSnapshot or both goalies missing data
      - Calls resolveGoalieComposite for each side
      - goalieEdgeDelta = clamp(homeComposite.composite - awayComposite.composite, -0.06, 0.06)
      - modelWinProb = clamp(consensusImpliedHome + goalieEdgeDelta, 0.05, 0.95)
        where consensusImpliedHome is pre-computed from the game h2h candidates
      - Returns { modelWinProb, edge: modelWinProb - impliedProb, projection_source }
      - projection_source = "NHL_GOALIE_COMPOSITE" when both goalies have data,
        "NHL_GOALIE_PARTIAL" when one side only
   c. In buildCandidates: after the isMlbSport block, add analogous isNhlSport block
      that attaches nhlSignal to candidates when resolveNHLModelSignal returns non-null
   d. In scoreCandidate: add isNhlSport override block parallel to useMlbModelSignal block

**Acceptance**:
1. resolveNHLModelSignal is exported from signal-engine.js
2. getLatestNhlModelOutput is exported from packages/data/index.js
3. projection_source is "NHL_GOALIE_COMPOSITE" when both goalies have data
4. projection_source is "NHL_GOALIE_PARTIAL" when only one side has data
5. resolveNHLModelSignal returns null (no crash) when nhlSnapshot is null or both goalies missing
6. Unit test: elite home goalie (savePct=0.928, gsax=8.5) vs weak away goalie (savePct=0.901, gsax=-2.1)
   produces modelWinProb for the home side > vig-removed consensus implied alone
7. Unit test: null nhlSnapshot input -> resolveNHLModelSignal returns null
8. scoreCandidate uses nhlSignal.edge as edgePct for NHL moneyline candidates when signal is non-null
9. All existing signal-engine and run-potd-engine tests still pass
10. MLB and NBA edgePct are unchanged -- nhlSignal block is isNhlSport-gated only
