# WI-0881: Wire NHL goalie composite into POTD signal-engine moneyline scoring

**ID**: WI-0881
**Goal**: Add a goalie-composite model signal to POTD NHL moneyline scoring,
analogous to the MLB pitcher-quality signal. NHL h2h markets are too tight for
cross-book consensus vig-removal alone to reliably produce edgePct > 0. Adding a
goalie quality differential bias (via resolveGoalieComposite from WI-0823) gives
NHL candidates genuine model-backed edge scores.

**Scope**:
- apps/worker/src/jobs/potd/signal-engine.js
- apps/worker/src/jobs/potd/run_potd_engine.js
- packages/data/src/db/cards.js   (add getLatestNhlModelOutput helper)
- packages/data/index.js          (re-export getLatestNhlModelOutput)
- apps/worker/src/jobs/potd/__tests__/signal-engine.test.js

**Out of scope**:
- Adding h2h to NHL odds config (WI-0880)
- Changing the NHL main model pipeline (run_nhl_model.js, cross-market.js)
- Totals or spread candidate scoring for NHL
- New DB tables or migrations

**Background**:
resolveGoalieComposite(savePct, gsax) is exported from
apps/worker/src/models/nhl-pace-model.js (WI-0823, shipped). card_payloads
raw_data JSON for NHL cards stores goalie data as:
  goalie_home_gsax, goalie_home_save_pct (primary keys)
  goalie.home.gsax, goalie.home.save_pct (legacy fallback)
  goalie_away_gsax, goalie_away_save_pct (primary keys)
  goalie.away.gsax, goalie.away.save_pct (legacy fallback)
Note: DB field is save_pct; resolveGoalieComposite param is savePct -- map on read.

**DB shape for getLatestNhlModelOutput(gameId)**:
  SQL: SELECT raw_data FROM card_payloads
       WHERE game_id = ? AND sport = "icehockey_nhl"
       ORDER BY created_at DESC LIMIT 1
  Parse: const rd = JSON.parse(row.raw_data)
  Return: {
    homeGoalie: {
      savePct: rd.goalie_home_save_pct ?? rd.goalie?.home?.save_pct ?? null,
      gsax:    rd.goalie_home_gsax    ?? rd.goalie?.home?.gsax    ?? null,
    },
    awayGoalie: {
      savePct: rd.goalie_away_save_pct ?? rd.goalie?.away?.save_pct ?? null,
      gsax:    rd.goalie_away_gsax    ?? rd.goalie?.away?.gsax    ?? null,
    },
  }
  Returns null if no row found.

**Implementation notes**:
1. Add getLatestNhlModelOutput(gameId) to packages/data/src/db/cards.js.
   Uses getDatabase() (single-writer contract). Re-export from packages/data/index.js.

2. In run_potd_engine.js:
   a. Add getLatestNhlModelOutput to the existing require('@cheddar-logic/data')
      destructure at the top of the file (same block as getLatestOdds, lines 7-17).
      Without this, gatherBestCandidate gets ReferenceError at runtime.
   b. In gatherBestCandidate, attach nhlSnapshot to NHL games:
      const candidateGame =
        sport === "MLB" && game?.gameId ? { ...game, oddsSnapshot: getLatestOdds(game.gameId) || null }
        : sport === "NHL" && game?.gameId ? { ...game, nhlSnapshot: getLatestNhlModelOutput(game.gameId) || null }
        : game;

3. In signal-engine.js:
   a. At the top of the file, add import alongside the existing mlb-model require:
        const { resolveGoalieComposite } = require('../../models/nhl-pace-model');
      Without this, resolveNHLModelSignal gets ReferenceError at runtime.

   b. Add isNhlSport(sport) = sport === "NHL" || sport === "icehockey_nhl"

   c. Add resolveNHLModelSignal(game) that:
      - Returns null if !game.nhlSnapshot or both goalies missing data
      - Calls resolveGoalieComposite for home and away goalies:
          const homeComposite = resolveGoalieComposite(
            game.nhlSnapshot.homeGoalie.savePct,
            game.nhlSnapshot.homeGoalie.gsax
          );
          const awayComposite = resolveGoalieComposite(
            game.nhlSnapshot.awayGoalie.savePct,
            game.nhlSnapshot.awayGoalie.gsax
          );
      - goalieEdgeDelta = clamp(homeComposite.composite - awayComposite.composite, -0.06, 0.06)
      - consensusImpliedHome = computeConsensus(game.market?.h2h || [], 'MONEYLINE').homeImplied
        (use the private computeConsensus already in signal-engine.js; do NOT re-derive from
        American odds — the medianImplied path avoids mixed-sign discontinuity)
      - homeModelWinProb = clamp(consensusImpliedHome + goalieEdgeDelta, 0.05, 0.95)
      - Returns { homeModelWinProb, projection_source }
        (no edge field -- edge is selection-specific, derived in scoreCandidate, see note 3e)
      - projection_source = "NHL_GOALIE_COMPOSITE" when both goalies have data,
        "NHL_GOALIE_PARTIAL" when one side only
        (detect partial: either goalie's savePct and gsax both null => that side is neutral)

   d. In buildCandidates: after the isMlbSport block, add analogous isNhlSport block
      that attaches nhlSignal to all candidates for that game when resolveNHLModelSignal
      returns non-null. The same signal object is shared across HOME and AWAY candidates
      for the same game -- scoreCandidate inverts homeModelWinProb for AWAY (see 3e).

   e. In scoreCandidate: add isNhlSport override block parallel to useMlbModelSignal block.
      Edge is selection-specific -- do NOT use nhlSignal.edge (there is none). Instead:

        const nhlSignal = candidate.nhlSignal ?? null;
        const useNhlModelSignal =
          isNhlSport(candidate.sport) &&
          candidate.marketType === 'MONEYLINE' &&
          Number.isFinite(nhlSignal?.homeModelWinProb);

        if (useNhlModelSignal) {
          const modelWinProb =
            candidate.selection === 'HOME'
              ? nhlSignal.homeModelWinProb
              : round(1 - nhlSignal.homeModelWinProb, 6);
          const modelEdge = round(modelWinProb - impliedProb, 6);
          // ... (same return shape as useMlbModelSignal block, substituting nhlSignal.projection_source)
        }

      This mirrors exactly how resolveMLBModelSignal handles HOME vs AWAY in mlb-model.js:
      side === 'HOME' → win_prob_home; side === 'AWAY' → 1 - win_prob_home.
