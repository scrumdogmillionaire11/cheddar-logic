# WI-0881: Wire NHL goalie composite into POTD signal-engine moneyline scoring

**ID**: WI-0881
**Goal**: Add a goalie-composite model signal to POTD NHL moneyline scoring, analogous to the MLB pitcher-quality signal path. Currently NHL moneyline candidates (unblocked by WI-0880) rely on cross-book consensus vig-removal to generate edge, which rarely produces edgePct > 0 on tight NHL h2h markets. Adding a goalie quality differential bias (via the existing resolveGoalieComposite from WI-0823) gives NHL candidates genuine model-backed edge scores.
**Scope**:
- apps/worker/src/jobs/potd/signal-engine.js
- apps/worker/src/jobs/potd/run_potd_engine.js
- apps/worker/src/jobs/potd/__tests__/signal-engine.test.js
- apps/worker/src/jobs/potd/__tests__/run-potd-engine.test.js

**Out of scope**:
- Adding h2h to NHL odds config (WI-0880)
- Changing the NHL main model pipeline (run_nhl_model.js, cross-market.js)
- Totals or spread candidate scoring for NHL
- Adding new DB tables or migrations

**Background**:
resolveGoalieComposite(savePct, gsax) already exists in apps/worker/src/models/nhl-pace-model.js (WI-0823, shipped). It returns { factor, composite, source } where composite is a bounded adjustment multiplier. The difference (homeComposite - awayComposite) represents a goalie quality differential that can shift the vig-removed fair probability for each team.

**Implementation notes**:
1. In run_potd_engine.js gatherBestCandidate, attach nhlSnapshot to NHL games the same way oddsSnapshot is attached for MLB. The nhlSnapshot should come from getLatestOdds(game.gameId) (same call) -- investigate what fields are available. If goalie data is absent from the odds snapshot, query card_payloads for the latest NHL model output by game_id to get homeGoalie/awayGoalie (siera/savePct-equivalent fields).
2. In signal-engine.js, add isNhlSport(sport) helper and resolveNHLModelSignal(game) function. This function should: extract home/away goalie savePct+gsax from game.nhlSnapshot; call resolveGoalieComposite for each side; compute goalieEdgeDelta = (homeComposite.composite - awayComposite.composite) bounded to [-0.06, +0.06]; apply delta to the vig-removed consensusImplied to derive modelWinProb; return { modelWinProb, edge: modelWinProb - impliedProb, projection_source }.
3. In buildCandidates, attach nhlSignal to NHL moneyline candidates if resolveNHLModelSignal returns a valid signal (same pattern as mlbSignal attachment).
4. In scoreCandidate, add an isNhlSport override block that uses nhlSignal when present, analogous to the useMlbModelSignal block.

**Acceptance**:
- resolveNHLModelSignal is exported and returns { modelWinProb, edge, projection_source } when goalie data is present
- projection_source is NHL_GOALIE_COMPOSITE when both goalies have data; NHL_NEUTRAL when neither has data (in which case signal should return null and consensus scoring proceeds)
- scoreCandidate uses nhlSignal.edge as edgePct for NHL moneyline candidates when signal is non-null
- A unit test: elite home goalie (savePct=0.928, gsax=8.5) vs below-avg away goalie (savePct=0.901, gsax=-2.1) produces a higher modelWinProb for the home side than the vig-removed consensus alone
- All existing signal-engine and run-potd-engine tests still pass
