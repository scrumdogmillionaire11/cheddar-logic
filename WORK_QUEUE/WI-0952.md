# WI-0952: Stale-Only Moneyline Auto-Recovery and Single-Retry Flow

**ID**: WI-0952
**Goal**: Convert stale-only execution-gate failures into deterministic targeted refresh + single retry so recoverable moneyline plays are not permanently dropped.

**Depends on**: WI-0950, WI-0951

**Acceptance**:

- When execution gate fails only on stale snapshot, runner triggers one targeted odds refresh attempt and one deterministic retry.
- Retry path is idempotent and guarded to prevent infinite loops or duplicate publishes.
- Non-stale blockers (`NO_EDGE`, `CONFIDENCE`, integrity gates) do not trigger retry flow.
- Payload metadata records stale recovery attempt and final outcome for audit.
- Tests cover stale-only success recovery, stale-only failed recovery, and mixed-blockers no-retry behavior.

**Stale-Only Detection Logic (Decision Tree)**:

Runner MUST retry if and only if **ALL** of the following are true:

```
IF execution_gate_result.blocked = true
   AND execution_gate_result.freshness_decision.blocked_by_freshness = true
   AND execution_gate_result.freshness_decision.tier in ['STALE_VALID', 'EXPIRED']
   AND no other blocker flag is true (NO_EDGE, CONFIDENCE, INTEGRITY_FAIL, etc.)
   AND stale_recovery_attempt_count < 1  // Guard: max once per model run
THEN
   trigger_targeted_refresh()
   retry_model_execution()
ELSE
   do_not_retry()
```

Key semantics:
- `blocked_by_freshness=true` means freshness was the PRIMARY blocker
- If ANY other blocker co-exists (e.g., NO_EDGE=true), stale recovery does NOT trigger (this is "mixed blockers")
- `attempt_count` is stored in runner state to prevent infinite loops across slots
- Tier distinction: `STALE_VALID` always retries; `EXPIRED` retries IFF `allowStaleIfNoNewOdds=true` per WI-0950 contract (rare, anti-silencing case)

**Targeted Refresh Strategy**:

```
refresh_scope: SINGLE_SPORT (MLB or NHL, not mixed)
refresh_target: ALL_GAMES in active/future state for the sport
refresh_method: Enqueue pull_odds:{sport}:emergency job with priority=HIGH
refresh_timeout: 120 seconds (abort if not complete by then)
refresh_success_check: After refresh, re-fetch latest snapshot and confirm age < 30 minutes (FRESH threshold from WI-0950)
retry_triggers_on: age < 30min (confirmed FRESH) OR age < 60min + allowStaleIfNoNewOdds=true
```

If refresh times out or snapshot remains stale after refresh → log diagnostic, DO NOT retry (treat as failed recovery).

**Idempotency Guard: Dedup Key Schema**:

```
stale_recovery_key = "{sport}:{game_id}:{slot_start_iso}:{model_run_uuid}"
```

- `sport`: "mlb" or "nhl"
- `game_id`: External game ID from fixture (e.g., ESPN ID)
- `slot_start_iso`: ISO 8601 UTC, normalized to minute precision (e.g., "2026-04-15T19:00:00Z")
- `model_run_uuid`: Unique execution ID of the model run (prevents same model run retrying twice)

Dedup logic:
- Before triggering refresh, check Redis: `EXISTS stale_recovery_key`
- If exists, skip refresh + retry (already attempted in this slot)
- If not exists, SET key with 10-minute expiry (covers retry window)
- Retry model execution uses SAME run UUID, so duplicate attempts within 10m are caught

**Payload Metadata Schema**:

All cards passing through stale recovery MUST include this metadata:

```json
{
  "stale_recovery": {
    "attempted": true,
    "triggered_at": "2026-04-15T19:15:30.123Z",
    "refresh_executed": true,
    "refresh_snapshot_age_before_ms": 85000,
    "refresh_snapshot_age_after_ms": 12000,
    "refresh_duration_ms": 45000,
    "retry_executed": true,
    "retry_gate_result": "PASS_EXECUTION_GATE",
    "final_status": "EXECUTABLE",
    "attempt_count": 1
  }
}
```

Fields:
- `attempted`: Boolean (true = stale recovery was triggered)
- `triggered_at`: ISO 8601 timestamp when recovery started
- `refresh_executed`: Boolean (whether odds refresh was attempted)
- `refresh_snapshot_age_before_ms`: Age in ms before refresh
- `refresh_snapshot_age_after_ms`: Age in ms after refresh
- `refresh_duration_ms`: How long the refresh job took
- `retry_executed`: Boolean (whether retry was executed after refresh)
- `retry_gate_result`: Pass string from gate result (e.g., "PASS_EXECUTION_GATE", "FAILED_NO_EDGE", "BLOCKED_AFTER_RETRY")
- `final_status`: Final card outcome ("EXECUTABLE", "BLOCKED", "PROJECTION_ONLY")
- `attempt_count`: Always 0 or 1 (no retries > 1)
