# WI-1244: Produce a Decision Blocker Audit Report

**ID**: WI-1244
**Goal**: Quantify why `LEAN` / canonical `SLIGHT_EDGE` rows are not promoting by reporting blocker and demoter distributions over recent settled windows.

**Context**:

The repo already has a narrow `LEAN -> PLAY` promotion path. Before any threshold or promotion changes are made, the team needs a measured view of which blockers and demoters dominate by sport and market. This WI is audit-only and must not change decision thresholds.

**Scope**:
- `packages/models/src/decision-pipeline-v2.js`
- `packages/models/src/decision-pipeline-v2-edge-config.js`
- `apps/worker/src/jobs/report_decision_blockers.js` (new)
- `apps/worker/src/__tests__/report_decision_blockers.test.js` (new)
- `WORK_QUEUE/WI-1244.md`

**Out of scope**:
- Any threshold tuning or promotion-scope expansion
- Any change to results, cards, or POTD read surfaces
- Any change to settlement logic

**Acceptance**:
- A report job can aggregate blocker and demoter reason-code frequencies over 30/60/90-day windows.
- Output is grouped by sport and market, with explicit counts for key blockers such as freshness, contradiction, adverse line movement, sigma fallback, and edge sanity rails.
- Output is exportable in a machine-readable format suitable for follow-on analysis.
- Regression coverage proves known blocked scenarios are classified into the expected blocker buckets.
