# WI-0978: Implement auto-rollback on post-deploy SLO breach

**ID**: WI-0978
**Goal**: Automatically rollback to previous known-good release when post-deploy health thresholds are breached.

**Depends on**: WI-0977

**Scope**:
- `.github/workflows/deploy-production.yml`
- `scripts/deploy-pi.sh`
- `scripts/pi-health-check.sh`
- `docs/runbooks/rollback-policy.md`

**Out of scope**:
- New observability platform adoption
- Long-term performance tuning

**Acceptance**:
- Deploy workflow defines rollback trigger thresholds (error rate/health checks) during soak window.
- On threshold breach, workflow initiates rollback to prior release artifact.
- Rollback event includes reason, failed SHA, restored SHA, and verification status.

**Owner agent**: unassigned
**Time window**: 2026-04-16 -> TBD
**Coordination flag**: needs-sync

**Tests to run**:
- `npm --prefix web run lint`
- `npm --prefix web run test:ci:smoke`

**Manual validation**:
- Simulate post-deploy health failure and confirm rollback workflow executes.
- Verify restored release SHA matches prior known-good deploy.
