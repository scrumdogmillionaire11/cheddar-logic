# WI-0873: Full-Game Moneyline Model -- projectFullGameML()

**ID**: WI-0873
**Goal**: Add projectFullGameML() to mlb-model.js computing full-game moneyline win probability from the full-game run differential via logistic function.

**Scope**:
- apps/worker/src/models/mlb-model.js (new function projectFullGameML)
- apps/worker/src/models/__tests__/mlb-model.test.js (4 new tests)
- apps/worker/src/jobs/run_mlb_model.js (wire as full_game_ml card in computeMLBDriverCards; add to module.exports)

**Out of scope**: Skellam/NB simulation (future), extra-innings tail, POTD wiring (WI-0874), bullpen REST data

**Background**:
No full-game ML model exists. data/mlb-research.md recommends deriving win probability from
P(R_H > R_A) using the joint run distribution. For a production v1, the logistic function applied
to the run differential is the correct approximation. The coefficient for 9 innings should be 0.5
(vs 0.8 for F5) because 9-inning differentials are larger in absolute magnitude.

**Implementation notes**:
projectFullGameML(homePitcher, awayPitcher, mlHome, mlAway, context):
  - Call projectFullGameTotal() to get homeProj/awayProj run means
  - runDiff = homeFullMean - awayFullMean
  - winProbHome = 1 / (1 + exp(-0.5 * runDiff))  [0.5 = full-game coeff; comment why]
  - De-vig mlHome/mlAway with two-sided normalization (same as projectF5ML)
  - homeEdge = winProbHome - fairHome
  - LEAN_EDGE_MIN = 0.04; CONFIDENCE_MIN = 6
  - Return null if projectFullGameTotal returns null/NO_BET
  - Emit FULL_GAME_ML_DEGRADED if projection_source is DEGRADED_MODEL

**Acceptance**:
1. projectFullGameML() uses projectFullGameTotal() run projections, not ERA formula
2. winProbHome coefficient is 0.5; comment in code explains difference from F5 (0.8)
3. Two-sided de-vig normalization applied before edge calculation
4. Test: team with lower projected run total has win prob < 0.5
5. Test: equal projections yield winProbHome between 0.49 and 0.51
6. computeMLBDriverCards includes full_game_ml card when h2h line is present
7. npm --prefix apps/worker test --no-coverage passes, zero regressions
