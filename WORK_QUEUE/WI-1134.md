# WI-1134: Decompose API Games Route Handler Into Query-Service-Transform Layers

**ID**: WI-1134
**Goal**: Reduce fragility and performance risk in api games by decomposing the large handler into bounded modules with explicit stage budgets.

**Requirements**:
- WI-1134-API-01: Route handler orchestration is decomposed into query, service, and transform layers with explicit module boundaries.
- WI-1134-PERF-01: Stage-level timing instrumentation exists for query, service, and transform phases with deterministic metric keys.
- WI-1134-REG-01: Existing `/api/games` smoke, repair-budget, and missing-data contract behavior remains stable.

**Execution plan**:
- `.planning/phases/WI-1134/WI-1134-01-PLAN.md`

**Scope**:
- web/src/lib/games/route-handler.ts
- web/src/lib/games/query-layer.ts
- web/src/lib/games/service-layer.ts
- web/src/lib/games/transform-layer.ts
- web/src/lib/games/perf-metrics.ts

**Out of scope**:
- Business logic policy changes for card decisioning
- Cards endpoint refactor

**Acceptance**:
- `route-handler.ts` delegates query, service, and transform responsibilities to extracted modules and no longer contains mixed stage logic in one monolithic block.
- `route-handler.ts` imports `query-layer.ts`, `service-layer.ts`, `transform-layer.ts`, and `perf-metrics.ts` and executes stages in query -> service -> transform order.
- Stage metrics emit all three keys for every request path: `games.query.ms`, `games.service.ms`, `games.transform.ms`.
- Forced timeout/fallback scenarios still return a valid `response_mode` and do not throw uncaught errors.
- All required regression commands pass from repo root:
  - `npm --prefix web run test:api:games:market`
  - `npm --prefix web run test:api:games:repair-budget`
  - `node web/src/__tests__/api-games-missing-data-contract.test.js`
