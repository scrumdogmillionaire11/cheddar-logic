# WI-1124: Fail-Closed Auth Policy For Protected Surfaces

**ID**: WI-1124
**Goal**: Enforce fail-closed authentication and entitlement checks for protected web pages and API routes.

**Scope**:
- web/src/lib/api-security/auth.ts
- web/src/lib/api-security/config.ts
- web/src/lib/games/route-handler.ts
- web/src/app/api/cards/route.ts
- web/src/app/api/cards/[gameId]/route.ts
- web/src/app/wedge/page.tsx
- web/src/app/fpl/page.tsx
- web/src/__tests__/api-auth-walls-default-deny.test.js
- web/src/__tests__/page-auth-wall-contract.test.js

**Out of scope**:
- Admin endpoint policy redesign
- Payment and subscription product logic changes
- `/api/results`, `/api/model-outputs`, `/api/team-metrics`, `/api/results/projection-settled`, `/api/props/player-shots` — these routes have no entitlement gate today and are intentionally public read surfaces; reclassifying them as protected is a separate work item

**Implementation notes**:

**auth.ts — close the `rbacEnforcement` env-var bypass:**
`requireAuth`, `requireRole`, and `requireEntitlementForRequest` all short-circuit to `ok: true` when `SECURITY_CONFIG.rbacEnforcement` is `false`. Remove this escape hatch from these three functions. If dev environments need auth bypassed, that bypass must be controlled by a production guard in `config.ts` (e.g., only allow `rbacEnforcement=false` when `NODE_ENV !== 'production'`, throwing at startup otherwise). Do not leave any codepath that returns `ok: true` without verifying a token when `rbacEnforcement` is the only gate.

**config.ts — add explicit public-route allowlist:**
Add a `PUBLIC_ROUTES` set (or equivalent `isPublicRoute(pathname)` helper) listing routes that are intentionally unauthenticated. This is the "explicit allow-public flag" required by acceptance. At minimum document the shape; downstream WIs in this series (`WI-1133`, `WI-1134`) depend on this contract existing.

**wedge/page.tsx and fpl/page.tsx — intended posture is PUBLIC:**
Both pages have auth commented out with `// AUTH DISABLED: Commenting out auth walls to allow public access`. The intended posture for both is **public** (no auth gate). The task here is not to re-enable auth; it is to replace the commented-out block with an explicit `// PUBLIC_SURFACE: no auth required — registered in PUBLIC_ROUTES` annotation and add both paths to the `PUBLIC_ROUTES` allowlist in `config.ts`. Do not restore the redirect-to-login logic on these pages.

**route-handler.ts — no entitlement call exists today:**
Add `requireEntitlementForRequest(request, RESOURCE.CHEDDAR_BOARD)` to the games route handler in the same pattern as `cards/route.ts` (line 320) and `cards/[gameId]/route.ts` (line 314). Return early with the error response if `!access.ok`.

**New test file spec — `api-auth-walls-default-deny.test.js`:**
Test against the live route handler with a real DB fixture or a mock that satisfies `ensureDbReady`. Required cases:
- `GET /api/cards` with no Authorization header → 401
- `GET /api/cards/[gameId]` with no Authorization header → 401
- `GET /api/games` (route-handler) with no Authorization header → 401
- `GET /api/cards` with a well-formed but invalid JWT → 401
- `GET /api/cards` with a valid token and `RESOURCE.CHEDDAR_BOARD` role → 200 (non-empty success or empty data, not error)
- When `ENABLE_RBAC` env var is missing entirely (not set to `false`), requests without token → 401 (confirms fail-closed is the default, not opt-in)

**New test file spec — `page-auth-wall-contract.test.js`:**
Test the page-level posture declarations. Required cases:
- `/wedge` is present in `PUBLIC_ROUTES` (or `isPublicRoute('/wedge')` returns `true`)
- `/fpl` is present in `PUBLIC_ROUTES` (or `isPublicRoute('/fpl')` returns `true`)
- A route not in `PUBLIC_ROUTES` (e.g., `/admin`) returns `false` from `isPublicRoute`
- The `PUBLIC_ROUTES` set contains no routes from the cards or games API families (guards against accidental over-allowlisting)

**Acceptance**:
- `requireEntitlementForRequest` returns `{ ok: false, status: 401 }` when called with no token, regardless of `ENABLE_RBAC` env var value (env var bypass removed in prod).
- `requireEntitlementForRequest` returns `{ ok: false, status: 401 }` when `ENABLE_RBAC` env var is absent.
- `/api/cards`, `/api/cards/[gameId]`, and `/api/games` all gate on `requireEntitlementForRequest` and return 401 to unauthenticated requests.
- `/wedge` and `/fpl` are registered in `PUBLIC_ROUTES` in `config.ts` with a comment explaining why; commented-out auth blocks replaced with explicit public-surface annotations.
- Any route not in `PUBLIC_ROUTES` and not explicitly calling an entitlement function is a scope gap — document in out-of-scope if intentional.
- Regression: `npm --prefix web run test:api:cards:security` and `npm --prefix web run test:cards-lifecycle-regression` continue to pass.
