# WI-0526: NHL Props Price Wiring + Blocked Shots Market Expansion

**ID:** WI-0526
**Goal:** Wire real SOG prop prices into NHL shots decisioning (Phase 1), then add blocked-shots ingestion (Phase 2).
**Owner agent:** copilot
**Time window:** 2026-03-20
**Coordination flag:** none
**Branch:** agent/copilot/WI-0526-nhl-props-price-wiring

---

## Context

`pull_nhl_player_shots_props.js` already fetches and stores `over_price`/`under_price` for `player_shots_on_goal` from DraftKings, FanDuel, and BetMGM. `getPlayerPropLine()` in the data package already SELECTs them. But `run_nhl_player_shots_model.js` only reads `.line` — prices are never read, never passed to `projectSogV2`, and therefore never used in decisioning.

`projectSogV2` (WI-0525) is fully implemented and tested (25 tests green) but is not imported in the model job at all. Its Stage 2 pricing layer — `edge_over_pp`, `ev_over`, `opportunity_score` — is dormant.

Current state:

| Market        | Pulled | Priced | Used by model |
| ------------- | ------:| ------:| -------------:|
| Shots on goal |    Yes |    Yes |        **No** |
| Blocked shots |     No |     No |            No |

## Phase 1: Wire SOG Prices

### Scope

- `apps/worker/src/jobs/run_nhl_player_shots_model.js`
- `apps/worker/src/models/__tests__/nhl-sog-price-wiring.test.js` (new)

### Out of scope

- `player_blocked_shots` market (Phase 2)
- Changes to `pull_nhl_player_shots_props.js`
- Changes to `nhl-player-shots.js` model functions
- Decisioning threshold changes (FIRE/HOLD/PASS gate values unchanged)

### Acceptance (Phase 1)

- [ ] `run_nhl_player_shots_model.js` imports and calls `projectSogV2`
- [ ] `over_price` and `under_price` are read from `realPropLine` (already returned by `getPlayerPropLine`)
- [ ] `projectSogV2` is called with those prices and the existing shot stats
- [ ] Card payload `decision` object includes `v2.edge_over_pp`, `v2.ev_over`, `v2.opportunity_score`, `v2.flags`
- [ ] `odds_backed: true` set when both prices present; `odds_backed: false` when prices missing
- [ ] `opportunity_score` surfaced at `decision.opportunity_score` for ranking
- [ ] `MISSING_PRICE` flag propagated into the card when line exists but prices absent
- [ ] Tests prove priced SOG props have non-null `opportunity_score`; projection-only props have `opportunity_score: null`

## Phase 2: Blocked Shots Market Expansion

### Scope

- `apps/worker/src/jobs/pull_nhl_player_shots_props.js` — extend to fetch `player_blocked_shots`

- `apps/worker/src/models/__tests__/nhl-blocked-shots-ingestion.test.js` (new)

### Out of scope

- Model scoring for blocked shots (separate future WI)
- Card generation for blocked shots (separate future WI)
- Any changes to SOG flow

### Acceptance (Phase 2)

- [ ] `pull_nhl_player_shots_props.js` fetches both `player_shots_on_goal` and `player_blocked_shots` per event
- [ ] Rows stored with `prop_type = 'blocked_shots'` in `player_prop_lines`
- [ ] SOG rows continue to use `prop_type = 'shots_on_goal'` — no collision
- [ ] No impact on existing SOG model run
- [ ] Tests prove both `prop_type` values can be stored and queried independently

  "SELECT player_id, json_extract(payload_data,'$.decision.opportunity_score') AS opp_score,
