# WI-1024: NBA Phase 3A — Residual learning layer for total projection

**ID**: WI-1024
**Goal**: Add a residual correction term that learns team-specific, pace-tier-specific, and total-band-specific biases from settled NBA totals history, with shrinkage and caps so it corrects persistent patterns without overfitting noise.

**Depends on**: WI-1020 (global rolling bias active), WI-1021 (dynamic baselines active), WI-1027 (accuracy schema includes `pace_tier`, `total_band`, `game_date`/`settled_at`)

**Scope**:

* `apps/worker/src/models/residual-projection.js` (extend for NBA)
* `apps/worker/src/jobs/run_nba_model.js`
* `apps/worker/src/models/__tests__/nba-projection-parity.test.js`

**Out of scope**:

* ML model training (no external pipelines)
* NHL/MLB residual models
* Regime detection (WI-1025)
* UI changes

## Acceptance

### 1) Residual lookup function

Add:

```js
computeNbaResidualCorrection({ db, homeTeam, awayTeam, paceTier, totalBand, month })
```

Returns:

```js
{
  correction,            // number (points)
  source,                // 'full' | 'team_pace_band' | 'team_band' | 'team' | 'global' | 'none'
  samples,               // integer
  segment,               // string descriptor of the segment used
  shrinkage_factor       // number in [0,1]
}
```

### 2) Segment hierarchy (first match wins)

Evaluate segments in this order; use the first with sufficient samples:

1. `team × paceTier × totalBand × month` → **min 15 samples**
2. `team × paceTier × totalBand` → **min 15 samples**
3. `team × totalBand` → **min 15 samples**
4. `team` → **min 10 samples**
5. Global → uses WI-1020 rolling bias (`source: 'global'`)

If no segment meets thresholds, return `{ correction: 0, source: 'none', samples: 0 }`.

> Team match is `(home_team = ? OR away_team = ?)`.

### 3) Residual definition

For a matched segment:

```sql
AVG(actual_total - raw_total) AS mean_residual
```

Where:

* `raw_total` = model total **before** WI-1020/WI-1024 corrections
* `actual_total` = final game total

### 4) Query shape (parameterized)

Base template:

```sql
SELECT AVG(actual_total - raw_total) AS mean_residual, COUNT(*) AS n
FROM projection_accuracy_line_evals
WHERE sport = 'nba'
  AND market_family = 'NBA_TOTAL'
  AND actual_total IS NOT NULL
  AND raw_total IS NOT NULL
  AND settled_at < datetime('now')
  AND (home_team = ? OR away_team = ?)
  /* + optional filters per segment:
     AND pace_tier = ?
     AND total_band = ?
     AND strftime('%m', COALESCE(game_date, settled_at)) = ?
  */
```

**Notes**:

* Do **not** `LIMIT` the aggregation. Use all qualifying rows.
* `game_date` falls back to `settled_at` if not present.

### 5) Shrinkage (mandatory)

Let:

* `mean_residual` from the segment
* `n` = sample count for the segment
* `globalBias` = WI-1020 rolling bias (points)

Compute:

```js
shrinkage = Math.min(1, n / 30)
correction = mean_residual * shrinkage + globalBias * (1 - shrinkage)
```

Behavior:

* `n < 30` → blended toward global
* `n ≥ 30` → full segment correction

### 6) Segment correction cap

Clamp:

```js
correction = clamp(correction, -5.0, 5.0)
```

If clamped, log:

```txt
[NBAModel] [RESIDUAL] segment correction clamped from X to ±5.0
```

### 7) Combined override ceiling (WI-1020 + WI-1024)

Let:

* `rollingBias` (WI-1020)
* `residualCorrection` (this WI)

If:

```js
Math.abs(rollingBias + residualCorrection) > 6.0
```

Then scale **only the residual term**:

```js
allowedResidual = 6.0 * Math.sign(rollingBias + residualCorrection) - rollingBias
residualCorrection = allowedResidual
```

This preserves WI-1020 while bounding total adjustment.

### 8) Application point (projection path)

* Apply **after** WI-1020 rolling bias
* Apply **once** to the projection source of truth (same rule as WI-1022)

```js
adjustedTotal = baseTotal + rollingBias + residualCorrection
```

* Downstream fields must inherit this value (no re-subtraction)

### 9) Raw data stamping

On each card:

```js
raw_data.residual_correction = {
  correction,            // final applied residual term
  source,                // segment used
  samples,               // n for chosen segment
  segment,               // human-readable descriptor
  shrinkage_factor       // 0–1
}
```

### 10) Logging

Log selection path:

```txt
[NBAModel] [RESIDUAL] source=team_pace_band samples=22 shrinkage=0.73 correction=+2.1
```

Fallback:

```txt
[NBAModel] [RESIDUAL] source=global (fallback) samples=0
```

## Tests

### In `nba-projection-parity.test.js`

Cover:

1. **Full segment (n ≥ 30)**

   * `shrinkage = 1.0`
   * correction = `mean_residual`

2. **Partial shrinkage**

   * `n = 10` → `shrinkage ≈ 0.33`
   * `n = 20` → `shrinkage ≈ 0.67`

3. **Hierarchy selection**

   * highest-specificity segment chosen first
   * falls back correctly when insufficient samples

4. **Global fallback**

   * no qualifying segments → returns WI-1020 bias

5. **Segment cap**

   * raw correction > 5 → clamped to ±5

6. **Combined cap (±6)**

   * rollingBias + residual exceeds ±6 → residual scaled down

7. **No data**

   * returns `{ correction: 0, source: 'none' }`

### In `run-nba-model.test.js`

* Residual correction is applied **once** to projection total
* Downstream projection fields match adjusted total
* `raw_data.residual_correction` exists and is populated correctly

---

## Why shrinkage is mandatory

Without shrinkage:

* small samples produce large corrections
* model overfits transient streaks

With shrinkage:

* `n = 3` → mostly global bias
* `n = 15` → 50/50 blend
* `n = 30` → full trust

This enforces stability and prevents noise amplification.
