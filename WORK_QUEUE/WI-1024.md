# WI-1024: NBA Phase 3A — Residual learning layer for total projection

**ID**: WI-1024
**Goal**: Add a residual correction term that learns team-specific, pace-tier-specific, and total-band-specific biases from settled game history — with shrinkage and caps so it corrects real patterns instead of memorizing seasonal noise.

**Depends on**: WI-1020 (global calibration active), WI-1021 (dynamic baselines active), WI-1027 (schema has pace_tier, total_band, driver_contributions)

**Scope**:
- `apps/worker/src/models/residual-projection.js` (extend for NBA)

**Out of scope**:

**Acceptance**:
- `computeNbaResidualCorrection({ db, homeTeam, awayTeam, paceTier, totalBand, month })` queries `projection_accuracy_line_evals` for matching features and returns `{ correction, source, samples, segment }`.
- Feature lookup dimensions (in order of specificity):
  1. Full segment: `team × paceTier × totalBand × month` (minimum 15 samples)
  2. Partial: `team × paceTier × totalBand` (minimum 15 samples)
  3. Partial: `team × totalBand` (minimum 15 samples)
  4. Global team: `team` (minimum 10 samples)
  5. Global: all NBA totals — uses WI-1020 rolling bias, returns `source: 'global'`
- Correction is computed as `mean_residual = AVG(actual_total - raw_total)` for the matched segment.
- **Shrinkage toward global mean is required**: `correction = mean_residual * shrinkage + globalBias * (1 - shrinkage)` where `shrinkage = Math.min(1, n / 30)`. At 30+ samples, full segment correction. Below 30, blended toward global.
- **Segment correction cap**: `abs(correction) ≤ 5.0 points`. If exceeded, clamp and log: `[RESIDUAL] segment correction clamped from N to ±5`.
- **Base model override ceiling**: the total adjustment from WI-1020 + WI-1024 combined cannot exceed ±6.0 points. If `abs(rollingBias + residualCorrection) > 6`, reduce the residual term proportionally.
- `raw_data.residual_correction` stamped: `{ correction, source, samples, segment, shrinkage_factor }`.
- Unit tests: known residual table returns expected correction with correct shrinkage; empty table returns 0; shrinkage formula verified at n=10, n=20, n=30; cap enforced at 6.0.

**Segment query (parameterized)**:

Team bucket lookup is by team appearance in either position:

```sql
SELECT AVG(actual_total - raw_total) AS mean_residual, COUNT(*) AS n
FROM projection_accuracy_line_evals
WHERE sport = 'nba'
  AND market_family = 'NBA_TOTAL'
  AND actual_total IS NOT NULL
  AND raw_total IS NOT NULL
  AND settled_at < datetime('now')
  AND (home_team = ? OR away_team = ?)
  AND pace_tier = ?
  AND total_band = ?
  AND strftime('%m', game_date) = ?
ORDER BY settled_at DESC
LIMIT 30
```

`pace_tier` and `total_band` from the WI-1027 schema columns. `game_date` derived from `settled_at` or a `game_date` column if added.

**Why shrinkage is mandatory**:

Without shrinkage, a team that went 3/3 over in a specific total band and pace tier will get a +6 correction applied immediately. That is noise. With shrinkage at n=3: `correction = mean_residual * 0.10 + globalBias * 0.90`. At n=15: `correction = mean_residual * 0.50 + globalBias * 0.50`. At n=30: full segment correction. This is the correct behavior.
