## Context

The current snapshot card (built in WI-0761) shows latest-per-phase status with age. Two gaps identified in WI-0792:

1. A card showing `failed` gives no indication of how long it has been failing. `failed` from 30 seconds ago (first occurrence) looks identical to `failed` from 45 minutes ago (9 consecutive checks). The streak count is the key triage signal.

2. Some checks skip writing a health row when no games are in window (e.g., `mlb:f5_market_availability` writes nothing when no MLB games are within T-6h). The snapshot retains the last written result indefinitely. A 23h-old `ok` card looks live. It should be visually distinguished.

## Implementation Spec

### 1. Consecutive failure streak

**Where:** `buildSnapshot()` in [web/src/app/admin/page.tsx](web/src/app/admin/page.tsx)

**How:** After building the snapshot (latest row per phase/check), compute the streak by scanning the full `health` history array for each snapshot key:

```ts
function computeStreak(rows: PipelineHealthRow[], phase: string, checkName: string): number {
  const key = `${phase}:${checkName}`;
  let streak = 0;
  for (const row of rows) {
    if (`${row.phase}:${row.check_name}` !== key) continue;
    if (streak === 0) {
      streak = 1; // first (latest) row sets the baseline status
    } else if (row.status === rows.find(r => `${r.phase}:${r.check_name}` === key)!.status) {
      streak++;
    } else {
      break;
    }
  }
  return streak;
}
```

Actually simpler: rows are ordered newest-first; find all rows for this key, walk until status changes.

**Display:** On the snapshot card, when `streak >= 2`, show a small badge below the status badge:
- `failed × N` in red/muted red
- `warning × N` in yellow/muted yellow
- Don't show for `ok` streaks (noise)

### 2. Stale-card muting

**Threshold:** `created_at` age > 35 minutes (1 check cycle = 5min, buffer = 30min extra)

**Display:** When a card is stale:
- Reduce card opacity (`opacity-50` or similar)
- Replace the age display with a pill: `check dormant`
- Suppress the status dot animation (if any)
- Still show the last known status as dim text for reference

**Rationale for 35min:** The health check runs every 5min. A card older than 35min means at least 6 consecutive cycles produced no row for this check — it's definitely dormant, not just slow.
