# WI-0623: Decompose cards-page-client.tsx into sub-components

**ID**: WI-0623
**Goal**: Break `web/src/components/cards-page-client.tsx` (3,662 lines) into focused sub-components with shared context/reducer so the page is testable and doesn't re-render on every state mutation.

**Background**:
`cards-page-client.tsx` manages filters, display state, sport toggling, modal logic, and settlement rendering in a single React component. Re-renders on every state mutation, unmaintainable, impossible to unit test subsections.

**Target structure**:
```
web/src/components/cards/
  CardsPageContext.tsx     — shared context + useReducer for filter/sport state
  CardsFilter.tsx          — filter bar and sport toggle UI
  CardsList.tsx            — virtualized card list rendering
  CardSettlementModal.tsx  — settlement detail modal
  CardMispriceTag.tsx      — misprice warning badge (soft line, dispersion)
  index.tsx                — CardsPageClient shell (thin orchestrator)
```

**Out of scope**:
- Changes to card data contracts or API
- New filter/display features

**Acceptance**:
- `CardsPageClient` in `index.tsx` is under 200 lines.
- Each sub-component can be rendered in isolation in tests.
- `npx tsc --noEmit --project web/tsconfig.json` passes.
- `npm --prefix web run build` passes.
- Existing filter and settlement behavior is unchanged.
