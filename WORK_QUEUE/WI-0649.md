# WI-0649: FPL GW run-in fixture difficulty integration (GW31–38)

**ID**: WI-0649
**Goal**: Integrate a fixture difficulty rating (FDR) lookup into the FPL Sage transfer and captaincy recommendations for the GW31–38 run-in, so that recommendations surface "easy run" and "hard run" context alongside standard projections.

**Background**:
FPL season ends GW38 (~May 2026). The chip engine thresholds were recalibrated (BB/TC, 2026-03-29). The next missing layer is fixture difficulty context in the run-in window. Currently, recommendations output raw projection values without flagging that, e.g., a player faces 5 of their last 8 games against top-4 defenses. This is the most impactful missing signal for late-season FPL decisions.

**Scope**:
- `cheddar-fpl-sage/src/fixture_difficulty.py` (new module — reads from FPL API `fixtures` endpoint)
- `cheddar-fpl-sage/src/recommendations.py` (extend captaincy + transfer logic with FDR scores)
- `cheddar-fpl-sage/backend/routers/advisor.py` (expose FDR context in response)
- `cheddar-fpl-sage/tests/test_fixture_difficulty.py` (new test file)

**Out of scope**:
- UI changes beyond surfacing existing FDR fields already returned by the backend
- Historical FDR accuracy analysis
- Double gameweek detection (separate concern)

**Acceptance**:
- `fixture_difficulty.py` fetches FPL `fixtures` endpoint and returns per-player average FDR for a configurable GW window (default: next 6 GWs)
- FDR scores are included in advisor response as `run_in_fdr: { easy_gws: int, hard_gws: int, avg_fdr: float }`
- Captaincy recommendation reason explicitly mentions `EASY_RUN` or `HARD_RUN` when avg_fdr < 2.5 or > 3.5
- `pytest cheddar-fpl-sage/tests/test_fixture_difficulty.py` passes
- `pytest cheddar-fpl-sage/tests/` exits 0
