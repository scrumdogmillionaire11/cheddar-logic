## Two-Stage Decomposition Spec

### Stage A — `computeNhl1pForecast` (new, in `nhl-totals-status.js`)

Computes the numeric forecast envelope from model and market totals. Pure function, no side effects.

```js
function computeNhl1pForecast({ modelTotal, marketTotal, hasRequiredInputs = true }) {
  const safeModelTotal = Number(modelTotal);
  const safeMarketTotal = Number(marketTotal);
  const delta =
    Number.isFinite(safeModelTotal) && Number.isFinite(safeMarketTotal)
      ? safeModelTotal - safeMarketTotal
      : NaN;
  const absDelta = Number.isFinite(delta) ? Math.abs(delta) : NaN;
  return {
    modelTotal: safeModelTotal,
    marketTotal: safeMarketTotal,
    delta,
    absDelta,
    forecastValid: Boolean(hasRequiredInputs) && Number.isFinite(delta),
  };
}
```

Export via `module.exports`:
```js
module.exports = { classifyNhlTotalsStatus, computeNhl1pForecast };
```

### Stage B — `classifyNhlTotalsStatus` (refactored, in `nhl-totals-status.js`)

Extend the existing signature with two new optional fields:

| New field | Type | Default | Description |
|---|---|---|---|
| `forecast` | `object \| null` | `null` | Stage A output. If provided, `delta`/`absDelta` are read from it; internal recomputation is skipped. If absent, function computes internally (backward compat). |
| `thresholds` | `object \| null` | `null` | Per-side threshold config. If absent, defaults apply. |

**Threshold shape and defaults:**
```js
const DEFAULT_THRESHOLDS = {
  over:  { play: 1.0, slightEdge: 0.5 },
  under: { play: 1.0, slightEdge: 0.5 },
};
```

**Internal logic changes:**

1. Before existing guards, derive active thresholds:
   ```js
   const sideKey = (side || '').toLowerCase(); // 'over' | 'under'
   const sideThresholds =
     (thresholds?.[sideKey] ?? DEFAULT_THRESHOLDS[sideKey] ?? DEFAULT_THRESHOLDS.over);
   const playThreshold = sideThresholds.play;
   const slightEdgeThreshold = sideThresholds.slightEdge;
   ```

2. Replace the three hardcoded numeric comparisons:
   - `absDelta >= 1.0` → `absDelta >= playThreshold`
   - `absDelta >= 0.5` → `absDelta >= slightEdgeThreshold`
   - `absDelta < 0.5` in `FLOOR_GUARD_FORCE_PASS_DELTA_LT_0_5` → `absDelta < slightEdgeThreshold`

3. The `ANTI_FLATTENING_RESTORE_PLAY` guard uses `playThreshold`:
   - `absDelta >= 1.0 && status === 'SLIGHT EDGE' && !explicitlyCapped` → `absDelta >= playThreshold && ...`

4. Reason codes that embed numeric literals (`BASE_PLAY_DELTA_GTE_1_0`, `BASE_SLIGHT_EDGE_DELTA_GTE_0_5`) must not be changed — they are historical audit values and must remain stable strings.

5. If `forecast` is provided in input, read `delta` and `absDelta` from it instead of recomputing:
   ```js
   const { delta, absDelta } = forecast ?? computeNhl1pForecast({ modelTotal, marketTotal, hasRequiredInputs });
   ```
   The `forecastValid` field from Stage A maps to the existing `!hasRequiredInputs || !Number.isFinite(delta)` guard — use `forecast ? !forecast.forecastValid : (!hasRequiredInputs || !Number.isFinite(delta))` for the early PASS exit.

## Required New Tests in `nhl-totals-status.test.js`

Add a new `describe('computeNhl1pForecast', ...)` block and a new `describe('classifyNhlTotalsStatus — independent thresholds', ...)` block. Do not modify existing tests.

**`computeNhl1pForecast` contract tests (add 3):**
1. `computeNhl1pForecast({ modelTotal: 6.5, marketTotal: 5.5 })` → `{ delta: 1.0, absDelta: 1.0, forecastValid: true }`

   - Assert: `result.status === 'PLAY'`

   - Assert: `result.status === 'SLIGHT EDGE'`
