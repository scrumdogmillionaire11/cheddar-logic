# WI-1237: Unify `/api/cards` and `/api/cards/[gameId]` Inventory Logic

**ID**: WI-1237
**Goal**: Align the per-game cards endpoint with the main cards endpoint so valid cards are not suppressed and multi-card inventories survive dedupe.

**Context**:

`/api/cards/[gameId]` still applies broader settlement suppression than `/api/cards`, and both endpoints default to dedupe by `game_id + card_type`, which is unsafe for prop-heavy or multi-market games. This WI unifies filtering and moves the dedupe identity closer to the actual market/card identity.

**Scope**:
- `web/src/app/api/cards/route.ts`
- `web/src/app/api/cards/[gameId]/route.ts`
- `web/src/lib/cards/query.ts`
- `web/src/__tests__/api-cards-per-game-inventory.test.js` (new)
- `WORK_QUEUE/WI-1237.md`

**Out of scope**:
- Any change to `/api/games` dedupe or degraded-state handling
- Any change to settlement write behavior
- Any change to results-route inclusion rules

**Acceptance**:
- `/api/cards/[gameId]` uses the same settled/run-scope filtering contract as `/api/cards`.
- One settled card in a game no longer suppresses unrelated unsettled cards for the same game.
- Default dedupe no longer collapses distinct same-type cards solely because they share `game_id + card_type`.
- Multi-market and prop-heavy games return their full intended inventory.
- Regression coverage proves multiple cards of the same `card_type` can coexist when their market identity differs.
