# WI-0708: API Contract Expansion for Posture-Aware Outputs

## Context

Depends on WI-0707 (canonical RiskPostureConfig must be in place).

Currently the `/api/analyze/interactive` response sends back `risk_posture` as a
passthrough string and a textual `risk_note`. Nothing in the response tells the
frontend:
- what posture was actually applied
- whether the captain was a template play or a differential
- what the hit ROI breakdown is
- why the recommendation differs from the safe route
- whether outputs are consistent with the stated posture

The result is a response that *labels* posture without *explaining* it.

## Goal

Add six posture-aware fields as an additive contract layer on top of the existing
response dict. No existing fields renamed or removed.

## New Response Fields

### Root level additions

```json
{
  "posture_used": "AGGRESSIVE",
  "posture_signal": "CHASE"
}
```

`posture_signal` derives from `get_posture_config(resolved_risk_posture).posture_signal_default`.

### Captain additions

```json
{
  "captain": "Salah",
  "captain_style": "DIFFERENTIAL"
}
```

Derivation rule (applied in result_transformer or contract_transformer):
- ownership_pct < 20 → "DIFFERENTIAL"
- ownership_pct >= 50 → "TEMPLATE"
- else → "BALANCED"

### Hit recommendation block

```json
{
  "hit_recommendation": {
    "suggested_hit": -4,
    "projected_net_gain": 2.3,
    "delta_next2_pts": 6.9,
    "delta_next6_pts": 10.8,
    "break_even_threshold": 1.5,
    "reason": "Short-term upside clears aggressive hit threshold."
  }
}
```

Use `posture_hit_allowed()` from risk_aware_filter.py (WI-0707) to populate.
If no hit is being taken, `suggested_hit = 0` and `reason = "No hit recommended."`

### Posture explanation block

```json
{
  "posture_explanation": {
    "summary": "This recommendation leans into upside and accepts modest volatility.",
    "key_tradeoff": "You gain captain ceiling and fixture upside but lose some floor.",
    "why_not_safer": "The safer route does not gain enough ground for aggressive posture.",
    "why_not_more_aggressive": null
  }
}
```

Generation rules (template strings, not LLM):
- `summary`:
  - AGGRESSIVE: "This recommendation leans into upside and accepts higher variance."
  - BALANCED: "This recommendation balances expected return against risk."
  - CONSERVATIVE: "This recommendation protects rank and minimises downside."
- `key_tradeoff`:
  - AGGRESSIVE: "You gain ceiling and differential leverage but accept volatility."
  - BALANCED: "Moderate structure improvement with controlled risk."
  - CONSERVATIVE: "You preserve floor and template alignment at the cost of upside."
- `why_not_safer`: non-null for AGGRESSIVE — "Safer option does not meet aggressive gain threshold." | null for CONSERVATIVE/BALANCED
- `why_not_more_aggressive`: non-null for CONSERVATIVE — "More aggressive option carries more volatility than conservative posture allows." | null for AGGRESSIVE/BALANCED

### Consistency check

```python
def build_posture_consistency_issues(
    posture_name: str,
    captain_style: str,
    hit_suggested: int,
    projected_net_gain: float,
) -> list[str]:
    issues = []
    if posture_name == "AGGRESSIVE" and captain_style == "TEMPLATE":
        issues.append(
            "Captain is template pick — consider a differential for aggressive posture."
        )
    if posture_name == "CONSERVATIVE" and hit_suggested < 0 and projected_net_gain < 6.0:
        issues.append(
            "Hit gain below conservative threshold — conservative posture would not normally approve this hit."
        )
    return issues
```

Include as `posture_consistency_issues: []` in response root.

## Where to Add in result_transformer.py

After `resolved_risk_posture` is resolved (line ~1080) and before the final `result`
dict is returned, inject:

```python
from backend.config.risk_posture import get_posture_config
posture_cfg = get_posture_config(resolved_risk_posture)

# posture_signal
result["posture_used"] = resolved_risk_posture
result["posture_signal"] = posture_cfg.posture_signal_default

# captain_style — derive from captain ownership if available
captain_ownership = result.get("captain_ownership_pct") or 25.0
if captain_ownership < 20:
    captain_style = "DIFFERENTIAL"
elif captain_ownership >= 50:
    captain_style = "TEMPLATE"
else:
    captain_style = "BALANCED"
result["captain_style"] = captain_style

# hit_recommendation — build from available transfer deltas
result["hit_recommendation"] = _build_hit_recommendation(result, posture_cfg)

# posture_explanation
result["posture_explanation"] = _build_posture_explanation(resolved_risk_posture)

# consistency check
result["posture_consistency_issues"] = build_posture_consistency_issues(
    resolved_risk_posture,
    captain_style,
    result.get("hit_recommendation", {}).get("suggested_hit", 0),
    result.get("hit_recommendation", {}).get("projected_net_gain", 0.0),
)
```
