## Context

`NBA_IMPACT_PLAYERS` at [apps/worker/src/jobs/run_nba_model.js](apps/worker/src/jobs/run_nba_model.js#L90) is a 41-name hardcoded set. Problems:
- **Traded players:** Dejounte Murray (traded mid-season) still triggers cap on his old team when he never suits up there.
- **Missing breakouts:** Any player who becomes All-Star caliber after the list was drafted is missed.
- **Wrong team association:** The set has players but no team binding — a player injury on a team that has a road game today still won't trigger the away-team cap.

ESPN injury data is already fetched as part of `getTeamMetricsWithGames()`. The `games` field includes recent game participants. The injury designation (`OUT`, `DTD`, `Doubtful`) is published by ESPN before game time and is the canonical data source already in the pipeline.

## Implementation Notes

1. In `fetchPlayerInjuries(league, teamId)` call (or parse existing ESPN team data), extract players designated `OUT`/`DOUBTFUL`/`DTD` for today's game.
2. Filter to starters or top-3 scorers by averaging last-5-game points from the `games` array.
3. Return a `impactPlayerOutList: string[]` array from `getTeamMetricsWithGames()`.
4. Replace `NBA_IMPACT_PLAYERS.has()` check with `impactPlayerOutList.includes()`.
5. Keep the `NBA_TEAM_ABBR_MAP` — still needed for ESPN abbreviation mapping.
