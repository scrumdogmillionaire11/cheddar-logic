# WI-1158: POTD Pipeline And Near-Miss Health Visibility In Dr. Claire + Admin

**ID**: WI-1158
**Goal**: Extend health monitoring to include POTD pipeline status and near-miss shadow performance, surfaced in both Dr. Claire outputs and `/admin`.

- Execution is wave-ordered so worker report semantics, admin API contract wiring, and admin UI visibility can be validated independently.

**Out of scope**:

- Changing `/wedge` market decision thresholds or execution logic

**Acceptance**:

- [WI-1158-POTD-01] Dr. Claire report includes a POTD health section with explicit status and actionable signals.
- [WI-1158-POTD-02] POTD health captures at minimum: last successful run recency, today fired/no-pick state, candidate volume, and near-miss shadow settlement freshness.
- [WI-1158-POTD-03] `/api/admin/model-health` includes POTD health payload without breaking existing sport snapshot shape for current consumers.
- [WI-1158-POTD-04] `/api/admin/pipeline-health` includes POTD lane checks (or equivalent surfaced POTD pipeline checks) so failures are attributable by phase.
- [WI-1158-POTD-05] `/admin` renders a dedicated POTD health card/panel that includes near-miss status counts and recency indicators.
- [WI-1158-LEGACY-06] Dead or legacy POTD/near-miss health branches superseded by the canonical pipeline-health path are removed (or explicitly documented as intentionally retained with reason).
- Admin/read-path handling is resilient to partial historical POTD data (missing rows should degrade to `no-data`/`stale`, not crash).

## Must-haves (goal-backward)

**truths**:
- Operators can see POTD lane health in Dr. Claire output with explicit status and reasons, not generic freshness text.
- POTD health semantics are consistent across worker report output, `/api/admin/model-health`, `/api/admin/pipeline-health`, and `/admin` rendering.
- Missing or sparse historical POTD/near-miss data degrades safely to deterministic `no-data`/`stale` semantics.
- Existing admin consumers remain compatible because new fields are additive and legacy snapshot shape is preserved.
- Superseded legacy POTD/near-miss health branches in scoped files are removed or intentionally retained with explicit rationale.

**artifacts**:
- `apps/worker/src/jobs/dr_claire_health_report.js` and `apps/worker/src/jobs/__tests__/dr_claire_health_report.test.js` implement and verify POTD and near-miss health status semantics.
- `web/src/app/api/admin/model-health/route.ts` and `web/src/__tests__/api-admin-model-health.test.js` expose additive POTD health payload while preserving existing shape.
- `web/src/app/api/admin/pipeline-health/route.ts` and `web/src/__tests__/api-admin-pipeline-health.test.js` expose stage-attributable POTD pipeline health checks.
- `web/src/app/admin/page.tsx` renders a dedicated POTD health panel using the canonical API fields and fallback semantics.

**key_links**:
- `dr_claire_health_report.js` POTD health status derivation -> `/api/admin/model-health` payload mapping -> `/admin` POTD panel rendering.
- `/api/admin/pipeline-health` POTD lane stage checks -> `/admin` degraded/stale reason visibility.
- Near-miss settlement freshness/counters in worker payload -> admin API fields -> UI recency/count indicators.
- Additive API contract tests -> admin route handlers -> no breakage for existing sport snapshot consumers.
- Legacy branch retirement assertions -> scoped files -> canonical POTD health path is the only active branch.
