# WI-1213: F5 Proxy Settlement Policy + Gray Zone Contract

**ID**: WI-1213

**Goal**:
Define and lock the MLB F5 proxy-settlement policy so clear zones are officially graded while gray-zone projections are tracked for calibration only.

**Scope**:

- `docs/decisions/ADR-0007-mlb-f5-full-model-projection-contract.md`
- `apps/worker/src/audit/projection_evaluator.js`
- `apps/worker/src/audit/__tests__/projection_evaluator.test.js`

**Out of scope**:

- DB migration work
- Web/API reporting changes
- Settlement runner insertion flow changes in `settle_projections.js`

**Acceptance**:

- F5 projection buckets are explicitly defined with stable boundaries:
  - `<= 3.24` (`STRONG_UNDER_3_5`)
  - `3.25-3.49` (`LEAN_UNDER_3_5`)
  - `3.50-3.74` (`GRAY_LOW_TRACK_UNDER_4_5`)
  - `3.75-4.24` (`GRAY_DEAD_ZONE_NO_CALL`)
  - `4.25-4.50` (`GRAY_HIGH_TRACK_OVER_3_5`)
  - `4.51-4.75` (`LEAN_OVER_4_5`)
  - `>= 4.76` (`STRONG_OVER_4_5`)
- Clear zones map to official proxy grading only (`U3.5` and `O4.5`).
- Gray-zone buckets are marked `NO_CALL`/`TRACK_ONLY` and never treated as official side W/L.
- No fallback side derivation from legacy/action/classification fields is introduced.
- Boundary tests cover every threshold edge and pass.
