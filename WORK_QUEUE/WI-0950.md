# WI-0950: Execution Gate Freshness Contract Realignment for Moneyline

**ID**: WI-0950
**Goal**: Replace hardcoded 5-minute absolute freshness threshold with cadence-aligned contract. A snapshot is valid if it's within the known odds-pull interval + grace window at time of decision, not if arbitrary time has passed. Unblock valid edges currently suppressed by stale-time logic that ignores scheduler reality.

**Acceptance**:

1. **Contract Type**: `ExecutionFreshnessContract` exported from `execution-gate-freshness-contract.js` with fields:
   - `cadenceMinutes: number` — expected odds pull interval (e.g., 60 for hourly)
   - `graceMultiplier: number` — tolerance buffer as multiplier (e.g., 1.25 for +25%)
   - `hardMaxMinutes: number` — absolute fail-safe cap before forced block
   - `allowStaleIfNoNewOdds: boolean` — anti-silencing flag: do not block if no newer odds are available

2. **Sport-Specific Defaults** (documented in contract file + inline):
   - **MLB**: `{ cadenceMinutes: 60, graceMultiplier: 1.25, hardMaxMinutes: 120, allowStaleIfNoNewOdds: true }`
   - **NHL**: `{ cadenceMinutes: 60, graceMultiplier: 1.25, hardMaxMinutes: 120, allowStaleIfNoNewOdds: true }`
   - **NBA**: `{ cadenceMinutes: 60, graceMultiplier: 1.25, hardMaxMinutes: 120, allowStaleIfNoNewOdds: true }`
   - Rationale: Each sport uses hourly pulls; 60m cadence * 1.25 grace = 75m window. Hard max 120m prevents truly stale (multi-hour) snapshots from silently passing.

3. **Three-Tier Freshness Logic** in `evaluateExecution()`:
   - 🟢 **FRESH**: `snapshotAgeMs <= (cadenceMinutes * 60_000)`
   - 🟡 **STALE_VALID**: `(cadenceMinutes * 60_000) < snapshotAgeMs <= (cadenceMinutes * graceMultiplier * 60_000)` — DO NOT BLOCK if `allowStaleIfNoNewOdds=true`
   - 🔴 **EXPIRED**: `snapshotAgeMs > (hardMaxMinutes * 60_000)` — ALWAYS BLOCK
   - Update blocked-by reason to distinguish: `STALE_SNAPSHOT:VALID_WITHIN_CADENCE` vs `STALE_SNAPSHOT:EXPIRED_HARDMAX`

4. **Configurable at Runtime**:
   - `EXECUTION_FRESHNESS_CONTRACT` env var (optional JSON override): allows ops to adjust sport-specific thresholds without code change
   - Default: use sport-specific values from contract file
   - Fallback to 5-minute legacy default if not set (for debug/manual runs)

5. **Structured Logging** (non-breaking addition to evaluateExecution return):
   - Emit `freshness_decision` object in all `evaluateExecution()` results with fields:
     ```json
     {
       "snapshot_age_ms": 45000,
       "sport": "mlb",
       "cadence_ms": 3600000,
       "threshold_ms": 4500000,
       "tier": "STALE_VALID",
       "blocked_by_freshness": false,
       "reason": "within cadence grace window"
     }
     ```
   - Workers and runners to emit `[EXECUTION_GATE_FRESHNESS]` log line pre-execution for observability

6. **Contract Exported for Dependent WIs**: `getContractForSport(sport: string) => ExecutionFreshnessContract` callable from `execution-gate.js`, so WI-0951/0952/0953 can import and reason about thresholds

7. **Tech Debt Consolidation**:
   - Retire `MODEL_FRESHNESS_MAX_AGE_MINUTES` from `check_pipeline_health.js`; replace with import from freshness contract (cross-reference as "health check window is 4x moneyline hard-max per operational needs")
   - No legacy magic numbers (5-minute, 240-minute) left in execution path; all constants in contract file with comments explaining origin
   - Deprecation: if any legacy branch retained, annotate with `TODO WI-####: remove after contract stabilization`

8. **Tests (Boundary + Three-Tier coverage)**:
   - `FRESH` tier: 30 seconds, 30 minutes (all pass, no freshness block)
   - `STALE_VALID` tier: 
     - Exactly at `cadence` (60 min): PASS (FRESH upper bound)
     - Just over `cadence` (60:01): PASS (STALE_VALID lower bound)
     - At grace boundary (75 min): PASS (grace upper bound)
     - Just over grace (75:01): FAIL with `STALE_SNAPSHOT:EXPIRED_HARDMAX` if >120m, else PASS
   - `EXPIRED` tier: 130 minutes (always FAIL with `STALE_SNAPSHOT:EXPIRED_HARDMAX`)
   - Anti-silencing: snapshot at 90 min with `allowStaleIfNoNewOdds=true` → PASS despite being stale
