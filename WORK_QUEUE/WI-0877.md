# WI-0877: F5 Total Synthetic-Line Edge Path (without-odds mode)

**ID**: WI-0877
**Goal**: Replace the SYNTHETIC_FALLBACK/PASS-only F5 total driver in `withoutOddsMode` with a real edge calculation that uses `projectTeamF5RunsAgainstStarter()` and a synthetic market line, emitting a live `FULL_MODEL` card when the projection clears the run-edge threshold.

**Scope**:
- `apps/worker/src/jobs/run_mlb_model.js` — new `computeSyntheticLineF5Driver()` helper; replace `projectionFloorDriver` block with new driver that tries full-model edge first and falls back to SYNTHETIC_FALLBACK only on degrade
- `apps/worker/src/jobs/__tests__/run_mlb_model.test.js` — 5 new tests covering: missing offense profiles → SYNTHETIC_FALLBACK, f5_runs null → SYNTHETIC_FALLBACK, edge below threshold → PASS, OVER fires at projectedBase >= 5.0, UNDER fires at projectedBase <= 3.0

**Out of scope**:
- `projectF5ML()` / F5 money-line side (that's WI-0871)
- Full-game total (WI-0872)
- Probability-based (Poisson CDF) edge reformulation — run-based edge consistent with existing `selectMlbGameMarket` path
- Any DB schema changes
- Changes to `buildBettingSurfacePayloadPredicate` / `/cards` filter logic

**Background**:
`computeProjectionFloorF5()` already computes a projected F5 floor via simple ERA arithmetic and the `withoutOddsMode` branch stores it as a `SYNTHETIC_FALLBACK` driver (status: PASS, action: PASS, explicitly blocked by `/cards`). This was a deliberate placeholder — "show a projected line but never generate plays." Now that `projectTeamF5RunsAgainstStarter()` (WI-0821) and offense profiles (WI-0830) are live, we can compute a real edge against a synthetic market line and emit actual cards.

The synthetic line is chosen from the two standard F5 totals: 3.5 (low-offense/elite-pitching games) or 4.5 (all others). Midpoint = 4.0: if `projectedBase < 4.0` → `syntheticLine = 3.5`, else `syntheticLine = 4.5`. Run-edge threshold is the existing `MLB_F5_EDGE_THRESHOLD = 0.5`. Dead zone is 3.0 < projectedBase < 5.0 — intentionally conservative; only extreme projections fire.

**Implementation notes**:

New helper: `computeSyntheticLineF5Driver(mlb, context, gameId)`
- `mlb` is the parsed `rawData.mlb` object from the odds snapshot
- Returns `null` if `mlb.home_offense_profile == null || mlb.away_offense_profile == null` (caller falls back to existing SYNTHETIC_FALLBACK PASS behavior)
- `homeResult = projectTeamF5RunsAgainstStarter(mlb.home_pitcher, mlb.away_offense_profile, context)` — away offense sees home starter
- `awayResult = projectTeamF5RunsAgainstStarter(mlb.away_pitcher, mlb.home_offense_profile, context)` — home offense sees away starter
- Returns `null` if either `result.f5_runs === null` (missing_inputs on either side; caller falls back to SYNTHETIC_FALLBACK)
- `projectedBase = homeResult.f5_runs + awayResult.f5_runs`
- `syntheticLine = projectedBase < 4.0 ? 3.5 : 4.5`
- `edge = projectedBase - syntheticLine`
- Confidence: start at 6 (synthetic line carries risk vs real 7); subtract 1 for each side whose `degraded_inputs` is non-empty; floor 4
- If `edge >= MLB_F5_EDGE_THRESHOLD && confidence >= 8`: prediction = 'OVER', status/action/classification = 'FIRE'
- If `edge <= -MLB_F5_EDGE_THRESHOLD && confidence >= 8`: prediction = 'UNDER', status/action/classification = 'FIRE' (note: starting at 6 means you need no degraded inputs on either side for FIRE; one degraded side → confidence 5 → WATCH at best; two → 4 → PASS)
- `projection_source: 'FULL_MODEL'` (not SYNTHETIC_FALLBACK — must pass `/cards` filter)
- `reason_codes` always includes `'SYNTHETIC_LINE_ASSUMPTION'`; add `'DEGRADED_INPUT_HOME'` / `'DEGRADED_INPUT_AWAY'` per side
- Return the full driver card object shaped identically to existing `projectionFloorDriver`

Wire-up in `withoutOddsMode` block:
- Replace the `const projectionFloorDriver = ...` ternary with:
  1. Try `const syntheticEdgeDriver = computeSyntheticLineF5Driver(mlb, context, gameId)` first
  2. If non-null: use as the driver
  3. If null: fall back to existing `projectionFloorF5` SYNTHETIC_FALLBACK PASS block (preserve exactly as-is for no-offense-profile degrade case)
- The `mlb` object must be parsed from `gameOddsSnapshot` at the top of the block (already done for `f5MlResult` a few lines earlier — reuse the same parse)
- `context` = `{ park_run_factor: mlb.park_run_factor ?? null, temp_f: mlb.temp_f ?? null, wind_mph: mlb.wind_mph ?? null, wind_dir: mlb.wind_dir ?? null, roof: mlb.roof ?? null }`

**Acceptance**:
1. `computeSyntheticLineF5Driver()` returns null when `home_offense_profile` or `away_offense_profile` is absent → caller produces SYNTHETIC_FALLBACK PASS card (existing behavior preserved)
2. `computeSyntheticLineF5Driver()` returns null when either `projectTeamF5RunsAgainstStarter` returns `f5_runs: null` → same fallback
3. `projection_source` is `'FULL_MODEL'` on emitted synthetic edge cards (not `'SYNTHETIC_FALLBACK'`), so `/cards` does not filter them
4. OVER fires when `projectedBase >= 5.0` against `syntheticLine = 4.5`
5. UNDER fires when `projectedBase <= 3.0` against `syntheticLine = 3.5`
6. Dead zone: `3.0 < projectedBase < 5.0` → confidence 6 but `|edge| < 0.5` or `|edge| >= 0.5` but confidence < 8 if degraded → PASS or WATCH, never FIRE
7. `reason_codes` always contains `'SYNTHETIC_LINE_ASSUMPTION'` on synthetic edge cards
8. `npm --prefix apps/worker test --no-coverage` passes, zero regressions
