# WI-0903: Negative-Path Safety Audit and Coverage Pack

**ID**: WI-0903
**Goal**: Add explicit negative-path tests for suppressed, downgraded, hidden, and blocked output states where operational safety is most likely to fail silently.

**Depends on**: WI-0901, WI-0902

**Scope**:

- `web/src/__tests__/negative-path-cards-games-fixtures.test.js`
- `web/src/__tests__/negative-path-pass-visibility-defaults.test.js`
- `apps/worker/src/jobs/__tests__/negative-path-gates.test.js`
- `apps/worker/src/jobs/__tests__/negative-path-settlement-live-truth.test.js`
- `docs/audits/negative-path-coverage.md`

**Out of scope**:

- Positive-path tuning
- UI style/layout changes
- New market models

**Acceptance**:

- Coverage includes: odds present but no play, play selected then downgraded, projection-only hidden by default, missing starter/goalie blocks, stale line blocks execution, UI default filter hides valid PASS, and settlement cannot rewrite live truth.
- Each scenario has deterministic fixture inputs and expected reason/status outcomes.
- Test failures are behavior-first and do not depend on source-string matching.

**Requirements coverage map**:

- R1 (cards/games negative fixture corpus with deterministic outputs, including documented fixture matrix) -> Task 1
- R2 (UI default PASS-hidden behavior and explicit include-PASS behavior) -> Task 2
- R3 (worker gate failures return explicit reason/status outcomes) -> Task 3
- R4 (settlement cannot rewrite live truth state) -> Task 4
- R5 (audit matrix maps each negative path to assertions and rationale) -> Task 1

**Preflight dependency gate**:

- Confirm WI-0901 reason-code taxonomy is merged so fixtures assert stable reason codes.
- Confirm WI-0902 parity fixtures are available so negative-path fixture shapes stay aligned.
- Baseline pass before edits:
  - `node web/src/__tests__/api-endpoint-parity-fixtures.test.js`
  - `npm --prefix web run test:filters`
  - `npm --prefix apps/worker run test -- --testPathPattern="settle_pending_cards.phase2|run_mlb_model|run_nhl_model|run_nba_model"`

**Must-haves (goal-backward)**:

**Truths**:

- A candidate suppressed after valid odds still emits explicit no-play reason/status.
- A selected play downgraded by a later gate emits downgrade reason and layer attribution.
- Projection-only recommendations are hidden by default and visible only when explicitly enabled.
- Missing starter/goalie context and stale lines deterministically block execution with explicit reasons.
- Settlement paths do not mutate already-live truth outcomes.

- `web/src/__tests__/negative-path-cards-games-fixtures.test.js` provides shared negative fixture corpus for cards/games.
- `web/src/__tests__/negative-path-pass-visibility-defaults.test.js` locks default PASS-hidden and include-PASS behaviors.
- `apps/worker/src/jobs/__tests__/negative-path-gates.test.js` locks blocked/downgraded gate behavior and reason/status outcomes.
- `apps/worker/src/jobs/__tests__/negative-path-settlement-live-truth.test.js` locks settlement no-rewrite invariants for live truth.
- `docs/audits/negative-path-coverage.md` documents matrix coverage and expected failure semantics.
