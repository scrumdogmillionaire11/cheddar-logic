# WI-0898: Env-Var Drift Audit for Play Qualification and Visibility

**ID**: WI-0898
**Goal**: Produce a deterministic config truth table for all environment variables that can change qualification, downgrade behavior, execution gating, projection-only handling, and UI visibility.

**Depends on**: none

**Scope**:
- apps/worker/src/jobs/**/*.js
- apps/worker/src/models/**/*.js
- web/src/app/api/**/*.ts
- web/src/lib/**/*.ts
- packages/models/src/**/*.js
- packages/data/src/**/*.js
- docs/audits/env-var-truth-table.md
- docs/runbooks/config-overrides.md

**Out of scope**:
- Changing threshold values or defaults
- Removing variables from runtime
- Altering deployment secrets

**Acceptance**:
- A single table lists each env var, default, read location, behavior impact, safe range, dangerous values, and test coverage status.
- Variables are classified as live, overridden in prod, stale/dead, or shadowed aliases.
- Qualification-impacting and downgrade-impacting vars are explicitly tagged.
- Projection-only and UI visibility toggles are explicitly tagged.
- Dead vars are listed with deletion recommendation and blast-radius note.

**Owner agent**: unassigned
**Time window**: 2026-04-11 -> TBD
**Coordination flag**: needs-sync

**Tests to run**:
- npm --prefix web run build
- npm --prefix apps/worker run test

**Manual validation**:
- Compare production env template to the truth table and confirm no undocumented overrides remain.
- Validate at least one known qualification gate end-to-end by toggling only one variable and observing expected surfaced status change.
