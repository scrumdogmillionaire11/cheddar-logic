# WI-0898: Env-Var Drift Audit for Qualification, Downgrade, and Execution Gating

**ID**: WI-0898
**Goal**: Produce a deterministic, searchable config truth table for environment variables that change play qualification, downgrade behavior, execution gating, and sizing. This is a prerequisite for WI-0906 (continuity audit) which requires a definitive map of where edge/confidence/size signals are gated by config.

**Depends on**: none

**Scope** (NARROWED — qualification and execution gating only):

**Must-audit files (qualification, downgrade, gating logic)**:
- `packages/models/src/edge-calculator.js` — edge measurement and edge-related toggles
- `packages/models/src/decision-pipeline-v2.js` — qualification gates, downgrade decision branches
- `apps/worker/src/jobs/execution-gate.js` — execution eligibility and quality state gating
- `apps/worker/src/jobs/run_nba_model.js` — sport-specific downgrade and qualification gates
- `apps/worker/src/jobs/run_nhl_model.js` — sport-specific downgrade and qualification gates
- `apps/worker/src/jobs/run_mlb_model.js` — sport-specific downgrade and qualification gates
- `apps/worker/src/jobs/potd/run_potd_engine.js` — POTD-specific qualification and downgrade gates
- `apps/worker/src/jobs/potd/signal-engine.js` — POTD signal tier and confidence gates
- `docs/audits/env-var-truth-table.md` — output artifact
- `WORK_QUEUE/WI-0898.md` — this work item

**Out of scope** (explicitly deferred):
- UI visibility toggles (web layer env vars) — separate future work item
- Projection-only modes — separate future work item
- Changing threshold values or defaults
- Removing variables from runtime
- Altering deployment secrets
- Dead/stale var deletion recommendations (mark only; decision-maker action)

**Coordination flag**: `needs-sync`
- Synchronization point: WI-0906 will read the truth table produced by WI-0898 and use it to validate continuity lineage. Both items must complete and be reviewed before staging.
- Shared touchpoint: `docs/audits/env-var-truth-table.md` (created by WI-0898, referenced by WI-0906)

**Acceptance Criteria** (linked to WI-0906 requirements):

1. **Deterministic truth table exists** (supports WI-0906-REQ-01: edge→confidence→size lineage)
   - Single markdown table: `docs/audits/env-var-truth-table.md`
   - Schema columns: `Var Name | Default | Read Location (module/function) | Qualification Impact | Downgrade Impact | Execution Gate Impact | Safe Range | Dangerous Values | Test Coverage (file:test_name or "N/A") | Status (live/overridden/stale) | Notes`
   - Minimum 25 rows (estimate: 1 per sport + POTD + core gates)
   - Example 3 rows provided below (see Output Format section)

2. **All qualification-impacting vars are identified and classified** (supports WI-0906-REQ-02: breaks where size ≠ 0 despite non-actionable)
   - Each row explicitly tagged: `qual_impact: true` or `qual_impact: false`
   - Examples: `NO_BET_IF_*`, `DOWNGRADE_IF_*`, `QUALIFICATION_MIN_*` marked `qual_impact: true`

3. **All downgrade-impacting vars are identified and classified** (supports WI-0906-REQ-02: fragile confidence breaks)
   - Each row explicitly tagged: `downgrade_impact: true` or `downgrade_impact: false`
   - Examples: `DEGRADED_IF_*`, `STALE_EDGE_HOURS` marked `downgrade_impact: true`

4. **All execution-gating vars are identified** (supports WI-0906-REQ-04: remediation guards for zero-size)
   - Each row explicitly tagged: `exec_gate_impact: true` or `exec_gate_impact: false`
   - Examples: `ENABLE_*_MODEL`, `FORCE_NO_BET` marked `exec_gate_impact: true`

5. **Test coverage is measurable and documented** (replaces unmeasurable "test coverage status")
   - Each row lists either:
     - `test_file:test_name` (e.g., `apps/worker/src/jobs/__tests__/execution-gate.test.js:should_zero_size_on_no_bet_gate`)
     - OR `none_found` (auditor searched for var name in test files; no test exists)
   - Auditor provides grep search evidence in PR comments (e.g., "searched `rg DOWNGRADE_IF_STALE_EDGE_HOURS apps/worker/src/jobs/__tests__/ -- found in execution-gate.test.js line 142")

6. **Dead/stale vars are marked (decision-maker action deferred)**
   - Each row tagged: `status: live | status: overridden_in_prod | status: stale | status: dead_candidate`
   - For `dead_candidate`: include field `dead_reason` (e.g., "var exists in code but no reads detected in current flow paths as of 2026-04-17")
   - Do NOT recommend deletion (that is out-of-scope); only mark for review

7. **Shadowed aliases are documented** (supports finding continuity breaks from config confusion)
   - Example: If `OLD_THRESHOLD` and `NEW_THRESHOLD` both exist, row notes: "alias for NEW_THRESHOLD; should remove OLD_THRESHOLD (out of scope)"

8. **One end-to-end manual validation gate performed** (supports manual acceptance)
   - Gate 1 (primary): `execution_gate.js` + `NO_BET_IF_STALE_EDGE_HOURS` (toggle: check zero sizing)
   - Gate 2 (secondary): `run_nba_model.js` + `DOWNGRADE_IF_EDGE_MOVE_PCT` (toggle: check downgrade status)

**Owner agent**: unassigned

**Time window**: 2026-04-17 -> 2026-04-24 (1 week sprint ending when WI-0906 can start)

**Tests to run**:
- `npm --prefix apps/worker run test` (must pass; any new tests for env var mocks)
- `npm --prefix web run build` (sanity check; web not in primary scope but ensure no regressions)

**Manual validation**:

1. **Gate 1 Test: NO_BET_IF_STALE_EDGE_HOURS qualification override**
   - Setup: Local dev environment with `.env.local` or `.env.test`
   - Action: Set `NO_BET_IF_STALE_EDGE_HOURS=0.5` (override to 30 minutes)
   - Action: Trigger NBA model run with a card that would normally qualify
   - Observe: Model output shows card with `quality_status: "NO_BET"` and `stake: 0`
   - Expected: Env var in truth table row correctly identifies this as `qual_impact: true` and `exec_gate_impact: true`
   - Pass criteria: Stake is 0, reason-code includes env var name

2. **Gate 2 Test: DOWNGRADE_IF_EDGE_MOVE_PCT downgrade gate**
   - Setup: Local dev environment
   - Action: Set `DOWNGRADE_IF_EDGE_MOVE_PCT=0.5` (override to 50% move threshold)
   - Action: Trigger NHL model run with an edge that has a 40% intraday move (below threshold)
   - Observe: Model output shows card with `quality_status: "DEGRADED"` (not `NO_BET`)
   - Expected: Env var in truth table row correctly identifies this as `downgrade_impact: true` and `qual_impact: false`
   - Pass criteria: Status is DEGRADED, stake is reduced (not zero), reason-code includes env var name

3. **Final verification**: Compare truth table to production `.env.example` or `.env.prod-template`
   - Ensure every env var in truth table with `status: live` or `status: overridden_in_prod` has a documented value/comment in the template
   - Report any undocumented overrides as "config debt" for future audit
