# WI-0543: Static asset drift hardening (multi-release retention + targeted purge + global stale guard)

**ID**: WI-0543
**Goal**: Eliminate production CSS/JS static-asset 404s caused by stale HTML or cached bundles by preserving recent `_next/static` generations, tightening cache policy, and adding a global stale-asset fallback guard.

**Out of scope**:

**Acceptance**:
- Deploy workflow archives recent `_next/static` generations with retention controls (14 days, max 20 snapshots), merges retained assets into current build static output, and performs rollback to previous `.next` on verification failure.
- Cloudflare purge uses URL-list HTML entry paths (`/`, `/cards`, `/fpl`) instead of `purge_everything`.
- Cache headers explicitly enforce immutable caching for `/_next/static/:path*` and no-store for HTML entry routes (including `/`).
- Global stale-asset guard is mounted at app root and handles stale `_next/static` JS/CSS failures with one reload-per-session fallback.
- Cards page stale-asset handling uses shared detector/session key to avoid conflicting reload semantics.
