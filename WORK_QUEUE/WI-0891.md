# WI-0891: Remove Live Split-Brain Between Settlement Backfill and /api/games Selection

**ID**: WI-0891
**Goal**: Eliminate split-brain behavior where settlement-time backfill updates to card_display_log can change what /api/games surfaces as true_play.

**Depends on**: WI-0890

**Scope**:
- apps/worker/src/jobs/settle_pending_cards.js
- web/src/lib/games/route-handler.ts
- apps/worker/src/jobs/__tests__/settle_pending_cards.phase2.test.js
- web/src/__tests__/api-games-missing-data-contract.test.js

**Out of scope**:
- Settlement math for grading wins/losses
- New database tables
- Changes to /api/cards routes

**Acceptance**:
- Settlement backfill no longer writes fields that can change live true_play ranking for active game surfaces.
- /api/games true_play selection uses a single consistent path even when display log rows exist.
- Existing backfill remains available for historical reporting where needed.
- Tests cover both active-run and no-active-run paths.

**Owner agent**: unassigned
**Time window**: 2026-04-11 -> TBD
**Coordination flag**: needs-sync

**Tests to run**:
- npm --prefix apps/worker test -- --testPathPattern="settle_pending_cards.phase2"
- node web/src/__tests__/api-games-missing-data-contract.test.js

**Manual validation**:
- Run settlement job twice on same dataset and confirm /api/games true_play remains unchanged for active cards.
- Verify no new card_display_log mutations are introduced for live-play authority fields.
