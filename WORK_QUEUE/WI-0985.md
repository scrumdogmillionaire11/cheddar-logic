# WI-0985: Canonical Quick Filter Preset Helpers

**ID**: WI-0985
**Goal**: Extract canonical preset matching and apply/reset helpers, then switch the filter panel to consume them so preset behavior is deterministic without relying on duplicated filter-panel-local logic.

**Depends on**: WI-0983

**Scope**:

- web/src/lib/game-card/presets.ts
- web/src/lib/game-card/preset-helpers.ts
- web/src/components/filter-panel.tsx
- web/src/__tests__/filters-presets-contract.test.js
- web/src/__tests__/preset-helpers.test.js

**Out of scope**:

- Cards page shell/layout changes outside the filter panel
- Diagnostics bucket behavior
- Changing non-preset filtering semantics

**Acceptance**:

- Preset matching and apply/reset logic is exported from a canonical helper module instead of living only inside the filter panel component.
- The filter panel consumes the canonical helper path for preset apply, active-state matching, and re-click reset behavior.
- Helpers derive preset application from mode defaults rather than stale inherited UI state.
- Active preset matching is order-insensitive and ignores irrelevant hidden/default-only fields.
- Helpers are covered by direct contract tests that do not depend on the generic cards page smoke harness.
