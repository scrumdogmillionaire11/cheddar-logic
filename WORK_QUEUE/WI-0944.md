# WI-0944: MLB Full-Game Market De-Suppression (Totals + Moneyline)

**ID**: WI-0944
**Goal**: Reduce over-suppression in MLB full-game totals and moneyline while preserving edge discipline and deterministic guardrails.

**Scope**:

- `apps/worker/src/models/mlb-model.js`
- `apps/worker/src/models/__tests__/mlb-model.test.js`
- `apps/worker/src/models/__tests__/mlb-model-gate.test.js`
- `apps/worker/src/jobs/run_mlb_model.js`
- `WORK_QUEUE/WI-0944.md`

**Out of scope**:

- New market types or schema/table additions
- Watchdog bypasses or disabling pipeline guard checks
- Changes to NBA/NHL model logic
- Forcing publish of non-edge plays

**Acceptance**:

- Mandatory funnel trace is implemented for last N=50 MLB games and reports stage counts: total_candidates_created, passed_projection, passed_edge_threshold, passed_volatility_threshold, passed_driver_support, passed_f5_contradiction, passed_confidence, final_official_plays.
- Funnel output includes per-stage drop percentages and is produced in deterministic format.
- Top 2 suppressors are identified from funnel data with exact code condition and candidate impact percentage.
- Full-game totals logic is updated to reduce late-innings noise sensitivity without removing edge-based discipline.
- Volatility edge threshold is converted to dynamic form with multiplier cap and absolute threshold cap <= 0.65.
- F5 contradiction is downgraded from hard veto to soft penalty (tier downgrade) for full-game totals.
- Full-game moneyline adds a separate adjustment layer (bullpen asymmetry, home inning edge, late leverage factor) instead of relying solely on total/run projection output.
- Full-game moneyline edge threshold is reduced to 0.025 with confidence gate requirement capped at 6.
- Redundant hard gates are collapsed so no more than two independent hard gates remain after projection; remaining checks become soft penalties.
- Deterministic tests cover: PASS->PLAY, PASS->PASS, and PLAY unchanged scenarios for MLB full-game markets.
