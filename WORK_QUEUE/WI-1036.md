# WI-1036: Default `/api/cards` to today's games only; add `?date=all` opt-out

**ID**: WI-1036
**Goal**: The `/api/cards` route currently returns cards for any game within the model's 36-hour lookahead window. This causes tomorrow's cards to appear in the UI alongside today's. Add a `?date=` query param that defaults to today (UTC date) so only current-day games are returned by default. Pass `?date=all` to restore full-horizon behavior for operators or future UI toggles.

**Depends on**: none

**Scope**:
- `WORK_QUEUE/WI-1036.md`
- `.planning/STATE.md`
- `web/src/app/api/cards/route.ts`

**Out of scope**:
- `web/src/app/api/cards/[gameId]/route.ts` ā€” per-game lookup, no date window applies
- Any UI component changes (API change is sufficient; UI can pass `?date=all` when needed)
- Worker model jobs ā€” no changes to how cards are generated or stored
- `getOddsWithUpcomingGames` ā€” no changes to the 36-hour ingestion window

**Behavior spec**:

| `?date=` param | Behavior |
|---|---|
| omitted (default) | Filter to `DATE(g.game_time_utc) = DATE('now')` |
| `?date=2026-04-19` | Filter to `DATE(g.game_time_utc) = '2026-04-19'` |
| `?date=all` | No date filter applied (current behavior) |

The date comparison is always against `g.game_time_utc` (UTC) ā€” no timezone conversion needed because game dates are stored in UTC and the display context is already UTC-aware.

**Implementation**:

In `route.ts`, after existing param extraction (around line 345), add:

```ts
const dateParam = searchParams.get('date') ?? null; // null ā†’ today default
const showAllDates = dateParam === 'all';
const gameDateFilter =
  !showAllDates && dateParam && /^\d{4}-\d{2}-\d{2}$/.test(dateParam)
    ? dateParam
    : !showAllDates
    ? new Date().toISOString().slice(0, 10) // today UTC
    : null;

if (gameDateFilter) {
  baseWhere.push("DATE(g.game_time_utc) = ?");
  baseParams.push(gameDateFilter);
}
```

This requires the existing `LEFT JOIN games g` in `buildSql` to be present for both dedupe modes ā€” it already is (both CTE and flat query join `games g`).

**Acceptance**:
- `GET /api/cards?sport=mlb` returns only cards for `DATE(game_time_utc) = today` (UTC)
- `GET /api/cards?sport=mlb&date=all` returns cards across the full lookahead window (current behavior)
- `GET /api/cards?sport=mlb&date=2026-04-19` returns only April 19 games
- `GET /api/cards?sport=mlb&date=invalid` falls back to today (malformed date treated as today)
- `npm --prefix web run build` passes with no new TypeScript errors
