## Fixture schema (required for all)

```json
{
  "fixture_id": "nba_2026_04_01_total_clean_01",
  "sport": "NBA",
  "description": "Clean total edge — full odds, FIRE status, executable",
  "input_contract": "ODDS_SNAPSHOT",
  "input": {
    "game_id": "nba_test_game_001",
    "sport": "NBA",
    "home_team": "Boston Celtics",
    "away_team": "Golden State Warriors",
    "game_time_utc": "2026-04-02T00:00:00Z",
    "total": 224.5,
    "total_price_over": -110,
    "total_price_under": -110,
    "spread_home": -7.5,
    "spread_away": 7.5,
    "spread_price_home": -112,
    "spread_price_away": -108,
    "h2h_home": -320,
    "h2h_away": 260,
    "captured_at": "2026-04-01T18:00:00Z"
  },
  "expected": {
    "input_hash": "RECOMPUTE_ON_FIRST_RUN",
    "execution_status": "EXECUTABLE",
    "classification": "PLAY",
    "market_type": "TOTAL",
    "reason_codes_must_not_include": ["PASS_NO_EDGE", "BLOCKED_MISSING_PRICING"]
  }
}
```

Notes:
- `input_hash` should be set to `"RECOMPUTE_ON_FIRST_RUN"` initially; after first snapshot run, replace with the actual computed hash. This prevents stale hashes blocking CI before baselines are established.
- `expected` fields are the minimum required; additional fields can be added for stricter comparison.
