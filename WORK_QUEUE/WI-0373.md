# WI-0373: Canonical Settlement P/L Calculation (Forward-Only)

**ID**: WI-0373
**Goal**: Ensure settlement P/L is computed by a single canonical formula for newly settled cards, with deterministic handling for wins/losses/pushes and malformed/missing odds.

**Scope**:
- `WORK_QUEUE/WI-0373.md`
- `WORK_QUEUE/README.md`
- `apps/worker/src/jobs/settle_pending_cards.js`
- `apps/worker/src/__tests__/settle-coverage.test.js`
- `docs/ops-runbook.md`

**Out of scope**:
- Historical resettlement/backfill of existing rows
- DB schema migrations
- Results UI redesign beyond existing W/L-first behavior

**Acceptance**:
- New settlements use one canonical P/L computation path with explicit rules for:
  - `win`, `loss`, `push`
  - positive/negative American odds
  - null/invalid/zero odds (P/L nullable, non-fatal)
- Settlement logs expose P/L-computation anomalies without failing W/L grading.
- Regression tests validate formula outputs and edge cases.
- Runbook documents how to interpret nullable/optional P/L values operationally.

**Owner agent**: unassigned
**Time window**: TBD -> TBD
**Coordination flag**: needs-sync

- `CARDS_API_BASE_URL=http://127.0.0.1:3000 node web/src/__tests__/ui-results-smoke.test.js`

**Manual validation**:
- Settle a controlled sample containing win/loss/push with mixed odds signs.
- Verify W/L counts are correct even when P/L is null for malformed odds.
- Verify `/api/results` continues to return W/L metrics as primary and P/L as optional metadata.
