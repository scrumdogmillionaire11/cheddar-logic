# WI-0794: Activate admin API auth wall and add Next.js middleware guard

> **STATUS: DEFERRED** — Auth infrastructure not ready. Revisit when auth rollout is scoped. (2026-04-05)

**ID**: WI-0794
**Goal**: Enable `ENABLE_AUTH_WALLS=true` in production and add a `web/src/middleware.ts` that enforces authentication on all `/api/admin/*` routes at the Edge layer so no new admin route can accidentally be left open.

**Scope**:

- `web/src/middleware.ts` (create)
- `web/src/app/api/admin/pipeline-health/route.ts`
- `web/src/app/api/admin/odds-ingest/route.ts`
- `web/src/app/api/admin/audit/route.ts`
- `.env.production` (add `ENABLE_AUTH_WALLS=true`, `ADMIN_API_SECRET=<placeholder>`)

**Out of scope**:

- Auth implementation for public routes (`/api/cards`, `/api/games`, `/api/results`)
- JWT revocation persistence (WI-0796)
- UI auth flows (magic link, login page)

**Acceptance**:

- `GET /api/admin/pipeline-health` returns 403 without `x-admin-secret` header when `ENABLE_AUTH_WALLS=true`
- `GET /api/admin/odds-ingest` returns 403 without correct header
- `GET /api/admin/audit` returns 403 without correct header (it already has the guard code — just needs activation)
- Next.js `middleware.ts` matches `/api/admin/:path*` and rejects requests missing the header even if an individual route handler forgets to call `performSecurityChecks`
- `.env.production` contains `ENABLE_AUTH_WALLS=true` and `ADMIN_API_SECRET` placeholder with generation instructions
