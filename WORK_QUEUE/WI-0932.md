# WI-0932: NHL execution wrapper HOLD gate for goalie/injury uncertainty

**ID**: WI-0932
**Goal**: Enforce explicit HOLD behavior when goalie confirmation or key injury certainty is insufficient, instead of letting uncertain games appear as normal slight-edge outputs.

**Depends on**: WI-0931

**Scope**:

- `apps/worker/src/jobs/run_nhl_model.js`
- `packages/models/src/decision-pipeline-v2.js`

**Out of scope**:

- Reweighting NHL drivers or changing projection formulas
- Stale snapshot source fixes in scheduler/pull jobs
- Book-source integrity checks for mixed-book line/price mismatches

**Acceptance**:

- Wrapper sets HOLD-equivalent state (action/status/reason) when goalie is unconfirmed or key injury certainty fails configured criteria.
- Confirmed-input games remain eligible for FIRE/WATCH/PASS as determined by the decision pipeline.
- PASS remains reserved for true no-edge or hard-invalidation paths, not generic uncertainty.
- Tests cover both unknown-goalie and confirmed-goalie paths and assert deterministic reason codes.
