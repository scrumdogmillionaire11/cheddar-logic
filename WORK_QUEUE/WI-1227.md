# WI-1227: Add Direct Unit Tests for resolvePrimaryReason and resolveTerminalReasonFamily

**ID**: WI-1227
**Goal**: Add a dedicated test file that directly exercises `resolvePrimaryReason()` and `resolveTerminalReasonFamily()` at unit level, covering every explicit branch and the current catch-all; establishes a regression baseline before WI-1225 and WI-1226 make structural changes.

**Context**:

`resolvePrimaryReason()` and `resolveTerminalReasonFamily()` in `decision-pipeline-v2.js` have no direct unit tests. They are only exercised indirectly through integration tests (`decision-pipeline-v2-*.test.js`) that build full payloads. This means:

- The silent `SUPPORT_BELOW_PLAY_THRESHOLD` fallback (WI-1225) and the `NO_PRIMARY_SUPPORT` misrouting (WI-1226) are not caught by the existing test suite.
- Changes to either function must be validated by running large integration tests rather than fast focused tests.

This WI adds the baseline. WI-1225 and WI-1226 will add to it; this WI must not depend on those changes being in place (it documents current behavior, including the fallback being `SUPPORT_BELOW_PLAY_THRESHOLD`).

Both functions are not currently exported. They should be exported from `decision-pipeline-v2.js` under a test-surface export (or via `module.exports` if the module already uses CommonJS) so they can be required directly by tests. Alternatively, test them via a thin wrapper if the module pattern makes direct export impractical.

**Scope**:

- `packages/models/src/decision-pipeline-v2.js` — export `resolvePrimaryReason` and `resolveTerminalReasonFamily` (test surface only; no behavior change)
- `packages/models/src/__tests__/decision-pipeline-v2-reason-resolution.test.js` (new)
- `WORK_QUEUE/WI-1227.md`
- `.planning/STATE.md`

**Out of scope**:

- Any behavior changes to either function (→ WI-1225, WI-1226)
- Integration-level tests (those already exist in `decision-pipeline-v2-*.test.js`)

**Acceptance**:

- New test file directly imports and calls `resolvePrimaryReason` and `resolveTerminalReasonFamily`.
- `resolvePrimaryReason` branch coverage includes: watchdog blocked, UNPRICED/COTTAGE, play-capped reason, proxy-capped, PLAY/LEAN status, support below lean, edge below play_edge_min, null edgePct fallback.
- `resolveTerminalReasonFamily` branch coverage includes: PLAY_ELIGIBLE, WATCHDOG_DATA_QUALITY, EXACT_WAGER_FAIL, LINE_NOT_CONFIRMED, PRICING_UNAVAILABLE (including NO_PRIMARY_SUPPORT current behavior), POLICY_BLOCK, EDGE_INSUFFICIENT, EXECUTION_GATE fallback.
- All tests pass against current unmodified code (they document existing behavior).
- No existing tests regress.
