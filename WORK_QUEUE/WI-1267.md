# WI-1267: Centralize Game Card Eligibility Contract

**ID**: WI-1267
**Goal**: Create one machine-readable eligibility contract for game cards so transform exclusion reasons and filter actionability use the same status/reason model.

**Scope**:
- `web/src/lib/game-card/transform/index.ts`
- `web/src/lib/game-card/filters.ts`
- `web/src/components/cards/shared.ts`
- `web/src/lib/game-card/eligibility.ts` (new)
- `web/src/__tests__/game-card-decision-authority.test.ts`

**Out of scope**:
- Worker decision logic
- `/api/results`
- Styling-only changes
- Changing betting thresholds

**Acceptance**:
- Eligibility contract returns `eligible`, `reason_code`, `surface_mode`, and `actionability`.
- Transform and filters consume the same contract.
- `plays > 0` cannot coexist with hidden card state unless reason code explains why.
- Tests cover PLAY/LEAN/PASS with EXECUTABLE/BLOCKED/PROJECTION_ONLY combinations.
- Projection-only MLB rows keep `mlb-projection-surface` behavior.

**Manual validation**:

- Verify actionable and blocked permutations expose consistent reason codes across transform and filters.

**Guard for WI closeout**:
- Do not change threshold policy; only unify eligibility semantics.
