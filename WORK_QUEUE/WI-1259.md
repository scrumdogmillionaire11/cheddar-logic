# WI-1259: Align MLB F5 ML Side Selection to Team F5 Run Splits

**ID**: WI-1259
**Goal**: Ensure MLB F5 ML side selection always follows the higher projected team F5 runs from the shared team-split projection path.

**Context**:

Current `projectF5ML` can select side by de-vig edge versus market implied probability. In practice this can select the lower projected team when market prices are skewed. For projection-card integrity, side selection must follow projected team split winner (`projected_home_f5_runs` vs `projected_away_f5_runs`).

**Scope**:

- `apps/worker/src/models/mlb-model.js`
- `apps/worker/src/models/__tests__/mlb-model.test.js`
- `WORK_QUEUE/WI-1259.md`

**Out of scope**:

- F5 total projection formula changes
- Full-game ML side-selection logic
- Odds ingestion or de-vig normalization changes

**Acceptance**:

- `projectF5ML` selects `HOME` when `projected_home_f5_runs > projected_away_f5_runs` and `AWAY` when the reverse.
- If projected runs are effectively tied, selection returns `PASS`.
- Edge diagnostics (`edgeH`, `edgeA`) remain available in reasoning for observability, but do not drive side selection.
- Add regression tests that fail under old edge-based selection and pass with projection-based side selection.

- Validate tie/noise scenario returns `PASS`.
