# WI-1265: Add Canonical Game Row Reconciliation Job

**ID**: WI-1265
**Goal**: Add a deterministic cleanup/reconciliation path that identifies stale external-ID game rows once `game_id_map` links them to canonical game IDs.

**Scope**:
- `packages/data/src/db/games.js`
- `apps/worker/src/jobs/reconcile_game_ids.js` (new)
- `apps/worker/src/__tests__/reconcile-game-ids.test.js` (new)
- `WORK_QUEUE/WI-1265.md`

**Out of scope**:
- Hard deletion of historical rows
- Breaking existing `game_id_map` constraints
- Changing odds ingestion identity rules

**Acceptance**:
- Job detects external-ID rows mapped to canonical IDs.
- Job reports rows eligible for merge/retirement.
- First implementation is safe/dry-run by default.
- No destructive cleanup occurs unless explicitly enabled.
- Tests prove mapped duplicate rows are detected and unmapped rows are untouched.

**Owner agent**: unassigned
**Time window**: 4-6 hours
**Coordination flag**: needs-sync
**Depends on**: WI-1264

**Tests to run**:
- `npm --prefix apps/worker test -- reconcile-game-ids`

**Manual validation**:
- Run the reconciliation job in dry-run mode against a DB snapshot containing mapped duplicates.
- Confirm output identifies only mapped external-ID duplicates and does not mutate rows in dry-run mode.

**Guard for WI closeout**:
- Default execution must remain non-destructive.
- Any destructive mode must be explicit and separately documented.
