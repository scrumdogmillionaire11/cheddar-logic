# WI-1150: Fix POTD NHL Model Snapshot Lookup (Wrong Sport Filter)

**ID**: WI-1150
**Goal**: Fix `getLatestNhlModelOutput` so it returns goalie data for NHL games, enabling NHL moneyline candidates to receive `edgeSourceTag: 'MODEL'` and compete for POTD selection.

**Scope**:
- `packages/data/src/db/cards.js`
- `packages/data/__tests__/card-payload-sport.test.js`

**Out of scope**:
- Changes to NHL model runner (`run_nhl_model.js`)
- Changes to `insertCardPayload` normalization logic
- Changes to POTD signal-engine or run_potd_engine

**Root cause**:
`insertCardPayload` normalizes the `sport` field to lowercase (line 753: `card.sport.toLowerCase()`), so NHL cards are stored as `sport = 'nhl'`. The `getLatestNhlModelOutput` query uses `WHERE sport = 'icehockey_nhl'` — this never matches, so the function always returns `null`. When it returns `null`, `buildCandidates` in `signal-engine.js` cannot inject an `nhlSignal`, so `scoreCandidate` falls through to `CONSENSUS_FALLBACK`, and `isModelBackedCandidate` rejects all NHL candidates.

**Implementation**:

**Pre-flight check (run before coding — STOP if it fails)**

Verify that `nhl-model-output` cards in prod contain real goalie fields (not mock/legacy format):

```sql
SELECT json_extract(payload_data, '$.goalie_home_save_pct'),
       json_extract(payload_data, '$.goalie_home_gsax'),
       json_extract(payload_data, '$.goalie_away_save_pct'),
       json_extract(payload_data, '$.goalie_away_gsax')
FROM card_payloads
WHERE card_type = 'nhl-model-output'
AND created_at > datetime('now', '-24 hours')
LIMIT 5;
```

**STOP if all goalie columns return NULL.** This means the payload schema differs from what this WI expects (e.g., cards were written by the legacy mock runner, not the real pipeline). Do not proceed with Tasks 1-2; return the WI for revision with the actual `payload_data` structure from prod.

If numeric values are present in the goalie columns, proceed.

**Task 1 — `packages/data/src/db/cards.js` — fix the query predicate**

In `getLatestNhlModelOutput` (line ~1243), change:

```js
// BEFORE:
WHERE game_id = ? AND sport = 'icehockey_nhl'

// AFTER:
WHERE game_id = ? AND card_type = 'nhl-model-output'
```

This matches the 1135 rows that actually exist in prod (confirmed via DB query 2026-04-23). The `nhl-model-output` card type is written by `apps/worker/src/models/index.js` (line 952); `run_nhl_model.js` lists it in `NHL_DRIVER_CARD_TYPES` (line 458) as a **reader**, not a writer. These cards contain `goalie_home_save_pct`, `goalie_home_gsax`, `goalie_away_save_pct`, `goalie_away_gsax` in `payload_data`.

No other changes to the function body — existing field extraction logic (`rd.goalie_home_save_pct`, `rd.goalie?.home?.save_pct`, etc.) remains correct.
