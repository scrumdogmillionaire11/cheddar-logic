# WI-1150: Fix POTD NHL Model Snapshot Lookup (Wrong Sport Filter)

**ID**: WI-1150
**Goal**: Fix `getLatestNhlModelOutput` so it returns goalie data for NHL games, enabling NHL moneyline candidates to receive `edgeSourceTag: 'MODEL'` and compete for POTD selection.

**Scope**:
- `packages/data/src/db/cards.js`
- `packages/data/src/db/cards.test.js` (or nearest existing NHL test file)

**Out of scope**:
- Changes to NHL model runner (`run_nhl_model.js`)
- Changes to `insertCardPayload` normalization logic
- Changes to POTD signal-engine or run_potd_engine

**Root cause**:
`insertCardPayload` normalizes the `sport` field to lowercase (line 753: `card.sport.toLowerCase()`), so NHL cards are stored as `sport = 'nhl'`. The `getLatestNhlModelOutput` query uses `WHERE sport = 'icehockey_nhl'` — this never matches, so the function always returns `null`. When it returns `null`, `buildCandidates` in `signal-engine.js` cannot inject an `nhlSignal`, so `scoreCandidate` falls through to `CONSENSUS_FALLBACK`, and `isModelBackedCandidate` rejects all NHL candidates.

**Implementation**:

**Task 1 — `packages/data/src/db/cards.js` — fix the query predicate**

In `getLatestNhlModelOutput` (line ~1243), change:

```js
// BEFORE:
WHERE game_id = ? AND sport = 'icehockey_nhl'

// AFTER:
WHERE game_id = ? AND card_type = 'nhl-model-output'
```

This matches the 1135 rows that actually exist in prod (confirmed via DB query 2026-04-23). The `nhl-model-output` card type is the canonical model output card written by `run_nhl_model.js` and contains `goalie_home_save_pct`, `goalie_home_gsax`, `goalie_away_save_pct`, `goalie_away_gsax` in `payload_data`.

No other changes to the function body — existing field extraction logic (`rd.goalie_home_save_pct`, `rd.goalie?.home?.save_pct`, etc.) remains correct.

**Task 2 — add a regression test**

Add a test to the relevant cards test file that:
1. Inserts a `card_payloads` row with `sport = 'nhl'` (lowercase, as `insertCardPayload` normalizes it), `card_type = 'nhl-model-output'`, and goalie fields in `payload_data`
2. Calls `getLatestNhlModelOutput(gameId)` and asserts it returns `{ homeGoalie: { savePct, gsax }, awayGoalie: { savePct, gsax } }`
3. Asserts the function returns `null` when no matching row exists

**Acceptance**:
- `getLatestNhlModelOutput(gameId)` returns a non-null object with goalie fields for a game that has an `nhl-model-output` card in `card_payloads`
- POTD audit log shows NHL moneyline candidates with `edgeSourceTag: 'MODEL'` instead of `NON_MODEL_SOURCE` after next model run
- All existing card DB tests continue to pass
