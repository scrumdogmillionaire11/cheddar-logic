# WI-0815: Propagate `EDGE_SANITY_CLAMP_APPLIED` to watchdog CAUTION in `computeTotalEdge`

| Field | Value |
|---|---|
| **ID** | WI-0815 |
| **Goal** | When the NHL edge sanity clamp fires in `computeTotalEdge` (raw edge > 18%), force `watchdog_status='CAUTION'` so the play-cleanliness gate blocks the card from PLAY classification. Eliminates the path where model miscalibration produces a clamped-but-confident PLAY card. |
| **Priority** | Medium |
| **Depends on** | None |
| **Coordination flag** | None |

---

## Background

`computeTotalEdge` in `packages/models/src/edge-calculator.js` applies a sanity clamp for NHL-style totals (sigma ≤ 3):

```js
if (isNhlStyleTotal && Math.abs(edge) > 0.18) {
  edge = Math.sign(edge) * 0.18;
  railFlags.push('EDGE_SANITY_CLAMP_APPLIED');
}
```

A raw edge above 18% means the projection and the market line are dramatically misaligned — the sigma is wrong, the projection is broken, or the odds data is stale. The clamp caps the output at 18%, but then proceeds to pass that value through `computeConfidence` with `watchdogStatus` at its default (no degradation), producing a high-confidence PLAY signal on what is genuinely pathological input.

The `EDGE_SANITY_CLAMP_APPLIED` flag is stored in `rail_flags` within the card payload but is never forwarded to the watchdog or cleanliness gate. The play-cleanliness profile for NHL/NHL (`require_watchdog_ok: true`) would block it — but only if `watchdogStatus` were `'CAUTION'`.

This is a one-function fix inside `computeTotalEdge`.

## Detailed Implementation

### Task 1 — Forward clamp event to `confidenceContext`

**File:** `packages/models/src/edge-calculator.js`, inside `computeTotalEdge`

**Current:**

```js
let edge = p_fair - p_implied;
if (isNhlStyleTotal && Math.abs(edge) > 0.18) {
  edge = Math.sign(edge) * 0.18;
  railFlags.push('EDGE_SANITY_CLAMP_APPLIED');
}
const edgePoints = mu - L;

const result = {
  edge: Number(edge.toFixed(4)),
  ...
  confidence: computeConfidence({
    baseConfidence: 0.88,
    ...confidenceContext,
  }),
```

**After:**

```js
let edge = p_fair - p_implied;
let effectiveConfidenceContext = confidenceContext;
if (isNhlStyleTotal && Math.abs(edge) > 0.18) {
  edge = Math.sign(edge) * 0.18;
  railFlags.push('EDGE_SANITY_CLAMP_APPLIED');
  // Force watchdog CAUTION so play-cleanliness gate blocks PLAY classification.
  // A >18% raw edge signals broken sigma or stale odds — do not promote to PLAY.
  effectiveConfidenceContext = {
    ...confidenceContext,
    watchdogStatus: 'CAUTION',
  };
}
const edgePoints = mu - L;

const result = {
  edge: Number(edge.toFixed(4)),
  ...
  confidence: computeConfidence({
    baseConfidence: 0.88,
    ...effectiveConfidenceContext,
  }),
```

This is the complete change. `computeConfidence` already applies a `-0.1` penalty for `watchdogStatus === 'CAUTION'` and the `resolvePlayCleanlinessProfile` for NHL already sets `require_watchdog_ok: true`, so this single line change causes the cleanliness gate to block PLAY promotion for any edge-clamped NHL total.
