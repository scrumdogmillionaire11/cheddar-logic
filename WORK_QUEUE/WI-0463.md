# WI-0463 — Skater injury factor in NHL totals model

## Status
IN PROGRESS

## Goal
Make `predictNHLGame()` aware of high-impact skater injuries so the pace/totals
model reduces expected goals when confirmed-out skaters are missing.

## Background
MoneyPuck enrichment already writes `raw_data.injury_status.home/away` (array of
`{ player, status, detail }`) onto every odds snapshot before model inference.
`predictNHLGame()` has never consumed this data.  A team missing 2–3 top-6
forwards can suppress goals by 7–10%, but the model was blind to it.

## Design
- **`computeSkaterInjuryFactor(injuries)`** (new helper in `models/index.js`):
  Counts confirmed-out skaters (status contains "out", "ir", "ltir", or
  "suspended"). Returns `max(0.88, 1.0 - count × 0.035)` or `null` if no
  confirmed-out players.  Factor is `null` (no-op) when injury data is absent
  (fail-open).
- **`predictNHLGame()` new params** (`nhl-pace-model.js`):
  `homeSkaterInjuryFactor = null`, `awaySkaterInjuryFactor = null`.
  Applied after step 1 (L5 blend) and before step 2 (pace), multiplying
  `homeOffRating` / `awayOffRating`.  Factor recorded in `adjustments.home/away.skater_injury`.

## Constants
| Constant | Value | Notes |
|---|---|---|
| `SKATER_INJURY_FACTOR_PER_OUT` | 0.035 | 3.5% reduction per confirmed-out skater |
| `SKATER_INJURY_FACTOR_MIN` | 0.88 | Cap at 12% total reduction |

## Acceptance criteria
- [ ] 1 confirmed-out skater → factor 0.965
- [ ] 3 confirmed-out → factor 0.895
- [ ] 4 confirmed-out → capped at 0.88
- [ ] 0 confirmed-out → factor `null` (no adjustment)
- [ ] DTD/Questionable statuses are ignored
- [ ] No injury data → factor `null` → model unchanged
- [ ] `adjustments.home.skater_injury` present in output when factor applied
