## Root cause

`ScreenshotParser.extract_slots()` ignores the uploaded image entirely and
returns `_SYNTHETIC_NAMES[slot_index]` — a hardcoded list of 15 made-up player
names. `PlayerRegistry` has only 19 hardcoded players. No real OCR runs.

---

## Tasks

### Task 1 — Bootstrap player registry from FPL API

Replace `_SAMPLE_PLAYERS` in `player_registry.py` with a bootstrap loader:

```python
async def bootstrap_from_fpl_api() -> None:
    """Fetch https://fantasy.premierleague.com/api/bootstrap-static/
    and populate the registry with all current-season players.
    Falls back to _SAMPLE_PLAYERS if the request fails (dev/offline).
    """
```

- Call `GET https://fantasy.premierleague.com/api/bootstrap-static/`
- Extract `elements[].id`, `elements[].web_name`, `elements[].first_name`,
  `elements[].second_name` to build both a `web_name` and `full_name` lookup
- Store as `{ player_id: display_name }` in the registry
- Call `await bootstrap_from_fpl_api()` inside `@app.lifespan` in `main.py`
  (after ProductStore, before server ready)
- If the FPL API is unreachable, log a warning and fall back to `_SAMPLE_PLAYERS`

## Implementation notes

- `pytesseract` requires `tesseract-ocr` binary. Add to `requirements.txt` and
  note in README that `brew install tesseract` / `apt install tesseract-ocr` is
  a system dependency.
- FPL bootstrap API is publicly accessible; no auth token needed.
- `web_name` (e.g. "Salah") scores better in fuzzy match against cropped text
  than `second_name` alone; store both and match against the higher scorer.
- Layout crop regions will need empirical tuning against real device screenshots.
  First pass: assume standard FPL mobile app aspect ratio (9:16 portrait).
