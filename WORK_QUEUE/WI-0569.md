# WI-0569: Market evaluator — execution selector (all markets)

**ID**: WI-0569
**Goal**: Generalize the best-line/best-price execution selector to cover all market types — totals, moneylines, and team totals — not just spreads. Each market should expose `best_line_for_[side]`, `best_price_for_[side]`, and `best_book_for_[side]` separately, as best line and best price can come from different books.

**Background**:
In a prior session we added `spreadHomeBook` / `spreadAwayBook` for spreads. But we collapsed best-line and best-price together (the book with the best spread line also gets its price used). This is wrong: sometimes a different book has a better price on the same line. This WI:
1. Separates best-line selection and best-price selection for spreads (fix the existing impl)
2. Adds execution selectors for totals (best over/under line + best over/under price + books) and moneylines (best home/away price + books)
3. Stores all execution fields in the DB and exposes them in the API

**Scope**:
- `packages/odds/src/market_evaluator.js` (new module, coordinate with WI-0568)
  - `selectBestExecution(entries, marketType)` — returns:
    ```
    spread: {
      best_line_home, best_line_home_book,
      best_line_away, best_line_away_book,
      best_price_home, best_price_home_book,
      best_price_away, best_price_away_book
    }
    total: {
      best_line_over, best_line_over_book,
      best_price_over, best_price_over_book,
      best_price_under, best_price_under_book
    }
    h2h: {
      best_price_home, best_price_home_book,
      best_price_away, best_price_away_book
    }
    ```
  - Best line = max(line) for the favored side (most points for bettor)
  - Best price = max(price in American odds) independently of line — i.e., the book offering the best juice, regardless of which line they have
- `packages/odds/src/normalize.js`
  - Replace the current best-line spread logic with calls to `selectBestExecution`
  - Add total and h2h execution fields to the `odds` object
- `packages/data/db/migrations/039_add_execution_to_odds_snapshots.sql`
  - Add columns for total and h2h execution (spread columns already exist from today's work):
    - `total_best_line_over REAL`, `total_best_line_over_book TEXT`
    - `total_best_price_over INTEGER`, `total_best_price_over_book TEXT`
    - `total_best_price_under INTEGER`, `total_best_price_under_book TEXT`
    - `h2h_best_price_home INTEGER`, `h2h_best_price_home_book TEXT`
    - `h2h_best_price_away INTEGER`, `h2h_best_price_away_book TEXT`
    - `spread_best_price_home_book TEXT`, `spread_best_price_away_book TEXT` (price and line books now separate)
- `packages/data/src/db.js` — `insertOddsSnapshot`
- `apps/worker/src/jobs/pull_odds_hourly.js` + `refresh_stale_odds.js`
- `web/src/app/api/games/route.ts`
- `web/src/components/cards-page-client.tsx` — GameData.odds interface + display

**Out of scope**:
- Consensus computation (WI-0568)
- Misprice detection (WI-0570)
- Team totals and props (low liquidity — defer)

**Acceptance**:
- For a game with 5 books: best-line book and best-price book can differ; both are stored and returned.
- Total execution fields populated: `total_best_line_over`, `total_best_line_over_book`, `total_best_price_over`, `total_best_price_over_book`, `total_best_price_under`, `total_best_price_under_book`.
- ML execution fields populated: `h2h_best_price_home`, `h2h_best_price_home_book`, etc.
- Card display shows best price book for moneyline cards (e.g. "ML: +146 (DraftKings)").
- TypeScript compiles cleanly.
