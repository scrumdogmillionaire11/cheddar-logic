# WI-0562: Isolate mutating web tests to a temporary DB path

**ID**: WI-0562
**Goal**: Prevent web test scripts from mutating whichever DB path happens to be configured in the environment by forcing a dedicated temp test DB.

**Scope**:
- `web/src/__tests__/api-dedupe.test.js`
- `web/src/__tests__/api-games-filter.test.js`
- `web/src/__tests__/api-cards-sport-filter.test.js`
- `web/src/__tests__/api-cards-lifecycle-regression.test.js`
- `web/package.json` (optional: test scripts that set `CHEDDAR_DB_PATH` explicitly)
- `docs/` test runbook (if command contract is documented)

**Out of scope**:
- Production DB schema changes
- Worker job logic changes
- Rewriting tests to pure mocks

**Background**:
Multiple web tests execute `INSERT/DELETE/UPDATE` statements against `card_payloads`, `games`, `game_results`, `run_state`, and `job_runs` via shared DB helpers. Today they use whatever DB path resolves at runtime, which can be dangerous if environment config points to non-ephemeral data.

Examples:
- `web/src/__tests__/api-dedupe.test.js`
- `web/src/__tests__/api-games-filter.test.js`
- `web/src/__tests__/api-cards-sport-filter.test.js`
- `web/src/__tests__/api-cards-lifecycle-regression.test.js`

**Acceptance**:
- Scoped mutating tests always run against a temp/test DB path (for example under `/tmp/cheddar-logic/`), not implicit environment defaults.
- Tests fail fast with a clear safety error if DB path resolves to known production-like locations (for example `/opt/data/`).
- Test setup/teardown remains idempotent and leaves no residual rows in shared dev DBs.
- README/runbook documents the enforced DB-path behavior for local and CI runs.

- Run scoped tests with a temp path and confirm they pass and clean up test rows.
