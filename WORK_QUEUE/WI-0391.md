# WI-0391: Active Plays Toggle + In-Progress Tracking

**ID**: WI-0391
**Goal**: Add a user-facing Active Plays toggle and canonical in-progress play lifecycle so cards visibility is controlled by explicit state, not implicit time-based filtering.

**Scope**:

- `WORK_QUEUE/WI-0391.md`
- `web/src/components/cards-page-client.tsx`
- `web/src/app/api/games/route.ts`
- `web/src/app/api/cards/route.ts`
- `web/src/app/api/cards/[gameId]/route.ts`

**Out of scope**:

- Settlement grading logic (`apps/worker/src/jobs/settle_pending_cards.js`)
- ESPN/final score ingestion (`apps/worker/src/jobs/settle_game_results.js`)
- Historical backfill/resettlement of existing records

**Acceptance**:

- Cards UI exposes an explicit toggle to switch between active-only and full-history card visibility.
- API responses support the toggle via an explicit lifecycle filter (not `expires_at` and not raw game start-time cutoffs).
- Settled games remain excluded from active-only views while remaining accessible when full-history mode is selected.
- Behavior is consistent between `/api/games`, `/api/cards`, and `/api/cards/[gameId]`.
- Add regression coverage for at least one settled game and one in-progress game path.
