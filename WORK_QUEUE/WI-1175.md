# WI-1175: Suppress Same-Game POTD Near-Miss Line Variants

**ID**: WI-1175
**Goal**: Prevent POTD near-miss tracking from displaying stale or alternate line variants from the same game and market as the official POTD play.

**Context**:
- Production on 2026-04-24 showed official POTD `San Antonio Spurs @ Portland Trail Blazers UNDER 219.5`.
- The near-miss pending list also showed `San Antonio Spurs @ Portland Trail Blazers UNDER 220.5`.
- Current near-miss grouping includes line value, so `NBA|same-game|TOTAL|219.500` and `NBA|same-game|TOTAL|220.500` are treated as different groups.
- This lets stale or alternate same-game lines survive as near misses after the official winner is excluded.

**Scope**:
- `apps/worker/src/jobs/potd/run_potd_engine.js`
- `apps/worker/src/jobs/potd/__tests__/run-potd-engine.test.js`

**Out of scope**:
- Historical production DB cleanup or backfill.
- Changing POTD official-play selection or Kelly sizing.
- Changing settlement identity keys for `potd_shadow_candidates`.
- UI redesign of the POTD page.

**Acceptance**:
- POTD shadow-candidate suppression uses canonical game identity, not raw `game_id` alone.
- If `game_id` is missing, candidates are matched by normalized sport + play/game date + normalized home/away teams.
- Suppression is same sport + same canonical game + same normalized market, regardless of side or line.
- `candidate_identity_key` remains settlement-safe and line-specific, e.g. `sport|game|market|selection|line`.
- Market keys are normalized before comparison.
- Same-day reruns purge/replace stale same-game/same-market shadow candidates from prior runs before new rows are persisted.
- Near-miss dedupe still keeps the best candidate for non-winning games/markets.
- Tests cover game-id-vs-team-derived matching, totals side suppression, spread side suppression, market alias normalization, and stale-row rerun cleanup.

- If moneyline candidates are eligible in this engine path, MONEYLINE winner suppresses opposing MONEYLINE side near-miss for same canonical game.
- Same-day rerun removes/replaces stale same-game/same-market shadow rows seeded from prior run before persisting current rows.
