# WI-1116: Results Surface Mapping for Confidence Buckets and Fragile Directions

**ID**: WI-1116
**Goal**: Upgrade `/results` so each displayed confidence bucket is transparently mapped to the underlying attribution fields, and clarify how weak/fragile directions relate to bucket records.

**Scope**:

- `web/src/app/results/page.tsx`
- `web/src/components/results/ProjectionResultsTable.tsx`
- `web/src/__tests__/ui-results-smoke.test.js`

**Out of scope**:

- Backend confidence score formula changes
- New market families or settlement pipeline changes
- Altering historical grading outcomes

**Depends on**:

- `WI-1115`

**Acceptance**:

- Projection Confidence Engine area shows bucket mapping context with explicit threshold text for LOW/WATCH/TRUST/STRONG and a weak-direction policy note.
- The UI displays explicit wording that weak directions (`edge_distance < 0.15`) are excluded from directional W/L and still included in MAE/bias auditing.
- If `FRAGILE` is presented in UI, it is explicitly defined as a presentation label tied to weak/no-edge logic and not treated as a native confidence band.
- `/results` renders at least one card-level attribution detail using API-provided fields (`projection_raw`, `synthetic_line`, `edge_distance`, `confidence_band`) so bucket placement is inspectable.
- UI remains responsive and readable on mobile and desktop.
