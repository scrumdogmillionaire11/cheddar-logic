# WI-0762: Build `pull_public_splits.js` job to populate Action Network public betting columns in `odds_snapshots`

| Field              | Value |
|--------------------|-------|
| **ID**             | WI-0762 |
| **Goal**           | Build a `pull_public_splits.js` pull job that fetches public betting percentages from the Action Network API and writes into the 8 stub columns added by migration 055, then wire the splits into the NHL/NBA model consensus signal |
| **Priority**       | High (schema stub with permanently-NULL columns affects model signal quality) |
| **Owner agent**    | Unassigned |
| **Time window**    | TBD |
| **Coordination flag** | `needs-sync` — touches scheduler registration in `main.js` and model signal wiring |

---

## Problem

Migration `055_add_public_splits_to_odds_snapshots.sql` added 8 columns to `odds_snapshots`:

```
public_bets_pct_home, public_bets_pct_away
public_handle_pct_home, public_handle_pct_away
public_tickets_pct_home, public_tickets_pct_away
splits_source, splits_captured_at
```

**No pull job has ever written to any of these columns.** Zero references exist in `apps/`, `packages/data/src/`, or `web/src/`.

**Impact:**
- All 8 columns are permanently NULL
- The intended consensus signal (public vs sharp divergence) is completely absent from the NHL and NBA models
- Migration 055 has been shipping in schema for unknown number of releases with no data

---

## Scope

| File | Change |
|------|--------|
| `apps/worker/src/jobs/pull_public_splits.js` | New job — fetch Action Network public betting % for active NHL/NBA/(optionally MLB) games; upsert `public_bets_pct_home`, `public_bets_pct_away`, `public_handle_pct_home`, `public_handle_pct_away`, `public_tickets_pct_home`, `public_tickets_pct_away`, `splits_source='action_network'`, `splits_captured_at=NOW()` into `odds_snapshots` matched by `external_game_id` or `(sport, home_team, away_team, game_date)` |
| `apps/worker/src/schedulers/main.js` | Register `pull_public_splits` in the pre-model window (after odds pull, before model run) |
| `apps/worker/src/jobs/run_nhl_model.js` | Read `public_bets_pct_home/away` from `odds_snapshots` and incorporate as a divergence flag into consensus classification |
| `apps/worker/src/jobs/run_nba_model.js` | Same as NHL — wire public splits divergence signal |
| `packages/adapters/src/action-network.js` | Add `fetchPublicSplits(sport, gameIds)` method; use documented public endpoint (see `docs/parsers/action_network_public_splits.md`) |
| `packages/adapters/src/__tests__/action-network.test.js` | Add tests for `fetchPublicSplits` normalization |
