# WI-0492: Soccer Model Phase 2 — Edge Repair + CLV Telemetry

**ID**: WI-0492
**Goal**: Replace circular xG edge calculation with real `model_prob - implied_prob`. Wire CLV ledger for Tier 1 odds-backed cards. First actionable edge signal.

**Depends on**:

- WI-0491 (xG foundation, Phase 1 complete)

**Scope**:

- `apps/worker/src/jobs/run_soccer_model.js` (replace `edge_ev` calculation for Track 1 markets)
- `apps/worker/src/jobs/settle_soccer_clv.js` (new — post-game CLV settlement job)
- `packages/data/src/db.js` (add CLV settlement functions; `clv_ledger` table already exists from WI-0485)
- `apps/worker/package.json` (add `job:settle-soccer-clv` npm script)
- `apps/worker/src/schedulers/main.js` (queue CLV settlement job post-game)
- `env.example` (add `ENABLE_SOCCER_CLV_SETTLEMENT=false` flag)

**Out of scope**:

- League-specific thresholds (Phase 3)
- Away-win hard caps (Phase 3)
- Projection win-rate tracking (Phase 4)
- Prop model expansion (Phase 5)

**Acceptance**:

1. **Edge calculation repair:** For all `soccer_ml`, `soccer_game_total`, and `soccer_double_chance` cards in Track 1:
   - Pull `home_xg`, `away_xg` from `soccer_team_xg_cache`
   - Call `computeXgWinProbs()` from `xg-model.js` → `{ homeWin, draw, awayWin }`
   - Fetch `implied_prob` from Odds API snapshot (vig-removed)
   - Calculate `edge_pct = model_prob - implied_prob` (where `model_prob` matches pick side)
   - **Result:** edge distribution is no longer always-positive; ~40-60% of cards should have negative edge

2. **Model confidence field:** Add to all Track 1 `payloadData`:
   - `FULL`: xG cache hit (home_xg ≥ 0.5 and away_xg ≥ 0.5)
   - `PARTIAL`: one team xG missing (use league average as fallback)
   - `LOW`: both teams xG missing (fall back to vig-removed `implied_prob` as fair prob, log WARNING)

3. **CLV ledger wiring:**
   - On every Tier 1 odds-backed card insert into `card_payloads`: call `recordClvEntry(card_id, game_id, market_key, pick_side, displayed_odds, card_created_at)`
   - `settle_soccer_clv.js` runs post-game (configurable schedule, e.g., T+24h): fetch closing odds from Odds API, call `settleClvEntry()`, compute CLV implied probability difference
   - Test with `ENABLE_CLV_LEDGER=true` flag

4. **No breaking changes:** Existing card payloads unchanged except for `edge_ev` value and new `model_confidence` field. All existing decision routing unaffected.

5. **Zero negative CLV on home picks:** Once 30+ settled plays per market per league: validate average CLV >= +0.05% (breakthrough efficiency; expect +1.5% long-term per spec)
