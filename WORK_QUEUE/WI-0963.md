# WI-0963: Unify worker->web decision authority contract for MLB cards

**ID**: WI-0963
**Goal**: Remove split-brain decision semantics between worker payload fields and web transform fallbacks by enforcing one canonical decision envelope path for MLB cards.

**Depends on**: none

**Architecture constraints (must hold)**:

- ADR-0004 governs this WI: worker-owned decision_v2 remains the only decision authority; web/API/UI consume canonical persisted fields and must not recompute MLB verdicts when canonical fields are present.
- Canonical MLB rows must flow through decision_v2.canonical_envelope_v2 first for official status, primary reason, and selection semantics.
- Any compatibility fallback is allowed only for legacy MLB rows where canonical envelope fields are absent; this WI must not introduce any new fallback heuristic or second authority path.
- Scope is intentionally partitioned to avoid active-file conflicts: apps/worker/src/jobs/run_mlb_model.js remains with WI-0964, and web/src/lib/game-card/transform/decision-surface.ts reason-surface refinements remain with WI-0965.

**Out of scope**:
- MLB model threshold tuning
- Scheduler cadence/freshness policy changes
- Non-MLB market decision redesign

**Acceptance**:
- Web rendering no longer depends on legacy fallback chains when canonical decision_v2.canonical_envelope_v2 is present.
- Worker output for MLB cards guarantees canonical envelope completeness for official status, reason code, and selection semantics.
- Regression tests prove authoritative totals and moneyline rows preserve surfaced status/selection without transform-time re-scoring.
- Any remaining fallback path is explicitly limited to legacy rows and covered by tests.

**Execution plan**:
- Phase 1: Harden the worker-side contract only in apps/worker/src/utils/decision-publisher.js. Ensure MLB cards that already carry decision_v2 publish a complete decision_v2.canonical_envelope_v2 for official status, primary reason, and surfaced selection semantics. Add targeted unit assertions in apps/worker/src/utils/__tests__/decision-publisher.v2.test.js. Do not edit apps/worker/src/jobs/run_mlb_model.js in this WI.
- Phase 2: Hard-cut web consumers to canonical persisted fields for MLB rows in web/src/lib/games/route-handler.ts and web/src/lib/game-card/transform/index.ts. When canonical envelope fields exist, prefer them over legacy action/status/classification fallback chains. Any legacy fallback path must be explicitly gated to rows missing canonical envelope fields.
- Phase 3: Add regression coverage proving authoritative MLB totals and moneyline rows preserve intended surfaced status/selection, and that legacy fallback remains available only when canonical envelope is absent.
