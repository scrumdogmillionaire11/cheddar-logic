**Acceptance**:

1. **Database schema:** `soccer_team_xg_cache` table created with columns: `sport`, `league`, `team_name`, `home_xg_l6`, `away_xg_l6`, `fetched_at`, `cache_date`. CRUD functions exported from `db.js`.

2. **Data ingestion:** `pull_soccer_xg_stats.js` successfully pulls rolling 6-game xG from FBref via `soccerdata` Python library for EPL, MLS, UCL. Weighted recency applied (most recent game = 2x weight). Fail-open behavior: on FBref unavailable, logs WARNING and continues (xG cache remains NULL-able for Phase 2 fallback).

3. **Poisson model:** `xg-model.js` exports pure functions with full test coverage:
   - `poissonPmf(k, lambda)` — PMF accurate to 4 decimal places
   - `computeXgWinProbs({ homeXg, awayXg, league })` → `{ homeWin, draw, awayWin }` summing to 1.0 (within 0.001 tolerance)
   - `computeXgTotalProb({ homeXg, awayXg, totalLine, direction })` → probability(over/under)
   - `applyLeagueHomeAdj(xg, league)` → adjusted xG (+0.12 EPL, +0.09 MLS, +0.10 UCL per spec)
   - `getLeagueSigma(league)` → sigma value (1.18 EPL, 1.24 MLS, 1.15 UCL)

4. **Scheduler integration:** xG pull queued before every soccer model run (same cadence as `pull_soccer_player_props.js`). Job key idempotency applied.

5. **Placeholder field:** `model_confidence` field added to `run_soccer_model.js` payloadData skeleton (value set to null pending Phase 2; no breaking changes to existing payloads).

6. **Zero breaking changes:** All existing soccer card payloads unchanged. Decision pipeline routing unaffected. No changes to `run_soccer_model.js` decision logic.
