**Implementation Notes**:

This fix requires choosing between two approaches:

#### Option A: Normalize to uppercase (recommended)

- Matches application hardcoding in model jobs (sport: 'NBA')
- Matches user-facing data expectations
- Modify `insertCardPayload()` to call `normalizeSportValue()` OR use `sport.toUpperCase()`
- Migrate existing lowercase cards: `UPDATE card_payloads SET sport = UPPER(sport);`
- Update API query to uppercase: `const sport = sportParam ? sportParam.toUpperCase() : null;` (already exists, just ensure consistency)

#### Option B: Normalize to lowercase (alternative)

- Matches odds_snapshots and games tables (already lowercase)
- Modify model jobs to pass lowercase sport: `generateCard({ sport: 'nba', ... })`
- Modify API query to lowercase: `const sport = sportParam ? sportParam.toLowerCase() : null;`
- Migrate existing uppercase cards: `UPDATE card_payloads SET sport = LOWER(sport);`

**Recommended**: Option A (uppercase) — simpler change, requires only db.js modification and data migration.
