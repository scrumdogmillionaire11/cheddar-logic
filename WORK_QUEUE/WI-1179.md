# WI-1179: Add Modern MLB/NHL Model Payload Compatibility in Data Extractors

**ID**: WI-1179
**Goal**: Make POTD model snapshot reads compatible with modern MLB/NHL payload schemas while preserving legacy schema support.

**Context**:

- POTD currently reads MLB/NHL model snapshots through `packages/data/src/db/cards.js`.
- Production payloads now include modern top-level model fields for MLB and PASS/evidence NHL rows that can be non-actionable.
- Existing extractor logic primarily targets legacy schema keys and can return null despite actionable modern data.

**Scope**:

- `packages/data/src/db/cards.js`
- `packages/data/__tests__/card-payload-sport.test.js`
- `WORK_QUEUE/WI-1179.md`
- `.planning/ROADMAP.md`
- `.planning/STATE.md`
- `.planning/phases/WI-1179/**`

**Out of scope**:

- Changes to `apps/worker/src/jobs/potd/run_potd_engine.js` fire/publish orchestration.
- Changes to `EDGE_SOURCE_CONTRACT`, `edgeSourceTag`, or `isModelBackedCandidate` semantics.
- NHL producer writes (handled in a follow-up WI).

**Acceptance**:

- `getLatestMlbModelOutput(gameId)` supports modern top-level schema (`model_prob|p_fair`, `edge`, `selection.side`, optional `price`, `line`, `market_type`) and legacy `drivers[0]` schema.
- `getLatestMlbModelOutput(gameId)` prefers modern actionable values when both modern and legacy structures exist.
- `getLatestNhlModelOutput(gameId)` returns actionable model signal data only when payload is non-PASS and contains finite model probability inputs required by the POTD consumer.
- PASS/evidence NHL rows without actionable model inputs are treated as non-actionable (extractor returns null or equivalent non-actionable result).
- Existing extractor tests remain green and new tests cover modern MLB schema and NHL PASS/evidence non-actionable behavior.
