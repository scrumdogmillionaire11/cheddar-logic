# WI-0779: Consolidate dual type locations in web/ (web/src/types vs web/src/lib/types)

| Field              | Value |
|--------------------|-------|
| **ID**             | WI-0779 |
| **Goal**           | Merge the two parallel type directories in `web/` into a single authoritative structure: ambient `.d.ts` stubs stay in `web/src/types/`, all runtime types move to `web/src/lib/types/`, and enforce the split via barrel exports |
| **Priority**       | Low-Medium |
| **Owner agent**    | Unassigned |
| **Time window**    | TBD |
| **Coordination flag** | `needs-sync` — modifies `web/src/lib/types/**` which is a serialized shared touchpoint |

---

## Problem

`web/` has two active type directories with overlapping concerns:

- `web/src/types/` — contains `cheddar-logic-data.d.ts` and `cheddar-models.d.ts` (ambient declaration stubs for CJS packages)
- `web/src/lib/types/` — contains runtime TypeScript types: `canonical-play.ts`, `game-card.ts`, and related files

There is no enforced rule on which directory to use for new types. Runtime types have ended up mixed alongside ambient declarations; contributors adding new types have no clear guidance on where they belong.

---

## Scope

| File | Change |
|------|--------|
| `web/src/types/` | Keep only ambient `.d.ts` declaration files (stubs for CJS packages) |
| `web/src/lib/types/` | Canonical home for all runtime TypeScript types; ensure an `index.ts` barrel exports all public types |
| `web/src/lib/types/index.ts` | Create or update to re-export all types in the directory |
| Any `web/src/` import that uses a wrong path | Update imports to the canonical path |

## Out of Scope

- Changing type definitions or adding new fields to existing types
- Migrating the CJS ambient declarations to ESM (separate ESM migration concern)
- Changes to `packages/` types

---

## Acceptance

- `web/src/types/` contains only `.d.ts` files
- `web/src/lib/types/index.ts` exists and exports all runtime types
- `grep -r "from '.*web/src/types/[^d]" web/src/` returns zero results (no imports of runtime types from the ambient dir)
- `cd web && npm run build` succeeds with no new TypeScript errors

- `ls web/src/types/` shows only `*.d.ts` files
- `ls web/src/lib/types/` includes an `index.ts` barrel
