# WI-0755: mlb-f5 projection card ŌĆö render raw projection value with MAE-friendly display

**ID**: WI-0755
**Goal**: Replace the blank zone on `mlb-f5` PROJECTION_ONLY cards with a clean projection display showing the model's projected F5 combined runs, an explicit "no market line" label, and no direction badge.

**Background**:
`mlb-f5` cards with `basis=PROJECTION_ONLY` have no market line ŌĆö no odds are fetched for F5 totals. The model emits a projected combined F5 runs total (e.g., "4.2 runs"). Accuracy is measured as MAE (mean absolute error) against actual F5 result after game completion ŌĆö not as a binary hit/miss against a line.

The card currently shows a blank zone. The display must not imply a bet direction (no OVER/UNDER badge) and must not show a line or odds field. The label should make clear this is a model output only.

**Scope**:
- `web/src/components/cards/GameCardItem.tsx` ŌĆö add `isMlbF5Projection` branch in card body render
- `web/src/components/cards/game-card-helpers.tsx` ŌĆö add `formatF5ProjectionBlock` helper if needed

**Out of scope**:
- Any changes to the worker / model / card_payloads
- Results tracking, MAE calculation, or actual result ingestion (WI-0756, WI-0757)
- nhl-pace-1p display (WI-0754)
- Inferring a line from the projection value (explicitly forbidden ŌĆö see design decision in party mode discussion)

**Acceptance**:
- A card with `cardType === 'mlb-f5'` and `basis === 'PROJECTION_ONLY'` (or `execution_status === 'PROJECTION_ONLY'`) renders a projection block containing:
  - Label: `F5 Total ŌĆö Model Projection`
  - Projected value: `Projected: X.X runs`
  - Sub-label: `No market line ┬Ę MAE tracking`
  - No line field, no direction badge, no bet odds, no FIRE/WATCH/PASS badge
- If an `mlb-f5` card ever has an odds-backed line in the future (execution_status ŌēĀ PROJECTION_ONLY), it falls through to the normal render path unchanged
- `npm --prefix web run lint` passes on changed files
