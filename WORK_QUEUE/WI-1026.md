# WI-1026: NBA Phase 3C — Market intelligence utilization

**ID**: WI-1026
**Goal**: Wire `sharp_divergence` and `splits_divergence` into confidence and tier computation so market-intelligence conflicts reduce play quality and market-intelligence alignment can provide a small confirmation boost. This layer adjusts confidence only; it does not change projection math or pricing logic.

**Depends on**: WI-1025 (regime detection active)

**Priority note**: This is the lowest-leverage WI in the program. Projection quality remains primary. Market intelligence may refine confidence, but it must never compensate for weak projection quality or poor EV logic. ## Acceptance

### 1) New market-intelligence modifier function

Add:

```js id="nba6r1"
applyMarketIntelligenceModifier({
  baseConfidence,
  sharpDivergence,
  splitsDivergence,
  edge,
})
```

Returns:

```js id="nba6r2"
{
  adjustedConfidence,
  multiplier,
  reasonCodes,
}
```

Where:

* `adjustedConfidence` is the final confidence after clamp
* `multiplier` is the single applied market-intelligence multiplier
* `reasonCodes` is an array of applied reasons

### 2) Modifier rules

Apply these rules exactly:

#### `SHARP_VS_PUBLIC`

If:

```js id="nba6r3"
sharpDivergence === 'SHARP_VS_PUBLIC'
```

Then:

* `multiplier = 0.85`
* add reason code: `SHARP_VS_MODEL_CONFLICT`

#### `PUBLIC_TRAP_RISK`

If:

```js id="nba6r4"
(splitsDivergence === 'PUBLIC_HEAVY_HOME' || splitsDivergence === 'PUBLIC_HEAVY_AWAY') &&
edge < 0.04
```

Then:

* `multiplier = 0.88`
* add reason code: `PUBLIC_TRAP_RISK`

#### `SHARP_CONFIRMATION`

If:

```js id="nba6r5"
sharpDivergence === 'SHARP_ALIGNED'
```

Then:

* `multiplier = 1.05`
* add reason code: `SHARP_CONFIRMATION`

Rules:

* this is a small boost only
* final confidence must still be capped at `0.90`

### 3) Combination logic

If multiple signals apply:

* Use **one multiplier only**
* Do **not** multiply them together
* Choose the **most conservative** applicable multiplier

Examples:

* `SHARP_VS_PUBLIC` + `PUBLIC_TRAP_RISK` → use `0.85`
* `SHARP_ALIGNED` + no risk signals → use `1.05`
* risk signal + alignment signal together → use the lower-risk multiplier, not both

This prevents compounded overreaction.

### 4) Confidence clamp

Final adjusted confidence must be clamped:

* floor: `0.45`
* ceiling: `0.90`

Implementation:

```js id="nba6r6"
adjustedConfidence = clamp(baseConfidence * multiplier, 0.45, 0.90)
```

### 5) Tier interaction

This modifier affects only:

* `confidence`
* tier outcome derived from confidence

This modifier must **not** affect:

* `edge`
* `p_fair`
* `ev_passed`
* `recommend_bet_type`

### 6) Raw data stamping

Stamp on the card payload:

```js id="nba6r7"
raw_data.market_intel_modifier = {
  multiplier,
  reason_codes,
}
```

If no signals apply:

```js id="nba6r8"
{
  multiplier: 1.0,
  reason_codes: [],
}
```

### 7) Application point in `run_nba_model.js`

Apply market-intelligence modification **after** base confidence is computed from model / pricing logic and **before** tier is finalized from confidence.

Flow:

```js id="nba6r9"
const baseConfidence = computePricedCallCardConfidence(...)

const marketIntel = applyMarketIntelligenceModifier({
  baseConfidence,
  sharpDivergence: oddsSnapshot?.raw_data?.sharp_divergence ?? null,
  splitsDivergence: oddsSnapshot?.raw_data?.splits_divergence ?? null,
  edge,
})

const finalConfidence = marketIntel.adjustedConfidence
```

Then use `finalConfidence` for tier mapping.

### 8) Unit tests

Cover:

1. `SHARP_VS_PUBLIC`

   * `edge = 0.06`
   * confidence reduced by ~15%
   * reason includes `SHARP_VS_MODEL_CONFLICT`

2. `PUBLIC_TRAP_RISK`

   * `splitsDivergence = 'PUBLIC_HEAVY_HOME'`
   * `edge = 0.03`
   * confidence reduced by ~12%
   * reason includes `PUBLIC_TRAP_RISK`

3. `SHARP_CONFIRMATION`

   * small boost applied
   * final confidence capped at `0.90`

4. Combined risk signals

   * `SHARP_VS_PUBLIC` + `PUBLIC_HEAVY_HOME` + `edge = 0.03`
   * multiplier = `0.85`
   * not `0.85 * 0.88`

5. No signals

   * multiplier = `1.0`
   * empty reason codes

6. Floor clamp

   * low confidence with negative multiplier effect
   * final confidence not below `0.45`

7. Raw data stamp

   * `market_intel_modifier` exists with multiplier and reasons
