# WI-0579 — 1P Cards Reuse Full-Game V2 Anomaly Flag Against Wrong Mu

**ID:** WI-0579
**Goal:** Run `projectSogV2` independently for 1P cards rather than reusing the `v2AnomalyDetected` flag from the full-game run. The full-game `v2AnomalyDetected` is computed against a full-game `mu` (e.g. 3.2 SOG); the 1P card uses a scaled mu (~2.0), so the anomaly check is comparing the wrong reference value.
**Severity:** HIGH
**Owner agent:** unassigned
**Status:** OPEN
**Time window:** —
**Coordination flag:** none

## Problem

In `run_nhl_player_shots_model.js`, the 1P card block reuses `v2AnomalyDetected` that was set during the full-game projection pass. `projectSogV2` returns an anomaly flag when `|mu - v2_mu| / sigma > threshold`. The full-game `mu` is typically 2.8–3.5; the 1P `mu` is scaled to ~55–60% of that (~1.6–2.0). Using the full-game anomaly flag for the 1P pricing decision means:

1. A legitimate full-game anomaly suppresses a valid 1P card.
2. A legitimate 1P anomaly is invisible because the full-game pass was clean.

## Scope

- `apps/worker/src/jobs/run_nhl_player_shots_model.js` — compute a fresh `v2AnomalyDetected_1p` (or `v2Result_1p`) by calling `projectSogV2` with the 1P mu and 1P line.
- `apps/worker/src/jobs/__tests__/run_nhl_player_shots_model.test.js` — add a test that full-game anomaly does NOT propagate to 1P card when 1P is clean.

## Out of Scope

- Changes to `projectSogV2` model math.
- Changes to full-game card logic.
- Any other model jobs.

## Acceptance

- [ ] 1P card block calls `projectSogV2(mu_1p, line_1p, sigma_1p, ...)` separately from the full-game call.
- [ ] `v2AnomalyDetected` for the full-game run is not used in the 1P pricing block.
- [ ] Test: full-game anomaly=true + 1P anomaly=false → 1P card NOT suppressed.
- [ ] Test: 1P anomaly=true → 1P pricing nullified regardless of full-game state.
- [ ] All existing tests pass.
