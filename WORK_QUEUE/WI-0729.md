# WI-0729: Create model audit rollout runbook

**ID**: WI-0729
**Goal**: Write the operator manual for the audit system: how to run it locally, how to approve a baseline, how to quarantine a card family, what counts as model decay vs code drift, and what happens when audit and performance disagree.

**Scope**:
- `WORK_QUEUE/WI-0729.md`
- `docs/model_audit_rollout_runbook.md` — new file

**Out of scope**:
- Changes to any audit tooling or fixture code
- Changes to CI config
- Changes to any runbook outside `docs/model_audit_rollout_runbook.md`

**Acceptance**:
- `docs/model_audit_rollout_runbook.md` exists and covers all six sections:
  1. How to run audit locally (`audit:all`, `audit:scorecard`, per-family)
  2. How to approve a baseline (review diff → set `baseline_reviewed: true` → add `_baseline_change_note`)
  3. How to quarantine a card family (set `card_family_status: QUARANTINED` in family registry, add reason + date)
  4. What counts as model decay vs code drift (decision rubric)
  5. What to do when audit passes but performance is collapsing (escalation path)
  6. CI enforcement: what failures are blocking, what requires human override
- The runbook references the family registry file (WI-0730) by path.
- The runbook references the scorecard command (WI-0731) by name.
