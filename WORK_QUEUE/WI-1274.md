# WI-1274: Remove Spurious Workspace `package-lock.json` Files and Consolidate to Root Lockfile

## ID
WI-1274

## Goal
Remove incorrectly tracked per-workspace `package-lock.json` files, regenerate the root `package-lock.json` to cover all workspaces, and update CI to install workspace dependencies via root `npm ci`.

## Background
The original concern that `packages/models/node_modules` was committed was false. `node_modules/` is already ignored and not tracked.

The real issue is that five workspace-level `package-lock.json` files are tracked alongside the root lockfile. In an npm workspaces monorepo, workspace lockfiles are not used by root installs and can diverge from the root lockfile over time.

**Known pre-existing state:** The root `package-lock.json` is currently out of sync with workspace dependencies — `npm ci` from root fails today with `Missing: uuid@9.0.1 from lock file`. This is because `packages/adapters` and `apps/worker` declare `uuid@^9.0.0`, which the root lockfile does not resolve. Running `npm install` from root (first step of this WI) will regenerate the root lockfile to cover all workspace dependencies. This lockfile regeneration is **expected and required** — it is not an out-of-scope dependency upgrade.

## Tracked workspace lockfiles to remove
- `packages/models/package-lock.json`
- `packages/data/package-lock.json`
- `packages/adapters/package-lock.json`
- `packages/odds/package-lock.json`
- `apps/worker/package-lock.json`

## Scope
- Remove the five listed workspace lockfiles from git
- Regenerate root `package-lock.json` via `npm install` to cover all workspace deps
- `.github/workflows/ci.yml`
- `.gitignore`
- `WORK_QUEUE/WI-1274.md`

## Out of scope
- Manually upgrading any version specifier in any `package.json` file
- Package architecture changes
- Runtime model/worker logic changes
- Removing or modifying `web/package-lock.json`
- Removing or modifying `node_modules/` ignore rules

## Required execution order

1. Delete the five workspace lockfiles from disk (do not `git rm` yet — do that after `npm install`)
2. Run `npm install` from repo root — this regenerates the root `package-lock.json` to include all workspace dependencies
3. Verify `npm ci` now succeeds from root
4. `git rm` the five workspace lockfiles
5. Add `.gitignore` guards
6. Update `.github/workflows/ci.yml`
7. Commit all changes

## Required CI behavior

**Workspace install jobs** (`validate`, `worker-tests`):
- Replace all `npm --prefix packages/<pkg> ci` and `npm --prefix apps/worker ci` steps with a single root `npm ci`
- `cache-dependency-path` must point to root `package-lock.json` only

**Web-scoped install jobs** (`validate`, `web-results-performance-budget`):
- These jobs run `npm --prefix web run test:ci:smoke` and `npm --prefix web run test:api:results:performance-budget`
- Keep `npm --prefix web ci` for any job that invokes web scripts, since `web/` has its own install surface
- `web/package-lock.json` stays in `cache-dependency-path` for those jobs only

## `.gitignore` guard

`apps/` contains only `worker/` (confirmed). Add these guards:

```gitignore
/packages/models/package-lock.json
/packages/data/package-lock.json
/packages/adapters/package-lock.json
/packages/odds/package-lock.json
/apps/worker/package-lock.json
```

- Root `.gitignore` prevents those exact lockfiles from being re-added.

- CI workspace jobs install dependencies through root `npm ci`.

- The root `package-lock.json` changes are limited to resolving workspace dependencies that were previously only in workspace lockfiles (e.g., `uuid@9.x` for `packages/adapters` and `apps/worker`). If the diff shows changes outside workspace dependency resolution, stop and explain.
