# WI-0629: Test suite for settle_mlb_f5.js

**ID**: WI-0629
**Goal**: Add a test file for `settle_mlb_f5.js` covering `gradeF5Card`, `fetchF5Total` (mocked), and the per-game gamePk lookup path (including the doubleheader fix from WI-0626).

**Background**:
`apps/worker/src/jobs/settle_mlb_f5.js` is 235 lines with non-trivial grading logic and an MLB API fetch, but has zero test coverage. `gradeF5Card` is a pure function and is highly testable. The per-game gamePk lookup is the site of the correctness bug in WI-0626; tests here will prevent regressions.

**Scope**:
- `apps/worker/src/jobs/__tests__/settle_mlb_f5.test.js` (new) — unit tests for:
  - `gradeF5Card`: OVER/UNDER win/loss/push/null cases, edge cases around push threshold
  - `fetchF5Total`: mock fetch responses for complete game, incomplete game (<5 innings), API failure
  - `settleMlbF5` integration path (in-memory DB): pending F5 card is settled correctly; non-F5 cards are skipped; cards with no gamePk map entry are counted as failed

**Out of scope**:
- Database migration tests (WI-0626 scope)
- Live MLB API calls

**Acceptance**:
- `npm --prefix apps/worker test -- settle_mlb_f5` passes.
- `gradeF5Card` test cases cover: OVER wins, OVER loses, UNDER wins, UNDER loses, push (exact), push (within 0.05), PASS prediction returns null.
- `fetchF5Total` is tested with mocked fetch for normal and error cases.
