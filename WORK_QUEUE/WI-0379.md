## Design

### `CanonicalGoalieState` — field spec

```js
/**
 * @typedef {Object} CanonicalGoalieState
 *
 * @property {string}       game_id          - The game this state applies to
 * @property {'home'|'away'} team_side        - Which team
 *
 * @property {'CONFIRMED'|'EXPECTED'|'UNKNOWN'|'CONFLICTING'} starter_state
 *   - CONFIRMED:   authoritative source confirms this starter for this game
 *   - EXPECTED:    named starter is resolved but not officially confirmed
 *   - UNKNOWN:     no starter resolved (no name, no reliable source)
 *   - CONFLICTING: two or more sources disagree and cannot be reconciled
 *
 * @property {'USER_INPUT'|'SCRAPER_NAME_MATCH'|'SEASON_TABLE_INFERENCE'|'MERGED'} starter_source
 *   - The highest-authority source that produced this state
 *
 * @property {string|null}  goalie_name      - Display name of the resolved starter, or null
 *
 * @property {'ELITE'|'STRONG'|'AVERAGE'|'WEAK'|'UNKNOWN'} goalie_tier
 *   - Statistical quality tier; UNKNOWN when name is unresolved or metrics missing
 *
 * @property {'HIGH'|'MEDIUM'|'LOW'|'NONE'} tier_confidence
 *   - Confidence in the goalie_tier assignment
 *   - HIGH:   name confirmed + metrics join succeeded
 *   - MEDIUM: name expected + metrics join succeeded
 *   - LOW:    name resolved but metrics missing or partial
 *   - NONE:   no goalie identity; tier is UNKNOWN
 *
 * @property {'FULL'|'DEGRADED'|'NEUTRALIZED'|'BLOCKED'} adjustment_trust
 *   - FULL:        CONFIRMED starter; full goalie tier math allowed
 *   - DEGRADED:    EXPECTED starter; math runs at reduced weight
 *   - NEUTRALIZED: UNKNOWN starter; no directional adjustment applied
 *   - BLOCKED:     CONFLICTING state; no adjustment; official totals ineligible
 *
 * @property {string[]}     evidence_flags
 *   - Audit trail of flags accumulated during arbitration and resolution
 *   - Examples: 'CONFLICTING_SOURCE_EVIDENCE', 'METRICS_JOIN_FAILED',
 *               'SEASON_TABLE_INFERENCE_ONLY', 'STALE_USER_INPUT'
 */
```

### Derivation rules for `adjustment_trust`

These rules are fixed and must be implemented in `makeCanonicalGoalieState()`:

| `starter_state` | `tier_confidence` | `adjustment_trust` |
|-----------------|------------------|--------------------|
| CONFIRMED       | HIGH or MEDIUM   | FULL               |
| CONFIRMED       | LOW or NONE      | DEGRADED           |
| EXPECTED        | any              | DEGRADED           |
| UNKNOWN         | any              | NEUTRALIZED        |
| CONFLICTING     | any              | BLOCKED            |

### Backward-compat shim in `predictNHLGame()`

The legacy boolean params must remain readable but are fully replaced by the canonical state object:

```js
// Legacy (deprecated — do not read in NHL totals path after WI-0383)
// @deprecated use homeGoalieState instead
homeGoalieConfirmed = false,
awayGoalieConfirmed = false,

// New
homeGoalieState = null,  // CanonicalGoalieState | null
awayGoalieState = null,  // CanonicalGoalieState | null
```

When `homeGoalieState` is non-null, it takes precedence. When null, the model falls back to the legacy boolean shim behavior until WI-0383 removes it from the authoritative path.

## Tests Required

### Unit tests for `nhl-goalie-state.js`

```
makeCanonicalGoalieState — CONFIRMED + HIGH confidence → adjustment_trust FULL
makeCanonicalGoalieState — CONFIRMED + LOW confidence → adjustment_trust DEGRADED
makeCanonicalGoalieState — EXPECTED + any confidence → adjustment_trust DEGRADED
makeCanonicalGoalieState — UNKNOWN + any → adjustment_trust NEUTRALIZED
makeCanonicalGoalieState — CONFLICTING + any → adjustment_trust BLOCKED
makeCanonicalGoalieState — with CONFLICTING_SOURCE_EVIDENCE in evidence_flags → preserved in output
makeCanonicalGoalieState — missing required fields → throws or returns validation error
```

### Regression tests for `nhl-pace-model.js`

```
predictNHLGame — existing tests must pass unchanged (legacy param shim active)
predictNHLGame — homeGoalieState provided → overrides legacy homeGoalieConfirmed
predictNHLGame — homeGoalieState null + legacy boolean present → legacy shim fires
```

## Failure Modes & Guards

### Failure mode: type defined but not validated at construction
**Guard:** `makeCanonicalGoalieState()` validates all fields and throws on invalid `starter_state` or `adjustment_trust` enum values.

### Failure mode: legacy params still read as authoritative after this WI
**Guard:** deprecation annotation is required; WI-0383 will add the enforcement test.

### Failure mode: `evidence_flags` not carried through to model output
**Guard:** `predictNHLGame()` output must include `homeGoalieState` and `awayGoalieState` as pass-through fields so downstream consumers can inspect them.
