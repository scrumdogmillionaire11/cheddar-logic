# WI-1148: Migrate Projections Sections To /results/projections

**ID**: WI-1148
**Goal**: Move projection settlement and accuracy content out of the main results page into the dedicated `/results/projections` page, leaving `/results` as a pure betting/odds view.

**Scope**:
- web/src/app/results/page.tsx
- web/src/app/results/projections/page.tsx
- web/src/lib/results/transform-layer.ts (if projection fetch logic needs extracting)

**Out of scope**:
- Changes to any API routes
- New projection-specific filters or controls (those are WI-1149)
- Changes to the betting view sections (summary, decision tiers, ledger)

**Acceptance**:
- `/results` renders: Betting Record Summary, Betting Decision Tiers, Betting Ledger. No projection sections.
- `/results/projections` renders: Projection Settlement cards (NHL 1P, MLB F5), calibration bands, Brier scores, and accuracy metrics (MAE, bias, directional accuracy).
- The projection accuracy fetch (currently deferred via `requestIdleCallback`) becomes a primary load in the projections page — no idle deferral needed since it's the primary content.
- The projections page fetches from `/api/results/projection-settled` and `/api/results/projection-accuracy` directly.
- `/api/results` response no longer needs to include `projectionSummaries` in the betting view (it can remain in the payload but the UI ignores it).
- All existing test suites pass after migration.

- `/results/projections` shows projection settlement and accuracy sections; no betting ledger visible.
- Projection accuracy metrics (MAE, Brier score) load without delay on the projections page.
- Sport filter on `/results` still correctly scopes the betting ledger and decision tiers.
