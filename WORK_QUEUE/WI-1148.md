# WI-1148: Confidence Band Drilldown - Auditable Row List Per Band Pill

**ID**: WI-1148
**Goal**: Make each confidence band pill in the Projection Confidence Engine clickable, expanding an inline row-by-row game log (matchup, date, direction, outcome) so confidence performance can be audited play by play without leaving `/results`.

**Scope**:

- `web/src/app/results/page.tsx`
- `web/src/components/results/ProjectionResultsTable.tsx`
- `web/src/__tests__/ui-results-smoke.test.js`
- `web/src/__tests__/integration/results-behavioral-contract.test.ts`

**Out of scope**:

- API shape changes - all data needed already arrives via `/api/results/projection-settled` and `/api/results/projection-accuracy`
- Worker-side confidence normalization (covered by WI-1143 through WI-1146)
- Adding new confidence bands or changing thresholds
- Pagination or export features

**Requirements covered**:

- UI-AUDIT-01: Confidence band pills are auditable entry points to row-level evidence.
- UI-AUDIT-02: Weak-direction exclusions are transparent and reconcilable to summary counts.
- UI-AUDIT-03: Canonical confidence tiers remain primary UI contract; legacy labels are audit metadata only.
- UI-AUDIT-04: Superseded confidence summary/legacy presentation code in scope is removed, not left dormant.

**Acceptance**:

- Each confidence summary pill in the `by_confidence_band` breakdown is interactive when it has at least one eligible row.
- Clicking a pill expands an inline list of graded directional rows for that band with: date, matchup, direction label, outcome (WIN/LOSS), and confidence score.
- Expanded list is collapsible (click pill again to close).
- Rows with `weak_direction_flag = 1` are excluded from the drilldown list and excluded count is shown per band.
- If a band has 0 eligible rows, its pill is non-interactive (no pointer cursor, no expand behavior).
- HIGH/STRONG-derived entries are visually distinct from LOW/WATCH-derived entries using existing confidence badge styling.
- Canonical confidence tier rendering remains `LOW|MED|HIGH` for primary results UI; legacy labels can appear only as audit context metadata where needed.
- No regression in existing Confidence Engine summary metrics or `ProjectionResultsTable` row rendering.
- Dead/superseded confidence presentation code replaced by this drilldown is removed from scoped files (no dormant duplicate summary-row UI paths).

**truths**:

- User can click a confidence pill and see auditable row-level evidence for that band.
- User can collapse an expanded band and return to compact summary view.
- User can reconcile weak-direction exclusions from drilldown counts to summary telemetry.
- User cannot expand empty bands.

**artifacts**:

- `web/src/app/results/page.tsx` provides expandable pill + drilldown state, filtering, and exclusion count wiring.

- `web/src/__tests__/ui-results-smoke.test.js` verifies pill interactivity/collapse and non-interactive empty-band behavior.
- `web/src/__tests__/integration/results-behavioral-contract.test.ts` verifies weak-direction exclusion and canonical-tier invariants.

**key_links**:

- Confidence pill click handler -> active expanded band state -> filtered `mappedProjectionAccuracyRows` list.
- Drilldown filter predicate -> excludes `weak_direction_flag = 1` rows -> excluded count per band.
- Summary `by_confidence_band` metrics -> displayed pill labels/counts -> drilldown row reconciliation.
- Confidence drilldown additions -> no regression to `ProjectionResultsTable` canonical row tier display.
