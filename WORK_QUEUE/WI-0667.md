# WI-0667: Sharp divergence annotation ‚Äî `computeSharpDivergenceAnnotation` + `SHARP_MONEY_OPPOSITE` / `SHARP_ALIGNED` tags

**ID**: WI-0667
**Goal**: When Circa (sharp-money) handle is on the **opposite** side of our projection, tag the card `SHARP_MONEY_OPPOSITE`. When Circa handle is on the **same** side, tag it `SHARP_ALIGNED`. Both tags are purely informational ‚Äî no status demotion, ever. The double signal (blue "Sharp Divergence" pill + amber "Public Heavy" pill showing simultaneously) is the visual cue to consider the other side.

---

## Background / Spike

WI-0667 was originally written as a *gate*: use DK public bet % as a proxy for sharp money, then demote PLAY‚ÜíLEAN when public is >72%/68% on our side. That design was superseded twice:

1. **WI-0776** lands actual Circa Sports handle % (`circa_handle_pct_home/away`) ‚Äî a direct sharp-money signal. The public-% proxy is no longer needed.
2. **User correction (2026-04-04)**: We do not want automatic status demotion. The value is in *tracking* sharp divergence against our picks, not blocking them. Let the pick stand; annotate it; measure the outcome.

**Correct design:**
- `SHARP_MONEY_OPPOSITE` ‚Äî Circa handle skews >= 40 pp toward the *opposite* side from our `direction` (i.e. ~70%+ of Circa handle on the other side)
- `SHARP_ALIGNED` ‚Äî Circa handle skews >= 40 pp toward the *same* side as our `direction` (i.e. ~70%+ of Circa handle with us)
- Below threshold (< 40 pp difference): no annotation

**The double-signal:** When a card has `SHARP_MONEY_OPPOSITE` AND `publicBetsPct > 65` (DK), both pills render simultaneously ‚Äî blue "Sharp Divergence" + amber "Public Heavy (X%)". No new pipeline logic needed for the amber side; it already fires from raw `publicBetsPct` in `market-signals.ts`. The visual combination is the cue to consider the other side.

**`SHARP_ALIGNED` = new pill.** When Circa agrees with our projection: emerald "Sharp Aligned" pill. Confidence booster.

**Analytics (post-ship):** accumulate rows where `sharp_money_opposite = true` in `card_payloads`; measure win rate vs baseline to verify or challenge model edge against Circa.

## Acceptance

### Pipeline function `computeSharpDivergenceAnnotation(payload, oddsSnapshot)`

- **No-op**: when `oddsSnapshot.circa_handle_pct_home == null` OR `oddsSnapshot.circa_handle_pct_away == null` ‚Äî returns immediately, payload unchanged
- **Input resolution**: determine which Circa column is "our side" (`payload.direction === 'HOME'` uses `circa_handle_pct_home`; `'AWAY'` uses `circa_handle_pct_away`)
- **`SHARP_MONEY_OPPOSITE`**: when Circa handle on the **opposite** side exceeds our side by >= 40 pp (‚âà 70%+ on the other side) ‚Äî push `'SHARP_MONEY_OPPOSITE'` onto `payload.tags`; set `payload.sharp_money_opposite = true`
- **`SHARP_ALIGNED`**: when Circa handle on our side exceeds the opposite side by >= 40 pp (‚âà 70%+ with us) ‚Äî push `'SHARP_ALIGNED'` onto `payload.tags`; set `payload.sharp_aligned = true`
- **Below threshold** (difference < 40 pp): no tags added, payload unchanged
- **No status mutation**: `official_status` is never modified by this function
- **In-place mutation contract**: mutates `payload.tags` directly ‚Äî same object reference the caller holds. No `decision-publisher.js` changes required.
- **Wiring**: inserted in `buildDecisionV2` **after** `applyNbaTotalQuarantine` and **before** `resolvePrimaryReason`
- **Context**: `oddsSnapshot` from `context.oddsSnapshot`, which already flows into `buildDecisionV2` from model job call sites

### Threshold constant

```js
// At top of decision-pipeline-v2.js, alongside PLAY_EDGE_MIN / LEAN_EDGE_MIN
// 40 pp = ~70% on one side (circa_home + circa_away ‚âà 100 for spread/ML)
const SHARP_CIRCA_DIVERGENCE_THRESHOLD = 40;
```

const hasSharpDivergence =
  primaryReasonCode === 'PASS_SHARP_MONEY_OPPOSITE' || // legacy dead code ‚Äî preserved for old cards

## Analytics queries (post-ship)

```sql
-- Win rate on picks where Circa was against us
SELECT
  COUNT(*)                                                                        AS total_picks,
  SUM(CASE WHEN outcome = 'WIN' THEN 1 ELSE 0 END)                               AS wins,
  ROUND(100.0 * SUM(CASE WHEN outcome = 'WIN' THEN 1 ELSE 0 END) / COUNT(*), 1)  AS win_pct
FROM card_payloads
WHERE json_extract(payload, '$.sharp_money_opposite') = 1
  AND outcome IS NOT NULL;

-- Win rate when Circa aligned with our pick
SELECT
  COUNT(*)                                                                        AS total_picks,
  SUM(CASE WHEN outcome = 'WIN' THEN 1 ELSE 0 END)                               AS wins,
  ROUND(100.0 * SUM(CASE WHEN outcome = 'WIN' THEN 1 ELSE 0 END) / COUNT(*), 1)  AS win_pct
FROM card_payloads
WHERE json_extract(payload, '$.sharp_aligned') = 1
  AND outcome IS NOT NULL;
```
