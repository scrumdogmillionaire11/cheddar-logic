FPL Transfer Recommendation Engine

Production-ready decision framework — v1.0 spec

Must-Have Factors
Expected Points (xPts)
Probability-weighted mean points over target horizon
MUST-HAVE
Formula
xPts = Σ(P(start) × E[pts|started]) per GW
Source
FPL API + Opta/Understat fixture xG/xA
Data Risk
FPL API lags; injury news not real-time
Fallback
Use 90-day rolling avg pts if current-form blank
Value (xPts per £m)
Efficiency ratio — core to value-first ranking
MUST-HAVE
Formula
Value = xPts_3GW / price_now
Source
FPL API price + computed xPts
Data Risk
Price changes mid-GW invalidate ratio
Fallback
Recalc at deadline; cache with TTL 1hr
Fixture Difficulty Rating (FDR)
Opponent strength adjusted for H/A, home record, form
MUST-HAVE
Formula
FDR_adj = base_FDR × H/A_factor × opp_form_decay
Source
FPL official FDR + derived H/A split stats
Data Risk
FPL FDR is notoriously lagged (seasonal reset)
Fallback
Use xG against / xGA last 5 as substitute
Minutes / Start Probability
Probability of 60+ min start; rotation risk score
MUST-HAVE
Formula
P(start) = sigmoid(minutes_pct × injury_decay × news_score)
Source
FPL API minutes + press conf NLP + Twitter
Data Risk
Manager news often drops <2hrs before deadline
Fallback
Apply continuous risk penalty; hard-exclude only explicit unavailable status
Blank / Double Gameweek Flag
Boolean + magnitude (0 GW, 1 GW, 2 GW) per player
MUST-HAVE
Formula
DGW multiplier = 1.8×; BGW = hard exclusion
Source
FPL fixtures API + manual calendar
Data Risk
Fixture reschedules & cup replays cause errors
Fallback
Alert + flag for manual review; do not auto-exclude
Transfer Cost / Hit Economics
Net EV vs. cost of -4pt hit; opportunity cost of FT banking
MUST-HAVE
Formula
Hit ROI = (xPts_in − xPts_out − 4) / risk_sd
Source
Manager's current squad state + engine xPts
Data Risk
No fallback needed — computed internally
Fallback
Recommend hit only if ROI > 1.5σ above threshold

Team Attack / Defense Strength
Season-level xG / xGA per 90, rolling last 8 GW
NICE-TO-HAVE
Use case
Spot teams in form vs. teams on decline
Source
FBref / Understat team stats
Fallback

Variance / Ceiling Profile
Std dev of pts distribution; haul probability (%)
NICE-TO-HAVE
Use case
Risk-mode selection; captain variance vs. floor picks
Source
Computed from historical pts distribution
Fallback
Use position-level median SD as default
Strategy Mode
Master Scoring Formula
TransferScore(p) = w1 × norm(xPts_horizon(p)) // Expected points + w2 × norm(value(p)) // Points per £m + w3 × norm(fix_quality(p)) // Fixture score + w4 × P(start, p) // Minutes security + w5 × norm(set_piece_bonus(p)) // Role context − w6 × norm(ownership_risk(p)) // Differential penalty + w7 × norm(DGW_multiplier(p)) // Double GW premium − hit_cost_adjusted(n_hits) // Transfer hit penalty
Form Decay — Anti-Recency Bias
// Exponential decay over last N GWs form_score(p) = Σ_{i=1}^{N} pts[i] × λ^(N-i) λ = 0.75 // decay rate (tune per backtest) N = 10 // window (not 5 — avoids noise) // Blend with season baseline xPts_adj(p) = α × form_score(p) + (1−α) × season_avg(p) α = 0.45 // season avg still carries 55% weight
Value-First Candidate Ranking
// Step 1: Hard filter (see Constraints panel) candidates = filter_hard_constraints(all_players) // Step 2: Compute composite EV score EV(p) = xPts_3GW(p) EV_price(p) = EV(p) / price(p) Ceiling(p) = xPts_3GW(p) + 1.5 × sd_pts(p) Floor(p) = xPts_3GW(p) − 0.75 × sd_pts(p) VAdjEV(p) = EV(p) − risk_aversion × sd_pts(p) // Step 3: Rank by strategy Conservative → sort by Floor(p) DESC Balanced → sort by VAdjEV(p) DESC Aggressive → sort by Ceiling(p) DESC Value-first → sort by EV_price(p) DESC (primary ranking)

Hard Constraints (fail = exclude candidate)
✕
Blank GW exclusion: P(playing) = 0 → auto-exclude. Do not recommend into blanks.
!
Start probability risk treatment: remove hard 75% gate. Use a continuous minutes-risk penalty and confidence badge; only hard-exclude explicit unavailable/out statuses.
✕
Budget constraint: squad_value + transfer_delta ≤ ITB + selling_price. No breach.
✕
Team limit: ≤ 3 players per club in squad post-transfer.
✕
Injury/unavailability: status = "Doubtful" or "Out" → exclude. Decay news confidence by 50% after 48hrs without update.
✕
Hit ROI < 1.0: Never recommend a hit if xPts_in − xPts_out < 4. Block the recommendation.
Soft Constraints (warn, don't block)
!
High ownership (TSB > 40%): Flag as template pick. Reduce differential bonus. Still recommend if EV warrants it.
!
Price rise risk: Flag if price rose >0.2 in last 3 days. Warn "may be priced in."
!
3+ GW difficult fixture run: Degrade FDR score but don't exclude if short-horizon xPts still strong.
!
Team without DGW: If rival candidates have DGW and target doesn't, surface the trade-off in explainability text.
!
Thin squad depth at position: Warn if bringing in a 3rd player from a club with upcoming injury risk.

Explainability Rules

Every recommendation must surface 2–4 human-readable rationale sentences. Template engine below maps score components to output text.

Transfer In Rationale Template
Show: primary driver (highest weighted factor), fixture context, value anchor, and one risk caveat.
"{player} is your strongest {horizon}-GW pick at {position}. [IF top driver = value] → "At £{price}m, they offer {ev_price:.1f} pts/£m — {X}% above position average." [IF top driver = fixture] → "They face {opp} ({FDR_adj} FDR) {H/A}, one of their easiest runs in {N} GWs." [IF DGW] → "A double gameweek in GW{X} adds ~{dgw_bonus:.1f} expected pts to their projection." [IF set piece] → "As confirmed pen taker, their ceiling rises to ~{ceiling:.0f} pts over {horizon} GWs." [RISK caveat always] → "Risk: {injury_note / rotation_note / ownership_note}."
Transfer Out Rationale Template
Always justify why we are selling — not just why the target is better. Avoids "they look good" recommendations without context.
"{outgoing} ranks {Nth} in your squad by {horizon}-GW xPts. [IF blank] → "They blank in GW{X}, bringing their {horizon}-GW projection down to {xPts:.1f}." [IF form drop] → "xPts has declined {pct:.0f}% over the last {N} GWs vs. season average." [IF fixture] → "A run of {FDR} fixtures makes them a low-value hold right now." [IF price] → "Selling now recovers £{sell_price:.1f}m — {delta:.1f}m above target cost."
Hit Recommendation Template
Hits must clear a higher bar. Rationale must show the net EV math explicitly.
"Taking a −4pt hit for {player_in} over {player_out}: xPts gain: +{delta_xpts:.1f} over {horizon} GWs Hit cost: −4 pts Net gain: +{net:.1f} pts ({confidence} confidence) Recommended: {YES if net > 4 and ROI > 1.5 else NO}"
Anti-Pattern Warning Labels
!
Recency trap: "Note: {player}'s last-3-GW form is above their season average — score is partially adjusted for regression."
!
Template herd: "This is a high-ownership pick ({TSB}%). Consider {alt_player} as a lower-ownership alternative with similar xPts."
✕
BGW trap: "⚠ {player} blanks in GW{X}. They have been excluded from this horizon's ranking."

Failure Modes / Anti-Patterns
🪤
Recency bias trap
Overweighting last 2–3 GW hauls. Fix: λ-decay + season baseline anchor (55% weight)
🐑
Template herd
Always recommending the most-owned player. Fix: ownership penalty in scoring + surface differentials
📉
Start-prob over-filter
Hard thresholds hide rotating value picks. Fix: replace hard gate with continuous minutes-risk penalty + confidence label
📅
Blank tunnel vision
Excluding blank-week players from 3/5 GW view. Fix: exclude from GW score only; include in horizon total
💸
Hit-happy engine
Recommending hits where xPts delta barely clears 4. Fix: require ROI > 1.5 with confidence interval
🔢
Overfit to last season
Weights tuned on 2023 over-index that season's patterns. Fix: rolling 3-year backtest; retune each August
