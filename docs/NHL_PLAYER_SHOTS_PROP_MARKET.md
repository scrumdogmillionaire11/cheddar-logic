# NHL Player Shots On Goal — Prop Market Process

## Overview

The NHL player shots on goal (SOG) prop market pipeline produces PROP cards for upcoming NHL games. Cards are generated by comparing a statistical model projection (mu) against real O/U market lines sourced from The Odds API.

## Data Flow

```
NHL API (api-web.nhle.com)
  └── pull_nhl_player_shots.js        — Fetches L5 game logs per player
        └── player_shot_logs table     — Stores shots, TOI, opponent per game

The Odds API (api.the-odds-api.com)
  └── pull_nhl_player_shots_props.js  — Fetches player SOG O/U lines
        └── player_prop_lines table    — Stores real market lines per player+game

run_nhl_player_shots_model.js
  ├── Reads: player_shot_logs (L5 data)
  ├── Reads: games (upcoming NHL schedule)
  ├── Reads: player_prop_lines (real O/U lines)
  ├── Runs: nhl-player-shots model (calcMu, calcMu1p, classifyEdge)
  └── Writes: card_payloads (PROP cards for HOT/WATCH edges)
```

## Job Execution Order

Run in this order before each NHL card cycle:

1. `npm --prefix apps/worker run job:pull-schedule-nhl` — refresh game schedule
2. `npm --prefix apps/worker run job:pull-nhl-player-shots` — fetch L5 player logs
3. `npm --prefix apps/worker run job:pull-nhl-player-shots-props` — fetch real prop lines
4. `npm --prefix apps/worker run job:run-nhl-player-shots-model` — generate cards

## Environment Variables

| Variable | Required | Default | Purpose |
|---|---|---|---|
| `ODDS_API_KEY` | Yes | — | The Odds API key |
| `NHL_SOG_PLAYER_IDS` | Yes | — | Comma-separated player IDs to track |
| `NHL_SOG_PROP_EVENTS_ENABLED` | No | `false` | Must be `true` to enable prop line fetching |
| `NHL_SOG_SLEEP_MS` | No | `500` | Sleep between player log fetches (rate limit guard) |
| `NHL_SOG_PROP_SLEEP_MS` | No | `1000` | Sleep between event prop fetches (rate limit guard) |
| `NHL_SOG_FETCH_RETRIES` | No | `4` | Retry count for NHL API fetch failures |

## Token Cost

The Odds API charges 1 token per event per market per fetch.

- `pull_nhl_player_shots_props` calls 1 event fetch per game in the 36h window
- Typical NHL slate: 5-8 games/day = 5-8 tokens/run
- If run 2x/day: ~10-16 tokens/day, ~300-500 tokens/month for this market

Enable via `NHL_SOG_PROP_EVENTS_ENABLED=true` only when ready to consume quota.

## Market Line Resolution

Lines are resolved from `player_prop_lines` with bookmaker priority:
1. DraftKings
2. FanDuel
3. BetMGM
4. Any available

If no real line exists (job not run, player not listed, API no-data), the model runner falls back to a **synthetic line** (mu ± random offset, rounded to nearest 0.5). Cards created with synthetic lines include `market_line_source: "synthetic_fallback"` in the `decision` payload block.

## Model

The SOG model (`apps/worker/src/models/nhl-player-shots.js`) is a JS port of `cheddar-nhl/src/nhl_sog/engine/mu.py`.

**Inputs:** L5 SOG values (most recent first), shotsPer60, projToi, opponentFactor, paceFactor, isHome

**Outputs:**
- `mu` — expected SOG for full game
- `mu1p` — expected SOG for 1st period (~32% of full-game mu)

**Edge classification thresholds (backtest-calibrated Feb 2026):**
- HOT: |edge| >= 0.8 and confidence >= 0.50
- WATCH: |edge| >= 0.5 and confidence >= 0.50
- COLD: everything else

Cards are only created for HOT or WATCH tiers.

## Card Types

| card_type | period | Description |
|---|---|---|
| `nhl-player-shots` | full_game | Full game SOG O/U |
| `nhl-player-shots-1p` | first_period | First period SOG O/U |

## Prop Line DB Schema

```sql
player_prop_lines (
  id TEXT PRIMARY KEY,
  sport TEXT NOT NULL,
  game_id TEXT NOT NULL,           -- FK to games.game_id
  odds_event_id TEXT,              -- The Odds API event ID
  player_name TEXT NOT NULL,
  prop_type TEXT NOT NULL,         -- 'shots_on_goal'
  period TEXT NOT NULL,            -- 'full_game' | 'first_period'
  line REAL NOT NULL,              -- O/U line (e.g., 3.5)
  over_price INTEGER,              -- American odds (e.g., -115)
  under_price INTEGER,
  bookmaker TEXT,
  fetched_at TEXT NOT NULL
)
```

## Known Limitations / Future Work

- **1P prop lines:** The Odds API does not consistently offer `player_shots_on_goal_1p` lines. First-period cards use synthetic fallback by default.
- **Player name matching:** Real line lookup is case-insensitive by `player_name`. If The Odds API uses a different name format than the NHL API pull, lines may not match — monitor `market_line_source: "synthetic_fallback"` in card payloads as a signal.
- **Team abbrev expansion:** `TEAM_ABBREV_TO_NAME` in the model runner is incomplete. Add NHL team abbreviations as needed.
- **opponentFactor / paceFactor:** Both hardcoded to 1.0. Future enrichment should source these from team metrics.
