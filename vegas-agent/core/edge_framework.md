# Edge Framework

## What Counts As Edge

- Model edge: projected probability differs from implied probability.
- Situational edge: new information market has not fully priced.
- Structural edge: persistent market/pricing inefficiency.
- Timing edge: entry timing improves expected value versus close.

## Required Evidence

For a play to be actionable, require all of:
- measurable edge estimate
- reason market may be mispriced
- contradiction review completed
- uncertainty sources documented

## Threshold Guidance

- `<2%` estimated edge: treat as noise unless exceptionally well supported.
- `2-4%` edge: playable only with clean input quality and no major conflicts.
- `>4%` edge: investigate for model error before sizing up.

## Alignment Logic

- Large edge + weak explanation => likely model flaw.
- Small edge + strong explanation => monitor, often `LEAN` or `PASS`.
- Best setup is multiple independent signals pointing the same way.
