# CHANGES

Optional scratch notes and patch logs per work item.

## Convention
- One optional file per work item: `CHANGES/WI-####.md`
- Keep entries concise and factual:
  - Scope
  - Files changed
  - Validation commands
  - Risks or follow-ups

`WORK_QUEUE/` remains the authoritative planning and scope source. `CHANGES/` is implementation telemetry only.
